APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Currently the SNMP DSA configuration only interacts with the
   SNMP API to alter the timeout property value
   (defaultTimeoutMsec) but this does not take into account
   another API property of retry count (defaultRetries (which is
   3)).
   
   The timeout is the time between the first failure and the first
   retry with each subsequent retry timeout being double the
   previous - for example a 1 second timeout value results an
   actual timeout value of 15 seconds:
   
   1s + 2s + 4s + 8s = 15s
   
   and as 15 second DSA timeout setting translates
   into 3 minute 45 seconds before timeout actually occurs:
   
   15s + 30s + 60s + 120s = 225s = 3m 45s
   
   The result is that all values of timeout are then multiplied by
   15 through the application of the retry count property in the
   API.
   
   The SNMP DSA must leverage the SNMP API defaultRetries property
   and the interaction of these configuration settings needs to be
   appropriately documented too.
   
   
    
   
   

LOCAL FIX
 *  There is currently no option but to apply timeout values that
   result in actual timeouts being multiples of 15 thereof.
