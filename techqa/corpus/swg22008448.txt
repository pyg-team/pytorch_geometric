PROBLEM(ABSTRACT)
 When the IBM Transformation Extender Advanced (ITXA) / IBM Standards Processing Engine (SPE) support team needs to review issues related to the SPE Transform Service the following information the following information should be sent to the support group with your case. 

RESOLVING THE PROBLEM

There are several sets of information that is needed by the ITXA Support group to investigate SPE Transform Service issues. 


 * Detailed problem description [http://www-01.ibm.com/support/docview.wss?uid=swg22007234] 
 * ITXA Version information [http://www-01.ibm.com/support/docview.wss?uid=swg22007234] 
 * ITXA debug / troubleshooting logs [http://www-01.ibm.com/support/docview.wss?uid=swg22008337] 
 * Input data 
 * Output data observed 
 * Expected output data 
 * How is the Transform process being invoked (IBM Transformation Extender, Sterling B2B Integrator, IBM Information Bus, Java code, XSLT)?
    * IBM Transformation Extender  * Map source (mms files) and type trees (mtt files) for all maps involved, including run maps 
       * If the ITX map is invoking the SPE adapter [https://www.ibm.com/support/knowledgecenter/SSVSD8_9.0.0/com.ibm.websphere.dtx.adapspe.doc/concepts/c_SPE_Adapter.htm], provide the artifacts that the SPE adapter needs to function (i.e: Envelopes, Code lists, Schemas, etc.) along with ITXA - IBM Transformation Extender SPE Adapter logs [http://www-01.ibm.com/support/docview.wss?uid=swg22008337#SPEAdapter] 
       * Can the ITX invoked transform map be successfully executed from the ITX Design Studio and/or Command Server? If no, open a case with the IBM Transformation Extender support team providing the ITX team with the above information. If yes, collect the information listed above and open a case with the ITXA support team 
       * Enable the ITX Map Audit log [https://www.ibm.com/support/knowledgecenter/en/SSVSD8_9.0.0/com.ibm.websphere.dtx.md.doc/concepts/c_map_design_MapAudit_Settings.htm] and provide the resulting log 
       * ITX Version Information [http://www-1.ibm.com/support/docview.wss?rs=2320&context=SSVSD8&q1=versioning&uid=swg21243594]
      
      
    * Sterling B2B Integrator  * The Business Process being invoked, simplified if possible and preferably in bpml format 
       * ITXA - Sterling B2Bi Integrator logs [http://www-01.ibm.com/support/docview.wss?uid=swg22008337#B2Bi] 
       * If the Transform Service invokes an ITX map, provide the information listed above for IBM Transformation Extender 
       * If the Transform Service invokes a Sterling map, provide the compiled Sterling (txo) map 
       * Dump Info [http://www-01.ibm.com/support/docview.wss?uid=swg21562125] results
      
      
    * IBM Information Bus  * The Message Flow being invoked, simplified if possible 
       * ITXA - IBM Information Bus logs [http://www-01.ibm.com/support/docview.wss?uid=swg22008337#IIB] 
       * The information listed above for IBM Transformation Extender 
       * IBM Information Bus version: mqsiservice -v 
      
      
    * XSLT  * The XSLT map
      
      
    * Java code  * Java code used to invoke ITXA 
       * Version of java: java - version
