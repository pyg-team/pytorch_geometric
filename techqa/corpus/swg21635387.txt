Title: IBM Error "java.lang.NumberFormatException: null" seen when sending message to IBM Initiate web service - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When sending a properly formatted message to the IBM Initiate web service you get a return message with "<faultstring>java.lang.NumberFormatException: null</faultstring>". 

SYMPTOM
Here is an example of a full message response received when sending a message to the IBM Initiate web wervice:
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/ [http://schemas.xmlsoap.org/soap/envelope/]" xmlns:xsd="http://www.w3.org/2001/XMLSchema [http://www.w3.org/2001/XMLSchema]" xmlns: xsi="http://www.w3.org/2001/XMLSchema-instance [http://www.w3.org/2001/XMLSchema-instance]">
<soapenv:Body>
<soapenv:Fault>
<faultcode>soapenv:Server.userException</faultcode>
<faultstring>java.lang.NumberFormatException: null</faultstring>
<detail>
<ns1:hostname xmlns:ns1="http://xml.apache.org/axis/ [http://xml.apache.org/axis/]">localhost</ns1:hostname>
</detail>
</soapenv:Fault>
</soapenv:Body>
</soapenv:Envelope>


CAUSE
When setting up the IBM Initiate web services SDK on Apache Tomcat some versions of Apache Tomcat do not define the shared folders for web applications.

RESOLVING THE PROBLEM
If you are seeing this error and your IBM Initiate web services SDK is running on Apache Tomcat you will need to modify the catalina.properties file. 

The catalina.properties file is found in your ${catalina.home}/conf directory.

You will need to modify the line shared.loader in the catalina.properties file. It should look like the following when you are done:
shared.loader=${catalina.home}/shared/classes,${catalina.home}/shared/lib/*.jar
