   ClassNotFoundException is logged for the same class on which ORB

LOCAL FIX
 *  If possible avoid the meta request by ensuring that both the
   server and the client have the same version of class.
   
   
    
   
   

PROBLEM SUMMARY
 *  The problem is caused because meta request was being sent to
   Z/WAS control region instead of the servant region.
   
   
    
   
   

PROBLEM CONCLUSION
 *  ORB has been updated to ensure that meta requests are sent to
   the servant region in Z/WAS.
