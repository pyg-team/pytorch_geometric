Title: IBM Installation fails with error: User does not have root authority - United States

Text:
DEL2INST; install fails ACUINI0061I; user authority TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Installation of Deployment Engine fails with below errors:
com.ibm.ac.si.install.InstallUnauthorizedException: ACUINI0061I User, 1054 does not have root authority!
at com.ibm.ac.si.install.SIInstaller.validateUnixUserAuthority(SIInstaller.java:1997) 

SYMPTOM
You may see below errors in DEInstall.log and/or de_trace.log when installing an application. These error suggest that DE was attempting to install as root, although user could be a non-root user:


com.ibm.ac.si.install.InstallUnauthorizedException: ACUINI0061I User, 1054 does not have root authority!
at com.ibm.ac.si.install.SIInstaller.validateUnixUserAuthority(SIInstaller.java:1997)
at com.ibm.ac.si.install.SIInstaller.validateUserAuthority(SIInstaller.java:1921)
at com.ibm.ac.si.install.SIInstaller.setInstallMode(SIInstaller.java:6023)
at com.ibm.ac.si.install.SIInstaller.initialize(SIInstaller.java:2343)
at com.ibm.ac.si.install.SIInstaller.<init>(SIInstaller.java:2219)
at com.ibm.ac.si.install.SIInstaller.<init>(SIInstaller.java:2187)
at com.ibm.ac.si.install.SIInstaller.<init>(SIInstaller.java:2175)
at com.ibm.ac.si.install.SIInstaller.getInstalledSIClasspath(SIInstaller.java:1410)
at com.ibm.ac.coi.ext.ia.DEBootstrapInstall.install(DEBootstrapInstall.java:319)
at com.ibm.ac.coi.ext.ia.DEBootstrapWorkerThread.run(DEBootstrapWorkerThread.java:253)


CAUSE
This error can be a result of an improperly created non-root user account on Solaris platform. If a non-root user is added using simplified command syntax (eg; useradd -m testuser), no directory will be created in /home or /export/home on Solaris. But Solaris will set the $HOME variable as /home/<username> (<username> refers to the id of the non-root user being created). 

Solaris will regard /export/home/<username> as user's home directory. Since this directory does not exist so DE may fail to install the application.


RESOLVING THE PROBLEM
1. Installation being performed as non-root user:


If user's home directory does not exist and installation is being performed as a non-root user then create user's home directory and set HOME variable to that directory. Then perform the installation again. But if user's home directory exist and you still see this error then this error can be ignored.

2. Installation being performed as a root user:
If the application is being installed as root user and you see above errors then it should be investigated. Please open a PMR for assistance from IBM.
