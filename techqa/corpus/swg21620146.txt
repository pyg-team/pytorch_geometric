Title: IBM Relocate Application Engine/Workplace/WorkplaceXT shared configuration directory - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 If the Application Engine/Workplace/WorkplaceXT's shared configuration directory needs to be changed post-installation to a new location, follow the procedure outlined. 

CAUSE
If the configuration directory should need to change location for any reason.

RESOLVING THE PROBLEM
In order to change the configuration directory, perform the following procedure 

 * Back up any files before editing.

 * In your installed version of AE/Workplace/WorkplaceXT, change the "configurationDirectory" parameter of the web.xml file in one of the following paths appropriate to your installation to point to the new configuration directory location.

 * Windows: 
   FileNet\AE\Workplace\WEB-INF\web.xml 
   IBM\FileNet\AE\Workplace\WEB-INF\web.xml 
   IBM\FileNet\WebClient\WorkplaceXT\WEB-INF\web.xml 
   
   Linux/UNIX: 
   FileNet/AE/Workplace/WEB-INF/web.xml 
   IBM/FileNet/AE/Workplace/WEB-INF/web.xml 
   IBM/FileNet/WebClient/WorkplaceXT/WEB-INF/web.xml 
   
   Web.xml example for AE/Workplace on Windows: 
   <context-param> 
   <param-name>configurationDirectory</param-name> 
   <param-value>C:\\Program Files\\FileNet\\Config\\AE</param-value> 
   </context-param> 
   
   Web.xml example for AE/Workplace on Linux/UNIX: 
   <context-param> 
   <param-name>configurationDirectory</param-name> 
   <param-value>//opt//IBM//FileNet//Config//AE</param-value> 
   </context-param> 
   
   Web.xml example for WorkplaceXT on Windows: 
   <context-param> 
   <param-name>configurationDirectory</param-name> 
   <param-value>C:\\Program Files\\FileNet\\Config\\WebClient</param-value> 
   </context-param>

 * Copy the contents of the either the Config\AE or Config\WebClient directory from the local installation to the new shared location referenced in Step Two. 
 * Rebuild the ear/war file via one of the following commands in one of the following directory examples:

 * Rebuild the AE/Workplace ear/war with: 
   
   Windows: 
   create_app_engine_ear.bat 
   create_app_engine_war.bat 
   
   Linux/UNIX: 
   create_app_engine_ear.sh 
   create_app_engine.war.sh 
   
   Rebuild the WorkplaceXT ear/war with: 
   
   Windows: 
   create_web_client_ear.bat 
   create_web_client_war.bat 
   
   Linux/UNIX: 
   create_web_client_ear.sh 
   create_web_client.war.sh 
   
   
   Locate the command in: 
   
   Windows: 
   FileNet\AE\deploy 
   IBM\FileNet\AE\deploy 
   IBM\FileNet\WebClient\deploy 
   
   Linux/UNIX: 
   FileNet/AE/deploy 
   IBM/FileNet/AE/deploy 
   IBM/FileNet/WebClient/deploy 

Once the ear/war is rebuilt, then redeploy it to your application server.
