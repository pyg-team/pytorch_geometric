Title: IBM Calendar entries show the wrong time. - United States

Text:
 TECHNOTE (FAQ)

QUESTION
The end user sees calendar entries are off by one or several hours in the past or future. Within the Basic Notes Client all display correctly.

ANSWER
This behavior can occur on Windows 7, but mainly on Windows Terminal Server, Citrix XenApp Server, Citrix XenDesktop because of the multi-user OS which allows access from multiple ICA clients located across various countries and time zones. 
This can be a third-party software bug. If you experience this issue, download and apply Citrix hotfix PSE450R05W2K3002.
Another Citrix hotfix PSE450R06W2K3025 [#231478] is required to be installed and the registry key needs to be populate with the executable i.e. notes2.exe.
Please check whether the Microsoft "August 2011 cumulative time zone update for Windows operating systems" http://support.microsoft.com/kb/2570791 [http://support.microsoft.com/kb/2570791] got implemented.

Note: This does not apply for Windows XP or Windows 2003 Servers because they don't have tzres.dll in C:\Windows system path.

Notes parameters are set according to the desired timezone:
Timezone=
DSTLAW=
DST=

The Notes Client must gather (please see TN 1424260 [http://www.ibm.com/support/docview.wss?uid=swg21424260]) task reports within systemsummary.txt: 
TimeZone Details 
================ICU4J Info================= 
ICU4J TZ Data Version is 2010j 
XPD ICU4J Data Update Patch version is 3.8.1.v20100526_2010j 
Current TZ is: GMT+0100, offset: 1, In DST=false, Time in this 
Timezone 17:58:1

Troubleshooting:
Customer exported the two registry keys:

Note: Romance Standard Time is just a sample, please use your own timezone name and settings.

[HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Time Zones\Romance Standard Time]
"MUI_Display"="@tzres.dll,-300"
"MUI_Dlt"="@tzres.dll,-301"
"MUI_Std"="@tzres.dll,-302"
"Display"="(UTC+01:00) Brussels, Copenhagen, Madrid, Paris"
"Dlt"="Romance Daylight Time"
"Std"="Romance Standard Time"
"TZI"=hex:c4,ff,ff,ff,00,00,00,00,c4,ff,ff,ff,00,00,0a,00,00,00,05,00,03,00,00,\
00,00,00,00,00,00,00,03,00,00,00,05,00,02,00,00,00,00,00,00,00

and

[HKLM\SYSTEM\CurrentControlSet\Control\TimeZoneInformation]
"Bias"=dword:ffffffc4
"DaylightBias"=dword:ffffffc4
"DaylightName"="Romance Daylight Time"
"DaylightStart"=hex:00,00,03,00,05,00,02,00,00,00,00,00,00,00,00,00
"StandardBias"=dword:00000000
"StandardName"="Romance Standard Time"
"StandardStart"=hex:00,00,0a,00,05,00,03,00,00,00,00,00,00,00,00,00
"TimeZoneKeyName"=""
"DynamicDaylightTimeDisabled"=dword:00000001
"ActiveTimeBias"=dword:ffffffc4

Note:
"TimeZoneKeyName"="" is empty!

Solution:

Following keys should match: 

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Time Zones] [HKLM\SYSTEM\CurrentControlSet\Control\TimeZoneInformation] Key: [Romance Standard Time]
[HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Time Zones\Romance Standard Time] "TimeZoneKeyName"="Romance Standard Time" 
(the English expression as it apprears as key under \Times Zones "MUI_Dlt"="@tzres.dll,-301" "DaylightName"="@tzres.dll,-301" "MUI_Std"="@tzres.dll,-302" "StandardName"="@tzres.dll,-302" 
Please modify and adjust the registry accordingly.
