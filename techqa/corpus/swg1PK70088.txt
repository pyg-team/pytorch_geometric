APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Environment: Microsoft Windows2003 Server
   WebSphere Application Server Network Deployment 6.1.0.9
   
   Problem: The "Pretest existing pooled connections" value is
   changing back to the default when another field is changed.
   -
   Problem Details:
   
   This is the panel where the setting is modified:
   
   Data sources > Default Datasource > WebSphere Application
   Server data source
   
   First, check the "Pretest existing pooled connections" and save
   the configuration. Then, bring up the panel again and change
   the "Minimum connections", and then bring up the panel again
   and see that Pretest existing pooled connections" is
   unchecked. The resource.xml file now shows
   testConnection="false" again.
   
   
    
   
   

LOCAL FIX
