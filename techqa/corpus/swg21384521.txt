Title: IBM Getting DPR-ERR-2079  when hit import or dispatcher link in Cognos Administration - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Customer is getting a "DPR-ERR-2079 Firewall security 
rejection. Your request was rejected by the firewall"
when they are trying to go to dispatcher or the import content link 

SYMPTOM
"DPR-ERR-2079 Firewall security 
rejection. Your request was rejected by the firewall"

CAUSE
Customer is having both 8.3 and 8.4 running on the same system. What they have did is moved the cogstartup.xml from their 8.3 to 8.4. Both system is running at the same time. We have forgot to take out the cookies system under Cognos configuration -> Actions -> Edit Global Configuration -> General. once this has been change, issue disappeared.

RESOLVING THE PROBLEM
Reset cookie settings to default: Cognos configuration -> Actions -> Edit Global Configuration -> General. 

We can not use the reset to default button.

By default the value of Domain and Path is empty, Secure flag enabled is set to false.
