Title: IBM IIB Execution Group memory growth with FTM of 1 to 2 GB per week - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Memory growth in the IBM Integration Broker (IIB) Execution Groups for the Financial Transaction Manager (FTM) product can creep at a rate of 1 to 2 GB per week, attributed to IBM WebSphere Transformation Extender (WTX) map nodes. 

SYMPTOM
Continual memory rise, only released when the Execution Group (EG) is stopped.


CAUSE
This has been identified as a product defect under APAR PI53922.

ENVIRONMENT
Financial Transaction Manager (FTM) running under IIB / WTX.

DIAGNOSING THE PROBLEM
Standard Unix tools used to monitor memory usage of the IIB processes.

RESOLVING THE PROBLEM
Obtain and apply the fix for APAR PI53922.
