Title: IBM TIP has slow performance or shuts down due to too many open files error - United States

Text:
TIPL2; TIPL2CONF slow performance; too many open files TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Users may experience slow performance from Tivoli Integrated Portal if the limit on the number of files that can be opened simultaneously is set too low. 

SYMPTOM
The SystemOut.log file will shows below error messages:

[7/18/12 10:18:23:291 GMT+05:30] 00000038 DCIConfigurat W com.ibm.tivoli.nam.config.DCIConfigurationManager.createConfigurationProperties() IOException: Too many open files
[7/18/12 10:18:25:861 GMT+05:30] 00000029 Ffdc I com.ibm.ffdc.impl.Ffdc setFailsafeProvider FFDC1000I: FFDC provider com.ibm.ws.ffdc.impl.FfdcProvider@159a159a aborted, Exception follows
[7/18/12 10:18:25:861 GMT+05:30] 00000029 Ffdc I com.ibm.ffdc.impl.Ffdc setFailsafeProvider FFDC1001I: FFDC provider aborted with exception {0}
java.security.PrivilegedActionException: java.io.IOException: Too many open files255)


DIAGNOSING THE PROBLEM
If SystemOut.log file is showing above errors then run below command to find out what is the System limit on number of open files. 

[root@user] /# ulimit -a


core file size (blocks) 1000000
data seg size (kbytes) unlimited
file size (blocks) unlimited
max memory size (kbytes) unlimited
stack size (kbytes) 8192
cpu time (seconds) unlimited
max user processes unlimited (1)
pipe size (512 bytes) 8
open files 1000
virtual memory (kbytes) 2105343


RESOLVING THE PROBLEM
If you see above errors in the log and the limit on "Open files" is low then change it to at least 8000. 

Restart TIP for this change to take effect.
