Title: IBM 0C4 ABEND when processing a SINK file in a run map with ITX Batch Command Server on z/OS - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When using the IBM Transformation Extender (ITX) Command Server on z/OS Batch to execute a map that in turn uses the RUN() function to call a sub map which has a SINK adapter, an 0C4 ABEND may occur. 

SYMPTOM
CEE3204S The system detected a protection exception (System Completion Code=0C4).
From compile unit WINMVS7:/scm/builds/9.0.0.1/source/64/os390/mvs/mvs.c
at entry point addMercfile
at statement 2782
at compile unit offset +000000001A3DA8F8
at entry offset +0000000000000480
at address 000000001A3DA8F8.


CAUSE
This has been identified as a product defect under APAR PI78604.

ENVIRONMENT
ITX Command Server on z/OS

DIAGNOSING THE PROBLEM
A standard map (not calling a RUN function) does not crash.

RESOLVING THE PROBLEM
Obtain and apply fix for APAR PI78604.
