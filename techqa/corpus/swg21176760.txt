Title: IBM Error: Packet contains no data for creating a replicated schema repository - United States

Text:
1176760; CQ; clearquest; CQMS; multisite; mkreplica; multiutil; replica; import; packet; clan; Packet contains no data for creating a replicated schemarepository TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 This technote explains how to resolve issues when replicating a new user database in an existing IBM速 Rational速 ClearQuest速 MultiSite速 environment. The mkreplica -import command throws an error when creating a new user database that is in the same CLAN (replicated Schema Repository) as an already existing replicated Schema Repository. 

CAUSE
Schema Repository replica-creation packets are not generated when replicating a new user database in the same CLAN. The following scenario will help us to understand the problem further: 

 1. Has a schema repository with 2 user databases, PROD & SAMPL. 
 2. Create 2 sites LEX and CUP with family PROD. 
 3. Now run mkreplica -export from LEX and the family is SAMPL. 
 4. Try to import the exported file on CUP site. The following commands generated the error message below:
    
    multiutil mkreplica -export -clan RATL -site LEX -family SAMPL -u admin -p cqadmin852 -fship -workdir d:\temp\cqms cqcup:CUP
    
    multiutil mkreplica -import -site CUP -repository CUPSQL -vendor ORACLE -dbologin dbom pass -connectopts 

 * HOST=XXXX;SID=YYYY;SERVER_VER=8.1;CLIENT_VER=8.1;LOB_TYPE=LONG -data CUPSQL -vendor ORACLE -dbologin dbou pass -connectopts HOST=XXXX;SID=YYYY;SERVER_VER=8.1;CLIENT_VER=8.1;LOB_TYPE=LONG "D\ProgramFiles\Rational\ClearCase\var\shipping\ms_ship\incoming\mk_CUP_19-March-04_11-17-50.xml"
   
   Multiutil: Packet contains no data for creating a replicated schemarepository.
   

RESOLVING THE PROBLEM
To add a new user database replica Family to an existing Clan, create a vendor database for the user database replica. 

 
Note: Instead of entering the database parameters for the a new replicated schema repository, use the CLAN name and indicate the following login information:


multiutil mkreplica -import -clan RATL -site CUP -data CUPSQL -vendor ORACLE -dbologin dbou pass -connectopts HOST=XXXX;SID=YYYY;SERVER_VER=8.1;CLIENT_VER=8.1;LOB_TYPE=LONG "D: \ProgramFiles\Rational\ClearCase\var\shipping\ms_ship\incoming\mk_CUP_19-March-04_11-17-50.xml
