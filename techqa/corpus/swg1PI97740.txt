APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When using the HANDLEIN() function to call more than 64K RUN
   map occurrences, one of the following problems may occur.
   
   1) A Crash
   
   2) Disk write error - return code 16
   
   3) Could not open work files - return code 36 (meaning - A
   scratch file for the HANDLEIN function could not be created.)
   
   The problem does not occur is the HANDLEIN() function is
   replaced with the ECHOIN() function.
   
   
    
   
   

LOCAL FIX
 *  ITXCQ - ITX00058845
   GM / GM
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   IBM Transformation Extender customers who calls HandleIn
   function over 64k times
   
   Problem Description:
   Map produces undefined results, crashes or fails with "Disk
   Write Error" if the map calls HandleIn over 64k times.
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  Correct code to handle over 64k calls to HandleIn function from
   the same map.
   
   Applies to:
   9.0.0.3
