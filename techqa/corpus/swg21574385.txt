Title: IBM Instructions on how to setup the assumeAutoFs to remote - United States

Text:
assumeAutoFs; CIT; remote TECHNOTE (FAQ)

QUESTION
 How do I setup the assumeAutoFs to remote? 

ANSWER
 

The instructions on how to setup the assumeAutoFs to remote are below:
************************************************************************************

For assumeAutoFs flag there is no server CLI command to set it (unlike
for excluding directories), so an SQL command needs to be run.

To set the assumeAutoFs flag to 'remote' value for agent with given ID
the command is following:

insert into adm.cit_scan_attribute (agent_id, name, value, is_remote)
values ( <agent_id_here>, 'assumeAutoFS', 'remote', 0)
insert into adm.cit_scan_attribute (agent_id, name, value, is_remote)
values ( <agent_id_here>, 'assumeAutoFS', 'remote', 1)

There are two insert command to add this assumeAutoFS parameter to both
local and remote software scan configuration files.

To set the flag for all agents on given platform do the following:

insert into adm.cit_scan_attribute (platform, name, value, is_remote)
values ('<platform_name>', 'assumeAutoFS', 'remote', 0)
insert into adm.cit_scan_attribute (platform, name, value, is_remote)
values ('<platform_name>', 'assumeAutoFS', 'remote', 1)

the <PLATFORM> can be one of following values:
AIX | HPUX | JVM | Linux | Solaris | Windows | i5/OS

After this is set, also the SW scan configuration file version needs to be
updated in adm.agent table in the following way:

For changing only given agent:

update adm.agent set sw_scan_config_time = current timestamp where
id=<agent_id_here>

for setting parameter for given platform:

update adm.agent set sw_scan_config_time = current timestamp where
os_name='<platform_name>'

**********************************************************************************

NOTE: 

Please note that this procedure applies also to stand-alone scan configuration.
