ABSTRACT
 This document contains links to the download page for IBM Content Collector V3.0 Fix Pack 1. 

DOWNLOAD DESCRIPTION
This fix pack includes the following components:


 * Text Search Support for IBM Content Manager V8 
 * P8 Content Search Services Support.

It does not include the server component. 
IBM CONTENT COLLECTOR DOWNLOADS
 *  All versions [http://www.ibm.com/support/docview.wss?uid=swg27040223] 
 *  Version 3.0.0 Fix Pack 1- selected tab, 

Apply IBM Content Collector V3.0 Fix Pack 1 to upgrade the Text Search Support of the following products to Text Search Support for IBM Content Manager V3.0.0.1: 

 * IBM Content Collector 2.1.1, 2.2, or 3.0 
 * IBM CommonStore for Lotus Domino V8.4 
 * IBM CommonStore for Exchange Server V8.4

Apply IBM Content Collector V3.0 Fix Pack 1 to upgrade the P8 Content Search Services Support component of IBM Content Collector 2.2 or 3.0 to IBM Content Collector P8 Content Search Services Support 3.0.0.1. 

Below is a list of the problems that were fixed in this fix pack. APAR number Component APAR title HE11479 [http://www.ibm.com/support/docview.wss?uid=swg1HE11479] P8 Content Search Services Support IBM Content Collector does not always index embedded messages with IBM Content Search Services HE11489 [http://www.ibm.com/support/docview.wss?uid=swg1HE11489] P8 Content Search Services Support IBM Content Collector does not always index the correct ICCMailboxId attribute value with IBM Content Search Services HE11555 [http://www.ibm.com/support/docview.wss?uid=swg1HE11555] P8 Content Search Services Support IBM Content Search Services Support does not extract archive attributes for documents in derived document classes correctly HE11490 [http://www.ibm.com/support/docview.wss?uid=swg1HE11490] Text Search Support for IBM Content Manager afuIndexTool does not repair the TIEREF of items if the TIEREF values is NULL HE11510 [http://www.ibm.com/support/docview.wss?uid=swg1HE11510] Text Search Support for IBM Content Manager The IBM Content Collector indexer for text search does not update the TIEFLAG value of items with a severe error (IDXRC 200) (If an APAR link does not work, the detailed information for that APAR is not yet available from the Support page.
