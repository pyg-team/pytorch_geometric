Title: IBM Rational Administrator will not start on Windows 2008 Server 64 bit  or Windows 2003 Server 64 bit - United States

Text:
RN7.1; RN7.0.2; RN 7.1.1; RN7.1.1.1; SRX; Rational Administrator; Rational Robot; Rational RequisitePro; Requisite Pro; Req Pro; Robot; TM; ReqPro; RATLC01273500 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Rational Administrator will not start on the 64-bit version of Windows 2008 or Windows 2003 Server. The licensing application incorrectly indicates that the product is running on a terminal server. 

SYMPTOM
An error occurs when you attempt to start Rational Administrator. The error message states: 


Administrator has detected that this is a terminal server session. Administrator does not support running in this environment. Click OK to exit.


CAUSE
Licensing software indicates that the OS is a terminal server when it is not.



ENVIRONMENT
Windows 2008 Server 64 bit 

Windows 2003 Server 64 bit

\\HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Rational Software\Rational Administrator


 

To do this: 

 1.  Go to Run and type regedit. 
 2. Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Rational Software\Rational Administrator[/code>]
    NOTE: In some cases this key may not exist, so you will have to create the keys that are missing in the path. Follow the steps below, replacing CTXS[ with the missing key value from the tree above. 
 3. Right-click in the right pane and select New -> Key. 
 4. Type in CTXS for the key name. 
 5. Close Regedit. 
 6. Start Rational Administrator.
