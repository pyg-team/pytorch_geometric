APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The HTTP Server for z/OS does not return a Content-Length field
   with a value of zero in the response if there is no reponse
   body.
   This does not comply with RFC261 Section 9.2 Options.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All users of the HTTP Server for z/OS who    *
   *                 have the OPTIONS method enabled.             *
   ****************************************************************
   * PROBLEM DESCRIPTION: There is no content-length response     *
   *                      header returned with the response to an *
   *                      OPTIONS request.                        *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The HTTP Server for z/OS does not create a Content-Length
   response header as part of the response for an OPTIONS request.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Changed the code of the HTTP Server for z/OS so that the
   response header "Content-Length: 0" is part of the response to
   an OPTIONS request. As directied by RFC 2616 Section 9.2
   OPTIONS.
   
   The following COMPID is affected by these changes:
   
   5697D4300 HTTP Server for z/OS  Version 5
   
   PTF08F
   The code changes are stored in CMVC under defect PK64295
   
   * Cross Reference between External and Internal Names
