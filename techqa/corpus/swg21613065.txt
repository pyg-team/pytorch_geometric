Title: IBM Portal 8 silent install fails with CRIMA1076E on Windows if path uses standard " \ " delimiter - United States

Text:
Portal 8 silent install fail Windows / \ CRIMA1076E manageprofiles TECHNOTE (TROUBLESHOOTING)

PROBLEM
After manually editing a Portal 8 response file, the installation may fail on Windows if the path in certain properties are set using the standard " \ " delimiter.

SYMPTOM
Installation Manager logs and the console output will report an error similar to this:

C:\ProgramData\IBM\Installation Manager\logs\native\20120925_1317.log 

========================================

CAUSE
The installer uses properties in the response file where it expects to have forward slashes instead of back slashes. In the case of the output directory, the resulting path would be mixed with different slashes and causes this problem on Windows.



RESOLVING THE PROBLEM
1. Uninstall the existing WAS 

2. Change the following properties in the response file to use forward slashes: 


<data key='user.was.forward.installLocation,com.ibm.websphere.PORTAL.SERVER.v80' value='C:/IBM/WebSphere/AppServer'/>
<data key='user.configengine.forward.binaryLocation,com.ibm.websphere.PORTAL.SERVER.v80' value='C:/IBM/WebSphere/ConfigEngine'/>
<data key='user.wp.forward.profilepath,com.ibm.websphere.PORTAL.SERVER.v80' value='C:/IBM/WebSphere/wp_profile'/>

Note: When recording a response file, these are already set correctly.

3. Retry the installation
