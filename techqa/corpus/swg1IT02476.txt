APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When sharing conversation is enabled and an MQ error occurs
   (e.g. max channel limit reached), the MQ connection might fail.
   High CPU utilization might be observed.
   
   
    
   
   

LOCAL FIX
 *  Disable sharing conversation.
   
   
    
   
   

PROBLEM SUMMARY
 *  An administrator might see high CPU utilization and DataPower
   in a hang condition with DataPower MQ Queue Manager objects
   using sharing conversation. This might occur with a high traffic
   load at the time of an MQ error.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in 5.0.0.16, 6.0.0.8, 6.0.1.4, and 7.0.0.2.
