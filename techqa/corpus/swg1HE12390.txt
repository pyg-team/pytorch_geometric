APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  User is incorrectly able to delete Data Source from a Request
   when there is a Notice in draft state. This results in
   custodians listed in
   $Recipients & $RecipientNames in the Hold Notice not being
   updated.
   
   
    
   
   

LOCAL FIX
 *  No
   
   
    
   
   

PROBLEM SUMMARY
 *  User cannot delete Data Source from the Request->Data Sources In
    Scope -> Edit tab when there is a Notice in draft state.
   However the same delete operation is allowed if user edits a
   Request and tries to delete a Data Source from there. User
   should NOT be allowed to delete Data Sources in scope from the
   Request Edit page as well."
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in Fix Central in 6033 iFix1.
