Title: IBM Windows 7 clients getting Connection Reset when viewing tiff images - United States

Text:
Daeja viewer; tif; tiff TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Clients on Windows 7 get Connection Reset on IBM FileNet Application Engine (AE/Workplace) attempting to view tiff images. 

SYMPTOM
Users located overseas upgraded to Windows 7 are not able to view tiff images. Users in the US upgraded to Windows 7 are able to view tiff images.

CAUSE
A network policy for Windows 7 users located overseas prevented them from downloading jar files. If ji.jar can not be loaded to the client, the viewer will not load.

DIAGNOSING THE PROBLEM
Enable Java console on the client station to get full error.

RESOLVING THE PROBLEM
P8 administrators will need to work with their internal network teams to determine if there are network policies preventing downloads so they can be revised to allow jar downloads from the AE/Workplace server to the client workstation.
