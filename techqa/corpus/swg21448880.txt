Title: IBM How to crawl documents in Lotus Quickr for WebSphere Portal Version 8.5 - United States

Text:
 TECHNOTE (FAQ)

QUESTION
After creating a Seed list crawler to crawl documents in a Lotus Quickr for WebSphere Portal v8.5 server, the crawler collected documents from portal pages instead of documents in the Quickr document libraries. How do I configure the crawler to collect documents from Quickr document libraries?

ANSWER
Beginning with OmniFind Enterprise Edition, Version 8.5 Fix Pack 7, Lotus Quickr for WebSphere Portal, Version 8.5 is supported. 

Requirements: To integrate IBM OmniFind Enterprise Edition v8.5 with Lotus Quickr for WebSphere Portal v8.5, you must specify the following option when you run the es.wp61.install.jar script to set up the integration: 

-InstallType quickr 

The integration script is required to support secure search of Lotus Quickr for WebSphere Portal documents. 

 

Procedure: Follow these steps to configure the crawler to collect documents from the Quickr document libraries: 

 

 1. In the OmniFind Enterprise Edition administration console, create a Seed list crawler and specify Quickr as the repository type. 
 2. Check the crawler ID to locate the crawler configuration directory, such as: ES_NODE_ROOT/master_config/crawler_ID 
 3. Detach the seedlistcrawler_ext.xml file, which is attached to this document and shown below, to the crawler configuration directory. For example: ES_NODE_ROOT/master_config/Collection_ID.SEEDLIST_96705/seedlistcrawler_ext.xml

 * 
 * <?xml version="1.0" encoding="UTF-8"?>
 * 
 * <ExtendedProperties>
 * 
 * <AppendChild XPath="/Crawler/DataSources/Server/Target" Name="SeedlistExtraParameter">&amp;Source=com.ibm.lotus.search.
 * 
 * plugins.seedlist.retriever.cai.CAIRetrieverFactory</AppendChild>
 * 
 * </ExtendedProperties>


4. Restart the crawler session.
