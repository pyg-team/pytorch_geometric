Title: IBM TIP will not start after the SSL certificate has expired or has been deleted - United States

Text:
TIPL2; TIPL2SSL; SSL; Certificate Management TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Tivoli Integrated Portal will not start after the default certificate has expired or was deleted 

CAUSE
TIP requires a valid self-signed or CA certificate to start

DIAGNOSING THE PROBLEM
Check the systemOut.log for the following error from the time of the startup attempt:

###
[10/10/12 10:03:31:346 PDT] 0000072c WSX509KeyMana E CWPKI0024E: The certificate alias "<certificate alias>" specified by the property com.ibm.ssl.keyStoreServerAlias is not found in KeyStore "<TIP HOME>/profiles/TIPProfile/config/cells/TIPCell/nodes/TIPNode/key.p12"
###

RESOLVING THE PROBLEM
Either of the following methods can be applied:




Option 1: Disable TIP security, create a self-signed certificate, and enable security 

 1. Disable security using the instructions in the following technical note:
    "Disable security for Tivoli Integrated Portal"
    https://www-304.ibm.com/support/docview.wss?uid=swg21499504 [https://www-304.ibm.com/support/docview.wss?uid=swg21499504]
    
 2. Log back into TIP and create a self-signed certificate OR "Receive" the signed CA certificate if one was requested from and provided by a CA. Full instructions on replacing the default Websphere certificate are documented here: 
    http://www-01.ibm.com/support/docview.wss?uid=swg21607528 [http://www-01.ibm.com/support/docview.wss?uid=swg21607528]
    
 3. Enable security as described in the technical note in step 1.
    
    


Option 2: Create a self-signed certificate via command line using the ikeycmd tool. 
The instructions for creating a self-signed certificate are detailed in the following technical note: 
Creating a self-signed certificate for TIP via command line [http://www-01.ibm.com/support/docview.wss?uid=swg21613840]
