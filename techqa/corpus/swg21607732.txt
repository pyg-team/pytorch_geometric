QUESTION
 ODBC Probe : How do you configure the probe for DB2 

ANSWER
 

Note: The ODBC Probe is now replaced by the JDBC Probe. 

 

Please refer to the Support's Guide to the ODBC probe document. 

The general configuration for DB2 on Solaris is discussed within. 

The probe property file would be like: 


vi db2_odbc.props
# Define DB2 settings
ODBCSource : 'DB2'
ODBCUser : 'db2v95'
ODBCPassword : 'netcool'
ODBCTable : 'PROBE_TABLE'
ODBCFieldSelect : '*'
ODBCSelect : 'READ != 1'
ODBCMarkerColumn : 'SERIAL'
ODBCUpdate : 'READ=1'
ODBCTimeField : 'TIMESTAMP'
ODBCTimeFormat : '%Y-%m-%d %H:%M:%S'
ODBCInterval : 60
# ODBCMaxRetries : 3
# ODBCRecoveryFile : ''
# ODBCWarnings : 0
# MessageLevel : 'debug'
# Define file locations
MessageLog : '/opt/nrv73/omnibus/log/db2_odbc.log'
ODBCRecoveryFile : '/opt/nrv73/omnibus/var/db2_odbc.reco'
RawCaptureFile : '/opt/nrv73/omnibus/var/db2_odbc.cap'
SAFFileName : '/opt/nrv73/omnibus/var/db2_odbc.saf'
RulesFile : '/opt/nrv73/omnibus/probes/solaris2/db2_odbc.rules'
#EOF
