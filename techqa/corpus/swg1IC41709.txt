 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  chepoch -actual and lsepoch -actual fail with the following
   error:
   C:\>multitool lsepoch -actual newrep@\myvob
   For VOB replica "\myvob":
    multitool: Error: Incompatible versions of ClearCase software
   and/or databases:
   "servers.ibm.com:C:\VOBSTORE\myvob.vbs".
   
   multitool: Error: Trouble opening VOB database:
   "servers.ibm.com:C:\VOBSTORE\myvob.vbs".
   
   C:\>multitool chepoch -actualnewrep@\myvob
   multitool: Error: Incompatible versions of ClearCase software
   and/or databases:
   "servers.ibm.com:C:\VOBSTORE\myvob.vbs".
   
    multitool: Error: Trouble opening VOB database:
   "servers.ibm.com:C:\VOBSTORE\myvob.vbs".
    multitool: Error: Operation "remote_get_replica_properties"
   failed
    ("server2.ibm.com"): incompatible versions of ClearCase
   software and/or databases.
   
   CAUSE:: The vob server at one site upgraded to ClearCase
   2003.06(R)  and left the vob at feature level 3.
   The other site stayed at ClearCase 2002.05 and left the feature
   level at 3.  This defect is an incompatibility of 2003.06 with
   a down rev multisited vob.
   
   
    
   
   

LOCAL FIX
 *  Upgrade both sites to ClearCase 2003.06
   OR
   Downgrade the 2003.06 site to 2002.05
   OR
   Don't use the "-actual" option of lsepoch/chepoch
     - Do lsepoch AT the replica
     - Do chepoch by specifying all the epoch numbers
   
   
    
   
   

PROBLEM SUMMARY
 *  See APAR Error Description.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Updated the implementation of chepoch -actual and lsepoch
   -actual to more efficiently gather epoch information on a remote
   replica.
