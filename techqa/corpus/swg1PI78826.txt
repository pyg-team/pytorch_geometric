APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using the parse function, the data stream may output with
   overlaid 'corruption' (data from eariler in the card / map)
   though this will not trigger any kiind of map failure.
   
   Example expected output:
   
   UNB+UNOA:3+STYLUSSTUDIO:1+DATADIRECT:1+20051107:1159+6002'
   
   ...corrupted in output to:
   
   UNB+U{Cities|123456789,......ADIRECT:1+20051107:1159+6002'
   
   
    
   
   

LOCAL FIX
 *  WTXRTC - 60514
   PB / PB
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   IBM Transformation Extender customers who use the PARSE
   function.
   
   Problem Description:
   Errors in output data after a PARSE.
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  The PARSE function has been modified to correctly set our offset
    after the PARSE.
   
   Applies to:
   8.4.1.4, 9.0.0.1
