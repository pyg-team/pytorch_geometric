APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Following attributes was introduced in KYN ODI without affinity
   exclusions, what may cause incorrect data in tables:
   
   
   Object: Garbage Collection Analysis (table KYNGCACT)
   attribute: Max Heap KBytes
   introduced: 7.1.0.2 IF3
   
   Object: Application Health Status (table KYNAPHLTH)
   attribute SCE
   introduced: 7.1.0.2
   
   Object: J2C Connection Pools (table KYNJ2C)
   attribute: Connection Type
   introduced: 7.1.0.2 IF6
   
   
   Support with version equal or newer then introduced attribute
   will not work properly with older agents.
   Example:
   Some columns in table "Garbage Collection Analysis" may contain
   wrong data when:
   agent 7.1.0.2 is connected to TEPS with support 7.1.0.2 IF3 or
   newer.
   
   
    
   
   

LOCAL FIX
 *  Update KYN agent to the same version as KYN supports.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users of KYN supports 7.1 FP2, FP3, using
   * agents 7.1 FP2 or older.
   ****************************************************************
   * PROBLEM DESCRIPTION: Missing *AFFIN property in ODI causes
   * wrong cooperation with older agents.
   ****************************************************************
   * RECOMMENDATION: Update support for KYN agent to version
   * 7.1.0.3 IF3 or newer.
   * Update agents with version 7.1.0.2 to version 7.1.0.2 IF6
   * or newer.
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  d213292
   
   Missing property *AFFIN of new attributes causes, that TEPS
   recognized all KYN agents as handling new attribute, what may
   be a reason of wrong content of affected table.
   
   List of affected tables:
   
   Object: Garbage Collection Analysis (table KYNGCACT)
   attribute: Max Heap KBytes
   introduced: 7.1.0.2 IF3
   
   Object: Application Health Status (table KYNAPHLTH)
   attribute SCE
   introduced: 7.1.0.2
   
   Object: J2C Connection Pools (table KYNJ2C)
   attribute: Connection Type
   introduced: 7.1.0.2 IF6
   
   WARNING: The error will not be fixed for KYN agents
   with versions 7.1.0.2 IF1-IF5. Do update that agents
   to version 7.1.0.2 IF6 or newer.
   
   The fix for this APAR is contained in the following maintenance
   packages:
           | interim fix | 7.1.0.3-TIV-ITCAMAD-IF0003
