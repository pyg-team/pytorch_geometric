APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  An SQLERROR -302 naming variable/parameter 16 is issued if a
   table being added to a subscription contains a column comment
   that exceeds 49 characters in length.  The attempt to map the
   subscription fails.  Diagnostic information accompanying the
   message is "paaAddTableToCatalog; 13269".
   
   
    
   
   

LOCAL FIX
 *  Change the column comment(s) to be less than 50 characters in
   length.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users with tables that have column comments  *
   *                 containing 50 or more characters.            *
   ****************************************************************
   * PROBLEM DESCRIPTION: When attempting to truncate and store   *
   *                      the first 50 characters of a table's    *
   *                      column comments, the comment C-string   *
   *                      is improperly terminated.  DB2 issues   *
   *                      an SQLERROR -302 as a result.           *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When importing a table into the catalogue, the table's column
   comments are improperly processed.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Logic to properly nul-terminate the C-string has been added.
   
   
    
   
   

TEMPORARY FIX
 *  Change the column comment(s) to be less than 50 characters in
   length.
