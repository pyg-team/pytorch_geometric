APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  A class in Apache Commons FileUpload contains code that triggers
   code execution as part of deserialization. The Apache Commons
   FileUpload library exists in IBM Business Process Manager (BPM)
   and makes this class available on the class path. If any code on
    the Application Server deserializes Java objects from an
   ObjectInputStream, this code is vulnerable to remote code
   execution.
   
   CVEID: CVE-2016-1000031
   DESCRIPTION: Apache Commons FileUpload, as used in Novell NetIQ
   Sentinel and other products, could allow a remote attacker to
   execute arbitrary code on the system, caused by deserialization
   of untrusted data in DiskFileItem class of the FileUpload
   library. A remote attacker could exploit this vulnerability to
   execute arbitrary code under the context of the current process.
   CVSS Base Score: 9.8
   CVSS Temporal Score:
   See https://exchange.xforce.ibmcloud.com/vulnerabilities/117957 
   for the current score
   CVSS Environmental Score*: Undefined
   CVSS Vector: (CVSS:3.0/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H)
   
   PRODUCTS AFFECTED
   IBM BPM
   IBM BPM Advanced
   IBM BPM Standard
   IBM BPM Express
   IBM WebSphere ESB
   IBM WebSphere ESB Registry Edition
   IBM BPM ESB

 *  A fix that upgrades the version of Apache Commons FileUpload
   component to 1.3.3 is available for WebSphere Process Server
   V7.0.0.5, WebSphere Enterprise Service Bus V7.0.0.5, IBM BPM
   V7.5.1.2, V8.0.1.3, V8.5.0.2, V8.5.5.0, V8.5.6.0 cumulative fix
   02, V8.5.7.0 cumulative fix 2017.06, V8.6.0.0 and V8.6.0.0
   cumulative fix 2017.12, and will be included in future releases
   of IBM BPM.
   
   For WebSphere Process Server V7.0.0.5, IBM BPM V7.5.1.2 and
   V8.0.1.3 contact IBM support and request the interim fix
   JR58572.
