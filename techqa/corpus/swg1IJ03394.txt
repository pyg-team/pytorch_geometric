   Fail-over no longer working after RBA intigration.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Following the installation of the Runbook Automation add-on  *
   * in an Impact cluster , when a user edits the default values  *
   * of the Netcool/Impact derby                                  *
   * database to configure launch-in-context support and          *
   * integration with IBM Runbook Automation, failover to the     *
   * secondary cluster member no longer                           *
   * occurs successfully as RBA can't connect to the secondary    *
   * Impact cluster.                                              *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * 1. create a text file in the impact install containing the   *
   * encrypted RBAAPIKeyPassword.                                 *
   * 2.  edit rba policy code ,  to retrieve the encrypted        *
   * RBAAPIKeyPassword from the text file.                        *
   ****************************************************************
