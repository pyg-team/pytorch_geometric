Title: IBM Is the generic index file format for OnDemand 2.2 releases supported in the later versions of OnDemand? - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 Can I use the following generic index file format, found in Indexing Reference version 2 release 2, in the later versions of OnDemand?

FIELD NAMES BEGIN:
index_1_name
...
index_32_name
FIELD NAMES END:
index_1_value
...
index_32_value
input_filename
offset
length
index_1_value
...
index_32_value
input_filename
offset
length
... 

ANSWER
The old format is supported in CMOD 8.3, but not in 8.5. . 

Readme 8.5.0.2: 
The old Generic Index file format used prior to Content Manager OnDemand
8.3 is no longer supported.
