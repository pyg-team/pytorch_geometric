APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When ODM is deployed on JBoss 6.1 EAP and running with an IBM
   JDK the following warnings are seen in the server log when
   deploying a ruleapp from the Business Console:
   [2015-05-27 16:53:07,553] ERROR [stderr] (http-/10.202.41.206:
   8080-10) [WARNING 0008] Warning: The encoding 'UTF-16' is not
   supported, using 'UTF-8'.
   [2015-05-27 16:53:07,572] ERROR [stderr] (http-/10.202.41.206:
   8080-10) [WARNING 0008] Warning: The encoding 'UTF-16' is not
   supported, using 'UTF-8'.
   [2015-05-27 16:53:07,575] ERROR [stderr] (http-/10.202.41.206:
   8080-10) [WARNING 0008] Warning: The encoding 'UTF-16' is not
   supported, using 'UTF-8'.
   ...
   
   
    
   
   

LOCAL FIX
 *  add    <path name="org/apache/xml/serializer" />  in jrules-res-
   management-JBOSS61EAP.ear\META-INF\jboss-deployment-structure.
   xml
   as in:
   ...
         <!-- Add path for the IBM JDK -->
             <path name="org/apache/xerces/dom" />
             <path name="org/apache/xml/serializer" />
   ...
   
   
    
   
   

PROBLEM SUMMARY
 *  jboss EAP6.1 classloading policy is very different and require
   special instructions.
   
   
    
   
   

PROBLEM CONCLUSION
 *  the package of Jboss EAR is modified
