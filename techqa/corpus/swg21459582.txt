PROBLEM(ABSTRACT)
 You are deploying your first event binding in CICS Transaction Server for z/OS (CICS TS) and the bundle definition (BUNDDEF) fails to install or deploy. You receive message DFHRL0002 E applid A severe error (code X'022E') has occurred in module DFHRLPM if running CICS TS V4.1. If you are running CICS TS V4.2, the code is X'0272'. You also receive message EYUXL0905E reporting an abend AAM4 (abendAAM4) in BCXI. 

SYMPTOM
For CICS TS 4.1: DFHRL0002 [http://publib.boulder.ibm.com/infocenter/cicsts/v4r2/index.jsp?topic=/com.ibm.cics.ts.messages.doc/cics_mc/dfhrl/RL0002.html] A severe error (code X'022E [http://publib.boulder.ibm.com/infocenter/cicsts/v4r2/index.jsp?topic=/com.ibm.cics.ts.doc/dfhs6/topics/dfhs6_rl.html]') has occurred in module DFHRLPM

CAUSE
Missing RDO group DFHRL that was added in CICS TS V4.1



ENVIRONMENT
CICS TS V4.1 and V4.2



DIAGNOSING THE PROBLEM
For CICS TS V4.1: 

 * The X'022E' in the DFHRL0002 E messages equates to tid_rlpk_internal_error. DFHRLPM in the rlpk_load_bootstrap_icm routine calls the CICS loader domain to acquire the DFHRLMF program. If there is an error, sever_error id_rlpk_internal_error is returned.
   
   

For CICS TS V4.2:  * The X'0272' in the DFHRL0002 E messages equates to tid_rlpk_acquire_program_error. DFHRLPM in the rlpk_load_bootstrap_icm routine calls the CICS loader domain to acquire the DFHRLMF program. If there is a error, severe_error tid_rlpk_acquire_program_error is returned.
   


In this instance, since the DFHRL RDO group was missing, there was no program entry for DFHRLMF and the DFHLDLD iproc LDLD_ACQUIRE will return a LDLD_RESPONSE of LDLD_EXCEPTION and LDLD_REASON of LDLD_PROGRAM_NOT_DEFINED resulting in the DFHRL0002 E with code X'022E' in CICS TS V4.1 and X'0272' in CICS TS V4.2. 

If you access the MSGUSR in the joblog for the region where the DFHRL0002 E message is displayed and enter a FIND command for string "DFHRL" and it is not found, then you are missing the DFHRL RDO group. . If the DFHRL RDO group has been installed the MSGUSR will contain message: 

DFHAM4893 [http://publib.boulder.ibm.com/infocenter/cicsts/v4r2/index.jsp?topic=/com.ibm.cics.ts.messages.doc/cics_mc/dfham/AM4893I.html] I mm/dd/yyyy hh:mm:ss applid Install for group DFHRL [http://publib.boulder.ibm.com/infocenter/cicsts/v4r2/index.jsp?topic=/com.ibm.cics.ts.migration.doc/dfheh/dfhe5_rdo_new_ibm_dfhrl.html] has completed successfully. 

The AAM4 abend for CICSPlex SM method BCXI occurs as a result of CPSM attempting to perform the install for the BUNDDEF resource and receiving a failure due to the lifecycle environment missing. 
RESOLVING THE PROBLEM
Upgrade your CICS TS regions using the DFHCSDUP utility [http://publib.boulder.ibm.com/infocenter/cicsts/v4r2/index.jsp?topic=/com.ibm.cics.ts.migration.doc/dfheh/dfhe5_rdo_csd_upgrade.html] and ensure DFHLIST is specified in your CICS system initialization table (SIT) GRPLIST parameter. Also, make sure that the DFHLIST includes the new RDO group DFHRL as well as DFHRS, DFHWU and DFHWEB2.

Note: If you have the PTF for CICS APAR PM40297 [http://www.ibm.com/support/docview.wss?uid=swg1PM40297] applied, you will receive new message DFHRL0123E:

DFHRL0123 [http://publib.boulder.ibm.com/infocenter/cicsts/v4r2/topic/com.ibm.cics.ts.messages.doc/cics_mc/dfhrl/RL0123E.html] E The CICS resource life-cycle manager failed to acquire PROGRAM program_name for BUNDLE resource bundle_name.
