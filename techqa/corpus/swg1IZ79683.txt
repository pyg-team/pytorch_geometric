Title: IBM IZ79683: CONFIGURATION ANALYSER WITH SNMP PROBE NOT INSTALLED PROPERLY INOMNIBUS 7.1/7.2/7.2.1 - United States

Text:

APAR STATUS
 * CLOSED AS FIXED IF NEXT.
    
   
   

ERROR DESCRIPTION
 *  Configuration Analyser with SNMP Probe does not execute in
   OMNIbus 7.1/7.2/7.2.1 because the mttrapd.check.jar is installed
   in $OMNIHOME/probes/java/, instead of the correct path
   $OMNIHOME/probes/solaris2/.
   
   
    
   
   

LOCAL FIX
 *  manually copy mttrapd.check.jar file from $OMNIHOME/probes/java/
   to $OMNIHOME/probes/solaris2
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All user who use Config Analyser with mttrapd probe on Unix
   platform.
   ****************************************************************
   PROBLEM DESCRIPTION:
   Config Analyzer not able to start with this error:
   "It appears as though tests are not installed for mttrapd.
   Please check the Configuration Analyser user documentation for a
   list of compatible Integrations and the versions needed.
   An error has caused the program to terminate abnormally. Please
   re-run it and set CheckLevel to DEBUG_MIN in the probe
   properties file, and then check
   <$NCHOME>/omnibus/log/mttrapd.check.log for more details."
   
   The <probe>.check.jar should be deploy at $OMNIHOME/probes/java/
   instead of $OMNIHOME/probes/$ARCH for Unix platform. While for
   Windows platform, it should be in $OMNIHOME/probes/$ARCH.
   ****************************************************************
   RECOMMENDATION:
   Upgrade to mttrapd probe patch to ver 11.2
   Upgrade ConfigAnalyzer to ver 1.2.0
   Upgrade Omnibus to:-
   7.1.0.13 or
   7.2.0.10 or
   7.2.1.9 or
   7.3.0.3
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION

TEMPORARY FIX
