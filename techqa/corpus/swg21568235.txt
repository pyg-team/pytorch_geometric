Title: IBM java.security.NoSuchAlgorithmException running taskrunner or solutioninstaller - United States

Text:
taskrunnner; psi; solutioninstall; solution installer; crypto; security; TPAEINST TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Running taskrunner or solution installer receive java.security.NoSuchAlgorithmException: Cannot find any provider supporting AES/ECB/PKCS5Padding 

CAUSE
incorrect jre version

RESOLVING THE PROBLEM
The error may be caused by a mismatch with the Java versions with the Deployment Engine (DE). 

To resolve the issue: 

1. Stop the IBM ADE Service 

2. Edit the c:\program files\ibm\common\acsi\setenv.cmd file 

3. Change the SI_JAVA_HOME variable to point to the ibm\smp\jre folder 

4. Change the SI_EXT_DIRS variable to point to the ibm\smp\jre folder and remove C:\WINDOWS\Sun\Java\lib\ext 

 (See example file below with the changes) 

5. Start the IBM ADE Service 

Taskrunner and solutioninstaller should run now 

 

setenv.cmd file, changes in bold

set ACU_COMMON=C:\Program Files\IBM\Common\acsi
set SI_PATH=C:\Program Files\IBM\Common\acsi
set SI_JAVA_HOME=C:\ibm\smp\jre

if "%SI_HAS_MODIFIED_PATH%" == "" goto MODPATH
goto AFTERMOD
:MODPATH
set PATH=%SI_PATH%\bin;%SI_PATH%\lib;%PATH%
set SI_HAS_MODIFIED_PATH=true
:AFTERMOD

set SI_ENV_DIR=%ACU_COMMON%\si_envs
set SI_EXT_DIRS=C:\ibm\smp\jre\lib\ext;C:/Program Files/IBM/Common/acsi\lib
set SI_INSTALL_MODE=0
