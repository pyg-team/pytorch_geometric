Title: IBM TWS 9.1  user security setting on local OS - United States

Text:
dash; local os; authentication; tws TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 TDWC, Tivoli Dynamic Workload Console, is one of the Tivoli Workload Scheduler components, and runs on DASH/JazzSM with WebSphere 8.5.0.1. Customer wants to add Local OS registry , steps are outlined here since this is not possible via console. 

RESOLVING THE PROBLEM
Local OS registry is supported in federated repository on WAS V8.5 environment. To add local OS registry to federated repository, customer need to configure it via wsadmin command. WAS admin console does not support adding local OS registry in federated repository. Following is wsadmin command to add local OS registry;

--------------------------------------------
LocalOS configuration steps as provided by Websphere/VMM teams :

(1)
$AdminTask createIdMgrCustomRepository {-adapterClassName
com.ibm.ws.wim.adapter.urbridge.URBridge -id localOS }

(2)
$AdminTask addIdMgrRepositoryBaseEntry {-id localOS -name o=localOS}

(3)
$AdminTask addIdMgrRealmBaseEntry {-name defaultWIMFileBasedRealm
-baseEntry o=localOS}

(4)
$AdminTask setIdMgrCustomProperty {-id localOS -name
uniqueUserIdProperty -value uniqueId }

(5)
$AdminTask setIdMgrCustomProperty {-id localOS -name
userSecurityNameProperty -value uniqueName }

(6)
$AdminTask setIdMgrCustomProperty {-id localOS -name
userDisplayNameproperty -value displayName }

(7)
$AdminTask setIdMgrCustomProperty {-id localOS -name
uniqueGroupIdProperty -value uniqueId }

(8)
$AdminTask setIdMgrCustomProperty {-id localOS -name
groupDisplayNameProperty -value displayName }

(9)
$AdminTask setIdMgrCustomProperty {-id localOS -name
groupSecurityNameProperty -value uniqueName }

(10)
$AdminConfig save 

--------------------------------------------

Here is a tested scenario : 

1. Create a dummy admin user on WIM and assign all related authorities.
2. Modify "Primary administrative user name" to the dummy admin user.
3. Restart WebSphere.
4. Remove the original TDWC admin user from WIM.
5. Perform to add LocalOS configuration steps provided by WebSphere.
6. Restart WebSphere.
-> Now, both WIM and localOS are used as user repositories.
7. Add necessary roles to the new TDWC local OS user.
8. Modify "Primary administrative user name" to the new TDWC local OS user.
9. Delete WIM from Federated repositories.
10. Restart WebSphere.
