LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: IBM WebSphere Application Server V6.0.2 and  *
   *                 V6.1 users running the versionInfo and       *
   *                 historyInfo utilities on HP and Solaris      *
   *                 operating systems.                           *
   ****************************************************************
   * PROBLEM DESCRIPTION: OutOfMemory exception is thrown when    *
   *                      running versionInfo and historyInfo     *
   *                      utilities on HP and Solaris.            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The default maximum heap size of Solaris and HP JDKs is 64MB
   only. If you have done a lot of maintenance or stack
   product installs on a WebSphere Application Server
   installation, the xml which tracks the history of all these
   installs will grow to a point that the versionInfo and
   historyInfo utilities require more than 64MB of memory to
   process this xml file. When this happens, an OutOfMemory
   exception will be thrown.
   
   
    
   
   

PROBLEM CONCLUSION
 *  To prevent the OutOfMemory exception, the default max heap
   size being used to run the versionInfo and historyInfo
   utilities on HP and Solaris is increased to 256MB.
   
   The fix for this APAR is currently targetted for inclusion in
   fix packs 6.0.2.29 and 6.1.0.19. Please refer to the Recommended
   Updates page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
