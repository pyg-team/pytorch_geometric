APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Reopening a Collection Plan to add a new custodian resets
   custodian status from "Completed" to "Not Executed". It is
   time consuming for the Plan owner to manually set it to the
   original status when there are lots of custodians on the plan.
   
   
    
   
   

LOCAL FIX
 *  Manually set the status for each custodian
   
   
    
   
   

PROBLEM SUMMARY
 *  Reopening a Collection Plan to add a new custodian resets
   custodian status from "Completed" to "Not Executed". It is
   time consuming for the Plan owner to manually set it to the
   original status when there are lots of custodians on the plan.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix is available now on Fix Central, as part of Atlas 6.0.3
   Fix pack 4
