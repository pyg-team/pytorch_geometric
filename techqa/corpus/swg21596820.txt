Title: IBM CCL-BIT-0006 when building cube based on FM package - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Transformer model is using a Framework Manager package that contains relatively long requests.
The cube generation stops while requesting data from database and the complete message is as follows :
CCL-BIT-0006 The connection closed before the request is processed. If you are using WebSphere Application Server, to reduce the frequency of this error, increase the Persistent Timeout parameter for the Web container transport chains in the administrative console. Increase the time in 10-15 second intervals until the error no longer or rarely occurs. 

CAUSE
Web server connection timeout

RESOLVING THE PROBLEM
 

* If using Apache web server :

Increase KeepAliveTimeout in Apache configuration file httpd.conf
Unix/Linux : /etc/httpd/httpd.conf
Windows: /Program Files/Apache Group/Apache/conf/httpd.conf

* If using IIS 7.0/7.5 :

Select Default Web Site.
In the Features View, double-click the "HTTP Response Headers" feature.
Once in HTTP Response Headers feature page, right-click anywhere and select "Set Common Headers".
Verify the "Enable HTTP keep-alive" option is selected.
Now right-click Default Web Site and select "Manage Web Site", "Advanced Settings".
Expand "Connection Limits" and increase "Connection Time-out (seconds)"
