Title: IBM PM59935; 8.0.0.3: console error when updating transientidp repository - United States

Text:
PM59935; PM59935; PM59935 DOWNLOADABLE FILES

ABSTRACT
 Configuring or updating a custom repository of Federated Repositories though the administrative console shows an error message although the updated configuration is saved. 

DOWNLOAD DESCRIPTION
PM59935 resolves the following problem:

ERROR DESCRIPTION:? 
CWWIM5019E console error when updating transientidp repository 

LOCAL FIX:? 

PROBLEM SUMMARY:?

USERS AFFECTED:
All users of IBM WebSphere Application Server V8.0

PROBLEM DESCRIPTION:
Configuring or updating a custom repository of Federated Repositories though the administrative console would show an error message though the updated configuration is saved.

RECOMMENDATION:
None

While Configuring or updating a custom repository of Federated Repositories though administrative console from the panel Security -> Global Security -> Federated Repositories can result in the following error message even if the custom repository implementation class is in the classpath as
required. 




CWWIM5019E Adapter class name is missing or is not valid: com.ibm.wps.vmm.adapter.inmemory.InMemoryAdapter

PROBLEM CONCLUSION:? 
Code changes have been made to load the custom repositoy class using the correct classloader which resolves this issue. 

The fix for this APAR is currently targeted for inclusion in fix pack 8.0.0.4 Please refer to the Recommended Updates page for delivery information: 
http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980] 


Note: This interim fix can also be installed using Install Manager (IM) with the 
Web-based ("live") repository provided by IBM.
