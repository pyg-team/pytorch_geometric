APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Launching  Analysis Studio (blank analysis) using a DQM package
   and Hebrew as Content Language will fail with:  XQE-GEN-0002 An
   unexpected exception occurred: 'CM-REQ-4342 An error occurred
   with the client.' WHILE [CCLMsg: system text='CM-REQ-4342 An
   error occurred with the client.']    Launching Report Studio
   using a DQM package and Hebrew/Arabic Product and Content
   locales will fail with a CM error while trying to load
   metadata..     Please note that:   - the problem doesn't occur
   if the package is published as classic (DQ not checked)   -
   problem doesn't occur for DQ package if the content locales for
   Hebrew and Arabic are removed from "Cognos Configuration"   -
   the problem occurs if, when launching the browser,  the "Hebrew"
   language is selected under User Preferences. Problem doesn't
   occur anymore if the language is changed to Default (English)
   - the problem is not specific for packages containing Hebrew or
   Arabic. It van be reproduced against "Go Sales and Retailers"
   created under English language     Error (see error trace
   attached):   CM-REQ-4159 Content Manager returned an error in
   the response header. The error "cmAuthenticateFailed CM-CAM-4005
   Unable to authenticate. Check your security directory server
   connection and confirm the credentials entered at login." can be
   found in the response SOAP header:   CM-REQ-4342 An error
   occurred with the client.
   
   
    
   
   

LOCAL FIX
 *  In Cognos Connection use English as content language.
