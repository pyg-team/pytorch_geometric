APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The Bi Directional Gateway will terminate as a service, if you
   try to run the gateway to a site that has a combination of high
   event load to process + High latency nework.
   
   Often the failure is observed whilst its resizing the NGTK
   component.
   
   
    
   
   

LOCAL FIX
 *  run objectserver via command line.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   Users that run OMNIbus gateways as Windows services
   ****************************************************************
   PROBLEM DESCRIPTION:
   When a gateway service on Windows is started, with a large
   number of events needing to be synchronized across a slow
   network connection, the gateway service would timeout and the
   gateway would not be started.
   ****************************************************************
   RECOMMENDATION:
   Apply the maintenance package listed in the Problem Conclusion.
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  Event processing has been postponed until after the service is
   marked as started, so that the gateway service will no longer
   timeout.
   
   The fix for this APAR is contained in the following maintenance
   packages:
   
        | fix pack | 7.2.0-TIV-NCOMNIbus-FP0009
        | fix pack | 7.2.1-TIV-NCOMNIbus-FP0008
        | fix pack | 7.3.0-TIV-NCOMNIbus-FP0002
