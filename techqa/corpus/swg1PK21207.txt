Title: IBM PK21207: AN IOEXCEPTION IS THROWN WHEN ISSUING A LONG-RUNNING SCHEDULER COMMAND USING LRCMD THROUGH THE ON-DEMAND ROUTER (ODR) - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When  attempting to issue a long-running command scheduler
   command using lrcmd.[bat|sh] through the On-Demand Router (ODR),
   an IOException will be returned as follows:
   
   C:\IBM\WebSphere\AppServer\bin>lrcmd -cmd=submit
   -xJCL=C:\SimpleCIxJCL.xml -port=<ODR port> -host=<ODR hostname>
   
   CWLRB4940I: com.ibm.ws.batch.wsbatch : -cmd=submit
   -xJCL=C:\SimpleCIxJCL.xml -port=<ODR port> -host=<ODR hostname>
   
   CWLRB4880E: -cmd=submit -xJCL=C:\SimpleCIxJCL.xml -port=<ODR
   port>
   -host=<ODR hostname> failed:
              com.ibm.ws.batch.wsbatch: java.io.IOException:
   Connection
   close: Read failed.  Possible end of stream encountered.

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All users of WebSphere Extended Deployment   *
   *                 version 6.0.1.1.                             *
   ****************************************************************
   * PROBLEM DESCRIPTION: When  attempting to issue a             *
   *                      long-running scheduler command using    *
   *                      lrcmd.[bat|sh] through the On-Demand    *
   *                      Router (ODR), an IOException will be    *
   *                      returned as follows:                    *
   *                                                              *
   *                      C:\IBM\WebSphere\AppServer\bin>lrcmd    *
   *                      -cmd=submit -xJCL=C:\SimpleCIxJCL.xml   *
   *                      -port=<ODR port> -host=<ODR hostname>   *
   *                                                              *
   *                      CWLRB4940I: com.ibm.ws.batch.wsbatch :  *
   *                      -cmd=submit -xJCL=C:\SimpleCIxJCL.xml   *
   *                      -port=<ODR port> -host=<ODR hostname>   *
   *                                                              *
   *                      CWLRB4880E: -cmd=submit -xJCL=C:\Simple *
   *                      CIxJCL.xml -port=<ODR port>             *
   *                      -host=<ODR hostname> failed:            *
   *                      com.ibm.ws.batch.wsbatch: java.io.      *
   *                      IOException:Connection close: Read      *
   *                      failed.  Possible end of stream         *
   *                      encountered.                            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The problem was caused by assuming in the code the every SOAP
   request will have a web module with web service and operation
   node defined. Therefore for the cases where web service and
   operation nodes do not exist, trying to use the operation map
   to get a key results in a null pointer exception. To the
   caller this wrapped up exception shows up as an IO Exception.
   
   
    
   
   

PROBLEM CONCLUSION
 *  For each SOAP over HTTP request, check for the definition of
   web service and web service operation nodes, if they exist use
   them to fetch the workclasses and perform further request
   classification as necessary, otherwise, use the original
   request mapping to route request.
