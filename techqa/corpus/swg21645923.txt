Title: IBM Unable to login to FileNet Workplace XT getting error, java.lang.StackOverflowError - United States

Text:
java.lang.StackOverflowError; Workplace XT login TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 While attempting to login to Workplace XT the following error page displayed:

Error Page Exception 

SRVE0260E: The server cannot use the error specified for your application to handle the Original Exception printed below: 

Error Message: java.lang.StackOverflowError 
Error Code: 500 
Error Stack: 
java.lang.StackOverflowError 
at javax.faces.context.FacesContext.getAttributes(FacesContext.java:57)
at javax.faces.context.FacesContext.getAttributes(FacesContext.java:64) 

SYMPTOM
The following error messages are logged in the Workplace XT WebSphere application server (WAS) systemout.log:

[7/26/XX 10:10:28:362 EDT] 00000074 MyfacesConfig E Both MyFaces and the RI are on your classpath. Please make sure to use only one of the two JSF-implementations. 
[7/26/XX 10:11:16:371 EDT] 000000b3 servlet E com.ibm.ws.webcontainer.servlet.ServletWrapper service Uncaught service() exception thrown by servlet faces: java.lang.StackOverflowError


CAUSE
The Class loader order for the Workplace XT - Manage Module is set to, Classes loaded with parent class loader first, in the WebSphere Administrative Console.

ENVIRONMENT
WebSphere

RESOLVING THE PROBLEM





 1. From the WAS Administrative Console - WebSphere enterprise application - Workplace XT application - Manage Module, change the Class Loader order to Classes loaded with local class loader first (parent last).
    
    
 2. Restart the Workplace XT WAS instance.
