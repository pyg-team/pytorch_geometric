PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: WebSphere Process Server 6.1.0.x users.      *
   ****************************************************************
   * PROBLEM DESCRIPTION: The Websphere Process Server            *
   *                      encounters ClassCastException during    *
   *                      restart.                                *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The root cause of this issue is the SOACore can not find the
   data type registry for header message.
   FamilyProcessContextHeader, we found the data type registry
   information is managed by
   com.ibm.bpe.sca.headers.HeadersPackage, and the SOACore cannot
   de-serialize the header into expected type if the
   HeadersPackage has not been initialized.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This issue will be fixed in a future fix pack.
