Title: IBM CLI0125E after db2  client upgrade from v9.7 or v10.1  to v10.5 or higher - United States

Text:
CLI0125E; client upgrade ; db2z v9; stored procedure TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After client upgrade application that connects to DB2 v9 or above on z/OS server started to report error CLI0125E on stored procedure call 

SYMPTOM
CLI based application reports error CLI0125E during call of the stored procedure following the client upgrade


CAUSE
Change in default settings for CommitOnEOF when application is running against zOS server

In v9.7 and v10.1 the default value for CommitOnEOF is 0;
Starting from v10.5 the default value for CommitOnEOF for connection to zOS server v9 and above
is 1.

RESOLVING THE PROBLEM
Change the value of keyword CommitOnEOF to 0 to resolve an error CLI0125E reported after upgrade of the db2 client level.
