PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server V6.0.2 and V6.1                      *
   ****************************************************************
   * PROBLEM DESCRIPTION: WebSphere Connect JDBC driver           *
   *                      V3.7.0.65 may throw exception           *
   *                      "IO Error writing temp file: Negative   *
   *                      seek offset"                            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   After upgrading WebSphere Application Server environment from
   V6.0.2.29 to V6.0.2.39, the following exception from
   applications that use the WebSphere-embedded Connect JDBC
   driver to connect to SQL Server might occur.
   
   java.sql.SQLException: [IBM][SQLServer JDBC Driver]IO Error
   writing temp file: Negative seek offset
   at
   com.ibm.websphere.jdbc.base.BaseExceptions.createException(Unkno
   wn Source)
   
   
   WebSphere Application Server Version 6.0.2.39 ships with
   Embedded connect JDBC driver V3.7.0.65 that was causing this
   problem.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The problem has been fixed in the WebSphere Embedded connect
   JDBC driver V3.7.0.68.
   The fix for this APAR is currently targeted for inclusion in
   fix packs 6.0.2.43 & 6.1.0.35.  Please refer to the Recommended
   Updates page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
