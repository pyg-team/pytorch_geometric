LOCAL FIX

PROBLEM SUMMARY
 *  If IE is used to download a 2MB file using SSL but the Save
   To dialog is not completed for more than client-connect-
   timeout seconds, IE reports the download was completed
   successfully but only a partial file ( 90KB) gets saved.
   
   Also, the same partial-file failure can be observed for an
   SSL download that reuses an SSL connection (a persistent
   connection) with the user waiting a little more than
   persist-con-timeout seconds before completing the Save To
   dialog.
   
   
    
   
   

PROBLEM CONCLUSION
 *  WebSEAL now resets a client SSL connection if any error/
   timeout occurs on any read/write operation other than the
   initial read.  The client-connect-timeout webseald.conf
   parameter is now used for timing-out only the initial read
   of a connection after the accept() succeeds.  The persist-
   con-timeout webseald.conf parameter is now used for timing
   out the initial read of a persistent connection.  A new,
   optional, webseald.conf parameter is being introduced,
   intra-connection-timeout, whose value is used to time-out
   all other read/write operations on an SSL connection to the
   client.  The default value for this new parameter is the
   value of the client-connect-timeout parameter.
   
   This solution does not affect TCP connections to the
   client.
   
   Codefix in patch 4.1-AWS-FP07.
