The same problem will happen with Build Utility 7552 on z/OS platform

CAUSE
An internal debugging message was injected into the normal build output message.

RESOLVING THE PROBLEM
The verbose stack trace can safely be ignored; However, if this needs to be removed then it can only be accomplished via a product fix.


If you experience this problem and require a test fix to remove those extra messages then please contact IBM support for the test fix. For information on how to contact IBM support see the 'IBM Software Support Handbook' link in the 'Related Information' section.
