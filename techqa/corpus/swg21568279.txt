Gather Documentation  1. Stop the Oracle Extended Agent 
 2. Set RAS1 trace settings below 
 3. Reconfigure the Oracle Extended Agent 
 4. Recreate problem and take screen print of issue 
 5. Process <install path>/bin/tacmd pdcollect from the command line 


Send the pdcollect jar/tar file, screen prints, and all useful Oracle environmental information including oracle logs, listener.ora and tnsnames.ora files if required 


Oracle Extended Agent Trace Settings  * Update the Oracle Extended Agent environment file:
 * Windows 
 *  <install path>\TMAITM6\KRZENV [http://www-1.ibm.com/support/docview.wss?&uid=swg21266129] 
 * 
 * 
 *  <install path>/config/rz.ini 
 * Set trace values for KBB_ RAS1 and KBB_RAS1_LOG 
 *  KBB_RAS1=ERROR (UNIT:krz ALL) (UNIT:knz ALL) (UNIT:kra ALL) KBB_RAS1_LOG set the following for more and larger log files 
 * 
 * 
 * Further specific filtering options, remove krz above and use only specific UNIT 
 *  krzalert: the Alert Log component krzbasic: the basic functions component krzcomm: the common connection component krzmutex: the event and mutex component krzora: the OCI(Oracle Call Interface) related functions component krzxml: the XML component
