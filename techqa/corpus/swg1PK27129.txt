Title: IBM PK27129: MESSAGE DELIVERY FAILURES USING MQSERVER - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using an environment with multiple clients communicating between
   WSAS and WMQ using the WebSphere MQ (WMQ) Link and WMQ Server,
   under heavy load messages may fail to be delivered to their
   destinations.
   
   The logs may show several errors, including:
   WTRN0078E: An attempt by the transaction manager to call
   start on a transactional resource has resulted in an error
   and
   CWSIS1000E: The requested action could not be carried out as
   it does not comply with the XA transaction protocol.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users of WebSphere Application Server SIBus  *
   ****************************************************************
   * PROBLEM DESCRIPTION: During any kind of XA Transactional     *
   *                      Messaging, the user could see           *
   *                      exceptions and FFDCs.                   *
   *                      XidAlreadyKnownException and other      *
   *                      XA exceptions are possible.             *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The fix prevents concurrent access to some of the XA Transaction
   data.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The problem is fixed by this APAR.
