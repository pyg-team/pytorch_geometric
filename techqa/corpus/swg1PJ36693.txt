Title: IBM PJ36693: An error occurs when the search criteria defined for a search te mplate includes multiple object stores with subclass filtering. - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Creating a search template across two object stores with
   subclass filtering and a choice list configured, and then saving
   it results in a "Catastrophic failure Operator 'eq' should have
   two child nodes" error.
   
   Steps to Reproduce:
   This error is reproducable when you create a search template
   across 2 object stores with sub class filtering.  Below is the
   procedure to reproduce the issue:
   CE server:
   
   a) OS 1:
     1) Create a doc class (i.e: class1)
     2) Assign any string property to this document class (i.e: Cc)
   b) OS 2:
    1) Create a doc class (i.e: class2)
    2) Assign a choice list property to this document class.
   Choice list can be String, Single.
   c) Alias document class from OS 2 to OS1 (ie: copy the GUID of
   class 2 to assign it to class 1 under Properties-Advance tab of
   class 1).
   
   Workplace:
   a) Logon to Workplace-Author-Advance-Search Designer.
   b) Create a search template by:
      1) Select both OS under Object Stores tab
      2) Select the document class from OS 1 (i.e: class1) under
   the Object Type Tab
      3) Select the choice list from the document class of OS 2
   (ie.: class2) and the string property from the document class of
   OS 1 (ie.: class1) under the Search Criteria tab.
      4) Saving the search template will generate the error.
