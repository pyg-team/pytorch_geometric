LOCAL FIX
 *  Turn off activity tracing.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   Users using async consume with MQ activity trace and getting
   messages of length > 8192 bytes
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   This problem occurred because the channel process was
   incorrectly attempting to dereference a NULL pointer which
   caused a memory exception when generating activity trace.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The MQ code has been modified to prevent the memory exception.
