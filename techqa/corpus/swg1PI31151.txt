APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If number of objects exceeds the List View Page Size setting,
   then hitting NEXT will cause the List View pane ?resizer? to
   disappear.
   Hitting PREV  will not restore the ?resizer?.
   
   Prerequisites:
   
   OP 7.0.0.2  Solutions
   1.Log into OpenPages APP as an Administrator
   2. Change Profile to OpenPages Modules 7.0.0 Master
   a)     On top menu bar, click Administration > Users, Groups an
   Domains
   b)     Click on the System Administrator 
   OpenPagesAdministrator
   c)      Click the Edit button and select OpenPages Modules 7.0.0
   Master in the Profile dropdown field and click Save button
   
   3. Create a new Entity and child Processes
   a)       Using the Navigation bar, select "Organization" and
   click on "Business Entities".
   b)       Click on the "Add new..." icon.
   c)       Enter "ENT01" in the "Name:" field, and click on the
   "Create" button
   d)       Click on ENT01 and then click on Processes under
   Associations on left pane
   e)       Click Actions > Add a new Process
   f)        Enter P1 in the name field and fill in the required
   fields (description, process owner), and click Save button
   g)       Repeat steps e and f, 5 more times  to create
   additional processes named P2 thru P6 instead of P1
   
   4. Modify List View Page Size to 5 in the Administration Setting
   a)       On top menu Navigate to Administration > Settings
   b)       Expand folders to OpenPages > Applications > GRCM >
   List View
   c)       Click on Page Size and set the value to 5 and click the
   Save button
   
   Steps to Reproduce:
   
    Login to OpenPages App as OpenPagesAdministrator
    On top menu, click on Organization > Business Entities
    Click on ENT01 and click SOXProcess under Associations on left
   pane
    Lengthen the List View pane by click and holding the bottom
   middle line of the pane, moving the mouse down and releasing the
   left mouse button
    Click on the Next link adjacent to the Actions button
    Observe that List View pane reverts back to original size and
   that the bottom line of the pane window disappears and confirm
   that you are unable to resize the pane.
    Hit the Prev link adjacent to the Actions button
    Confirm that the bottom line of the List View pane is still
   missing
   
   Expected Results:
   
   Should be able resize the List View pane
   
   Actual Results:
   
   Bottom of List View pane disappears
   
   Error Message:
   
   None
   
   
    
   
   

LOCAL FIX
 *  Naviagate back to Home Page and enter the List View again
