Title: IBM Execute via Data Movement Service fails: DM-RCI-0110 CCL-RCI-0002 DPR-ERR-2014 DPR-ERR-2082 - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When attempting to run a simple fact build on an AIX server from Data Manager workbench on a Windows machine, Execute locally and Remobe execute work fine but Execute via Data Movement Service fails with an error message. Attempting to execute the published task on the AIX server via Cognos Connection causes the task stay in a pending state and never starts. 

SYMPTOM
Error 

'RCI_DMSRunSpecification' returned with 1 message 

1. ERROR 
DM-RCI-0110 Data Manager RCI Interface Error (RCI_SoapFaultException - caf) Failed to execute DataMovementTask. 

CCL-RCI-0002 The Soap Server Returned a Soap fault. 
The error messages returned from the WASP engine: 
DPR-ERR-2014 Unable to load balance the request because no nodes in the cluster are available, or no nodes are configured for the service: dataMovementService. 
DPR-ERR-2082 An error has occurred. Please contact your administrator. 
The complete error has been logged by CAF with SecureErrorID:(date):(time)#1 


ENVIRONMENT
64-bit BI server and 32-bit Data Manager on AIX with IBM Websphere application server.



RESOLVING THE PROBLEM
Rebuild all EAR files for the 32-bit install, the ServletGateway,p2pd and dsnet.ear. Redeploy and restart the 32-bit WAS Cognos Gateway and Cognos Application servers.
