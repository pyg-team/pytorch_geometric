APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using a subtraction sign in an Impact event reader filter
   
   e.g. FirstOccurrence > LastOccurrence - 60
   
   prevents event processing.  Errors in netcool.log
   
   com.micromuse.common.parser.PolicyException: Unhandled
   Exception: java.lang.ArithmeticException in policy:  at line: 1
   
   
    
   
   

LOCAL FIX
 *  Where possible, avoid using subtractions in your event reader
   filter.
   
   
    
   
   

PROBLEM SUMMARY
 *  Filter with subrtraction causes Reader not to work
   
   
    
   
   

PROBLEM CONCLUSION
 *  If Event Reader filter includes non impact operation such as:
   LastOcurrence > getdate - 60 , it throws an exception with
   invalid operands. getdate is an omnibus filter not Impact's. If
   the user want to do that, one has to add a property to avoid the
   check aganist the filter.
   impact.<readername>.parserestrictionfilter=false.
