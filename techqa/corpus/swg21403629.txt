Title: IBM ITCAM for WebSphere shows disconnected in TEP workspace server summary. - United States

Text:
CAMADALL TECHNOTE (TROUBLESHOOTING)

THIS DOCUMENT APPLIES ONLY TO THE FOLLOWING LANGUAGE VERSION(S):
 US English 

PROBLEM(ABSTRACT)
 ITCAM for WebSphere TEP workspace "WebSphere Agent" and workspace view "Application Servers Summary" shows status of disconnected for a server subnode name. 

SYMPTOM
In the TEP workspace "WebSphere Agent" the user will see a status of disconnected for a server subnode name but the WAS JVM will actually be running. 


CAUSE
User performed un-configure for the ITCAM data collector as a WAS App Server and then 

reconfigured the data collector for WAS Portal Server. The user did not do a restart of the JVM 
between these two activities. 


ENVIRONMENT
ITCAM for WebSphere Application Server Data Collector



DIAGNOSING THE PROBLEM
Looking at the TEPS console one of the WebSphere profiles is listed as a WebSphere App Server and the new functioning profile is listed as WebSphere Portal Server. 



RESOLVING THE PROBLEM
Delete the <hostname>yn.ctx file in the <ITMHOME> directory

 

PRODUCT ALIAS/SYNONYM
 ITCAM4AD ITCAMfAD
