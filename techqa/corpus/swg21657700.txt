Title: IBM AIX 6.1 UpdateInstaller Exit Code 64 installing TWS fixpacks - United States

Text:
was updi TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 TWS fixpack installation fail with the following error: possible error, 64, installing the updateinstaller. 

SYMPTOM
You can find the following into summary.log: 

Installing updateinstaller to destination ...
/software/tws/TWS_8.5.1/fp_4/AIX/./../AIX/eWAS/UpdateInstaller/install -silent -OPT silentInstallLicenseAcceptance="true" -OPT disableOSPrereqChecking="true" -OPT disableEarlyPrereqChecking="true" -OPT installLocation="/software/tws/UPDI" -OPT skipStartMenu=true -OPT traceFormat=text -OPT traceLevel=SEVERE
Please check for log files in the "updilogs" subdirectory of the current user's home directory. If no logs have been produced there, please try running the install command again, with this additional parameter: -is:javaconsole

Restoring lock file ...
Possible error, 64, installing the updateinstaller.
For more information, refer to log files in directory: /software/tws/UPDI/logs
ERROR: The script ended abnormally. The reason is: possible error, 64, installing the updateinstaller.
The script exit code is 64


RESOLVING THE PROBLEM
Here there are steps you need to to solve your issue.

1) MANUAL UPDI INSTALLATION (disabling the Disk Space Check) Using the workaround specified in the WAS technote at the following link:

http://www-01.ibm.com/support/docview.wss?uid=swg21456660 [http://www-01.ibm.com/support/docview.wss?uid=swg21456660]

1.a) Check that the application server is not running, using: 

ps -ef | grep java 

and if it is running stop it. 

1.b) Move to the TWS 851 FixPack installation media, at location: 

<FP_IMAGE>/AIX/eWAS/UpdateInstaller 

1.c) Run the WebSphere UPDI installation running the following command, as a single line, to install the UPDI disabling the disk space check: 

./install -is:javaconsole -OPT silentInstallLicenseAcceptance=true 
-OPT disableOSPrereqChecking=true -OPT disableEarlyPrereqChecking=true 
-OPT installLocation=/sys_apps_01/tws/maestro/UPDI -OPT 
skipStartMenu=true -OPT traceFormat=ALL -OPT disableDiskSpaceCheck=true 
-W calculatediskspaceInstallWizardBean.active=false -W 
checkprereqs.active=false -W 
prereqsfailedpanelInstallWizardBean.active=false -silent 


2) TWS INSTALLATION - WAS PAK INSTALLATION (disabling the Disk Space Check) 

2.a) Backup the <TWS_IMAGE_DIR>\tws_tools\updateBobcat.sh 

2.b) Edit the <TWS_IMAGE_DIR>\tws_tools\updateBobcat.sh 

2.c) Disable the prerequisite check by changing the following line: 


$UPDATEINSTALLER/update.sh -W 
maintenance.package="$UPDATEINSTALLER/maintenance/$PATCH_DIR.pak" -W 
product.location="$EWASROOT" -W update.type="install" -silent 

with the this one: 

$UPDATEINSTALLER/update.sh -OPT disableNonBlockingPrereqChecking="true"
-W maintenance.package="$UPDATEINSTALLER/maintenance/$PATCH_DIR.pak" -W
product.location="$EWASROOT" -W update.type="install" -silent 


2.d) Save the <TWS_IMAGE_DIR>\tws_tools\updateBobcat.sh 

2.e) Resume the TWS installation setting to SUCCESS the UPDI Installation and to READY the WAS Update step
