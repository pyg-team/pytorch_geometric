PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All users of WebSphere Message Broker V6.1   *
   *                 with HTTPInput or HTTPRequest nodes who      *
   *                 wish to send or receive HTTP Query Strings.  *
   ****************************************************************
   * PROBLEM DESCRIPTION: It is currently difficult for users     *
   *                      to url-decode HTTP QueryStrings after   *
   *                      they enter an HTTPInput node or         *
   *                      url-encode them before they are sent    *
   *                      our by an HTTPRequest node.             *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   It is currently difficult for users to url-decode HTTP
   QueryStrings after they enter an HTTPInput node or url-encode
   them before they are sent our by an HTTPRequest node.
   
   
    
   
   

PROBLEM CONCLUSION
 *  WebSphere Message Broker was modified to enhance HTTP Query
   string support.
   
   Topic ac04565 (HTTPInput Node) will be updated to include
   details of how to enable enhanced query string support
   on the HTTPInput node.
   
   Topic ac04595 (HTTPRequest Node) will be updated to include
   details of setting the outbound query string based
   on the settings in the LocalEnvrionment
   
   Target availability V6.1.0.4
