Title: IBM PK29441; 6.1.0.1: Repeated admc0001e messages into log files - United States

Text:
PK29441; PK29441; PK29441 DOWNLOADABLE FILES

ABSTRACT
 "java.io.NotSerializableException" messages are repeatedly written to server log. 

DOWNLOAD DESCRIPTION
PK29441 resolves the following problem:

ERROR DESCRIPTION: 
The following exception is filling up log files in WebSphereÂ® Application Server version 6.0.2.9 ND or Base environment : 

[6/19/06 9:34:19:216 PDT] 0000003e JMXObjectOutp E ADMC0001E: 
Unable to serialize object: com.sun.jndi.ldap.LdapCtx@2bbf257a 

[6/19/06 9:34:19:366 PDT] 0000003e GenericSerial E ADMC0008E: 
The system failed to marshal the object of type 

[Ljava.lang.Object;: com.sun.jndi.ldap.LdapCtx 

These errors are logged several times per second. 

LOCAL FIX: 
Ignore messages and increase file system space. 

PROBLEM SUMMARY

USERS AFFECTED:
Application Server version 6 users.

PROBLEM DESCRIPTION:
java.io.NotSerializableException repeatedly written to server log.

RECOMMENDATION:
None

Notification code receives a java.io.NotSerializableException while attempting to send a message. It then queues the message to try and send again. This results in repeated java.io.NotSerializableException entries in the server log:

java.io.NotSerializableException: com.sun.jndi.ldap.LdapCtx
at java.io.ObjectOutputStream.writeObject0(ObjectOutpu
tStream.java(Compiled Code))
at java.io.ObjectOutputStream.defaultWriteFields(Object
OutputStream.java(Inlined Compiled Code))
at java.io.ObjectOutputStream.writeSerialData(ObjectOutput
Stream.java(Compiled Code))
at java.io.ObjectOutputStream.writeOrdinaryObject(ObjectOu
tputStream.java(Compiled Code))
at java.io.ObjectOutputStream.writeObject0(ObjectOutputStre
am.java(Compiled Code))
at java.io.ObjectOutputStream.defaultWriteFields(ObjectOutp
utStream.java(Compiled Code))
at java.io.ObjectOutputStream.defaultWriteObject(ObjectOutp
utStream.java(Compiled Code))
at java.lang.Throwable.writeObject(Throwable.java:679)

PROBLEM CONCLUSION: 
The notification code has been updated to not retry sending the message if it results in a 
java.io.NotSerializableException exception. 

The fix for this APAR is currently targeted for inclusion in Fix Packs 6.0.2.15 and 6.1.0.4.
