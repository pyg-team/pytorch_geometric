Title: IBM IC78008: MQ QUEUE MANAGER OBJECT WITHOUT A REQUEST QUEUE MIGHT GO DOWN UNDER HIGH LOAD USING URL-OPEN - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If you set up a Multi-Protocol Gateway with HTTP front side
   handler and a stylesheet to create connections
   by using url-open to retrieve messages from an MQ ReplyQueue,
   then if the concurrent connections reach the total connection
   limit for an MQ Queue Manager (QM), the state of the MQ QM
   object might go down.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Affected is the use of a MQ QM object without a request queue.
   
   The MQ QM object might go down when it reaches the connection
   limit.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix is available in 3.8.1.16, 3.8.2.7, 4.0.1.4 and 4.0.2.1
   
   
    
   
   

TEMPORARY FIX
