 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Error Message: EJB invocation throws a ClassCastException when a
   non serializable element packed in an object array.Test expects
   a java.io.NotSerializableException whereas ORB throws
   ClassCastException: Class incompatible with java.io.Serializable
   .
   Stack Trace: java.lang.ClassCastException:
   suite.r80.base.asynch.errRemote.shared.Helper incompatible with
   java.io.Serializableat com.ibm.rmi.io.***.write(***:48)at
   com.ibm.rmi.iiop.IDRWriter.fastWriteValInternal(IDRWriter.java:1
   011)at
   com.ibm.rmi.iiop.IDRWriter.fastWriteVal(IDRWriter.java:1046)at
   com.ibm.rmi.iiop.IDRWriter.write_value(IDRWriter.java:636)at
   com.ibm.rmi.iiop.IDRWriter.write_value(IDRWriter.java:674)at
   com.ibm.rmi.iiop.IDRWriter.write_value(IDRWriter.java:627)
   .
   N/A
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  The ORB does not verify whether the class implements the
   java.io.Serializable interface and hence throws incorrect
   exception.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This APAR will be fixed in the following Java Releases:
      7 R1 SR1       (7.1.1.0)
   .
   The solution is to verify that the class implements
   java.io.Serializable interface, if not throw
   java.io.NotSerializableException as expected.
