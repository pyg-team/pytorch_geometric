APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After defining new the BigFix datasource in BFI with some
   existing data and some custom analysis whose content size is
   closed to 2MB, during first
   ETL import, the BigFix database may hit the Disk out of Space
   issue. Once can see a similar error message in the BFI tema.log:
   2015-12-15 00:11:46 (+0:06:10.692) INFO: ETL from IEM CATE -
   RawDatasourceAnalysis              (0x000000000865BB55 -
   0x0000000008E9C4EF): Failed
   2015-12-15 00:11:46 (+0:00:00.068) ERROR: Sequel::DatabaseError:
   Java::ComIbmDb2JccAm::SqlException: The file system is full..
   SQLCODE=-968, SQLSTATE=57011, DRIVER=3.64.104
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * When you add a new data source to BigFix Inventory and the
   * data source contains custom analyses whose content size is
   * close to 2MB, the BigFix database might run out of space
   * during the first import.
   * EXPECTED RESULTS:
   * The problem does not occur.
   ****************************************************************
   * RECOMMENDATION: Apply appropriate update when available.
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this APAR is contained in the following maintenance
   packages:

    | Update | ILMT 9.2.3.0
