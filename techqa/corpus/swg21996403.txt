Title: IBM Why do I see IBM Content Manager timestamps displayed in GMT instead of local time in IBM Content Navigator? - United States

Text:
ICMGMTSYSATTRTS; ICMGMTUSERATTRDATE; ICMGMTUSERATTRTIME; cmbicmsrvs TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Why do I see IBM Content Manager timestamps displayed in GMT instead of local time in IBM Content Navigator? 

CAUSE
The following parameters in cmbicmsrvs.ini file under .../navaigator.war/WEB-INF/classes/connectors directory determines whether a Content Manager timestamp attribute is displayed in GMT or in local time.
ICMGMTSYSATTRTS=
ICMGMTSYSATTRDATE=
ICMGMTUSERATTRTS=
ICMGMTUSERATTRDATE=
ICMGMTUSERATTRTIME=



ENVIRONMENT
IBM Content Navigator V2.0.3
IBM Content Manager V8.5
IBM Content Collector V3.0

RESOLVING THE PROBLEM
The following parameters in cmbicmsrvs.ini file under .../navaigator.war/WEB-INF/classes/connectors directory determines whether a Content Manager timestamp attribute is displayed in GMT or in local time.
ICMGMTSYSATTRTS=
ICMGMTSYSATTRDATE=
ICMGMTUSERATTRTS=
ICMGMTUSERATTRDATE=
ICMGMTUSERATTRTIME=

1. Make sure these parameters are part of the cmbicmsrvs.ini file, add any missing parameters to the list.





2. If the intended setting is to display the timestamp property in GMT set the values for all the parameters listed to 'FALSE' OR if the intended setting is to display the timestamp property in local time, set the values for all the parameters listed to 'TRUE'. 


3. Restart the application server hosting IBM Content Navigator.
