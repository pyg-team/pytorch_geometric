Title: IBM FixPack4 install fails with Invalid Password - United States

Text:
ncp_db_access.pl; libpam.so TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Though GA version was installed without any issues, FixPack4 upgrade fails with 'Invalid Password' error in connecting to NCIM database - how to fix it? 

CAUSE
Missing library. 

ENVIRONMENT
ITNM Core, TIP & DB2 database running on 3 different hosts.

DIAGNOSING THE PROBLEM
If you run ncp_db_access.pl script, you would end up with following error:


Level: info Msg: Attempting to connect to database (DBI:DB2:DATABASE=PITNM;HOSTNAME=db2host.ibm.com;PORT=50000;PROTOCOL=TCPIP)...
install_driver(DB2) failed: Can't load '/opt/IBM/tivoli/netcool/precision/perl/lib/site_perl/5.8.8/i686-linux-thread-multi/auto/DBD/DB2/DB2.so' for module DBD::DB2: libpam.so.0: cannot open shared object file: No such file or directory at /opt/IBM/tivoli/netcool/precision/perl/lib/5.8.8/i686-linux-thread-multi/DynaLoader.pm line 23 


RESOLVING THE PROBLEM
Install 32bit version of libpam module and re-run the install.
