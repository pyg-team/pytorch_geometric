Title: IBM IIB or WMB Startup trace for biphttplistener - United States

Text:
biphttplistener; startup trace; trace TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Procedure to collect the startup trace of biphttplistener in IBM Integration Bus (IIB) or WebSphere Message Broker (WMB). 

RESOLVING THE PROBLEM



STARTUP TRACE INSTRUCTIONS FOR WINDOWS/UNIXES

 * 
 * 
 *  Note: The broker needs to be running to enable the HTTPListener trace override property.  
   mqsichangeproperties brokerName -b httplistener -o HTTPListener 
   -n traceOverrideLevel -v debug
   
   Shutdown and restart the broker to pick up changes.
   Recreate the problem, then issue the following commands to process the trace records. 
   
   mqsireadlog brokerName -t -b httplistener -f -o listenertrace.xml
   
   mqsiformatlog -i listenertrace.xml -o listenertrace.out
   
   If possible, review listenertrace.out to confirm that issue was captured. 
   Issue the following command to disable active trace setting. 
   
   mqsichangeproperties brokerName -b httplistener -o HTTPListener 
   -n traceOverrideLevel -v none
   
   Shutdown and restart the broker to pick up changes. 
   
   
   



STARTUP TRACE INSTRUCTIONS FOR Z/OS

 * 
 * 
 *  Note: The broker needs to be running to enable the HTTPListener trace override property.  
   Edit and submit the BIPCHPR job with the following flags/settings:
   
   -b httplistener -o HTTPListener -n traceOverrideLevel -v debug
   
   Shutdown and restart the broker to pick up changes.
   Recreate the problem, then run the following jobs using the specified flags to process the captured trace records. 
   
   BIPRELG job - read trace records
   
   -t -b httplistener -f -o listenertrace.xml
   
   BIPFMLG job - format trace records
   
   -i listenertrace.xml -o listenertrace.out
   
   If possible, review listenertrace.out to confirm that issue was captured. 
   Edit and submit the BIPCHPR job with the following flags/settings:
   
   -b httplistener -o HTTPListener -n traceOverrideLevel -v none
   
   Shutdown and restart the broker to pick up changes.
