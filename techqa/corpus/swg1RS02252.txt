 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When concurrently executing a ruleset and requesting a XU Dump,
   it may randomly happends that
   - deadlock leads to threaddumps similar to :
         thread: Thread-7 id: 0x1abf7200 (owns monitor for
   com/ibm/rules/res/xu/engine/cre/internal/CREManager object :
   0x4820993de0) waiting for =>
         thread: Thread-27 id: 0x1acc9800 (owns monitor for
   ilog/rules/res/xu/ruleset/internal/IlrRulesetUsageInformationImp
   l object : 0x481b94de28) waiting for =>
         thread: Thread-7 id: 0x1abf7200 (owns monitor for
   com/ibm/rules/res/xu/engine/cre/internal/CREManager object :
   0x4820993de0)
   - ruleset execution requests are blocked
   - connection pool overflow
   
   
    
   
   

LOCAL FIX
 *  Avoid XU dump request
   
   
    
   
   

PROBLEM SUMMARY
 *  Use of synchronized method and structure leads to a deadlock
   when the XU  build the xu dump
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code is fixed
