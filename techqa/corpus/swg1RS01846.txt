ERROR DESCRIPTION
 *  When starting ODM Studio with an ODM useris which is already
   logged-in, the second login attempt with force-logoff option
   may fail when the dataserver response time is long (e.g. over
   4s)
   
   
    
   
   

LOCAL FIX
 *  This issue is specific to the ODM Studio startup sequence and
   does not affect the API, a manual workaround is to force the
   user logoff using a separate java program that uses the ODM API
   to login the userid and then logoff to reset the session.
