APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using Impact(6.1.1) to pull the data from ITM(6.2.3) data
   provider via UI data provider DSA in Dash.The TEPS server swap
   increase is caused by getting itcam data vi impact ui data
   provider.
   CURI will cache dataset instance for performance improvement and
   consistent pagination.
   
   http get and delete request need to be in pair, get will create
   a dataset instance(when bring up a page), delete will destroy a
   dataset instance(when close page).
   
   In the TEPS server log,we can view the get code and can't find
   delete code while using Impact to get data from ITM data
   provider before displaying to DASH.
   
   We used impact function getbyfilter to get data in UI data
   provider.
   What function can we use to delete request to avoid TEPS memory
   leak ?
   
   
    
   
   

LOCAL FIX
 *  n/a
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Uses impact function getbyfilter to get data in UI data      *
   * provider.  CURI will cache dataset instance for performance  *
   * improvement and consistent pagination.                       *
   * TEPS UIDP is not clearing cache, therefore memory leaks.     *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The GetByFilter function in a policy to request both GET than
   DELETE within the same function call.
   The DELETE action will be only executed if delete=true is on the
   query string.
   
   The fix for this APAR is contained in the following
   maintenance packages:
   | Fix Pack | 6.1.1-TIV-NCI-FP0002
