Title: IBM FileNet Image Services Resource Adapter (ISRA) error: java.util.MissingResourceException. Can't find bundle for base name - United States

Text:
ISRA 2.0; BEA WEBLOGIC 7.0; JAVA.UTIL.MISSINGRESOURCEEXCEPTION JAVA.UTIL.MISSINGRESOURCEEXCEPTION: C TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 FileNet Image Services Resource Adapter (ISRA) 2.0 error: java.util.MissingResourceException. Can't find bundle for base name 

RESOLVING THE PROBLEM
FileNet Image Services Resource Adapter (ISRA) error: java.util.MissingResourceException. Can't find bundle for base name. 


The client received this error when starting ISRA through BEA WebLogic. On startup an exception is thrown concerning the ISRA:

<Feb 25, 2003 1:25:04 PM CET> <Notice> <WebLogicServer> <000327> <Starting WebLogic Admin Server "necl011" for domain "filenet">
java.util.MissingResourceException: Can't find bundle for base name com.filenet.is.ra.util.FNISRAMessages, locale en_US
at java.util.ResourceBundle.throwMissingResourceException ResourceBundle.java:707)
at java.util.ResourceBundle.getBundleImpl(ResourceBundle.java:678)
at java.util.ResourceBundle.getBundle(ResourceBundle.java:554)
at com.filenet.is.ra.util.FN_IS_MessageImpl.<clinit>(Unknown Source)
at java.lang.Class.forName0(Native Method)
at java.lang.Class.forName(Class.java:115)
at com.filenet.is.ra.util.FN_IS_LocaleException.<clinit>(Unknown Source)

at com.filenet.is.ra.spi.FN_IS_SpiManagedConnectionFactory.createManaged
Connection(Unknown Source)
at weblogic.connector.common.internal.ConnectionPool.makeResources(Conne
ctionPool.java:986)

The solution (From Release Notes) is: 


Put the ISRA.jar file in server's CLASSPATH environment variable. 

To do this, edit the file <WebLogic Installation Directory>\weblogic700\server\bin\startWLS.cmd to include ISRA.jar in the CLASSPATH environment variable. 

Restart the server.
