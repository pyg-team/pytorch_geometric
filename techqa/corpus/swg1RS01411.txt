ERROR DESCRIPTION
 *  When augmenting WAS profile with the Decision Server Event and
   the message below is observed:
   
   "INSTCONFPARTIALSUCCESS: Profile augmentation succeeded, but
   some
   non-fatal actions failed. For more information, consult
   /opt/ibm/WAS/logs/manageprofiles/AppSrv01_augment.log"
   
   From the augment log, this is noted:
   Cannot locate
   /opt/ibm/WAS/profileTemplates/wbe/default/../defaulters/wbeMqMsg
   ingTransTypeDefaulter.ijc but execution can continue
   
   In
   <ODM801>\ODM\config\wbeProfileTemplate\wbe\default\actionRegistr
   y.xml that it consists of the following in augmenting WBE
   Messaging Configuration:
   <argument key="wbeMqMsgingTransType" values="1"
   required="false">
   <defaulter
   path="../defaulters/wbeMqMsgingTransTypeDefaulter.ijc"
   priority="160" />
   <validator
   path="../validators/wbeMqMsgingTransTypeValidator.ijc" />
   </argument>
   
   Under both defaulters and validators folders, the two ijc do not
   exist.
   
   The same augment declaration is found in the other two files:
   <ODM801>\ODM\config\wbeProfileTemplate\wbe\managed\actionRegistr
   y.xml
   <ODM801>\ODM\config\wbeProfileTemplate\wbe\management\actionRegi
   stry.xml
   
   
    
   
   

LOCAL FIX
 *  1. The wbeMqMsgingTransTypeDefaulter.ijc is there to enable
   the capability of specifying a default value ("BINDINGS") for
   command manageprofile's "-wbeMqMsgingTransType" parameter when
   "-wbeMsgingType MQ_JMS_Messaging" is specified and
   "-wbeMqMsgingTransType" is not provided. You can always bypass
   the problem by specifying a value for the
   "-wbeMqMsgingTransType" parameter.
   
   2. The wbeMqMsgingTransTypeValidator.ijc is there to enable
   the capability of validating the value of manageProfile's
   "-wbeMqMsgingTransType" parameter when "-wbeMsgingType
   MQ_JMS_Messaging" is specified. As long as you use
   "Default_Messaging", "MQ_JMS_Messaging", or "No_Messaging" as
   the value for "-wbeMqMsgingTransType", you will be fine.
