Title: IBM Impact Console Integration causes blank page DASH - United States

Text:
DASH; JazzSM; Impact; Console Integration; 7.1; Blank Page TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After having added a console integration to see the Impact 7.1 GUI, users login into DASH see a blank page and have no access to any menus nor see any data 

SYMPTOM
Blank page in DASH for any user


CAUSE
Not fully established but clearly related to the console integration

ENVIRONMENT
DASH 1.1.0.3 and Impact 7.1 FP6

DIAGNOSING THE PROBLEM
log files (SystemOut.log) show nullpointer exceptions for servlet


[8/17/16 11:18:41:898 BST] 00000078 servlet E com.ibm.ws.webcontainer.servlet.ServletWrapper service SRVE0014E: Uncaught service() exception root cause About: javax.servlet.ServletException: javax.portlet.PortletException: java.lang.NullPointerException


RESOLVING THE PROBLEM
use consolecli.sh to find out the details of the console integration 

cd /opt/IBM/JazzSM/ui/bin
[netcool@jazzsmnonroot bin]$ ./consolecli.sh ListConsoleIntegrations --username jazzsm --password netcool
Console Integration ID : Federation.1471426451534
Console Integration Name : Impact71
Console Integration URL : protocol (HTTP or HTTPS) impactUIhost:impactUIPort/ibm/console/rest 
[netcool@jazzsmnonroot bin]$ ./consolecli.sh DeleteConsoleIntegration --username jazzsm --password netcool --id Federation.1471426451534
CTGWA4017I The command completed successfully.

Once the console integration is deleted connecting to DASH displays the default initial page for the user
