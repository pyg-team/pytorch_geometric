   - Examples of things to document:
   
   To use Windows authentication - add ;integratedSecurity=true to
   the database name, via the GUI
   
   
   i.e.  the ds file will look like this.
   
   <Data
   source>.MS-SQLServer.BACKUPDATABASE=database;integeratedSecurity
   \=true
   
   or use the GenericSQL DSA.
   
   JDBCDRIVER=com.microsoft.sqlserver.jdbc.SQLServerDriver
   
   and
   
   1) Integrated security is supported by the jdbc on windows
      operating system only. That is using
             integratedSecurity=true.
       If thisoption is used, it looks for the sqljdb_auth.dll file
       in the library path.
       That was why I mentioned, it was dll and can't be seen by
       non-windows operating systems.
   
   
   2) If you want to use integrated security with any other
      operating system that is supported by MSSQL JDBC (in our
      case Linux), another  option has to be added which is
          authenticationScheme=JavaKerberos
      This options tells MSSQL JDBC that it's not windows and
      should use kerberos instead.
