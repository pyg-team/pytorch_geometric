Title: IBM IZ22327: MANUALLY ENTERED GL ACCOUNT VALUE IN ASSETS GETS NULLED OUT WHEN A LOCATION VALUE IS SELECTED. DOES NOT HAPPEN WITH GLNAVIGATOR - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Manually entered Gl Account value in Assets gets nulled out when
   a location value is selected. Does not happen with glnavigator.
   
   Steps:
   Launch App Designer and select Asset application.
   Add a textbox for a GLACCOUNT attribute.
   Enter GLNAVIGATOR in the lookup property.
   Save.
   In Assets enter a new Asset record and manually enter a known
   valid GL Account.
   Enter a Location value of BR230 and tab out.
   GL Account field is nulled out.
   
   If instead of manually entering the GL Account in the first step
   you select it using the GL Navigator, the value remains when a
   location value is entered.
   Client is also reporting that this happens when a Failure Class
   is selected.
   
   
    
   
   

LOCAL FIX
 *  Use GL Navigator

PROBLEM CONCLUSION
 *  1.In Assets application created a new Asset record and manually
   entered a known valid GL Account (6200-350-SAF).

   Account field is not nulled out.
   
   3.Instead of manually entering the 'GL Account ' in the first
   step, selected the GLACCOUNT using the lookup(GL Navigator),
