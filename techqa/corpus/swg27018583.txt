Known Issues 
Problem: Users may get OutOfMemory exception for WebSphere Insurance Content Pack while 
deploying Common Services / Solution Scenario PI files in WebSphere Integration Developer 

Solution: Please increase MaxHeapSize to 1300 using following steps and retry deployment. 

To set the heap size on Distributed Platforms:

1. Log on to the WebSphere Process Server administrative console.
2. Click Servers > Server Types > WebSphere application servers, and then click server1 in the navigation pane.
3. Go to (+) Java and Process Management > Process Definition > Java Virtual Machine.
4. Set the Initial Heap Size to 512 and the Maximum Heap Size to 1300.
5. Click Apply, and then Save to save your changes to the master configuration.

Note: To enable these settings, restart WebSphere Process Server. 

Problem: Users may get Stack Overflow before importing any optional jar files for a Content Pack. 

Solution: Apply the following setting in JVM of WebSphere Process Server to increase maximum Java thread stack size to 1 mega byte. 

To set the Maximum Java thread stack size: 

1. Log on to the WebSphere Process Server administrative console.
2. Click Servers > Server Types > WebSphere application servers, and then click server1 in the navigation pane.
3. Go to (+) Java and Process Management > Process Definition > Java Virtual Machine.
4. Set the Generic JVM arguments to -Xss 1m
5. Click Apply, and then Save to save your changes to the master configuration.

Note: To enable these settings, restart WebSphere Process Server.
