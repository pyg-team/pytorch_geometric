APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After an appliance restart, the ISAM Client may not come up.
   The logs may show the following error:
   
   [0x81a00006][tam][error] tam(dp_sstam): tid(4347135): IBM
   Security Access Manager authorization client log message: IBM
   Security Access Manager server error: azn_initialize :
   HPDCO1053E   Could not bind socket to port (0, 8).
   
   
    
   
   

LOCAL FIX
 *  Try one of the following as a workaround:
   1. Disable ISAM objects before an appliance reload.
   2. Reboot appliance after appliance reload.
   
   
    
   
   

PROBLEM SUMMARY
 *  The fix changes how the ISAM component is restarted when the
   firmware is restarted.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix is available in 7.0.0.17, 7.1.0.14, 7.2.0.11, 7.5.0.5,
   7.5.1.4 and 7.5.2.2
   
   For a list of the latest fix packs available, please see:
   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
