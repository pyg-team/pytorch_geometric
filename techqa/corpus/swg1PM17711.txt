Title: IBM PM17711: FAILED TO CREATE CANDIDATE FOR HP JDK ON MEMORY LEAK PAGE OF VE - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Installed and configured ITCAM for J2EE v6.1.0.4 J2SE DC on a
   J2SE application.
   On VE console select PROBLEM DETERMINATION->MEMORY
   DIAGNOSIS->Memory Leak
   Select Group :LY, Server: h1.d1.com.J2SE.8110(L2), then press OK
   button.
   Create a New Candidate Wait Time enter 3 minutes,then press save
   button.
   Then on this page, an error message is popped up, "CYNVE0739E:
   This feature is not supported for Sun JDK and is disabled by
   default"
   Check Existing Candidates,the candidate has been created.
   
   The error message is wrong about JDK vendor. The candidate
   should not be added if the JDK is not supported.
