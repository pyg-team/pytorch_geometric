INSTALLATION INSTRUCTIONS
Note: You do not need to download the compressed files that contain the update; IBM Installation Manager downloads and installs the update directly from the IBM update repository.



If you download and install this fix pack locally, before you start the installation process, you must add the repository location for the fix pack in IBM Installation Manager. For details, see the Download Package and Add repository location for fix pack sections.

DOWNLOAD PACKAGE
If you are performing a local installation of the fix pack, you can download the fix pack by using either Download Director or FTP from the table below. If you cannot use either of these methods, download the fix pack by using HTTP from the following URL: 

http://download.boulder.ibm.com/ibmdl/pub/software/rationalsdp/v75/rbd/7512/zips/ [http://download.boulder.ibm.com/ibmdl/pub/software/rationalsdp/v75/rbd/7512/zips/]

After the download is complete, extract the compressed files in the appropriate directory. 


Adding the repository location for the fix pack

To add the fix pack repository location to IBM Installation Manager:


 1. Start IBM Installation Manager. 
 2. On the Start page of Installation Manager, click File > Preferences > Repositories. 
 3. On the Repositories page, click Add Repository. 
 4. In the Add repository window, browse to or enter the file path to the diskTag.inf file. This file is located in the disk1 sub-directory in the directory where you extracted the compressed files. Click OK. 
 5. To close the Preference page, click OK.

Support Information Before you contact IBM Rational Software Support, gather the background information that you need to describe the problem. When you describe a problem to an IBM software support specialist, be as specific as possible and include all relevant background information so that the specialist can help you solve the problem efficiently. To save time, know the answers to these questions:

 * What software versions were you running when the problem occurred?
 * Do you have logs, traces, or messages that are related to the problem?
 * Can you reproduce the problem? If so, what steps do you take to reproduce it?
 * Is there a workaround for the problem? If so, be prepared to describe the workaround.
