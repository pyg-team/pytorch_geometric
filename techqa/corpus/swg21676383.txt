Title: IBM Unable to launch the Graphic Process Modeler (ProcessModeler.jnlp) when using a Java 7 JDK/JRE - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Java Webstart (javaws) is attempting to launch the IBM Sterling B2B Integrator Graphic Process Modeler with a Java 7.x JDK/JRE and is throwing an Application Error. "Unable to launch the application 

SYMPTOM
Unable to Launch the GPM. The follow exception is being thrown. 

java.lang.SecurityException: Missing required Permissions manifest attribute in main jar: http://11.11.11.11:10000/gbm/pmodeler/FlowBuilder.jar
at com.sun.deploy.security.DeployManifestChecker.verifyMainJar(Unknown Source)
at com.sun.deploy.security.DeployManifestChecker.verifyMainJar(Unknown Source)
at com.sun.javaws.Launcher.doLaunchApp(Unknown Source)
at com.sun.javaws.Launcher.run(Unknown Source)
at java.lang.Thread.run(Unknown Source)


CAUSE
Increased security settings in the Java 7 Control Panel prohibit Java Webstart from launching the application


ENVIRONMENT
Loading the Graphic Process Modeler on a Windows Client with Java 7.x JDK/JRE 



RESOLVING THE PROBLEM
From the Windows Control Panel, open the Java Control Panel 

Select the "Security" Tab
Select "Edit the Site List" 
Select "Add" and add the IBM Sterling B2B Integrator URL,
Select "OK"
Select "Continue" in the "Security Warning - HTTP Location" Pop-up
Select "OK" one more time to save the changes

Another option is to lower the Security Slider Bar to Medium and it will allow all Java Applications to run after presenting a security prompt.

If adding a security exception is not a possibility, then the following article can be used to force the GPM to use a 1.6 JRE:

RELATED INFORMATION
 Start with 1.6 JRE when both 1.7 and 1.6 installed [http://www-01.ibm.com/support/docview.wss?uid=swg21613292]
