ERROR DESCRIPTION
 *  In the FileNet Enterprise Manager, an error will come up when
   adding a property definition to the "property definitions"
   tab of a class.
   The steps to reproduce the issue are as follow:
   1. Logon the FEM and open an object store
   2. Create a property template
   3. Add the property template into the "property definition"
   tab of the root Document class
   4. Click apply and propagate button.
   5. The error comes up.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  P8 users using Enterprise Manager (3.5, 4.0, 4.0.1)Problem occur
   
   
    
   
   

PROBLEM CONCLUSION
 *  Problem is fixed in P8 4.0.1 FP2 by a dialog that asks the user
   if they want to apply any changes made to the class definitions.
    Enterprise Manager will then call Save() on the object before a
   ttempting to propagate changes to subclasses.
