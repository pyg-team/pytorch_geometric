APAR STATUS
 * CLOSED AS DOCUMENTATION ERROR.
    
   
   

ERROR DESCRIPTION
 *  RAA Information Center mentions Rollback function of
   Installation Manager as being usable with RAA, but in fact, the
   rollback functionality is not supported by RAA installation.
   
   
    
   
   

LOCAL FIX
 *  Use Installation Manager to Uninstall (after backing up
   contents of <RAA Installation Directory>/config , and then use
   Installation Manager to install desired level of RAA code.
   Uninstalling and reinstalling RAA using Installation Manager
   will not remove the RAA database from DB2.
   
   
    
   
   

PROBLEM SUMMARY
 *  RAA documentation indicated Roll back feature of Installation
   Manager could be used to revert to previous RAA code level, but
   that is incorrect.  Installation Manager Roll back is not
   supported by RAA.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Removed topic "Reverting updates to previous version".  Updated
   "Installation Manager overview" topic -added note that
   "rollback is not available for RAA".
