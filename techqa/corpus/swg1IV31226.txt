APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After clearing the state of an event reader and restarting it,
   sometimes, it starts reading events where StateChange is greater
   than some non-zero number (it should start reading events where
   StateChange > 0).
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * After clearing the state of EventReader and restarting it,   *
   * the EventReader does not pick up events from StateChange     *
   * &gt; 0, rather some other recently processed StateChange     *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   When the EventReader was shut down prior to clearing the state,
   some events had been processed but not &quot;checkpointed&quot;,
   meaning the eventreader hadn't written to its file that the
   StateChange of the events was finished. When the EventReader was
   restarted, Impact wrote the old checkpoint to the .statechange
   file and read in it for its first query.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Don't write out a checkpoint to file after a Clear State was
   issued, before the EventReader makes its first query.<br/>The
   fix is contained in 511 FP2
   
   
    
   
   

TEMPORARY FIX
