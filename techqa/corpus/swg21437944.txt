PROBLEM(ABSTRACT)
 You are using the CICS Explorer and are connecting to your CICSPlex SM (CPSM) Web User Interface (WUI) server CMCIPORT for the first time. The connection fails with message IZE0105E or CNX0104E. Your CPSM WUI server MSGUSR contains messages DFHWB0114, DFHWB0727, and DFHAC2236 with an abend AWB2 in DFHWBXN. 

SYMPTOM
Your CICS Explorer displays the following message text in the status area: 

IZE0105E or CNX0104E Connection failed with exception org.xml.sax.SAXParseException: The markup in the document preceding the root element must be well-formed.

Your CPSM WUI server MSGUSR has the following messages:

DFHWB0114 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/DFHWB/DFHWB0114.html] yyyy/mm/dd hh:mm:ss applid CWXN A non-HTTP request has been received by an HTTP service. The request has been rejected. Host IP address: nn.nn.nn.nn. Client IP address: nn.nn.nn.nnn. TCPIPSERVICE: EYUCMCIT 
DFHWB0727 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/DFHWB/DFHWB0727.html] yyyy/mm/dd hh:mm:ss applid CWXN CICS Web Interface attach processing could not attach the requested alias transaction CWWU. Userid: userid. Host IP address: nn.nn.nn.nn. Client IP address:nn.nn.nn.nnn. TCPIPSERVICE: EYUCMCIT
DFHAC2236 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/DFHAC/DFHAC2236.html] yyyy/mm/dd hh:mm:ss applid Transaction CWXN abend AWB2 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/AWxx_abend_codes/AWB2.html] in program DFHWBXN term ????. Updates to local recoverable resources will be backed out.


CAUSE
The GRPLIST in the SIT did not contain the new CICS TS 4.1 or later RDO group DFHWU.


ENVIRONMENT
CICS TS V4.1 or later; CICSPlex SM V4.1 or later WUI Server using CMCI;



DIAGNOSING THE PROBLEM
The majority of IZE0105E or CNX0104E connection failure messages can be resolved by examining the CICS TS MSGUSR logs for CICS errors related to attaching the CWXN or CWWU transaction identifiers.

Here is a sample CICS Explorer log file showing the exception for the CNX0104E -> 
.log [/support/docview.wss?uid=swg21437944&aid=2] .log [/support/docview.wss?uid=swg21437944&aid=1]



RESOLVING THE PROBLEM
Upgrade your DFHCSD [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.migration32.doc/dfhe5_rdo_csd_upgrade.html] to use the CICS TS V4.1 or later DFHLIST which includes the new RDO group DFHWU [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.migration32.doc/dfhe5_rdo_new_ibm_dfhwu.html]. 

In addition to DFHWU, the other RDO groups new to CICS TS 4.1 or later are DFHRL [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.migration32.doc/dfhe5_rdo_new_ibm_dfhrl.html], DFHRS [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.migration32.doc/dfhe5_rdo_new_ibm_dfhrs.html], and DFHWEB2 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.migration32.doc/dfhe5_rdo_new_ibm_dfhweb2.html].

The WUI server MSGUSR will contain the following messages if the DFHWU RDO group was in the GRPLIST:
DFHXM0101 yyyy/mm/dd hh:mm:ss applid userid CSSY TRANSACTION definition entry for CWWU has been added.
DFHAM4893 Iyyyy/mm/dd hh:mm:ss applid Install for group DFHWU has completed successfully.

See related item [http://www.ibm.com/support/docview.wss?uid=swg21438547] if the CWXN and CWWU transactions exist but you receive the DFHWB0361 for a security failure,

If you are still receiving errors, see the MustGather for CICS Explorer errors and incorrect output [http://www.ibm.com/support/docview.wss?uid=swg21385861] to determine what documentation to gather before contacting CICS Support.
