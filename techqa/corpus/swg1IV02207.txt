 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using a Filter of the form:
   
   FieldA not like 'ABC' AND FieldB != ''
   
   produces an error in the Filter Analyzer although the
   EventReader does work,
   but one does have to apply the two EventReader properties:
   
   impact.[EventReaderName].consolidatefilters=false
   impact.[EventReaderName].uselikeoperator=true
   
   first the get the EventReader to run with such a Filter and then
   to get it to use the Filter in its SQL select statement.
   
   The Filter Ananlyzer error is of the format:
   
   Filter contains the following syntax error
   FieldA not like 'ABC' AND FieldB != '' : rule includes an
   unsupported expression
   
   
    
   
   

LOCAL FIX
 *  N/A - The EventReader and Filter work fine with the application
   of the properties:
   
   impact.[EventReaderName].consolidatefilters=false
   impact.[EventReaderName].uselikeoperator=true
   
   Just the Filter Analyzer will report this error - it will not
   adversely affact operation.
   
   
    
   
   

PROBLEM SUMMARY
 *  FILTER ANALYZER INCORRECTLY REPORTS AN ERROR WITH FILTER FORM:
   FIELDA NOT LIKE 'ABC' AND FIELDB != ''
   
   
    
   
   

PROBLEM CONCLUSION
 *  As a result, the filter analyzer throws an error and event
   reader doesn't start.
   
   This APAR is fixed in the following maintenace release:
   5.1.1.1-TIV-NCI-IF0003
