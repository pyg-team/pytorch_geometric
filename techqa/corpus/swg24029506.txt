Title: IBM SPSS Collaboration and Deployment Services 4.1 Fix Pack 2 - United States

Text:
SPSS; SPSS patch; Collaboration; Deployment DOWNLOADABLE FILES

ABSTRACT
 This fix pack will upgrade your IBM SPSS Collaboration and Deployment Services 4.1 installation to IBM SPSS Collaboration and Deployment 4.1 FP2 (4.1.0.2). 

DOWNLOAD DESCRIPTION
This fix pack provides important product corrections for IBM SPSS Collaboration and Deployment Services 4.1. It has gone through a targeted regression test suite aimed at validating roll up of escalated defect resolutions that have gone in this fix pack.




Issues Corrected: Fix list [https://www.ibm.com/support/docview.wss?uid=swg27021308]

PREREQUISITES
This fix pack must be applied over an existing SPSS Collaboration and Deployment Services 4.1 installation.

INSTALLATION INSTRUCTIONS
 

[http://www.ibm.com/i/v14/icons/fw_bold.gif] SPSS Collaboration and Deployment Services 4.1 FP2 repository installation instructions [javascript:void(0);] [javascript:void(0);] (click to expand) Prior to installation
 * Back up PASW Collaboration and Deployment Services repository database. 
 * Verify that the PASW Collaboration and Deployment Services application server is in the following state:  * JBoss: shut down 
    * WebSphere: running, server should be restarted prior to patch apply to clear JNI handles 
    * WebLogic: shut down 
    * Oracle: shut down
   
   
 * Verify that PASW Collaboration and Deployment Services repository database is running. 


Installing

1. Download 4.1-IM-SCaDSBASE-FP002.zip. 2. Extract the content of the archive to a temporary location on the PASW Collaboration and Deployment Services repository host file system. 

3. Execute CADS_4.1.2_install.jar as follows: 

 * GUI mode:
 * 
 * console mode: java -jar CADS_4.1.2_install.jar -i console


Note: On UNIX systems use console mode. 4. Review the end-user license agreement, specify PASW Collaboration and Deployment Services repository installation directory, and proceed with the execution. The program will create patching subdirectory in the specified location. 

5. Change to <PASW Collaboration and Deployment Services repository installation directory>/patching/CADS_PATCH_4.1.2 directory. 

6. On System i, use QShell environment. 

7. On UNIX and System i systems, make sure that apply-patch.sh or apply-patch.qsh has execute permission. 

8. Execute the patch script appropriate to the system where Collaboration and Deployment Services is installed: 

 * Windows:
 * 
 * 
 * UNIX:
 * 
 * 
 * System i:
 * 
 * 

9. When prompted, enter the Collaboration and Deployment Services administrator user name and password and proceed with patch installation. 10. If the patch installation completes successfully, a corresponding message will be displayed. 

11. Start PASW Collaboration and Deployment Services repository application server. If you are using WebSphere, restart the server. 


Post-installation


 * If you must reinstall Statistics and Modeler adapters after applying PASW Collaboration and Deployment Services Patch 4.1.2 to a repository running on WebSphere 7 cluster, use WebSphere administrative console to stop all applications and make sure the node manager is running before installing the adapters.
