Title: IBM Infosphere Information Server ODBC for Hive Connector Fails Connecting to Hive/CDH Hadoop Data Source - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Infosphere Information Server ODBC for Hive connector fails connecting toHive/CDH Hadoop data source. 

SYMPTOM
Check the odbc trace output for errors: 

SQLSTATE = S1T00
NATIVE ERROR = 0
MSG = [IBM(DataDirect OEM)][ODBC Apache Hive Wire Protocol driver]
Timeout expired.

SQLSTATE = S1000
NATIVE ERROR = 1202
MSG = [IBM(DataDirect OEM)][ODBC Apache Hive Wire Protocol driver]Error
switching to requested database (database). 

See the following url for information on enabling the odbc tracing for a connector:
http://www-01.ibm.com/support/docview.wss?uid=swg21657912 [http://www-01.ibm.com/support/docview.wss?uid=swg21657912]


CAUSE
Setting authentication in HIVE ODBC driver entry in the odbc.ini file.


DIAGNOSING THE PROBLEM
Check whether the HIVE ODBC driver configuration for authentication enabled.




RESOLVING THE PROBLEM
Disable authentication for the HIVE ODBC driver in the odbc.ini file.
