Title: IBM Daeja ViewONE Virtual Image does not display - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 This issue can be seen when using the embedded Daeja ViewONE Virtual viewer in a Content Navigator or DataCap Navigator system. 


SYMPTOM
In DataCap Navigator: 

Unable to display document: Image load failed

[/support/docview.wss?uid=swg22011904&amp;aid=1]

In Content Navigator:

Unable to display document: Annotations failed to load: Annotation data is not in the correct format

[/support/docview.wss?uid=swg22011904&amp;aid=2]


CAUSE
The jace.jar file is not present in the expected location so this causes a classloader error to occur for the new



ENVIRONMENT
 IBM DataCap Navigator or IBM Content Navigator



DIAGNOSING THE PROBLEM
Review the systemOut log files with debugging enabled and look for the following error:

java.lang.NoClassDefFoundError:com.filenet.api.exception.EngineRuntimeException
at com.ibm.ecm.protocol.icn.Handler.openConnection(Handler.java:128)
at java.net.URL.openConnection(URL.java:991)
...
Caused by: java.lang.ClassNotFoundException: com.filenet.api.exception.EngineRuntimeException
at java.net.URLClassLoader.findClass(URLClassLoader.java:607)
at com.ibm.ws.bootstrap.ExtClassLoader.findClass(ExtClassLoader.java:243) 

 

Whilst different symptoms are seen on the client depending on whether the system used is Content Navigator or DataCap Navigator, this same error text shows in the SystemOut logs in either case.



RESOLVING THE PROBLEM
The issue is resolved by copying jace.jar to the navigator.war\WEB-INF\lib folder
