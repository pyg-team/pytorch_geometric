Title: IBM Installation Manager and IBM Packaging Utility 32-bit applications crash on a 64-bit x86 RHEL 7 machine - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When running the IBM Installation Manager or IBM Packaging Utility 32-bit application using the Graphical User Interface (GUI) on a 64-bit RHEL 7 platform, the applications will crash with a JVM terminated error. 

SYMPTOM
The GUI crashes and a JVM termination window pops up with the following text: 

 JVM terminated. Exit code=160


RESOLVING THE PROBLEM
The IBM Installation Manager and IBM Packaging Utility 64-bit applications support RHEL 7.0 x86 on 64-bit architecture. 

To resolve the issue with the IBM Installation Manager, follow the Knowledge Center instructions for converting the 32-bit Installation Manager to 64-bit using the -reinstallIM option. See the 1.8.2 Knowledge Center topic for changing the bit mode of Installation Manager [http://www-01.ibm.com/support/knowledgecenter/SSDV2W_1.8.2/com.ibm.cic.agent.ui.doc/topics/t_bit_mode.html?lang=en]. 

 

To resolve the issue with the IBM Packaging Utility, simply un-install the 32-bit Packaging Utility application and reinstall the 64-bit application.
