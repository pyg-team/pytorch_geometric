APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Searching for users with the Portal "directory search" dialog
   is  sometimes very slow in production.
   When searching for users to give them access, sometimes it
   takes > 30 seconds (or more) to return results (and therefore
   appears that no results are returned)
   Sometimes results are returned very fast
   The issue has been seen when using the following standard
   product dialogs:
   WCM Easy Authoring "Select Owners"; "Select Authors"; "Select
   Users"
   (dialog popup says "Launch the directory searchdialog")
   Newsletters "Select Authors"; "Select Users"; "Grant * Access"
   (dialog popup says "Launch the directory searchdialog")
   Quickr "Add Members"
   The issue occurs when searching for both users and groups
   The issue has not been seen in the Portal "Manage Users and
   Groups" portlet dialog
   This is not impacted by the browser being used.
   
   
    
   
   

LOCAL FIX
 *  L3 have supplied 7.0.0.23-WS-WAS-TFAX51432.pak  as a test fix
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server                                      *
   ****************************************************************
   * PROBLEM DESCRIPTION: Virtual member manager (VMM) is not     *
   *                      returning correct results when using    *
   *                      a search exression including LDAP and   *
   *                      LA properties                           *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Virtual member manager (VMM) is not returning the correct
   results when a complex search expression contains LDAP and LA
   properties.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This fix handles complex search expressions with LDAP and LA
   properties.
   
   The fix for this APAR is currently targeted for inclusion in
   fix packs 7.0.0.31, 8.0.0.8 and 8.5.5.1.  Please refer to the
   Recommended Updates page for delivery information:
