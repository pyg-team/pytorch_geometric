Title: IBM Deployment Engine registry files read-write permissions - United States

Text:
DEL2CONF; deployment engine registry files permission TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The Deployment Engine registry files have read-write permissions for any user, thus not security compliance. 

SYMPTOM
The Deployment Engine repository files show below permissions. The directories for root and non-root user installations are different. So change DE_INSTALLATION_DIR with correct path in the following. 

$ cd <DE_INSTALLATION_DIR>/repos/IURegistryDB/seg0 

$ ls -l 

-rwxrwxrwx 1 tivoli tivadmin 8192 Jul 27 16:57 cb1.dat
-rwxrwxrwx 1 tivoli tivadmin 8192 Jul 20 10:18 cc0.dat
-rwxrwxrwx 1 tivoli tivadmin 8192 Jul 20 10:18 cd1.dat
-rwxrwxrwx 1 tivoli tivadmin 8192 Jul 20 10:18 ce1.dat
-rwxrwxrwx 1 tivoli tivadmin 12288 Jul 20 10:18 cf0.dat
-rwxrwxrwx 1 tivoli tivadmin 898 Jul 20 10:18 service.properties


CAUSE
It is a product defect and is resolved in DE 1.4.0.15. This DE version will be rolled into different products at various times. 

You can check DE version by running below command:

[/support/docview.wss?uid=swg21610750&amp;aid=1]

Root user: 
/usr/ibm/common/acsi/bin/de_version.sh 
Non-root user: 
$HOME/.acsi_<username>/bin/de_version.sh

[/support/docview.wss?uid=swg21610750&amp;aid=2]
Global Install:
Run below command from command prompt:

cd C:\Program Files\IBM\Common\acsi 
de_version.cmd 


Local Install:
Run below command from command prompt:

cd C:\Documents and Settings\<username>\asci_<username> 
de_version.cmd



RESOLVING THE PROBLEM
Use below work around to change file permissions. This work around is available in DE 1.4.0.15 or later versions. 


[/support/docview.wss?uid=swg21610750&amp;aid=3]
For root user Installation: 
Run below command:
/usr/ibm/common/acsi/bin/de_security.sh -singleUser

For non-root user Installation: 
Run below command:
$HOME/.acsi_<username>/bin/de_security.sh *singleUser 

 
[/support/docview.wss?uid=swg21610750&amp;aid=4]
Global Install:
Run below command from command prompt:

cd C:\Program Files\IBM\Common\acsi 
de_security.sh *singleUser 


Local Install:
Run below command from command prompt:

cd C:\Documents and Settings\<username>\asci_<username> 
de_security.sh *singleUser

If you like to upgrade to DE 1.4.0.15 then please open a PMR so IBM support team can provide upgrade package.
