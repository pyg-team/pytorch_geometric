ERROR DESCRIPTION
 *  Severity:   2
   
   
   Approver: N/A
   
   Compid:   5724B96OO
   
   Release:  6.2 and 6.2.1
   
   Abstract:   FAILS TO MONITOR ORACLE LISTENER WHEN DATABASE IS
   INACTIVE
   
   Detailed Recreation Procedure:
   1. Login the Oracle Database instance and shutdown it.
   su - oracle
   sqlplus / as sysdba
   shutdown immediate
   
   2. Make sure the listener is active
   
   lsnrctl status
   
   3. Go to ITM TEP, check Oracle->OracleInstanceName->Processes
   Node, make
   a view to display the KORLISTD ODI table data, you will found
   that no
   data is display. and Oracle_Server_Not_Active is trigger.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ***************************************************************
   * USERS AFFECTED:
   ITM for Database:Oracle Agent Version 6.20FP1
   ***************************************************************
   * PROBLEM DESCRIPTION:
   Oracle Agent can not monitor the Oracle listener when Oracle
   Server
   is inactive.
   ***************************************************************
   * RECOMMENDATION:
   apply 6.2.0.1-TIV-ITM_ORA-IF0008 when available
   ***************************************************************
   Oracle Agent can not monitor oracle listener because Agent
   find out the Oracle server is inactive and stop all
   database related monitor
   activities including listener monitoring.
