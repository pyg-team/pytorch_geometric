Title: IBM How to disable and remove the rich text information when generating a Rational SoDA report on a Rational Software Modeler 7.0.5.x and 7.5.x model - United States

Text:
richtext; rich text; html tags; report; documentation; soda; html; richt-text; 1307493 TECHNOTE (FAQ)

QUESTION
 This technote explains how you can disable and remove the rich text information stored in a IBM Rational Software Modeler, IBM Rational Software Architect for Websphere or Standard Edition model so that it becomes plain text and can be displayed with using the "Documentation" attribute in your IBM Rational SoDA report . 

CAUSE
Since version 7.0.5.x of Rational Software Modeler and IBM Rational Software Architect there is an option to have the documentation displayed in a rich-text format. 

When executing a Rational SoDA report, the documentation display in the generated MicrosoftÂ® Word document will contain the HTML tags that represents the rich-text format in the model.

By default, the Display rich text on diagram and documentation option is enabled as shown below:

[/support/docview.wss?uid=swg21307493&aid=1] [/support/docview.wss?uid=swg21307493&aid=1]


ANSWER
To prevent the SoDA Generation from displaying the HTML tags in the generated Word document , disable the rich-text formatting from the Modeling preference: 

 

 1. Click Window > Preferences > Modeling > Miscellaneous.
    
 2. Clear the Display rich text on diagrams and documentation check box.
    
 3. Click OK.
    
 4. Close the model(s)


The next time you will reopen the model, the documentation will not show this rich text option anymore: 

[/support/docview.wss?uid=swg21307493&aid=2] [/support/docview.wss?uid=swg21307493&aid=2] 

but it will show the HTML tags: 
[/support/docview.wss?uid=swg21307493&aid=3] [/support/docview.wss?uid=swg21307493&aid=3] 

Deactivating the option will only affect the new documentation that the user will add. 





To remove the existing HTML tags from the documentation field: 
 1. Click Window > Preferences > Modeling > Miscellaneous.
    
 2. Clear the Display rich text on diagrams and documentation check box.
    
 3. Select the Display text conversion action in Refactor menu check box.
    
 4. Click Apply and OK.
    
 5. In the Project Explorer view, right-click the model and click Refactor > Convert Documentation to Plain Text.
    
    [/support/docview.wss?uid=swg21307493&amp;aid=4]
    
 6. Save the model:


The next SoDA report generation will no longer display the HTML tags. 

Additionally, with SoDA 7.0.2 iFix01 or 7.0.1.3 iFix01 or 7.0.0.4 iFix01 or later versions of SoDA a new attribute was called "RichTextDocumentation" added so that you can generate your report against models containing Rich Text documentation.
