APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The update of a portlet instance via XMLAccess script fails
   with the following exception:
   
   com.ibm.wps.command.xml.XmlFormatException: EJPXA0049E: Input
   syntax error in line (nnn) column (nnn): the XML input does not
   conform to the XML schema PortalConfig_7.0.0.xsd. unknown
   org.xml.sax.SAXParseException: cvc-complex-type.2.4.a: Invalid
   content was found starting with element 'parameter'. One of
   '{preferences, supported-publishing-event,
   supported-processing-event}' is expected.
   
   The portlet instance that fails to update contains updates on
   both parameter data and preference data.
   
   The XML script has been either produced by XMLAcess, or by the
   releasebuilder.
   
   
    
   
   

LOCAL FIX
 *  Edit the xml document that failed to execute.
   
   Locate all <portletinstance> sections that contain both
   <parameter> and <perferences> sections.
   
   Put all <preferences> and <parameter> sections into
   the correct order.
   In the failing scenario, the order was
   <portletinstance ..>
   <preferences..>
   (..)
   <parameter ..>
   (..)
   </portletinstance>
   
   Move all <parameter> sections before the first
   <preferences> section, so that the sections appear in the
   correct position:
   
   <portletinstance ..>
   <parameter..>
   (..)
   <preferences ..>
   (..)
   </portletinstance>
