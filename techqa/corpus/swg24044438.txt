Title: IBM ITCAM Agent for WebSphere MQ Version 7.3.0 Fix Pack 02 Provisional Fix 01 - United States

Text:
 DOWNLOADABLE FILES

ABSTRACT
 This provisional fix only contains updated agent executables for ITCAM agent for Websphere MQ. It provides toleration support for IBM MQ v9.0.3 and v9.0.4 and includes a few APARs. 

DOWNLOAD DESCRIPTION
Fix files for this fix:


 lx8263_v7302_kmq_LA01.tar.gz (Linux x86_64)
aix523_v7302_kmq_LA01.tar.gz (AIX)
win64_v7302_kmq_LA01.zip (Windows 64 bit)

PREREQUISITES
Make sure that your ITCAM agent for WebSphere MQ is at V730 FP02. Refer to following link about the V730 FP2.
http://www-01.ibm.com/support/docview.wss?uid=swg24043057 [http://www-01.ibm.com/support/docview.wss?uid=swg24043057]

INSTALLATION INSTRUCTIONS
 - Apply LA 01 on Linux/Unix 
1) Stop running MQ agent instances
2) Backup kmqagent, kmqlocal and kmqremote under <ITMHOME>/<arch>/mq/bin directory.
For example, rename them.
3) Copy the updated files from the tar.gz for your Linux or AIX to the above directory.
4) Start all MQ agent instances.

- Apply LA 01 on Windows 
1) Stop all running MQ agent instances from "Manage Tivoli Enterprise
Monitoring Services(MTEMS)" panel and ensure there are no kmqlocal or
kmqagent process in windows task manager.

2) Backup the original MQ agent binaries (kmqagent.exe, kmqlocal.exe and
kmqremote.exe) in <ITMHOME>\TMAITM6_x64 directory.

3) Replace with the same files in win64_v7302_kmq_LA01.zip. If you have
multiple MQ agent instances, copy kmqagent.exe, kmqlocal.exe and
kmqremote.exe to kmqagent_<instance>.exe, kmqlocal_<instance>.exe and
kmqremote_<instance>.exe respectively.

4) Restart MQ agent instances from the MTEMS.

Note: <ITMHOME> is the directory where your mq agent is installed.
<arch> is the platform code, such as aix523, lx8263 etc.
