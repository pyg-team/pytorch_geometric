APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In SOAP 1.1 specification, Client and Server are two valid
   faultcodes, while in SOAP 1.2 specification, the corresponding
   faultcodes are changed into Sender and Receiver. When Liberty
   Profile receives such a SOAP 1.2 fault message from WebSphere
   Application Server, Liberty Profile issues:
   "{http://www.w3.org/2003/05/soap-envelope}Server is not a
   standard Code value".
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  IBM WebSphere Application Server users of   *
   *                  JAX-WS web services applications            *
   ****************************************************************
   * PROBLEM DESCRIPTION: The unified SOAP fault code "Server"    *
   *                      created by WebSphere Application        *
   *                      Server does not meet SOAP 1.2           *
   *                      specification.                          *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When creating a unified SOAP fault message, "Server" is
   set as the fault code, however, "Server" is not a standard
   code value of SOAP 1.2 specification. When a Liberty Profile
   server receives such a SOAP 1.2 fault message from WebSphere
   Application Server, the Liberty Profile server issues:
   "{http://www.w3.org/2003/05/soap-envelope}Server is not a
   standard Code value".
   
   
    
   
   

PROBLEM CONCLUSION
 *  We now differentiate SOAP 1.1 and SOAP 1.2 unified fault
   messages. We use Server as the fault code for SOAP 1.1
   messages, and use Receiver as the fault code for SOAP 1.2
   messages.
   
   This new behavior is controlled with a JVM property:
   com.ibm.ws.webservices.useReceiverInUnifiedSOAP12Fault
   
   This JVM property must be set to true to get the new behavior
   with Receiver as the fault code for SOAP 1.2 messages. The
   default value is false and does not change behavior.
   
   The fix for this APAR is currently targeted for inclusion in
   fix packs 7.0.0.31, 8.0.0.8 and 8.5.5.2.  Please refer to the
   Recommended Updates page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
