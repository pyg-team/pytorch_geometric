Title: IBM Disable SSLv3 in DataPower when communicating with WebSphere MQ - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How do I prevent DataPower from using SSLv3 when communicating with WebSphere MQ ? 

CAUSE
Due to various vulnerabilities reported in SSLv3 protocol, it is highly recommended to disable SSLv3 across all configuration objects in DataPower.

ANSWER
1. First make sure to Quiesce all domains and services to stop traffic to the appliance. System quiesce and unquiesce commands can be invoked by navigating to Administration --> Main --> System Control.


2. There are two things you need do to prevent DataPower from communicating with WebSphere MQ using SSLv3.

When DataPower mq-qm object is using Key Repository (GSkit):

(a). In DataPower Control Panel, navigate to "Configure MQ Queue Manager" page. Under the "Connections" tab, configure the parameter "SSL Cipher Specification" to use one of the cipher specifications listed below.

For DataPower versions 6.0.0 and later:

TLS_RSA_WITH_AES_128_CBC_SHA
TLS_RSA_WITH_AES_256_CBC_SHA
TLS_RSA_WITH_NULL_SHA256
TLS_RSA_WITH_AES_128_CBC_SHA256
TLS_RSA_WITH_AES_256_CBC_SHA256

For DataPower versions prior to 6.0.0:

TLS_RSA_WITH_AES_128_CBC_SHA
TLS_RSA_WITH_AES_256_CBC_SHA
AES_SHA_US

(b). In WebSphere MQ channels select the same TLS protocol as DataPower uses from the channel CipherSpec.

When DataPower mq-qm object is using an SSL Proxy Profile:
