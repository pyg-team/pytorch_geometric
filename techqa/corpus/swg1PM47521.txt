ERROR DESCRIPTION
 *  AdminApp Edit MapSharedLibForMod fails with ADMA0112E on version
   8.0. It works if we run the same command on version 8.0
   wsadmin client itself, it fails only when we run the command
   from version 7.0 or version 6.1 wsadmin client.
   
   For example the following command running from version 7.0
   
   AdminApp.edit('AppName', '[ -MapSharedLibForMod [[ AppName META-
   INF/application.xml AXIS2 ]]]' )
   
   fails with the following exception.
   
   ADMA0112E: Validation error in task Assign shared libraries to
   application or each module
   The required column uri is not specified.
   
   
    
   
   

LOCAL FIX
 *  execute the command on version 8.0 .
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server version 8.0                          *
   ****************************************************************
   * PROBLEM DESCRIPTION: When editing an application using the   *
   *                      wsadmin AdminApp.edit command or        *
   *                      calling                                 *
   *                      AppManagement.getApplicationInfo        *
   *                      method from a client program with       *
   *                      v7.0 or below connected to a 8.0        *
   *                      deployment manager, the edit fails      *
   *                      with an exception.                      *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When editing an application using the wsadmin AdminApp.edit
   command or calling AppManagement.getApplicationInfo method
   from a client program with WebSphere Application Server v7.0 or
   below connected to a WebSphere Application Server v8.0
   deployment manager, the edit fails with an exception.
   The exception from wsadmin will be a validation exception
   similar to the following:
   ADMA0112E: Validation error in task Assign shared libraries to
   application or each module.
   The required column uri is not specified.
   The exception from setApplicationInfo will be an
   MBeanException similar to the following:
   javax.management.MBeanException: Exception thrown in
   RequiredModelMBean while trying to invoke operation
   setApplicationInfo
    &#162;&#226;&#130;&#172; &#166;
   Caused by:
   com.ibm.websphere.management.exception.AdminException:
   java.lang.Exception: Invalid ObjectName format
   commsvc.rest.war,WEB-INF/web.xml
