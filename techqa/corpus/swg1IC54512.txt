ERROR DESCRIPTION
 *  The OSS runmqsc process loops when the Guardian runmqsc
   is TACL STOP.
   
   
    
   
   

LOCAL FIX
 *  fix required
   
   
    
   
   

PROBLEM SUMMARY
 *   If a Guardian runmqsc process is TACL STOP
   the OSS runmqsc process that performs the actual work loops.
   The problem is related to the process not handling a error to
   its stdin file.
   The condition also ocuurs if the telnet session that the
   Guardian process is run on fails. The OSS process consumes 100%
   of the cpu it is running in.
   
   
    
   
   

PROBLEM CONCLUSION
 *   Correct the process handling of stdin file
   errors.
   .
   Released in WMQ V5.3.1.2 fix pack for HP NonStop Server
