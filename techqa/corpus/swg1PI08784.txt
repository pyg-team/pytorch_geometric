ERROR DESCRIPTION
 *  Multiple fixes bring DataQuant for WebSphere 1.2 up to Fix Pack
   19 level.
   Fixes include:
   QMFDQ-15363 SQL query text is not displayed correctly in the
   SQL query editor when using Internet Explorer web-browser for
   running the application.
   QMFDQ-15364 Delete row does not work on rows that were newly
   added using Table Editor.
   QMFDQ-15516 The "DDM0006. Data format exception detected."
   connection.
   QMFDQ-15554 Web links do not work when the application is run
   in cluster mode.
   QMFDQ-15885 The colums selected in the Promted Query editor
   are sorted in reverse order.
   QMFDQ-15888 Row conditions cannot be selected in the Prompted
   Query editor.
   QMFDQ-15889 An extra graphic attribute is added in the SQL
   query editor after changing the row conditions.
   QMFDQ-15898 SBCS% instead of DBCS% is added to the graphic
   column by generated SELECT statement in the Prompted Query
   editor and the Diagram view.
   QMFDQ-15944 The request to JSLibraryServiceHandler is not
   cached.
   QMFDQ-15945 The connection time to a repository increases for
   each subsequent user logging on to this repository.
   QMFDQ-15966 In some cases, the Prompted Query editor can
   generate SQL text that exceeds the limit of 79 characters.
   QMFDQ-15968 User login information for a datasource cannot be
   entered when the "Apply logon to subsequent connections during
   application session" option is selected.
