APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The 64-bit Windows plugin fails to load the GSK security
   library causing HTTPS requests to fail. This happens because
   the plugin tries to load the 32-bit GSK library instead of the
   64-bit library. An error message similar to the following can
   be found the plugin log:
   
   ERROR: lib_security: loadSecurityLibrary: Failed to load gsk
   library from C:\Program Files\IBM\WebSphere\Plugins\gsk8\gsk8_32
   ERROR: ws_transport: transportInitializeSecurity: Unable to
   load security library
   ERROR: ws_server: serverAddTransport: Failed to initialize
   security. Secure transports are not possible.

   The 64-bit Windows plugin fails to load the GSK security
   library causing HTTPS requests to fail. This happens because
   the plugin tries to load the 32-bit GSK library instead of the
   64-bit library. An error message similar to the following can
   be found the plugin log:
   ERROR: lib_security: loadSecurityLibrary: Failed to load gsk
   library from C:\Program
   Files\IBM\WebSphere\Plugins\gsk8\gsk8_32
   ERROR: ws_transport: transportInitializeSecurity: Unable to
   load security library
   ERROR: ws_server: serverAddTransport: Failed to initialize
   security. Secure transports are not possible.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The 64-bit Windows WAS plugin was updated to load the 64-bit GSK
   library instead of the 32-bit library.
   
   This fix is targeted for IBM HTTP Server fix packs:
   - 9.0.0.6
