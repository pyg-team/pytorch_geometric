Title: IBM Failed to parse date string using MSK - United States

Text:
MSK; datetotime conversion; $date; datetime; parse TZ; time zone conversion; probe time zone; date/time; date time TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 datetotime conversion 

$date = datetotime("Tue Dec 19 17:33:11 MSK 2000" , "EEE MMM dd HH:mm:ss vv yyyy") 

fails in probe.rules file.


SYMPTOM
Errors in probe debug log file and incorrect time is output, for example: 

"Error: E-ETC-102-036: Fail to parse date string: 
Tue Dec 19 17:33:11 MSK 2000 using format: EEE MMM dd HH:mm:ss vv yyyy. 
Error code: U_PARSE_ERROR 

Warning: W-UNK-000-000: Failed to convert "Tue Dec 
19 17:33:11 MSK 2000" to DATETIME using format "EEE MMM dd HH:mm:ss vv 
yyyy" - using current time"


CAUSE
The probe EMS uses the time zone abbreviation 'MSK', however the ICU database - which the probe libraries are developed on - does not recognise it as a valid time zone for date/time conversions.


RESOLVING THE PROBLEM
Use 

$date = datetotime("Tue Dec 19 17:33:11 Europe/Moscow 2000", "EEE MMM dd HH:mm:ss VV yyyy") 

in the rules file instead.
