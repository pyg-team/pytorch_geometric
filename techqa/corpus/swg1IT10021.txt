ERROR DESCRIPTION
 *  Documentum client builder does not create build-dfc-client-rpm
   on StoredIQ data server.
   
   Environment: StoredIQ 7.6.0.2 and 7.6.0.3 data server.
   
   Error noted:
   Building RPM siq-war-dfc-client-201.0.0.0-1.x86_64.rpm
   Copying RPM siq-war-dfc-client-201.0.0.0-1.x86_64.rpm to current
   directory
   cp: cannot stat
   /deepfs/documentum/buildroot/RPMS/x86_64/siq-war-dfc-
   client-201.0.0.0-1.x86_64.rpm': No such file or directory
   
   
    
   
   

LOCAL FIX
 *  Add clean step in /usr/local/bin/build-dfc-client-rpm at the
   end of the SPECFILE section:
   Name: $rpmname
   Summary: The siq-war-dfc-client RPM package for Linux.
   Group: Services/Network
   Version: $dctmver
   Release: $release
   License: Proprietary
   AutoReqProv: no
   Requires: siq-war-dctm
   Vendor: StoredIQ
   %description
   The webapp that contains the Documentum DFC .jar files.
   %files
   $dctmwardir
   %pre
   # n/a
   %post
       cp $siqwardir/xmlrpc*.jar $dctmwardir/
       cp $siqwardir/ws-commons-util-1.0.1.jar $dctmwardir/
       cp $siqwardir/commons-logging-1.1.jar $dctmwardir/
   %clean
   #Added_clean_step
