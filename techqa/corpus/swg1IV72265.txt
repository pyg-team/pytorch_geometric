ERROR DESCRIPTION
 *  Abstract:
   KQI agent has a concurrency issue when processes data received
   from java
   sub-process.
   This issue that may cause "Signal Memory Access Violation" and
   agent
   down.
   Users Affected:
   WebSphere Message Broker Monitoring Agent users
   Environment:
   All platforms.
   Problem Description:
   If there are multiple units of response data received from java
   sub-process (read from pipe) at the same time, the receive queue
   in main
   process may be accessed by two thread concurrently.
   One of them may remove the data while another is still using it.
   Signal
   11 is received from function "KQI_Sub_Process::run()".
   The related call stack in core file is shown as below.
