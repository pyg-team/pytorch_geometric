ERROR DESCRIPTION
 *  ITNM 4.1.1
   Omnibus 8.1 installed with IM
   
   configOMNI script run prior to installation of ITNM 4.1.1
   
   documentation
   
   http://www-01.ibm.com/support/knowledgecenter/SSSHRK_4.1.1/com.i [http://www-01.ibm.com/support/knowledgecenter/SSSHRK_4.1.1/com.i]
   bm.networkmanagerip.doc_4.1.1/itnm/ip/wip/install/reference/nmip
   _ins_configomnicmdlineoptions.html?lang=en
   
   When -m is used this fails with the error
   
   Execution terminating with error code 7. ./ConfigOMNI[633]:
   /opt/IBM/tivoli/netcool/omnibus/install/nco_patch:
   not found [No such file or directory]
   
   the problem is that 8.1 NOI objectserver has neither
   of these installation mechanisms.  It's not COI and too new for
   the preinstallation method.
   
     Report "Installing probe $MTTRAPD_PROBE_PACKAGE"
   
      if [ -d COI ]; then
          echo "LICENSE_ACCEPTED=true" >$TMP/probe_response.txt
          echo "PROBE_OR_GATE_LOCATION=$PWD"
   >>$TMP/probe_response.txt
          $OMNIHOME/install/nco_install_integration -i silent -f
   $TMP/probe_response.txt
      else
          NAME=&#180;ls | tail -1&#180;
          $OMNIHOME/install/nco_patch -batch -force -install $NAME
   1>$TMP/$OBJ_NAME.log 2>&1
      fi
      if [ $? -ne 0 ]; then
           Error "Installing probe $MTTRAPD_PROBE_PACKAGE failed.
   Please see $TMP/$OBJ_NAME.log" 7
      else
           rm -rf $TMP/$OBJ_NAME.log
      fi
   ===============
   
   here is the NOI doc it is correct to not use -m
   
   http://www-01.ibm.com/support/knowledgecenter/SSTPTP_1.3.0/com.i [http://www-01.ibm.com/support/knowledgecenter/SSTPTP_1.3.0/com.i]

   pt.html?cp=SSTPTP_1.3.0%2F2-1-7-2&lang=en
   
   And here is the ITNM doc where we talk about using -m
   
   http://www-01.ibm.com/support/knowledgecenter/SSSHRK_3.9.0/com.i [http://www-01.ibm.com/support/knowledgecenter/SSSHRK_3.9.0/com.i]
