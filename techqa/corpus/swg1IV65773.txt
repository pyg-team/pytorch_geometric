LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Tivoli System Automation Application Manager
   * (SA AM) 4.1 users
   ****************************************************************
   * PROBLEM DESCRIPTION:
   * The SA AM automation engine might end up in an infinite loop,
   * making the communication to the first level automation domains
   * impossible. The WebSphere Application server 'SystemOut.log'
   * shows an EEZConnectionFailedException:
   *   com.ibm.eez.sdk.exceptions.EEZConnectionFailedException:
   *   EEZF0006E A java.rmi.RemoteException: CORBA NO_MEMORY
   *   0x4942f982 No; nested exception is:
   *    org.omg.CORBA.NO_MEMORY:
   *    >> SERVER (id=12345678, host=xx.xx.xx.xx) TRACE START:
   *    >>   org.omg.CORBA.NO_MEMORY: Unable to read value from
   *     underlying bridge : Java heap space  vmcid: IBM
   *     minor code: 982 completed: No
   *    >>  at com.ibm.rmi.iiop.CDRReader.read_value
   *     (CDRReader.java:2429)
   *    >>  at com.ibm.eez.dmn.client._EEZDaemon_Tie.handleEvent
   *     (_EEZDaemon_Tie.java:587)
   *
   * The SA AM traces do show the message with ID CHRM1058I, and a
   * resource reference which has its ObservedState switching
   * between 'WasOnline' and 'Online'.
   ****************************************************************
   * RECOMMENDATION:
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The internal automation Engine (Logic deck) has been adjusted to
   address this issue. The trigger to enter the infinite loop has
   been removed.
   .
   The official fix for this problem is included in Tivoli System
   Automation Application Manager 4.1 fix pack 1.
