APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Customer installed Sterling Secure Proxy and ran the
   bin/configureEngineSsl.sh -x  utility to change the default
   keystore passphrase from "password" to one that had an
   ampersand (&), e.g. "bad&pass".  When they attempted to start
   up the engine, they got
   Startup did not succeed. Terminating:
   com.sterlingcommerce.hadrian.common.xml.XmlParsingException:
   Error on line 10: The reference to entity "pass" must end
   with the ';' delimiter.
   The Customer had to reinstall the engine and refrain from using
   a keystore passphrase that included an ampersand.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  The SSP engine or CM may not start correctly after using
   configureEngineSsl.bat/bin or configureCmSsl.bat/bin to change
   the keystore password to a string containing special
   characters. Depending on the location of the special character
   in the string, the error message will be something like:
   Startup did not succeed. Terminating:
   com.sterlingcommerce.hadrian.common.xml.XmlParsingException:
   Error on line 10: The reference to entity "pass" must end with
   the ';' delimiter.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Code has been added to correctly encapsulate passwords so that
   special characters are not processed as such.
   
   
    
   
   

TEMPORARY FIX
 *  Sent Customer fix for RTC453348.
