Title: IBM IZ22413: IF-00001 BREAKS IMPACT 4.0.2 SERVICES - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Customer install IF-00001 for Impact 4.0.2 and he was not able
   to connect to object servers and on the gui display only 4
   services show up.  The problem is that selfmonitoring has this
   two properties set to true:
   
   impact.selfmonitoring.queuestatus.enable=true
   impact.selfmonitoring.autostartup=true
   
   
    
   
   

LOCAL FIX
 *  L3 has provided the following work around:
   
   1) If you have the patch install and currently experience the
   issue.  Do the following:
    - Turn OFF impact server by doing $NCHOME/bin/wasce stop
    - cd $NCHOME/impact/etc/vi
   <Your_servername>_selfmonitoring.props
    - set the following option to false:
   
    impact.selfmonitoring.queuestatus.enable=false
    impact.selfmonitoring.autostartup=false
   
    - Turn ON impact server by doing $NCOME/bin/wasce start
   
   2) If you don't have the patch install, then do the following
   before install this patch:
    -- Turn OFF impact server by doing $NCHOME/bin/wasce stop
    - cd $NCHOME/impact/etc/vi
   <Your_servername>_selfmonitoring.props
    - set the following option to false:
   
   
    impact.selfmonitoring.autostartup=false
   
    - Turn ON impact server make sure everything is OK
    - Then apply the patch
