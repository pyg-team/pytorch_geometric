ERROR DESCRIPTION
 *  Currently the API will block if the server is still running but
   has entered into a "sick but not dead" state.
   This APAR adds an option to timeout the rule execution request
   if it exceeds a value and move on to the next server in the
   HBRSSIDLIST
   
   
    
   
   

LOCAL FIX

 *  ****************************************************************
   * USERS AFFECTED: ODM users where zRES server hangs            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   If the zRES server hangs for any reason during rule
   execution then clients waiting for reply will also hang.

TEMPORARY FIX
