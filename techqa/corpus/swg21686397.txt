Title: IBM Unable to log on or no logon page after performing new install of FileNet Workplace XT. - United States

Text:
Workplace XT no logon page TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The behavior may be different depending on the browser being used. Users will see a blank page, a logon page or a generic error box that says, A system error has occurred, when accessing the Workplace XT URL. 

When using Firefox, users may see the XT logon page. When attempting to log on, users will see a dialog box that says, A system error has occurred.

When using Internet Explorer, users will see a blank browser page or a dialog box that says, A system error has occurred.

In both cases, users are not able to log on to Workplace XT.

CAUSE
Application security not enabled in WebSphere.


RESOLVING THE PROBLEM
Look in the WebSphere systemout.log for the application server instance where Workplace XT is deployed. After seeing the "A system error has occurred" dialog, look for activity similar to the exception above in the systemout.log. 

 

If exceptions like the one above are being logged in the Workplace XT systemout.log, perform the following steps:


 

 1. In the WebSphere console hosting the Workplace XT application, navigate to Security
    
    
 2. Global security and confirm the "Enable application security" check box is checked.
    
    
 3. If it is not checked, check the box, apply the change and then save the change to the WebSphere master configuration. 
    
    
 4. Since this is a global security setting, the WebSphere application server should be restarted.
