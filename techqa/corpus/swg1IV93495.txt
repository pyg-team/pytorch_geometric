ERROR DESCRIPTION
 *  GL Account data gets populated in the PLUSDACGDETAIL table for
   the associated Work Order when the Perform Estimation function
   is performed.  When the Perform Acceptance function is done, no
   values on the Work Order get replaced.  This occurs even if the
   CU Acceptance Options/GL Account/Replace setting in
   Organizations(T&D) is set.
   
   Steps to reproduce:
   -Log in to maxdemo instance as wilson
   -Go To Administration | Organizations (T&D)
   -In the Organization field enter "eaglena" and press Enter
   -Click EAGLENA to open the record
   -Select Action | T&D Options | CU Estimating Options
   -Click the checkboxes for:
    Allocate accounting to work order tasks?
    Generate accounting details records
   -Click OK
   -Select Action | T&D Options | CU Acceptance Options
   -In the GL Account section click the radio button for Replace
   -Click OK
   -Save the record
   -Log out, then log in

   Station Detail
