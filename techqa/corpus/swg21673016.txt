Title: IBM Xmlaccess is unable to connect to Portal during a CF upgrade - United States

Text:
upgrade; host file; host name; xmlaccess; could not connect to portal TECHNOTE (TROUBLESHOOTING)

PROBLEM
Xmlaccess is unable to connect to Portal during a CF upgrade

SYMPTOM
During a CF upgrade it will fail and the ConfigTrace.log shows the following....

[xmlaccess] EJPXB0006I: Connecting to URL

http:/ /<fully qualifed host name>:10039/wps/config

[xmlaccess] EJPXB0009E: Could not connect to portal.
[xmlaccess] java.net.UnknownHostException: <fully qualified host name>
[xmlaccess] EJPXB0016E: An error occurred on the client:
<fully qualified host name>
--- Exception Thrown ---
/opt/IBM/WebSphere/PortalServer/pzn/prereq.pzn/config/includes/pzn_cfg.x
ml:358: EJPXB0016E: An error occurred on the client:
<fully qualified host name
at com.ibm.wps.xmlaccess.XmlAccessTask.fail(Unknown Source)


CAUSE
In this case the problem was caused because the customer had mistyped the host name value in the machines host file

ENVIRONMENT
Linux
Portal 8.0.0.1 Express


DIAGNOSING THE PROBLEM
Review ConfigTrace.log

RESOLVING THE PROBLEM
Correct the host name value on the machine host file and reattempt the CF upgrade
