Title: IBM Rational Developer for System z 9.1.1.3 fails to launch after installing IMS Explorer 3.2.1.12 - United States

Text:
Eclipse; compatbility; RDz; IMS; Explorer; fail; launch; start TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After installing IMS Explorer 3.2.1.12, the existing installation of IBM Rational Developer for System z 9.1.1.3 fails to launch with the error, "An error has occurred. See the log file 
C:\Users\abcde\.eclipse\org.eclipse.platform_4.1.0_...\configuration\151619420732.log. 

ENVIRONMENT
Windows

DIAGNOSING THE PROBLEM
In the log file in C:\Users\abcde\.eclipse\org.eclipse.platform_4.1.0_...\configuration\151619420732.log. The error is an Eclipse error. 

The 151619420732.log shows: 
org.eclipse.osgi 
Application error 
java.lang.IllegalStateException: Unable to acquire application service. 
Ensure that the org.exlipse.core.runtime bundle is resolved and started (see config.ini.) at 
org.eclipse.core.runtime.internal.adaptor.EclipseAppLauncher.start(Unknown Source.)
... 


RESOLVING THE PROBLEM
IMS Explorer 3.2.1.12 and RDz 9.1.1.3 with Data Studio 4.1.1 use different Eclipse base versions and are incompatible. 

IMS Explorer 3.2.x uses Eclipse 4.6 and RDz 9.1.x and Data Studio 4.1.1 uses Eclipse 4.2. 


To completely resolve: 

 * Uninstall IMS Explorer 3.2.1.12 
 * Reinstall RDz 9.1.1.3 with Data Studio 4.1.1 
 * Install IMS Explorer 3.1.x.x
   


Another option is to upgrade RDz 9.1.1.3 to IDz 14.1 and Data Studio 4.1.3 to work with IMS Explorer 3.2.1.12.  

Check the compatibility of related products in the IBM Software Compatibility Matrix at https://www.ibm.com/software/reports/compatibility/clarity/index.html [https://www.ibm.com/software/reports/compatibility/clarity/index.html]
