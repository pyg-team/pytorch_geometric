LOCAL FIX
 *  * If you see the problem when you creating new SIBUS ,
   unrelated to any of the BPM deployment environment , complete
   the SIBUS configuration by creating database for ME datastore.
   or delete the unused SIBUS
   
   * You may also see this problem, if you saved a new Deployment
   environment which had problems creating the database tables
   when generating the DE environment.   You can complete the
   manually creating the required database  to complete the DE
   creation or delete the new Deployment environment using
   BPMConfig -delete option  and recreate the DE environment after
   fixing the previous database creation failures
   
   
    
   
   

PROBLEM SUMMARY
 *  No additional information is available.
   
   
    
   
   

PROBLEM CONCLUSION

TEMPORARY FIX
 *  You can add message engines for the SI (system integration) bus
   , or remove it if it is not required.
