修订程序和下载信息 
Domino 平台 Fix Central ID 文件名与下载链接 9.x 所有平台 DominoServer_9x_Applets_Jan2014 DominoServer_9x_Applets_Jan2014.zip [http://www.ibm.com/support/fixcentral/swg/selectFix?product=ibm%2FLotus%2FLotus+Domino&fixids=DominoServer_9x_Applets_Jan2014&function=fixId&parent=ibm/Lotus] 8.x 所有平台 DominoServer_8x_Applets_Jan2014 DominoServer_8x_Applets_Jan2014.zip [http://www.ibm.com/support/fixcentral/swg/selectFix?product=ibm%2FLotus%2FLotus+Domino&fixids=DominoServer_8x_Applets_Jan2014&function=fixId&parent=ibm/Lotus] 

安装新的 Applet 步骤:  * 下载并解压缩 .zip 文件 
 * 关闭 Domino 服务器 (或者执行 tell HTTP quit 退出 HTTP 任务) 
 * 备份现有文件 
 * 拷贝新的文件到 \Domino\data\domino\java，如提示文件已存在，请覆盖。



.zip 包含的文件如下:  * actionbar.jar 
 * editor.jar 
 * nvapplet.jar 
 * outline.jar 
 * hotfix license


注意: 上述 .jar 文件对于 Domino 8.x 和 9.x 都是相同的。他们唯一的区别在于 Hotfix 授权协议，一个是针对 8.x，一个是针对 9.x 版本的. 


常见问题 
这些新的 applet 解决什么问题? 
1.7u51 JVM 现在要求 applet 在 manifest 文件中明确声明类名称和需要获取的权限。然后开发人员对 .jar 文件做了签名，并增加了 JVM 浏览器插件的安全性。相关问题报告：  * SPR# SODY9DDBD5 - 安装 Java 7 Update 45 后，Applet 报告 "Application: Unknown" , 安装 Java 7 Update 51 后 Applet 无法运行 
 * SPR# SODY9FFEYE - 安装 Java 7 Update 51 后，Domino 的 Java applet 需要列在白名单中才能进行交互


这些 applet 是否会与现有的热修订程序或修订程序集产生冲突？ 
不会。这些新的 applet 不会影响服务器上现存的热修订程序。即使安装了其他热修订程序，管理员依然可以将这些 .jar 文件拷贝到服务器上。 

除了手工替换 .jar 文件，有没有其他的解决方法？ 
没有。 

iNotes 用户是否受到影响? 
iNotes 用户不受影响，iNotes 模板不使用 Applet。 

Traveler 服务器是否受到影响? 
这个问题不影响 Traveler 服务器。Traveler 服务器也不使用 applet。这个问题仅影响通过浏览器访问 applet 的用户界面。Traveler 的前端客户端不是浏览器，所以不会受到影响。 

Notes 客户端是否受到影响? 
一般不会受到影响，除非安装了 Domino Designer 客户端，在浏览器中预览的功能可能会受到影响。替换 .jar 文件后问题将得到解决。 

如果没有更新 applet ，你会看到什么错误信息？ 
"Missing required Permissions manifest attribute in main jar" 相关信息
