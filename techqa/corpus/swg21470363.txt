Title: IBM "Unable to cast COM object of type 'System.__ComObject' ... Library not registered" when launch report inside "Reports - Run" - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 User launches Controller. User clicks "Reports - Run". User selects report, and clicks "Run". User receives error message.

User acknowledges error (by clicking OK), but receives further error messages. 

SYMPTOM
Error #1 

 
Information
Standard Error
Source: ControllerForms
Description: Unable to cast COM object of type 'System.__ComObject' to interface type 'Excel.Application'. This operation failed because the QueryInterface call on the COM component for the interface with IID
'{000208D5-0000-0000-C000-000000000046}' failed due to the following error: Library not registered
{Exception from HRESULT: 0x8002801D (TYPE_E_LIBNOTREGISTERED)).
at Cognos.Controller.ControllerForms.clsDataEntry..ctor(Object&oDispatcher, Object& oExcel)
[OK]

Error #2

Information

Error #3

Information


CAUSE
Invalid installation of Microsoft Office on client device.


RESOLVING THE PROBLEM
In one real-life customer case, the solution was to: 

1. Logon to the client device as a Windows administrator
2. Uninstall all MS Office-related software
3. Install a supported version of Microsoft Office
4. Apply all the latest Microsoft patches (for that version of MS Office)
5. Test.
