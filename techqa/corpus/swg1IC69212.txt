Title: IBM IC69212: SOME REPORTS IN TIVOLI STORAGE MANAGER REPORTING AND MONITORING 6.1 CANNOT BE EXPORTED AS PDF, PS AND XLS - United States

Text:
AIX SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS FIXED IF NEXT.
    
   
   

ERROR DESCRIPTION
 *  Tivoli Storage Manager Reporting and Monitoring uses Tivoli
   Common Reporting engine and reports are generated within this
   component. For some views it can happen that the report creation
   fails with below message when format PDF, PS or XLS is selected.
   Reports in HTML format will work.
   
   CTGTRE034E A failure occurred attempting to run and format the
   report [ /TSMReports/TSM_client_schedule_status ] . The report
   cannot be run or formatted.
   
   Customer/L2 diagnostics:
   To verify if you are affected of this APAR you can review
   ReportingEngine log file and look for the following lines:
   
   Month DD, YYYY HH:MM:SS AM/PM
   org.eclipse.birt.report.engine.api.impl.ReportEngine
   SEVERE: An error happened while running the report. Cause:
   java.lang.ArrayIndexOutOfBoundsException: Array index out of
   range: 4
           at
   org.eclipse.birt.report.engine.layout.pdf.PDFImageLM.getArea(PDF
   ImageLM.
   java:531)
   
   This TSM APAR relates to TCR bug TCR1023.
   
   Tivoli Storage Manager Versions Affected:
   6.1 Reporting and Monitoring
   
   Initial Impact:
   Low
   
   Additional Keywords:
   TCR TSM TIP AC ISC RM Report export KSKAGENT ITM Warehous
   Warehouse
   
   
    
   
   

LOCAL FIX
 *  Run report as HTML, then print as PDF.
