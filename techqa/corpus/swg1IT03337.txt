Title: IBM IT03337: UNABLE TO LAUNCH GPM WHEN USING ORACLE JRE 1.7.0_51 OR LATER - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Unable to launch the Graphic Process Modeler (GPM) from
   Sterling B2B Integrator 5.2.4.2 on machines using a Oracle
   JDK/JRE 1.7.0 update 51 or later.
   
   Java Console shows the following error message:
   
   #### Java Web Start Error:
   #### Missing required Permissions manifest attribute in main
   jar: http://<SI_Address>:<baseport>/gbm/pmodeler/FlowBuilder.jar [http://<SI_Address>:<baseport>/gbm/pmodeler/FlowBuilder.jar]
   
   
    
   
   

LOCAL FIX
 *  STRRTC - 433275
   RR / RR
   
   Circumvention:
   On machines running the GPM:
   - Go to the Java Control Panel
   - Click on the Security tab
   - Click on the Edit Site List button
   - Click Add in the Exception Site List window
   - Click in the empty field under the Location field to enter
   the URL, using the format:
   http://<SI_Address>:<baseport> [http://<SI_Address>:<baseport>]
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   Sterling B2B Integrator users on Oracle JDK/JRE 1.7.0_51 or
   later.
   
   Problem Description:
   Users are unable to launch the GPM on machines using Oracle
   JDK-JRE 1.7.0_51 or later.
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  Resolution Summary:
   Resolved the issue by adding the following attributes to the
   Manifest file:
   
   Permissions
   Application-Name
   Codebase
   
   
   Oracle JDK/JRE 1.7.0_51 or later requires this. This now gets
   included when building the platform_asi jar
   Users are now able to run the GPM.
