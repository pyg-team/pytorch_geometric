Title: IBM How to enable verbose garbage collection for the Jazz Build Engine - United States

Text:
Jazz Build Engine; performance; verbose garbage collection; Java memory; JBE; RTC; Rational Team Concert; gc; verbose gc; jbe TECHNOTE (FAQ)

QUESTION
 How can you enable verbose garbage collection for the Jazz Build Engine (JBE) in IBM Rational Team Concert (RTC)? 

CAUSE
Verbose garbage collection can help investigate JBE performance issues.

ANSWER
Procedure: 

 1. Locate the file jbe.ini. This file will contain lines similar to:
    
    # Uncomment following 2 lines to configure JRE.
    #-vm
    #/TODO/path/to/jre/bin
    --launcher.suppressErrors
    -vmargs
    -Dosgi.requiredJavaVersion=1.6 
    
    
 2. Add following line:
    -verbose:gc 
    
    
    
 3. Add this flag, optionally, if you want to determine where the output log should go:
    -Xverbosegclog[:<file>[,<X>,<Y>]] 
    
    See here for syntax details: Garbage Collector command-line options [http://publib.boulder.ibm.com/infocenter/javasdk/v6r0/index.jsp?topic=%2Fcom.ibm.java.doc.diagnostics.60%2Fdiag%2Fappendixes%2Fcmdline%2Fcommands_gc.html].
    
    
 4. Save the file.
    
    
 5. Restart the JBE.


This will help gather memory-related Java performance data.
