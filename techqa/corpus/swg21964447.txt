Title: IBM How do we find the current DB2 Advanced Copy Services (ACS) version and compatibility matrix installed? - United States

Text:
ACS; version; compatibility matrix TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 What is the command used to check the current DB2 Advanced Copy Services (ACS) version installed? 

ENVIRONMENT
DB2 Advanced Copy Services (ACS)

RESOLVING THE PROBLEM
Run the below command to find the current version installed:


 * 
 * - /home/<instance_name>/sqllib/acs/acsd -v


For v9.7 we ship  * $ acsd -v
 * 
 * TSM ACS management daemon (acsd) 6.1.0.0 (Build 367P) compiled Mar 5 2009


For v10.1 we ship  * $ acsd -v
 * 
 * IBM Tivoli Storage FlashCopy(R) Manager management daemon (acsd) 2.2.1.2 (Build 474Z2) compiled Mar 29 2011


For v10.5 we ship  * $ acsd -v
 * 
 * IBM Tivoli Storage FlashCopy(R) Manager management daemon (acsd) 3.2.0.0 (Build 590) compiled Oct 22 2012


On v10.1 and v10.5 we also ship a tarball containing v4.1  * $ acsd -v
 * 
 * IBM Tivoli Storage FlashCopy(R) Manager management daemon (acsd) 4.1.0.2 (Build 649J) compiled Apr 25 2014


Note: 
For HP and Sun, we ship only the v4.1 tarball.
