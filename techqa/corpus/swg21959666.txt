Title: IBM 5.3.0.6-ISS-XGS-All-Models-IF0002 - United States

Text:
5.3.0.6-ISS-XGS-All-Models-IF0002; 5.3.0.6; Interim Fix; AllModels; All Models; IF2; LCD; XGS TECHNOTE (FAQ)

QUESTION
 What fixes are included in 5.3.0.6-ISS-XGS-All-Models-IF0002? 

ANSWER
Abstract

IBM Security Network Protection (XGS)
Interim Fix version: IF0002
Applicable versions: 5.3.0.6

Problems resolved

APAR IV74056 - Fixes issue in which LCD panel shows garbage after numerous reboots. 

APAR IV73743 - Fixes the condition where event emails would not always be sent when more than three recipients were specified. Also corrects for the same recipient list being duplicated upon policy reload, which created an erroneously high recipient count that would also causes this high recipient count failure.

APAR IV73393 - Fixes the issue where syslog_sshd crashes with sig6

APAR IV73399 - Fixes an issue where XGS cannot add management address and shows it as invalid value.

APAR IV73264 - Fixes the issue where static routing table entry is removed from the active table after reboot.

APAR IV73274 - Fixes automatic reboot on XGS where captive portal is being used.

Fixes an issue where alpsd crash with signal 49 - watchdog might erroneously cause a timeout and make alpds crash with signal 49 do to an inconsistency in the way the timeout time delta was calculated.

Disables IBM HTTP Server RC4 Cipher by default. Includes advance tuning parameter to enable if necessary. e.g lmi.security.RC4 = true

Modules impacted

- alpsd, dpdk, mesa_config, mesa_eventsd, mesa_syslogd, XGS_53.war,
GLG_events, mesa_lcd, mesa_control, pktcapd 

Known Issues

1) Fixpack does not allow for rollback. If rollback is done and appliance is then rebooted, then the appliance will get into an unconfigured state. The only way to recover is to reimage the box or change to the secondary partition.
Change to secondary partition:
a) reboot
b) choose none selected partition from the grub menu
c) press enter

Notes

- After applying fix pack, a reboot is required for the changes that are made by the fix pack to apply 
- Clear the browser cache on any browsers that are used to manage the appliance.
- If the Event Id GLGSY0038W appears in your System Events Event Log, then the operating system is experiencing a temporary failure. However, the appliance is still protecting your network. Contact IBM Support for assistance.
