DOWNLOAD DESCRIPTION
ERROR DESCRIPTION: 


Ship Java 8 SR5 FP10 for WebSphere Application Server traditional bundled Java 8 

LOCAL FIX: 
None

PROBLEM SUMMARY: 
Ship Java 8 SR5 FP10 for WebSphere Application Server traditional bundled Java 8 .

USERS AFFECTED:
Ship Java 8 SR5 FP10 for WebSphere Application Server traditional bundled Java 8 
This combination of WebSphere Application Server and Java is available with fix pack 855.11 and later.

Linux PPC64 LE does not support embedded Java 8 at this time, since the existing embedded Java on that platform is Java 7.1, and it is not going out of service yet. Please continue to use the Java 8 extension offering for your Java 8 needs on this platform.

PROBLEM DESCRIPTION: 
Ship Java 8 SR5 FP10 for WebSphere Application Server traditional bundled Java 8 

RECOMMENDATION:
Apply this i-fix to your SDK to upgrade your SDK 8 to SDK8 SR5 FP10.
NOTE:
This iFix can ONLY be applied on:
a) WebSphere Application Server traditional bundled Java 8 V855.11 and above with bundled Java 8 installed


PROBLEM CONCLUSION: 
Applying this apar will upgrade your SDK 8 to SDK8 SR5 FP10 (Oracle 1.8.0_u161 ).

The fixes in this Java SDK are found here:
https://www.ibm.com/developerworks/java/jdk/fixes/8/#SR5FP10 [https://www.ibm.com/developerworks/java/jdk/fixes/8/#SR5FP10]

When downloading the fix from fix central, please also download the readme.txt with instructions for installation.
