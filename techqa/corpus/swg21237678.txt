Title: IBM DB2 Agent not running with libkra.a could     
not be loaded. - United States

Text:
5724B96DoTR; upgrade libkra.a module invalid magic number. TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Customer has installed Omegamon CDs for 04R1M4B and installed on 
AIX partitions. UX and UL appear to work correctly, but UD (DB2 
Universal Database) agent does not start and issues following message in 
the log: 
exec(): 0509-036 Cannot load program 
/optnonaix/candle/aix516/ud/bin/kuddb2 because of the following errors: 
0509-150 Dependent module /optnonaix/candle/aix516/lib/libkra.a could 
not be loaded. 
0509-103 The module has an invalid magic number. 


CAUSE
Configuration file needs to be updated.

RESOLVING THE PROBLEM
Please have the customer remove all occurances of the string 

$CANDLEHOME$/$BINARCH$/lib: 
from the file ud.ini in the config subdirectory. 

Backup ud.ini to ud.save.ini 
edit ud.ini and remove the string. In vi enter 
:%s/$CANDLEHOME$\/$BINARCH$/lib:// 
save ud.ini 
Start the UD agent
