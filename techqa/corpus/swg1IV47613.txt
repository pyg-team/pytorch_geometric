APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Environment
   SAP agent, 7.1
   SAP Basis version 701 and 702
   
   PROBLEM
   Overflow condition causes ABAP dumps during CCMS log
   processing by SAP agent. This occurs when the Alert Unique
   Number exceeds 10 digit length.
   in ABAP program "/IBMMON/SAPLITM6" in "/IBMMON/ITM_CCMS_ALERTS":
   
   Short text
       Overflow when converting from 2160022548
   Error analysis
   An exception occurred that is explained in detail below.
   The exception, which is assigned to class
   'CX_SY_CONVERSION_OVERFLOW', was not caught in procedure
   "/IBMMON/ITM_CCMS_ALERTS" "(FUNCTION)", nor was it propagated by
   a RAISING clause.
   Information on where terminated
       Termination occurred in the ABAP program "/IBMMON/SAPLITM6"
   - in "/IBMMON/ITM_CCMS_ALERTS".
       The main program was "SAPMSSY1 ".
   
       In the source code you have the termination point in line
   611 of the (Include) program "/IBMMON/LITM6U59".
   
   
    
   
   

LOCAL FIX
 *  To continue using the agent, disable CCMS alert monitoring
   functionality.
   
   Steps to disable CCMS alerts function module:
   No change on agent side.
   SAP Side:
   1) Transaction Code: SE16
   2) Give the table name as: /IBMMON/ITM_CNFG
   3) Click on Function key F5 or create entries button.
   4) Add the entry of /IBMMON/ITM_ALERTS and NO.
   5) And click on Save.
   
   To check if the entry is stored:
   Transaction Code: SE11
   Give the table name as: /IBMMON/ITM_CNFG
   Click on Function key F7.
   
   And then F8 and the entry will be visible here.
