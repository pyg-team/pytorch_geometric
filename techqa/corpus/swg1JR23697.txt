Title: IBM JR23697: WHEN USING THE ANYTYPE, SIMPLE TYPES ARE NOT RETURNED (SCA) - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  An  SCA(BPEL module) which is exposed as a webservice, The
   interface
   message type is an array of "any" types.
   When complex types are passed into the webservice call,
   everything
   is fine: the types and data are correct.
    However, when a simple type is passed into the webservice call,
   this
   simple type is transformed into a DataObject, AND the data is
   reported
   as lost (value is emptystring).
   
   
    
   
   

LOCAL FIX
 *  no local fix
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: WebSphere Process Server 6.0.1 users         *
   ****************************************************************
   * PROBLEM DESCRIPTION: Without the fix, simple type data       *
   *                      within an xsd:anyType is lost during    *
   *                      SCA web services invocation.            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   An SCA(BPEL module) which is exposed as a webservice, the
   interface message type is an array of "any" types. When
   complex types are passed into the webservice call, everything
   is fine: the types and data are correct.
   However, when a simple type is passed into the webservice call,
   this simple type is transformed into a DataObject, and the
   data is reported as lost (value is emptystring).
   
   
    
   
   

PROBLEM CONCLUSION
 *  Simple type data within an xsd:anyType will not be lost
   during SCA web services invocation.
   
   
    
   
   

TEMPORARY FIX
