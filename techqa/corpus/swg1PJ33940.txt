Title: IBM PJ33940: EAR FILE IS NOT RECREATED AFTER UPGRADING TO 1.1.2.0, WHEN EAR W AS SPECIFIED AS THE DEPLOYMENT TYPE. - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When peforming an upgrade install from WorkplaceXT 1.1.1.x to
   1.1.2.0 and an EAR file was specified for deployment, the
   upgrade installer fails to regenerate the EAR file.
   Is there a workaround? :
   Yes. Once the upgrade installation has completed, the user
   will need to navigate to the deploy directory (Ex. C:\Program
   Files\FileNet\WebClient\deploy) and manually execute the
   create_web_client_ear.bat file. The EAR file can then be
   deployed following the documentation.
   Actual Results:
   The Web Client EAR file does not get regenerated when
   performing an upgrade from 1.1.1.x to 1.1.2.0
   Expected Results:
   For the EAR file to be recreated automatically as part of the
   upgrade install.
