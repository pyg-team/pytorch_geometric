Title: IBM How to use archcmd with SSL - United States

Text:
ICCSAP configuration archcmd SSL HTTPS TECHNOTE (FAQ)

QUESTION
 How can archcmd be used with SSL to obtain the archint.ini from IBM Content Navigator (ICN)? 

CAUSE
By default, archcmd is using the Java runtime shipped with IBM Content Collector for SAP (ICCSAP). This is usually not configured to trust IBM Content Navigator (ICN) and therefore connections using SSL are failing. 

ANSWER
Other than the command shown in ICN, there are some more options available, as the JVM knows about already created keystores and truststores.
The additional options are:
-Djavax.net.ssl.keyStore="/path/to/keystore" -Djavax.net.ssl.keyStorePassword="password" -Djavax.net.ssl.trustStore="/path/to/truststore" -Djavax.net.ssl.trustStorePassword="password"


Assuming the archcmd call provided by ICCSAP ICN plugin looks as below: 

archcmd.sh com.ibm.esd.configplugin.util.DownloadService -url _https://hostname:port/navigator/ -desktop sapadmin -user icmadmin -id 12 


The resulting archcmd call would look as follows: 


archcmd.sh -Djavax.net.ssl.keyStore="/path/to/keystore" -Djavax.net.ssl.keyStorePassword="password" -Djavax.net.ssl.trustStore="/path/to/truststore" -Djavax.net.ssl.trustStorePassword="password" com.ibm.esd.configplugin.util.DownloadService -url _https://hostname:port/navigator/ -desktop sapadmin -user icmadmin -id 12
