Title: IBM cmuexitd fails to start after running makeexits for cmuexits.c. - United States

Text:
STERLINGTRB TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 cmuexitd fails to start after running makeexits for cmuexits.c. 

SYMPTOM


Problem occurs when running Connect:Enterprise UNIX version 2.4.03 Build 236 on AIX 5.3 and makeexits was run using IBM C-compiler version 8.

Error Message  

 

exec(): 0509-036 Cannot load program /sci/connect/dengs1/ce2402GA/aix/bin/cmuexitd because of the following errors:

0509-150 Dependent module libc.a(shr.o) could not be loaded.

0509-022 Cannot load module libc.a(shr.o).

0509-026 System error: A file or directory in the path name does not exist.


RESOLVING THE PROBLEM
Solution 

Change the following line in $CMUHOME/etc/ceustartup and $CMUHOME/etc/profile or where ever you source Enterprise environment variables.

Change:

LIBPATH=$CMUHOME/aix/lib

To:

LIBPATH=$CMUHOME/aix/lib:/usr/lib
