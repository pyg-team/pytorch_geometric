APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  While enabling search optimization with business data variable
   aliases that contain non-ASCII characters, you see an error like
   "ORA-00972 identifier is too long alias column name".
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  When you use UTF8 characters in the alias name for business data
   variables, which need more than 1 byte, and use the
   SchemaGenerator, you might see an error like "ORA-00972
   identifier is too long alias column name" if the bytes needed to
   represent the alias name as the column name in a table are too
   many.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix is available for IBM BPM V8.5.7 cumulative fix 2016.06 on
   request.
   
   A fix will be included in IBM BPM V8.5.7 cumulative fix
   2016.12 that generates non-UTF8 column names for business
   data variable aliases that contain non-ASCII characters.
   
   To determine whether the IBM BPM V8.5.7 cumulative fix 2016.12
   is available and download it if it is, complete the following
   steps on Fix Central:
   
   1. On the Select product tab, select WebSphere as the product
   group, IBM Business Process Manager with your edition from the
   WebSphere options, All as the installed version, and All as the
   platform, and then click Continue.
   2. In the Text field, enter "cumulative fix", and click
   Continue.
