APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The JabberService does not handle on outage of the
   jabberserver well.
   Updates have been made to the JabberServer and JabberDriver
   class files
   to improve the reliability of the JabberServer.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * High CPU usage when the jabber server that the jabber        *
   * service is connected to goes down.                           *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   IF the Jabber Server that the Impact JabberService is connected
   to goes down, CPU pegging occurs.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The high CPU usage issue was fixed by having smack do the
   connection
   and login successfully before allowing echomine to try the
   login.
   The fix for this APAR is contained in the following
   maintenance packages:
   |Fix Pack|5.1.1-TIV-NCI-FP0003
   |Fix Pack|6.1.0-TIV-NCI-FP0002
