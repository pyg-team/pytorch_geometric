APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  This internal APAR resolves the following issues:
   1) "ARYA107E-No useable image copy found for tablespace" for
    Undo SQL recovery plans involving LOB tablespaces.
   2) ARYB209E - SQL error occurred in ARYCRL9D on line 313.
    SQLCODE = -805.
   3) Bind warnings for SQLCODE=-206 TOKENS=TP.LIMITKEY_INTERNAL
    after v11 to v12 migration.
   4) ARYX65535I during the Recovery JCL generation.
   5) PLAN setting is ignored by the batch utility if PIT=CURRENT.
   6) ABEND U0066 for Undo SQL recovery plans.
   7) Typo in ARYR411E message.
   8) Recovery fails trying to recover an object at the SG level.
   9) VSAM ICs are not used as a source of recovery.
   10) Incorrect PLAN IS INVALID message for cloned object.
   11) DSNUM not coded for IC during Fast Replication Recovery.
   12) Contention errors during the parallel recovery of
    COPY YES indexes.
   13) SLR records for auxiliary relations are not updated after
    some ALTERs.
   14) 'Reset ISPF client passwords' does not work.
   15) ARYX093S - An unexpected error occurred
    (./AryAgentRequestGenRecPlans.cpp,294) for empty profiles.
   16) Message "ARYZ090E - Any 'automatic' image copy should be
    of 'Local Primary' type." is issued incorrectly randomly.
   17) Default indexspace bufferpool can't be specified for ARY
    database.
   18) Default value of 'Qualifier for Db2 Automation Tool objects'
    is changed from 'HAA' to 'DLC'.
