PROBLEM(ABSTRACT)
 Error -- "java.util.MissingResourceException: Can't find bundle for base name resources/extn/extnbundle, locale en_KR" 

SYMPTOM


User are getting the following error - "java.util.MissingResourceException: Can't find bundle for base name resources/extn/extnbundle." This exception is displayed for all the locales configured in the Sterling application.

RESOLVING THE PROBLEM


This error occurs if both extnbundle.properties and extended extnbundle.properties are not present in the EAR (under resources.jar).

Check the resources/extn folder for the presence of extnbundle.properties or extended extnbundle.properties after exploding the EAR. If not present, place the appropriate properties file, rebuild the resources.jar, rebuild the EAR and redeploy. Please refer to Sterling Installation and Customization Guides for further information on building the jars, EAR and deployment.
