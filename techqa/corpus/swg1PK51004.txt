LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users of service integration bus links       *
   *                 between messaging engines on different       *
   *                 buses in the default messaging provider in   *
   *                 WebSphere Application Server 6.x             *
   ****************************************************************
   * PROBLEM DESCRIPTION: The service integration bus link        *
   *                      between messaging engines on            *
   *                      different buses does not recover if     *
   *                      one of the messaging engines is         *
   *                      restarted.                              *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   If one of the messaging engines at either end of a service
   integration bus link is restarted, the service integration bus
   link does not recover successfully.  A CWSIT0075E: THE LINK
   UUID <VALUE> IS ALREADY DEFINED error is thrown and the
   service integration bus link does not restart successfully.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This fix resolves the problem by enabling a service
   integration bus link to recover from a messaging engine being
   restarted.
