Title: IBM -804 abend when trying to browse either SYSIBM.SYSTABLESPACESTATS or SYSINDEXSPACESTATS with Optim 5.5.2 - United States

Text:
BIGINT; DECFLOAT TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 -804 abend when trying to browse either SYSIBM.SYSTABLESPACESTATS or SYSINDEXSPACESTATS with Optim 5.5.2 

SYMPTOM
-804 abend when trying to browse either SYSIBM.SYSTABLESPACESTATS or SYSINDEXSPACESTATS with Optim 5.5.2

Error message similar to this will be logged:

DSNT408I SQLCODE = -804, ERROR: AN ERROR WAS FOUND IN THE APPLICATION PROGRAM INPUT PARAMETERS FOR THE SQL STATEMENT, REASON 16 DSNT418I SQLSTATE = 07002 SQLSTATE RETURN CODE DSNT415I SQLERRP = DSNXROHB SQL PROCEDURE DETECTING ERROR DSNT416I SQLERRD = -319 0 0 -673594048 30 0 SQL DIAGNOSTIC INFORMATION DSNT416I SQLERRD = X'FFFFFEC1' X'00000000' X'00000000' X'D7D9C540' X'0000001E' X'00000000' SQK DIAGNOSTIC


CAUSE
Optim 5.5.2 does not support the BIGINT or DECFLOAT data types. 

ENVIRONMENT
Optim for zOS v5.5.2 and earlier, DB2 for zOS v9

DIAGNOSING THE PROBLEM
Attempts to browse either SYSIBM.SYSTABLESPACESTATS or SYSINDEXSPACESTATS with Optim 5.5.2 will fail with a -804 error.

Error message similar to this will be logged:

DSNT408I SQLCODE = -804, ERROR: AN ERROR WAS FOUND IN THE APPLICATION PROGRAM INPUT PARAMETERS FOR THE SQL STATEMENT, REASON 16 DSNT418I SQLSTATE = 07002 SQLSTATE RETURN CODE DSNT415I SQLERRP = DSNXROHB SQL PROCEDURE DETECTING ERROR DSNT416I SQLERRD = -319 0 0 -673594048 30 0 SQL DIAGNOSTIC INFORMATION DSNT416I SQLERRD = X'FFFFFEC1' X'00000000' X'00000000' X'D7D9C540' X'0000001E' X'00000000' SQK DIAGNOSTIC

RESOLVING THE PROBLEM
Upgrade to Optim v6.1.1 or higher
