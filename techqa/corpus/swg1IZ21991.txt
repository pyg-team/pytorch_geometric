APAR STATUS
 * CLOSED AS DOCUMENTATION ERROR.
    
   
   

ERROR DESCRIPTION
 *  Documentation describes performance improvement incorrectly;
   
   If the Domain Name Server (DNS) is not resolving IP addresses
   quickly,
   you can improve performance by
   setting the NoNameResolution property to 0.
   
   
   NoNameResolution should be set to 1 so that DNS is not used
   
   
    
   
   

LOCAL FIX
 *  set NoNameResolution : 1
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All users of the for SNMP EMS Probe.         *
   ****************************************************************
   * PROBLEM DESCRIPTION: The IBM Tivoli Netcool/OMNIbus SNMP EMS *
   *                      Probe Reference Guide provides          *
   *                      incorrect information about the         *
   *                      NoNameResolution property.              *
   ****************************************************************
   * RECOMMENDATION:                                              *
   *                                                              *
   ****************************************************************
   The description for the NoNameResolution property should be
   changed as follows:
   
   If the Domain Name Server (DNS) is not resolving IP addresses
   quickly, you can improve  performance by setting the
   NoNameResolution property to 1.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The product documentation will be updated with the next release
   of the probe.
   
   
    
   
   

TEMPORARY FIX
