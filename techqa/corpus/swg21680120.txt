Title: IBM BMXAA7025E When Installing TAMIT - United States

Text:
TAMIT install; BMXAA7025E; TPAEUPDATEDB; 7210_23 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempts to install TAMIT on top of 7.1.1.12 results in the error BMXAA7025E during updatedb. 

SYMPTOM
After installing Maximo Asset Management 7.1.1.9 or later and then installing TAMIT to the same environment, will fail on updatedb with the error:
BMXAA7025E -- Script failed while running.;
sql#942;
ORA-00942: table or view does not exist

-- BEGINUPDATEDBERROR
-- Error running file: V7210_23
-- UPDATEDBFILE:V7210_23.dbc
-- Script: Error
psdi.util.MXSystemException: dbchange#ScriptFailure


CAUSE
The dependency for which scripts to run first is incorrect in the newer versions.

ENVIRONMENT
When installing TAMIT 7.2 on top of TPAE you will run in to this error.

DIAGNOSING THE PROBLEM
When installing TAMIT and the install fails, check the updatedb log. The full transcript of the error will look something like this, depending on database platform:

RESOLVING THE PROBLEM
You can change the dependencies. Update file c:\ibm\smp\maximo\applications\maximo\properties\product\tsd_common.xml so that the depends section prereqs contentcatalog.
The tsd_common.xml file should change from:
<depends>common_pmp,search_pmp</depends>
to:
<depends>common_pmp,search_pmp,contentcatalog</depends>
