Title: IBM Content Engine error adding Document Title with parentheses "()" - United States

Text:
XT SRVE0293E parameter invalid value web.xml TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 SRVE0293E parameter has an invalid value error returned when Document Title contains "()" 

SYMPTOM
When adding a document from Workplace XT and using parentheses in the Document Title, the following error is logged in the WebSphere SystemOut.log file:

[7/12/13 10:27:34:906 PDT] 00000012 webapp E com.ibm.ws.webcontainer.webapp.WebApp logServletError SRVE0293E: [Servlet Error]-[faces]: com.filenet.wcm.api.BaseRuntimeException: The FORM:PROPERTY_GRIDDocumentTitle parameter has an invalid value.



CAUSE
An obsolete entry was found in the web.xml file for Workplace XT causing a security filter to be invoked to scan the parameters for specific types of tags and content.

ENVIRONMENT
CE 5.1.0-003
WorkplaceXT 1.1.5-001
WAS 7.0.0.27

RESOLVING THE PROBLEM
The error referred to the FORM:PROPERTY_GRIDDocumentTitle parameter which was found in the following PageSecurityFilter entry in the web.xml file:

<filter>
<filter-name>PageSecurityFilter</filter-name>
<filter-class>com.filenet.ae.toolkit.server.servlet.filter.SecurityFilter</filter-class>
<init-param>
<param-name>parameters</param-name>
<param-value>
FnAjaxMethod,FnParam_browserOffset1,FnParam_browserOffset2,clientId,fieldId,wId,wIdField,actionId,page,objectType,favoritesMode,keywordMode,storedSearchMode,targetUrl,


actionObjectId,actionObjectStoreName,actionObjectType,droptarget,op,_responseURL,FnParam_path,dataType,treeNodeUid,appType,fileCount,id,isCheckin,jsfViewId,multiSelectAction,
sessionKey,targetTreeNodePath,objectStoreName,classId,label,FnParam_actionObjectStoreName,level,
FORM:PROPERTY_GRIDDocumentTitle
</param-value>
</init-param>
</filter>

This entry was carried over from an older version of the web.xml file and was no longer needed. The entry was removed, Workplace XT was redeployed and the issue was resolved.
