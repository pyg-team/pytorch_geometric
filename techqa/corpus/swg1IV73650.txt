ERROR DESCRIPTION
 *  Uninstalled PVU based software in TAD4d not updated by ITIC in
   SCCD
   
   PERFORMANCE ISSUE
   No
   
   STEPS TO REPRODUCE:
   *Taking IBM Websphere Application Server Network Deployment as
   example.
   1. Sign TAD4d audit report on 02 May 2015 - for 01 Apr to 30
   Apr.
   (a) Run ITIC import for TAD4d after 02 May 2015.
   (b) Login to SCCD and run PVU Sub Capacity Based
   Audit Report (pvusubaudit.rptdesign).
   - Find one computer with IBM Websphere Application Server
   Network Deployment installed.
   - e.g. COMPA
   2. Uninstall IBM Websphere Application Server Network
   Deployment from COMPA on 05 May 2015.
   3. Sign TAD4d audit report on 12 May 2015 - for 01 May to 10
   May.
   (a) Run ITIC import for TAD4d after 12 May 2015.
   *There is a possibility COMPA is still show as installed in
   this report depending on the PVU high water mark in TAD4d.
   4. Sign TAD4d audit report on 22 May 2015 - for 11 May to 20
   May.
   *In order to confirm, open signed TAD4d audit report on 22 May
   2015 and make sure COMPA is not reported.
   (a) Run ITIC import for TAD4d after 22 May 2015.
   (b) Login to SCCD and run PVU SUb Capacity Based
   Audit Report (pvusubaudit.rptdesign).
   (c) COMPA is still having discovered capacity in PVU Sub
   Capacity Based Audit Report (pvusubaudit.rptdesign) <-- The
   issue.
   
   CURRENT ERRONEOUS RESULT:
   Uninstalled PVU based software in TAD4d not removed by ITIC in
   SCCD even it no longer appears in signed TAD4d audit report.
   
   EXPECTED RESULT:
   Uninstalled PVU based software in TAD4d should be removed by
   ITIC in SCCD if it no longer appears in signed TAD4d audit
   report.
