Title: IBM Resolving JMSCMQ0001: WebSphere MQ call failed  when connecting  to MQ version  7.5 - United States

Text:
MQ; CHANNEL; TPAEINTEGRATION TECHNOTE (FAQ)

QUESTION
 How to resolve JMSCMQ0001: WebSphere MQ call failed with compcode '2' ('MQCC_FAILED') reason '2035' ('MQRC_NOT_AUTHORIZED') when configuring Maximo to use an MQ queue 

CAUSE
The QMGR configuration feature in MQ 7.5

ANSWER
When configure the Queue Connection Factory by navigating to JMS > Queue Connection Factories and providing all required attributes values for WebSphere MQ messaging provider and then testing a connection to MQ the error JMSCMQ0001: WebSphere MQ call failed with compcode '2' ('MQCC_FAILED') reason '2035' ('MQRC_NOT_AUTHORIZED') is present. 

To resolve : 

Navigate to ibm\WebSphere MQ\bin and run runmqsc maximo (maximo is QMGR name) 

DISPLAY QMGR notice configuration parameter CHLAUTH(ENABLED) That means that Channel Authorization Enabled. Please open the attached Pic1.bmp file. 

run 

ALTER QMGR CHLAUTH(DESABLED) 

run 

DISPLAY QMGR noticed CHLAUTH(DISABLED) 

Restart MQ and Application Server and retest connection
