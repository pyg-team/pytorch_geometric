APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  There is a need to address the detection of the bad
   certificate, early in the Fix Pack install process, and
   rollback (prior to Impact code changes get placed on the
   environment).
   
   
    
   
   

LOCAL FIX
 *  Fix for this will be included in a future Fix Pack release
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * There are occasions when a Client might go through the       *
   * process of renaming their Impact servers hostnames.  If they *
   * do so, and forget to update the keystore with the            *
   * appropriate values; the next time a command (such as         *
   * nci_version_control or nci_trigger) is run, it will fail.    *
   * When running the nci_version_control "uncoall" command, the  *
   * following exception is being generated:                      *
   *                                                              *
   * ./opt/IBM/tivoli/impact/bin/nci_version_control NCI uncoall  *
   * impactadmin                                                  *
   * javax.net.ssl.SSLHandshakeException:                         *
   * java.security.cert.CertificateException: No name matching    *
   * <hostname>  found                                            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   he nci_version_control is used during the Fix Pack upgrade
   process, to unlock all of the locked files (such as policies)
   within impact.
   
   If the nci_version_control fails to execute properly, the Fix
   Pack installer will attempt to rollback the installation but,
   there will be occasions where the rollback will leave the
   environment in a unstable state; requiring manual actions to put
   it back to the proper state.
   
   To address this issue, the Impact Health Check logic has been
   intetrated into the latest Impact 7.1.0 Fix Pack.  This logic is
   also avilable via a download (for Clients not updating to Fix
   Pack 10 or above).
