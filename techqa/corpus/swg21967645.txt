Title: IBM After Rebooting the server, the server is not coming up - United States

Text:
WAS; recovery; restart TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempts to restart IBM Collaborative Lifecycle Management (CLM) application it fails with the CRJAZ1972E error in application server log. 

SYMPTOM

 * Attempting to display application admin page fails with timeout exception
   
   
 * You see the following error in the <WAS-PROFILE>/logs/systemout.log file:

CAUSE
Your IBM Websphere Application Server (WAS) temp directory is corrupted. The possible root cause of corruption might be forced termination of the WAS process.

ENVIRONMENT
WAS as application server



RESOLVING THE PROBLEM
WORKAROUND: 

 1. Stop WAS.
    
    
 2. Remove the contents from these directories under the profile root(e.g. <WAS_HOME>/<AppServer>/profiles/<Profile>), except for the noted help directory: temp, wstemp, workspace .
    
    
    Note: Do not remove this directory, which is required for the CLM local help: temp/{jts_node}/{jts_server}/clmhelp_war .
    
    
 3. Restart WAS.
