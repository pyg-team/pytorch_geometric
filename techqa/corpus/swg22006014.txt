Title: IBM Security Bulletin: The BigFix Platform versions 9.1 and 9.2 have security vulnerabilities that have been addressed via patch releases - United States

Text:
 SECURITY BULLETIN

SUMMARY
 The BigFix Platform verions 9.1 and 9.2 have some vulnerabilities associated with the zlib library, as well as Cross Site Scripting and XML External Entity that have been addressed in patch releases 9.1 patch 11 and 9.2 patch 11 respectively 

VULNERABILITY DETAILS

DESCRIPTION: zlib is vulnerable to a denial of service, caused by an out-of-bounds pointer arithmetic in inftrees.c. By persuading a victim to open a specially crafted document, a remote attacker could exploit this vulnerability to cause a denial of service.

AFFECTED PRODUCTS AND VERSIONS
BigFix Platform Version 9.1, BigFix Platform Version 9.2

REMEDIATION/FIXES
 



CVE-2016-9840, CVE-2016-9841, CVE-2016-9842, CVE-2016-9843, CVE-2017-1219:
BigFix Platform 9.1, BigFix Platform 9.2: 

To remediate these vulnerabilities please apply the upgrade-patch (9.1 patch 11 or 9.2 patch 11) by looking for the associated upgrade-patch fixlet in the Console, launching and applying it. 

CVE-2017-1203:
Case Set 1:

BigFix Platform 9.1, BigFix Platform 9.2: 
(i) Cross site scripting - Console - Client Compliance Policy Wizard is not protected for java script injection 
(ii) Cross site scripting - Server/Relay - BESServer/Relay URL are not protected against javascript injection 
(iii)Cross Site Scripting - Console Fixlet import can accept java injection in description

To remediate these vulnerabilities please apply the upgrade-patch (9.1 patch 11 or 9.2 patch 11) by looking for the associated upgrade-patch fixlet in the Console, launching and applying it. 

Case Set 2:

(i) Cross site scripting - Server/Relay - Unauthenticated client traffic

BigFix Platform 9.1/BigFix Platform 9.2:
To remediate these vulnerabilities please apply the upgrade-patch (9.1 Patch 11 or 9.2 Patch 11) by looking for the associated upgrade-patch fixlet in the Console, launching and applying it. Then follow the following post configuration step on the System where the Primary BigFix Server is installed.

Update or set the new masthead parameter "minimumSupportedClient=9.0" using the the following BESAdmin tool command:

On Windows:
BESAdmin.exe /minimumSupportedClient=<VALUE> /sitePvkLocation=<LICENSE_PVK_FILE> /sitePvkPassword=<LICENSE_PVK_PWD>

On Linux:
/opt/BESServer/bin/BESAdmin.sh -minimumSupportedClient=<VALUE> -sitePvkLocation=<LICENSE_PVK_FILE> -sitePvkPassword=<LICENSE_PVK_PWD>


NOTE: Any clients at version 8.2 or lower in the deployment will stop working as soon as minimumSupportedClient is set to 9.0.

Please see the 9.1 Patch 11 and 9.2 Patch 11 Release notes and documentation for additional information about the minimumSupportedClient parameter. 

Case Set 3:
WebUI:

For customers on running BigFix 9.2.6 or greater and running WebUI please refer to the following Security Bulletin for instructions on how to remediate the WebUI cases associated with this vulnerability (CVE-2017-1203) here: http://www-01.ibm.com/support/docview.wss?uid=swg22005246 [http://www-01.ibm.com/support/docview.wss?uid=swg22005246]
