   Reproducible Y/N: Y
   
   Description of problem:
   Modification of a table comment on database side is not properly
   managed when the Rose model is synchronized.
   
   Steps to reproduce:
   A1. Modify only a comment (more than 300 characters) of a
   table on the database side.
   A2. Synchonize the Rose model with the database
   A3. No difference is displayed
   
   B1. Modify a comment (more than 300 characters) and a column
   of a table on the database side
   B2. Synchronize the Rose model with the database
   B3. Differences are displayed for both the column and the
   comment
   B4. Import all into the model
   B5. the column modification is imported but not the comment one
   
   C1. Modify on the database side a small comment of a table
   which has a primary key
   C2. Synchronize the Rose model with the database
   C3. Import the difference into the model
   C4. the comment modification is not imported
   C5. the name of the constraint for the primary key has been
   renamed to PK_<table name>
