PROBLEM SUMMARY
 *  In a Tivoli Integrated Portal Single Sign On environment, it is
   not possible to bypass the logon dialog when launching multiple
   Tivoli Enterprise Portal instances.
   
   Enabling Single Sign On does not keep the logon dialog from
   appearing when using Java Webstart to launch Tivoli Enterprise
   Portal from Tivoli Integrated Portal.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The file that launches Tivoli Enterprise Portal was changed so
   it would allow the Single Sign On configuration to work
   correctly.
   
   
   The fix for this APAR is contained in the following maintenance
   packages:
   
     | fix pack | 6.2.2-TIV-ITM-FP0008
     | fix pack | 6.2.3-TIV-ITM-FP0001
   
   
    
   
   

TEMPORARY FIX
 *  Make the following change in the tep.jnlp file located in
   <CANDLEHOME>\CNB on Windows or in <CANDLEHOME>/cw on UNIX
   
   1-Locate the following lines near the top of the file:
   <jnlp
   spec="1.0+"
   codebase="http://$HOST$:$PORT$/"
   href="tep.jnlp">
   
   2-Remove the 'href' attribute to fix the problem (do not forget
   to keep the ending angle bracket):
   <jnlp
   spec="1.0+"
   codebase="http://$HOST$:$PORT$/">
