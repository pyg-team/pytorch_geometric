Title: IBM Special characters cause syntax error in the ESQL - United States

Text:
MQIINFO syntax error ESQL dash TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The following ESQL generates a syntax error in IBM Integration Bus (IIB) or WebSphere Message Broker (WMB):

SET OutputRoot.XML.TestMsg.Field-1 = 'Hello World'; 

CAUSE
The problem is caused by the dash in the field 'Field-1'. 

RESOLVING THE PROBLEM
 When using special characters you must encapsulate the field names within double quotes. 

Example:
SET "OutputRoot"."XML"."TestMsg"."Field-1" = 'Hello World';
