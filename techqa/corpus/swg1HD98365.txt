Title: IBM HD98365: The CSLD crawler does not recognize the correct size of a Lotus Domino database if it is greater than 2 GB and less than 4 GB - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Notes mail db size obtained by crawler is incorrect when the
   mail db size is greater than 4294967295 bytes
    The DB size we got is from the return value of the function
   LNINT LNDatabase::GetSize(),    (ININT is definded as unsigned
   32-bit integer value of which can  be 0  ~ 4*1024*1024*1024-1)
   
   
    
   
   

LOCAL FIX
 *  Disable select databases by size
   
   
    
   
   

PROBLEM SUMMARY
 *  The CSLD crawler did not always recognize the actual size of the
   Lotus Domino database. For databases greater than 2 GB and
   less than 4 GB,the database size reported in the trace for
   the crawler was a negative value.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The CSLD crawler was changed to correct this problem.
