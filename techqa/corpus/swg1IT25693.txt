PROBLEM SUMMARY
 *   The following problems are resolved in FixPack
   V5.3.1.15:
   APAR IT24026 - Mitigation for the following security
                  vulnerabilities in OpenSSL Malformed X.509
                  IPAddressFamily could cause OOB read
                  (CVE-2017-3735).
   APAR IT24145 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT24145] - In a situation where report messages are re-route
                  to a dead letter queue they may have an incorrect
                  'format' field in their MQDLH header.
   APAR IT24388 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT24388] - MQ for NonStop server, reinitialisation of
                  multiple slave repository mangers in parallel
                  results in deadlock,  FDC RM220005 with AMQ9511
                  and AMQ9448 logged.
   
   APAR IT24742 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT24742] - Enabling of FORMAT2 support for queue files.
   APAR IT24999 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT24999] - If a cluster queue is set to PUT(DISABLED) and
                  messages are in transit the target queue manager
                  can throw an FDC "incorrect application process
                  opener context stored in mqopen for mqput" or the
                  cluster channel to this queue manager may be
                  terminated.
   APAR IT25657 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25657] - SDCP terminates without having created output or
                  incomplete output.
   APAR IT25659 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25659] - Non-native applications may abend in READX, due
                  to implicit cast of readcnt to the wrong type.
   
   APAR IT25662 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25662] - Messages can be put on PUT(DISABLED) local queues
                  if their CLUS() attribute is set.
   APAR IT25665 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25665]
   APAR IT25666 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25666]
   APAR IT25669 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25669]
   APAR IT25671 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25671] - Errors AMQ9511 and AMQ9448 are logged along with
                  FFST regarding probeid: RM220005 and probe type:
                  AMQ9511.
   APAR IT25667 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25667] - AMQRRMFA processes create redundant opens to othe
                  AMQRRMFA processes, when one reinitializes due to
                  an error (error logged AMQ9448).
   APAR IT25677 [http://www-01.ibm.com/support/docview.wss?uid=swg1IT25677] - When a close message is overtaken by a cpu down
                  message the cleanup done as a reaction to the clo
                  will fail since the data it tries to access is no
                  valid anymore, resulting in error 4022 when tryin
                  to lock a mutex.
   APAR IT25692 - Mitigation for the following vulnerabilities in
                  OpenSSL Constructed ASN.1 types with a recursive
                  definition could exceed the stack (CVE-2018-0739)
