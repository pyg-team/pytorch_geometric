APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When there are many MQ front side handlers on a single service
   and the either the MQ connection is broken, or the service is
   removed and attempted to be restarted, DataPower may see CPU
   usage grow.
   
   It is possible this cpu usage will remain high causing
   unresponsiveness until the service or domain is manually
   disabled.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  CPU might be high, when MQ connections are broken.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in 7.0.0.17, 7.1.0.14, 7.2.0.11, 7.5.0.5,
   7.5.1.4 and 7.5.2.2
   
   For a list of the latest fix packs available, please see:
   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
