APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  the admin-password not being decrypted when sent to external
   endpoint. During searchDN, password is correctly decrypted
   before hitting the LDAP server. However, during auth-group, the
   password is selected without decrypting it.
   
   
    
   
   

LOCAL FIX
 *  If user disable the "Use Authorization Group" it works fine.
   
   
    
   
   

PROBLEM SUMMARY
 *  LDAP Authorization group was previously not fully functional.
   Configured password would not be correctly processed and passed
   onto ldap endpoint. In addition, user in multiple authorization
   group would be erroneously marked as invalid.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix for LDAP Authorization group was the proper processing of
   the username and password during authorization group step along
   with fixing the erroneously marked user in multiple groups. Fix
   will be available all 508x branches going starting from 5083.
