Title: IBM Scheduled reports are not running in Cognos Express - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Scheduled reports will run manually but not when scheduled. 

What are the steps for running NC_Drop for Cognos Express? 

SYMPTOM
Scheduled reports are not running


RESOLVING THE PROBLEM
1) Backup your Cognos Express content Store(http://www-01.ibm.com/support/knowledgecenter/SSMR4U_10.2.1/com.ibm.swg.ba.cognos.ig_exprss.10.2.1.doc/t_backingupexpresscontent.html%23Backing [http://www-01.ibm.com/support/knowledgecenter/SSMR4U_10.2.1/com.ibm.swg.ba.cognos.ig_exprss.10.2.1.doc/t_backingupexpresscontent.html%23Backing]upExpresscontent)
2. Open a command prompt and go to location:
<cx_install_root>/ids/
3. List the files in that directory and run 'cm_ids_svr_custom.cmd' or run it straight from the directory
4. It should then run and take you back to the prompt.
5. Run command 'dbaccess'
6. Use the arrow keys and go to Menu 'Database' -> select the content store database name
7. You are prompted for username and password. Use username - icognosexpress and password - (Blank)
8. Select the 'Use SQL Query language' menu using arrow keys
9. Using arrow keys, select the manu 'Use Editor'
10. A notepad window should open
11. Open Windows explorer and go to '<Install Location>\configuration\schemas\delivery' -> Informix
12. Open the NC_Drop sql script in notepad -> copy all the sql lines and paste it into notepad opened in step 11.
13. Menu File -> Save
14. Go to Menu Run and Run the script. The script should run and return message that the tables are altered successfully.
