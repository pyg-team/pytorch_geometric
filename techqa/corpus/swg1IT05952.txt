APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  With the compression option in HTTP front side handler on, a
   returned HTTP 500 response might include an extra '0' in the end
   of message even there is no Transfer-encoding:  chunked header.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Affected is the use of a Web Service Gateway when an input
   message does not match any of the operations tied to a port.
   
   If that port has compression turned on on the Front Side
   Handler, an extra '0' might be returned with the HTTP 500
   response.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in 6.0.0.14, 6.0.1.10, 7.0.0.7 and 7.1.0.5.
   
   For a list of the latest fix packs available, please see:
   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
