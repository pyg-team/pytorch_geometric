PROBLEM(ABSTRACT)
 Use this procedure to trace the Configuration Manger startup. 

Gathering (Mustgather) this information before calling IBM Support will help familiarize you with the troubleshooting process and save you time. 

RESOLVING THE PROBLEM
 
Tracing the Configuration Manager:

1. Stop the Configuration Manager.


 * mqsistop <configmgr name> 
 * 

2. Start trace.  * 
 * mqsiservice <configmgr Name> -r TraceSize=2097151 
 * 
 * mqsiservice <configmgr name> 
 * -r Trace=debug


3. Start the Configuration Manager. 
 * mqsistart <configmgr name>
 * 

4 Recreate the error. 

5. Stop the Configuration Manager. Make sure the Configuration Manager is stopped. 
 * mqsistop <configmgr name> 


6. Retrieve the trace log for the specified components. 
 * mqsireadlog <configmgr name> -t -b service -f -o service_configmgr.xml
   mqsireadlog <configmgr name> -t -b agent -f -o agent_configmgr.xml


7. Format the XML log files. 
 * mqsiformatlog -i service_configmgr.xml -o service_configmgr.txt
   mqsiformatlog -i agent_configmgr.xml -o agent_configmgr.txt
 * 

8. Stop trace. 
 * mqsiservice <configmgr name> -r Trace=none
 * 
 * mqsiservice <ConfigMgr Name> -r TraceSize="" 


9. After a PMR is open, submit the files service_configmgr.txt and agent_configmgr.txt to IBM Support for problem determination.
