APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If you have tuned your eWAS and add an Xmn value it could cause
   eWAS to fail. For example, you added the following to your
   server.xml :-
   
   -Dsun.rmi.dgc.ackTimeout=10000 -Xdisableexplicitgc
   -Xgcpolicy:gencon
   -Xmn1024m -DSTANDALONE" executableJarFileName=""
   disableJIT="false">
   
   The installation script does not take the Xmn value into account
   and can cause a bad value. This causes eWAS to fail in start up
   with error :-
   
   JVMJ9GC019E -Xmn too large for -Xms
   JVMJ9VM015W Initialization error for library j9gc26(2): Failed
               to initialize
   Error: Could not create the Java Virtual Machine.
   Error: A fatal exception has occurred. Program will exit.
   
   
    
   
   

LOCAL FIX
 *  Remove Xmn value from your server.xml before upgrade.
   
   
    
   
   

PROBLEM SUMMARY
 *  Fixed TWS code to set the Xms value taking in account also the
   Xmn value set on the WebSphere profile.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This apar will be fixed in 9.2.0-TIV-TWS-FP0002,
   9.1.0-TIV-TWS-FP0003
   
   
    
   
   

TEMPORARY FIX
 *  Remove Xmn value from your server.xml before upgrade.
