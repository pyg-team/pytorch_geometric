PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users that use RDz 7.5.1                     *
   ****************************************************************
   * PROBLEM DESCRIPTION: The following error was logged and the  *
   *                      client could not log on RDz Daemon.     *
   *                      FEK117E socketpair() failed.            *
   *                      reason=(EDC5127I Too many open files    *
   *                      in system.)                             *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The Server process could not handle the client's logon request
   owing to NullPointerException.
   So, the Daemon created a lot of Server processes and exhausted
   the pipe resources.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Now, the Server process is modified not to cause
   the NullPointerException.
