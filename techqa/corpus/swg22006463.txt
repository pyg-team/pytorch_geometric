Title: IBM Security Bulletin: IBM Development Package for Apache Spark might create a remote exploitation vector against old Internet Explorer browsers through XSS - United States

Text:
 SECURITY BULLETIN

SUMMARY
 IBM Development Package for Apache Spark addresses the following vulnerability.

The vulnerability is a potential cross-site scripting (XSS) attack on a Web UI client; server-side analytical processing by Apache Spark is not affected and data is not compromised. 

VULNERABILITY DETAILS
CVEID: CVE-2017-7678 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2017-7678]
DESCRIPTION: Apache Spark is vulnerable to cross-site scripting, caused by improper validation of user-supplied input by the web UI. A remote attacker could exploit this vulnerability using a specially-crafted URL to execute script in a victim's Web browser within the security context of the hosting Web site, once the URL is clicked. An attacker could use this vulnerability to steal the victim's cookie-based authentication credentials.
CVSS Base Score: 6.1
CVSS Temporal Score: See https://exchange.xforce.ibmcloud.com/vulnerabilities/128608 [https://exchange.xforce.ibmcloud.com/vulnerabilities/128608] for the current score
CVSS Environmental Score*: Undefined
CVSS Vector: (CVSS:3.0/AV:N/AC:L/PR:N/UI:R/S:C/C:L/I:L/A:N) 

AFFECTED PRODUCTS AND VERSIONS
 

Affected IBM Development Package for Apache Spark Affected Versions IBM Development Package for Apache Spark, v1.x All versions IBM Development Package for Apache Spark, v2.x Version 2.0.0.0 - 2.1.1.0 REMEDIATION/FIXES
 

IBM Development Package for Apache Spark Remediated Versions IBM Development Package for Apache Spark, v2.x Version 2.1.1.1, or later 
For IBM Development Package for Apache Spark, v1.6.3.1 and earlier versions, IBM recommends upgrading to a fixed, supported version of the product. 
WORKAROUNDS AND MITIGATIONS
Use only modern/latest browsers, such as FireFox or Chrome, when interacting with the Web UI ; or, ensure that any potentially affected Internet Explorer browsers are patched to current standards. 

Alternatively, avoid use of the Web UI (Off by default) until you can apply a fixed version.
