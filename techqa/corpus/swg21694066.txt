Title: IBM Oracle Gateway failed to start after changing soft and hard limits - United States

Text:
oracle gateway soft hard limits tab chk insufficient space E-REG-002-029 E-UNK-000-000 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The soft and hard limits was increased and decreased and then the gateway failed to start up. 

SYMPTOM
Warning: W-UNK-000-000: Statistics: Overload threshold exceeded. (97.575488>=10)
Error: E-REG-002-029: Soft limit exceeded for region : status_upd_workarea
Error: E-UNK-000-000: WorkAreaQData: Region 'status_upd_workarea': Insufficient space in data segment. Attempting to take data off-line.


CAUSE
The cache directory of the gateway still have the old .tab and .chk files with the previous setting for the soft and hard limits.

DIAGNOSING THE PROBLEM
Check and compare the size of the .tab and .chk file against the soft and hard limits setting in the gateway props file.

RESOLVING THE PROBLEM
Stop the gateway process if it is still running.
Delete the cache directory of the gateway "$OMNIBUS/var/nco_g_oracle" .
Restart the gateway and the cache directory will be recreated on startup.
