Title: IBM PJ38957: A search template cannot be added to object store if the Marking or Choice list property view is set to Hidden or Read Only. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Search template cannot be saved when it includes hidden or read
   only Marking or Choice list property.
   
   Steps to reproduce:
   
      1. Create a Marking Set at the P8 domain level.
      2. Create a new CE string property template using FileNet
   Enterprise Manager and
   assign the marking set created in step 1 when creating the
   property template.
      3. Assign the new marking property template to at least one
   document class using FileNet Enterprise Manager.
   
      4. Log on to Workplace as a user that is able to run Search
   Designer.
      5. In Search Designer, select at least one object store to
   search on the Object Stores tab, and then click the Search
   Criteria tab.
      6. Select "Hidden" or "Read only" from the View drop-down
   menu on
   the Search Criteria tab.
      7. Select the Marking property created in step 2 from the
   Property drop-down menu on the Search Criteria tab.
   
      8. Select "is not null" from the Operator drop-down menu on
   the
   Search Criteria tab.
      9. If you attempt the "Add new" operation (either by
   clicking the button on the Search Designer toolbar or from the
   File menu), an empty dialog box is displayed with an "i" icon
   in it.  See below.
   
   Expected to be able to save the search template.
   
   The impact to the customer/system:
   Unable to create search templates with certain options when
   searching on a marking property.
