Title: IBM PI44996: DEFAULT WCM RICH TEXT CKEDITOR FAILS TO INSERT HTML TABLES USING THE MENU BUTTON - United States

Fixes integrated in WebSphere Portal 8.5.0.0 Combined Cumulative Fixes [http://www-01.ibm.com/support/docview.wss?uid=swg24037786]



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The default WCM rich text editor ( CKEditor ) fails to insert
   HTML tables using the menu button "Insert Table" in both
   authoring portlet as well as inplace editing.
   
   The following error messages can happen:
   "Unable to get property 'undefined' of undefined or null
   reference."
   or
   "TypeError: _359 is undefined"
   at
   /wps/PA_WCM_Authoring_UI/layers/full.js Line 2
   or
   "TypeError: _12f is undefined"
   at
   ...head.js Line 7253, col 4
   
   The problem was introduced with CF06 for Portal 8.5.0.0.
   
   
    
   
   

LOCAL FIX
 *  Use HTML source code editing instead and manually enter the
   HTML code for the table construct.
