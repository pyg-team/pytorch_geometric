Title: IBM InfoSphere Content Collector Web Application fails to retrieve emails. - United States

Text:
ICC; FilterElement TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 I am trying to retrieve archived emails using IBM InfoSphere Content Collector by accessing the link to a stubbed message but I am getting a blank page instead of actual email? 

SYMPTOM
The following error can be seen from the IBM InfoSphere Content Collector Web Application log. 

For example, in 
<Content Collector installDir>\WebApplication\afu\logs\trace.log :

[2009-09-23] [09:57:31.161] 24 WebContainer : 0 SEVERE java.lang.NoSuchMethodError: com/filenet/api/property/FilterElement.<init>(Ljava/lang/Integer;Ljava/lang/Long;Ljava/lang/Boolean;Ljava/lang/String;Ljava/lang/Integer;)V [com.ibm.afu.session.AFUContextContainer: getRepositoryService(442)]
com.ibm.afu.session.AFUContextContainer.init(AFUContextContainer.java:367)
com.ibm.afu.session.AFUContextContainer.getRepositoryService(AFUContextContainer.java:439)


CAUSE
If your FileNet P8 Content Engine Server and FileNet P8 Content Engine Client versions are different you will have this problem.


ENVIRONMENT
IBM InfoSphere Content Collector with FileNet P8 Content Engine 4.0 as the archive repository



DIAGNOSING THE PROBLEM
The validation of the configuration web services is successful and P8 connector is correctly chosen for the web application. 



RESOLVING THE PROBLEM
 

 1. Upgrade the FileNet P8 Content Engine Client to match with FileNet P8 Content Engine Server on the IBM InfoSphere Content Collector. 
 2. From IBM InfoSphere Content Collector Web Application directory <Content Collector installDir>\WebApplication, run afu_ewas_uninstall.bat to uninstall IBM InfoSphere Content Collector Web Application. 
 3. Modify the file called afu_ewas_install.bat in IBM InfoSphere Content Collector Web Application directory <Content Collector installDir>\WebApplication to change FNCEPATH to point to the path of the newly installed FileNet P8 Content Engine Client . Save the file. 
 4. Next, run afu_ewas_install.bat. 
 5. In the IBM InfoSphere Content Collector Configuration Manager, check to make sure all parameters are filled in correctly under Email Service > Web Application and Email Service > Configuration web service. 
 6. Restart the IBM InfoSphere Content Collector web application service. 
 7. Update the description of the configuration web service in IBM InfoSphere Content Collector Configuration Manager. Save the changes. Configuration web service will be restarted automatically.
