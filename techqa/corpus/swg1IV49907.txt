ERROR DESCRIPTION
 *  Go To and Start Center links are always shown on the header bar
   even when the options are un-granted for the specific group.
   
   
   STEPS TO RECREATE:
   1. Login to SCCD with maxadmin user.
   2. Go To -> System Configuration -> Platform Configuration ->
   System
   Properties" application.
   3. Open "mxe.webclient.homeButtonHeaders" property.
   4. Set 1 to this property (it is default).
   5. Do "Live Refresh" (if necessary).
   6. Go To -> Security -> Security Groups application.
   7. Click "new group" icon.
   8. Input group name (e.g. "TESTGROUP"), and check "Independent
   of
   Other Groups".
   9. Save it.
   10. Open "Applications" tab.
   11. Select "Self Service Center" application.
   12. Set "Grant Access" Off/Uncheck at the following options.
   - Show GoTo link in page header.
   - Show Start Center link in page header.
   13. Go To -> Security -> Users application.
   14. Click "new user" icon.
   15. Set user name (e.g. "TESTUSER").
   16. Click "Set Password" button and set password.
   17. Open "Groups" tab.
   18. Click "New Row" button, and set "TESTGROUP" group created
   at Step 6
   19. Logout from SCCD.
   20. Login to SCCD with "TESTUSER" user created at Step 14
   21. Go To -> Self Service -> Self Service Center application.
   "Go To" and "Start Center" link is shown on header bar. <-- The
   issue
   
   It happens to both SCCD 7.5.0.1 and 7.5.0.2. And it works
   correctly in
   SCCD 7.5.1.
   
   Make sure during test mxe.webclient.homeButtonHeaders = 1.
