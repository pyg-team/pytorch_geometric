Title: IBM Information Server DataStage ODBC Connector Oracle Wire Protocol error ORA-12899: value too large for column - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 A DataStage job using the ODBC Connector and the Oracle Wire Protocol driver returns the error:

ODBC function "SQLExecute" reported: 
SQLSTATE = HY000: Native Error Code = 12,899: 
Msg = [IBM(DataDirect OEM)][ODBC Oracle Wire Protocol driver]
[Oracle]ORA-12899: value too large for column "xxx"."xxx"."xxx" (actual: 4, maximum: 1) 
(CC_OdbcDBStatement::executeInsert, file CC_OdbcDBStatement.cpp, line 702) 

CAUSE
Incorrect setting for the Oracle Wire Protocol driver parameter EnableNcharSupport

RESOLVING THE PROBLEM
In the .odbc.ini file, which is in the DSEngine directory, change the value of the Oracle Wire Protocol driver parameter for your data source from 

EnableNcharSupport=0
to
EnableNcharSupport=1
