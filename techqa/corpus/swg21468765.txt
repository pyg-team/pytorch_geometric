Title: IBM Running HPII_start and get error: too many open files using IBM FileNet Image Services - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Running HPII_start failed, get error: too many open files using Image Services. 

CAUSE
Failed when nofiles exceeded limit after only 245 batches.
The status files are being removed prior to closing the HPII data files. 


RESOLVING THE PROBLEM
Edit the HPII_cfg file removing or commenting out the 2 lines 

;ProgressFileFlag=1 
;FNProgressDirectory=/fnsw/local/bin/hpii/status

Engineering is working on a fix for future release.
