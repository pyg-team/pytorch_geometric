APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If a filter like Severity!=Type is used, when Filter Syntax is
   checked following error message is displayed:
   Filter Syntax Analysis Results:
   The following filters have syntax errors:
   .... Syntax Error
   :rule includes an unsupported expression
   
   This message needs to be changed or removed as is misleading
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * The Filter Analyzer reports the error: "rule includes an     *
   * unsupported expression" when comparing two ObjectServer      *
   * fields against each other.                                   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  Introduce a new props: impact.analyzer.osfields to bypass such
   syntax error.
   
   IMPACT_HOME/etc/server.props
   impact.analyzer.osfields=true
   
   The fix for this APAR is contained in the following
   maintenance packages:
   | Fix Pack | 7.1.0-TIV-NCI-FP0013
