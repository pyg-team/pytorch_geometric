APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Issues triggering from a database source using WTX 8.4.1.4.
   
   The map runs correctly under Command Server, but when triggered
   from the Launcher, fails to connect to the database.
   
   Audit log shows:
   
   <Platform>Launcher for Windows - Version 8.4.1(7)</Platform>
   
   <ExecutionSummary MapStatus="Error" mapreturn="12"
   ElapsedSec="0.0044" BurstRestartCount="0">
      <Message>Source not available</Message>
      <CommandLine>C:\Simple\Database\Test1.mmc</CommandLine>
   
   <SourceReport   card="1"   adapter="DB"   bytes="0"
   adapterreturn="-1069">
      <Message>Required command line options missing</Message>
      <Settings>-T+ -event 201</Settings>
   </SourceReport>
   
   
    
   
   

LOCAL FIX
 *  WTXCD - WTX00048469
   PB / PB
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   IBM Transformation Extender customers who use Launcher and
   define watches to trigger from DB
   
   Problem Description:
   Launcher fails to trigger from DB. The listener detects a
   condition to trigger of DB, Launcher starts the map but the map
   fails with source not available. The adapter error code is 1069.
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  Correct the core engine code to properly retrieve MDQ
   information from the mmc file when map executes GET from DB
   target on a source event card.
   
   Applies to:
   8.4.0.6, 8.4.1.4, 9.0.0.0
