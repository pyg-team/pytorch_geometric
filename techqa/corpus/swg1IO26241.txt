ERROR DESCRIPTION
 *  Directory Server audit logs currently display if the connection
   is using either SSL or TLS for each operation.
   This enhancement request will take care of displaying SSL and
   TLS protocol version info corresponding to the connection.
   This enhancement also clarifies if the operation is happening
   on SSL port with "SSL" keyword OR a secure TLS connection on
   non-SSL port with "STARTTLS" keyword.
   
   
    
   
   

LOCAL FIX
 *  Not Available.
   
   
    
   
   

PROBLEM SUMMARY
 *  Currently SDS is capable of supporting SSL V3.0, TLS V1.0,
   TLS V1.1 and TLS V1.2 secure connection protocols.
   
   If end user does SSL port based operations like
   search/adds/deletes/unbinds, Audit log header only shows 'SSL'
   and for startTLS based operation it only logs 'TLS'.
   
   This enhancement is to include SSLV3, TLS10, TLS11, or TLS12
   keywords based on the underlying secure protocol version that?s
   in use.
   For example, "SSL" will be replaced by "SSL TLS12" when TLS v1.2
   is used via SSL port. Similarly "TLS" will be replaced by
   "STARTTLS TLS12" when TLS v1.2 is used via startTLS operation on
   regular non-SSL port.
