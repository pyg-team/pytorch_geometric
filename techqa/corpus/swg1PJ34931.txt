   Description of Problem (full details)
   When searching document using search template or simple search
   on the object store which the CRB enabled, the format of the
   "Ranking" in the searched result is not correct, too many
   places to the right of the decimal are being shown. AE version
   is per185.106.

   Does this occur on more than one station / server? :

   Can the problem be replicated on an internal system?  Y or N
   By Who (L3, Support, etc..)
   Steps to Reproduce in search designer:
   1. Click Search Designer under Author--> Advanced Tools
   2. Select the Object Store which the CBR enabled
   3. Click search criteria tab, Under AND Property & Content
   conditions, set:
   View       Words/Phrases      Modifiers
   Editable      Smoke setup  Any
   4. Click Tools--> Options, Check the "Rank Content?". and
   click OK
   5. Then Save the search template
   6. Add some documents which the content contain "Smoke
   setup".
   7. Run the search template
   Steps to Reproduce in simple search:
   1. Edit User Preferences by Click Preferences,
   2. Select Search tab, select the OS which is CBR enabled by
   clicking "Change Search In",
   3. Set the value of Relevancy Ranking is on.
   3. Click Apply.
   4. Select Documents tab, set the value of Show Content
   Contains to Show.
   5. Click Apply.
   6. Go to Search tab in workplace, type "Smoke", click
   search button.
   Expected result:
   Got the document you added, and the value will be shown under
   "Ranking" as the format: xx.xx% (for example 77.42%)
   Actual result:
   The format of the value under Ranking is xx.xxxxxxxxxxx% (for
   example: 83.50999999999999%), please refer to the attachment.
