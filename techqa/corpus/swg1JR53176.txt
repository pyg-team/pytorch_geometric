LOCAL FIX
 *  Workaround for the problem is to add the following information
   in the
   StandAlone.Properties file
   
   bpm.de.sourceInfo.versionInfo=8.5.0.0
   bpm.de.sourceInfo.productType=Advanced
   
   
    
   
   

PROBLEM SUMMARY
 *  When you run the DBUpgrade command and the profile type is
   stand-alone, the command doesn't load the deployment environment
   and you receive a java.lang.NullPointException
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix is available for IBM BPM V8.5.5.0 so that the deployment
   environment loads when you run the DBUpgrade command even if the
   profile type is stand-alone.
   
   On Fix Central (http://www.ibm.com/support/fixcentral), search
   for JR53176:
   
   1. Select IBM Business Process Manager from the product
   selector, the installed version to the fix pack level, and your
   platform, and then click Continue
   2. Select APAR or SPR, enter JR53176, and click Continue.
   
   When you download fix packages, ensure that you also download
   the readme file for each fix. Review each readme file for
   additional installation instructions and information about the
   fix.
