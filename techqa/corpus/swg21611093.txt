PROBLEM(ABSTRACT)
 Using the sarpccp show mapping command




CAUSE
The show mapping command displays RPC endpoint map elements. Every element correspond to an object UUID, interface identifier, annotation, and binding information. The binding information contains an RPC protocol sequence, a network address, and an endpoint (a port in case of UDP and TCP) in square brackets. All SARPCD server processes such as SFS or cicsas registers with three protocols namely CIPC, tcp and udp. For example, rpc-prot-seq:network-addr[endpoint]. 

sarpccp show mapping 
or
sarpccp show mapping [-i if-id [-v versions]] [-o object-uuid] 


Options 

-i 
Defines an interface identifier (optional). Only one interface can be shown in a single operation. If specified, only elements containing this interface identifier are shown. 

The interface identifier value (<interface id>) has the following form: 
interface-uuid[major-version.minor-version] 

The UUID is a hexadecimal string, and the version numbers are decimal strings. 

For example: -i ec1eeb60-5943-11c9-a309-08002b102989,1.1

-v 
Indicates how a specified interface version is used (optional). If it is used without the -i option, the -v option is ignored. 

-o 
Defines an object (<object>) to be shown (optional). 

The UUID is a hexadecimal string. 
For example: -o 3c6b8f60-5945-11c9-a236-08002b102989


Example 

1. sarpccp show mapping

2. sarpccp show mapping -i 004c9bcc-bf17-1bef-b1d9-9e62091c0000,1.0


Sample output:

<object> 7fa868f2-fabc-11e1-9623-001a64910fbc
<interface id> 004c9bcc-bf17-1bef-b1d9-9e62091c0000,1.0
<string binding> ncacn_ip_tcp:1.22.33.44[40119]
<annotation> /.:/cics/sfs/hostname

<object> 7fa868f2-fabc-11e1-9623-001a64910fbc
<interface id> 004c9bcc-bf17-1bef-b1d9-9e62091c0000,1.0
<string binding> ncacn_ip_udp:1.22.33.44[34119]
<annotation> /.:/cics/sfs/hostname

<object> 8421cb2a-fd88-11e1-9306-00025576217a
<interface id> 80878000-1892-11d0-91bb-08005ace014c,1.0
<string binding> ncacn_cics_ipc:[1905020024]
<annotation> /.:/cics/chregion/ts Region Definition

The <annotation> string is used to query and determine the server process corresponding to the binding. For example, to get the process associated with the binding "/.:/cics/chregion/ts", you need to run:

# tkadmin query process -server /.:/cics/chregion/ts
430140
# ps -ef | grep 430140
cics 430140 921620 0 17:28:22 - 0:00 cicsrl chregion 611319885 6 a0000000 b0000000
#
