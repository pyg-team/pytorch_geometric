Title: IBM Creating P8 domain for CPE 5.2 fails with Member Not Found - United States

Text:
LDAP; Directory Config TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempts to create the Directory Service Configuration for the IBM FileNet P8 Content Platform Engine (CPE) domain fail via FileNet Enterprise Manager (FEM) with Member Not Found 

SYMPTOM
After entering the LDAP host and bind user to create the Directory Service via FEM, a popup message states Member Not Found and the P8 Domain can not be created


CAUSE
A previous version of FEM was installed and not removed before installing CPE 5.2

DIAGNOSING THE PROBLEM
Check the FEM dll versions:
right click FileNet_Api.dll the detail tab should show 5.x matching the version of CPE installed.

RESOLVING THE PROBLEM
Uninstall FEM from the Windows applet and reinstall the CE .Net API or use ACCE on a supported browser.
