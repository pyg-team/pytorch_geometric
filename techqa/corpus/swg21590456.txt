PROBLEM(ABSTRACT)
 Unable to configure Content Search Services servers on FileNet Content Engine 

SYMPTOM
Using FileNet Content Manager 5.0/5.1 the following error occurs when Configuring IBM Content Search Services servers on Content Engine:

File = .\FnWizCreateTextSearchServer.cpp, Line = 246, Rev. =
$Revision: 1.0 $
Source =
Description = The Content Engine server cannot connect to the IBM
Content Search Services server <CSS Server name> located on host
<hostname>, port 8191.
--------------------------------
Error Code: 0
Error Source: FileNet.Api

HR = 0x80040218
HR Description = Error 0x80040218
MinorCode = 0x00000a94


CAUSE
Invalid CSS authentication token

DIAGNOSING THE PROBLEM
Attempt to configure Content Search Services servers on Content Engine using an invalid authentication token.

RESOLVING THE PROBLEM
When Content Search Services is first installed you are provided a valid authentication token. Do not change this but write it down because it is needed later to configure CSS for CE. It can only be changed after the fact by using the "configTool generateToken"




Re-run the wizard to configure Content Search Services in FileNet Enterprise Manager using the original valid authentication token.
