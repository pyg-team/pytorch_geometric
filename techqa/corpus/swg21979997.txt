Title: IBM Adding custom report templates for Cognos
Workspace Advanced - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How do I add custom report templates for Cognos
Workspace Advanced 

ANSWER
IBM® Cognos® Workspace Advanced has predefined report templates that the report authors can use when creating new reports. If the predefined report templates do not meet the needs of report authors, custom templates can be created and made available in Cognos Workspace Advanced. The procedures for adding custom report templates depend on custom interface profiles. You need to use a different procedure if the custom interface profiles are enabled in IBM Cognos Business Intelligence and when they are not enabled.
Adding custom report templates when custom user interface profiles are enabled
To make the template available to the report authors, you need to modify a number of files in the IBM Cognos Business Intelligence installation directory and add the custom report template to a custom user interface profile.

Before you begin:
The custom report template specification must already be created. 
User interface profiles must be enabled.

You need to modify the following files in the Cognos Business Intelligence installation directory:
/webcontent/pat/res/templates.xml
/webcontent/pat/res/Resources.xml
/webcontent/pat/res/reportstudio_en.xml
/webcontent/uipe/res/uipe.xml

After you finish editing these files, your custom report template should be available for selection in the New dialog box in Cognos Workspace Advanced.

Procedure
1. In IBM Cognos Administration, on the Library tab, create a new Cognos Workspace Advanced profile.

2. In the c10_location/webcontent/pat/res/templates.xml file, search for the <xmlFragment id="ReportTemplates"> element, and add your custom template name.

For example, if your custom template name is My custom list, you add it to the templates.xml file in the following way:
<resources>
<xmlFragment id="ReportTemplates">
<template name="My custom list"
<template name="Blank"
<template name="List"
<template name="Crosstab"
Save and close the templates.xmlfile.

3. In the c10_location/webcontent/pat/res/Resources.xml file, search for the listView id="New" element, and add a new listItem element for your custom
report template.

In the following example, My custom list is added as the first listItem element for listItems.
<listView id="New" view="icon" clipLabels="false">
<listItems>
<listItem idsLabel="IDS_LBL_NEW_MY_CUSTOM_LIST" icon="icon_my_custom_list.gif" value="My custom list"/>
<listItem idsLabel="IDS_LBL_NEW_BLANK_REPORT" icon="icon_blank.gif" value="Blank"/>
<listItem idsLabel="IDS_LBL_NEW_BLANK_REPORT" icon="icon_blank.gif" value="Blank"/>
<listItem idsLabel="IDS_LBL_NEW_LIST_REPORT" icon="icon_list.gif" value="List"/>
</listItems>
</listView>
Save and close the Resources.xml file.

4. In the c10_location/webcontent/pat/res/reportstudio_en.xml file, add a new string element for the custom report template.

In the following example, the string id="IDS_LBL_NEW_MY_CUSTOM_LIST" represents the My custom list template.
<string id="IDS_LBL_JOIN_OPERAND" type="Control Label"> Query:</string>
<string id="IDS_LBL_NEW_MY_CUSTOM_LIST" type="Control Label">My custom list</string>
<string id="IDS_LBL_NEW_BLANK_REPORT" type="Control Label">Blank</string>
<string id="IDS_LBL_NEW_LIST_REPORT" type="Control Label">List</string>
<string id="IDS_LBL_NEW_CROSSTAB_REPORT" type="Control Label">Crosstab</string>

The reportstudio_en.xml file is used to translate the labels into different languages. 

Save and close this file.

5. In the c10_location/webcontent/uipe/res/uipe.xml file, search for the listView id="New" element, and add a new listItem element for the new template.

In the following example, the <listItem id="My custom list" idsLabel="IDS_LBL_NEW_MY_CUSTOM_LIST" smallIcon=""/> represents the custom template.

</toolboxes>
<listViews>
<listView id="New" idsDescription="IDS_LISTVIEW_DESC_New" idsLabel="IDS_LISTVIEW_LBL_New">
<listItem id="My custom list" idsLabel="IDS_LBL_NEW_MY_CUSTOM_LIST" smallIcon=""/>
<listItem id="Blank" idsLabel="IDS_LBL_NEW_BLANK_REPORT" smallIcon=""/>
<listItem id="List" idsLabel="IDS_LBL_NEW_LIST_REPORT" smallIcon=""/>
<listItem id="Crosstab" idsLabel="IDS_LBL_NEW_CROSSTAB_REPORT" smallIcon=""/>

This step adds the custom template to the list of user interface objects in the Cognos Workspace Advanced user interface profile. Save and close the uipe.xml file.

6. Include the new template in the Cognos Workspace Advanced user interface profile that you created in step 1.
a. In Cognos Administration, click the Library tab.
b. For the Cognos Workspace Advanced user interface profile that you created in step 1, click the Actions menu icon, and then click Edit. The User Interface Profile Editor is displayed.
c. In the Profile Options box, expand the User Interface Objects node, and click List Views > New. You should see the custom template name in the list of views.
d. Select the check box for the custom template name, and click OK.

7. Open Cognos Workspace Advanced, choose the edited Cognos Workspace Advanced user interface profile, and click Create New to create a new report.

The new template should be available in the New dialog box.

Adding custom report templates when custom user interface profiles are not enabled

To make the template available to the report authors, you need to modify a number of files in the IBM Cognos Cognos Business Intelligence installation directory.

Before you begin:
The custom report template specification must already be created.
You need to modify the following files in the Cognos Business Intelligence
installation directory:
/webcontent/pat/profiles/profile_bua_standalone.xml
/webcontent/pat/res/templates.xml
/webcontent/pat/res/Resources.xml
/webcontent/pat/res/reportstudio_en.xml
After you finish editing these files, your custom template should be available for selection in the New dialog box in Cognos Workspace Advanced.

Procedure
1. In the /webcontent/pat/profiles/profile_bua_standalone.xml file, search for the listView id="New" action="show" element, and add a new listItem element that corresponds with the custom report template.
In the following example, the listItem value="My custom list" represents the My custom list template.
<listViews>
<listView id="New" action="show">
<listItem value="My custom list"/>
<listItem value="Blank"/>
<listItem value="List"/>
<listItem value="Crosstab"/>
</listView>
</listViews>


2. In the c10_location/webcontent/pat/res/templates.xml file, search for the <xmlFragment id="ReportTemplates"> element, and add your custom template name.
For example, if your custom template name is My custom list, you add it to the templates.xml file in the following way:
<resources>
<xmlFragment id="ReportTemplates">
<template name="My custom list"
<template name="Blank"
<template name="List"
<template name="Crosstab"

Save and close the templates.xmlfile.

3. In the c10_location/webcontent/pat/res/Resources.xml file, search for the listView id="New" element, and add a new listItem element for your custom report template.

In the following example, My custom list is added as the first listItem element for listItems.
<listView id="New" view="icon" clipLabels="false">
<listItems>
<listItem idsLabel="IDS_LBL_NEW_MY_CUSTOM_LIST" icon="icon_my_custom_list.gif" value="My custom list"/>
<listItem idsLabel="IDS_LBL_NEW_BLANK_REPORT" icon="icon_blank.gif" value="Blank"/>
<listItem idsLabel="IDS_LBL_NEW_BLANK_REPORT" icon="icon_blank.gif" value="Blank"/>
<listItem idsLabel="IDS_LBL_NEW_LIST_REPORT" icon="icon_list.gif" value="List"/>
</listItems>
</listView>

Save and close the Resources.xml file.

4. In the c10_location/webcontent/pat/res/reportstudio_en.xml file, add a new string element for the custom report template.

In the following example, the string id="IDS_LBL_NEW_MY_CUSTOM_LIST" represents the My custom list template.
<string id="IDS_LBL_JOIN_OPERAND" type="Control Label"> Query:</string>
<string id="IDS_LBL_NEW_MY_CUSTOM_LIST" type="Control Label">My custom list</string>
<string id="IDS_LBL_NEW_BLANK_REPORT" type="Control Label">Blank</string>
<string id="IDS_LBL_NEW_LIST_REPORT" type="Control Label">List</string>
<string id="IDS_LBL_NEW_CROSSTAB_REPORT" type="Control Label">Crosstab</string>

The reportstudio_en.xml file is used to translate the labels into different languages.

Save and close this file.


5. Open Cognos Workspace Advanced, and click Create New to create a new report.
The new template should be available in the New dialog box.
