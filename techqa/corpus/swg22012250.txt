CAUSE
Misconfiguration

RESOLVING THE PROBLEM
Take the signer of the above certificate in the error above from WebSphere Application Server (WAS) and put it in the trust store of the RAD. 

Export certificate from WAS & import it into RAD.

The important part is to determine which certificate needs to be exported from the WAS server, and where it needs to be imported on the workstation.

On the WAS Server you must review the “End Point Security“ for the “Application Server” and review which SSL configuration is used for this endpoint. You must export the certificate from this trust store. 

This certificate must then be imported into 
<RAD/RSA_install_dir>\jdk\jre\lib\security\cacerts and when requested Trust the new Certificate.

- Update ssl.client.props file to support TLSv1.2.
