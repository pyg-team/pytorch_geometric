Title: IBM Derby DB not running as Primary - United States

Text:
Derby; failover; primary; secondary TECHNOTE (FAQ)

QUESTION
 I've configured my Impact server cluster to use Derby failover, but my designated Derby Primary server is running as the Derby Secondary server, why ? 

ANSWER
The designation of Derby Primary and Secondary hosts in the UI definitions is treated "secondarily", the primary determination of what Derby host will be Primary, is the Impact server that is running as 
Primary - the Derby Primary DB will always be running on that host
