LOCAL FIX

PROBLEM SUMMARY
 *  Prior to this APAR, XMLAccess attempts to open an HTTPS
   connection using the SSL context, if the XMLAccess servlet URL
   specifies the https:// protocol. There is  no option to select a
   different context, such as TLS, or SSLv3.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Command line parameter "-protocol <value>" was introduced, which
   allows the user to specify the context.
   Note that the HTTPS context must be supported and activated in
   the application server configuration.
   
   Manual Steps:
   After the APAR has been installed, the XML Configuration
   Interface (xmlaccess) supports a new command line parameter,
   which can be used to determine the protocol of the https
   connection to the xmlaccess servlet. The syntax of the parameter
   is:
   
         -protocol <protocol>
   
   where <protocol> specifies the protocol, as for example SSL,
   SSLv1, SSLv3, TLS.
   Note that only protocols  can be selected, which are supported
   and enabled by the application server. The parameter is only
   evaluated if the url of the xmlaccess servlet selects a secure
   (https) connection. Otherwise, the parameter is ignored.
   
   Usage example:
   
   xmlaccess.sh -url https://www.samplehost.com:10040/wps/config [https://www.samplehost.com:10040/wps/config]
   -parameter SSLv3 ...
   
   determines that the SSLv3 protocol is selected to connect to the
   xmlaccess servlet.
