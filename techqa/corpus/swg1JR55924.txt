APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Installing WebSphere IFix PI50993 or WebSphere 8.5.5.9 causes
   Information Server to report java.lang.NoClassDefFoundError
   for the Apache Http component classes.
   For example:
   ... webapp E com.ibm.ws.webcontainer.webapp.WebApp
   logServletError  SRVE0293E: [Servlet Error]-[REST2]:
   java.lang.NoClassDefFoundError:
   org.apache.http.impl.client.DefaultHttpClient
   at java.lang.Class.getDeclaredMethodsImpl(Native Method)
   at java.lang.Class.getDeclaredMethods(Class.java:1001)
   
   
   See the "Local fix" section for temporary manual steps to work
   around the problem.
   
   
   Alternatively, patches have been published to address this
   problem.  See Security bulletin
   http://www.ibm.com/support/docview.wss?uid=swg21982420 for
   details, and refer to
   http://www.ibm.com/support/docview.wss?uid=swg21996739 or
   http://www.ibm.com/support/docview.wss?uid=swg24043188 for
   updated installation instructions.
   
   After applying the ISF rollup patch, the manual steps in the
   "Local fix" section of this document must be undone as follows:
       Delete the copied *http*-4.2.1.jar files in
           C:\IBM\WebSphere\AppServer\profiles\InfoSphere\classes
           <IIS Install>/ASBServer/apps/lib/iis/classes
       Restart WebSphere
   
   
    
   
   

LOCAL FIX
 *  Copy the http*.jar files
       from
   C:\IBM\WebSphere\AppServer\profiles\InfoSphere\lib\iis\20thirdpa
   rty
       to
          C:\IBM\WebSphere\AppServer\profiles\InfoSphere\classes
       -and-
          <IIS Install>/ASBServer/apps/lib/iis/classes
   Restart Websphere
