Title: IBM An error occured while trying to access server when accessing Oracle Controller database - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After successfully creating and running steps on a new Oracle Controller database when opening in the Controller client and attempting to connect the message "An error occured while trying to access server&quot; is displayed. 

SYMPTOM
An examination of the Windows Event Viewer Application log reveals the following error message indicating an ORA-12638 issue

>  Error occured at 9/27/2006 9:12:27 PM in Cognos Controller, Error No=5, Source=FrangoDirect.ConnectionDirect.GetUserInfo#ControllerProxyClient, Description=System.Web.Services.Protocols.SoapException: Server was unable to process request. ---> System.Data.OracleClient.OracleException: ORA-12638: Credential retrieval failed
at System.Data.OracleClient.DBObjectPool.GetObject(Object owningObject, Boolean& isInTransaction)
at System.Data.OracleClient.OracleConnectionPoolManager.GetPooledConnection(String encryptedConnectionString, OracleConnectionString options, OracleConnection owningObject, Boolean& isInTransaction)
at System.Data.OracleClient.OracleConnection.OpenInternal(OracleConnectionString parsedConnectionString, Object transact)
at System.Data.OracleClient.OracleConnection.Open()
at System.Data.Common.DbDataAdapter.QuietOpen(IDbConnection connection, ConnectionState& originalState)
at System.Data.Common.DbDataAdapter.FillFromCommand(Object data, Int32 startRecord, Int32 maxRecords, String srcTable, IDbCommand command, CommandBehavior behavior)
at System.Data.Common.DbDataAdapter.Fill(DataSet dataSet, Int32 startRecord, Int32 maxRecords, String srcTable, IDbCommand command, CommandBehavior behavior)
at System.Data.Common.DbDataAdapter.Fill(DataSet dataSet)
at Cognos.Controller.Proxy.ControllerReportNetService.RepNet.GetUserInfo(String sGuid, String sUser, String passportId)
at Cognos.Controller.Proxy.CCRWS.GetUserInfo(String sGuid, String sUser, String passportId)
--- End of inner exception stack trace ---, HelpFile= HelpContext=0



CAUSE
ORA-12638 - Credential retrieval failed.

ENVIRONMENT


RESOLVING THE PROBLEM
Within the ORA_HOME\network\admin\sqlnet.ora file change line 

>  SQLNET.AUTHENTICATION_SERVICES= (NTS) to

SQLNET.AUTHENTICATION_SERVICES= (NONE)
