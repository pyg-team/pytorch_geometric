QUESTION
How do I configure Natural Language Query Whitelist on Watson Explorer Content Analytics?

ANSWER
To configure NLQ Whitelist:

 1. Open ES_NODE_ROOT/master_config/nlqservice/querymodifier.yml with a text editor
 2. Add PhraseWhitelistStrategy to ac_production profile like as
    ************************
    spring:
    profiles: ac_production
    languages:
    pathToPears: src/test/resources/pears
    strategies:
    default: PhraseWhitelistStrategy POSBasedNoiseWordRemoverStrategy DictionaryBasedNoiseWordRemoverStrategy DisjunctifyStrategy ACSyntaxTransformStrategy
    ************************
 3. Modify the full path of the whitelist file like as ************************
    phraseFilePath:
    default: /path/to/phrase/whitelist.txt
    ************************
    
    
 4.  Save changes and close the editor
    
    
 5.  Open the whitelist.txt above with UTF-8 aware text editor
    
    
 6.  Add the word which NLQ removal must exclude
    for example:
    ************************
    work<tab>-<tab>load
    ************************
    Notes: <tab> stands for a tab character. <tab> is needed among tokenized words
    
    
 7.  Save changes and close the editor
    
    
 8.  Execute commands below
    esadmin config sync
    esadmin nlqservice.<nid> restart
    Notes: <nid> stands for node ID, such as node1, node2
