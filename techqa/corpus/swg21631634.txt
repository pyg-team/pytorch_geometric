Title: IBM Cannot initialize the given in-memory storage manager due to exception org.apache.sandesha2 - United States

Text:
TPAEINTEGRATION; MIF; INTEGRATION FRAMEWORK; SANDESHA TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Unable to post requests to Maximo WebServices


SYMPTOM
Incoming WebServices requests fail with the following error:

Cannot initialize the given in-memory storage manager due to exception org.apache.sandesha2. 
SandeshaException: Sandesha2 Internal Error: DefaultPropertyBean is not set.

SessionContextMBeanAdapter findAttCausingNotSerializableException 
Miscellaneous data: Attribute "_WCSM_" is declared to be serializable 
but is found to generate exception "java.io.NotSerializableException" 
with message "psdi.webclient.system.websession.SimpleWebAppSession". 
Fix the application so that the attribute "_WCSM_" is correctly 
serializable at runtime. 



CAUSE
This error is caused by a conflict on WebSphere 8 and Maximo Axis2 Libraries.
WebSphere 8 delivers its own Libraries.

ENVIRONMENT
WebSphere 8

RESOLVING THE PROBLEM
1) From the meaweb module folder - 
applications\maximo\meaweb\webmodule\WEB-INF\classes\psdi\iface\webservices, find the MaxWebServiceProvider.class 

2) Rename it to MaxWebServiceProvider.bak 

3) Rebuild and Redeploy your maximo.ear file 

4) Re-test the WebService call.
