Title: IBM Tivoli Integrated Portal Installation failure with memory allocation errors - United States

Text:
TIPL2; TIPL2INST; Tivoli Integrated Portal; cannot allocate memory; java; IOException TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 IBM products which include Tivoli Integrated Portal (TIP) may fail if there is insufficient memory available to the installer. In this case the installation log will include the error "java.io.IOException: Cannot allocate memory." 

SYMPTOM
The installation program will run for a while, and then terminate as follows:
The following installation step failed: 

$IAGLOBAL_COI_STEP_ISC_DESC$

PRESS <ENTER> TO CONTINUE:


CAUSE
There is insufficient memory available to complete the Websphere installation.



ENVIRONMENT
All platforms



DIAGNOSING THE PROBLEM
Review the installation log located in the home directory of the OS user used to execute the installer. The installation log name will be similar to IA*.log. Look for the following error in the log:

Caused by: 

/opt/IBM/TCR/_uninst/TIPInstall/plan/install/MachinePlan_localhost/00002_IAGLOBAL_COI_STEP_ISC/ISC.post.install.xml:68: Execute failed: 
java.io.IOException: java.io.IOException: Cannot allocate memory. 


RESOLVING THE PROBLEM
Ensure that there is at least 2GB of RAM available to the TIP installer. Stop any other processes which are consuming high amounts of memory, or add more RAM to the system. If the installation is run from a VM guest, ensure the guest is allocated at least 2GB of RAM.
