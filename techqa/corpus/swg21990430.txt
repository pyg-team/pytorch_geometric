Title: IBM repotools -createTables command fails with ORA-01000 on Oracle 12 - United States

Text:
Oracle; driver; ORA-01000; create_tables; create; tables; 19632480; cursors TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempt to install IBM Collaborative Lifecycle Management (CLM) products fails with "ORA-01000: Maximum open cursors exceeded" error during repotools -createTables stage when Oracle 12 is used. 

SYMPTOM
You see the following error message in repotools-app_createTables.log file: 


CRJAZ0811E The package-to-schema map could not be detected because of an SQL error.
com.ibm.team.repository.common.TeamRepositoryException: CRJAZ0811E The package-to-schema map could not be detected because of an SQL error.
at com.ibm.team.repository.service.TeamRepositoryExceptionHelper.wrapException(TeamRepositoryExceptionHelper.java:58)
at com.ibm.team.repository.service.db.provider.AbstractDbProvider.doesTableOrViewExist(AbstractDbProvider.java:467)
at com.ibm.team.repository.service.db.provider.AbstractDbProvider.doesTableExist(AbstractDbProvider.java:430)
at sun.reflect.GeneratedMethodAccessor21.invoke(Unknown Source)
...
Caused by: java.sql.SQLException: ORA-01000: Maximum open cursors exceeded
at oracle.jdbc.driver.T4CTTIoer.processError(T4CTTIoer.java:450)
at oracle.jdbc.driver.T4CTTIoer.processError(T4CTTIoer.java:399)
at oracle.jdbc.driver.T4C8Oall.processError(T4C8Oall.java:1059) 






CAUSE
Defect in Oracle jdbc driver: Bug 19632480 - ORACLEDATABASEMETADATA.GETTABLES CURSOR CONNECTION LEAK [https://support.oracle.com/rs?type=bug&id=19632480].


ENVIRONMENT
Oracle jdbc driver in version 12.1.0.2.0.



DIAGNOSING THE PROBLEM
Check your Oracle jdbc driver if your version is affected by the defect above



RESOLVING THE PROBLEM
Contact Oracle support to get the driver which is not affected by Bug 19632480. [https://support.oracle.com/rs?type=bug&id=19632480] The example Oracle driver version not affected by this bug is 12.1.0.1.0.
