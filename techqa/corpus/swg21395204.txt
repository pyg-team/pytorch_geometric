Title: IBM How to change language of WebSphere Application Server log - United States

Text:
websphere; language; setting; RAD; WAS; locale TECHNOTE (FAQ)

QUESTION
 How do you modify the language of the WebSphere log for the IBM Rational Application Developer for WebSphere Software (RAD)? 

CAUSE
The language of the WebSphere Application Server log depends on the regional and language settings. The settings of either the computer that runs WebSphere Application Server or Rational Application Developer determines the language.

ANSWER
For WebSphere Application Server, pass system properties to the Java Virtual Machine (JVM) , these properties are the following.


-Duser.language=en -Duser.region=US 


Consider the following steps, 



 1. Select the server and start it, if needed.
    
    
 2. Right click on the server and run the Administration Console , see below.
    
    
    
    
    [/support/docview.wss?uid=swg21395204&aid=1] [/support/docview.wss?uid=swg21395204&aid=1]
    
    
    Alternatively Open https://localhost:9044/ibm/console/ [https://localhost:9044/ibm/console/] and login.
    9044 is the default value for the Admin console port 
    
    
 3. Open Servers > Application Servers. Then choose the server where you want to modify the language for the log.
    
    
    Example : server1 
    
    
    
    [/support/docview.wss?uid=swg21395204&amp;aid=2]
    
    
    
 4. Click Java and Process Management > Process definition in section Server Infrastructure,
    
    [/support/docview.wss?uid=swg21395204&aid=3] [/support/docview.wss?uid=swg21395204&aid=3]
    
    
 5. Choose beneath Additional Properties > JVM Properties.
    
    
    
    [/support/docview.wss?uid=swg21395204&amp;aid=4]
    
    
 6. Fill out the following parameter in the field Generic JVM arguments.
    
    -Xquickstart
    
    [/support/docview.wss?uid=swg21395204&amp;aid=5]
    
    
 7. Add the parameter: 
    
    -Duser.language=en -Duser.region=US 
    
    
 8. Click OK, and Save, Logout.
    
    
    
 9. Restart the server to apply the changes of the JVM.
