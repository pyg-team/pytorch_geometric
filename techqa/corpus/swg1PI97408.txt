ERROR DESCRIPTION
 *  Generation of marshaller jar for rule project with COBOL XOM
   fails when XOM path contains a Java XOM which depends on the
   COBOL XOM.
   
   
    
   
   

LOCAL FIX
 *  1. create a new rule project
   2. declare the COBOL XOM in this new rule project
   3. remove the COBOL XOM from the original rule project
   4. create a dependency of the original rule project to the
   new rule project

 *  The  generation of marshaller jar for rule project with cobol
   xom fails
