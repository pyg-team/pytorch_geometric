Title: IBM Restore  WAS proxy server service fails with security and role errors loading admin-authz.xml from RCSList - United States

Text:
SECJ0007E; SECJ0244I; WSVR0009E; nodeagent; CSList; syncNode TECHNOTE (TROUBLESHOOTING)

PROBLEM
You have accidentally decommissioned a WebSphere Application Server proxy application. After taking steps to return it to service you still get an error when trying to start the nodeagent: 

 
00000001 distSecurityC E SECJ0007E: Error during security initialization. The exception is com.ibm.ws.security.role.RoleBasedAppException: configureRoleBasedAuthz() error loading
admin-authz.xml from RCSList at com.ibm.ws.security.core.distSecurityComponentImpl.
configureRoleBasedAuthz(distSecurityComponentImpl.java:1594)





SYMPTOM
Error on attempting to load the nodeagent:
00000001 distSecurityC E SECJ0007E: Error during security initialization. The exception is com.ibm.ws.security.role.RoleBasedAppException: configureRoleBasedAuthz() error loading
admin-authz.xml from RCSList at com.ibm.ws.security.core.distSecurityComponentImpl.
configureRoleBasedAuthz(distSecurityComponentImpl.java:1594)




CAUSE
The admin did not sync the application node with the Deployment Manager.



ENVIRONMENT
WAS 8.5, Sametime 9.0



DIAGNOSING THE PROBLEM
Securities and roles error after restore of the server.



RESOLVING THE PROBLEM
Sync the WAS application node agent with the DMGR.

http://www.ibm.com/support/knowledgecenter/SS7K4U_8.5.5/com.ibm.websphere.zseries.doc/ae/rxml_syncnode.html [http://www.ibm.com/support/knowledgecenter/SS7K4U_8.5.5/com.ibm.websphere.zseries.doc/ae/rxml_syncnode.html]

For example: 


c:\IBM\WebSphere\AppServer\profiles\[proxy or application directory]\bin>syncNode deploymgr.domain.com 8703

You may need to add the WAS admin and password to the command line, e. g., -username wasadmin -password password.
