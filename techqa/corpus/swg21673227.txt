Title: IBM Hot fix for ITCAM Extended Agent for Oracle Database Version 6.3.1 Fix Pack 2 APAR IV60530 - United States

Text:
5724V09ORTR TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 APAR IV60530 (Extended Oracle Database agent v6.31 FP2 data collector process - krzclient/krzstart memory leak) opened on TCAM Extended Agent for Oracle Database Version 6.3.1 Fix Pack 2. The link of APAR IV60530:

http://eclient.lenexa.ibm.com:9082/search?fetch=source/APAR/IV60530 [http://eclient.lenexa.ibm.com:9082/search?fetch=source/APAR/IV60530] 

RESOLVING THE PROBLEM
To resolve the problem, you need to apply ITCAM Agent for Oracle Database Version 6.3.1 Fix Pack 2 Interim Fix 2 or later version. 6.3.1.2-TIV-ITM_KRZ-IF0002 includes fix of APAR IV60530. 

ITCAM Extended Agent for Oracle Database: 6.3.1.2-TIV-ITM_KRZ-IF0002
http://www-01.ibm.com/support/docview.wss?uid=swg24038442 [http://www-01.ibm.com/support/docview.wss?uid=swg24038442] 

 

If you are not able to apply formal fix temply, you can download hot fix from this technote, and apply it as the following steps: 

 1. Stop Extended Oracle Database agent. 
 2. Find krzclient on Linux/Unix or krzclient.exe on Windows from ITM installation directory. 
 3. Apply hot fix:
    on Linux/Unix:
    Replace <ITM_INSTALL_DIR>/<arch>/krzclient with <hot_fix>/<arch>/krzclient
    on Windows: 
    Replace <ITM_INSTALL_DIR>\<arch>\krzclient.exe with <hot_fix>\<arch>\krzclient.exe 
 4. Start Extended Oracle Database agent.
