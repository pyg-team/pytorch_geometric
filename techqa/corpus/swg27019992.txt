Title: IBM Fixed: Configuration Manager sometimes fails when you deploy IBM FileNet Content Engine on an application server that is running Oracle WebLogic Server with SSL configured - United States

Text:
 RELEASE NOTES

ABSTRACT
 Configuration Manager sometimes fails if Content Engine is deployed on an application server that is running a UNIX operating system and Oracle WebLogic Server that is configured with SSL.

A fix for this issue was applied to the IBM FileNet P8 Version 5.0 information center (http://publib.boulder.ibm.com/infocenter/p8docs/v5r0m0/index.jsp [http://publib.boulder.ibm.com/infocenter/p8docs/v5r0m0/index.jsp]) on www.ibm.com as of April 25, 2011. 

CONTENT
If you are configuring Oracle WebLogic Server with SSL for Content Engine, be aware that SSL configuration steps are documented in the topic "Enabling SSL for Content Engine" in the IBM FileNet P8 5.0.0 Information Center.
