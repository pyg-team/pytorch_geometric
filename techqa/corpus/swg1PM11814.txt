APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When using an Oracle database, an attempt to use a schema
   synonym for the datastore of a messaging engine results in
   "IllegalStateException: CWSIS1532E: The table,
   <synonym_name>.SIBOWNER, is missing.", and the messaging engine
   will not start.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  Users of the default messaging provider for *
   *                  IBM WebSphere Application Server Version    *
   *                  6.1 and Version 7.0                         *
   ****************************************************************
   * PROBLEM DESCRIPTION: When using a synonym as a schema name   *
   *                      for a data store, the name is not       *
   *                      found causing the messaging engine to   *
   *                      fail to start.                          *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The messaging engine datastore code looks for tables that
   match the schema and table type provided. If the schema is a
   synonym, then the table is not found even if it actually
   exists because the datastore code is only looking for table
   types 'table' and 'alias'. This results in the messaging
   engine failing to start, with the following error message
   being output:
   "IllegalStateException: CWSIS1532E: The table,
   <synonym_name>.SIBOWNER, is missing."
   
   The problem is known to affect Oracle databases using
   synonyms, and could affect other databases where synonyms are
   configured.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this APAR adds a custom property that allows
   the messaging engine code to find the table whether the schema
   is real or a synonym.
   
   The messaging engine custom property is as follows:
   sib.msgstore.jdbcAllowSynonymUse=true
   
   Details of setting messaging engine custom properties are
   available in the "Setting tuning properties of a messaging
   engine" section of the information center.
   
   In addition to enabling the above custom property, it might be
   necessary to set the following additional two custom
   properties:
   sib.msgstore.jdbcPerformColumnChecks=false
   sib.msgstore.jdbcUseDeleteInsteadOfTruncateAtStartup=true
   
   These properties are required in cases where vendor specific
   limitations that prevent the JDBC driver from obtaining column
   meta data for, or TRUNCATE, a table that is a synonym.
   These limitations are known to apply to Oracle databases,
   where the two additional custom properties must be set to
   utilize synonyms with a data store.
   
   The fix for this APAR is currently targeted for inclusion in fix
   pack 6.1.0.33 and 7.0.0.13. Please refer to the
   Recommended Updates page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
