Title: IBM Adding a user as a member of the Global Administrative Group - United States

Text:
Global Admin Group; GAG; Global Administrative group; group membership; how to add a member to Global Admin; groups; ITDS TECHNOTE (FAQ)

QUESTION
 How do I add a user as a member of the Global Administrative Group? 

ANSWER
The global administrative group is a way for the directory administrator to delegate administrative rights in a distributed environment to the database backend. Global administrative group members are users that have been assigned the same set of privileges as the administrative group with regard to accessing entries in the database backend. Members of the global administrative group do not have access to the audit log and have no privileges or access rights to any data or operations that are related to the configuration settings (configuration backend) of the directory server. All global administrative group members have the same set of privileges.

Note:
Global administrative group members have the authority to send the administrative control.


Example: 
In this example, we will go through the steps using command line utilities to add a new user, add that user as a member of the Global Administrative Group and then confirming their membership.

**Note: Use the "-?" with command line utilities for additional usage information. 

1.) The structure of the DIT (Directory Information Tree) in this environment: 

Syntax: idsldapsearch -p <port> -h <hostname> -D <adminDN> -w <password> -L -s <scope> -b <base> objectclass=*

==> idsldapsearch -p 389 -D cn=root -w secret -L -s sub -b "o=IBM,c=US" objectclass=*
dn: o=IBM,c=US
objectclass: organization
objectclass: top
o: IBM

dn: ou=people,o=IBM,c=US
ou: people
objectclass: top
objectclass: organizationalUnit

2.) Create an ldif file containing the definition for the testadmin user 
***Note: Update the password value and dn as appropriate for your environment and requirements. If the User you wish to add as a member of the Global Admin Group already exists, proceed to step 4 and substitute that Users' DN in place of "uid=testadmin,ou=people,o=IBM,c=US". 

dn: uid=testadmin,ou=people,o=IBM,c=US 
cn: testadmin 
cn: Test Administrator 
sn: testadmin 
objectclass: top 
objectclass: person 
objectclass: organizationalPerson 
objectclass: inetOrgPerson 
uid: testadmin 
userpassword: secret 

3.) Add the user via idsldapadd 

==> idsldapadd -p 389 -D cn=root -w secret -i testadmin.ldif 
Operation 0 adding new entry uid=testadmin,ou=people,o=IBM,c=US 

4.) Add the User to the Global Admin Group 

Syntax: idsldapmodify -p <port> -h <hostname> -D <adminDN> -w <password> 

==> idsldapmodify -p 389 -D cn=root -w secret 
dn: globalGroupName=GlobalAdminGroup,cn=ibmpolicies 
changetype: modify 
add: member 
member: uid=testadmin,ou=people,o=IBM,c=US 

Operation 0 modifying entry 
globalGroupName=GlobalAdminGroup,cn=ibmpolicies 

5.) Confirm membership to the Global Admin Group

Option 1: Perform an ibm-allmembers search on the Global Admin Group

Syntax: idsldapsearch -p <port> -h <hostname> -D <adminDN> -w <password> -s <scope> -b <baseDN of the Group> objectclass=* ibm-allmembers

==> idsldapsearch -p 389 -D cn=root -w secret -s base -b "globalGroupName=GlobalAdminGroup,cn=ibmpolicies" objectclass=* ibm-allmembers 

globalGroupName=GlobalAdminGroup,cn=ibmpolicies
ibm-allmembers=uid=testadmin,ou=people,o=IBM,c=US

Option 2: Perform an ibm-allgroups search on the user

Syntax: idsldapsearch -p <port> -h <hostname> -D <userDN> -w <password> -s <scope> -b <baseDN of the User> objectclass=* ibm-allgroups

==> idsldapsearch -p 389 -D "uid=testadmin,ou=people,o=IBM,c=US" -w secret -s base -b "uid=testadmin,ou=people,o=IBM,c=US" objectclass=* ibm-allgroups

uid=testadmin,ou=people,o=IBM,c=US
ibm-allgroups=globalGroupName=GlobalAdminGroup,cn=ibmpolicies
