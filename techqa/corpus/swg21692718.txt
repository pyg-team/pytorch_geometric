Title: IBM Calendar entries are incorrect when using Samara and Minsk time zones - United States

Text:
dst; russia; russian dst; russian; calendar; time zone; timezone; patch; Samara; Minsk TECHNOTE (TROUBLESHOOTING)

PROBLEM
After installing the Russian DST 2014 patch for Windows, Notes/Domino Calendar entries are incorrect when using either the Samara or Minsk time zones.

SYMPTOM
 

Steps to Reproduce (using Notes 8.5.3 FP6 client)

1. Install Russian DST 2014 patch for Windows - http://support2.microsoft.com/kb/2998527 [http://support2.microsoft.com/kb/2998527]

2. Change the OS time zone setting to one of the following two time zones.

[/support/docview.wss?uid=swg21692718&aid=1] [/support/docview.wss?uid=swg21692718&aid=1]

[/support/docview.wss?uid=swg21692718&amp;aid=2]

3. Install Notes client 8.5.3FP6

4. Install 853FP6SHF225.

5. Open Standard Notes client, create a calendar entry 12:00-1:00pm in Nov 20, Open calendar one week, one month etc view, it shows 5:00 - 6:00am. 


RESOLVING THE PROBLEM
This issue is tracked as SPR# WWAG9Q8Q5N. To resolve the issue, download and install the JVM updates that are provided in technote 1692733 - Security Bulletin: Multiple vulnerabilities in IBM Java SDK affect IBM Notes and Domino [http://www.ibm.com/support/docview.wss?uid=swg21692733]
