ERROR DESCRIPTION
 *  IBM WebSphere DataPower Appliances
   
   Firmware version: 3.8.1.0
   
   Problem description: When trying to use the wrong userid and/or
   password on MQ connection, the appliance crashes if sharing
   conversation is turned on.
   
   
   Steps to recreate:
   - Create an MQ Queue Manager object with sharing conversation
   turned on.
   - Configure the userid/password with an incorrect value.
   - Set the admin state of the MQ Queue Manager (QM)  to enabled.
   
   Expected result: the MQ QM should be down.
   
   
    
   
   

LOCAL FIX
 *  Workaround: Verify that the correct password/userid is used.
   
   
    
   
   

PROBLEM SUMMARY
 *  Crash when sharing conversation is on and the connection is
   rejected by channel exit
   
   
    
   
   

PROBLEM CONCLUSION
 *  Correct the assignment of varialbe to avoid using invalid value.
   This fix will be available on 3.8.1.2
