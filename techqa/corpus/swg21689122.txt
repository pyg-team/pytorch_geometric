Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)

CAUSE
There is an ObjectType that is not a real property. It is an internal property used to identify the type of object. There is nothing to prevent defining a custom property with the same name of ObjectType. 


FEM has made a performance improvement in displaying the contents of a folder by using an SQL query. This query uses this ObjectType property and displays the results, based on the value returned.

When their is a custom property ObjectType , it takes precedence over the system created ObjectType and the results do not show the object Metadata

As said earlier, there is no real way of fixing this, rather than changing the display and symbolic names.


RESOLVING THE PROBLEM
 

 1.  Access FEM
     
     
 2.  Access the Object Store displaying the issue
     
     
 3.  Right click on Search Results
     
     
 4.  Click New Search
     
     
 5.  On the Content Engine Query Builder tab under Select From Table choose Property Template
     
     
 6.  Under Column choose ID
     
     
 7.  Under Condition choose Equal To
     
     
 8.  In Value enter the GUID of the Custom Property ObjectType
     
     
 9.  Click OK
     
     
 10. Once the Object displays right click and choose Properties
     
     
 11. Change both the Property and Symbolic Name to a name other than ObjectType (the name cannot contain spaces)
     
     
 12. Refresh the Object Store to verify the changes are implemented.
     
     
 13. Items should display properly
