LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   Users attempting to create a Data Sample where one of the
   columns is an Informix datetime data type
   ****************************************************************
   PROBLEM DESCRIPTION:
   The creation of Samples involving Informix datetime fields was
   failing with the error "[IIS-CONN-ODBC-000004] ODBC function
   {0} reported:  SQLSTATE =
   22007: Native Error Code = 242: Msg = [Microsoft][ODBC SQL
   Server Driver][SQL Server]The conversion of a char data type to
   esulted in an out-of-range datetime   value."
   ****************************************************************
   RECOMMENDATION:
   Update to Information Analyzer 81 RUP4.  This change is also
   included in 8.1 Fix Pack 1.
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  In Information Analyzer 81 RUP4 you will be able to create a
   Data sample containing an Informix datetime column.
