APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Attachment filename in Japanese is unreadable when using IE11
   with mxe.doclink.securedAttachment=true
   
   PERFORMANCE ISSUE
   No
   
   STEPS TO REPRODUCE:
   1. Go to -> System Configuration -> Platform Configuration ->
   System Properties
   (a) Filter for 'mxe.doclink.securedAttachment' and expand the
   record.
   (b) Update the value to 'true' and save.
   (c) Select Action -> Live Refresh.
   2. Create a new file (with file name in Japanese character) in
   client machine with following filename.
   3. Go To -> Service Requests -> Service Requests application.
   (a) Create a new Service Request record e.g. 1001
   (b) Attachment paperclip -> Add New Attachments -> Add New File
   (c) Select file with created in (2)
   (d) Click OK to attach.
   (e) View Attachments -> Click to save/open the attachment.
   (f) Notice the filename is unreadable
   =_UTF-8_B_5pel5pys6Kqe44OG44K544OILmRvY3g=_=
   *The issue
   
   CURRENT ERRONEOUS RESULT:
   Attachment filename in Japanese is unreadable when using IE11
   with mxe.doclink.securedAttachment=true
   
   EXPECTED RESULT:
   Attachment filename in Japanese is readable when using IE11
   with mxe.doclink.securedAttachment=true
   
   ADDITIONAL INFO:
   1. IE11 (normal mode) vs Firefox.
   - Issue only seen in IE11 normal mode.
   2. Attachment with Japanese character file name -vs- Attachment
   with English character file name using same IE11 normal mode.
   - Issue only seen in attachment with Japanese character file
   name although both tested with IE11 normal mode.
   
   ENVIRONMENT:
   - SCCD 7.5.1.3
   - Tpae: 7.5.0.8
   - Browser: IE11 normal mode
   
   
    
   
   

LOCAL FIX
 *  Use IE11 compatibility mode
