ERROR DESCRIPTION
 *  With manual syndication, create two syndication pair which share
   a same library. If syndication pair A has some failed item from
   the library, the status will be pending.
   
   While the status of syndication pair B will be Idle even the
   pending item list is not empty. The status for syndication pair
   B is not correct if sever is restarted.
   
   When creating a new syndication pair including the library, the
   status is Idle but the pending item list is not empty.
   
   
    
   
   

LOCAL FIX
 *  None available
   
   
    
   
   

PROBLEM SUMMARY
 *  With manual syndication, create two syndication pair which share
   a same library. If syndication pair A has some failed item from
   the library, the status will be pending.
   
   While the status of syndication pair B will be Idle even the
   pending item list is not empty. The status for syndication pair
   B is not correct if sever is restarted.
   
   When creating a new syndication pair including the library, the
   status is Idle but the pending item list is not empty.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Should always check the pending item list when setting the
   status of the syndicator.
