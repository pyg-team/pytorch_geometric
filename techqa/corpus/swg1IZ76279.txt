ERROR DESCRIPTION
 *  Parsin exception during change person when a dropdown attribute
   contains &.
   
   Description: If an attribute value containing ampersan(&)
   character such
   as "Customer&Dist. Mgmt./Sales&Dist. Mgmt." is changed to drop
   down box
   with a search control and then the change person form is opened
   then the
   following exception occurs:
   <Exception><![CDATA[org.xml.sax.SAXParseException: The reference
   to
   entity "Dist." must end with the ';' delimiter.
   
   Steps to Duplicate:
   1) Add the attribute departmentnumber to the corporate tab of
   the
   default ITIM Person form.
   2) Configure the departmentnumber attribute as a DropDown Box
   control
   with the objectclass set to inetOrgPerson(or custom objectclass
   in
   customer case), and the attribute names departmentnumber for
   both.
   3) Create a person object with departmentnumber containing an &
   character.
   4) Attempt to modify the person created in step 3 in ITIM GUI.
   Parsing
   exception results.
   
   Desired Behavior: To be able to manage ITIM Person objects when
   an &
   character is contained in the attribute value for a DropDown Box
   control.
   
   Environment: ITIM V5.1 IF9 and IF10

 *  n/a
   
   
    
   
   

PROBLEM SUMMARY
 *  Users affected:
   Customers with an & (ampersand) in a drop down attribute.
