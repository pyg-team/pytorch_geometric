Title: IBM Maximo Performance Degrades Until Application Hangs and/or Crashes - United States

Text:
perf; degrade; slow; hang; stops; TPAEPERFORMANCE FLASH (ALERT)

ABSTRACT
 It has been reported where Maximo performance degrades over time until it appears that the Application Server has frozen and/or crashed. 

CONTENT

Every Maximo transaction uses an Application Server (WebLogic/WebSphere) thread or threads. When the transaction has finished processing, the thread is released back to the thread pool and is available for the next transaction. When threads are not released back to the thread pool after the transaction is completed, this is referred to as a "stuck thread". Stuck threads are eventually released and will not have the same effect upon your system. Threads that are never released back to the thread pool is referred to as a "thread deadlock". Deadlocks also have the negative impact of decreasing the available thread pool count. As threads are deadlocking the available thread pool can drop very low and transactions start waiting for resources to process. 

The first symptom of this problem is a slowdown in general Maximo Application Server performance. When the available thread pool gets to zero, the Maximo Server may stop running (crashes).


To determine if there are deadlocks and your system is experiencing this problem, you will need to generate a thread dump. See below document link. Provided is an example of a thread dump where this issue occurs. 

The below thread dump example has been logged as issue # 68008 and is fixed in Maximo 5.2 Patch3.
