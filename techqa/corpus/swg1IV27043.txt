   Problem Discription:
   When config the MQ FTE agent in silent mode, the instance name
   specified by option -o is lost in the agent .cfg file. As a
   result, no information is shown in MQ FTE workspace.  The issue
   only happens on ITM 622fp4 to 622fp9.  ITM623 releases works
   fine.
   
   Detailed Recreation step
   
   (1)install Monitoring Agent for WebSphere MQ File Transfer
   Edition 07.01.00.00
   (2)install ITM 622fp4 unix or linux os agent
   (3)silent config a mq agent by issuing
   ./itmcmd config -A -p /tmp/mqconfig.txt -o showmqinstance m6
   (4)finsh the config and check the file <machine>_m6_instance.cfg
   file,
    INSTANCE= [    SECTION=COORQM
   the instance is empty.
