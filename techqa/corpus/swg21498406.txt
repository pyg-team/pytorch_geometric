DIAGNOSING THE PROBLEM
WCM was not configured properly. When the /portalserver/wps.properties file was checked, the following parameter was found: WPInstallType=base 


In a Base install, WCM is not configured by defaul which is why the script failed. For more information, refer to the Portal wiki documentation, "Adding features to a base installation: WP7 [http://www-10.lotus.com/ldd/portalwiki.nsf/dx/Adding_features_to_a_base_installation_wp7]".


RESOLVING THE PROBLEM
To work around this issue: 


1. Install Portal --> Base. 

2. Apply the WCM critical fix.

3. Execute the following ConfigEngine script. This will deploy the missing local rendering portlet which is not installed in the Base version. 

 * /ConfigEngine.sh configure-wcm-authoring


4. Execute the following ConfigEngine script:  * ./ConfigEngine.sh update-wcm
