LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * Users using Case Analyzer with 5.2.1.7-P8CPE-FP007 or prior. *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * With Case Analyzer store created on Oracle database, event   *
   * processing stops with the  exception -                       *
   * "java.sql.SQLException: ORA-12899: value too large for       *
   * column", when there are multi-byte characters in the string  *
   * user defined field values.                                   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Upgrade to 5.5.0.0-P8CPE or above. New user defined fields   *
   * created after 5.5.0.0-P8CPE will not cause the mentioned     *
   * issue. For existing user defined fields created prior to     *
   * 5.5.0.0-P8CPE, follow the instructions in the below          *
   * technote:                                                    *
   * http://www-01.ibm.com/support/docview.wss?uid=swg22009650    *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  For new string user defined fields, the issue is fixed in
   5.5.0.0-P8CPE and above. Case Analyzer schema is altered to
   accommodate multi-byte characters.
