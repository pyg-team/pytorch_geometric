Title: IBM Unable to log in to TM1 Web - renders blank page - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When going to the URL http://<servername>/TM1Web a blank page is 
displayed rather than the login page. 

SYMPTOM
No error in the Web GUI, just the blank page, but the tm1web.log may show a message similar to the following: 

2012-08-03 17:06:18,880 [9] ERROR Applix.TM1.Web.Page.TM1WebError -Error Details:
System.InvalidOperationException: This implementation is not part of
the Windows Platform FIPS validated cryptographic algorithms.


CAUSE
Using FIPS (Federal Information Processing Standard) 


RESOLVING THE PROBLEM
If using FIPS refer to
1.) Microsoft KB article http://support.microsoft.com/kb/911722 [http://support.microsoft.com/kb/911722]
2.) MSDN Developer site on how to fix the issue: 
http://social.msdn.microsoft.com/Forums/en/clr/thread/7a62c936-b3cc-4493 [http://social.msdn.microsoft.com/Forums/en/clr/thread/7a62c936-b3cc-4493]
-a3cd-cc5fd18b6b2a 

1. In a text editor such as Notepad, open the application-level
Web.config file.
2. In the Web.config file, locate the <system.web> section.
3. Add the following <machineKey> section to the <system.web>
section:"
<machineKey validationKey="AutoGenerate,IsolateApps"
decryptionKey="AutoGenerate,IsolateApps" validation="3DES"
decryption="3DES"/>"
4. Save the Web.config file.
5. Restart the Microsoft Internet Information Services (IIS) service.
