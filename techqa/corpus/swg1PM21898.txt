LOCAL FIX

PROBLEM SUMMARY
 *  Some customers are experiencing odd behavior with the
   $TemplateCatalog object. It seems the parameter 'all', which was
   used in prior Portal versions causes a failure, however, the
   parameter 'root' which is in the Portal sample does behave
   properly.
   
   
    
   
   

PROBLEM CONCLUSION
 *  $TemplateCatalog find folder [all] returns now all template
   folders.
