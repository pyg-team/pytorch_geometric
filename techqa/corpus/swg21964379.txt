Title: IBM SPSS Statistics server write its temporary files in /tmp instead of SPSSTMPDIR - United States

Text:
SPSSTMPDIR /tmp statistics SPSS SPSSTMPDIR_DEFAULT TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 We have a main issue with SPSS Statistics server which writes its temporary files in /tmp instead of the path described in the variable SPSSTMPDIR set in the start procedure of Statistics. 
This fulfill /tmp very quickly. 

RESOLVING THE PROBLEM
Changing the following line in the statsenv.sh and restarting Statistics server solved the issue (removed the hyphen): 

SPSSTMPDIR=${SPSSTMPDIR:-$SPSSTMPDIR_DEFAULT}
export SPSSTMPDIR

replaced by:
SPSSTMPDIR=${SPSSTMPDIR:$SPSSTMPDIR_DEFAULT}
export SPSSTMPDIR
