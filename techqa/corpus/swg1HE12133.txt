ERROR DESCRIPTION
 *  Following are steps to replicate the problem
   1. Create a Matter and save it
   2. Create a Request with org unit ( with a few users assigned)
   and save
   3. Save the request. Confirm that the users from the org are
   part of  people in scope
   4. Add another user manually ( user should not be part of the
   org added earlier)
   5. Save and close
   6. Create a Preservation Plan and make sure you manually add
   custodians. Save the Plan
   7. Add System admin as the owner of the plan. Under
   custodians tab - Click add custodians and in the popup click
   add all
   8. Click Remove all
   9.  Add only the custodian that was added manually to the
   scope. Publish the preservation Plan
   10. Click on custodian tab and then click All on Hold button
   11. Edit the request and then delete the org and save. Under
   people in scope you will see only one user
   12. Map > Person > search for one of the person from the org
   and click all Holds and Collection, the matter is seen in the
   list
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  After removing a person from a preservation plan, the person
   still appears in Map>Person>All Holds and Collections for the
   associated matter.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This fix is available now from FixCentral in fixpack 6.0.1.7.
