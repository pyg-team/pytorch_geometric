Title: IBM BMXAA4198E - The GL attribute GLACCOUNT on object {1} requires an organization - United States

Text:
REST API; SR; INTEGRATION FRAMEWORK; MIF; TPAEINTEGRATION TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When creating new Service Requests using REST API, the error below is thrown if a GLACCOUNT is defined.



psdi.util.MXAccessException: BMXAA4198E - The GL attribute GLACCOUNT on 
object {1} requires an organization. Specify a value in the Organization field.

java:744) 

SYMPTOM
Integration is unable to create Service Requests


CAUSE
This behavior is also seen in the Service Requests application if the user directly input the GL Account in the field instead of using the GL Navigator dialog.

The GL navigator requires an ORGID / SITEID value in order to display and validate the GL Accounts for a specific record.


DIAGNOSING THE PROBLEM
Using Service Requests application, if the user directly inputs a GL 
Account value in the field, the same error will occur, because an ORGID has not been informed. This is how the GL Navigator is designed.

RESOLVING THE PROBLEM
Solution for this problem is to provide both ASSETORGID and ASSETSITEID when creating a new SR using REST API. 

This is a REST request sample, sending ASSETORGID and ASSETSITEID.
