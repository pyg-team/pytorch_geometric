Title: IBM Using TIMEOUTLENIENCY to allow processing of SCA asynchronous messages with expired LTPA tokens - United States

Text:
LTPA; expired tokens; asynchronous SCA messages; TIMEOUTLENIENCY; LTPA timeout; token expired; TokenExpiredException; SCA security TECHNOTE (FAQ)

QUESTION
 I have set the Lightweight Third Party Authentication (LTPA) timeout value to the maximum value allowed by internal enterprise security standards, but I still have asynchronous Service Component Architecture (SCA) messages with LTPA tokens that expire before the messages can be processed. How can I resolve this issue? 

CAUSE
WebSphere Process Server uses the security layer of WebSphere Application Server to create secure communications with LTPA tokens. By default, an LTPA token times out after 120 minutes, though it is possible for a token to expire even before the specified LTPA timeout value (see document: Token timeout behavior when LTPA is used as the authentication mechanism [http://www.ibm.com/support/docview.wss?uid=swg21320747] [http://www-01.ibm.com/support/docview.wss?rs=2307&uid=swg21320747]).
Because asynchronous SCA messages can sometimes stay in the message queue for a long time, LTPA tokens in those messages that use SCA security handlers can expire before the messages can be processed.

 *  Configuration setting or variable Valid and default values: 
   WebSphere Process Server V7.0.0.0, V7.0.0.1, and V7.0.0.2 Valid and default values: 
   WebSphere Process Server V7.0.0.3 LTPA timeout
   
   (Specified in the LTPA timeout value for forwarded credentials between servers field on the LTPA page in the administrative console) Valid value: An integer between 5 and 35971
   
   Units: Minutes
   
   Default value: 120 minutes Valid value: An integer between 5 and 35971
   
   Units: Minutes 
   
   Default value: 120 minutes TIMEOUTLENIENCY Valid value: A positive integer between 0 and 35791  *   *

 *  Open the administrative console. Click Environment > WebSphere Variables. Select the scope of the variable. Click New. Create a new variable called TIMEOUTLENIENCY and set its value according to your needs. Click OK. Restart the server.
