Title: IBM JR41849: THE DEFAULT VALUED DATES ARE USING INCORRECT FORMAT IN "NEW UI " - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Incorrect interpretation of a date by the product.
   
   Steps to reproduce:
   1)In the "My Setting" page, set the "Datetime Input
   format"/""Datetime Output format" setting to Default Format
   M/D/yy h:mm (2-digit year).
   2) Set the date value on an item to 01/30/2060 using the
   calender picker.
   3) You will notice that the value is being changed to
   01/30/1960.
   
   This issue is not reproducible in the classic UI.
   
   keyword: attribute, timestamp
   
   
    
   
   

LOCAL FIX
 *  Workaround: In "My Setting" set the value for  Datetime Output
   format and Datetime Input format to be User Select MM-dd-yyyy
   HH:mm:ss (4-digit year).
