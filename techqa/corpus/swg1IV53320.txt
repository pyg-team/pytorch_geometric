ERROR DESCRIPTION
 *  When we use Test connection button on the Oracle DSA (the one
   shown
   above) it corrupts the data source file for the datasource and
   impacts
   connectivity with datasource
   
   Example from PSUP (For Oracle HEA datasource)
   Custom URL in the datasource file (below is correct one) -
   working one
   
   DS_ORA_HEA.Oracle.CUSTOMURL=(DESCRIPTION_LIST\=
   (LOAD_BALANCE\=off)
   (FAILOVER\=on) (DESCRIPTION \= (ADDRESS \= (PROTOCOL \=
   TCP)(HOST \=
   ora-svc-ASS05PS-APP.nbndc.local)(PORT \= 1521))(CONNECT_DATA \=
   (SERVICE_NAME \= ASS05PS_APP.nbndc.local))) (DESCRIPTION \=
   (ADDRESS \=
   (PROTOCOL \= TCP)(HOST \=
   ora-svc-ASS05PS-APP-stby.nbndc.local)(PORT \=
   1521))(CONNECT_DATA \= (SERVICE_NAME \=
   ASS05PS_APP.nbndc.local))) )
   
   
   When we use Test Connection Button: above URL was getting
   changed in
   file to
   
   DS_ORA_HEA.Oracle.CUSTOMURL=&\#40;DESCRIPTION_LIST\=
   &\#40;LOAD_BALANCE\=off&\#41; &\#40;FAILOVER\=on&\#41;
   &\#40;DESCRIPTION
   \= &\#40;ADDRESS \= &\#40;PROTOCOL \= TCP&\#41;&\#40;HOST \=
   ora-svc-ASS05PS-APP.nbndc.local&\#41;&\#40;PORT \=
   1521&\#41;&\#41;&\#40;CONNECT_DATA \= &\#40;SERVICE_NAME \=
   ASS05PS_APP.nbndc.local&\#41;&\#41;&\#41; &\#40;DESCRIPTION \=
   &\#40;ADDRESS \= &\#40;PROTOCOL \= TCP&\#41;&\#40;HOST \=
   ora-svc-ASS05PS-APP-stby.nbndc.local&\#41;&\#40;PORT \=
   1521&\#41;&\#41;&\#40;CONNECT_DATA \= &\#40;SERVICE_NAME \=
   ASS05PS_APP.nbndc.local&\#41;&\#41;&\#41; &\#41;
   
   Brackets are getting replaced with &\#41; characters in the
   datasource
   file
   
   MDV in Error is 5.1.1-TIV-BSM-FP0002
