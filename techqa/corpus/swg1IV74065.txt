APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The ITM data provider provides the event results data for
   siuations but is missing the "owner" column. The owner column
   provides the userid that acknowledged the situation event.
   
   
    
   
   

LOCAL FIX
 *  no work around
   
   
    
   
   

PROBLEM SUMMARY
 *  IBM Tivoli Monitoring data provider did not provide Owner column
    for event row data.
   
   
   Added support so the Owner column shows up in the event payload
   from the IBM Tivoli Monitoring data provider.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Event requests UIR's will now contain the Owner column.  The
   owner column will contain an owenr ID if the ITM event has been
   acknowledged.
   
   The fix for this APAR is contained in the following maintenance
   packages:
