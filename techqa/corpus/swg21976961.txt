Title: IBM Does LightWeight JDBC adapter in IBM Sterling B2B Integrator return only one value while calling Oracle Stored Procedure? - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 In IBM Sterling B2B Integrator (SBI), the LightWeight JDBC adapter (LWJDBC adapter) does not return results of all output PARAMS while calling a Oracle Stored Procedure. 

SYMPTOM
For example : 

CREATE OR REPLACE PROCEDURE admin.IBM_TEST(oOut out number,oERR_CODE OUT VARCHAR2, oMSG_RESULT OUT VARCHAR2, iPROCESS_ID IN NUMBER,iSERIAL_NUMBER IN VARCHAR2,iSN_VERSION IN VARCHAR2,iPartNo in varchar2) IS
begin

Result: 


<?xml version='1.0' encoding='UTF-8'?><Result><Rows><ReturnValue>100</ReturnValue></Rows><OUTPUT_PARMS><PARM1>100</PARM1></OUTPUT_PARMS></Result>

RESOLVING THE PROBLEM
The SBI LWJDBC adapter can return ONLY one value when calling an Oracle stored procedure. This is by design.
