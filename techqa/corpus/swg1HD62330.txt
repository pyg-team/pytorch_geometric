Title: IBM HD62330: POSSIBLE TO UNLOCK PART REFERENCE LOCKED BY OTHER USER - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Possible to Unlock Part Reference locked by other
   user
   Scenario
   1. Logon to ENOVIA as User1 with Role.Org.Project .
   2. Create a product P1 and open it in Product Editor.
   3. Insert a part/instance under the product and
   lock only the part reference. Save the structure
   and exit LCA.
   4. Again login to ENOVIA as User2 with same
   context, Role.Org.Project.
   5. Open the same product P, expand the Product
   Structure
   6. Lock the  Usage of Part whose reference was
   already locked by other User1.
   7. Save and Select Part in List and choose the
   Unlock Selections option.
   => Actual result: Both Usage and Reference is
   unlocked
   But the Reference was locked by other user.
   Expected Result: Only Usage lock is unlocked.
   .
