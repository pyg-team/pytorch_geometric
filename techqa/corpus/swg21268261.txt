PROBLEM(ABSTRACT)
 In order to begin diagnosing any issue on ITDS version 6.0, the first step is to identify the version, including fixpack level of the ITDS environment. This document details the steps for determining the ITDS version information by supported platform.

Gathering version information on AIX 
Collect the output of the following commands: 

lslpp -l | grep ldap 
lslpp -l | grep db2 
lslpp -l | grep gsk 
ibmslapd -V 
idsilist -a 

If the Web Administration Tool is installed and configured collect the output from: 
./opt/IBM/ldap/V6.0/idstools/deploy_IDSWebApp.sh -v 

Gathering version information on HP 
Collect the output of the following commands: 

swlist | grep ldap 
swlist | grep -i db2 
swlist | grep -i gsk 
ibmslapd -V 
idsilist -a 

If the Web Administration Tool is installed and configured collect the output from: 
./opt/IBM/ldap/V6.0/idstools/deploy_IDSWebApp.sh -v 

Gathering version information on Linux 
Collect the output of the following commands: 

rpm -qa | grep -i ldap 
rpm -qa | grep -i db2 
rpm -qa | grep -i gsk 
ibmslapd -V 
idsilist -a 

If the Web Administration Tool is installed and configured collect the output from: 
./opt/ibm/ldap/V6.0/idstools/deploy_IDSWebApp.sh -v 

Gathering version information on Solaris 
Collect the output of the following commands: 

pkginfo | grep –i idsl 
pkginfo –l IDSlbc60 IDSl32p60 
pkginfo | grep –i db2 
pkginfo | grep –i gsk 
ibmslapd –V 
idsilist -a 

If the Web Administration Tool is installed and configured collect the output from: 
./opt/IBM/ldap/V6.0/idstools/deploy_IDSWebApp.sh -v 

Gathering version information on Windows 
Collect the output of the following commands: 

Dir “<Install_Drive>:\Program Files\IBM\LDAP\V6.0\bin” 
<Install_Drive>:\Program Files\IBM\SQLLIB\bin\db2level.exe 
Dir “<Install_Drive>:\Program Files\IBM\gsk” 
ibmslapd –V 
idsilist -a 

If the Web Administration Tool is installed and configured collect the output from: 
Dir “<Install_Drive>:\Program Files\IBM\LDAP\V6.0\deploy_IDSWebApp.sh -v
