Title: IBM PK59793; 6.1: Putting a server in maintenance mode returns HTTP 1.1 500 - United States

Text:
PK59793; PK59793; PK59793 DOWNLOADABLE FILES

ABSTRACT
 java.lang.NullPointerException is seen on the On-Demand Router (ODR) when HTTP requests are flowing. 

DOWNLOAD DESCRIPTION
PK59793 resolves the following problem:

ERROR DESCRIPTION: 
IBM® WebSphere® Extended Deployment V6.1 maintenance mode feature does not work as expected. After placing a server into maintenance mode, intermittent HTTP 1.1 500 responses are returned when attempting to establish a session. 

The server state was not being initialized. The following exception is thrown: 

HttpFilterCha E PROX0050E: Caught unhandled exception from doFilter in filter=XD load balance filter.; exception=java.lang.NullPointerException at 
com.ibm.ws.xd.dwlm.client.XDTargetServerImpl.isUp(XDTargetServerImpl.java:234) at
com.ibm.ws.dwlm.client.TargetServerImpl.isAvailable(TargetServerImpl.java:178) at 
com.ibm.ws.xd.dwlm.client.XDTargetServerImpl.isAvailable(XDTargetServerImpl.java:226) 

LOCAL FIX: 
N/A 

PROBLEM SUMMARY

USERS AFFECTED:
All users of WebSphere Extended Deployment V6.1 using the On Demand Router.

PROBLEM DESCRIPTION:
java.lang.NullPointerException is seen on the On-Demand Router (ODR) when HTTP requests are flowing.

RECOMMENDATION:
None

The following exception is seen in the ODR logs during traffic:

[1/21/08 9:59:58:095 PST] 00000018 HttpFilterCha E PROX0050E: Caught unhandled exception from doFilter in filter=XD load balance filter.; exception=java.lang.NullPointerException at com.ibm.ws.xd.dwlm.client.XDTargetServerImpl.isUp(XDTargetServerImpl.java:234) at
com.ibm.ws.dwlm.client.TargetServerImpl.isAvailable(TargetServerImpl.java:178) at
com.ibm.ws.xd.dwlm.client.XDTargetServerImpl.isAvailable(XDTargetServerImpl.java:226)

PROBLEM CONCLUSION: 
The code was fixed to avoid the exception and the fix was made available for the 6.1.0.0 release. The fix will be included in Service Pack 6.1.0.1 for WebSphere Extended Deployment.
