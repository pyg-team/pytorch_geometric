Title: IBM Operations Insight 1.2 installation Problem - United States

Text:
WebGUI 8.1; NOI; Netcool Operations Insight:Installation; Exception TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The Installation of Operations Insight into an existing WebGUI 8.1 fails with the following error in Installation Manager 
"typedef class com.ibm.tivoli.tacct.psc.install.server.config.IsWasRunning cannot be found
using the classloader AntClassLoader[/opt/IBM/netcool/omnibus_webgui/bin/OMNIbusWebGUI_IM.jar] [/opt/IBM/IMShared/files/com.ibm.noi.ea.antfile_1.2.0.201409291539.xml:657" 

SYMPTOM
The installation stops within installation manager. 

The line it stops at in the file specified in the error /opt/IBM/IMShared/files/com.ibm.noi.ea.antfile_1.2.0.201409291539.xml 

suggests there is a class it cannot find in the file the classpath property is pointing at: 

line 657 

<typedef name="isWasRunning" classname="com.ibm.tivoli.tacct.psc.install.server.config.IsWasRunning" classpath="${OMNIBUS_WEBGUI_HOME}/bin/OMNIbusWebGUI_IM.jar"/> 

Running jar tvf in that file can show that the class is not present within it

CAUSE
There are two different GA releases of WebGUI 8.1 

June 2014 GA version (build 493) and September 2014 GA version (build 615).



ENVIRONMENT
OMNIbus WebGUI 8.1



DIAGNOSING THE PROBLEM
Installation Manager graphical window and its logs show an exception



RESOLVING THE PROBLEM
Operations Insight needs the later version (build 615) in order to install. After this version is in the system, jar tvf shows the class the installer didn't find before
