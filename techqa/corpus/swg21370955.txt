CAUSE
Corrupt settings inside end user's ccr.config configuration file.



RESOLVING THE PROBLEM
Reset the end user's ccr.config file to the default settings. 

TIP: Some user-specific settings (for example 'local preferences') will be lost by doing this. Therefore, as a precaution, take a backup of the old file (or entire folder) in case the user requires those settings to be manually copied over again. 

 

Steps: 

Ask the 'bad' end user to do the following, when logged onto the 'bad' client device (for example Windows XP laptop) as his/her normal Windows user: 

 1. Close Controller (if it is running) 
 2. Click "Start - Run" 
 3. Type the following: %APPDATA% <enter> 
 4. Open the subfolder "Cognos" 
 5. Rename the subfolder "CCR" to "CCR.old" 
 6. Test
