 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Release:  7.2.1 IF07 and above
   
   Environment: ITCAM for SOA v7.2.1.7 and above
   OS: any
   
   Problem Description:
   HD agent(Warehouse Proxy) is unable to populate historical
   table Fault_Log_Table_610 (KD42ET) of KD4 agent(ITCAM for SOA)
   with error message:
   ERROR MESSAGE: "A value in the attribute Fault_String_Long_U
   after UTF8 conversion has a length (n) greater than the column
   size (1024). The value is printed in the hd trace file"
   It is caused by code changes in ITCAM for SOA 7.2.1 IF07, and
   null value of Fault_String_U is condition for this issue.
   Detailed Recreation Procedure:
   1) Enable historical data collection for table
   Fault_Log_Table_610 (KD42ET) of KD4 agent, but data is failed
   to be written into warehouse database.
   2) Open Warehouse Proxy error messages from TEP, below error
   message with 'Critical' error severity displays:
   A value in the attribute Fault_String_Long_U after UTF8
   conversion has a length (n) greater than the column size
   (1024). The value is printed in the hd trace file
   3) This issue happens sometimes only when Fault_String_U
   attribute of Fault_Log_Table_610 attribute group is null.
