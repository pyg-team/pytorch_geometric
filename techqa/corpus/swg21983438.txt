Title: IBM DASH Page Cycler Creates additional Page tabs each time through the cycle. - United States

Text:
DASHL2INST; DashTools; Cumulative Patch4; DASH 3.1.2.1; Page Cycler.; Cumulative Patch 3; IV85266 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Starting with DASH Cumulative Patch 4, the DashTools Page Cycler Widget creates new Page tabs each time it cycles through the list of Pages to display, each page name will show an incrementing number in brakets after
the name. name(1), name(2) etc 

SYMPTOM
When displaying the Pages defined in the Page Cycler Widget, instead of cycling through the pages to display, DASH creates a new Page Tabs each time it cycles through the list. Here is an example, you can see that the instance number increases over time. It will keep creating new tabs until you stop the Page Cycler. 

[/support/docview.wss?uid=swg21983438&aid=1] [/support/docview.wss?uid=swg21983438&aid=1]


CAUSE
Cause yet to be determined. Please refer to the Resolving the problem section below.



ENVIRONMENT
DASH 3.1.2.1 Cumulative Patch 4



DIAGNOSING THE PROBLEM
Please refer to the Symptom section above for see the behavior after Cumulative Patch 4 is installed. 

After you have uninstalled DASH Cumulative Patch 4, you should see the following with no increase in the number of Pages defined:

[/support/docview.wss?uid=swg21983438&aid=2] [/support/docview.wss?uid=swg21983438&aid=2]
