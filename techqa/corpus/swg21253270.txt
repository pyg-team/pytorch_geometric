ABSTRACT
 This document explains how to configure the JDBC drivers for WebSphere Business Integration Adapter for JDBC and provides examples (on the Windows platform) for Oracle, DB2 and SQL Server databases. 

CONTENT
When connecting to a database, the WebSphere Business Integration Adapter for JDBC needs corresponding JDBC drivers, and for the product to work efficiently, some configuration is required. The drivers are provided by the database providers, which are shipped with database products and available from the providers' Web sites. 

To configure the JDBC drivers, follow these steps: 

 1. Add the JDBC drivers to the class path.
    
    Edit the start_JDBC.bat file (on Windows) and remove the text REM before "SET JDBCDRIVERPATH=". Set the JDBCDRIVERPATH value with the file path of the drivers, as shown in the following examples:
    
    Oracle 
    SET JDBCDRIVERPATH=D:\Oracle_driver\ojdbc14.jar
    
    DB2
    SET JDBCDRIVERPATH=D:\DB2_driver\db2jcc.jar;D:\DB2_driver\db2jcc_license_cu.jar
    
    MS SQL Server
    SET JDBCDRIVERPATH=D:\sqlserver_driver\sqljdbc.jar 
    
    Save the start_JDBC.bat file. 
 2. Specify the values for the DatabaseURL and JDBCDriverClass properties.
    
    Edit the adapter configuration file using Connector Configurator; specify the corresponding values for the DatabaseURL and JDBCDriverClass properties, as shown in the following examples:
    
    Oracle
    DatabaseURL: jdbc:oracle:thin:@IP:1521:SID
    JDBCDriverClass: oracle.jdbc.driver.OracleDriver
    
    DB2
    DatabaseURL: jdbc:db2://IP:50000/DBName
    JDBCDriverClass: com.ibm.db2.jcc.DB2Driver
    
    MS SQL Server
    DatabaseURL: jdbc:sqlserver://IP:1433;DatabaseName=DBName;SelectMethod=cursor
    JDBCDriverClass: com.microsoft.sqlserver.jdbc.SQLServerDriver
    
    Note: Replace IP, SID, and DBName with the correct values for your environment.
    
    Save the adapter configuration file.
    
 3. Run the start_JDBC.bat file to start the adapter.
