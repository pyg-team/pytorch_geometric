APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  This APAR covers changes to the IBM MQ Queue Manager to
   disallow the use of CipherSpecs which specify cryptographic
   algorithms that are now
   considered to be broken or weak.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   This affects users of IBM MQ who are
   using SSL/TLS security on queue manager channels.
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   In line with industry security guidelines and research,
   IBM MQ now considers the following CipherSpecs to be weak:
   
   
   TLS_RSA_WITH_3DES_EDE_CBC_SHA
   ECDHE_ECDSA_3DES_EDE_CBC_SHA256
   ECDHE_RSA_3DES_EDE_CBC_SHA256
   
   
    
   
   

PROBLEM CONCLUSION
 *  The CipherSpecs identified in the list above will no longer be
   permitted by default when initiating MQ channels.
   
   To re-enable these CipherSpecs, please refer to the relevant
   release's documentation:
   
   7.1 : http://www-01.ibm.com/support/docview.wss?uid=swg1IV73287 [http://www-01.ibm.com/support/docview.wss?uid=swg1IV73287]
   7.5 : http://www-01.ibm.com/support/docview.wss?uid=swg1IV73287 [http://www-01.ibm.com/support/docview.wss?uid=swg1IV73287]
   8.0 :
   http://www.ibm.com/support/knowledgecenter/en/SSFKSJ_8.0.0/com.i
   bm.mq.sec.doc/q120565_.htm?view=kc#q120565___deptls [http://www.ibm.com/support/knowledgecenter/en/SSFKSJ_8.0.0/com.ibm.mq.sec.doc/q120565_.htm?view=kc#q120565___deptls]
   9.0 :
   https://www.ibm.com/support/knowledgecenter/en/SSFKSJ_9.0.0/com.
   ibm.mq.sec.doc/q120565_.htm?view=kc#q120565___deptls [https://www.ibm.com/support/knowledgecenter/en/SSFKSJ_9.0.0/com.ibm.mq.sec.doc/q120565_.htm?view=kc#q120565___deptls]
   
   
   Once re-enabled, the CipherSpec listed above can be used to
   transfer up to 32 GB of data before the connection is terminated
   with error AMQ9288. To avoid this error, either avoid using
   triple DES, or enable secret key reset when using these
   CipherSpecs.
