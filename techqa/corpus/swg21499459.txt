Title: IBM StackOverflowError when checking a project or building a ruleset - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Building or checking the consistency of large rule projects in Rule Team Server (RTS) or Decision Center can generate a StackOverflowError. 

SYMPTOM
The following message is shown in the application server log: 

SEVERE: ilog.rules.teamserver.brm.builder.IlrBuildException: null 
ilog.rules.teamserver.brm.builder.IlrBuildException: 
ilog.rules.teamserver.brm.builder.IlrBuildException: null 
at ilog.rules.teamserver.brm.builder.IlrBuilder.doConvert(IlrBuilder.java:585) 
at ilog.rules.teamserver.brm.builder.IlrBuilder.getBuildResult(IlrBuilder.java:359) 
at ilog.rules.teamserver.ejb.ruleset.IlrRulesetArchiveGenerator.doAddRuleArtifactToArchiveBuilder1(IlrRulesetArchiveGenerator.java:856) 
...
Caused by: java.lang.StackOverflowError 
at ilog.rules.brl.brldf.IlrBRLDefinitionHelper.getGrammarProperty(IlrBRLDefinitionHelper.java:1972) 
at ilog.rules.brl.brldf.IlrBRLDefinitionHelper.getStringProperty(IlrBRLDefinitionHelper.java:1752)
...


CAUSE
The JVM reaches the limit of the available thread stack size and throws an exception.


DIAGNOSING THE PROBLEM
Check the JVM settings for the server hosting RTS/Decision Center, and more specifically its -Xss parameter.



RESOLVING THE PROBLEM
The size of the JVM stack can be modified using the -Xss parameter. 

Increase its value to avoid the error.
