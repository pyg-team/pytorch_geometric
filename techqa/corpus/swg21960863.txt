Title: IBM LogJam vulnerability interim fix for embedded WebSphere Application Server - Install procedure - United States

Text:
CVE-2015-4000; LogJam; Log Jam; vulnerability; embedded WAS; eWAS; embedded WebSphere Application Server; IDSWebApp; fix pack; IDSWebApp; SDS WebAdmin; TDS WebAdmin; TDS Web Administration Tool; upgrade eWAS TECHNOTE (FAQ)

QUESTION
 How to install LogJam vulnerability interim fix for embedded WebSphere Application Server (provided by Directory Server)? 

CAUSE
LogJam vulnerability (CVE-2015-4000 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2015-4000]) requires that the Java (JRE/JDK) used by embedded WebSphere Application Server(eWAS) be updated with the interim fix PI42778 (for eWAS 7.0.0.31 or later) and with the interim fix PI42779 (for eWAS 6.1.0.47). 

ANSWER
The tables on the Directory server recommended fixes [http://www.ibm.com/support/docview.wss?uid=swg27009778] page provides the latest recommended eWAS fix levels. 

This LogJam Vulnerability CVE-2015-4000 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2015-4000] problem is fixed in eWAS (and WAS) with the interim fix PI42778 [http://www.ibm.com/support/docview.wss?uid=swg24040145] (for eWAS 7.0.0.31 or later) and with the interim fix PI42779 [http://www.ibm.com/support/docview.wss?uid=swg24040182] (for eWAS 6.1.0.47). 

This document provides installation procedure for Log Jam vulnerability interim fix for embedded WebSphere Application Server provided by Directory Server(SDS / TDS). 

1. Identify the eWAS install location. (eWAS_install_location) 

Here is the default install location of eWAS if SDS/TDS provided GUI was used during product installations. 

V6.4: 

Note: V6.4 provided full WAS 8.5.5, Use instructions provided in WebSphere Security Bulletin [http://www.ibm.com/support/docview.wss?uid=swg21957980]. Look into section "For V8.5.0.0 through 8.5.5.6 Full Profile". 

V6.3.1: 

AIX, Solaris: /opt/IBM/ldap/V6.3.1/appsrv
Linux: /opt/ibm/ldap/V6.3.1/appsrv
Windows: <ITDS_V6.3.1_install_location>\appsrv 

V6.3: 

AIX, Solaris: /opt/IBM/ldap/V6.3/appsrv
Linux: /opt/ibm/ldap/V6.3/appsrv
Windows: <ITDS_V6.3_install_location>\appsrv 

V6.2: 

AIX, HP-UX, Solaris: /opt/IBM/ldap/V6.2/appsrv
Linux: /opt/ibm/ldap/V6.2/appsrv
Windows: <ITDS_V6.2_install_location>\appsrv 

V6.1: 

AIX, HP-UX, Solaris: /opt/IBM/ldap/V6.1/appsrv
Linux: /opt/ibm/ldap/V6.1/appsrv
Windows: <ITDS_V6.1_install_location>\appsrv 

2. Find the eWAS version that is in use and determine the need for fix pack installation: 

# for AIX, HP-UX, Solaris and Linux, open a login shell
cd <eWAS_install_location>/bin
./versionInfo.sh 

# for Windows open a command prompt:
cd <eWAS_install_location>\bin
versionInfo.bat 

 

3. Install latest recommended eWAS and WASSDK fix packs: 

3. a. For eWAS 7.0 provided by SDS 6.3.1 or TDS 6.3, install latest recommended eWAS fix pack [http://www.ibm.com/support/docview.wss?uid=swg27009778] (that is 7.0.0.39 or later fix level: current latest fix level is 7.0.0.41). 

3. b. For eWAS 6.1 provided by TDS 6.2 or TDS 6.1, install eWAS fix pack 6.1.0.47 [http://www.ibm.com/support/docview.wss?uid=swg21404665#v61]. 

 

4. Install required LogJam vulnerability interim fix for eWAS (choose one of the following based on the eWAS fix pack level installed in step 3.): 

4. a. For eWAS 7.0.0.41 refer "embedded WebSphere Application Server 7.0 Fix Pack 41 (7.0.0.41) [http://www.ibm.com/support/docview.wss?uid=swg21981056]" 

4. b. For eWAS 6.1.0.47 download the platform specific interim fix file from PI42779 [http://www-01.ibm.com/support/docview.wss?uid=swg24040182] and install using Update installer procedure - step 5 [http://www.ibm.com/support/docview.wss?uid=swg21404270].
