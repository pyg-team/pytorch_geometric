Title: IBM PJ39345: UNABLE TO USE ENTRY TEMPLATE VIA FOLDER ASSOCIATION AFTER THE FOLDER WAS EXPORT/IMPORT FROM DEPLOYMENT MANAGER TOOL. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using Workplace XT, to add a document to a folder, the following
   error occurs if the entry template is associated to the folder
   and the folder was exported and imported using Deployment
   Manager.
   
   "The <template name> entry template is not available or no
   longer applies to this object store."
   
   Steps to Reproduce:
   1. From Workplace XT, create a new document entry templates.
   2. From Workplace XT, create a folder.
   3. From Workplace XT, associate the new folder with the document
   entry template.
   4. From Deployment Manager, export and import the folder
   and entry template.
   5. From Workplace XT, selected the new folder at the destination
   object store.
       Drag/drop a local document to and note the error.
   
   
    
   
   

LOCAL FIX
 *  Remove the entry template from the folder and re-add it using
   Workplace XT.

 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All who use entry templates associated with folders          *
