Title: IBM PJ36484: UNABLE TO LOGIN/AUTHENTICATE WHEN USER ACCOUNT' DISTINGUISHED NAME (DN) CONTAINS A '=' IN NAME - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Unable to login/authenticate when a user account's
   distinguishedname (DN) contains an '=' character in name.
   For instance, DN:  CN=user1, OU=Grp\=Eng, DC=region1, DC=ibm
   DC=com,  or   CN=user1, OU=GrpEng, DC=region\=1, DC=ibm, DC=com.
   
      User with DN that does not have '=' in name is
   able to login.
     The following exception is thrown at every
   attempt by users withan '=' in the DN.
   
          ERROR method name: getSecurityIdList
   principal name:   <username> Global Transaction: false
   User Transaction: false    Exception  Info: LDAP Provider
   Access failed for principal   '<username>'. Root cause: Invalid
   DN: CN=...
   com.filenet.api.exception.EngineRuntimeException:
   SECURITY_LDAP_PROVIDER_FAILED: LDAP Provider   Access
   failed for  principal '<username>'. Root cause: Invalid  DN:
   CN=...
   Caused by:
   com.filenet.api.exception.EngineRuntimeException:
   SECURITY_INVALID_DN: Invalid DN: CN=...
   
   Steps to Reproduce:
   1 Create OU with '=' character in name
   2 Add user to the OU
   3 Login with this user
   
   Note that the use of '=' in Short Name attribute is still not
   allowed by design.
