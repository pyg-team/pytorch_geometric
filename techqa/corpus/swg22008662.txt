Title: IBM How to start IBM Information Server Metadata Server when it hangs? - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How to start IBM Information Server Metadata Server on IBM DB2 database when it hangs with following error?

DSRA8650W : Error closing a JDBC child wrapper, com.ibm.ws.rsadapter.jdbc.WSJccPreparedStatement@47a09537

com.ibm.db2.jcc.am.SqlException: [jcc][10120][10943][3.66.46] Invalid operation: statement is closed. ERRORCODE=-4470, SQLSTATE=null

PersistenceUt E Database communication error - the database connection is no longer valid.

The database may be down or inaccessible via the network.

com.ibm.db2.jcc.am.DisconnectNonTransientConnectionException:

Error for batch element #1: [jcc][t4][2030][11211][3.66.46] A communication error occurred during operations on the connection's underlying socket, socket input stream, or socket output stream. Error location: Reply.fill() - insufficient data (-1). Message: Insufficient data.

ERRORCODE=-4499, SQLSTATE=08001 at com.ibm.db2.jcc.am.dd.a(dd.java:321)

com.ibm.db2.jcc.am.SqlException: [jcc][103][10843][3.66.46]

Non-recoverable chain-breaking exception occurred during batch

processing. The batch is terminated non-atomically.

ERRORCODE=-4225, SQLSTATE=null 

CAUSE
This could happen if the database transaction log is not large enough. The Metadata repository issues database rollback the query during startup and IBM WebSphere application server profile startup fails. 


ANSWER
Increase the transaction log size for IBM Information Server Metadata repository database on IBM DB2.
