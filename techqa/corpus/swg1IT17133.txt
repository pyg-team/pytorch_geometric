APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When an http proxy is used in a SOAPRequest/HTTPRequest node,
   the SSL SNI information is seen as proxy host name instead of
   the webservice endpoint address.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All users of WebSphere Message Broker and IBM Integration Bus
   using HTTPRequest/SOAPRequest nodes making SSL connections
   through a proxy.
   
   
   Platforms affected:
   z/OS, MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   When an http proxy is used in a SOAPRequest/HTTPRequest node,
   when SSL is enabled, the SSL SNI  value is set to the proxy host
   name instead of the webservice endpoint address.
   
   There are a number of resource name changes between WebSphere
   Message Broker and IBM Integration Bus Version 9.0.  For details
   visit
   http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e
   tools.mft.doc/bb23814_.htm [http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.etools.mft.doc/bb23814_.htm]
   
   
    
   
   

PROBLEM CONCLUSION
 *  The product will use the end server's fully qualified domain
   name as the SNI value when doing an SSL handshake through a
   proxy.
