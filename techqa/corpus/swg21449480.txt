Title: IBM Cause for exception java.sql.SQLException: [BEA][DB2 JDBC Driver]SQLDIAGGRP was specified and not null - United States

Text:
http://publib.boulder.ibm.com/infocenter/db2luw/v9r5/topic/com.ibm.db2.l uw.wn.doc/doc/c0051316.html TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The following exception is observed when running the RTS database script for a DB2 datasource from the installation manager on BEA Weblogic application server:

java.sql.SQLException: [BEA][DB2 JDBC Driver]SQLDIAGGRP was specified and not null 

SYMPTOM
The following exception stack is shown in the server log file: 

java.sql.SQLException: BEA;DB2 JDBC Driver;SQLDIAGGRP was specified and not null.
java.lang.RuntimeException: java.sql.SQLException: BEA;DB2 JDBCDriver;SQLDIAGGRP was specified and not null.
at ilog.rules.teamserver.ejb.service.IlrTransactionContext.close(IlrTransactionContext.java:172) 
at ilog.rules.teamserver.ejb.service.dao.IlrElementDAOJDBC.loadMessageFiles (IlrElementDAOJDBC.java:248) 
at ilog.rules.teamserver.ejb.service.IlrSessionFacadeImpl.loadMessageFiles (IlrSessionFacadeImpl.java:5199) 
... 
Caused by: java.sql.SQLException: ;BEA;DB2 JDBC Driver;SQLDIAGGRP was specified and not null. 
at weblogic.jdbc.wrapper.ResultSet.close(ResultSet.java:180) 
at ilog.rules.teamserver.ejb.service.IlrTransactionContext.close(IlrTransactionContext.java:169) 


CAUSE
The driver used with the RTS datasource is the one provided by BEA Weblogic. This driver is not supported.


RESOLVING THE PROBLEM
Use an IBM DB2 Universal JDBC Provider listed in the JRules Software Requirements page [http://www-01.ibm.com/support/docview.wss?rs=4043&uid=swg27018144]. 

You can download the DB2 JDBC drivers at www.ibm.com/software/data/db2/ [http://www.ibm.com/software/data/db2/].
