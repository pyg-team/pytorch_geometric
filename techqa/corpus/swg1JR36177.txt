PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  Users operating in a mixed cell             *
   *                  i.e., WebSphere Application Server(WAS)     *
   *                  and WebSphere Process                       *
   *                  Server(WPS) environment.                    *
   ****************************************************************
   * PROBLEM DESCRIPTION: J2EE application update on WAS          *
   *                      profile is invoking SCA Update tasks    *
   *                      and resulting in ffdcs and exceptions.  *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Service Component Architecture(SCA) Update task needs to be
   executed only for SCA modules.  A check to validate the
   SCA module is missing in the code.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A Check has been added to validate the SCA module before
   executing the update task.
