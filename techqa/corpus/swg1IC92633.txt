ERROR DESCRIPTION
 *  SI 5.2.4 on Windows with an IBM JDK fails to start if verbose
   GC is enabled.
   went through the performance tuning wizard and after performing
   setupfiles, I'm not able to start SI anymore.  I see the below
   within the Noapp_exe.log.  I did choose to turn on verbose GC
   in the performance tuning wizard.
   
   <verbosegc version="20101027_AA">
   JVMJ9VM007E Command-line option unrecognised: -Xloggc:
   E=\GIS_ROOT\SI52_NODE2\properties\servers.properties
   </verbosegc>
   
   
    
   
   

LOCAL FIX
 *  MA/MA
   
   Workaround:
   
   The output can be redirected using the following flag:
   .
   -Xverbosegclog[:<file>]
