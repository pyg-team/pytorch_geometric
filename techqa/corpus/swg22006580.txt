Title: IBM Security Bulletin: Non-configured connections could cause denial of service in IBM WebSphere MQ Internet Pass-Thru (CVE-2017-1118 ) - United States

Text:
 SECURITY BULLETIN

SUMMARY
 When using the MQIPT SecurityManager, IP addresses that are not configured in the MQIPT security policy file with the accept and resolve socket permissions, cause MQIPT to stop responding to connection attempts. 

VULNERABILITY DETAILS
CVEID:CVE-2017-1118 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2017-1118]
DESCRIPTION:IBM WebSphere MQ Internet Pass-Thru could allow an attacker to cause the MQIPT to stop responding due to an incorrectly configured security policy.
CVSS Base Score: 3.7
CVSS Temporal Score: Seehttps://exchange.xforce.ibmcloud.com/vulnerabilities/121156 [https://exchange.xforce.ibmcloud.com/vulnerabilities/121156]for the current score
CVSS Environmental Score*: Undefined
CVSS Vector: (CVSS:3.0/AV:N/AC:H/PR:N/UI:N/S:U/C:N/I:N/A:L) 

AFFECTED PRODUCTS AND VERSIONS
IBM WebSphere MQ Internet Pass-Thru v2.0 on all platforms 

IBM WebSphere MQ Internet Pass-Thru v2.1 on all platforms

REMEDIATION/FIXES
Download and apply the latest version of IBM WebSphere MQ Internet Pass-Thru, 2.1.0.3 from MS81:WebSphere MQ Internet Pass-Thru [http://www-01.ibm.com/support/docview.wss?uid=swg24006386] page.

WORKAROUNDS AND MITIGATIONS
None known
