ERROR DESCRIPTION
 *  Web content management doesn't show links to anonymous user
   
   To reproduce the scenario, following steps can be taken.
   
   1. Create an authoring template containing a link element,
   following the
   documentation
   http://publib.boulder.ibm.com/infocenter/wcmdoc/v6r1/topic/com.i [http://publib.boulder.ibm.com/infocenter/wcmdoc/v6r1/topic/com.i]
   bm.lotus
   .wcm.doc_v615/wcm/wcm_dev_elements_link_adding.html ,
   2. Create content using that template:
   1. At the link element part, choose a link to an existing LWCM
   content
   (using the Browse content button).
   2. Check the Use title of linked item option.
   3. Create a menu component listing the content items created
   using the
   authoring template.
   1. Insert an element creating a link in the menu results, e.g.
   <Element context="autoFill" type="content"
   key="link-element-name"
   format="tag"/>
   4. Display the menu in a 286 rendering portlet.
   To a logged user, the link is rendered correctly. However, an
   anonymous
   user does not see the link. The output looks as e.g.
   <a
   href="?1dmy&current=true&urile=wcm%3apath%3a/testlibrary/testsit
   e/testar
   ea/testcontent" ></a>
   The title is not placed between the <a> HTML tags.
   The anonymous portal user has the same access rights as the
   logged user
   on both the displayed content and the linked content.
