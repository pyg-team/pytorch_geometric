APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  J2EE apps are reported as 'Stopped' in WAS Admin Console, after
   WAS Data Collector v7.2 IF0014 is installed and configured for
   'nodeagent' as a FULL DC (not DC-Less). NOTE: The same J2EE
   apps are properly monitored (including App status) in TEP
   workspaces.
   This issue has only been observed in following ENVIRONMENT:
   WebSphere Platform 9.0.0.2
   [ND 9.0.0.2 cf021645.01] [JAVA8 8.0.3.22
   pxa6480sr3fp22-20161213_02]
   
   
    
   
   

LOCAL FIX
 *  Until a fix is released, you can configure nodeagent in DC-Less
   mode (JMX monitoring).
   
   
    
   
   

PROBLEM SUMMARY
 *  MBean registration updated to correct the problem.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this APAR is available in the following maintenance
   releases:
   
     | interim fix | 7.2.0.0-ITM-ITCAM_DC-IF0015
