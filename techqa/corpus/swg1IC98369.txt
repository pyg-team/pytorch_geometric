APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  While deploying a nested subflow with promoted properties,
   this error occurs:
   
   BIP3701E: A Java exception was thrown whilst calling the Java
   JNI method ''AdminAgentThread_getDeployMessage''.
    The Java exception was ''org.xml.sax.SAXParseException: The
   element type "SubFlowNode" must be terminated by the matching
   end-tag "".''.
    The Java stack trace was ''Frame : 0
   org.xml.sax.SAXParseException: The element type "SubFlowNode"
   must be terminated by the matching end-tag "".
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All users of IBM Integration Bus V9.0 who are using nested
   subflows.
   
   
   Platforms affected:
   MultiPlatform, z/OS
   
   ****************************************************************
   PROBLEM SUMMARY:
   When an application or a library that contains a nested
   subflow with promoted properties is deployed, then the deploy
   fails with the following error:
   
   BIP3701E: A Java exception was thrown whilst calling the Java
   JNI method ''AdminAgentThread_getDeployMessage''.
    The Java exception was ''org.xml.sax.SAXParseException: The
   element type "SubFlowNode" must be terminated by the matching
   end-tag "".''.
    The Java stack trace was ''Frame : 0
   org.xml.sax.SAXParseException: The element type "SubFlowNode"
   must be terminated by the matching end-tag "". There are a
   number of resource name changes between WebSphere Message Broker
   and IBM Integration Bus Version 9.0. For details visit
   http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e
   tools.mft.doc/bb23814_.htm [http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.etools.mft.doc/bb23814_.htm]
   
   
    
   
   

PROBLEM CONCLUSION
 *  The product now deploys the flow successfully.
