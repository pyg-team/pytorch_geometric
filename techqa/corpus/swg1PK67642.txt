APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Syndication fails for published items which have drafts. An
   AuthorizationException or LockedParentException is being thrown
   when syndication attempts to update the published version of any
   of these items on the syndicator.
   
   Recreate steps:
   
   1. Create a publish content "ContentA".
   2. Create another content "ContentB" which contains a link point
   to
   'ContentA'.
   3. Create a draft copy of ContentA.
   4. Do a syndication. (So both server has a draft and a publish
   content).
   5. On syndication, edit the draft copy, republish it and create
   a draft
   copy again.
   6. Do a syndication. (Fails)
   
   The exception causes the fail in the log is as below:
   
   [6/11/08 19:59:01:104 BST] 00000051 PackageProces W
   IWKPD1029X: Could
   not store doc: financial-solutions-centres, IdRef:
   {e884d780491f7b7e9c9bfcd51d17e395,
   com.aptrix.pluto.content.Content}
   
   
    
   
   

LOCAL FIX
 *  NA
   
   
    
   
   

PROBLEM SUMMARY
 *  If user update and re-publish a draft copy of a publish contetn,
   then create a draft copy again immediatly, meanwhile, the
   content is still be referenced by other content, then
   syndication of this content will fail due to the
   ReferentiallntegirtyException.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Not using the proper method to update the publish content with a
   draft copy. Change to using the proper method in order to avoid
   the ReferentialIntigirtyException.
