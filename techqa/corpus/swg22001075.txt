Title: IBM AppScan tests for CVE-2017-5638, CVE-2017-9805, and CVE-2017-9791 - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 Do IBM Security AppScan Standard and IBM Security AppScan Enterprise have tests for Apache Struts 2 command execution vulnerabilities, CVE-2017-5638, CVE-2017-9805, and CVE-2017-9791? 

ANSWER
In AppScan Standard 

 1. The tests for CVE-2017-5638 (Apache Struts2 Remote Code Execution) were added to Security Rules Version 8901, available for download on March 20, 2017. The tests were improved in the following versions of Security rules. If you upgrade your AppScan Standard to 9.0.3.6 iFix002 [http://www.ibm.com/support/docview.wss?uid=swg27047066#92], you may see the following tests in the AppScan Test Policy: 
    
    [/support/docview.wss?uid=swg22001075&aid=1] [/support/docview.wss?uid=swg22001075&aid=1]
    
    The tests were included into the Default, Infrastructure-Only and Complete predefined policies. 
    
    
    
 2. The tests for CVE-2017-9805 (Apache Struts2 REST Plugin XStream Remote Code Execution) were added by 9.0.3.6 iFix002 [http://www.ibm.com/support/docview.wss?uid=swg27047066#92], available for download on September 19, 2017. After installing iFix002, you may see the following tests in the AppScan Test Policy: 
    
    [/support/docview.wss?uid=swg22001075&aid=2] [/support/docview.wss?uid=swg22001075&aid=2]
    
    The tests were included into the Default, Infrastructure-Only and Complete predefined policies. 
    
    
    
 3. The tests for CVE-2017-9791 (Apache Struts2 with Struts1 Plugin Remote Code Execution) were added by 9.0.3.6 iFix003 [http://www.ibm.com/support/docview.wss?uid=swg27047066#93], available for download on October 16, 2017. After installing iFix003, you may see the following tests in the AppScan Test Policy: 
    
    [/support/docview.wss?uid=swg22001075&aid=3] [/support/docview.wss?uid=swg22001075&aid=3]
    
    The tests were included into the Default, Infrastructure-Only and Complete predefined policies. 



In AppScan Enterprise, the tests are available in version 9.0.3.7 released on November 28, 2017. 


Note: The AppScan products are not affected by this vulnerability as noted in AppScan Source, AppScan Standard and AppScan Enterprise are not affected by CVE-2017-5638 [http://www.ibm.com/support/docview.wss?uid=swg22000355].
