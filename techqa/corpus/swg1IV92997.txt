APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Creating a Database Event Reader Service with only a restriction
   filter and not a timestamp or key fields results in random
   events being read from the Database event source due to an
   invalid serial or timestamp used to iterate through the
   Database.
   
   
    
   
   

LOCAL FIX
 *  None known
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * 1.  An RFE introduced a new method, constructQueryWithCount. *
   * This method in the objectserversqldatasource                 *
   * class does not contain the orderby clause.                   *
   * [RFE] DatabaseEventReader: Configure Number of Rows fetched  *
   * per query (46476)                                            *
   * Events were being skipped/excluded from processing by the    *
   * databaseventreader.                                          *
   *                                                              *
   * 2. In 7.1 FP4. a formatdate function call was introduced to  *
   * the timestampfield in the persistToFile method in the        *
   * genericeventreader class.                                    *
   * this fails if the timestampfield is not a date so the state  *
   * file was not being updated                                   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Add an orderby clause to the query that queries the          *
   * datasource.                                                  *
   * this ensures that in the event that the keyfield being used  *
   * is not sequential, that no events will be skipped.           *
   * i.e if serial is being used but some serial numbers are      *
   * missing.                                                     *
   *                                                              *
   * The update to genericeventreader checks the content of the   *
   * timestamp field and if the formatdate conversion             *
   * fails throws an exception.                                   *
   ****************************************************************
