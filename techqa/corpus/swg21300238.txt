JSSE trace Instructions TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The procedure to collect a JSSE trace to debug HTTPS problems with HTTPRequest node in WebSphere Message Broker (WMB) or IBM Integration Bus (IIB). 

CAUSE
HTTPS configuration

RESOLVING THE PROBLEM
 

UNIX 

 1. Export the following environment variable in the Broker service ID's profile: IBM_JAVA_OPTIONS=-Djavax.net.debug=true 
 2. Restart the broker. 
 3. Recreate the problem. 
 4. Check the stderr and stdout under: /var/mqsi /components/<Brokername>/<EGUUID>



Windows  1. Export the following environment variable in the System profile: IBM_JAVA_OPTIONS=-Djavax.net.debug=true 
 2. Restart the broker 
 3. Recreate the problem. 
 4. Send the console.txt located in C:\ProgramData\IBM\MQSI\components\<broker>\<EG UUID> 
    for the failing DataFlowEngine. 
 5. Unset/remove the environment variable from the system profile.
