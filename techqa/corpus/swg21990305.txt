Title: IBM LDAP authentication issue - United States

Text:
SIQLegal ; authentication; port TECHNOTE (FAQ)

QUESTION
 After configuring the StoredIQ for Legal environment to use the company LDAP and imported users. Why the imported users are not able authenticate ? 

CAUSE
Incorrect LDAP port could cause this issue.

ANSWER
Configure LDAP server port correctly in StoredIQ for Legal LDAP configuration. If LDAP accepts SSL then use port 636 , if not then use default port 389.
