Title: IBM Security Bulletin: IBM Sterling Connect:Direct for HP NonStop is affected by OpenSSL vulnerability CVE-2014-3508 - United States

Text:
 SECURITY BULLETIN

SUMMARY
 Connect:Direct for HP NonStop is vulnerable to one of the information disclosure vulnerabilities reported by the OpenSSL Project on 6 August 2014. 

VULNERABILITY DETAILS

CVE-ID: CVE-2014-3508 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2014-3508]

Description: OpenSSL could allow a remote attacker to obtain sensitive information, caused by an error in OBJ_obj2txt. If applications echo pretty printing output, an attacker could exploit this vulnerability to read information from the stack.

Connect:Direct for HP NonStop is only impacted by this vulnerability when a Session Manager trace is enabled. Trace output from the SSL handshake might include sensitive information.

CVSS Base Score: 4.3
CVSS Temporal Score: See https://exchange.xforce.ibmcloud.com/vulnerabilities/95165 [https://exchange.xforce.ibmcloud.com/vulnerabilities/95165] for more information
CVSS Environmental Score*: Undefined
CVSS Vector: (AV:N/AC:M/Au:N/C:P/I:N/A:N)

AFFECTED PRODUCTS AND VERSIONS
IBM Sterling Connect:Direct for HP NonStop 3.5.00
IBM Sterling Connect:Direct for HP NonStop 3.5.01
IBM Sterling Connect:Direct for HP NonStop 3.6.0

REMEDIATION/FIXES
The recommended solution is to apply the fix as soon as practical. Please see below for information about the available fixes.



VRMF APAR Remediation/First Fix 3.5.00 IT03733 Apply 3.5.00 maintenance dated July 31, 2014, available on IWM [https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?source=swg-SterlngLegacyreq] 3.5.01 IT03733 Apply 3.5.01 maintenance dated July 31, 2014, available on IWM [https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?source=swg-SterlngLegacyreq] 3.6.0 IT03733 Apply 3.6.0.1, iFix 007, available on Fix Central [http://www-933.ibm.com/support/fixcentral/swg/selectFixes?parent=ibm~Other%2Bsoftware&product=ibm/Other+software/Sterling+Connect%3ADirect+for+HP+NonStop&release=3.6.0.1&platform=All&function=all] WORKAROUNDS AND MITIGATIONS
To avoid this vulnerability do not enable Session Manager traces.
