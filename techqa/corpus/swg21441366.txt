Title: IBM Procedure to trace a failing deploy in IBM Integration Bus or WebSphere Message Broker - United States

Text:
WBI Message Broker WBIMB WMQI MQSI Deploy Fail trace v7 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Use this procedure to capture a trace of a failing deploy in IBM Integration Bus (IIB) V9, V10, or WebSphere Message Broker V8. 

RESOLVING THE PROBLEM
Start tracing of broker agent and execution groups 

1. Start Trace


mqsichangetrace <brokername> -t -b -l debug 
mqsichangetrace <brokername> -t -e <egroup> -l debug -r 

where <brokername> is the name of your broker while <egroup> is the name of your execution group


2. Deploy and monitor for errors.

3. Stop all tracing when deploy errors occur. 

mqsichangetrace <brokername> -t -b -l none
mqsichangetrace <brokername> -t -e <egroup> -l none


Format the traces using the commands below:

1. Retrieve the trace log for the specified component. 
mqsireadlog <brokername> -t -b agent -f -o agenttrc.xml

2. Retrieve and format the XML log 
mqsiformatlog -i agenttrc.xml -o agenttrc.txt

3. Retrieve the trace log for the specified component. 
mqsireadlog <brokername> -t -e <egroup> -f -o <egroup>.xml

4. Format the XML log file 
mqsiformatlog -i <egroup>.xml -o <egroup>.txt

5. After a PMR is open, submit the files agenttrc.txt and <egroup>.txt to IBM Support for problem determination.
