Title: IBM Tivoli Common Reporting installation fails on AIX - United States

Text:
TCR; Tivoli Common Reporting; reporting; reporter TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 IBM® Tivoli® Common Reporting installation program fails on some AIX platforms. The Deployment Engine (DE) component is not able to fully initialize which causes problems during the Tivoli Common Reporting server installation. A newer version of the DE component is required, and a manual clean up before restarting the installation program. 

SYMPTOM
When the installation program fails, review the installer log files. 

The log.zip archive is created during the installation in the destination directory specified by the user:


 * Default value for UNIX - /opt/IBM/tivoli/tip


The archive file contains the following files: 
ia/IA-TIPInstall-00.log and de/USER/de_trace.log. 

Note: In case the archive file is not created, locate the following files in your system: 
 * USER_HOME_DIR/IA-TIPInstall-00.log (information logged by the Tivoli Integrated Portal installer)
 * de_trace.log (information logged by DE):  * for a root type installation 
      /usr/ibm/common/acsi/logs/root/de_trace.log
    * for a non-root type installation 
      USER_HOME_DIR/.acsi_USER/logs/USER/de_trace.log, 
      for example /home/users/tipadmin/.acsi_tipadmin/logs.de_trace.log

RESOLVING THE PROBLEM
To fix the problem replace the Deployment Engine component in the IBM Tivoli Common Reporting installation media. A newer version of Deployment Engine (1.3.1.9) is available as an interim fix 3 for IBM Tivoli Common Reporting 1.2.0.1. Download the upgrade and follow the instructions to fix the IBM Tivoli Common Reporting installation program: 

 
http://www.ibm.com/support/docview.wss?uid=swg24023685 [http://www.ibm.com/support/docview.wss?uid=swg24023685]

Before you start the fixed installer, remove all the previously installed files. The procedure of manual uninstallation of TCR is described in a separate Technote - Manual uninstallation of IBM Tivoli Common Reporting 1.2:

http://www.ibm.com/support/docview.wss?uid=swg21392921 [http://www.ibm.com/support/docview.wss?uid=swg21392921]

After cleaning your environment, start the installation program again with the Deployment Engine upgraded.
