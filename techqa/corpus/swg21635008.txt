PROBLEM(ABSTRACT)
 After installing Optim UDF with ODPP and running the verification program, the job fails with a -443 error and the output shown below:

X'00000000' X'00000000' SQL DIAGNOSTIC INFORMATION 


SYMPTOM
Cannot validate the license for the ODPP environment defined under the ENVVARS member, which points to the license file under environment variable LIBPATH


CAUSE
The 2 license files do not exist in the License directory under the environment variable ODPPLL 

ODPPKEYF.OPT
ODPPLICF.OPT


RESOLVING THE PROBLEM
The License program needs to run in the OMVS environment by running the below command, where the ODPPLic Program resides - under the /path/odpp/lib directory 

 Where CPUID is Mainframe CPUID and Key is the TDM Key for the Optim Z/OS Product: 


ODPPLicp OPTZOS CPUID=####### 
KEY=XXXXXXXXX 
LICPATH=/path/odpp/odpp_license_files
