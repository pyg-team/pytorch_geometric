CAUSE
 

/tmp folder is nearly full. 

Sufficient space not allocated to default tmp directory on ICN server node. 


RESOLVING THE PROBLEM
Assign enough drive space for the default tmp directory on the ICN server node; e.g. 15GB- /opt/IBM/Filenet/ECMClient/configure/tmp/ 


Also ensure that /tmp filesystem, e.g. AIX, Linux, etc., has been allocated sufficient space. A minimum of 4.5GB or more free space is required. 
It can be checked by using the command: df -kg /tmp

If the above do not resolve the issue, check your operating system for any Monitoring Agents. They may be interacting with the directory that is being used and this may block some file operations the deployment tool is trying to execute. Disable the Monitoring Agent or add the FileNet directories to exceptions.
