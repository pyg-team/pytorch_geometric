Title: IBM Error 500 when trying to login to DASH - United States

Text:
DASHL2; DASHL2CONF; cannot login; error 500; Filter ; [TIPISCLITEAuthorizationFilter]; could not be initialized TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Users are getting Error 500 and cannot log into Dashboard Application Services Hub. 

SYMPTOM
The following error message is being displayed: 

"An error occurred while processing request: /ibm/console/login.do 

Message: Filter [TIPISCLITEAuthorizationFilter]: could not be initialized" 


CAUSE
The XML files such as the navigation.xml and portletEntities.xml are missing in the WEB-INF directory: 

/opt/IBM/JazzSM/profile/config/cells/JazzSMNode01Cell/applications/isc.ear/deployments/isc/isclite.war/WEB-INF 


DIAGNOSING THE PROBLEM
Check the SystemOut.log for errors like these: 

ServiceManager init() .init :Exception occurred while initializing [com.ibm.isclite.service.datastore.navigation.NavigationService] service serviceorg.eclipse.emf.ecore.resource.impl.ResourceSetImpl$1$DiagnosticWrappedException: java.io.FileNotFoundException: /opt/IBM/JazzSM/profile/config/cells/JazzSMNode01Cell/applications/isc.ear/deployments/isc/isclite.war/WEB-INF/navigation.xml (No such file or directory) 

and

[5/7/15 7:31:13:208 CDT] 00000040 UpdatePortlet W UpdatePortletStore initPrltEntitiesResource PortletEntities Registry does not exist. It will be created.
ResourceMonitor ResourceMonitor Resource file /opt/IBM/JazzSM/profile/config/cells/JazzSMNode01Cell/applications/isc.ear/deployments/isc/isclite.war/WEB-INF/portletEntities.xmlnot exist! 



RESOLVING THE PROBLEM
Restore the missing or corrupted XML files from backup. 

Please see the following technote:
