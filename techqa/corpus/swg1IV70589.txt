   The entry="" is unexpected as the Key Value Name is a required
   value.  This APAR is to report that the resulting error message
   should identify which signature is in error and what that error
   is.  Additionally, the incorrect signature should just be
   skipped (allowing other signatures to be imported).
   
   
    
   
   

LOCAL FIX
 *  Remove the invalid signature from the catalog or correct the
   signature so that it specifies a key value name:
   entry="Something"
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   Import of a custom software catalog that contains signatures
   with empty registry key values fails
   Behavior after you install the update:
   Signatures with empty registry key values are not supported.
   They are skipped and the import finishes successfully.
   ****************************************************************
   * RECOMMENDATION: Apply appropriate update when available or
   * refer to the documentation update
