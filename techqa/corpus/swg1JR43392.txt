PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  WebSphere Process Server V7 Users           *
   ****************************************************************
   * PROBLEM DESCRIPTION: SCA Module destination queue was        *
   *                      deleted once the application target     *
   *                      endpoint address is changed             *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   During the updating of the application
   (either through direct update of the application or modifying
   the import via the admin console) the related queue are
   removed by SCA Container.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Now for the full uninstall logic SCA container will clean
   up these component destinations and also re-create it.
