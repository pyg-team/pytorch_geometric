Title: IBM Web GUI のユーザー設定ができません - United States

Text:
Webgui users; Webgui authentication; Japanese; 日本語 TECHNOTE(トラブルシューティング)

問題(概要)
 Web GUI をインストール後、どのユーザーの設定もできません。 

症状
Web GUI のインストール後、設定過程でユーザー認証が働きません。


解決方法
Web GUI 8.1 ではインストール後にユーザーを作成するために configuration tool を実行する必要があります。 OMNIbus Web GUI configuration tool を開始すれば Web GUI の設定に役立つでしょう。 

開始する前に：
OMNIbus Web GUI configuration tool を実行中に問題が発生した場合には更なる情報について ncwConfigUI_<タイムスタンプ>.log ファイルから参照できます。このログファイルのある場所は OMNIbus Web GUI のホームディレクトリの下の/configtools/logs です。
例：/opt/IBM/netcool/omnibus_webgui/configtool/logs 

手順：
1. configuration tool のディレクトリへ移動します。 :WEBGUI_HOME/configtool/<$プラットフォーム名>
2. ncwConfigUI コマンドを実行します。.

UNIX/Linux の例：
1. WEBGUI/HOME/configtool/linux.gtk.x86_64 へ移動します。
2. 以下のコマンドを実行します。
./ncwConfigUI -WASUserID <WAS管理ID> -WASPassword <WAS管理パスワード>

Windowsの例：
1. C:\IBM\netcool\omnibus_webgui\configtool\win32.win32.x86_64 へ移動します。
2. 以下のコマンドを実行します。
ncwConfigUI.exe -WASUserID <WAS管理ID> -WASPassword <WAS管理パスワード>

次に行なうこと： 

 * デフォルトユーザーとグループの作成のオプションを指定した場合、ログインにはこれらのユーザー認証を使用します。デフォルトユーザーの初期パスワードは 例えば ncoadmin と ncouser のパスワードはWebSphere Server の管理パスワードと同一になります。 
 * ユーザーとグループの作成オプションを指定しない場合、jazz for Service Management のユーザーとしてログインします。このユーザーの認証はインストール中に指定済みです。


注： OMNIbus WebGUI Configuration Tool に誤まったユーザー名、あるいはパスワードを入力してしまった場合には、デフォルトユーザー作成時、あるいは WebSphere Application Server 再起動時にエラーメッセージが表示されます。 
これを正すために OMNIbus Web GUI Configuration Tool に戻って、正しい認証を入力できます。 関連情報
