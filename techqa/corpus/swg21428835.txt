Title: IBM Error: “The Reference to Entity "Language" Must End With the ';' Delimiter” - United States

Text:
delimiter; ampersand; amp; org.xml.sax.SAXParseException TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Executing UpdateDB results in the error "The reference to entity "language" must end with the ';' delimiter”, specifically while processing script file V7110_props.dbc. 

SYMPTOM
Executing UpdateDB results in the error "The reference to entity "language" must end with the ';' delimiter”, specifically while processing script file V7110_props.dbc.


CAUSE
This is due to special characters in the XML (DBC) file. The '&' (ampersand) is a special character in XML, which needs to be escaped to work correctly. The XML Parser is trying to dereference "language" since it is followed by the ampersand causing the syntax error.

DIAGNOSING THE PROBLEM
The following full error will be displayed in the UpdateDB log file located in \ibm\SMP\maximo\tools\maximo\logs\

at org.apache.xerces.impl.XMLScanner.reportFatalError(Unknown Source)
at org.apache.xerces.impl.XMLScanner.scanAttributeValue(Unknown Source)


RESOLVING THE PROBLEM
To resolve the issue, wherever an ampersand appears in the V7110_props.dbc file add "amp;" (without quotes) then save the file and run UpdateDB again.
