Title: IBM PK49742: CONTENT LINKS FAIL TO SYNDICATE TO THE SUBSCRIBER - United States

Text:
 
APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Content links are not syndicating to the subscriber.
   
   Example scenario in which the issue was encountered:
   1. Two Site Areas A and B
   2. Site Area A content is created
   3. Site Area B content is linked to original Site Area A
   content.
   4. Syndication process does not syndicate the content link.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Problem Summary: Content links fail to syndicate to the
   subscriber
   
   Detailed Problem Description:
   
   Recreation scenario:
   Create library and set up syndicator/subscriber pair for live
   item syndication
   Create Site1, SiteArea1, SiteArea2, 2-Stage Workflow and
   Authoring Template that saves the item to both site areas
   Create content, save, move to next stage
   Syndicate
   
   When the syndication is complete, the content the content item
   will be syndicated, but the content link for the other site area
   will not be syndicated.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Problem Analysis:
   
   An event log entry is not being made for the content link when
   it is published.
   
   Failing Module: WCM - Web Content Management (General)
   
   Problem Solution:
   
   Change the event log processing to add an entry for content
   links when they are published.
   
   Affected Users: All Users
