Title: IBM DE upgrade fails with com.ibm.ac.si.install.impl.SINotCompatibleException - United States

Text:
install; failure; octet; decimal; version; ACUApplication; properties TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempt to upgrade IBM Deployment Engine (DE) fails with error com.ibm.ac.si.install.impl.SINotCompatibleException 

CAUSE
Incorrect version numbers in ACUApplication.properties 

ENVIRONMENT
IBM Tivoli product with IBM Deployment Engine (DE)

DIAGNOSING THE PROBLEM
For example, DE upgrade fails with below error message in DE_Install.log file :





SIInstaller install com.ibm.ac.si.install.impl.SINotCompatibleException: ACUINI0011I Deployment Engine 1.4.0.22 can not be upgraded to 1.4.0.16.
at com.ibm.ac.si.install.impl.SIInstaller.compatibilityInternal(SIInstaller.java:3251)
at com.ibm.ac.si.install.impl.SIInstaller.installInternal(SIInstaller.java:4030)

ERROR: The request to install Deployment Engine 1.4.0.16 cannot be performed. Deployment Engine 1.4.0.22 is already installed on the system. Deployment Engine 1.4.0.16 cannot upgrade Deployment Engine 1.4.0.22, but Deployment Engine 1.4.0.16 contains important changes that are not included in Deployment Engine 1.4.0.22. Please update the system with a version of Deployment Engine that contains the changes included in Deployment Engine 1.4.0.16. 

In ACUApplication.properties file, notice that list of compatible versions is corrupted.

In ACUApplication.properties file, Update the acu.compatibleVersions property DE 1.4 version numbers from 1.4.xx to 1.4.x.x - for example, update 1.4.016 to 1.4.0.16 (make it 4 octets)
