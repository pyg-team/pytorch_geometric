APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When running a DVS test suite for a ruleset using a managed XOM,
   after an update to the XOM the changes are not found and the
   ruleset parsing fails.
   
   
    
   
   

LOCAL FIX
 *  Execute a different test suite not using the same XOM to reset
   the  XOM classloader loaded in the SSP
   
   
    
   
   

PROBLEM SUMMARY
 *  the ssp keep the latest loaded classloader in memory. Each
   subsequent execution does not use the managed XOM it should
   
   
    
   
   

PROBLEM CONCLUSION
 *  the code is modified so that a new classloader is created and
   use for each  dvs test execution.
