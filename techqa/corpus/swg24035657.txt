ABSTRACT
 IBM Rational ClearCase 8.0.1.1 has been made generally available and contains fixes to version 8.0.1 including all predecessor fix packs. 

DOWNLOAD DESCRIPTION

Please read ClearCase Release Notes [http://www.ibm.com/support/docview.wss?&uid=swg27039506] for information about system requirements, new features, and other important release information.

Important Upgrade Information:
Please refer to 1572830 [http://www.ibm.com/support/docview.wss?&uid=swg21572830] prior to upgrading servers deploying web components. Specifically refer to the section labeled "Avoiding Problems".

Before upgrading to this release from ClearCase MultiSite 8.0.0.x (including 8.0), on servers that have feature level 6 VOB families, please refer to technote 1609810 [http://www.ibm.com/support/docview.wss?&uid=swg21609810] for important information.

If you plan to install this ClearCase release or to upgrade to this release on Solaris 11.1, please read technote 1649246 [http://www.ibm.com/support/docview.wss?&uid=swg21649246] prior to that to understand why those operations may fail.

If you wish to apply the latest updates to your system, check the Fixes by Version for Rational ClearCase [http://www.ibm.com/support/docview.wss?&uid=swg27023148] technote for more details.



TestFixes included and excluded in this Fixpack Technote 1566655 [http://www.ibm.com/support/docview.wss?&uid=swg21566655] lists TestFixes that are included in this release, as well as those that have not been incorporated into this release.

INSTALLATION INSTRUCTIONS
Descriptions of resolved defects are included in the ux_readme and windows_readme files in the Download package table.

Refer to technote 1380386 [http://www.ibm.com/support/docview.wss?&uid=swg21380386] for detailed installation instructions. 

If you to plan to use WebSphere Application Server v8.0 with IBM Rational ClearCase, IBM Rational ClearQuest or IBM Rational RequisitePro, please read technote 1569987 [http://www.ibm.com/support/docview.wss?&uid=swg21569987] before installing the products. 

If you plan to use Installation Manager on Solaris, Linux, or AIX in GUI mode, please refer to technote 1330190 [http://www.ibm.com/support/docview.wss?&uid=swg21330190] for more information.

You must install this update using the Updates Packages wizard in IBM Installation Manager. For information on installing updates, see the topic 'Installing Updates' in the IBM Installation Manager Information Center at 
http://publib.boulder.ibm.com/infocenter/install/v1r6/index.jsp?topic=/com.ibm.cic.agent.ui.doc/topics/t_install_updates.html [http://publib.boulder.ibm.com/infocenter/install/v1r6/index.jsp?topic=/com.ibm.cic.agent.ui.doc/topics/t_install_updates.html];.



 * By default, the update is installed from the IBM update repository located on a remote server.
   
 * 
 * It is also possible to download the update and install it locally.
 * 
 * 
 *  Download the repository from the Download Package table below. 
   
   Extract files to a temporary directory. In the IBM Installation Manager, click File > Preferences and add the temporary directory location to the list of repositories. 
   
   Use the Update Packages wizard in the Installation Manager to complete the update. 



If you are using ClearCase on Oracle Solaris 11, the appropriate GUI packages must be installed for the ClearCase GUIs to operate properly. Please see technote 1597074 [http://www.ibm.com/support/docview.wss?&uid=swg21597074] for more information. 

For the Rational ClearTeam Explorer and ClearTeam Explorer Extension offerings, use the ccrc.zip and ccrc_extension.zip download packages below. For installation instructions, see ClearCase Information Center [https://publib.boulder.ibm.com/infocenter/cchelp/v8r0m0/index.jsp?topic=/com.ibm.rational.clearcase.cc_ms_install.doc/topics/c_inst_ccrc.htm]. 


If you have previously modified the ClearCase default extended naming symbol (@@) on any Windows 64-bit system, please review technote 1618052 [http://www.ibm.com/support/docview.wss?&uid=swg21618052]. After upgrading to this release, you will need to reconfigure the symbol's value.
