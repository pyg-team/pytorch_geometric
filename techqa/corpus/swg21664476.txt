Title: IBM Memory leak in an integration server that contains an MQInput node - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 An integration server (execution group) with a deployed message flow containing an MQInput node leaks memory when the target input queue does not exist. 

CAUSE
APAR IC92060 [http://www-01.ibm.com/support/docview.wss?uid=swg1IC92060]

ENVIRONMENT
Any WMB/IIB using WebSphere MQ 7.5

DIAGNOSING THE PROBLEM
In your system log, you will find that the 'MQOPEN' request failed with MQRC 2085 (MQRC_UNKNOWN_OBJECT_NAME).

RESOLVING THE PROBLEM
The fix for this issue is included in WebSphere MQ 7.5.0.3.
There are also several local fixes detailed here [http://www-01.ibm.com/support/docview.wss?uid=swg1IC92060].
