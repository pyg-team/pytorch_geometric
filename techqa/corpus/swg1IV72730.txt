APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  PROBLEM:
   After applying a condition to the DATETIME field on the table
   row of Specifications tab in Asset/Work Order application,
   validation on the date is no longer working.
   
   STEPS TO REPRODUCE:
   - Goto Assets > 123 Assets (clone of Assets)
   - Click on Specifications tab (or open Asset 11400,
   Specifications tab)
   - Click on Date/Time field of either any 3 Specification
   attributes that is of Data Type SRPDATE (there is a condition
   set)
   - Highlight currnet date/time, enter new date (ie. 4/20/15),
   press tab to exit field
   - Field validation does not occur.
   
   ie. If you type 4/20/15 and press tab to exit the field when
   the condition is not applied, the remainder of the date is
   validated (ie. 4/20/15 3:00pm)
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Maximo Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * After applying a condition to the DATETIME field on the      *
   * table row of Specifications tab in Asset/Work Order          *
   * application, validation on the date is no longer working.    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Upgrade to latest release.                                   *
   ****************************************************************
   STEPS TO REPRODUCE:
   1) Goto Database configuration and select ASSETSPEC object
   2) Goto Attributes tab and add new row
      Attribute: SRP_DATETIME
      Type: DATETIME
   3) Configure database changes
   4) Goto Domains application select DATATYPE domain
   5) Add new row
      Internal Value: ALN
      Value: SRPDATE
      Description: DateTime Values
   6) Goto Conditional Expression Manager application and add new
   row
      Condition: SRPSPECDATE
      Description: SRP Specification Data Type is DATE TIME
      Type: EXPRESSION
      Expression: :ASSETATTRIBUTE.datatype = 'SRPDATE'
      Always Evaluate: Yes
   7) Save Condition.
   8) Goto Application Designer and select ASSET application
   9) Select Duplicate Application Definition from action menu
      Application: SRPASSET
      Description: SRP Assets
   10) Select Add/Modify Signature Options from action menu
   11) Add new row
       Option: SRPSPECDATE
       Description: SRP Spec Date
   12) Select Specifications Tab
   13) Add a table column between "Table Value" and "Start Measure"
   and edit properties of new column
       Attribute: SRP_DATETIME
   	Lookup: DATELOOKUP
   14) Switch to Advanced tab of table column properties dialog
       Signature Option: SRPSPECDATE
   15) Press Configure Conditional Properties button
       Security Group Name: EVERYONE
   	Condition Number: SRPSPECDATE
   	Condition is false property: inputmode
   	Condition is false value: readonly
   16) Save application
   17) Goto Security Groups Application and select EVERYONE group
   18) Goto Application tab filter for SRP Assets application
   19) Grant Listed options for This Application
   20) Save Security Group
   21) Sign Out and back into Maximo
   22) Goto Assets > SRP Assets application
   23) Click on Specifications tab (or open Asset 11400,
   Specifications tab)
   24) On Classification Detail Menu select Go To Classifications
   25) Create New Classification
       Classification: SRPCLASS
   	Description: SRP Classification
   26) Add New Use With
       Use With Object: ASSET
   27) Add 3 new Attributes
       Attribute: TEST01
   	Data Type: ALN
       Attribute: TEST02
   	Data Type: SRPDATE
       Attribute: TEST03
   	Data Type: NUMERIC
   28) Save and Return with Value
   29) Click on Date/Time field of TEST02 Specification attributes
   that is of Data Type SRPDATE (there is a condition set)
   30) Enter new date (ie. 6/6), press tab to exit field
   - Field validation does not occur and fill in the year and time.
   ie. If you type 6/6 and press tab to exit the field when the
   condition is not applied, the remainder of the date is validated
   (ie. it will convert to 6/6/15 3:00pm)
   
   
    
   
   

PROBLEM CONCLUSION
 *  Field validation works on table cells that have conditional
   inputmode of readonly.
   
   The fix for this APAR is contained in the following maintenance
   package:
   	 | release\fix pack | Fix Pack Release 7.5.0.9 TPAE
