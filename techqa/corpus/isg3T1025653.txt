QUESTION
 How to configure GUI session timeout configuration parameters in Symphony 7.2.0? 

CAUSE
N/A

ANSWER
There are 2 officially supported GUI session timeout parameters : 

1. SessionExpireTime 

This parameter refers how long your session may survive in inactive mode from Symphony WEBGUI. Upon timeout, it will re-direct to log-on page. This parameter can be configured from $EGO_CONFDIR/../../gui/conf/pmcconf/pmc_conf_ego.xml. The default value is 60 minutes. 

2. InvalidationTimeout 

This parameter refers to how long your session stays as valid session in inactive mode. If timeout occurs, your GUI session becomes invalid and your session information will be flushed from browser cache/cookie. You can configure this parameter value from $EGO_CONFDIR/../../gui/conf/server_gui.xml. The default value is 30 minutes. 

 

Ensure that the InvalidationTimeout value is equal to or greater than SessionExpireTime value configured so that the GUI session does not get invalidated while it is active. For example, set as below :

<Parameter> 

<Name>SessionExpireTime</Name> 

<Value>60</Value> 

</Parameter> 


<httpSession invalidationTimeout="120m"/>
