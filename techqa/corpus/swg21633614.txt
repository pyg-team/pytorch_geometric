RESOLVING THE PROBLEM
This problem occurs because of a corrupted SIB xml configuration file. This could have happened because something went wrong during backup or restore of the file. In this case, the sib-engines.xml file has messaging engines, but there are no bus members in sib-bus.xml. 

To resolve this issue delete the sib-engines.xml file and add the server or cluster to the bus and redo the configuration and start the messaging engine.
