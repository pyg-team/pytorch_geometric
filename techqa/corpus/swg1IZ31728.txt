   Abstract:   ITM for DB 6.2 Oracle agent goes down by Ctrl+C on
               Solaris 9
   
   Environment:
     Oracle Agent: Solaris 9, ITM for DB 6.2
   
   Problem Description:
     Oracle agent goes down when customer input Ctrl+C in the
   session in which he started Oracle agent.  Please see "Detailed
   Recreation Procedure" below.
   
   Detailed Recreation Procedure:
     This problem can be always recreate in house Solris 9 machine.
     1) Start telnet to Solaris 9 system, and login by root user.
        Make sure you use /sbin/sh (B shell)
     2) Start Oracle agent
        # $CANDLEHOME/bin/itmcmd agent start or
     3) Verify Oracle agent are running
        # $CANDLEHOME/bin/cinfo -r
     4) Input Ctrl+C
        # ^C
     5) Verify if Oraclea agent is running
        # $CANDLEHOME/bin/cinfo -r
   
     You will find the agent process is down.
