ERROR DESCRIPTION
 *  If you are using WebSphere Service Registry and Repository with
   an Oracle database, and have loaded a large amount of user data,
   when you select 'Service Operations' in the UI, the following
   exception is seen:
   
   A problem occurred executing the query "GSR0089E: An error
   occurred while dealing with the following query:
   "getAllUDPsForBsrURIList"" : {1}
   GSR0089E: An error occurred while dealing with the following
   query: "getAllUDPsForBsrURIList"
   GSR0597E: An error occurred when accessing the metadata store.
   org.springframework.jdbc.BadSqlGrammarException:
   StatementCallback; bad SQL grammar [SELECT DISTINCT
   A0.OBJ_TYP_CD, A0.SUBJ_ID, A0.OBJ_TYP_CD, A0.PRED_ID,
   A0.OBJ_TYP_CD, A0.OBJ_ID FROM w_statement A0 WHERE A0.SUBJ_ID IN
   (lots of numbers ....)
   ...
   maximum number of expressions in a list is 1000
   java.sql.SQLException: ORA-01795: maximum number of expressions
   in a list is 1000
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Oracle recommend that long SQL queries are split, this
   recommendation has been applied.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This fix is expected in 6.3.0.3 and 7.0.0.2
