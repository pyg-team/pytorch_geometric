Title: IBM PK78286; 7.0.0.1: null pointer from VMM code during server startup with security - United States

Text:
PK78286; PK78286; PK78286 DOWNLOADABLE FILES

ABSTRACT
 Null pointer exception during initialization of Virtual Member Manager's (VMM) dynacache 

DOWNLOAD DESCRIPTION
PK78286 resolves the following problem:

ERROR DESCRIPTION: 
NullPointer during the server startup after federated repository is configured with LDAP on V70. 

1. Configure virtual member manager (VMM) in Application Server administrative console. 
- enable admin security 
- enable application security 
- set 'Federated Repository' as current, and configure it pointing to a LDAP server 

2. Restart server1 

results: server1 fails to start with error:

PROBLEM SUMMARY:

USERS AFFECTED:
All users of WebSphere Application Server V6.1 and V7.0

PROBLEM DESCRIPTION:
Null pointer exception during initialization of virtual member manager's (VMM) dynacache.

RECOMMENDATION:
None

During initialization of virtual member manager (VMM) LDAP adapter, a null pointer exception occurs when dynacache is being initialized.

It occurs because dynacache service has not been started.
