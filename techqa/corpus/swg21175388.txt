Title: IBM AMQ9202 Channel event messages not generated - United States

Text:
channel events; amqmdain; AMQ9202 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 WebSphere MQ does not write channel event messages.

The channel goes into retry mode after being started, and the MQRC_CHANNEL_STOPPED event with MQRQ_CHANNEL_STOPPED_RETRY event is not generated. 

CAUSE
Working as designed.

RESOLVING THE PROBLEM
You need to do two things to receive channel event messages: 

 * Alter the queue manager to enable channel event messages. 
 * Alter the UNIX qm.ini or Windows Registry to enable channel stop events. 


Alter the queue manager to enable channel event messages. 

 * Enable the queue manager channel events using the alter qmgr command on UNIX or Windows.
   
   or


 * You can also alter the queue manager properties on Windows using the WebSphere MQ Explorer.



Example: alter qmgr chlev (enabled)
