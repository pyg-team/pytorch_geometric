Title: IBM Enabling SNMP, email, and user defined responses for OpenSignatures on the Network IPS - United States

Text:
SNMP; OpenSignatures; responses; email; SMTP; User Defined; User Specified; trons TECHNOTE (FAQ)

QUESTION
 How do you enable SNMP, email, and user-defined responses for OpenSignatures on the Network IPS (GX)? 

CAUSE
Response Filters and Central Responses do not allow you to directly configure responses for OpenSignatures.

ANSWER
To configure your GX to generate an SNMP, email, or user-defined response for all OpenSignatures, you need to add an advanced tuning parameter. To do this, open the Tuning Parameters policy and add the advanced parameter that is referenced below. Be sure to choose only the appropriate response values you want to use (multiple responses can be added by separating them with a comma as seen in the example below): 


Name: np.opensignature.user.response 

 * Value: SNMP:<*mySNMP> 
 * Value: EMAIL:<*myEmail> 
 * Value: User specified:<*myUserDefined>
   Example: np.opensignature.user.response=DISPLAY:WithoutRaw,EMAIL:<*myEmail>,SNMP:<*mySNMP> 

Note: The name inside the <> brackets correlates to the name of the object within the Response Objects policy. This policy can be found directly under the Repository and within the Shared Objects. The response does not work if this is not configured. 

If you only want to enable the response on an individual OpenSignature (or specific ones), you need to add the following two parameters in Tuning Parameters. 

Note: These parameters are the same across all firmware versions. 

Name: np.vs.0.issue.< 60000000+sid > 
Type: Boolean 
Value: on 

Name: np.vs.0.issue.< 6000000+sid >.user.response 
Type: String  * Value: SNMP:<*mySNMP> 
 * Value: EMAIL:<*myEmail> 
 * Value: User specified:<*myUserDefined>


For more information on OpenSignatures, see Configuring OpenSignatures [http://www.ibm.com/support/knowledgecenter/SSB2MG_4.6.2/com.ibm.ips.doc/tasks/configuring_open_signature_event_options.htm] on the IBM Knowledge Center.
