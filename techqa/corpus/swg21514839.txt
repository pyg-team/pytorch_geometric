Title: IBM Raising alerts when TO/TU agent goes offline - United States

Text:

QUESTION
 How can you raise alerts when the Transaction Collector (TU) or Transaction Reporter (TO) agent goes offline? 

ANSWER

The Agent Management Services (AMS) feature in IBM Tivoli Monitoring (ITM) allows you to detect when an agent goes offline. The AMS is part of the OS agent. 

The TU/TO agents ship with ktu_default.xml and kto_default.xml files in the following directory: 

Windows

ITM_HOME\TMAITM6\CAP

Unix/Linux

ITM_HOME/config/CAP 

Edit these files to change the line: 

<managerType>NotManaged</managerType>

to:

<managerType>ProxyAgentServices</managerType> 

Restart the agent. When the agent goes offline, you see an alert in the OS agent's Agent Management Services workspace.
