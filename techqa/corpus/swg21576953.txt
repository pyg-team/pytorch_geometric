Title: IBM Error importing credential vault slot - United States

Text:
credential; vault; slot; xmlaccess; EJPSK0044E; EJPXA0090W TECHNOTE (TROUBLESHOOTING)

PROBLEM
Attempts to import credential vault slot result in EJPSK0501E when using XMLAccess.

SYMPTOM

CAUSE
When importing credential vault slots, stale data may be encountered. Users who previously created a credential vault slot in the WebSphere Portal customization database no longer exist in the user repository (such as an LDAP server). However, the data still exists in the Portal database.


RESOLVING THE PROBLEM
Determine whether the user referenced by the error message should be found in the repository. If so, troubleshoot Virtual Member Manager (VMM) configuration (based on PUMA MustGather). 

If this user is not in the target server's user repository, remove the credential vault slot (identifiable by E_NO1UF4I1196E1036H4ALVI0OC5, in the example trace messages above) from the input XML and re-import.
