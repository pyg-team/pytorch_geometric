Title: IBM AMC Editor fails to launch from TEP - United States

Text:
71AMCON-TS; 71AMCON-GEN TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The Application Management Configuration (AMC) Editor fails to start due to conflict with ITCAM for AD .jar file. 

SYMPTOM
javaws*.trace file has the exception: 
java.lang.SecurityException: Signers of 'com.ibm.tivoli.logging.jflt.SystemPropertyValidator' do not match signers of other classes in package 


CAUSE
The cause of the problem is a conflict between "tk_jflt.jar" file from ITCAM for WAS and "rtcom.jar" 

from the AMC agent. Both .jar files include the 'com.ibm.tivoli.logging.jflt' classes, but are different implementations. This issue can happen after the T3 AMC agent is upgraded from 7.3 to 7.4.0.1.


DIAGNOSING THE PROBLEM
Enable logging and tracing.

From the command prompt on the system where you are running the java webstart client,
issue the command 'javaws -viewer'.

In the Advanced tab, check the 'Enable tracing' and 'Enable logging' boxes, then click OK:


[/support/docview.wss?uid=swg21685236&aid=1] [/support/docview.wss?uid=swg21685236&aid=1]

The log we are looking for is in one of these two directories depending on whether IBM or Sun Java is installed: 
C:\Users\<user>\AppData\LocalLow\IBM\Java\Deployment\log 
or 
C:\Users\<user>\AppData\LocalLow\Sun\Java\Deployment\log 

for example: 
C:\Users\IBM_ADMIN\AppData\LocalLow\IBM\Java\Deployment\log 

The file to look for is named 'javaws*.trace'. 
Attempt to start the AMC Editor, then check the latest javaws trace log for an error similar to this:

RESOLVING THE PROBLEM
The fix is to install IFIX 5 application support from the package 7.2.0.0-TIV-ITCAMAD_WS_ITM-IF0005. 


The application support for product (YN) must be installed on all TEMS, RTEMS and TEPS and TEP client machines. 


Select the 'ITCAM for Websphere Applications 7.20.00.00' support option to install support for the 'YN' agent..



On Unix/Linux, ITCAM for AD V07.20.00.05 application support needs to be installed for Portal
browser client component (tpw). The "cinfo -t" results should be similar to the following : PC APPLICATION SUPPORT DESC PLAT APP VER BUILD INSTALL DATE
yn ITCAM Agent for WebSphere Applications tms 07.20.00.05 201410061712 20150123 1133
yn ITCAM Agent for WebSphere Applications tps 07.20.00.05 201410061712 20150115 1420
-->yn ITCAM Agent for WebSphere Applications tpw 07.20.00.05 201410061712 20150123 1200

Clear the Java cache after the update. Configure the TEPS after updating the application support:  Unix/Linux - 

 /opt/IBM/ITM/bin/itmcmd config -A cq 

 Windows - 

 Reconfigure from the MTEMS: 

[/support/docview.wss?uid=swg21685236&amp;aid=2]

A work around is to use the TEP Desktop Client.
