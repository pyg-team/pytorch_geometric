ERROR DESCRIPTION
 *  Email retrieval is only checking the default P8 object store
   instead of all referenced object stores, resulting in
   MAPI_E_CALL_FAILED error trying to preview or retrieve
   documents from the other object stores
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ICC for Emails gives MAPI_E_CALL_FAILED error while searching
   the emails from multiple object stores.
   
   Email retrieval is currently only checking the default P8 object
   store instead of all referenced object stores. This results in
   MAPI_E_CALL_FAILED error trying to preview or retrieve documents
   from the other object stores.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Starting with IBM Content Collector 4.0.1 FP5 release, this
   issue has been fixed. User can now search mails from multiple
   object stores through Outlook Store Provider.
