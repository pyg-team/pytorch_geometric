APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The ReturnEvent policy function updates all variables which have
   an @ symbol in front of them, even if thoses variables are not
   part of the database which is defined within the Database
   Event Reader Service. The result is the policy executes
   producing a sql exception and the update not going through.
   
   
    
   
   

LOCAL FIX
 *  For now do not use the @ symbol in front of the variables that
   are not part of the database. If the @ was used in front of the
   variable so as to be able to access this variable globally, then
   the user need to be using the SetGlobalVar(), SetServerVar(),
   GetGlobalVar(), and GetServerVar() functions instead.
   
   
    
   
   

PROBLEM SUMMARY
 *  SQL execption when ReturnEvent with an @ symbol in front
    of variable which are not part of database
   
   
    
   
   

PROBLEM CONCLUSION
 *  The ReturnEvent policy function updates all variables which
   an @ symbol in front of them, even if thoses variables are not
   part of the database which is defined within the Database
   Event Reader Service. The result is the policy executes
   producing a sql exception and the update not going through.
   
   The fix for this APAR is contained in the following
   maintenance packages:
   | Interim Fix | 5.1.1.1-TIV-NCI-IF0003
