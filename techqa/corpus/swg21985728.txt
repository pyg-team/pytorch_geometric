Title: IBM APPX0108: The certificate specified has expired. - United States

Text:
TPAEANYWHERE; APPX0108; certificates TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When building Maximo Anywhere Windows applications you receive the error 'APPX0108: The certificate specified has expired.' 

SYMPTOM
[exec] C:\Program Files (x86)\MSBuild\Microsoft\VisualStudio\v14.0\AppxPa
ckage\Microsoft.AppXPackage.Targets(2519,5): error APPX0108: The certificate specified has expired. For more information about renewing certificates, see http:/
/go.microsoft.com/fwlink/?LinkID=241478. [C:\IBM\Anywhere\MaximoAnywhere\apps\As
setAudit\windows8\native\AssetAudit.jsproj]
[exec]
[exec] 4 Warning(s)
[exec] 1 Error(s)
[exec]
[exec] Time Elapsed 00:00:18.46


CAUSE
This is caused when the certificates used to sign the Maximo Anywhere applications have expired. 

RESOLVING THE PROBLEM
To resolve this issue you will need to create or update your certificates to ones at are not expiring. You can do this in Visual Studio with the following steps from the Maximo Anywhere Read me.


1. Open Visual Studio and browse to the C:\IBM\Anywhere\MaximoAnywhere\apps\{APPNAME}\windows8\native folder. 

2. Open the .jsproj file (for example AssetAudit.jsproj) 

3. From the Solution Explorer click on the package.appmanifest file 

3. Click on the Packaging tab and then click on choose certificate 

4. Click on Configure Certificate and select your new certificate or create a test certificate. 

 

The certificate will save to the windows8/native directory. 

 

5. Rebuild and Install your applications 

 

* The certificate should be installed to Trusted People on the device as well before installing the newly built applications.
