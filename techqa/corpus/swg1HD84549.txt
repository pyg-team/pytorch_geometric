APAR STATUS
 * CLOSED AS DOCUMENTATION ERROR.
    
   
   

ERROR DESCRIPTION
 *  SCENARIO:
   1) Start CATIA.
   2) Open H00434.CATProduct file in Assembly
   Design workbench.
   3) Click "Apply Material" icon. Library (ReadOnly)
   dialog box is displayed.
   4) Apply any material of Library to the Part1
   (Part1.1) present in the specification tree. The
   material is applied to Part1(Part1.1).
   5) Click File / Save Management on the menu bar.
   Save Management dialog box is displayed.
   6) Select the Save button.
   7) Select the OK button. H00434.CATProduct is
   saved.
   8) Close H00434.CATProduct Window.
   9) Click File / New on the menu bar. New dialog
   box is displayed.
   10) Select Product in the List of Types.
   11) Click OK in the New dialog box.
   Product1.CATProduct is created in Assembly
   Design workbench.
   12) Right Click on newly created product in the
   specification tree and click on Select Components /
   Existing Component in the context menu. File
   Selection window is displayed.
   13) Select H00434.CATProduct.
   14) Click Open button. H00434.CATProduct is
   inserted in Product1.CATProduct.
   15) Select View / tree expansion / Expand Second
   Level by menu bar. The specification tree is
   extended.
   16) Double click Part1(Part1.1) in the specification
   tree. Part1(Part1.1) is active.
   17) Right click on the material and select delete.
   .
   OBSERVATION:
   The material cannot be deleted as delete command
   is grayed out. Also the material cannot be deleted
   when product is active.
   .

PROBLEM CONCLUSION
 *  THIS MODIFICATION WILL BE INCLUDED IN THE
   DOCUMENTATION DELIVERED WITH CATIA
   VERSION 5 RELEASE 20 GA LEVEL.
   INCIDENT DIAGNOSIS:
   Material applied to part instance in product context
   cannot be deleted.
   .
   DOCUMENTATION REFERENCE:
   The following line is added under CATIA
   Infrastructure / Real Time Rendering / User Tasks /
   Applying a Material:
   .
   The material is stored in the document (part or
   product) which is root of the editor at the material
   application time. The modification or deletion of
   material is possible only when the root of the
   current editor is the same document.
   .
