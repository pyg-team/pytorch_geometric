Title: IBM IZE0106E Connect failed with error Not Found (cicsrgn) when connecting to CICS TS - United States

Text:

PROBLEM(ABSTRACT)
 You receive message IZE0106E Connect failed with error 'Not Found (cicsrgn)' when trying to connect the CICS Explorer to a CICS Transaction Server for z/OS (CICS TS) region. If running an earlier version of CICS Explorer (V1.1.0.x or V1.0.0.x), you will receive message CNX0106E Connection failed with error, 'CICSRGN, NOTPERMIT, USRID'. 

SYMPTOM
When the connection fails, newer releases of the CICS Explorer return message IZE0106E [http://pic.dhe.ibm.com/infocenter/ratdevz/v8r5/topic/com.ibm.cics.zos.help/topics/reference/ize0106e.html] Connect failed with error 'Not Found (cicsrgn)'. Whereas, older releases return message CNX0106E Connection failed with error, 'CICSRGN, NOTPERMIT, USRID'.


CAUSE
Security is enabled in the CICS region on the Host (SEC=YES), but the TCPIPService that the CICS Explorer is coming into has AUTHENTICATE=NO. This means that CICS will use the CICS default user id to do the authentication. Note: If you are using the supplied TCPIPService definition, DFH$WUTC, the default is AUTHENTICATE=NO. 

DIAGNOSING THE PROBLEM
Looking at the CICS job log, you will see that there are security violations being reported by message DFHXS1111 for the CICS default userid. This is because the default CICS userid did not have authority to run the CWWU transaction.


If you look at CICS trace it contains the following trace entry from when the failure occurs:
AP A334 WUEYU DATA - resultsetGetFailed RESOURCE(CICSRGN ) CONTEXT() SCOPE() CSD_RESOURCE(NO) API_FUNCTION(GET ) API_RESPONSE1(1031) API_RESPONSE2(1345). 
1031 = NOTPERMIT
1345 = USRID


RESOLVING THE PROBLEM
Alter the AUTHENTICATE parameter in the TCPIPService definition [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.resourcedefinition.doc/resources/tcpipservice/dfha4_attributes.html] that the CICS Explorer is using from your CICS region on the host to have the type of security you desire. For example, specify AUTHENTICATE=BASIC to obtain the user id and password from the client. 

If you are still receiving errors after altering the TCPIPService AUTHENTICATE option, see the MustGather for CICS Explorer errors and incorrect output [http://www.ibm.com/support/docview.wss?uid=swg21385861] to determine what documentation to gather before contacting CICS Support.
