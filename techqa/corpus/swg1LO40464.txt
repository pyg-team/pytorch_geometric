Title: IBM LO40464: WEBFORM SERVER THREADS GET HUNG WHILE UNDER HEAVY LOAD. - United States

ERROR DESCRIPTION
 *  While running a form in Webform Server under a high load the
   threads may get into a hung state.
   
   You will see the following messages in your WebSphere
   SystemOut.log for the associated server:
   
   [4/24/09 15:57:35:664 EDT] 000003ea ThreadMonitor W   WSVR0606W:
   Thread "WebContainer : 762" (000003ea) was previously reported
   to be hung but has completed.  It was active for approximately
   940198 milliseconds.  There is/are 1 thread(s) in total in the
   server that still may be hung.
   [4/24/09 15:57:35:830 EDT] 0000036b ThreadMonitor W   WSVR0606W:
   Thread "WebContainer : 664" (0000036b) was previously reported
   to be hung but has completed.  It was active for approximately
   1913979 milliseconds.  There is/are 0 thread(s) in total in the
   server that still may be hung.
   
   When in this state the server process will continue to grow in
   memory until all the memory on the server is exhausted which
   results in a crash.
   
   USER/PRODUCTS AFFECTED
   *************************
   
   Webform Server 3.5.0
   Forms that use XForms Submission
   
   
    
   
   

LOCAL FIX
 *  This issue will be resolved in the next release, 3.5.1.
   
   If you are encountering this issue and need it applied to
   3.5.0 please contact Support and quote this APAR number.
   
   
    
   
   

PROBLEM SUMMARY
 *  The Webform Server would crash after an XForms submission
   returned truncated data.  Also found and resolved a memory leak
   relatd to XForms submissions.
   
   These issues were resolved in 3.5.1.
   
   
    
   
   

PROBLEM CONCLUSION

TEMPORARY FIX
