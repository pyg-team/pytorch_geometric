ERROR DESCRIPTION
 *  Reported Release:     622
   Compid:               itm_scn_ID
   
   PROBLEM DESCRIPTION:
   
   It appears that there is a limitation with having non-ascii
   chars in the userid or in the distinguished name.
   
   Customer has NLS characters in distinguished names within LDAP.
   When a user tries to add another user into TEP, the characters
   are removed from the name and the user cannot login.
   
   
    
   
   

LOCAL FIX
 *  none - only remove the NLS characters from distinguished name
   
   
    
   
   

PROBLEM SUMMARY
 *  Non-ascii chars will not be accepted in user id or distinguished
   name
   
   
    
   
   

PROBLEM CONCLUSION
 *  In page 119 of ITM Troubleshooting guide, following info
   was added.
   ===============================================================
   
   Non-ASCII characters are not accepted in the User ID or the
   distinguished name field
   
   If you have non-ASCII characters in your User ID or
   distinguished names within LDAP, and then add another user into
   the Tivoli Enterprise Portal, the characters are removed from
   the name and the user cannot login. Remove the non-ASCII
   characters from distinguished name or the User ID.
   ===============================================================
