 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)

Sometimes all reports run without a problem, other times one or more reports result in error messages.

CAUSE
There will only be one cookie for every browser session, which will be sent through both I-Frames.
This causes problems when trying to run more than one report with regards to our internal security components.

RESOLVING THE PROBLEM


Use portlets to execute more than one report in one window. Every portlet will create it's own cookie.

When using one browser window to execute more than one report via I-Frames one or more reports can fail because of the usage of only one browser cookie. This causes problems with the internal authentication. If one of the reports gets in Async-mode the cookie will be sent and mix up things. As long as one report hits the Async-mode an error will be generated because of a mix-up of cookie entries.

Why is it working sometimes?
Because the reports do not get in Async-mode or the timing of the execution and the sending of the cookie avoids the problem. Steps:Access the portlets in Cognos Connection via "Tools"->"Portal Administration"
