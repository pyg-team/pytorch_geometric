APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When the IBM Lotus Domino template is enabled in IBM Content
   Collector, the Content Collector functions are accessed which
   includes searching for the Content Collector function image
   resources. In some environments, this search might take a long
   time causing delays and unresponsiveness on the Lotus Notes
   client workstation.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  When the IBM Lotus Domino template is enabled in IBM Content
   Collector, the Content Collector functions are accessed which
   includes searching for the Content Collector function image
   resources. In some environments, this search might take a long
   time causing delays and unresponsiveness on the Lotus Notes
   client workstation.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Install IBM Content Collector 3.0 Fix Pack 6 or 4.0 Fix Pack 3
   and set AFU_UNCHECK_IMAGE_RESOURCE = 1 in the notes.ini file.
   After you have installed the fix and set the variable, IBM
   Content Collector will skip checking the image resources.
   However, the attachment icon in the Lotus Notes view column
   might not be displayed properly.
   
   
    
   
   

TEMPORARY FIX
 *  Fix available in 3.0.0.5-IBM-ICC-Server-LA002 ,
   4.0.0.2-IBM-ICC-Server-LA016 and
   4.0.0.2-IBM-ICC-LA019 (Server and Notes Client)
