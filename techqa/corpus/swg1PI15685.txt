Title: IBM PI15685: AUTHORING UI DISPLAYS INVALID OPTIONS WHEN CHECKBOXES ARE DOUBLE CLICKED WITH IE - United States

Text:
  A FIX IS AVAILABLE
Fixes integrated in WebSphere Portal 8.0.0.1 Combined Cumulative Fixes [http://www-01.ibm.com/support/docview.wss?uid=swg24034497]



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When check boxes are double clicked within the Authoring UI
   using Internet Explorer only, invalid options are displayed to
   the user.
   
   Example 1:
   - in the library explorer, navigate into Components section
   - quickly double click the check box next to any item
   - the item will remain selected
   - now uncheck the check box for the same item
   - the Preview button as an example will still be enabled even
   though no items are actually selected
   
   
   Example 2:
   - open up a menu component item form in edit mode
   - double click the check box for the Locations criteria
   - the Locations criteria option will be selected
   - click the Locations checkbox again to unselect it
   - the Locations section will remain visible on the item form
   even though the checkbox is unchecked
   
   
    
   
   

LOCAL FIX
 *  Refresh the page to force the the current page to be reloaded in
   a clean state.
