 *  Helpful Content  *  Use BigFix Patch Management to protect your devices from Petya ransomware [https://www.ibm.com/developerworks/community/blogs/a1a33778-88b7-452a-9133-c955812f8910/entry/Use_BigFix_Patch_Management_to_protect_your_devices_from_Petya_ransomware?lang=en&acss=danl_4568_web]

      Server error: Database connection error in IBM Endpoint Manager (BigFix) [http://www.ibm.com/support/docview.wss?uid=swg21669070&acss=danl_4568_web] 
      Database Error: The database being connected to has a database version of "Enterprise 5.34", and this application requires a database version of "Enterprise 4.31". 
      After upgrade BFI/LMT server to version 9.2.7 LDAP authentication may not work in for some older LDAP types [http://aimrtp1.rtp.raleigh.ibm.com:8080/:%20http://www.ibm.com/support/docview.wss?uid=swg22001375&acss=danl_4568_web] 
      Users are no longer able to login to BFI/LMT server web console using domain accounts,but it was working before the upgrade. 
      CODIN0531E The db2fenc1 user name already exists on the system and cannot be used by the installer. [http://www.ibm.com/support/docview.wss?uid=swg22003992&acss=danl_4568_web] 
      The All-In-One ILMT installation on Linux server failed with the error above.
