APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using specific data, SPSS Statistics Desktop 24 backend engine
   crashes when the CTABLES command contain the /COMPARETEST
   STYLE=SIMPLE subcommand.
   
   No error message is captured in the Output Viewer when this
   issue occurs.
   No error message is captured in
   statistics_client_trace_file0.log file.
   
   On Windows, end-user is presented with:
   spssengine has terminated unexpectedly.
   
   On Mac, end-user is not presented with an error message.
   Application simply becomes unresponsive.
   
   Syntax being used is within expected parameters and should
   execute without errors or crashes.
   
   
    
   
   

LOCAL FIX
 *  It has been observed that if your Custom Tables does not contain
   any calculations, such as totals, categories, etc, the table
   output is generated.
