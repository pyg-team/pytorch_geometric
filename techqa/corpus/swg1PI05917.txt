PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  Users of WebSphere eXtreme Scale HTTP       *
   *                  session                                     *
   *                  management.                                 *
   ****************************************************************
   * PROBLEM DESCRIPTION: A hung thread detection message is in   *
   *                      the                                     *
   *                      WebSphere Application server web        *
   *                      container.                              *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Messages similar to the following messages are logged in the
   SystemOut and trace log files:
   ThreadMonitor W   WSVR0605W: Thread
   "WebContainer : 8" (0000009a) has
   been active for 726089 milliseconds
   and may be hung.  There is/are 1
   thread(s) in total in the server that
   may be hung.
   When the session ID from a request does not match the WebSphere
   eXtreme Scale session ID cookie, a permanent lock results on the
   WebSphere eXtreme Scale session ID key. Subsequent requests that
   contain this WebSphere eXtreme Scale session ID key are waiting
   indefinitely, resulting in the hung thread warnings.
   
   
    
   
   

PROBLEM CONCLUSION
 *  WebSphere eXtreme Scale ensures that any keys that are locked
   during a request are unlocked by the end of the request.
