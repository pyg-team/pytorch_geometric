APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When performing operations on dpa maps generated from ITX 9.0.0
   some functions (such as the Base64toText function) have
   incorrect outputs.
   
   
    
   
   

LOCAL FIX
 *  Using "IBM-1146_P100-1997"  in the dpa map  experiencing this
   seems to resolve the issue for ITX 9.0.
   
   
    
   
   

PROBLEM SUMMARY
 *  Affected is the use of WTX DPA map file.
   When calling some WTX functions like CTEXT() and CSERIESTOTEXT()
   to change data encoding, sometimes the function doesn't return
   the result in correct encoding.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in 7.5.2.14 and 7.6.0.7
   
   For a list of the latest fix packs available, please see:
   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
