Title: IBM T4 silent install fails on Windows under Active Directory Account - United States

Text:
T4; CRT; Active Directory; silent install; Windows; 71CRT-PLAN; 71CRT-INS; 71CRT-TS TECHNOTE (FAQ)

QUESTION
 
Why does the T4 Client Response Time agent installation fail in silent mode under an Active Directory account on a local Windows workstation? 

CAUSE

Under Windows, Tivoli only recognizes the Administrator account on the local workstation.


ANSWER
 

Scenario:


You log on under an Active Directory account that is a member of the Administrator's group. The silent install fails with very strange log messages. Here, you are not using Tivoli Configuration Manager (TCM) to remotely deploy the agent.

The abort messages make no sense at all, especially on workstations where Tivoli products have not previously been installed.
