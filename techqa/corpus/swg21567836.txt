Title: IBM Installing Rational Application Developer using Installation Manager results in "Java was started but returned exit code=8096" error - United States

Text:
8096; exit code; launchpad TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempts to install IBM Rational Application Developer for WebSphere Software (RAD) using IBM Installation Manager results in the error "Java was started but returned exit code=8096". 

SYMPTOM
You see the error message similar to the following: 


Java was started but returned exit code=8096
C:\TEMP\IBMIM7673501490945815724.tmp\jre_6.0.0.sr9_20110208_03\jre\bin\javaw.exe

[/support/docview.wss?uid=swg21567836&amp;aid=1]


CAUSE
The IBM Installation Manager cache directories are corrupted. 


RESOLVING THE PROBLEM
Procedure:


 1. Click Start > Control Panel > System > Advanced system settings > Environment Variables.
    
    
    
 2. Find the Windows temporary directory location in System Properties window.
    
    [/support/docview.wss?uid=swg21567836&amp;aid=2]
    
    
    
 3. Remove the following directories from the Windows temporary directory:  * ciclogs_<username> 
     * cicvolcache_<username> 
     * IBMIM<#>
    
    
    
    
    
 4. Start IBM Installation Manager
    
    
    
 5. Install IBM Rational Application Developer for WebSphere Software.
