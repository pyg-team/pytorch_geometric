LOCAL FIX

   installation on a different platform, such as Windows, or from
   the MQC71 SupportPac into the z/OS WebSphere Application Server.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   This issue affects users who have installed WebSphere MQ on z/OS
   and wish to apply the WebSphere MQ Resource Adapter, which is
   found in the java/lib/jca directory of the WebSphere MQ
   Installation, to a WebSphere Application Server instance, using
   the "Update RAR" functionality of the application server.
   
   
   Platforms affected:
   z/OS
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   A character conversion error during the build process caused
   some metadata in the resource adapter file (wmq.jmsra.rar) to be
   corrupted. This meant that the metadata could not be read
   correctly by the application server, causing the operation to
   fail with the error shown above.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The WebSphere MQ Resource Adapter build process has been updated
   to correct the character conversion error.
