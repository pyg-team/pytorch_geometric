QUESTION
 What general information should be collected to aid in IBM Rational Host On-Demand problem determination in order to and save time resolving Problem Management Records (PMRs)? 

 * Symptoms match known problems (rediscovery). 
 * There is a non-defect problem that can be identified and resolved. 
 * There is a defect that identifies a work around to reduce severity. 
 * Locating root cause can speed development of a code fix.

ANSWER
This document is intended to provide a detailed description of the documentation required by the Rational Host On-Demand support team to diagnose your problem. It contains the following sections: 

Gathering general information
Gathering component specific information
Submitting information to IBM Support
General hints and tips







Gathering general information 

If you open a PMR or SR, every problem record should include the following:  * A complete description of the problem 
 * The Host On-Demand version, release, and maintenance level 
 * The Operating System version, release, and maintenance level of server and client 
 * Related products version, and release levels, such as Web Server and Java 
 * A valid contact, contact phone number and contact email address

Submitting information to IBM Support 

Submitting documentation to the Host On-Demand support team 
 * You must open a Problem Management Record (PMR) or Service Request (SR) prior to sending doc to the Host On-Demand support teams. 
 * You can use Service Request to attach files to your PMRs. Visit the Software Support help page to learn more about Service Request (SR) [https://www-946.ibm.com/sr/help/]
