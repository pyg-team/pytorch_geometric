Title: IBM Security Bulletin: IBM InfoSphere BigInsights Console uses IBM WebSphere Application Server (WAS) Liberty profile that could allow serving servlets by classname on a per application basis (CVE-2015-1927). - United States

Text:
 SECURITY BULLETIN

SUMMARY
 Security Bulletin: IBM InfoSphere BigInsights Console uses IBM WebSphere Application Server (WAS) Liberty profile. Using WAS, serving servlets by classname is considered a security exposure and can be enabled on a per application basis in the development cycle (CVE-2015-1927). 

VULNERABILITY DETAILS
CVEID: CVE-2015-1927 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2015-1927] 
DESCRIPTION: Serving servlets by classname is considered a security exposure and is disabled by default. However, it can be enabled on a per application basis and this is sometimes used in development with the intent of removing it before going into production. The problem happens when it is enabled in development and then not removed for production - as per portal who raised the problem.

This has been fixed in WebSphere Application server, eWAS, and Liberty under APAR PI31622
This has been shipped with fixpack 8556.
CVSS Base Score: 6.8
CVSS Temporal Score: See https://exchange.xforce.ibmcloud.com/vulnerabilities/102872 [https://exchange.xforce.ibmcloud.com/vulnerabilities/102872] for the current score
CVSS Environmental Score*: Undefined
CVSS Vector: (AV:N/AC:M/Au:N/C:P/I:P/A:P) 


AFFECTED PRODUCTS AND VERSIONS
IBM InfoSphere BigInsights 2.x and 3.x that use IBM WebSphere Application Server Version 8.5 Liberty profile 

REMEDIATION/FIXES
Fix: 

 1. Stop all BigInsights Services 
 2. Apply Fix Pack for IBM WebSphere Application Server (WAS) Liberty profile. Fix needed is: Fix Pack 6 [https://www-304.ibm.com/support/docview.wss?uid=swg24040035] Level 8.5.5.6.  1. WAS References  1. Multiplatform Readme [http://www.ibm.com/support/docview.wss?uid=swg27045831] [http://www.ibm.com/support/docview.wss?uid=swg27036319] 
         2. WAS Security Bulletin http://www-01.ibm.com/support/docview.wss?uid=swg21959083 [http://www-01.ibm.com/support/docview.wss?uid=swg21959083]
        
        
    
    
 3. Start all BigInsights Services
