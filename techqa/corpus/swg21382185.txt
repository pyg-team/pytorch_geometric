Title: IBM BIP2135E Stored procedure data conversion errors in WMB or IIB - United States

Text:
mqiinfo; BIP2135E; MQSI_ODBC_ASSOCIATE_CONVERTER; IANAAppCodePage TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 You run a stored procedure against an Oracle database and receive the following error in IBM Integration Bus (IIB) or WebSphere Message Broker (WMB):

BIP2135E: Source character ''f3'' in field ''3c3f7 ... e44423e''cannot be converted to unicode from codepage '1208'. The source character is an invalid code point within the given codepage. 

CAUSE
ODBC settings

RESOLVING THE PROBLEM
 

 * Export the following environment variable in broker service Id's profile: MQSI_ODBC_ASSOCIATE_CONVERTER=522S1202
   
 * Add the following parameter to the database stanza in odbc64.ini (for 64-bit Execution Group) or odbc32.ini (for 32-bit Execution Group):
   IANAAppCodePage=106
