CAUSE
A port number is not required to be specified during a Launcher start command. In versions prior to WTX 8.4.0.0 , if a port number was specified, this would be ignored. But in WTX 8.4.0.0, the third parameter for the launcher start command is considered to be for a adminpropfile.xml file which is a launcheradmin properties file. Specifying a port number for this option results in an exception. Removing the port number during a launcher start command resolves the issue. 


Please note, there are new command line tools available in WTX 8.4.0.0 that support all of the configuration options available through the Management Console and Launcher Administration. A property file can be generated using the -export command line option of the launcheradmin command line tool. Please refer to the WTX Launcher documentation for more information.


RESOLVING THE PROBLEM
Removing the port number specified during the Launcher start command resolves the issue.
