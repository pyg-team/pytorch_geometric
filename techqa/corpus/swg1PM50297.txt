Title: IBM PM50297: Quick Scan View Users Cannot Access Jobs Created VIA XML Settings File - United States

Text:
 
APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  A customer who is an ASE administrator is trying to migrate
   Quick Scan Jobs from one instance to another
   They export the job properties of the job in the first instance
   and create a new job in the 2nd instance from the exported xml
   Advanced view users are able to see the Job and Report Pack
   Object and are able to access the Job as a Quick Scan view
   Quick Scan users cannot see the Job and only see a Report Pack
   Object
   Furthermore, if you are in Advanced View if you export a Report
   Pack to a settings file and then create a new Report Pack from
   the settings file, the Quick Scan user sees and object with a
   Quick Scan Job icon that they cannot access
