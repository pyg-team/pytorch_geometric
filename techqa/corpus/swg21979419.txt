Title: IBM Intermittent problem with Daeja ViewONE Virtual Viewer integrated with IBM Content Navigator - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 IBM Daeja ViewOne Virtual Viewer will not load the the document and the following error message appears:

"ji.streamer.splitter.f: Unable to generate page due to repository access error." 

SYMPTOM
Check the Virtual Server logs for the following:

WebContainer : 3 Feb 17, 2016 10:02:52 AM (000039922/000021000): ViewOnePlatform.navigator.tile> 10.0.216.14:7NeM5Okk8Bv1sKlTWtj9qyx Unable to retrieve document from repository: Connection timed out: connect
WebContainer : 3 Feb 17, 2016 10:02:52 AM (000039922/000000000): ViewOnePlatform.navigator.tile> streaming error
WebContainer : 3 Feb 17, 2016 10:02:52 AM (000039922/000000000): ji.streamer.splitter.f: Unable to generate page due to repository access error.
at ji.streamer.servlet.jiAbstractStreamerServlet.w(Unknown Source)


CAUSE
Often this will appear when the user is using HA environment with load balancer

ENVIRONMENT
 Daeja ViewONE Virtual Viewer integrated with IBM Content Navigator on WebSphere Application Server

DIAGNOSING THE PROBLEM
Viewer will load the files and server logs shows "Unable to generate page due to repository access error".

RESOLVING THE PROBLEM
Add the following parameters in the IBM Content Navigator "Document Streaming" tab and "Virtual" tab of Data ViewONE settings -> 

Document Streaming tab

docIdPrototype ${originalDocURL}
externalHost1 hostname/navigator
internalHost1 localhost:9043/navigator

Virtual tab
externalHost1 hostname/navigator
internalHost1 localhost:9043/navigator
