CAUSE
The installer is failing on attempting to create a file.

DIAGNOSING THE PROBLEM
When attempting to run a Deployment Engine command such as listIU.sh from the user's home directory/.acsi_username/bin, permission is denied. Permission is denied for any user, even root. 

Issue the command "mount", which showed the /home directory was not executable (noexec option)
$ mount
/dev/mapper/system-home on /home type ext3 (rw,noexec,nosuid,noatime,noacl) 


RESOLVING THE PROBLEM
When installed as a non-root user, the deployment engine by default installs in the user's home directory. The deployment engine installation includes files that must be executable. Change the deployment engine install location by exporting the IAGLOBAL_DE_INSTALL_LOCATION environment variable to an executable file system. For example 

export IAGLOBAL_DE_INSTALL_LOCATION= /opt/DE_HOME

The installation will still place some files in the non-root user's home directory, but they are non-executable files.
