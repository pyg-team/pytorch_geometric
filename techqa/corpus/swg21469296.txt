Title: IBM Java parseDouble vulnerability interim fix for embedded WebSphere Application Server - Install procedure - United States

Text:
CVE-2010-4476; parseDouble; vulnerability; embedded WAS; eWAS; embedded WebSphere Application Server; IDSWebApp; fix pack; IDSWebApp; TDS WebAdmin; TDS Web Administration Tool; upgrade eWAS TECHNOTE (FAQ)

QUESTION
 How to install Java parseDouble vulnerability interim fix for embedded WebSphere Application Server (provided by Tivoli Directory Server)? 

CAUSE
Java parseDouble vulnerability (CVE-2010-4476 [http://www.oracle.com/technetwork/topics/security/alert-cve-2010-4476-305811.html]) requires that the Java (JRE/JDK) used by embedded WebSphere Application Server(eWAS) be updated with the interim fix PM32387 [http://www.ibm.com/support/docview.wss?uid=swg21462019] (umbrella APAR for all WebSphere). 

ANSWER
The tables on the Directory server recommended fixes [http://www.ibm.com/support/docview.wss?uid=swg27009778] page provides the latest recommended eWAS fix levels. 

This problem is fixed in eWAS and SDK levels 7.0.0.17 and 6.1.0.39. If you are using eWAS / SDK levels greater than 7.0.0.17 and 6.1.0.39, there is no need to follow the procedure below. 

This document provides installation procedure for Java parseDouble vulnerability interim fix for embedded WebSphere Application Server provided by Tivoli Directory Server(TDS). 

1. Identify the eWAS install location. (eWAS_install_location) 

Here is the default install location of eWAS if TDS provided install shield GUI was used during products installations. 

V6.3.1: 

AIX, Solaris: /opt/IBM/ldap/V6.3.1/appsrv
Linux: /opt/ibm/ldap/V6.3.1/appsrv
Windows: <ITDS_V6.3.1_install_location>\appsrv 

V6.3: 

AIX, Solaris: /opt/IBM/ldap/V6.3/appsrv
Linux: /opt/ibm/ldap/V6.3/appsrv
Windows: <ITDS_V6.3_install_location>\appsrv 

V6.2: 

AIX, HP-UX, Solaris: /opt/IBM/ldap/V6.2/appsrv
Linux: /opt/ibm/ldap/V6.2/appsrv
Windows: <ITDS_V6.2_install_location>\appsrv 

V6.1: 

AIX, HP-UX, Solaris: /opt/IBM/ldap/V6.1/appsrv
Linux: /opt/ibm/ldap/V6.1/appsrv
Windows: <ITDS_V6.1_install_location>\appsrv 

2. Find the eWAS version that is in use and determine the need for fix pack installation: 

# for AIX, HP-UX, Solaris and Linux, open a login shell
cd <eWAS_install_location>/bin
./versionInfo.sh 

# for Windows open a command prompt:
cd <eWAS_install_location>\bin
versionInfo.bat 

 

3. Install eWAS and WASSDK fix packs - Use the following procedure only if you are using eWAS / SDK levels less than or equal to 7.0.0.15 and 6.1.0.37 

3. a. For eWAS 7.0 provided by TDS 6.3, install eWAS fix pack 7.0.0.13 OR 7.0.0.11. This problem is fixed in JDK provided by eWAS 7.0.0.17 (and later). The procedure below applies only if you are using 7.0.0.11 or 7.0.0.13. 

3. b. For eWAS 6.1 provided by TDS 6.2 or TDS 6.1, install eWAS fix pack 6.1.0.35 OR 6.1.0.33. The problem is fixed in JDK provided by eWAS 6.1.0.39 (and later). The procedure below applies only if you are using 6.1.0.33 or 6.1.0.35. 

 

4. Install required Java parseDouble vulnerability interim fix for eWAS (choose one of the following based on the eWAS fix pack level installed in step 3.): 

4. a. i. For eWAS 7.0.0.13 download the platform specific interim fix file from PM32175 [http://www.ibm.com/support/docview.wss?uid=swg24029103] and install using Update installer procedure - step 5 [http://www.ibm.com/support/docview.wss?uid=swg21404270]. 

4. a. ii. For eWAS 7.0.0.11 download the platform specific interim fix file from PM32173 [http://www.ibm.com/support/docview.wss?uid=swg24029110] and install using Update installer procedure - step 5 [http://www.ibm.com/support/docview.wss?uid=swg21404270]. 

4. b. i. For eWAS 6.1.0.35 download the platform specific interim fix file from PM32177 [http://www.ibm.com/support/docview.wss?uid=swg24029090] and install using Update installer procedure - step 5 [http://www.ibm.com/support/docview.wss?uid=swg21404270]. 

4. b. i. For eWAS 6.1.0.33 download the platform specific interim fix file from PM32184 [http://www.ibm.com/support/docview.wss?uid=swg24029112] and install using Update installer procedure - step 5 [http://www.ibm.com/support/docview.wss?uid=swg21404270].
