APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When using a simple .csv for the flatfile DSA
   
   #Test|#Count
   1|100
   2|200
   3|300
   
   Any time you use the above as a data provider, the data in the
   table does not show up.  However, if you use any other field
   besides "Count" it will render.  Seems that Impact is trying to
   process 'Count' instead of using it as Column name
   
   
    
   
   

LOCAL FIX
 *  Change #Count to something else.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * SQL / Flat File DSA won't return any data to the UI Data     *
   * Provider if the table contains a column with the name COUNT. *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Data Types exposed by the UI Data Provider will dynamically
   build a filter clause for the SQL query when retrieving table
   data. The filter builder will examine the incoming parameters
   from the table widget and incorporate them into the filter if
   the parameter and column name math, which includes the table
   widget's count parameter. This behavior is maintained for the
   purpose of backwards compatibility.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The UI Data Provider was updated to support a new property,
   filter.ignorelegacyparams that can selectively disable the
   legacy filter builder for a data type.
   
   The fix for this APAR is contained in the following maintenance
   packages:
   |Fix Pack | 7.1.0-TIV-NCI-FP0013
