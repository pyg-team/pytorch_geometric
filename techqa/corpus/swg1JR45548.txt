APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  There is an error in the coach framework when user designs the
   coach to reset the value in a table within a coach via boundary
   event.
   
   
    
   
   

LOCAL FIX
 *  n/a
   
   
    
   
   

PROBLEM SUMMARY
 *  Non-existing data binding (for configuration option) occurs when
   coach framework tries to refresh existing view instances with
   new data from server side.
   
   USER AFFECTED:
   Business Process Manager Advanced, Standard and Express users
   who have this kind a Coach with,
   -Table containing view that has variable as configuration option
   (e.g. Select control)
   -Uses server script to set/reset table data.
   
   PROBLEM DETAILED DESCRIPTION:
   When coach framework receives new data from server side, it
   tries to refresh the existing coach view instances. One of the
   step during the refresh is to update the data binding of
   configuration option. But some binding don't exist as the result
   of new data from server side.
   
   Reproduction,
   1. Originating customer provided a process application, with
   sample human service, containing a prepopulated table and
   ability to delete rows.
   2. Run the human service.
   3. Delete the first row.
   
   Result, error null will display. There is no error in the log
   file.
   
   Expected, the row will be deleted.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Problem has been fixed by enhancing coach framework logic, now
   it can tolerate the non-existing data binding.
