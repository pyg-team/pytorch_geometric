Title: IBM Stopping Atlas emails from non-production environment - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 In Atlas, How do you immediately stop the emails going out from non-production environment to users?. 

CAUSE
From to time, we do data refresh from Production environment to non-production environment. This datarefresh also copies the configuration and settings from Production environment making non-production environment looking like Production.

ANSWER
Please update APPSERVERURL and SMTPSERVER parameters as below in your non-production environment Atlas database and commit and restart the non-production environment application server. 

update COMPONENTPARAMETER set PARAMETERVALUE='INVALID-SMTPSERVER' where
PARAMETERNAME='SMTPSERVER';

update COMPONENTPARAMETER set PARAMETERVALUE='HTTP://NON-PRODSERVER/PolicyAtlas' where
PARAMETERNAME='APPSERVERURL';

COMMIT;

Restart the PolicyAtlas application or the Application Server.

The above steps will immediately stop emails going to users from non-production environment.

But you can also follow more detailed steps given in the following technote.
