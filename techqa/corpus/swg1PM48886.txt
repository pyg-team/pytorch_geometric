APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When using Site Management and the Resource Manager portlet to
   publish and promote pages from a source server to a target
   server, instances of the Web Content Viewer (JSR 286) portlet
   on published pages might fail to render on the target system.
   In such cases the system log file reports a
   NullPointerException as the cause of the failing portlet:
   
   
   [9/21/11 19:40:34:765 CEST] 0000004f servlet       E
   com.ibm.ws.webcontainer.servlet.ServletWrapper service
   SRVE0068E:
   Uncaught exception created in one of the service methods of the
   servlet Web Content Viewer (JSR 286) in application
   PA_WCMLRingPortJSR286.
   Exception created : javax.servlet.ServletException:
   javax.portlet.PortletException: java.lang.NullPointerException
   at com.ibm.ws.portletcontainer.invoker.impl.PortletServlet
   .doDispatch(PortletServlet.java:369)
   ...
   Caused by: java.lang.NullPointerException
   at
   com.ibm.workplace.wcm.app.ui.portlet.standard
   .JSRRenderingPortletPreferencesValidator
   .validate(JSRRenderingPortletPreferencesV
   alidator.java:704)
   Opened for release 7001
   
   
    
   
   

LOCAL FIX
 *  As a workaround delete the "WCM_TAGGING_CATEGORIES_BROADCAST"
   portlet preference from the failing instance of the Web Content
   Viewer (JSR 286) portlet.
   
   
    
   
   

PROBLEM SUMMARY
 *  A code fix for this issue is integrated into the Cumulative Fix
   24 (PM51190 [http://www-01.ibm.com/support/docview.wss?uid=swg1PM51190]) for IBM Web Content Manager V7.0.0.1.
