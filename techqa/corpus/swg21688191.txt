PROBLEM(ABSTRACT)

deployed the scanners, set the upload times, but I still have no

2014-09-21 19:10:20 (+0:00:00.265) WARN: UsageMatch: No app usage data present

Verify that the clients are collecting Application Usage data. 

From one of your clients, check to see what is in the following directory, this is the default path to the BES Client folder. 

C:\Program Files (x86)\BigFix Enterprise\BES Client\__BESData\__Global\UsageData 

Do you see any .stat files? there should be one for each running process on the client if this is set correctly. 
If you do not see any files or just a couple, check the following Registry: 

using Regedit 

Check: 
HKEY_SOFTWARE\Wow6432Node\BigFix\EnterpriseClient\Settings\_BESClient_Usagemanager_EnableAppUsageSummary 

Value = 1 


Check: 
HKEY_SOFTWARE\Wow6432Node\BigFix\EnterpriseClient\Settings\_BESClient_Usagemanager_EnableAppUsageSummaryApps 

Value = -: (there can be several configurations for this, the "-:" specifies to collect Application Usage Data for all processes on the client) 


The BES Client will check the computers process list every 5 seconds and update the .stat file for each process.


After the upload scan data for both the CIT and ITSIT scanners make sure the LMT/CIT/<compID>_citlog.xml file is uploaded to the server. In this example this is the data from a client whose computer ID is 10494614:
