ERROR DESCRIPTION
 *  When attempting to deploy a managed XOM from Rule Designer with
   insufficient authorization, the error message is confusing and
   inconsistent:
   Unexpected error:
   com.ibm.rules.res.tools.ant.xom.DeployManagedXOMException:
   The XOM repository service could not deploy file:
   C:\Users\xyzxyz &#65436;1\AppData\Local\Temp\miniloan-xom.zip
   The log file contains the actual reason for the failure:
   Access denied. The user "resMonitor" is not in any of the
   following roles: "resDeployers, resAdministrators".
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  An exception is thrown when the permission is not sufficient to
   deploy the XOM but it is caught and handled by Rule Designer
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code is fixed, similar message is prompted when deploying a
   XOM or a Ruleapp if permission is insufficient.
