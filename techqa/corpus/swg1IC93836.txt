APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  A web service request from a HTTPRequest node gets a response
   compressed with gzip and a corresponding content encoding
   header. The HTTPRequestnode decompresses the payload but leaves
   the content encoding header unchanged so that the broker reply
   to the client is incorrect. The content encoding should be
   removed
   before sending the reply to the client.
   
   
   
   
   
   
   
   
   Additional Symptom(s) Search Keyword(s): KIXREVxxx
   
   
    
   
   

LOCAL FIX
 *  None Available
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All users of WebSphere Message Broker V8.0 or IBM Integration
   Bus V9.0 and the HTTPRequest node.
   
   
   Platforms affected:
   MultiPlatform, z/OS
   
   ****************************************************************
   PROBLEM SUMMARY:
   The content encoding header for a HTTP response containing a
   zipped payload is not removed when the payload is processed
   and unzipped by the HTTPRequest node.
   
   
   
   
   
   There are a number of resource name changes between WebSphere
   Message Broker and IBM Integration Bus Version 9.0. For
   details visit
   http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e [http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e]
   tools.mft.doc/bb23814_.htm.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The product now removes the content encoding header at the point
   where the HTTPRequest node unzips the payload.
