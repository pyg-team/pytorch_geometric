Title: IBM 8096 Exit code when Rational Functional Tester connects to licence server - United States

Text:
IBM Rational Functional Tester; Rational Functional Tester; RFT; RFT 8.2; Java was started but returned exit code=8096; exit code=8096; 8096; exitdata; jar; SDP TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempts to start IBM Rational Functional Tester (RFT) v8.2.x.x or 8.3.x.x result in error "Java was started but returned ...8096" and an abrupt end. 

SYMPTOM
The error message is:

Java was started but returned exit code=8096

[/support/docview.wss?uid=swg21454409&aid=1] [/support/docview.wss?uid=swg21454409&aid=1] 


If you click OK, Rational Functional Tester will close abruptly.


You recently updated Rational Functional Tester. 

CAUSE
The newer Rational Functional Tester release is incompatible with the IBM Rational Common License Server version that you currently use.



ENVIRONMENT
 

 * Rational Functional Tester v8.2. 
 * Rational Functional Tester v8.3.

RESOLVING THE PROBLEM
Upgrade IBM Rational Common License Server to the newest version. 

See: Rational License Key Server downloads [http://www-947.ibm.com/support/entry/portal/search_results/software/rational/rational_license_key_server?filter=language:en+scope:stgsysx,dblue,ic,pubs%20+tsdoctype:D400,D410,D420,D430,D424,D425,D426,DA600,DB510&sortby=-dcdate_sortrange&prod=P410861Y68956E38&matches=0&sn=spe].
