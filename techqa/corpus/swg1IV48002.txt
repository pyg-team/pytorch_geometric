Title: IBM IV48002: KLZAGENT PROCESS HIGH CPU WHEN KLZ_PROCESS_SAMPLE_SECS=0 - United States

Text:
  A FIX IS AVAILABLE
IBM Tivoli Monitoring 6.3.0 Fix Pack 3 (6.3.0-TIV-ITM-FP0003) [http://www-01.ibm.com/support/docview.wss?uid=swg24036775]

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  IBM Tivoli Monitoring 6.3.0 FixPack 2 Linux OS Agent can
   generate high cpu usage when KLZ_PROCESS_SAMPLE_SECS=0 is set in
   the lz.ini file.
   
   
    
   
   

LOCAL FIX
 *  It is possible to workaround this issue restoring default value
   KLZ_PROCESS_SAMPLE_SECS=60 (or any other value different by 0).
   
   
    
   
   

PROBLEM SUMMARY
 *  Monitoring Agent for Linux OS process consumes high CPU when
   configured with KLZ_PROCESS_SAMPLE_SECS=0.
   
   
   Monitoring Agent for Linux OS process klzagent can generate high
   CPU usage when KLZ_PROCESS_SAMPLE_SECS=0 is set in the lz.ini
   file.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Code was update to correct the problem.
   
   The fix for this APAR will be contained in the following
   maintenance package:
   
   | FixPack | 6.3.0-TIV-ITM-FP0003
   | InterimFix | 6.3.0.2-TIV-ITM_LINUX-IF0001
   
   
    
   
   

TEMPORARY FIX
 *  Remove KLZ_PROCESS_SAMPLE_SECS=0 setting from the lz.ini file,
   or assign a non-zero value.
