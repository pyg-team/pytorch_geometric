Title: IBM Openpages 7.2 /cognos 10.2.2 - can not view reports in excel or pdf - DPR-ERR-2056 The Report Server is not responding error seen - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 User runs reports from within OpenPages and once the report is open in Cogos Viewer, select the option to VIew in Excel or View in PDF. The report is not rendered and a DPR-ERR-2056 is displayed on screen. 

SYMPTOM
Error on screen is DPR-ERR-2056 The Report Server is not responding. In cogserver.log file this error is seen - CAF-WRN-2082 An error has occurred. Please contact your administrator. The complete error has been logged by CAF 


CAUSE
The reports are using custom logos which are not present in the necessary directory on the report server.

ENVIRONMENT
OpenPages 7.2 / Cognos 10.2.2 / Linux

DIAGNOSING THE PROBLEM
Log into OpenPages, select a report to run, once open in Cognos Viewer select view in excel or pdf option, observer that report does not render in the chosen format but displays error.

RESOLVING THE PROBLEM
Check the following directory on the report server(s) on Linux:-




/opt/IBM/cognos/c10_64/webcontent/

This directory must contain the images for the logos that are used by the reports.
Copy the images to this directory, restart the Cognos and OpenPages services.
