APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  GOCCM cannot import DDL with
   GENERATED AS IDENTITY (START WITH 1)
   where
      GENERATED ALWAYS AS IDENTITY (START WITH 1)
   works OK. ADBCDTS don't recognize ALWAYS as optional. ADBCDTS
   issues the following messages.
   
   On IMPORT:
      ADBCDTS - Unsuccessful completion
                ADB9409E Registration could not be completed.
                  Reason Code . . . :             16
                  Reason  . . . . . : Failed while building the
                                      delta version.
                  Change ID . . . . :            961
      Register error. rc=            16
   
   Compare reports:
      ADB7751W The token ( was found in the Column list, but
               it is not valid.
      ADB7751W The token START was found in the Column list, but
               it is not valid.
      ADB7751W The token WITH was found in the Column list, but
               it is not valid.
      ADB7751W The token 1 was found in the Column list, but it
               is not valid.
      ADB7723E Unexpected token in table definition. The token
               is ).
   
   
    
   
   

LOCAL FIX
 *  Add "ALWAYS":
      GENERATED ALWAYS AS IDENTITY (START WITH 1)
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users of DB2 Administration Tool for z/OS    *
   *                 and DB2 Object Comparison Tool for z/OS      *
   *                 who import a change or perform a compare     *
   *                 using DDL.                                   *
   ****************************************************************
   * PROBLEM DESCRIPTION: The DDL reader fails to correctly       *
   *                      process DDL when a user tries to        *
   *                      import a change or compare using        *
   *                      a DDL that contains CREATE or           *
   *                      ALTER TABLE statement that has          *
   *                      the GENERATED clause without            *
   *                      the ALWAYS keyword.                     *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When importing a change or using compare to process DDL that
   contains CREATE or ALTER TABLE statement that has GENERATED
   clause without ALWAYS keyword, the following warning messages
   are observed:
   
   ADB7751W The token ( was found in the
   Column list, but it is not valid.
   ADB7751W The token START was found in
   the Column list, but it is not valid.
   ADB7751W The token WITH was found in
   the Column list, but it is not valid.
   ADB7751W The token 1 was found in the
   Column list, but it is not valid.
   ADB7723E Unexpected token in table
   definition. The token is ).
   
   
    
   
   

PROBLEM CONCLUSION
 *  Problem has been resolved.
