PROBLEM(ABSTRACT)
 Out of memory during installation of MCSF 8.5 on Unix or Linux, using IBM or JRockit JDKs 

SYMPTOM

RESOLVING THE PROBLEM


From the MCSF 8.5 Installation Guide, Preinstallation Setup for a UNIX or Linux Environment: Note at the bottom that ADDITIONAL_ANT_JAVA_TASK_ARGS should not be configured for IBM and JRockit JDKs.

ADDITIONAL_ANT_JAVA_TASK_ARGS=-XX:MaxPermSize=512m

ADDITIONAL_ANT_COMPILER_TASK_ARGS=-J-Xms256m -J-Xmx1408m

Check the Minimum Memory Requirements for memory parameter values based on your operating system. These parameter values are written to the sandbox.cfg file during installation. After installation, you can tune them if you are seeing Out-of-Memory errors. See the Selling and Fulfillment Foundation: Properties Guide for more information about these parameters.

To Reiterate: The ADDITIONAL_ANT_JAVA_TASK_ARGS property must not be set for IBM and JRockit JDKs.
