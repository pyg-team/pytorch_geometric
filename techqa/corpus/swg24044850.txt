PROBLEM SUMMARY: 
Ship Java 8 SR5 FP15 for WebSphere Application Server traditional and Liberty

USERS AFFECTED:
IBM WebSphere Application Server traditional v8.5.x.x and Liberty v16.0.0.2 and later Java 8 users.

PROBLEM DESCRIPTION: 
Ship Java 8 SR5 FP15 for WebSphere Application Server traditional and Liberty 

RECOMMENDATION:
Apply this i-fix to your SDK to upgrade your SDK 8 to SDK8 SR5 FP15.

NOTE:
This iFix can ONLY be applied on:
a) WebSphere Application Server traditional V8559 and above
b) Liberty V8555 and 16.0.0.2 above

NOTE:
This i-fix will back up, and then replace the WAS_HOME/java1.8_xx/jre/lib/security/java.security file. This file contains a security fix that is critical to keeping your environment secure. The back up file will be placed at WAS_HOME/java1.8_xx/jre/lib/security/java.security.bak. If you have previously updated this file, you will need to merge your changes into the new copy.

PROBLEM CONCLUSION: 
Applying this apar will upgrade your SDK 8 to SDK8 SR5 FP15 (Oracle 1.8.0_u171).

The fixes in this Java SDK are found here:
https://www.ibm.com/developerworks/java/jdk/fixes/8/#SR5FP15 [https://www.ibm.com/developerworks/java/jdk/fixes/8/#SR5FP15]

When downloading the fix from fix central, please also download the readme.txt with instructions for installation.

This Java SDK level was minimally tested with WebSphere 8.5.5.13.
