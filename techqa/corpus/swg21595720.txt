Title: IBM CNX0106E failed CMAS NOTPERMIT USRID when connecting to CICS Explorer - United States

Text:

PROBLEM(ABSTRACT)
 You receive message CNX0106E failed with error 'CMAS, NOTPERMIT, USRID'. This occurs when you attempt to connect the CICS Explorer to a CICSPlex SM Web User Interface (CPSM WUI) server using CICS management client interface (CMCI), The CMAS JESMSGLG contains a corresponding ICH408I message for the CONFIG.DEF resource profile. 

SYMPTOM
CNX0106E failed with error 'CMAS, NOTPERMIT, USRID'
ICH408 [http://publib.boulder.ibm.com/infocenter/zos/v1r13/topic/com.ibm.zos.r13.icha600/racfproc.htm#racfproc]I CONFIG.DEF


CAUSE
UACC of NONE for CONFIG.DEF resource profile instead of UACC of a minimum of READ.

The CICS Explorer connection process requires the CICSPLEX resource [http://pic.dhe.ibm.com/infocenter/cicsts/v4r2/topic/com.ibm.cics.ts.doc/eyua6/CICSPLEXtab.html] to build the CICSplex view. For existing WUI users, the DEFAULTCONTEXT and DEFAULTCMAS EYUPARMS can be set, and no security access is needed for the CONFIG.DEF resource profile. 

For a new CMCI user connecting to a secure WUI, the CICS Explorer requires the WUI to ask the CMAS to discover any and all CICSplexes so it can populate the CICSplex view tab. In order to populate this view, the Explorer requests the CICSPLEX resource table which is secured by the CONFIG.DEF resource profile. Therefore, the CICS Explorer user requires READ access to this profile in order to connect successfully.

This needs to be clearly documented in the "CICS Explorer prerequisites" topic. in the CICS Transaction Server for z/OS (CICS TS) information center. A feedback e-mail has been sent to request the update.

ENVIRONMENT
CICS Explorer V1.1 or higher connecting to a CICSplex SM WUI server; CICS TS V3.2; V4.1; V4.2;

DIAGNOSING THE PROBLEM
CICSPlex SM CMAS JESMSGLG contains message ICH408I indicating the CICS Explorer userid has insufficient access to the CONFG.DEF resource profile.

RESOLVING THE PROBLEM
Provide UACC of READ to the CONFIG.DEF resource profile in the ESM (External Security Manager).
