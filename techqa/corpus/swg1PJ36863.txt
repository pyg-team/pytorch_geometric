Title: IBM PJ36863: A search template cannot be checked into an object store when using any browser language other than English. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Checkin fails if the search template contains content criteria
   using the "All" or "Any" operator. This only occurs if the
   browser is configured to use a language other than English.
   
   Steps to reproduce:
   1 Set the browser's language to any non-English language, such
   as Chinese.
   2 Start the search designer and create a search template with
   one property and content criterion with the "All" operator.
   3 Check out the newly created search template.
   4 Select a different property for the existing criterion.
   5 Check in the search template.
   
   Actual Results:
   The following error occurs:"verityitemdata node must have only
   one child."
   
   Expected Results:
   The search template is successfully checked in.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * Users using browser language other than English and editing  *
   * a search template with content criteria                      *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Users using the browser language other than English got an   *
   * error message below when                                     *
   * they tried to check in a Search Template with content        *
   * criteria:                                                    *
   *                                                              *
   * "verityitemdata node must have only one child."              *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Customer will need to upgrade to 1.1.4.2-WPXT-FP002 or later *
   * to resolve the issue.                                        *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this issue was added to 1.1.4.2-WPXT-FP002.
   Users using that version of XT or later will be able to check in
   Search Templates without the error.
