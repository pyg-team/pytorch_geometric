STERLINGPRI TECHNOTE (FAQ)

QUESTION
 How to replace the Oracle ojdbc14.jar JDBC driver with a valid classes12.zip JDBC driver (SCI88483) 

ANSWER


 1.  
 2.  
 3.  
     
     Obtain the Oracle 9.2.0.5 JDBC driver (classes12.zip) and place in any directory on the GIS machine. 
 4.  
 5.  
 6.  
     
     Stop GIS 
 7.  
 8.  
 9.  
     
     Edit Sandbox.cfg under <SI-Home>\properties and change the following two parameters: 
     DB_DRIVERS=ojdbc14.jar 
     JDBC_DRIVER=ojdbc14.jar 
     To: 
     DB_DRIVERS=classes12.zip 
     JDBC_DRIVER=classes12.zip 
 10. 
 11. 
 12. 
     
     Edit Dynamicclasspath.cfg.in under <SI-Home>\properties and change this line: 
     DB_JAR=<Some-Path>\ojdbc14.jar 
     To: 
     DB_JAR=<Some-Path>\classes12.zip 
 13. 
 14. 
 15. 
     
     Copy Oracle 9.2.0.5 JDBC Driver (classes12.zip) to this directory: 
     <SI-Home>\jdk\jre\lib\ext 
     Under the same directory, rename or move ojdbc14.jar 
 16. 
 17. 
 18. 
     
     Under <SI-Home>\bin run: 
     setupfiles.cmd 
 19. 
 20. 
 21. 
     
     Under <SI-Home>\bin run: 
     Deployer.cmd 
 22. 
 23. 
 24. 
     
     Under <SI-Home>\bin run: 
     Install3rdparty.cmd oracle 1_2 -d <path-to-jdbcDriver>\classes12.zip 
 25. 
 26. 
 27. 
     
     Start GIS 
 28. 
 29. 
 30. 
     
     Run dump_info.cmd under <SI-Home>\bin and check the output to ensure that GIS is now using the new driver (classes12.zip).
