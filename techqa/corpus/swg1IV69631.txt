ERROR DESCRIPTION
 *  Approver:         MK
   
   Environment:      ITM 623
   
   Problem Description:
   When Watchdog is monitoring a non-OS agent and that agent is
   restarted with a different userid, the Watchdog workspace
   does not update the userid.   This works correctly on Windows.
   
   Recreate:
   To recreate problem you need set watchdog to watch non-OS
   agent. Then change run as user. Start non-OS agent with new
   user. Check AMS worksapce and you will see that User Name
   workspace is not updated with new user. When you restart UX
   agent then AMS workspaces is updated with new user.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  User Name field not updated when agent restarted with different
   userid.
   
   
   When Watchdog is monitoring a non-OS agent and that agent is
   restarted with a different userid, the Watchdog workspace
   Agents' Runtime Status does not update the User Name for the
   process.   This works correctly on Windows.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Code has been updated to update the userid.
   The fix for this APAR will be contained in the following
   maintenance packages:

   | InterimFix | 6.3.0.5-TIV-ITM_UNIX-IF0001
