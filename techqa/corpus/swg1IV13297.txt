APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The problem is that the AlertKey that is being set by the
   SelfMonitoring Service (for Service Monitoring events) takes
   into account the Cluster and Server name but doesn't populate
   the information specific to the Service which was
   stopped/started.  Therefore the ObjectServer GenericClear
   Automation (problem/resolution) action cannot distinguish
   between individual Service events and all Problem events are
   cleared by any single Resolution event.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  SelfMonitoring Service Monitoring events are being incorrectly
   handled by the ObjectServer GenericClear Automation.  Starting
   one Service will resolve ALL stopped Service events.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Updated AlertKey to include Service name
