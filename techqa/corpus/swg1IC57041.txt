LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users of WebSphere Enterprise Service Bus    *
   *                 and WebSphere Process Server v6.1, who are   *
   *                 using a Mediation Flow Component which       *
   *                 contains a XSLT primitive to transform       *
   *                 messages.                                    *
   ****************************************************************
   * PROBLEM DESCRIPTION: When a XSLT primitive processes a       *
   *                      message, two exception messages are     *
   *                      printed out into the FFDC log files,    *
   *                      of the form:                            *
   *                                                              *
   *                        java.lang.NoSuchMethodException       *
   ****************************************************************
   * RECOMMENDATION: If you encounter this problem, and are       *
   *                 concerned by the logging of these messages   *
   *                 you are advised to install the fix pack      *
   *                 which contains the code change associated    *
   *                 with this APAR.                              *
   ****************************************************************
   The FFDC messages which are printed out during the operation of
   the XSLT primitive are benign, in that as the user of the
   product you can safely ignore their meaning.  The XSLT
   primitive continues to operate in the appropriate manner,
   irrespective of the two FFDCs.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code fix associated with this APAR was targeted to be
   included within the following fix pack:
   
     v6.1.2 fix pack 1  (v6.1.2.1)
   
   
    
   
   

TEMPORARY FIX
