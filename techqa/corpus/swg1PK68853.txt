APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  WebSphere Application Server 61x, the 'com.ibm.CORBA.RequestTime
   out' variable replaces 'protocol_iiop_local_timeout'.
   
   The variable 'protocol_iiop_local_timeout' is deprecated in 61x.
   
   Currently you can set this 'Request Timeout' feature for the
   server by defining 'com.ibm.CORBA.RequestTimeout' property.
   
   How to specify: To specify this custom property, in the
   administrative console, click Application servers > server_name
   > > Container services > ORB service
   
   
   This property will reflect in the was.env of the server but NOT
   in the JESLOG of the server yet.
   
   This apar will throw the above mentioned property in the server
   joblog.
   
   
    
   
   

LOCAL FIX
 *  Currently you can set this 'Request Timeout' feature for the
   server by defining 'com.ibm.CORBA.RequestTimeout' property.
   
   How to specify: To specify this custom property, in the
   administrative console, click Application servers > server_name
   > > Container services > ORB service
