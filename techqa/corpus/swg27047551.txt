Title: IBM Documentation update - TurboIntegrator ASCII local variables - United States

Text:
 PRODUCT DOCUMENTATION

ABSTRACT
 TurboIntegrator ASCII local variables must be a standard ASCII printable character, with a decimal value between 33 and 127 inclusive. 

CONTENT
The following TurboIntegrator ASCII local variables must be set to a standard ASCII printable character, with a decimal value between 33 and 127 inclusive. Any ASCII character outside this range causes an error when importing the ASCII data. 

DatasourceASCIIDelimiter
DatasourceASCIIDecimalSeparator
DatasourceASCIIThousandSeparator
DatasourceASCIIQuoteCharacter 


Going above the decimal value 127 value enters the realm of multi-byte character sequences, which these variables do not support. Characters below the printable range - decimal values 00 thru 31 - are reserved as control characters, such as carriage return (CR), line feed (LF), and tab. Such control characters may get consumed/stripped by logic execution and should not be used for ASCII local variables. 

When setting any of these variables, the character can be specified as a character enclosed in single quotes or as an ASCII Char decimal code without quotes. 

For example, you can use either of the following statements to set the hyphen as the field delimiter for an ASCII data source in a TurboIntegrator process: 


DatasourceASCIIDelimiter='-';
DatasourceASCIIDelimiter=Char(45);
