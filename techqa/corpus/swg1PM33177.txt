 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If the user has installed a Base Portal Content installation and
   has not configured blogs/wikis, the 7001 upgrade will fail with
   the following exception:
   
   ================
   [WcmLibraryUpdate] WcmLibraryComponentUpdate: Attempting to
   connect to
   http://myportal.server.com:10039/wps/contenthandler?uri=wcmbw:id [http://myportal.server.com:10039/wps/contenthandler?uri=wcmbw:id]
   :applyUpdate. (Attempt 1 of 1)
   [WcmLibraryUpdate] WcmLibraryComponentUpdate: Success, reading
   content...
   [WcmLibraryUpdate] WcmLibraryComponentUpdate: content = ***
   ERROR WCM Library Component Update ***
   
   [WcmLibraryUpdate] ERROR: could not find a library with the
   specified wcm library name: Blog Solo Template
   v70{"identifier":"null","items":[null]}
   [WcmLibraryUpdate] null
   
   [WcmLibraryUpdate] WcmLibraryComponentUpdate: Not connected to
   server: *** ERROR WCM Library Component Update ***
   
   [WcmLibraryUpdate] ERROR: could not find a library with the
   specified wcm library name: Blog Solo Template
   v70{"identifier":"null","items":[null]}
   [WcmLibraryUpdate] null
   
   Target finished:
   action-post-apply-PTF-applyPTF7001-wp.oob.content/packager
   Target finished: post-apply-PTF
   Target finished: action-config-7001-ConfigEngine
   Target finished: CONFIG-WP-PTF-7001
   ================
   
   
    
   
   

LOCAL FIX
 *  Manual edits to the wp.blog.fp_cfg.xml can workaround the
   problem.
   
   
    
   
   

PROBLEM SUMMARY
 *  The check works on Server Base installs but was missing the case
   where an Enable or Extend ("Content") Base installation was
   present.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix is available from Fix Central:
   http://www.ibm.com/eserver/support/fixes/fixcentral/swg/quickord
   er?productid=WebSphere%20Portal&brandid=5&apar=PM33177 [http://www.ibm.com/eserver/support/fixes/fixcentral/swg/quickorder?productid=WebSphere%20Portal&brandid=5&apar=PM33177]
    You might need to type or paste the complete address into your
    Web browser.
   
   
    
   
   

TEMPORARY FIX
 *  Manually edit the wp.blog.fp_cfg.xml
