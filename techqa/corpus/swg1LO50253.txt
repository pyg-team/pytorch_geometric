ERROR DESCRIPTION
 *  GetNextDocument method of NotesView class cannot return correct
   document when
   time/date functions are used in the view selection formula.
   
   Steps:
   1. I attached reproducible database gettest.nsf. please save it
   in local data
   directory.
   2. Open the database
   3. Run Agent "Remove Document From TEST(Today)"
   
   Result:
   If the agent run correctly, all documents are removed from
   TEST(Today) view.
   However some docs are remained in the view.
   This problem also occurs in @Now, @Yesterday and @Tomorrow.
   
   If this behavior is spec, the customer request us to document
   about it in
   Designer Help.
   
   
    
   
   

LOCAL FIX
 *  use @TextToTime("Today") instead of @Today.  or  "view.update =
   false" in the
   lotusscript code.
