APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  IBM WebSphere Transformation Extender (WTX) maps are deployed to
   WebSphere Process Server (WPS) via an EAR file.
   
   The maps are called by invoking the Web Service that exposed by
   WPS.
   
   Prior to the upgrade, the result of the map can be clearly seen,
   arriving the map, at the specified input card.
   
   After the upgrade, the map receives one or more NULLs.
   
   This can be seen in the logs and the WTX input content trace
   (.mtr) file.
   
   
    
   
   

LOCAL FIX
 *  WTXCQ - WTX00046975
   PB / PB
   No known workaround
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   WebSphere Transformation Extender customers who use integration
   with WESB
   
   Problem Description:
   NULL returned from Mediation Module, after upgrading to WTX
   8.3.0.6.
   
   Platforms Affected:
   ALL
   
   
    
   
   

PROBLEM CONCLUSION
 *  Correct BPM processing by introducing the old find routine that
   only gets executed in WESB and when the type was not found by
   any other means.
   
   Applies to:
   8.3.0.6
