The SystemOut logs also contain the following:

00000073 CSIv2Effectiv W 
JSAS1477W: SECURITY CLIENT/SERVER CONFIG MISMATCH: The client security configuration (sas.client.props or outbound settings in GUI) does not support the server security configuration for the following reasons: 
ERROR 1: JSAS0809E: The current OID is RSA but this is not an Admin request.
ERROR 2: JSAS0603E: The server does not support SSL/TLS, but the client is configured to require it.


CAUSE
This error occurs when SSL is configured on the WebSphere Application Server running Workplace XT, but SSL is not configured in the WebSphere Application Server running Content Platform Engine. 


As of WebSphere 8.x, SSL is enabled by default in the application server and must be disabled for Workplace XT if not configured for the Content Platform Engine.


RESOLVING THE PROBLEM
In WebSphere Console for the Application Server hosting Workplace XT: 


1. Access Security > Global Security

2. Expand RMI/IIOP Security

3. For each of:
CSIv2 inbound communications
CSIv2 outbound communications
verify that the Transport is set to TCP/IP instead of SSL-required

4. Restart the application server
