Title: IBM TX map triggered by Launcher from a Database source fails execution with database adapter return code -1069 : Required command line options missing - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When running the IBM WebSphere Transformation Extender (WTX) / IBM Transformation Extender (ITX) Launcher product and triggering a map from an event in a database, it may be observed that the map is triggered but the database adapter fails with return code -1069 : Required command line options missing. 

SYMPTOM
Audit may show:

<Platform>Launcher for Windows - Version 8.4.1(7)</Platform>

<ExecutionSummary MapStatus="Error" mapreturn="12" ElapsedSec="0.0082" BurstRestartCount="0">
<Message>Source not available</Message>
<CommandLine>C:\Simple\Database\Test1.mmc</CommandLine>

<SourceReport card="1" adapter="DB" bytes="0" adapterreturn="-1069">
<Message>Required command line options missing</Message>
<Settings>-T+ -event 201</Settings>
</SourceReport>


CAUSE
The Launcher is incorrectly retrieving the MDQ / database connection information that is compiled into the map.

ENVIRONMENT
WTX/ITX maps running under Launcher version 8.4.1.3, 8.4.1.4 or 9.0.0.0 on any platform.

DIAGNOSING THE PROBLEM
Exactly the same map will run correctly under Command Server:

<Platform>Command Server for Windows - Version 8.4.1(7)</Platform>

<ExecutionSummary MapStatus="Valid" mapreturn="0" ElapsedSec="0.1594" BurstRestartCount="0">
<Message>Map completed successfully</Message>
<CommandLine>'C:\Simple\Database\Test1.mmc' -BI</CommandLine>
<ObjectsFound>10</ObjectsFound>
<ObjectsBuilt>1</ObjectsBuilt>

<SourceReport card="1" adapter="DB" bytes="114" adapterreturn="0">
<Message>Success</Message>
<TimeStamp>00:36:47 April 14, 2016</TimeStamp>
</SourceReport>

RESOLVING THE PROBLEM
Obtain and apply the fix for APAR PI60718
