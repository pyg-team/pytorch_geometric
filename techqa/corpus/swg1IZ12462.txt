Title: IBM IZ12462: Alter table statement with "NOT LOGGED INITIALLY WITH EMPTY TABLE" parameters additionally requires DELETE privilege. - United States

Text:
AIX 
APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 * AS OF THIS APAR, USERS WITH ALTER PRIVILEGE CAN NO LONGER ISSUE
   THE STATEMENT ALTER TABLE NOT LOGGED INITIALLY WITH EMPTY TABLE.
   USERS ALSO NEED TO HAVE DELETE PRIVILEGE IN ORDER TO ISSUE THIS
   STATEMENT SUCCESSFULLY.
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 * ****************************************************************
   USERS AFFECTED:
   ALL DB2 SYSTEMS ON ALL LINUX, UNIX, AND WINDOWS PLATFORMS AT
   SERVICE LEVELS VERSION 9.5 GA.
   ****************************************************************
   PROBLEM DESCRIPTION:
   AS OF THIS APAR, USERS WITH ALTER PRIVILEGE CAN NO LONGER ISSUE
   THE STATEMENT ALTER TABLE NOT LOGGED INITIALLY WITH EMPTY TABLE.
   USERS ALSO NEED TO HAVE DELETE PRIVILEGE IN ORDER TO ISSUE THIS
   STATEMENT SUCCESSFULLY.
   ****************************************************************
   RECOMMENDATION:
   GRANT DELETE PRIVILEGE TOGETHER WITH ALTER TO USER WHO IS
   REQUIRED TO ISSUE THIS ALTER STATEMENT.
   ****************************************************************
   
    
   
   

PROBLEM CONCLUSION
 * THE COMPLETE FIX FOR THIS PROBLEM FIRST APPEARS IN DB2 VERSION
   9.5 FIX PACK 1 AND ALL THE SUBSEQUENT FIX PACKS.
