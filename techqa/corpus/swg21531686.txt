Title: IBM Allocation errors when tranferring to HFS files (SCI92599) - United States

Text:
STERLINGPRI TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Allocation errors when tranferring to HFS files (SCI92599) 

SYMPTOM
SDE1708I 
SDE970CI 
Transfer to a HFS file submitted via the IUI interface fail with a dynamic allocation error, either SDE1708I or SDE970CI


CAUSE
HFS file name specified using incorrect syntax. The syntax varies depending on 
the RECEIVING Environment specified.

RESOLVING THE PROBLEM
When RECEIVING ENVIRONMENT=UNIX is specified, the HFS name MUST NOT be enclosed in single quotes e.g. /u/test/hsfname.txt

When RECEIVING ENVIRONMENT=OS390 is specified, the HFS name MUST be enclosed in single quotes e.g. '/u/test/hfsname.txt'
