NEW FEATURES OR UPDATES
Multi-tab recording for Internet Explorer 11 is supported starting from the following fix pack versions: 

 * IBM Security Privileged Identity Manager, Privileged Access Agent 2.1.1 
 * IBM Security Privileged Identity Manager, Privileged Access Agent fix pack 2.1.0-ISPIM-AA-FP0009 [http://www.ibm.com/support/docview.wss?uid=swg24044018] 
 * IBM Security Access Manager for Enterprise Single Sign-On AccessAgent fix pack 8.2.2-ISS-SAMESSO-AA-FP0015 [http://www.ibm.com/support/docview.wss?uid=swg24044168]


To enable this feature, deploy one of these supported fix packs and update to the Amazon Web Services (AWS), version 1.3 or later AccessProfile.

KNOWN ISSUES AND LIMITATIONS

APAR Description None. Internet Explorer Enhanced Protected Mode (EPM) is not supported. None. AccessProfile version 1.2
The following limitations apply to Internet Explorer only.  * Keyboard key strokes may not be recorded. 
 * Multiple tabs or windows for the same web application may not be recorded. 
 * Closing the first tab that initiated checkout or recording might result in check-in of the credential.


Only GUI based recording is supported.

PREREQUISITES
 

 * IBM Security Privileged Identity Manager 1.0.1 or later is deployed. 
 * If you have an existing Amazon Web Services AccessProfile, back up the existing AccessProfile, and then delete it before you upload the new Amazon Web Services AccessProfile.



For more information about AccessStudio or AccessProfiles, see the product documentation.

INSTALLATION INSTRUCTIONS
 

 1. Download the package. 
 2. Extract the AccessProfile to a location on your computer. 
 3. Upload the AccessProfile.  1. Open the AccessProfile with AccessStudio. 
     2. Select the AccessProfile from the Data type pane. 
     3. From the toolbar, click Upload selected data to IMS.
    
    



For more AccessProfiles, go to the AccessProfile Library [http://www-01.ibm.com/support/docview.wss?uid=swg21883991].
