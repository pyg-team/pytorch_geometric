Title: IBM How do I enable an IBM OnDemand Web Enablement Kit (ODWEK) trace for IBM WEBi? - United States

Text:

QUESTION
 How do I enable an IBM OnDemand Web Enablement Kit (ODWEK) trace for IBM Web Interface for Content Management (IBM WEBi)? 

ANSWER
To enable an ODWEK trace for IBM WEBi:


 1.  Open the IBM WEBi configuration tool by using the following commands:
      * Windows: <IBM WEBi install location>\admin\WEBiAdmin.bat 
      * UNIX: <IBM WEBi install location>/admin/WEBiAdmin.sh 
        
        For more information, see Starting the configuration tool [http://publib.boulder.ibm.com/infocenter/cmod/v8r4m1/topic/com.ibm.ciwbi.doc/cdnwc033.htm] in the Content Manager OnDemand Information Center. 
     
     
 2.  Click Servers and click the OnDemand tab to display the list of Content Manager OnDemand servers that are configured for IBM WEBi.
     
 3.  Select the Content Manager OnDemand server for which you want to enable the ODWEK trace and click Edit.
     
 4.  Under Configuration Parameters, click Add. 
     
 5.  From the list of parameters, select Trace Level, and then specify the desired ODWEK trace level. A trace level of 4 is recommended. Click OK.
     
 6.  Click Add again. 
     
 7.  From the list of parameters, select Trace Directory, and then specify the directory where you want to store the arswww.trace file. You must specify an existing file path. Click OK. 
     
 8.  Click OK to return to the Configure Servers dialog, and then click Finish. 
     
 9.  Run the WEBiconfig script:
      * Windows: <IBM WEBi install location>\wasScripts\WEBiconfig.bat update-files 
      * AIX/Linux: <IBM WEBi install location>/wasScripts/WEBiconfig.sh update-files
        
     
     
 10. If you use WebSphere Application Server Network Deployment, distribute your changes to your cluster nodes by clicking Ripple start in the deployment manager administrative console.


Important: After you gather the required trace files, disable the ODWEK trace. 
To disable the ODWEK trace, repeat the steps above, but set the Trace Level parameter to 0.
