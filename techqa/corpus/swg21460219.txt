QUESTION
 How do you adjust the timeout for Job Scheduling Console (JSC) or Tivoli Dynamic Workload Console (TDWC) sessions? 

CAUSE
The WebSphere LTPA token is set to time out some number of minutes after login. If you are logged into JSC or TDWC when this expires, you will be prompted to re-login. You want to extend this timeout to a longer period before being required to login again.

ANSWER
The WebSphere LTPA token timeout can be adjusted to allow longer login sessions for JSC and TDWC. 

The documentation for this can be found here:


WAS 7.0 (TWS 8.6)
http://www-01.ibm.com/support/knowledgecenter/SSEQTP_7.0.0/com.ibm.websphere.soafep.multiplatform.doc/info/ae/ae/usec_sec_domains_cache.html?cp=SSEQTP_7.0.0&lang=en [http://www-01.ibm.com/support/knowledgecenter/SSEQTP_7.0.0/com.ibm.websphere.soafep.multiplatform.doc/info/ae/ae/usec_sec_domains_cache.html?cp=SSEQTP_7.0.0&lang=en]

WAS 8.5 (TWS 9.x)
http://www-01.ibm.com/support/knowledgecenter/SSEQTP_8.5.5/com.ibm.websphere.base.doc/ae/usec_sec_domains_cache.html?lang=en [http://www-01.ibm.com/support/knowledgecenter/SSEQTP_8.5.5/com.ibm.websphere.base.doc/ae/usec_sec_domains_cache.html?lang=en]

WebSphere Application Server (WAS) must be stopped and restarted after setting the new value. You will be required to log into JSC or TDWC again.

You can also disable Automatic Generation of Tokens by setting autoGenerate="false".
