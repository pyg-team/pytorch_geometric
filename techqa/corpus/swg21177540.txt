PROBLEM(ABSTRACT)
 You receive message DFHSJ0503 Attempt to load DLL libjvm.so has failed. Runtime error message is EDC5205S DLL module not found. 

You also receive an abendASJD. This occurs when you attempt to run a Javaâ„¢ transaction in a CICS Transaction Server for z/OS region. 

SYMPTOM
DFHSJ0503 [http://publib.boulder.ibm.com/infocenter/cicsts/v4r1/index.jsp?topic=/com.ibm.cics.ts.messages.doc/cics_mc/dfhsj/SJ0503.html] and abend ASJD [http://publib.boulder.ibm.com/infocenter/cicsts/v4r1/index.jsp?topic=/com.ibm.cics.ts.messages.doc/cics_mc/ASxx_abend_codes/ASJD.html]


CAUSE
Incorrect path specified for JAVA_HOME parameter. Following are typical causes: 

 * The path does not exist. 
 * The path is misspelled. 
 * The case of letters in the path are incorrect. Java is case sensitive.

RESOLVING THE PROBLEM
Ensure that JAVA_HOME specifies the path for the correct IBM SDK for z/OS, Java Technology Edition subdirectories and JAR files. For a more complete description of the JAVA_HOME parameter see Building the JCICS sample programs [http://publib.boulder.ibm.com/infocenter/cicsts/v4r1/index.jsp?topic=/com.ibm.cics.ts.java.doc/topics/dfhpjmn.html] and Options for JVMs in a CICS environment [http://publib.boulder.ibm.com/infocenter/cicsts/v4r1/index.jsp?topic=/com.ibm.cics.ts.java.doc/topics/dfha2_jvmprofile_options.html]. [http://publibfi.boulder.ibm.com/cgi-bin/bookmgr/BOOKS/dfha2a02/3.3.2.1?SHELF=&DT=20040421185701]

 1. Enter TSO ISHELL from TSO. This will take you to the OpenMVS ISPF Shell. 
    
    
 2. Type the path name you are specifying on your JAVA_HOME parameter. For example, /usr/lpp/java/J6.0/. 
    
    
 3. Press Enter. For version Java 6.0, directories and files similar to the following should display:
