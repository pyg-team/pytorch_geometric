LOCAL FIX

PROBLEM SUMMARY
 *  Using RDz 9.1.1.1 and CICS Explorer 5.2.0 results in.
   IllegalArgumentException loop when selecting menu "OSGi
   Application Project Reference" in RDz. Selecting "OSGi
   Application Bundle Reference" works
   java.lang.IllegalArgumentException: Project facet osgi.app has
   not been defined. at org.eclipse.wst.common.project.facet.core
   .internal .FacetedProjectFrameworkImpl .getProjectFacet(Unknown
   Source) at org.eclipse.wst.common.project.facet.core
   .ProjectFacetsManager .getProjectFacet(Unknown Source) This
   exception can result if the WebSphere Application Server
   Developer Tools (WDT) is not installed.
   
   
    
   
   

PROBLEM CONCLUSION
 *  CICS Explorer was updated so that in the menu handler for the
   new button a check for the presence of a WDT plug-in is made
   using the same check used to filter out the menu items in the
   new wizard.   If the com.ibm.etools.aries.core plugin is not
   loaded then WDT is not installed and the menu item for OSGi
   Application Project reference should not be displayed.
   This fix will be made available in Version 5.2.0.4 of the CICS
   Explorer.
   For installation instructions please see:
   Ordering maintenance for the IBM CICS Explorer
   http://www.ibm.com/support/docview.wss?uid=swg21380083 [http://www.ibm.com/support/docview.wss?uid=swg21380083]
   
   
    
   
   

TEMPORARY FIX
