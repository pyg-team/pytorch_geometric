Title: IBM Sync pweb LDAP to WebSphere LDAP (TIP)  (TNPM Wireless) - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 Create user using TNPMW Native GUI, How to use it across TNPM and TCR at the same time ? 

ANSWER
1. Log in as an administrative user. 
2. If you need to add a new LDAP repository, open the Tivoli Integrated Portal administrative console by direct link: https://hostname:port_number/ibm/console/secure/securelogon.do [https://hostname:port_number/ibm/console/secure/securelogon.do] or from Tivoli Common Reporting user interface: https://hostname:port_number/ibm/console [https://hostname:port_number/ibm/console], from the navigation tree on the left, select Settings > WebSphere Admin Console, and Launch WebSphere Admin Console. When the console opens in a new window, perform the following steps: <Refer Img01 and Img02>.
a) Go to Security > Global security. <Refer Img03>

b) Select Federated repositories from the available realm definitions, then click Configure. 
<Refer Img04>
c) Click Manage repositories under Related Items. Then click Add to add a new LDAP Repository. 
<Refer Img05>
Sample : 
Global security [https://10.76.238.87:16316/ibm/console/com.ibm.ws.console.security.forwardCmd.do?forwardName=AdminSecurity.config.view] > Federated repositories [https://10.76.238.87:16316/ibm/console/adminSecurityDetail.do?forwardName=VirtualRealm.config.view&selectUserRegistry=WIM&configure=Configure...&secureApps=on&action=Edit&activeAuthMechanism=LTPA&sfname=userRegistries&lastPage=AdminSecurity.config.view&enabled=on&useDomainQualifiedUserNames=on&displayActiveUserRegistry=Federated+repositories&csrfid=ng530T6NWuFgTRCIMYj5Hsv] > Manage repositories [https://10.76.238.87:16316/ibm/console/com.ibm.ws.console.security.forwardCmd.do?forwardName=RepositoryConfig.content.main&lastPage=VirtualRealm.config.view&resourceUri=security.xml&parentRefId=WIMUserRegistry_1&contextId=cells%3ATIPCell&perspective=tab.configuration] 
<Refer Img06>
d) Enter LDAP security setting information. The primary host name must contain no spaces.
e) Select Require SSL communications if all LDAP communications should be encrypted. 
f) Select Centrally managed. 
g) Click OK 

Repository Identifier is any name to identify this repository.
Bind distinguished name and Bind Password is from the LDAP server setup.

Bind distinguished name : cn=Directory Manager 
Bind Password : Dirmgr01

3. Set the Group, Org container, Person Accounts entity, from the below link.
<Refer Img07>

*Base entry is the base path that TIP will work with to store users and groups.
1. Go to Global security -> Federated properties. On the General Properties page, under Additional Properties, click Supported Entity Types. The Supported entity types page opens.

2. Click the entity type Group, and configure the base entries for the default parent in the format ou=Groups, o=company,c=com using your server details.
"ou=Groups,o=pesistent,c=co,c=in"
<Refer Img08>

3. Repeat for entity type PersonAccount and OrgContainer.
<Refer Img09 and Img10>

4. Add Base entry to Realm 
a) Select Settings â†’ Global security.
From the Available realm definitions list, select Federated repositories and click Configure.
b) To add an entry to the base realm, Click Add Base entry to Realm.
<Refer Img11>
c) Enter the distinguished name (DN) of a base entry that uniquely identifies this set of entries in the realm. This base entry must uniquely identify the external repository in the realm. 
<Refer Img12>

Note: If multiple repositories are included in the realm, use the DN field to define an additional distinguished name that uniquely identifies this set of entries within the realm. For example, repositories LDAP1 and LDAP2 might both use o=ibm,c=us as the base entry in the repository. So o=persistent,c=com is used for LDAP1 and o=persistent2,c=com for LDAP2. The specified DN in this field maps to the LDAP DN of the base entry within the repository (such as o=persistent,c=com b). The base entry indicates the starting point for searches in this LDAP directory server (such as o= persistent,c=com c).

d) Click OK.
e) In the Messages area at the top of the Global security page, click the Save link and log out of the WebSphere Application Server console.

5. From the Available realm definitions list, select Federated repositories and click Set as current to mark the federated repository as the current realm. <Refer Img13>

6. Apply and save the changes. <Refer Img14>

7. Restart the TIP server to enable the configuration. 

a. Stop the server by entering the following command on one line:
/opt/IBM/tivoli/tipv2Components/TCRComponent/bin/stopTCRserver.sh tipadmin tipadmin
b. Start the server by entering the following command on one line:
/opt/IBM/tivoli/tipv2Components/TCRComponent/bin/startTCRserver.sh tipadmin tipadmin

8. Verify that the federated repository is correctly configured: 
a. In the navigation tree, click Users and Groups > Manage Users. 
b. Select User ID from the Search by list. 
c. Click Search to search Users in federated repository. This list includes users from both LDAP and the local file registry. <refer Img15 and Img16>
