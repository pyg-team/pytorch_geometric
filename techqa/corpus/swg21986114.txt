Title: IBM Web GUI 8.1 FP5 installation fails at post installation step - United States

Text:
FP5fails; Webgui 8.1 PF5 fail; CRIMA1217E TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When trying to install Web GUI 8.1 fp5, the installation fails due to some error at post installation step. 

CAUSE
The update to FP5 from FP2 fails on the deployment of WAR step

RESOLVING THE PROBLEM
Manual redeploy of exploded WAR in DASH resolved the issue.

(1) jar up the WAR content to a temp WAR file 
cd <JazzSM_home>/profile/installableApps/JazzSMNode01Cell/isc.ear/OMNIbusWebGUI.war 
jar -cvt <omnibus_webgui>/integration/tipInstallableApps/OMNIbusWebGUI_new.war. 

(2) Redeployed new OMNIbusWebGUI.war 
cd <JazzSM_home>/profile/bin/wsadmin.sh -conntype NONE 
$AdminApp update isc modulefile {-operation addupdate -custom forceRemove=true -contents 
<omnibus_webgui>/integration/tipInstallableApps/OMNIbusWebGUI_new.war 
-contenturi OMNIbusWebGUI.war -usedefaultbindings -contextroot 
/<context_root>/webtop} 
$AdminConfig save 
exit 

(3) Set Classloader="PARENT_LAST" for OMNIbusWebGUI.war in deployment.xml 
cd <JazzSM_home>/profile/config/cells/JazzSMNode01Cell/applications/isc.ear
/deployments/isc edit deployment.xml 
locate "module" block with attribute uri="OMNIbusWebGUI.war" 
within "classloader" block, add attribute mode="PARENT_LAST" 
e.g. 
<modules xmi:type="appdeployment:WebModuleDeployment" 
xmi:id="WebModuleDeployment_1421423463195" deploymentId="1" 
startingWeight="10000" uri="OMNIbusWebGUI.war" containsEJBContent="0"> 
<targetMappings xmi:id="DeploymentTargetMapping_1421423463195" 
target="ServerTarget_1421410731166"/> 
<classloader xmi:id="Classloader_1421423463195" mode="PARENT_LAST"/> 
</modules> 

(4) restart server 

Finally, uninstall FP2 , Install webgui 8.1 FP4 full installer, then successfully update it to FP5
