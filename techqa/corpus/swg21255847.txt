Title: IBM WASX7246E with Exception method ADMN0022E applying a Fix Pack - United States

Text:
problem apply APAR fixpack Commerce instance TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 You receive an error when applying WebSphere Commerce 6.0 APARs or Fix Pack 1 to the WebSphere Commerce instance when the WebSphere Commerce instance has WebSphere Application Server global security enabled. 

You see a similar error similar to the following in the 

WC_installdir/logs/update/actions/install/deployear_WC<instance_name>.log file

updateWASConfiguration:
[wsadmin] WASX7246E: Cannot establish "SOAP" connection to host "localhost" because of an authentication failure. Ensure that user and password are correct on the command line or in a properties file.
[wsadmin] Exception message (if any): "ADMN0022E: Access is denied for the getProcessType operation on Server MBean because of insufficient or empty credentials."
[wsadmin] WASX7213I: This scripting client is not connected to a server process; please refer to the log file D:\Program Files\IBM\WebSphere\AppServer\profiles\demo\logs\wsadmin.traceout for additional information.
[wsadmin] WASX8011W: AdminTask object is not available.
[wsadmin] Not output parameters
[wsadmin] WASX7017E: Exception received while running file "d:/Program Files/IBM/WebSphere/CommerceServer60/config/deployment/scripts/configUpdateForFP.jacl"; exception information: com.ibm.ws.scripting.ScriptingException: WASX7070E: The configuration service is not available.

        WC_installdir/instances/instance_name/properties/version/update/confighistory

        WC_userdir/instances/instance_name/properties/version/update/confighistory

        WC_installdir/instances/instance_name/properties/version/update/confighistory 
        On i5/OS 
        WC_userdir/instances/instance_name/properties/version/update/confighistory 
        
    
    
 2. Do one of the following steps depending on the level of maintenance:
     1. If you are having problems applying WebSphere Commerce 6.0 APARs to your instance(s), complete the following steps to apply the fixes:
        - Uninstall the APAR JR25011 from the WebSphere Commerce directory first.
        - Download the updated APAR JR25011 from the reference link below and apply it to the WebSphere Commerce product directory.
        
     2. If you are having problems with applying WebSphere Commerce 6.0 Fix Pack 1 to your instance(s), download the APAR IY94012 from the reference link below and apply it to the WebSphere Commerce product directory.
        
    
    
 3. Reapply the required maintenance (WebSphere Commerce 6.0 APARs or Fix Pack 1) to the WebSphere Commerce instance.


Note: APAR IY94012 is only applicable to the product level, not instance(s) level.
