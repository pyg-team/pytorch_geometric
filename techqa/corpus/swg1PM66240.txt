ERROR DESCRIPTION
 *  PU 1.53/1.5.2 Regression - PU corrupts repository when deleting
   FIXes. After certain WAS IFIX was deleted from repository using
   PU, installing WAS from the repository gives artifacts not found
   errors similar to below:
   
   Message ID: CRIMA1161E
   
   'jar file com.ibm.ws.exec.chmod_8.0.0.201203071708' not found in
   http://svr_addr/was80.repository.pmr. [http://svr_addr/was80.repository.pmr.]
   'jar file com.ibm.ws.patchinstaller_8.0.0.201203221330' not
   found in http://svr_addr/was80.repository.pmr. [http://svr_addr/was80.repository.pmr.]

   IBM Packaging Utility  1.5.3/1.5.2 Regression - Packaging
   Utility corrupts repository when deleting Fixes. After
   certain WAS IFIX was deleted from repository using Packaging
   Utility, installing WAS from the repository gives the
   artifacts not found error - Message ID: CRIMA1161E
   
   
    
   
   

PROBLEM CONCLUSION
 *  A defect was found in the way the IBM Packaging Utility
   processes shared install units in the reported scenario.  A
   fix has been made in the v1.6 IBM Packaging Utility to
   correct this problem.
