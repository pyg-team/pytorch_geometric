Title: IBM Users are seeing Firebug lite errors when accessing Connections - United States

Text:
IE11; Chrome ; event; gadgets; error; Firebug Error; issue; Found; Console TECHNOTE (TROUBLESHOOTING)

PROBLEM
When accessing Connections using either IE11 or Chrome, users will see "Firebug lite errors" or intermittently see error: "Firebug Error: Firebug GUI could not be created" and this may come up several times.

Note users don't have Firefox installed. Why would Connections be trying to launch Firebug?


SYMPTOM
When accessing Connections using either IE11 or Chrome, users will intermittently see error "Firebug Error: Firebug GUI could not be created" and this may come up several times.

Or they will get popup window with title "Firebug Lite 1.4.0"

[/support/docview.wss?uid=swg21989238&amp;aid=1]



ENVIRONMENT
Server Version:

Connections 5.5
Windows Server 2012
WebSphere 8.5.5.7
DB2 10.5.5

Client Version:
IE Version: 11.494.10586.0
Chrome: 51.0.2704.103



DIAGNOSING THE PROBLEM
1. It seems the issue started to occur when web browser submitted following

request when the IBM Connections Mail gadget was loaded: /connections/resources/socmail-client/gadgets/inbox.xml

2. The issue also occur when Calendar gadget was loaded:
/connections/opensocial/gadgets/ifr?url=https%3A%2F%2Fconnections.acme.net%2Fconnections%2Fresources%2Fweb%2Flconn.calendar%2FCalendarGadget.xml&container=default&view=default&lang=en&country=US&debug=0

In which Firebug lite console had been called : 
<script src="https://getfirebug.com/firebug-lite.js#startInNewWindow" 
type="text/javascript"></script>



RESOLVING THE PROBLEM
1. As a workaround, customer resolved the issue when browser calls IBM Connections Mail gadget by removing the content between <GadgetConfig> and </GadgetConfig> tag which is part of socialmail-discovery-config.xml


2. For the issue happened when browser calls Calendar widget
Administrator login to Websphere Application Server Integrated Solutions Console and checked: Troubleshooting -> Logs and Trace -> Logging and Tracing -> InfraCluster -> Diagnostic Trace Server -> Change Log Detail Levels

Found either of the following string:
org.apache.shindig.*=All
org.apache.shindig.gadgets.render.*=finest

Removed the above trace string and click Apply -> OK

Synchronized the nodes and restarted Application servers.
