CAUSE
The important log snippet is the following:
CWWIM4537E No principal is found from the 'CAMPUS\<USERID>' principal name.

The two obvious places to check would be the LDIF record for this user, along with the wimconfig.xml file from the DMGR profile:
How to obtain and read LDIF or LDAPSearch results: http://www-01.ibm.com/support/docview.wss?uid=swg21240886 [http://www-01.ibm.com/support/docview.wss?uid=swg21240886]
<DMGR_PROFILE>/config/cells/MdmCell01/wim/config/wimconfig.xml

The wimconfig.xml is the main configuration file and it stores the virtual member manager settings at the global (admin) level or at the domain level in a multiple security domain environment.

The LDAP Data Interchange Format (LDIF) is a representation of LDAP (Lightweight Directory Access Protocol) directory content and update requests. It contains directory content along with update requests, such as Add, Modify, Delete, and Rename, as a set of records, one record for each update request.


ANSWER
Work with your LDAP admin to review the files mentioned in the Cause section. If you are unable to make progress, then please engage WebSphere support
