APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  During the server termination, an abend 0C4-11 occurred on
   the SRB run in BBOOWRK+07E8 when it attempted to refer to an
   field created by IWMMCREA that was already freed by z/OS WLM
   resource manager as the address space was going away.
     Symptom             Description
     -------             -----------
     PIDS/5655N0100      Program id: 5655N0100
     RIDS/BBOOSRBF#L     Load module name: BBOOSRBF
     RIDS/BBOOSRBF       Csect name: BBOOSRBF
     AB/S00C4            System abend code: 00C4
     PRCS/00000011       Abend reason code: 00000011
     REGS/C11C8          Register/PSW difference for R0C:-11C8
     RIDS/BBORFRR#R      Recovery routine csect name: BBORFRR
     ...
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server V6.1 and 8.0.                        *
   ****************************************************************
   * PROBLEM DESCRIPTION: ABEND0C4 in bboosrbf/bboowork when      *
   *                      running in an address space that is     *
   *                      coming down.                            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   ABENDS0C4 in occurs in Websphere Application Server for z/OS
   SRB when it attempts to reference a WLM area that has already
   been freed by WLM's resource manager. WLM's resource manager
   has run because the address space is coming down.
   The SRB was processing a local communications clean up request
   when it abended.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Changed the recovery routine covering the SRB to not dump when
   an abend occurs in a SRB that is running in an address space
   that is coming down.
   
   APAR PM37976 is currently targeted for inclusion in Service
   Level (Fix Pack) 6.1.0.41 of WebSphere Application Server V6.1.0
   and Fix Pack 8.0.0.2 of WebSphere Application Server V8.0.
   
   The fix will also be delivered as sysrouted APAR PM38766 [http://www-01.ibm.com/support/docview.wss?uid=swg1PM38766] in
   WebSphere Application Server V7.0 for z/OS.
   
   Please refer to URL:
   //www.ibm.com/support/docview.wss?rs=404&uid=swg27006970
   for Fix Pack availability.
