APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Customer wants to be able to use the sendemail function with the
   DefaultSender and EncodingChar policy statements, as it was
   working pre-Fix Pack 1. Also one needs to use the latest
   geronimo jar file.
   
   
    
   
   

LOCAL FIX
 *  After adding Fix Pack 1 one needs to modify their policy to
   include the DefaultSender and EncodingChar policy statements, as
   well as adding the latest geronimo jar file.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All IBM Netcool/Impact users who are using SendEmail Action
   Function
   ****************************************************************
   PROBLEM DESCRIPTION:
   if CharEncoding is not set in SendEmail function and the user
   didn't download the new version of j2ee to handle special
   characters, it will throw an Exception.
   ****************************************************************
   RECOMMENDATION:
   User should upgrade to Netcool/Impact version
   4.2.0.1-TIV-NCI-IF0005
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  SendEmail doensn't need the CharEncoding nor the j2ee package by
   default.
   User can use SendEmail by default without any additional
   settings.
   
   
    
   
   

TEMPORARY FIX
