 *  javascript tag in remote rendering portlet not xdime compliant
   
   
    
   
   

PROBLEM CONCLUSION
 *  After the APAR is installed, in order to force the RRP not to
   display the javascript tag, do the following:
   
   After installing the APAR, to enable the behavior:
   * Login as portal admin
   * Navigate to the WP Admin page
   * Under Portlets, find the remote rendering portlet
   * click the wrench icon to add a new portlet parameter.  Add the
   following parameter:
   Name: RRP_DISPLAY_JSTAG
   value: false
   
   * CLick on add button
   * click Ok
   * Logout and login.
