Title: IBM Error java.io.FileNotFoundException invoking EJB JAX-WS web services client code - United States

Text:
Christy Ye; EJB client; java.io.FileNotFoundException; JAX-WS web services; EJB web services TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Invoking web services client code generated using IBM WebSphere Application Server v6.1 in the _service.java proxy file results in error "java.io.FileNotFoundException: .\META-INF\wsdl\<wsdl_file_name> (The system cannot find the path specified.)" 

SYMPTOM
 

 1. Create an EJB 2.1 project 
 2. Create a wsdl folder under the ejbModule\META-INF folder 
 3. Import the wsdl file to the ejbModule\META-INF\wsdl folder 
 4. Generate web services client using WebSphere Application Server v6.1 JAX-WS web services runtime 
 5. Invoke the generated web services code (for example generate a JSP client in Rational Application Developer for WebSphere Software and run the JSP client), and you will see similar exceptions in the console logs as follows:

CAUSE
Defect APAR PK94563 [http://ibm.com/support/docview.wss?uid=swg1PK94563] has been submitted to address this behavior.



The line to create the URL in the _service.java file contains an extra character, ".", which prevents the URL being resolved at runtime.

url = new URL("file:./META-INF/wsdl/ " ); 

Note the "dot" after "file:". 



RESOLVING THE PROBLEM
The fix for this defect/APAR is delivered in RAD and RSA v7.5.4+. 


WORKAROUND:

Remove the extra character "." from the URL in the _service.java proxy file.

url = new URL("file:./META-INF/wsdl/ " );

to 

url = new URL("file:/META-INF/wsdl/<wsdl_file_name>");

Note: There is no "dot" after "file:".
