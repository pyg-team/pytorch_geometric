Title: IBM Security Bulletin: Weak default password policy in IBM BigFix Compliance Analytics (CVE-2017-1196) - United States

Text:
 SECURITY BULLETIN

SUMMARY
 IBM BigFix Compliance Analytics uses a weak default password policy that could allow an attacker to easily guess user passwords. IBM BigFix Compliance Analytics has remediated this vulnerability. 

VULNERABILITY DETAILS
CVEID: CVE-2017-1196 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2017-1196]
DESCRIPTION: IBM BigFix Compliance (TEMA SUAv1 SCA SCM) does not require that users should have strong passwords by default, which makes it easier for attackers to compromise user accounts.
CVSS Base Score: 5.9
CVSS Temporal Score: See https://exchange.xforce.ibmcloud.com/vulnerabilities/123671 [https://exchange.xforce.ibmcloud.com/vulnerabilities/123671] for the current score
CVSS Environmental Score*: Undefined
CVSS Vector: (CVSS:3.0/AV:N/AC:H/PR:N/UI:N/S:U/C:H/I:N/A:N)

AFFECTED PRODUCTS AND VERSIONS
IBM BigFix Security Compliance Analytics 1.9.70

REMEDIATION/FIXES
 

Product VRMF APAR Remediation/First Fix IBM BigFix Compliance Analytics 1.9.79 IV96627 
Upgrade to IBM BigFix Compliance Analytics 1.9.79

Updates to Compliance/Compliance Analytics are available via Fixlets in the IEM Console. IBM BigFix Compliance Analytics 1.9.79 Fixlets were released 24th May 2017.

WORKAROUNDS AND MITIGATIONS
