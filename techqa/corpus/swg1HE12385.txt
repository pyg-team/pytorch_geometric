APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  During creation of a Hold Notice and Hold Notice
   Template,the application allows User to set the Auto-reminder
   with the option "Use Escalation Rules from Initial Notice". But
   the User can remove the Escalation rules from the Initial Notice
   post this. This creates an error situation that can potentially
   lead to the Send Reminder Notice Task failing.
   
   
    
   
   

LOCAL FIX
 *  Set up customer Hold Notice Templates correctly and inform
   the paralegals not to configure the Hold Notices to use the "Use
   Escalation Rules from Initial Notice" if there are no
   escalation rules for the Initial Notice
   
   
    
   
   

PROBLEM SUMMARY
 *  During creation of a Hold Notice and Hold Notice Template,the
   application allows User to set the Auto-reminder with the option
    "Use Escalation Rules from Initial Notice". But the User can
   remove the Escalation rules from the Initial Notice post this.
   This creates an error situation that can potentially lead to the
    Send Reminder Notice Task failing.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in Fix Central in 6033 iFix1.
   
   
    
   
   

TEMPORARY FIX
 *  Set up customer Hold Notice Templates correctly and inform the
   paralegals not to configure the Hold Notices to use the "Use
   Escalation Rules from Initial Notice" if there are no escalation
    rules for the Initial Notice.
