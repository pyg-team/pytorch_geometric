APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The DataPower device may restart due to large number of MQ
   connections being used for high volume traffic and the mq-qm
   object goes down resulting in many MQ errors with reason code of
   2059.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  When using MQ backend with connection timeout, the DataPower
   device may restart due to large number of MQ connections
   resulting in many MQ errors of 2059 reason code.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in 7.0.0.14, 7.1.0.11, 7.2.0.8, 7.5.0.2 and
   7.5.1.2

   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
