Migration documents for IBM Integration Designer and WebSphere Integration Developer 

Source artifact migration does not remove the resolutionScope attribute from migrated business processes (1218387) [http://www.ibm.com/support/docview.wss?uid=swg21218387&acss=dakc] 
Version: 6.0 
Source artifact migration from WebSphere Studio Application Developer Integration Edition does not remove the resolutionScope attribute from a migrated business process' partner links that have this attribute present. 

Backward compatibility of WebSphere Integration Developer 6.0.2 projects (1248401) [http://www.ibm.com/support/docview.wss?uid=swg21248401&acss=dakc] 
Version: 6.0.2 
WebSphere Integration Developer 6.0.2 projects cannot be used in WebSphere Integration Developer 6.0.1. 

All class files are not generated when the auto-build is off (1251647) [http://www.ibm.com/support/docview.wss?uid=swg21251647&acss=dakc] 
Version: 6.0.2 
In WebSphere Integration Developer 6.0.2, some class files are not properly generated when auto-build is set to OFF. 

Migration error received: The method logInfo(String) is undefined for the type ConvertEndChar (1256540) [http://www.ibm.com/support/docview.wss?uid=swg21256540&acss=dakc] 
Versions: 6.0.2, 6.0.2.2 and 6.1 
When trying to migrate a WebSphere InterChange Server repository, the following error is received: The method logInfo(String) is undefined for the type ConvertEndChar 

WebSphere Integration Developer V6.0.2 cannot deploy to WebSphere Process Server V6.0.1 (1256694) [http://www.ibm.com/support/docview.wss?uid=swg21256694&acss=dakc] 
Versions: 6.0.1, 6.0.1.1, 6.0.1.2 and 6.0.2 
WebSphere Integration Developer V6.0.2 cannot deploy to WebSphere Process Server V6.0.1. If you were to upgrade WebSphere Integration Developer to version 6.0.2, but did not upgrade WebSphere Process Server to the same version (6.0.2), a number of new features and functions would not be available. 

"IWAB0639E Error in generating Java files and deployment descriptors from WSDL file" error appears after generating Web service binding (1295544) [http://www.ibm.com/support/docview.wss?uid=swg21295544&acss=dakc] 
Version: 6.1 
If you generate a Web service binding on a new export created within WebSphere Integration Developer using an interface that was generated from a WebSphere InterChange Server migration, the Java files and deployment descriptors cannot be created successfully. 

Classpath migration issues when importing WebSphere Integration Developer modules (1313634) [http://www.ibm.com/support/docview.wss?uid=swg21313634&acss=dakc] 
Versions: 6.1, 6.1.0.1 and 6.1.2 
The project classpath (runtime JRE and server target) might not be migrated from WebSphere Process Server V6.0 to WebSphere Process Server V6.1 when importing WebSphere Integration Developer V6.0.x modules into WebSphere Integration Developer V6.1 and WebSphere Integration Developer V6.1.2. 

Build hangs after running the WebSphere Studio Application Developer migration wizard (1328928) [http://www.ibm.com/support/docview.wss?uid=swg21328928&acss=dakc] 
Version: 6.2 
After running the WebSphere Studio Application Developer migration wizard, the build may hang. You will not be able to initiate any other workspace operations or start a new build process. 

Build errors in a migrated workspace containing EJB client projects (1329131) [http://www.ibm.com/support/docview.wss?uid=swg21329131&acss=dakc] 
Version: 6.2 
After building a workspace that has been migrated using the WebSphere Studio Application Developer migration wizard, you may see build errors relating to the build path missing classes that are defined in EJB client projects. 

Build errors complaining a field does not exist in the Business Object (1444722) [http://www.ibm.com/support/docview.wss?uid=swg21444722&acss=dakc] 
Versions: 6.2, 6.2.0.1, 6.2.0.2, 6.2.0.3, 7.0, 7.0.0.1 and 7.0.0.2 
After migrating the projects from WebSphere Integration Developer 6.0.2 to 6.2.0, you may encounter unexpected build errors even though the projects built perfectly in 6.0.2. The build errors refer to the Java code that references some business objects. 

Failure in upgrading WebSphere Process Server test environment of WebSphere Integration Developer to V7.0.0.3 (1446037) [http://www.ibm.com/support/docview.wss?uid=swg21446037&acss=dakc] 
Version: 7.0.0.3 
During the installation of WebSphere Integration Developer V7.0.0.3 , when upgrading the WebSphere Process Server test environment to V7.0.0.3, you may experience an installation failure 

Upgrading WebSphere Process Server test environment to V7.0.0.3 for WebSphere Integration Developer (1448502) [http://www.ibm.com/support/docview.wss?uid=swg21448502&acss=dakc] 
Version: 7.0.0.3 
When upgrading the WebSphere Process Server test environment to V7.0.0.3, you may experience an installation error. 

Out of Memory error after upgrading to WebSphere Integration Developer 7.0.0.3 (1461125) [http://www.ibm.com/support/docview.wss?uid=swg21461125&acss=dakc] 
Version: 7.0.0.3 
When using WebSphere Integration Developer 7.0.0.3 for the first time after migrating, you may experience problems when connected from WebSphere Integration Developer 7.0.0.3 to ClearCase using the dynamic view. This problem may occur when you run a workspace build where maps are part of the projects and the active workspace is being used for the first time after the migration. The workspace operation seems to hang at the point where the map will be regenerated. WebSphere Integration Developer is not responsive during this time and ends with an Out of Memory exception after a couple of minutes. 

Business Integration Library not deploying with module in V7 Integration Designer when library migrated from project created in prior V6 releases (1498530) [http://www.ibm.com/support/docview.wss?uid=swg21498530&acss=dakc] 
Versions: 7.0, 7.0.0.1, 7.0.0.2 and 7.0.0.3 
When a NEW business integration library is originally created in version 6.1 IBM WebSphere Integration Designer then imported into version 7 , a user may observe this business integration library remains part of available libraries but is not included in Utility Jars. 

XML Map Multiple Mapping Error While Migrating to WebSphere Integration Developer V7.0 (1514954) [http://www.ibm.com/support/docview.wss?uid=swg21514954&acss=dakc] 
Versions: 7.0, 7.0.0.1, 7.0.0.2, 7.0.0.3 and 7.0.0.4 
Attempts to migrate from IBM WebSphere Integration Developer V6.2 to V7.0 result in Multiple Mapping to a head element not supported in this context errors when using IBM WebSphere Integration Developer. 

WebSphere Integration Developer v7.0.0.4 does not start with -vm jvm.dll and -Xmx1792m (1573139) [http://www.ibm.com/support/docview.wss?uid=swg21573139&acss=dakc] 
Version: 7.0.0.4 
With IBM WebSphere Integration Developer, v7.0.0.3 a user was able to run it with a large heap of Xmx1792. After the upgrade to WebSphere Integration Developer 7.0.0.4 the application would no longer start with the existing heap size of Xmx1792. 

Migrating artifacts with IBM Integration Designer (IID) (1614518) [http://www.ibm.com/support/docview.wss?uid=swg21614518&acss=dakc] 
Versions: 7.5.1 and 8.0 
You are importing a V7.5.1 Advanced Integration service into IBM Integration Designer V8, and it prompts you to migrate the artifacts. Should you migrate the artifacts all at once or one at a time? 

Changes for the WS-AT setting between WebSphere Process Server V6.1.2 and IBM Business Process Manager V8.0.1 (1639286) [http://www.ibm.com/support/docview.wss?uid=swg21639286&acss=dakc] 
Versions: 8.0, 8.0.1, 8.0.1.1 and 8.5 
After migrating to IBM Business Process Manager V8.0.1, the "App/EJB/Web" module is not created, so users cannot set the WS-AT setting in ejb-jar.xml. Additionally, since the transport is not JAX-WS under IBM Business Process Manager V8.0.1, the WS-AT setting cannot be enabled in the policy set. 

Source artifact migration hints and tips (7016291) [http://www.ibm.com/support/docview.wss?uid=swg27016291&acss=dakc] 
Versions: 6.0, 6.0.1, 6.0.2, 6.1, 6.1.2, 6.2 and 6.2.0.1 
The attached document provides hints and tips to consider when migrating source artifacts in WebSphere Integration Developer.
