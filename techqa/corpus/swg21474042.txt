Title: IBM JDBC application receives SqlException: Invalid parameter: Unknown column name COLUMN_NAME. ERRORCODE=-4460 - United States

Text:
getTables; JDBC; JCC; ERRORCODE; -4460; Invalid parameter: Unknown column name; DSNTIJMS; DESCSTAT; DB2; z/OS TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 A JDBC application using the IBM Data Server Driver for JDBC and SQLJ may receive the following SqlException when retrieving database object metadata from a DB2 for z/OS version 9.1 server, using the JDBC call java.sql.database.DataBaseMetadata.getTables: 

SYMPTOM
com.ibm.db2.jcc.am.SqlException: [jcc][10150][10300][3.57.82] Invalid parameter: Unknown column name <COLUMN_NAME>. ERRORCODE=-4460, 

SQLSTATE=null

at com.ibm.db2.jcc.am.bd.a(bd.java:60) 

at com.ibm.db2.jcc.am.bd.a(bd.java:103) 

at com.ibm.db2.jcc.am.hb.a(hb.java:1700) 

at com.ibm.db2.jcc.am.zl.a(zl.java:1642) 

at com.ibm.db2.jcc.am.zl.getString(zl.java:1485) 

...


CAUSE
If the The DESCSTAT subsystem parameter (DSNZPARM) on z/OS was set to NO (DESCSTAT=NO) when the DSNTIJMS install job for the DB2 Metadata Stored Procedures was performed, this problem may occur.
Setting DESCSTAT=YES is a requirement for these stored procedures.
For details, see: 
http://www.ibm.com/support/knowledgecenter/SSEPEK_10.0.0/java/src/tpc/imjcc_jccinstallwithdb2z.html [<a href=]" >Installing the IBM Data Server Driver for JDBC and SQLJ as part of a DB2 installation



RESOLVING THE PROBLEM
To resolve this problem, perform the following steps. 

1) Ensure that the DSNZPARM DESCSTAT is set to YES. 

2) Refresh the WLM application environment: 

 V WLM,APPLENV=<wlmenv>,REFRESH 

3) Re-run the DSNTIJMS install job
