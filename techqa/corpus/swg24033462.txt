INSTALLATION INSTRUCTIONS
 
Following are the two common ways to install this fix pack:


 * 
 * If IBM Rational Host Access Transformation Services Version 7.1 is already installed on your system, you can install this update using the Update Packages wizard in the IBM Installation Manager. The fix pack can be installed without the need to add a repository to the IBM Installation Manager Preferences. Simply ensure that the "Search service repositories during installation and updates" box is checked for Installation Manager "Preferences" for "Repositories".
 * 
 * 
 * If you are installing IBM Rational Host Access Transformation Services Version 7.1 for the first time or if you are installing an additional instance of it, you can install this update at the same time you install IBM Rational Host Access Transformation Services Version 7.1. Simply ensure that the "Search service repositories during installation and updates" box is checked for Installation Manager "Preferences" for "Repositories" . Then click Check for Other Versions and Extensions on the Install page of the Installing Packages wizard in the IBM Installation Manager. 
 * 


NOTE: In both install scenarios above, if the IBM Rational Host Access Transformation Services Version 7.1.0.7 fix pack does not install automatically, you may need to add a repository to the IBM Installation Manager Preferences. Refer to 
Finding fix packs for Rational Host Access Transformation Services, Versions 7.1.0.x and 7.5.0.x [https://www.ibm.com/support/docview.wss?uid=swg21450982]. 

For details about installing this fix pack, see the "Installing HATS maintenance" section in the Host Access Transformation Services Version 7.1.0.7 Release Notes [http://publib.boulder.ibm.com/infocenter/hatshelp/v71/topic/com.ibm.hats.doc/nl/en/doc/service/en/service.html#InstallMaint ] document.

HELPFUL HINTS FOR OBTAINING TECHNICAL ASSISTANCE:
Before you contact IBM Rational Software Support, gather the background information that you need to describe the problem. When you describe a problem to an IBM software support specialist, be as specific as possible and include all relevant background information so that the specialist can help you solve the problem efficiently. To save time, know the answers to these questions: 

 * What software versions were you running when the problem occurred? 
 * Do you have logs, traces, or messages that are related to the problem? 
 * Can you reproduce the problem? If so, what steps do you take to reproduce it? 
 * Is there a workaround for the problem? If so, be prepared to describe the workaround. 

If you have helpful information to diagnose or identify the problem on your system, you can provide this data by following the instructions to exchange information with IBM Technical Support [http://www.ibm.com/software/support/exchangeinfo.html].
