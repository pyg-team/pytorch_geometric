Title: IBM PM03510: MAP DESIGNER CAN CRASH WHEN USING THE XMLLIB->XSLT() FUNCTION WITH UTF-8 DATA AS THE INPUT - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  An abend occurs when using the XMLLIB->XSLT() function with
   UTF-8 data as the input.
   
   Map execution can end abnormally, causing the crash of either
   the Map Designer, Command Server, Launcher or API (depending on
   the execution method).
   
   
    
   
   

LOCAL FIX
 *  WTXCQ - WTX00032885
   PB / PB
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   Websphere Transformation Extender xmllib->XSLT function users
   
   Problem Description:
   Xmllib->XSLT function calls XALAN ICUCleanUp functions. As a
   result, WTX maps using ICU are unstable if they call XSTL
   function
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fixed a problem in xmllib library XSLT function not to perform
   ICU cleanup that can cause ICU corruption
   
   Delivered in:
   8.1.0.6, 8.2.0.5, 8.3.0.1
   
   
    
   
   

TEMPORARY FIX
