Title: IBM Rational Host On-Demand Fix Pack 1 (11.0.5.1) for 11.0.5 - United States

Text:
HOD; CSD5; ISB1; Host On-Demand On Demand HOD CSD 11.0 Fix Pack; HOD 11.0.5.1 DOWNLOADABLE FILES

ABSTRACT
 IBM Rational Host On-Demand 11.0.5.1 has been made generally available and contains fixes to version 11.0.5 including all predecessor fix packs. 

DOWNLOAD DESCRIPTION
Host On-Demand V11.0.5.1 Fix Pack is now available for download for Windows, AS/400, AIX, HPUXIA64 and z/OS.

APAR # for the Fix Pack is OA39450 - HOST ON-DEMAND FIX PACK V11.0.5.1

You may download this service update from Fix Central [http://www.ibm.com/eserver/support/fixes/] or place an order via the IBM Support Center. No physical media will be available for this Fix pack. 




NOTE: You must install V11.0.5.0 first be installing 11.0.5.1. 

To download the latest service updates for Host On-Demand, go to the Fix Central [http://www.ibm.com/eserver/support/fixes/]. 
The URL is http://www.ibm.com/eserver/support/fixes/ [http://www.ibm.com/eserver/support/fixes/].


Once at the Fix Central website, please follow these steps:


 1. Select Rational from the Product Family list. 
 2. Select Host On-Demand from the Product list. 
 3. Select 11.0.5.1 from the Installed Versions list. 
 4. Select the platform from the Platforms list. 
 5. Click Continue. 
 6. At the log in prompt, enter your IBM Web ID and password. *Note: This is the ID used to logon to Passport Advantage. 
 7. If you do not have an IBM Web ID, click on "register" to create an ID. Then you will need to submit a Self-nomination form to your Passport Advantage primary contact to give you authorization to download. 
    

You will be able to download the products to which you are entitled. If you are not entitled or authorized to download service, you will need to contact your company's Passport Advantage primary contact to give you authorization to download the code or Passport Advantage eCare [http://www-306.ibm.com/software/howtobuy/passportadvantage/pao_customers.htm]. Access this page and click on "Need Assistance" at the bottom of the page. 

The Fix Packs can be downloaded by customers who have a valid IBM Web ID. Fix Packs do not require the entitlement check. If any non-entitled customer tries to download a Refresh Pack, a Special build, or Tools, they will get an error message "The following fix cannot be downloaded due to an entitlement failure." The Fix name is shown below the message. 

Non-entitled customers need to contact their company's Passport Advantage Primary Contact and request authorization to download. To get authorization, they can log-on to Passport Advantage and send a Self-nomination to the primary contact. If unable to gain authorization, contact the Primary contact to download the package needed." 

If you have any questions or if you are unable to find the service you need, please contact the Host On-Demand or Personal Communication support center by calling 800-IBM-SERV or via Electronic Service Request (ESR) or IBMLink. 


NOTE to IBM Employees: 
You will need to authenticate to Fix Central with your IBM Intranet ID and IBM Web ID at the following site: https://port.rchland.ibm.com/support/fixcentral/ac/options 
then proceed to download the packages with the above steps.

HELPFUL HINTS FOR OBTAINING TECHNICAL ASSISTANCE:
Before you contact IBM Rational Software Support, gather the background information that you need to describe the problem. When you describe a problem to an IBM software support specialist, be as specific as possible and include all relevant background information so that the specialist can help you solve the problem efficiently. To save time, know the answers to these questions: 

 * What software versions were you running when the problem occurred? 
 * Do you have logs, traces, or messages that are related to the problem? 
 * Can you reproduce the problem? If so, what steps do you take to reproduce it? 
 * Is there a workaround for the problem? If so, be prepared to describe the workaround. 

If you have helpful information to diagnose or identify the problem on your system, you can provide this data by following the instructions to exchange information with IBM Technical Support [http://www.ibm.com/software/support/exchangeinfo.html].
