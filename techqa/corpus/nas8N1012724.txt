CERT TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 This document describes how to import a Certificate Authority using iKeyMan for System i Access for Windows. 

RESOLVING THE PROBLEM
IBM Key Management (iKeyMan) is used to import certificates for use of System i Access for Windows. This may be the only option to import a Certificate Authority for use when connecting to a System i SSL server. 

Step 1. Open iKeyMan one of three ways:

a. Select Windows Start>Run, and type cwbuisxe.exe /ikeyman

Initializing from RUN [/support/docview.wss?uid=nas8N1012724&amp;aid=1]

b. Select Windows Start>Programs>IBM System i Access for Windows>IBM Key Management:

Initializing from START [/support/docview.wss?uid=nas8N1012724&amp;aid=2]

c. Open the IBM System i Access for Windows folder on the desktop and select IBM Key Management:

Desktop icon [/support/docview.wss?uid=nas8N1012724&amp;aid=3] 

Initializing from Access folder [/support/docview.wss?uid=nas8N1012724&aid=4] [/support/docview.wss?uid=nas8N1012724&aid=4] Step 2. Select the Open button and select the cwbssldf.kdb file located in C:\Documents and Settings\All Users\Documents\IBM\Client Access\

Step 3. Use the pulldown in the middle of the panel to change from Personal Certificates to Signer Certificates. Then, select the Add... button located on the right panel, and change the Certificate file name to *.cer or whatever the extension of the certificate is. Next, select Browse... and navigate to the certificate file located on the PC.
