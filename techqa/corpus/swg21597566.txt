Title: IBM Configuring Identity Manager for FIPS compliance - United States

QUESTION
 How do you set up IBM Tivoli Identity Manager (ITIM) and its middleware for Federal Information Processing Standards compliance? 

ANSWER
From version 5.0, IBM Tivoli Identity Manager has supported Federal Information Processing Standards (FIPS) compliance. FIPS is both a set of acceptable encryption algorithms and a set of processes a site must follow. This document provides links to all of the documentation necessary to configure ITIM and it's middleware for FIPS mode, it does not cover any of the additional processes that must be adhered to for full FIPS compliance. 

FIPS also requires that all secure communication take place using Transport Level Security (TLS), as defined in RFC 2246. TLS is based on, but incompatible with, SSLv3. However, even though at a technical level the protocol is different, from a configuration standpoint, the setup is exactly the same as SSL. So in virtually all of our documentation, the setup steps are for "SSL" with little or no mention of TLS. It is important to understand that when configured for secure mode, all IBM components in the ITIM infrastructure will first attempt to communicate using TLS, before dropping down to SSL if that fails. When FIPS mode is enabled, the components will ONLY communicate securely using TLS, if that fails, the connection fails.
