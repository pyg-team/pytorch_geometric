Title: IBM CTGTRI157E error when connecting to Cognos reporting engine - United States

Text:
ITM TCR R&M TSM monitoring reporting DASH TEP TEPS TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Any operations that need a connection to the Cognos reporting engine failed with CTGTRI157E error. 

SYMPTOM
CTGTRI157EAn error occurred while connecting to the Cognos reporting engine. 

This error can occur on any operations that need a connection to the Cognos reporting engine. This includes, but is not limited to:
1. Running the datasource script. (To import Cognos reports and connects Tivoli Common Reporting to the WAREHOUS database)
2. View/create/generate reports.


CAUSE
Error connecting to the Cognos reporting engine. Most likely cause is Jazz for Service Management (JazzSM) service is not started.


RESOLVING THE PROBLEM
Stop/start JazzSM service. 

By default, JazzSM is not started automatically and does not restart after a system reboot. To stop/start JazzSM service: 

On Windows, 

cd jazzsm_install_dir\profile\bin 

stopServer.bat server1 

startServer.bat server1 

 

On AIX/Linux, 

cd jazzsm_install_dir/profile/bin 

./stopServer.sh server1 

./startServer.sh server1 

 

Where jazzsm_install_dir is the JazzSM installation directory. The default installation directory for JazzSM is: 

C:\Program Files\IBM\JazzSM for Windows 

/opt/IBM/JazzSM for AIX/Linux 

 

The JazzSM service can also be set to start automatically when the system starts. For information on how to start JazzSM service automatically, review The Tivoli Monitoring for Tivoli Storage Manager user guide, section "Starting the Jazz for Service Management service"
