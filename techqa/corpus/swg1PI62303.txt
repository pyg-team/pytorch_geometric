APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Previous versions of the bluemixUtility used TLS 1.0 when
   sending request to Bluemix, it has now been updated to use
   TLS 1.1 or 1.2
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server Liberty - Liberty Buildpack          *
   ****************************************************************
   * PROBLEM DESCRIPTION: Update the bluemixUtility to use TLS    *
   *                      1.1 or 1.2 instead of 1.0 when          *
   *                      communicating with Bluemix              *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Bluemix deprecated use of TLS 1.0 thus requiring an update to
   the bluemixUtility to use the correct versions.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Modified the bluemixUtility to use TLS 1.1 or 1.2 when
   communicating with Bluemix
   
   The fix for this APAR is currently targeted for inclusion in fix
   pack 8.5.5.10.  Please refer to the Recommended Updates page for
   delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
