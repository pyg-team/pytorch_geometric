Title: IBM System Director gives error '-4499' while changing root password for PureData for Operational Analytics ( PDOA ) - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 While changing the root password for PDOA appliance, IBM System Director could show status error. 



SYMPTOM
The symptom could be, for example- 


1) smstop could hang 
2) smstart might not return anything. 



CAUSE
It is a issue with shared memory 



DIAGNOSING THE PROBLEM
From the logcollector ( /opt/ibm/director/bin/logcollector ) below error message could be seen -> 

 

Timestamp : 2015/05/13 16:45:57.407
Severity : SEVERE
Class : com.ibm.lwi.database.datasources.proxy.ProxyDatasource
Method : getConnection()
Thread : Thread-9
Logger : com.ibm.lwi.database.datasources.proxy

Message : Failed to get connection for ID: usmi
Secondary : com.ibm.db2.jcc.am.io: [jcc][t4][2043][11550][4.7.85]
Exception java.net.ConnectException: Error opening socket to server
localhost/127.0.0.1 on port 50,010 with message: A remote host refused
an attempted connect operation.. ERRORCODE=-4499, SQLSTATE=08001



RESOLVING THE PROBLEM
In order to resolve this problem please follow the steps below - 


1. Open a command prompt.
2. Run the /opt/ibm/director/bin/smdb stop command.
3. Run the /opt/ibm/director/bin/smdb start command.
4. Restart the Director Server
