APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After applying PK74042 [http://www-01.ibm.com/support/docview.wss?uid=swg1PK74042] and using the JSR160RMI connector, the
   server will take 2-3 mins to stop where it took less than a
   minute before.  It doesn't affect the SOAP connector or programs
   using wsadmin, it only affects RAD.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server V7.0                                 *
   ****************************************************************
   * PROBLEM DESCRIPTION: System may take up to 3 minutes to      *
   *                      stop when using the JSR160RMI           *
   *                      connector to issue the stop command.    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When using an admin client and the JSR160RMI connector, it can
   take up to 3 minutes for the server to stop after receiving
   the command to stop.  This is due to the admin client using
   polling to retrieve notifications and the ORB checking for
   recent requests.  When the polling interval is less than the
   ORB check time of 5 seconds, the ORB will not stop until the 3
   minute timeout is reached.  The problem will only occur on
   systems that have PK74042 [http://www-01.ibm.com/support/docview.wss?uid=swg1PK74042] or 7.0.0.1 applied to the admin
   client.  If the notification listener is removed before the
   stop command, then the polling will finish and the stop will
   not wait.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The polling code for the admin client was changed to increase
   the polling interval when an exception was received from the
   server.  This will increase the polling interval to be greater
   than the ORB check time after a few tries.
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 7.0.0.5.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
