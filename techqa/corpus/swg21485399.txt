Title: IBM Getting 'Out of Memory' error when trying to run a Modeler stream in Modeler client connected to Modeler Windows 2008 64 bit Server. - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Trying to open Modeler stream in Modeler running on Windows 2008 Server 64 bit. Getting an 'Out of Memory' issue and Java Heap dumps.

javacore:

NULL ------------------------------------------------------------------------
0SECTION TITLE subcomponent dump routine
NULL ===============================
1TISIGINFO Dump Event "systhrow" (00040000) Detail "java/lang/OutOfMemoryError" received 
1TIDATETIME Date: 2013/04/19 at 11:37:15
1TIFILENAME Javacore filename: C:\Program Files\IBM\SPSS\Modeler\15\javacore.20130419.113654.2056.0006.txt
1TIREQFLAGS Request Flags: 0x81 (exclusive+preempt)
1TIPREPSTATE Prep State: 0x4 (exclusive_vm_access)



RESOLVING THE PROBLEM
Increase the Xmx256m value in the Modeler Server installation /config directory located in the jvm.cfg file.

example:
# JVM options
# These are cumulative
#options, "-Xms64m"
options, "-Xmx256m"
options, "-Djavax.xml.parsers.SAXParserFactory=org.apache.xerces.jaxp.SAXParserFactoryImpl"
options, "-Djavax.xml.parsers.DocumentBuilderFactory=org.apache.xerces.jaxp.DocumentBuilderFactoryImpl
