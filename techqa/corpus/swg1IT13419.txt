APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When the IBM MQ Appliance is restarted, the SSH daemon fails to
   start, preventing administrative SSH connections from being
   established with the appliance.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   This issue affects users of the MQ Appliance
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   A timing condition within the MQ Appliance could cause temporary
   process files to fail to be cleaned up following a reboot.
   
   If encountering this condition, the SSH daemon could fail to
   start, leaving the MQ Appliance unable to accept SSH
   connections.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The MQ Appliance has been updated to address the failure to
   remove these files on reboot.
