PDF: IBM Atlas Policy Suite Administrators Guide: System Configuration 
(AdministratorsGuide_SystemConfiguration.pdf)

Section: 
EMAIL_HEADER_EXTENSIONS component


 * Issue: 
 * 
 * 
 *  Some email clients do not recognize the string values that are shown in the table of example parameters: (Table 1. Parameters for composing a set of sensitivity entries. ). This component is only valid for hold notices and hold notice templates. The component parameters will only be used in the message body. 
 * 
 * Correction:
 * 
 * 
 * Use the EMAIL_HEADER_EXTENSIONS component to create SMTP header entries in hold notices and hold notice templates. 
 * The key of each EMAIL_HEADER_EXTENSIONS parameter is a variable in the form $VariableName.The value for the parameter is a string that replaces the variable before the message is sent. The variable are added to the body of the message. 
 * Different email clients interpret and handle the mail header differently, so you should test this feature before you implement it to ensure that you understand the behavior in the various email clients. 
 * Changes to this component are effective immediately after you save them. 
 * Parameters 
 *  Key Value $VariableName ReplacementString 
 * Each parameter defines a single variable that can be used in a hold notice or hold notice template. The parameter key must be in the form $VariableName, where the '$' is literal. 
 * There are no default parameters for this component.
