Title: IBM ACUEX0073E errror is occurring when backing up with de_backupdb script - United States

Text:
DEL2; ACUEX0073E ; de_backupdb; backup; restore TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The execution of the de_backupdb in Tivoli Integrated Portal, result in below errors:

ACUEX0073E An error has occurred when backing up or restoring the Deployment Engine installation database. 
Deployment Engine is in use. 

SYMPTOM
$ ./de_backupdb 

ACUDB0004I A database backup command was issued, but directory 
/usr/ibm/common/acsi/backupDBs, the default directory where database 
backup information is stored, does not exist. Deployment Engine will 
create the directory. 
Backing up database to /usr/ibm/common/acsi/backupDBs/201111220912.zip; 
please wait ... 
ACUEX0073E An error has occurred when backing up or restoring the 
Deployment Engine installation database. 
Deployment Engine is in use. 


CAUSE
.lock files in the logs directory may prevent the backup utility from working correctly. 

RESOLVING THE PROBLEM
Delete all the .lock* files from logs directory for Deployment Engine:


[/support/docview.wss?uid=swg21691434&amp;aid=1]
C:\Program Files\IBM\Common\acsi\logs
[/support/docview.wss?uid=swg21691434&amp;aid=2]
For root install :
/usr/ibm/common/acsi/logs
For non-root install :
<Install User's HOME>/.acsi_<install_username>/logs

Try the backup command again. 

If this does not resolve the issue then please send all the files from the logs directory for further investigation.
