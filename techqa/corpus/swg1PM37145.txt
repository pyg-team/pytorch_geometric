APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Custom Request definitions that instrument application methods
   with primitive char arguments must have these arguments passed
   in the equivalent Java object wrapper. The BCI helper for custom
   requests incorrectly specifies java/lang/Char as the wrapper
   class instead of java/lang/Character.
   
   Users Affected:
   All users of ITCAM for WebSphere 6.1, ITCAM
   for Application Diagnostsics using Custom
   Requests.
   
   Keyword/Symptom:
   NoClassDefFoundError Custom Request
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:
   *
   * All ITCAM for WebSphere 6.1/AD 7.1 Data collector configured
   * systems.
   *
   ****************************************************************
   *
   * PROBLEM DESCRIPTION:
   *
   * When a method that is monitored by ITCAM Data collector by
   * injecting byte code for a callback function contains a Java
   * char primitive data type in its arguments, it is incorrectly
   * incorrectly tried to be wrapped with a
   * java.lang.Char, which is a non-existent class, instead of
   * java.lang.Character.
   *
   * This results in NoClassDefFoundError for java/lang/Char class
   * in the ITCAM Data collector logs. The application itself will
   * not be affected by this error.  The method whose argument has
   * the primitvie char will not be monitored by ITCAM.
   *
   ****************************************************************
