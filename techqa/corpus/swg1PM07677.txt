Title: IBM PM07677: NEW SITE WIZARD TEMPLATES NOT DEPLOYING THE WCM HIDDEN PAGE IN NEW VIRTUAL PORTALS. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  New Site Wizard templates not deploying the WCM hidden page in
   new virtual portals. This can introduce problems working with
   WCM on a virtual portal.
   
   
   Symptoms include:
   
   After creating a new site using the site wizard:
   
   If you are using inline editing in a virtual portal and click
   the insert image icon in the default RTE the popup does not come
   up correctly.
   
   The popup window will have several copies of the insert image
   dialog, one for every portlet on the calling page.
   
   
    
   
   

LOCAL FIX
 *  For existing virtual portal created with the new site wizard,
   use xmlaccess to export hidden WCM page from base portal and
   import into virtual portal.
   
   Hidden page is: com.ibm.wps.hiddenpage.wcm.Authoring_Portlet
   
   For new virtual portals created with the new site wizard:
   
   Update the xml in the template used to create the site, similar
   to:
   
   <content-node action="update" active="true"
   allportletsallowed="false"
   content-parentref="6_000000000000000000000000A0"
   create-type="explicit" domain="rel"
   objectid="<value>" ordinal="3250"
   type="page"
   uniquename="com.ibm.wps.hiddenpage.wcm.Authoring_Portlet">
   <supported-markup markup="html" update="set"/>
   <parameter name="com.ibm.portal.Hidden" type="string"
   update="set"><![CDATA[true]]></parameter>
   <access-control externalized="false" owner="undefined"
   private="false">
   <role actionset="Privileged User" update="set">
   <mapping subjectid="@VpUserGroup@" subjecttype="user_group"
   update="set"/>
   </role>
   </access-control>
   <component action="update" active="true" deletable="undefined"
   domain="rel" modifiable="undefined"
   objectid="<value>" ordinal="200"
   orientation="H" type="container" width="undefined"/>
   </content-node>
   
   
    
   
   

PROBLEM SUMMARY
 *  New Site Wizard templates not deploying the WCM hidden page in
   new virtual portals. This can introduce problems working with
   WCM on a virtual portal.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Added the hidden WCM Authoring page for inline editing in the
   Department Internet and Internet site templates.
