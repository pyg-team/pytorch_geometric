Title: IBM JR54367: TEMP FILES ARE NOT DELETED AFTER YOU DEPLOY PROCESS APPLICATIONS ONLINE - United States

Text:
  FIXES ARE AVAILABLE
Version 8.5.0 Fix Pack 2 for the IBM Business Process Manager products [http://www-01.ibm.com/support/docview.wss?uid=swg24040683]
Version 8.5 Refresh Pack 7 for the IBM Business Process Manager products [http://www-01.ibm.com/support/docview.wss?uid=swg24041575]


SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When you deploy process apps from IBM Process Center, many
   temporary files are generated in the /tmp or %TEMP% folder.
   After several attempts to deploy process apps, the /tmp or
   %TEMP% folder is filled with temp files. These temp files are
   not cleaned up automatically.
   
   PRODUCTS AFFECTED
   IBM Business Process Manager (BPM) Advanced
   IBM BPM Standard
   IBM BPM Express
   
   
    
   
   

LOCAL FIX
 *  Delete files from the /tmp or %TEMP% folder.
   
   
    
   
   

PROBLEM SUMMARY
 *  No additional information is available.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix is available that removes the temp files from the /tmp or
   %TEMP% folder automatically after process apps are deployed
   online.
   
   On Fix Central (http://www.ibm.com/support/fixcentral), search
   for JR54367:
   
   1. Select IBM Business Process Manager with your edition from
   the product selector, the installed version to the fix pack
   level, and your platform, and then click Continue.
   2. Select APAR or SPR, enter JR54367, and click Continue.
   
   When you download fix packages, ensure that you also download
   the readme file for each fix. Review each readme file for
   additional installation instructions and information about the
   fix.
