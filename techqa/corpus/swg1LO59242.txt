Title: IBM LO59242: NOKIA DEVICE NOT RECOGNIZED AS SUPPORTING SECURITY POLICY. - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Customer reported that one of their Nokia devices was banned as
   a result of "Prohibit devices incapable of security compliance"
   even though the device was compliant with the security policy
   and had the security tarm installed from Nokia.
   
   
    
   
   

LOCAL FIX
 *  Problem only happens when selecting "Prohibit devices incapable
   of security"
   
   
    
   
   

PROBLEM SUMMARY
 *  Once a Nokia device was banned for violation of security policy,
   it was found to be difficult for the device to get un-banned
   when the secuirty violation is resolved.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The Lotus Traveler server was updated to handle this situation.
   It only happened for devices banned initially during
   registration as being not capable of security.
   
   
    
   
   

TEMPORARY FIX
 *  This fix is includedin in Lotus Traveler 8522 and will be
   inlcuded in all future releases.
   http://www.ibm.com/support/docview.wss?uid=swg24019529 [http://www.ibm.com/support/docview.wss?uid=swg24019529]
