APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Enabling the trace "com.ibm.config.eclipse.wtp=finer"
   causes a FileNotFoundException when installing an EJB 3.0
   jar file which has no deployment descriptor
   (META-INF/ejb-jar.xml), with
   exception text
   "java.io.FileNotFoundException: IWAE0065E Absolute path
   unknown",
   and occuring from
   org.eclipse.jst.j2ee.commonarchivecore.internal.strategy.
   TempZipFileLoadStrategyImpl.getAbsolutePath.
   A DeploymentDescriptorLoadException (which wraps the
   FileNotFoundException) also occurs, with the text:
   org.eclipse.jst.j2ee.commonarchivecore.internal.exception.
   DeploymentDescriptorLoadException: META-INF/ejb-jar.xml".
   
   
    
   
   

LOCAL FIX
 *  Disable trace and the problem should not occur.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server 7.0 (all versions up to but not      *
   *                  including 7.0.0.19) and 8.0.0.0.            *
   ****************************************************************
   * PROBLEM DESCRIPTION: FileNotFoundException when enabling     *
   *                      com.ibm.config.eclipse.wtp=FINER.       *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Enabling the trace "com.ibm.config.eclipse.wtp=FINER"
   can cause a FileNotFoundException when installing an EJB 3.0
   (or higher) jar file which has no deployment descriptor
   (META-INF/ejb-jar.xml).  For example:
   
   java.io.FileNotFoundException: IWAE0065E Absolute path
   unknown
   
   The FileNotFoundException will be shown as wrapped
   by a DeploymentDescriptorLoadException:
   
   org.eclipse.jst.j2ee.commonarchivecore.internal.exception.
   DeploymentDescriptorLoadException: META-INF/ejb-jar.xml
   
   For this problem, the exception stack will show the class
   "TempZipFileLoadStrategyImpl" and the method "getAbsolutePath":
   
   org.eclipse.jst.j2ee.commonarchivecore.internal.strategy.
   TempZipFileLoadStrategyImpl.getAbsolutePath.
   
   The exception only occurs when the specified trace is
   enabled.  The exception is shown in server trace log files.
   
   The exception occurs because trace attempts to display an
   archive path value which is not always available.  When the
   archive path value is not available, a FileNotFoundException
   results.  That the archive path value is not available is not
   a problem, and no exception should be displayed.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Trace code was updated to avoid displaying the
   FileNotFoundException when the trace specification
   "com.ibm.config.eclipse.wtp=FINER" is enabled.
   
   APAR PM22226 is currently targeted for inclusion in
   Service Level (Fix Pack) 7.1.0.19 of WebSphere
   Application Server 7.0, and is currently targeted for
   inclusion in Service Level (Fix Pack) 8.0.0.1 of WebSphere
   Application Server 8.0.
   
   Please refer to URL:
   //www.ibm.com/support/docview.wss?rs=404&uid=swg27006970
   for Fix Pack availability.
