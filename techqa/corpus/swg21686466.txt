Gather Documentation  1. Stop the agent 
 2. Set RAS1 trace settings below 
 3. Reconfigure the agent 
 4. Recreate problem and take screen print of issue 
 5. Process <install path>/bin/tacmd pdcollect from the command line 
    

Send the pdcollect jar/tar file, screen prints, and all useful environmental information to the PMR 

ITCAM Agent Trace Settings 
 * Update the Agent environment file:
 * Windows 
 * For 32 bit agent: <install path>\TMAITM6\K<pc>ENV 
 * For 64-bit agent: <install path>\TMAITM6_x64\K<pc>ENV 
 * Unix/Linux 
 * <install path>/config/<pc>.ini 
   
   i5/OS
   For MC agent: KMCTMP/KMSPARM(KBBENV)
   Trace log will be generated in KMCTMP\KMCAGENT<nn>
   
   For MQ agent: KMQTMP/KMSPA<suffix>(KBBENV)
   Trace log will be generated in KMQTMP\KMQ<suffix><nn>
   
   Note: 
   <pc> is the two-character product code, i.e. mc, mq, m6 and qi.
   <suffix> is the suffix id, e.g. 00001, 00002.
   <nn> is the log number, i.e. 01, 02 and 03.
   
 * Set trace values for KBB_ RAS1 and KBB_RAS1_LOG 
 *  KBB_RAS1 set the following for detailed trace option
   For mc agent:
   KBB_RAS1=ERROR (UNIT:kcf ALL) (UNIT:kmc ALL)
   
   For mq agent:
   KBB_RAS1=ERROR (UNIT:kmq ALL) (UNIT:kra ALL)
   
   For qi agent:
   KBB_RAS1=ERROR (UNIT:kqia ALL) (UNIT:kqij ALL)
   
   For m6 agent:
   KBB_RAS1=ERROR (UNIT:km6 ALL) 
 *  KBB_RAS1_LOG set the following for more and larger log files
   COUNT=05 LIMIT=10 PRESERVE=1 MAXFILES=9
