PROBLEM(ABSTRACT)
 You run CICS Transaction Server for z/OS (CICS TS) and receive a loop of DFHAP0001 messages reporting abend 0C1 (abend0C1). In this case, the set of abends starts with a task receiving an OMEGAMON XE for CICS abend RLCP indicating too much CPU was used. You are using CA InterTest. 

CAUSE
CA-InterTest issues an EXEC CICS ABEND for the OMEGAMON abend that causes the abend 0C1. If Intertest had not been involved, the RLCP would have went through without any incident.

RESOLVING THE PROBLEM
[/support/docview.wss?uid=swg21202265&amp;aid=1]CA Technologies [http://www.ca.com/] Support recommends adding the various OMEGAMON for CICS abend codes (those beginning with RLxx) to the IN25ABEN module. Afterwards CA Intertest will not intercept the abend codes. In this case, the DFHAP0001 abend loops stopped after updating IN25ABEN.
