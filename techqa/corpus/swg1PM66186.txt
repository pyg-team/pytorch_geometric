LOCAL FIX
 *  A valid (alternate) solution, post migration is to follow the
   steps:
   
   Login to DMGR Integrated Solutions Console
   Under Applications > Application Types > WebSphere enterprise
   applications
   Select MigratedThemes Enterprise Application
   Select the Update button
   Select Replace the entire application
     Choose the Remote file system, providing the path to the
   MigratedThemes.ear from the Primary Node of the First Cluster
   migrated's &lt;PortalServer&gt;/installableApps directory
     Choose Fast Path and Specify binding to use : use new
   bindings
     Select default for remaining steps, confirming proper mappings
   for clusters.
   Now, Select all nodes from System administration &gt; Nodes
   view
   Select Synchronize
