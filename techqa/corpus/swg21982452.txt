Title: IBM Audit Notification service (ANServer) will not start up, the ODBC System DSN connectivity test fails and Sterling Gentran:Server for Microsoft Windows. - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 You are running a ODBC connectivity test to troubleshoot why ANServer.exe is not starting, and the connection fails with error, [Microsoft][ODBC SQL Server Driver][Shared Memory]SSL Security error. 

SYMPTOM
Running connectivity tests...
Attempting connection
[Microsoft][ODBC SQL Server Driver][Shared Memory]SSL Security error
TESTS FAILED!


CAUSE
The check box for 'Use Strong Encryption for Data' is marked

RESOLVING THE PROBLEM
It is quite acceptable to use the security feature of ODBC, Gentran software is not responsible for creating or maintaining. 


If unsure, Unchecked the 'Use Strong Encryption for Data' check box on the ODBC configuration and test again.
