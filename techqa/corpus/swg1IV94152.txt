APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Sybase agent SDA failed when agent installed on non-windows
   platform.
   This issue is caused because TEMA_SDA and ITM_MANIFEST_PATH was
   not set in environment.
   
   Detailed Re-creation Procedure:
   1. Install sybase agent on non-windows platform.
   2. Enable SDA support on TEMS, TEPS and agent machine.
   3. Restart the Sybase agent and check on ITM machine if support
   is seeded.
   
   Related Files and Output:
   SDA failed for Sybase agent with below error in agent logs
   Ignoring TEMA_SDA Configuration! Agent SDA package not found for
   PRODUCT "OY".
   
   Environment:
   All supported non-windows OS versions.
   Sybase server ASE versions 15.0, 15.7, 16.0.
   ITCAM for Applications Monitoring Agent for Sybase Server
   v06.20.01.00.
   
   
    
   
   

LOCAL FIX
 *  add  below variable to oy.ini file:
   
   
   export TEMA_SDA=Y
   
   restart the agent
   
   
    
   
   

PROBLEM SUMMARY
 *  Sybase agent SDA support seeding failed when agent installed on
   non-windows platforms.
   Issue is because the variables TEMA_SDA and ITM_MANIFEST_PATH
   are not set in the environment.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Added the required varibles with EXPORT in the oy.config.
   
   
    
   
   

TEMPORARY FIX
