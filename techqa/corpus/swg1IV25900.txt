ERROR DESCRIPTION
 *  product: Netcool/Impact 6.1.0
   
   abstract: Impact stops processing events :
             java.lang.OutOfMemoryError
   
   problem: Impact hung logs show error message like this:
   
    ERROR [ImpactServerManagedBean] Failed to
    register/unregister policy
    java.lang.OutOfMemoryError
   
   appears that the OutOfMemory is from a backup of events.
   When Impact process events it grabs a batch of events and
   process them. When the events are processed batch is
   checkpointed. Each batch can be run at the same time, but they
   have to be checkpointed in order and can't be checkpointed
   before the previous batch.
   The logs seem to show one batch is not finishing.
   This causes the hang. batches after this one can't be
   checkpointed.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * When processing events Impact server will stop processing    *
   * events and generate an OutOfMemory Exception                 *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * None                                                         *
   ****************************************************************
   The .state file in the impact/etc directory is now longer being
   updated.  This means that impact is no longer processing events.
   After a while, Impact will start having OutOfMemory errors.
