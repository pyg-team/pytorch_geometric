Title: IBM Configuration of Data Studio web console fails on systems with unsupported default locales - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 If the default locale of the system is not supported by Data Studio web console, the configuration of the product fails. 

SYMPTOM
The following message is displayed the command window:

An error has occurred. See log file "<DSWC install dir>\workspace\.metadata\.log"

The most recent log file will contain a message like the following example:

Caused by: java.util.MissingResourceException: Can't find bundle for base name com/ibm/im/dt/dca/dswc/resource/nls/dswc_customization, locale en_GB
at java.util.ResourceBundle.throwMissingResourceException (ResourceBundle.java:1510)
at java.util.ResourceBundle.getBundleImpl(ResourceBundle.java:1333)
at java.util.ResourceBundle.getBundle(ResourceBundle.java:807)


RESOLVING THE PROBLEM
Change the locale of the system to "en_US" before starting the configuration of the Data Studio web console.
