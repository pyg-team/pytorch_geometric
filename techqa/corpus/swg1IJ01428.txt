 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  OS and version:       Windows Server 2012 R2 Standard
   Product Version:      OMNIbus 8.1.0.10
   
   Ping runs and stops via Windows Services. When using Windows
   Services to stop the ping probe, it doesn't do a graceful
   shutdown - it crash and the error generate in Windows Event Log.
   
   
    
   
   

LOCAL FIX
 *  Testfix provided
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * Netcool/OMNIbus probes running as Windows Services on        *
   * Microsoft Windows.                                           *
   * Defect was observed on Windows Server 2012 R2 Standard.      *
   * Problem occurs in Netcool/OMNibus installations 8.1 fixpack  *
   * 4 to 8.1 fixpack 15.                                         *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * When a probe is installed as a Windows Services and is       *
   * stopped via the Service Controller it would occasionally     *
   * crash. Application Error 1000 or 1001 is subsequently listed *
   * in the Windows Event Log.                                    *

   * Conclusion.                                                  *
