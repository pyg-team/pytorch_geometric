APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  You use the CICS Explorer V5.1.0.1 or V5.1.1.0 right-click menu
   action "New from..." to create a new program definition for
   your CICS TS V4.1 region and the Program Definition wizard does
   not open.  The Error Log shows:
   
   Unhandled event loop exception
   
   
   com.ibm.cics.model.IllegalCICSAttributeException: the
   unsupported marker value has no external representation
   
   You can successfully use the right-click action "New..." to
   create a program definition for your CICS TS 4.1 region in
   either the CSD or EYUDREP in BAS.
   
   The problem occurs on the "New from..." action due to the
   JVMSERVER attribute changes in CICS TS 4.2 and above.
   
   
   
   Additional Symptom(s) Search Keyword(s): KIXREVWJB
   RTC 75464
   DFHCSD
   internalToExternal
   
   
    
   
   

LOCAL FIX
 *  Use the Right-click menu "New..." action instead of "New from.."
   
   
    
   
   

PROBLEM SUMMARY
 *  User chooses a CICS region of v4.1 or earlier, right-clicks on a
   PROGDEF and chooses 'New From...' from the context menu.
   UnhandledEventLoopException is registered in the error log and
   the wizard fails to open, with an exception recorded in the
   error log. This occurs because the PROGDEF creation wizard
   attempts to make use of the JVMSERVER attribute, which is not
   available in regions before 4.2.
   Additional Keywords: CICSTS32 CICSTS41
   
   
    
   
   

PROBLEM CONCLUSION
 *  CICS Explorer is changed so that it will not access the
   JVMSERVER value on regions earlier than CICS TS V4.2.
   .
   This fix will be made available in Version 5.1.1.1 of
   the CICS Explorer.
   For installation instructions please see:
   Ordering maintenance for the IBM CICS Explorer
   http://www.ibm.com/support/docview.wss?uid=swg21380083 [http://www.ibm.com/support/docview.wss?uid=swg21380083]
