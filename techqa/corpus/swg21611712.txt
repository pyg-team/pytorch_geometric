Title: IBM Users unable to login to FileNet WorkplaceXT or FileNet Enterprise Manager - United States

Text:
sufficient; required; Security:090304; authentication failed; Control Flag TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Unable to login to FileNET WorkplaceXT (WPXT) or FileNET Enterprise Manager (FEM) 

SYMPTOM
Cannot login to FileNET WorkplaceXT or FEM. 

Error: the user is not authenticated.


CAUSE
Change the authentication Control Flag from REQUIRED to SUFFICIENT in the Weblogic Console.

The Control Flag attribute is the Java Authentication and Authorization Service (JAAS) control flag that determines how the LoginModule for the WebLogic Authentication provider is used in the login sequence.

A SUFFICIENT value specifies this LoginModule need not succeed. If it succeeds, control returns to the application. If it fails and other Authentication providers are configured, authentication proceeds down the LoginModule list.


ENVIRONMENT
CE 5.1 / WorkplaceXT 1.1.5 / Linux 2.6.18 / Weblogic 10.3.3 / Oracle 11G

RESOLVING THE PROBLEM
 

 1. Go into the Weblogic Console
    
    
 2. Go to Configuration
    
    
 3. Go to Common
    
    
 4.  Set the JAAS Authentication control flag from REQUIRED to SUFFICIENT
    
    
 5. Restart the WorkplaceXT JVM
    
    
 6. Attempt to login into WorkplaceXT and FEM
