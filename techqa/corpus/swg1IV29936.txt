APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The exception in stored procedure happened, by example when proc
   called with wrong parameter type. After N exceptions (where N is
   number of "Maximum SQL Connection" of Datasource) any call
   return with timeout. From  above-stated we suppose then oracle
   connection locked in exception case.
   
   Impact lock connection only if stored procedure repeat
   exception.
   
   
    
   
   

LOCAL FIX
 *  L3 updated,  The OracleStoredProcedure in Impact is not
   invalidating the connection after a syntax error. May need code
   change to resolve the issue.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Cannot make Oracle stored procedure call after receiving     *
   * syntax error from a stored procedure call                    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   Impact was reusing a bad database connection after the syntax
   error in the stored procedure call
   
   
    
   
   

PROBLEM CONCLUSION
 *  The bad connection was removed so Impact will create a new, good
   connection to Oracle to use for subsequent jdbc calls. The fix
   for this APAR is contained in Impact5.1.1 FP2
