ERROR DESCRIPTION
 *  Error Description&#65496;
   Category search does not work in InfoSphere MDM Server for PIM
   new UI when search_ignorecase is set to Yes.
   
   Steps to reproduce:
   1.  Set search_ignorecase=Yes in
   $TOP/etc/default/common.properties
   
   2.  Load an item and click on 'Categories' tab to map
   categories to an item.
   
   3.  In Search String type *test* which should bring back
   categories with Test, test results.
   
   Note: Only certain categories are returned, for example
   categories with 'test' and 'Test' results are omitted.
   
   Expected behavior is that the search should return all results
   and treat the search as case insensitive.
   
   Keyword: invalid search
   
   
    
   
   

LOCAL FIX
 *  Local Fix&#65496;
   None
