Title: IBM Map test fails - javax.xml.soap.SOAPExeception - United States

Text:
map test; javax.xml.soap.SOAPException; INVALID_CHARACTER_ERR; invalid or illegal XML character; Interrupted Man TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Running map test and gets javax.xml.soap.SOAPException error. 

EDIDeenvelope using same map throws Interrupted Man, Service error.

INVALID_CHARACTER_ERR: An invalid or illegal XML character is specified 

SYMPTOM
Map Test error - javax.xml.soap.SOAPException error
EDIDeenvelope error - INVALID_CHARACTER_ERR: An invalid or illegal XML character is specified


CAUSE
Standard rule on Output side of map 
Update/Process Data
XPath = /ProcessData/TransactionSetControlNumber/text()

ENVIRONMENT
Map Editor, Standard or Extended rule Update/Process Data, XPATH to evaluate

DIAGNOSING THE PROBLEM
Open map, Print to File Extended Rules, and Record Details. Search for Standard or Extended rules and check that values are specified properly

RESOLVING THE PROBLEM
Right-click on field/element that contains the invalid XPATH value, select Properties, Standard Rule tab, in the Please enter the XPATH to evaluate option, place the node/field name in process data where you want this fields value to appear.
Example:
(entering a value of) TransactionNumber 
(will yield this in Process Data) <TransactionNumber>12345</TransactionNumber>
