Title: IBM IV46355: RAPID7 NEXPOSE SCANNER DISPLAYS AN ERROR WHEN THE SITE NAME PATTERN FIELD CONTAINS AN AMPERSAND (&) CHARACTER. - United States

ERROR DESCRIPTION
 *  On the Admin tab, Rapid7 NeXpose scanners that use the import
   type Import Site Data - Adhoc Report via API cannot support
   ampersand characters (&). The error message displays when a
   regular expression value in Site Name Pattern field contains an
   ampersand (&) character.
   
   
   The following error message is displayed in /var/log/qradar.log:
   
   [FATAL] [NOT:0000003000][IP address/- -] [-/- -]Unable to start
   ScanManager, no scans will be performed
   ScanManager could not initialize.
   scanManager.ScanManager.loadScannerConfig(ScanManager.java:8 94)
   scanManager.ScanManager.init(ScanManager.java:1011)
   scanManager.ScanManager.onInit(ScanManager.java:338)
   Naming.initializeNewComponent(Framew orksNaming.java:951)
   Naming.getApplicationScopedComponent (FrameworksNaming.java:931)
   Context.getSingletonInstance(Framework sContext.java:1109)
   ScannerDaemon.startScannerDaemon(ScannerDaemon.j ava:77)
   vis.scanManager.ScannerDaemon.access$000(ScannerDaemon.java:19)
   vis.scanManager.ScannerDaemon$1.run(ScannerDaemon.java:182)
   Caused by:
   javax.xml.bind.UnmarshalException
   - with linked exception:
   ParseException: The entity name must immediately follow the '&'
   in the entity reference.]
   
   
    
   
   

LOCAL FIX
 *  In the Rapid7 NeXpose scanner configuration, you can use a
   regular expression in the Site Name Pattern field that does not
   include an ampersand (&) character.
   
   
    
   
   

PROBLEM SUMMARY
 *  This issue has been resolved in QRadar SIEM 7.2.4.
