PROBLEM(ABSTRACT)
 This technote provides guidance for resolving the error "com.ibm.ws.exception.RuntimeWarning: com.ibm.ws.exception.ConfigurationWarning: Multiple ConfigurationWarning exceptions encountered" when starting the IBM WebSphere Portal v6.x server in IBM Rational Application Developer for WebSphere software v7.x. 

SYMPTOM

com.ibm.ws.exception.RuntimeWarning: com.ibm.ws.exception.ConfigurationWarning: Multiple ConfigurationWarning exceptions encountered

CAUSE
The message com.ibm.ws.exception.ConfigurationWarning: Multiple ConfigurationWarning exceptions encountered can be encountered in a number of situations; one being the Portlet Project WAR is already deployed to the Portal server and you are trying to deploy another Portlet with the same name or Context root.



RESOLVING THE PROBLEM
To resolve "com.ibm.ws.exception.ConfigurationWarning: Multiple ConfigurationWarning exceptions encountered", you can uninstall all other Portlet applications not needed from the WebSphere Application Server Admin Console. 

 

 1. Right-click on defined started Portalv6 server in Rational Application Developer v7 
     * Go to Add and Remove Projects... and remove any deployed EARs from the server by bringing the deployed applications from the "Configured Projects" list to the "Available Projects" list. Click OK when done.
       
       
    
    
 2. Right-click on defined started Portalv6 server in Rational Software Architect v7 and choose the option "Run Administrative Console" to run the WebSphere Application Server Admin console. 
     * Use your WebSphere Auth credentials to log in to the server.
       
       
    
    
 3. Click the Applications from the left side and go to Enterprise Applications.
    
    
 4. Scroll through the 6 or 7 pages of deployed application and make sure that all applications are running except the "DefaultApplication", "query" and "ivtApp" which will be in a Stopped status.
    
     * Uninstall all application that you have deployed and not needed and are visible. If they are "Started", stop and then uninstall by using the functionality provided in the above buttons in the Admin console.
       
       
    
    
 5. Restart the Portal v6 and make sure that you have no errors when starting the Portal Server.
    
    
 6. Right-click on defined started Portal v6 server in Rational Software Architect v7; go to Add and Remove Projects...
     * Add you application present in the "Available Projects" list to the "Configured Projects" list.
