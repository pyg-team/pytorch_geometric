Title: IBM Warning message ADMA0115W occurs when manually updating to Rational Asset Manager version 7.0.0.1 on WebSphere Application Server - United States

Text:
RN7.0.0.1; RAM; asset manager; 1270907 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When updating IBM® Rational® Asset Manager that was manually installed on WebSphere® Application Server from version 7.0 to version 7.0.0.1, you might see a number of ADMA0115W warning messages stating that resource assignments are not found within the scope of the module com.ibm.ram.repository.web.ws.was. 

SYMPTOM
When using the Administrative console to update Rational Asset Manger server application (named RAM1WebApplication) that was manually installed on WebSphere Application Server, you might see warnings similar to the following example when you click Finish in the update wizard: 


ADMA0115W: Resource assignment of name url/RAM_Persist and type java.net.URL, with JNDI name url/RAM_Persist is not found within scope of module com.ibm.ram.repository.web with URI com.ibm.ram.repository.web.war,WEB-INF/web.xml deployed to target WebSphere:cell=ramser1Node01Cell,node=ramser1Node01,server=server1.


RESOLVING THE PROBLEM
You can safely ignore these warnings; they are expected.
