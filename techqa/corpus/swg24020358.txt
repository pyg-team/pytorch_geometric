Title: IBM JR30316:  Instance creation fails because DBA credentials not available - United States

Text:
instance creation; failed; DBA; dbaPassword; dbaName DOWNLOADABLE FILES

ABSTRACT
 Create an IBM WebSphere Commerce instance using the Instance Creation Wizard. Instance creation fails with error stating the database administrator password was not enabled. 

DOWNLOAD DESCRIPTION
 


The problem occurs when: 

 * You select Use an existing DB2 database or Oracle tablespace in the database panel. 
 * You select Use staging server in the staging panel.


Creating a managed content staging instance fails when specifying an existing database because the DBA user information is blank. 
Use one of these solutions: 

 1. Perform the following steps:  1. Upgrade to WebSphere Commerce Version V6.0 Fix Pack 5 if not yet installed. 
     2. Apply APAR JR30316 [https://www.ibm.com/support/docview.wss?uid=swg1JR30316]. 
     3. Run the Instance Creation Wizard to create a WebSphere Commerce instance.
        
    
    
 2. Perform the following steps:  1. Run Commerce Instance Creation Wizard with Generate properties file only enabled on first panel. 
     2. Complete all panels; then click Finish. The wizard generates the following file: 
        WC_installdir/ [http://publib.boulder.ibm.com/infocenter/wchelp/v6r0m0/topic/com.ibm.commerce.base.doc/misc/mabhelp.htm]instances/instance_name [http://publib.boulder.ibm.com/infocenter/wchelp/v6r0m0/topic/com.ibm.commerce.base.doc/misc/mabhelp.htm]/properties/createInstance.properties 
     3. Edit the generated properties file and enter values for the following properties:

Post-instance creation steps (required for solution 1 only):  *  Check WC_installdir/ [http://publib.boulder.ibm.com/infocenter/wchelp/v6r0m0/topic/com.ibm.commerce.base.doc/misc/mabhelp.htm]instances/instance_name [http://publib.boulder.ibm.com/infocenter/wchelp/v6r0m0/topic/com.ibm.commerce.base.doc/misc/mabhelp.htm]/logs/createInstanceANT.log, for the following warning message : 
   
   WARNING: workspaces is not enabled due to missing dbaName or dbaPassword, please run config_ant ...
   Continue with steps 3 and 4 only if the above warning message is found. (optional) Navigate to WC_installdir/ [http://publib.boulder.ibm.com/infocenter/wchelp/v6r0m0/topic/com.ibm.commerce.base.doc/misc/mabhelp.htm]bin directory. (optional) Run the following command to enable workspaces: 
 *  
   config_ant ConfigureWorkspaceComponent -DinstanceName= instance_nameDdbaName= dbaname -DdbaPassword= dbaPassword -DdbUserPassword= dbUserPassword -DcmDBUserPwd= cmDbUserPassword
