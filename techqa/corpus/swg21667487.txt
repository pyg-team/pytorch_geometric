Title: IBM Authentication portal session timeout information for the Security Network Protection appliance - United States

Text:
Authentication; Portal; Session; timeout TECHNOTE (FAQ)

QUESTION
 How long, by default, does a session remain active when a user authenticates through the Security Network Protection (XGS) authentication portal? 

ANSWER
Users who authenticate through the portal are timed out after 24 hours. This timeout can be altered via the following advanced tuning parameters:

Configure Advanced Tuning Parameters: 

 1. Log into the XGS appliance Local Management Interface (LMI) 
 2. Go to: Management System Settings > Advance Tuning Parameters 
 3. Click New > Add the tuning parameter into the Advanced Tuning Parameter windows


Name: session.expirytimeout 
Values: 86400 (default is 24 hours) 
Description: Your description 

Name: session.idletimeout 
Values: 86400 (default is 24 hours) 
Description: Your description 

Notes:  * The session.expirytimeout parameter is the primary parameter that sets the timeout value. The session.idletimeout parameter is not used but should always be set to the same value as the session.expirytimeout parameter. So, users should always edit both parameters when modifying the timeout value. 
   


Passive authentication (e.g via TLES) uses the below advanced tuning parameter that is related to session timeout: 

Name: session.passivetimeout 
Values: 14400 (default is 4 hours) 
Description: Your description 

Note: The default value for the parameters is 14400 seconds (4 Hours) for users authenticated passively via Active Directory. It is not relevant for portal authentication.
