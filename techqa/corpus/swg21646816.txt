Title: IBM ITCAM Configuration issue with WebSphere XD - United States

Text:
CAMADALL; CAMADWXDDC WebSphere XD ITCAM configuration WASX8021E TECHNOTE (TROUBLESHOOTING)

THIS DOCUMENT APPLIES ONLY TO THE FOLLOWING LANGUAGE VERSION(S):
 US English 

PROBLEM(ABSTRACT)
 When configuring ITCAM DC receive WebSphere Authentication Error WASX8021E. 

SYMPTOM
When configuring the ITCAM DC get the following Webspher Authentication Error:
WASX8021E: the wsadmng with version "7.0.0.27" connected to an incompatible server 
or deployment manager with node version "6.1.1.5", which is not 
supported." 


CAUSE
wsadmin picks up the Virtual Enterprise version instead of the Application Server version.

DIAGNOSING THE PROBLEM
On starting wsadmin the following warning message appears: 
WASX8022E: Incompatible node version; exception information: 
com.ibm.ws.scripting.ScriptingException 
: WASX8021E: The wsadmin with version "x.x.x.x" connected to 
an incompatible server or deployment manager with node version 
"y.y.y.y", which is not supported. 
Where y.y.y.y is the incorrect version number for the server 
connected to.

RESOLVING THE PROBLEM
Apply WAS APAR PM82538 available in WebSphere fix pack 7.0.0.29 or higher. Refer to the Recommended Updates page for delivery information: 
http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
