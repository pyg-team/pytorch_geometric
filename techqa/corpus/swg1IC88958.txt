Title: IBM IC88958: MALFORMED HTTP REQUESTS MIGHT RESULT IN LOG MESSAGE WITH CORRUPTED MESSAGE INSERTS. - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When this error is logged as a result of malformed HTTP
   requests, the message insert values might be corrupted.
   
   0x80e005ea
   Malformed method headers (%s) for HTTP request, from URL %s.
   Explanation:
   
   The HTTP request method did not conform to one specified by RFC
   2616 or was prohibited by your configuration.
   Administrator response:
   
   Check your method against RFC 2616 and your configuration. The
   most commonly employed method is POST.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Corrected problem where HTTP services might log message with
   corrupted message inserts
   when  malformed HTTP request is received.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix will be available in 3.8.2.17.

   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
