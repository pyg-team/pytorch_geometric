ERROR DESCRIPTION
 *  Error Message: When processing system dumps containing corrupt
   data, the jextract tool sometimes produces an invalid XML file
   with a missing &lt;/frame&gt; closing tag in the &lt;stack&gt;
   stanza.
   This causes a parse error when trying to process the dump in
   tools like MAT, jdmpview and Dump Analyzer.
   .
   Stack Trace: N/A
   .
   Example bad XML block:
   &lt;vmthread id=&quot;0x3a57f300&quot; obj=&quot;0x165efe8&quot;
   native=&quot;0x12dc&quot; state=&quot;Running&quot; &gt;
   &lt;stack start=&quot;0x16b530&quot; end=&quot;0x16fd48&quot;
   &gt;
   &lt;frame method=&quot;0x3ab3a690&quot; pc=&quot;0x6&quot;
   arguments=&quot;0x16fa68&quot; /&gt;
   &lt;frame method=&quot;0x3ab3a780&quot;
   pc=&quot;0x43e8fc81&quot; line=&quot;587&quot; /&gt;

   processing stack 427668B0&lt;/error&gt;
   &lt;/stack&gt;
   &lt;/vmthread&gt;
   
   
    
   
   

LOCAL FIX
 *  Fix up the XML by hand.
   
   
    
   
   

PROBLEM SUMMARY
 *  If the Java stack is corrupt in the dump, the stack walker in
   jextract can crash. That crash was caught but not managed
   properly - so the closing &lt;/frame&gt; tag was left off.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The JVM has been modified to close the frame tag if the stack
   walker crashes during jextract.
