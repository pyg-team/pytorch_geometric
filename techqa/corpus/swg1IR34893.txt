Title: IBM IR34893: AE IR33957 FIX COMPLETION. BLANK LINES DO NOT SHOW BETWEEN MESSAGE LINES EVEN THOUGH EAMFBMSG IS DEFINED CORRECTLY. - United States

Text:
 
APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  EAMFBMSG is defined with multiple line feeds between message
   lines but those blank lines do not print on the 4610.
   .
   Use the following scenario to recreate:
   1)  Using ADXCSJ0L, I went into EAMFBMSG.DAT and changed the
       line OUR MONTHLY ADVANTAGE SWEEPSTAKES to
            MMM MONTHLY ADVANTAGE SWEEPSTAKES (this is only to
            validate the change for the blank line numbers took).
       Prior to the OUR MONTHLY ADVANTAGE SWEEPSTAKES, change the
       value ,1" to ,4".
   2)  Stop/start Checkout Support.
   3)  Reload options.
   4)  Sign on/off.
   5)  Ring a transaction using a Preferred Customer Card.
   6)  Total, tender.
   7)  Review the Customer Receipt.  You will find that the Message
       update from OUR to MMM was changed but there are no blank
       lines between YOU'RE AUTOMATICALLY ENTERED INTO
                     MMM MONTHLY ADVANTAGE SWEEPSTAKES.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  EAMFBMSG is defined with multiple line feeds between message
   lines but those blank lines do not print on the 4610.
   
   
    
   
   

PROBLEM CONCLUSION
 *  THE CODE WAS CHANGED TO ALLOW ANY NUMBER OF LINE
   FEEDS AS DEFINED IN EAMFBMSG.DAT FILE.  APAR
   IR33957 [http://www-01.ibm.com/support/docview.wss?uid=swg1IR33957] RESTRICTED THIS TO ONE (1) LINEFEED
   
   
    
   
   

TEMPORARY FIX
