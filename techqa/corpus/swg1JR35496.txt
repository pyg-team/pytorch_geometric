ERROR DESCRIPTION
 *  In adapter end-point configuration, the "event query for
   processing events" property always gets set back to Standard,
   even if you change the setting. For example, if
   I set the property to User-Defined(Dynamic), save it, exit,
   go back into assembly
   diagram, the "event query for processing events" property is set
   back to
   "Standard" always.
   
   
    
   
   

LOCAL FIX
 *  Need ifix for this issue. Customer running 6.2.0.1
   
   
    
   
   

PROBLEM SUMMARY
 *  On JDBC inbound endpoint configuration in WID edit environment,
   the value of "event query type for processing events" gets back
   to default "Standard" even if you change to
   "User-Defined(Dynamic)" and save it.
   
   
    
   
   

PROBLEM CONCLUSION
 *  6.2.0.3_IF04 fix it.The value of "event query type for
   processing events" is changed to the desired value after saving
   changes.
