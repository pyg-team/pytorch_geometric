APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  By default, databases are automatically enabled for automatic
   storage and the system generated tablespaces use automatic
   storage by default. SYSCATSPACE and USERSPACE1 use DMS under
   the covers with AUTORESIZE = YES (by default). Auto-resize can
   be turned off using ALTER TABLESPACE but when db2look is used
   to capture the DDL for a database, it is not showing if
   auto-resize is off
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  DB2LOOK NOT DISPLAYING AUTORESIZE INFORMATION FOR SYSTEM
   GENERATED TABLESPACES
   
   
    
   
   

PROBLEM CONCLUSION
 *  Problem is fixed in 9.5 Fix Pack 3 for Linux, UNIX, and Windows.
