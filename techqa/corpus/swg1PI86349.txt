APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  thankUsing Google Chrome, when a Add New widget is created on
   the Dashboard panel, on clicking the Add New button from the
   Dashboard panel, the Add New Wizard is displayed but the
   dashboard panel overlaps on the Add New Wizard and the panel
   moves with along with the cursor.
   
   
   Workaround:
   
   Left click anywhere on the screen for the panel to disappear.
   
   Prerequisites:
   
   OP 7.3.0.1
   
   A)	Set up profile
   1. Log into the OpenPages UI as OpenPagesAdministrator
   2. Navigate to Administration -> Users, Groups and Domains
   3. Click on OpenPagesAdministrator, click the Edit button,
   select OpenPages Modules 7.3.0 Master for Allowed   Profiles
   and Current Profile, then click Save
   
   B)	Create a new Panel for Add New widget
   1.	Login to OP UI as OpenPagesAdministrator using Google Chrome
   browser.
   2.	On the OpenPages Home Page, select the Dashboard tab
   
   3.	Click on Create New Panel
   4.	In the Create New Panel Wizard, enter a Panel Title, e.g. My
   Title
   5.	Add a new widget
   a.	Widget Type: Add New
   b.	Object Type: any Object Type, e.g. Control
   
   6.	Click on Create Panel
   
   
   
   Steps to Reproduce:
   
   1.	Using Google Chrome browser, login to OP UI as
   OpenPagesAdministrator.
   2.	On the OpenPages Home Page, select the Dashboard tab.
   3.	Click on the Add New button under the Panel titled My Title.
   4.	The Dashboard panel overlaps on the Add new Wizard and the
   panel moves along with the cursor.
   
   
   Expected Results:
   
   Add New Wizard should display without any overlap of the
   Dashboard Panel.
   
   Actual Results:
   
   Dashboard Panel overlaps on the Add New Wizard and the panel
   moves along with the cursor.
   
   Error Message:
   
   No error messages.
   
   
    
   
   

LOCAL FIX
 *  click anywhere on the screen for the panel to disappear.
