ERROR DESCRIPTION
 *  There are changes of experiencing hung threads in production
   environment due to changes delivered in 6053 ifix5 which makes
   use of a getting a child evidence in target case.  This call

 *  APAR PO03257 was included in release 6.0.5.4 iFix2
   Please see release Notes:
   http://www-01.ibm.com/support/docview.wss?uid=swg27041970 [http://www-01.ibm.com/support/docview.wss?uid=swg27041970]
   PO03257, CGISS-7231 - Hung Threads in production
   The changes delivered in 6.0.5.3 ifix5 to get the source case
   by making use of a child evidence in target case was done in a
   very inefficient way. To resolve this, changes were made to
   remove the call which was returning large number of records and
   made use of case header details of target case to get the
   source case from which evidence was shared or created.
