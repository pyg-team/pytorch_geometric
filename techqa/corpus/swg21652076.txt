Title: IBM Host On-Demand server ends with "Out of memory" error - United States

Text:
HOD; out of memory; out-of-memory; service manager TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Running IBM Rational Host On-Demand server on Microsoft Windows with more than 4G physical memory ends with OutOfMemory error if default memory is used with the java command to start the server. 

SYMPTOM
A java heap dump and java core dump is generated at the time of the failure and the Host On-Demand server ends. Sessions going through the redirector get disconnected. 


CAUSE
The OutOfMemory condition was a result of a known issue with IBM JRE 6 and fixed in IBM JRE 6 SR 12 and later. 

RESOLVING THE PROBLEM
To resolve the problem, you can do one of the following:


 * Upgrade Host On-Demand to version 11.0.8 or later as these versions use IBM JRE 6 SR 12 or higher.
   
   
 * Increase the xmx value which Host On-Demand uses by following these steps:
   
   Disclaimer This solution contains information about modifying the system registry. Before making any modifications to the Microsoft Registry Editor, it is strongly recommended that you make a backup of the existing registry. For more information describing how to back up the registry, refer to Microsoft Knowledge Base article 256986 [http://support.microsoft.com/kb/256986]
   
   
    1. Open the Windows registry editor (Start > Run > type regedit)
       
       
    2. Navigate to HKEY_LOCAL_MACHINE > SYSTEM > CurrentControlSet > services > IBMServiceManager > Parameter
       
       
    3. Right click on AppParameter
       
       
    4. Select Modify
       
       
    5. Back up the current value of the Parameter before making any modifications, save the value in a file
       
       
    6. Insert the parameter -Xmx512m as shown below. 
       
       Note: Do not copy and paste the value, type it in.
       
       -Xrs -Xmx512m -DFIPS=on -Djava.net.preferIPv4Stack=true -classpath
       "C:\Program Files\IBM\HostOnDemand\lib\sm.zip;C:\Program
       Files\IBM\HostOnDemand\lib\jt400.jar;C:\Program
       Files\IBM\HostOnDemand\lib\ibmjndi.jar;C:\Program
       Files\IBM\HostOnDemand\lib\jndi.jar;C:\Program
       Files\IBM\HostOnDemand\lib\jsdk.jar;C:\Program
       Files\IBM\HostOnDemand\lib\ods.jar;C:\Program
       Files\IBM\HostOnDemand\lib;C:\Program Files\IBM\HostOnDemand\bin;"
       com.ibm.eNetwork.HODUtil.services.admin.NCServiceManager "C:\Program
       Files\IBM\HostOnDemand" 
       
       
    7. Restart the HOD Service Manager for the above changes to go into effect.
