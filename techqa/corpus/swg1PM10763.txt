ERROR DESCRIPTION

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All z/OS users of the IBM WebSphere         *
   *                  Application Server V7.0 with CEA            *
   *                  (Communications Enabled Applications)       *
   *                  Feature Pack who create a profile with one  *
   *                  template type and augment with a different  *
   *                  template type.                              *
   ****************************************************************
   * PROBLEM DESCRIPTION: CEA profile augmentation failed on      *
   *                      z/OS when using a template type         *
   *                      different from the type used to         *
   *                      create the profile.                     *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When a z/OS user created a profile using one template type
   profileTemplates/cell/default, and subsequently augmented
   it using a different template type
   profileTemplates/CEA/ceafep.default, the augmentation failed
   (INSTCONFFAILED).
   
   
    
   
   

PROBLEM CONCLUSION
 *  With this fix, a z/OS user can now create a profile using
   profileTemplates/cell/default and then augment it successfully
   using profileTemplates/CEA/ceafep.default
   
   
    
   
   

TEMPORARY FIX
