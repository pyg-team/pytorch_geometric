Title: IBM Modeler adapter updates will not push down to the C&DS Deployment Manager client - United States

Text:
CADS CNDS Dep Man client deployment manager client windows modeler adapters updates plug-ins plugins pem update URL adapters pull down pulled down adapters won't come down adapters won't apply adapters won't install TECHNOTE (FAQ)

QUESTION
 Modeler adapter updates will not push down to the C&DS Deployment Manager client



As a result, the Deployment Manager client is unable to define a Modeler Server resource definition nor is the client able to drag a stream file on to the job canvas of a new job. 

ANSWER
Two possible causes are known to cause this problem:

1) Proxy server. If the client having the difficulty receiving the updates is set to use a proxy, temporarily disable the proxy settings or have an IT administrator put the C&DS Deployment Manager URL into an exception list

2) Check APAR PI76801:
http://www.ibm.com/support/entdocview.wss?uid=swg1PI76801 [http://www.ibm.com/support/entdocview.wss?uid=swg1PI76801]

Which describes the following two workarounds:

A) * Delete the folder
<C&DS_Server_Install>\\components\pemplugins\update\com.spss.mgmt.modeler.feature_8.0.0.0_site,
* Only leave the com.spss.mgmt.modeler.feature_8.0.0.1_site

B) * Comment out the line: '<childlocation="./com.spss.mgmt.modeler.feature_8.0.0.0_site"/>' in
compositeArtifacts.xml and compositeContent.xml.

For both workarounds, you can restart the SPSS C&DS Deployment Manager thick client application without restarting the SPSS C&DS Server. The Deployment Manager client should get the plugin updates correctly. 

If none of the above two items resolves the problem, then as an alternative would be to apply the updates to the Deployment Manager client manually. Here are the steps to do that: 


======================================================
1. Create a new folder named 'plugins' under 

"C:\Users\<UserAccount>\AppData\Roaming\SPSSInc\Deployment Manager"

Note, if the plugins folder already exists in that location, then move to Step 2

2. Navigate to the "<C&DS Server Install>\components\pemplugins\update\com.spss.mgmt.modeler.feature_8.1.0.0_site\plugins" directory and copy the jar files found there to the plugins directory from step 1 above. I.E. "C:\Users\<UserAccount>\AppData\Roaming\SPSSInc\Deployment Manager\plugins"

3. Using a text editor, open the "bundles.info" file found here by default:

"C:\Program Files\IBM\SPSS\Deployment\8.0\DeploymentManager\configuration\org.eclipse.equinox.simpleconfigurator\bundles.info"

Then add the 7 lines listed below based on the version of C&DS and Modeler you are using. Note also that you will need to modify the file path based on the actual environment, meaning change the <UserAccount> with the end-user's actual Windows login:
