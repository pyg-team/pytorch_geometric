APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Problem Description:  When the Watchdog code which is part of
   the OS agent is monitoring the WebSphere MQ Monitoring Agent
   (kmq) and the instance name contains a dot ("."), the agent
   displays the status of STOPPED and  Process ID=0 even when the
   instance is managed by Watchdog and the instance is running.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  When the Watchdog code which is part of the Monitoring Agent for
   Windows is monitoring the WebSphere MQ Monitoring Agent (kmq)
   and the instance name contains a dot ("."), the status for that
   instance is displayed as STOPPED and Process ID=0 even when the
   instance is running.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code was changed to correctly parse the instance name when
   it contains a dot (".").
   
   The fix for this APAR is contained in the following maintenance
   packages:

     | fix pack | 6.2.3-TIV-ITM-FP0004
     | fix pack | 6.3.0-TIV-ITM-FP0002
