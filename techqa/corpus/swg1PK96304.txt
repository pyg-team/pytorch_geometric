APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The Common Event Infrastructure (CEI) configuration
   administrative task does not support automatically
   configuring the DB2 DataServer driver based DataSource.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  IBM WebSphere Application Server V7.0 users *
   *                  who configure Common Event                  *
   *                  Infrastructure(CEI) with a DB2 database.    *
   ****************************************************************
   * PROBLEM DESCRIPTION: Configuring the Common Event            *
   *                      Infrastructure with a DB2 database      *
   *                      does not support data source based on   *
   *                      the IBM Data Server Driver for JDBC.    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The configuration process of Common Event Infrastructure with
   DB2 database creates the WebSphere Application Server data
   source based on the DB2 Universal JDBC Driver Provider (XA)
   driver version (considered as the default). Common Event
   Infrastructure configuration supports the data source to be
   configured with a JDBC Provider using IBM Data Server Driver
   as well. If you need to base your data source on the IBM Data
   Server Driver for JDBC, you will be able to do so by using a
   manual procedure via the Administrative Console. The
   automatic configuration process for Common event
   Infrastructure should support this case as well.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The configuration process with DB2 database has been changed
   by adding to the configEventServiceDB2DB configuration
   Admin Task a new optional parameter -jdbcDriverVersion that
   will accept the following possible/acceptable values 3.0 /
   4.0. The usage of 3.0 values does configure the JDBC provider
   based on the DB2 Universal JDBC Driver Provider (XA) driver
   version whereas using the 4.0 value will produce the JDBC
   provider based on the IBM Data Server Driver for JDBC version.
   The introduced behavior is:
   a. -jdbcDriverVersion not specified in the AdminTask - The
   configuration will be performed using 'DB2 Universal JDBC
   Driver Provider (XA)', therefore 3.0 driver version.
   b. -jdbcDriverVersion specified but no value supplied - Will
   raise an exception as -jdbcDriverVersion needs a parameter,
   then if the user go so far as to use the parameter but does
   NOT supply a value, we give an error stating like in the
   following example "CEIIN0739E No value was specified for the
   jdbcDriverVersion parameter. &#175;   &#189;
   c. -jdbcDriverVersion specified with the '3.0' value supplied
   - The configuration will be performed using ' DB2 Universal
   JDBC Driver Provider (XA)', therefore 3.0 driver version.
   d. -jdbcDriverVersion specified with the '4.0' value supplied
   - The configuration will be performed using 'DB2 Using IBM JCC
   Driver (XA)', therefore based on the provider that supports
   JDBC 4.0 specification using the IBM Data Server Driver for
   JDBC and SQLJ 4.0 driver version.
   e. -jdbcDriverVersion specified with a value OTHER than 3.0 or
   4.0 - Will raise an exception like in the following example:
   "CEIIN0701E The parameter jdbcDriverVersion contains the value
   <wrong_value>, which is not valid. Valid values are: 3.0, 4.0."
   
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 7.0.0.9.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
