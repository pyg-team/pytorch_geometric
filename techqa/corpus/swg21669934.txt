QUESTION
 InfoSphere DataStage parallel job drops rows emitting next warning. What's the meaning of the message and how to eliminate it?

Item #: 42
Event ID: 219
Timestamp: 2014-04-10 16:53:12
Type: Warning
User Name: dsadm
Message Id: IIS-DSEE-TBLD-00008
Message: Transformer_1,0: Field 'APT_Decimal::ErrorBase: From: the source decimal has even precision, but non-zero in the leading nybble, or is too large for the destination decimal..' from input dataset '0' is NULL. Record dropped.


CAUSE
IIS-DSEE-TBLD-00008 message appears when there is decimal data conversion error.

ANSWER
Meaning of 

 * "the source decimal has even precision, but non-zero in the leading nybble" message

 * "is too large for the destination decimal..' from input dataset '0' is NULL" message

To find out what value causes the conversion error, attach a reject link to the problem Transformer stage, and review decimal column values in the rejected rows. 

If the decimal values in rejected rows are correct, adjust the precision and scale of the decimal column in Columns tab of the Transformer. 
If the value in reject row is corrupt, clean it up in the data source.
