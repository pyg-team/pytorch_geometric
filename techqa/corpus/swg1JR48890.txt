APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Process Portal performs too many DB operations when checking
   user authorization for process instances.
   
   In Process Portal, the following behavior may occur
   intermittently, for a given task:
   
   1) The task cannot be launched (clicking on the task only shows
   the task details and business data).
   
   2) The Actions dropdown menu for the task is not displayed.
   
   Please notice that this iFix is a backport iFix of JR44650 [http://www-01.ibm.com/support/docview.wss?uid=swg1JR44650]
   
   
    
   
   

LOCAL FIX
 *  Refreshing the browser page has been observed to resolve this
   issue when observed, but not every time.
   
   
    
   
   

PROBLEM SUMMARY
 *  Excessive DB trips to check whether user can view instance in
   process
   
   PROBLEM DETAILED DESCRIPTION:
   Process Portal performs too many DB operations when checking
   user authorization for process instances.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Reduce API calls that lead to DB access in the code path of the
   process instance actions.
