Text:
GUI; UI; simplesrm; logon; login; unable; fail; failing TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)

CAUSE
The errors indicate depletion of transaction log space.


RESOLVING THE PROBLEM
Change LOGSECOND db2 parameter using command: 


db2 update db cfg for <dbname> using logsecond <new_value> 

where <new_value> will be the new value you want (for example double the current value you have) 

You can find the current value for the LOGSECOND parameter by reviewing the output from the GET DATABASE CONFIGURATION command - look at the Infocenter: 

http://publib.boulder.ibm.com/infocenter/db2luw/v9/topic/com.ibm.db2.udb.admin.doc/doc/r0001954.htm [http://publib.boulder.ibm.com/infocenter/db2luw/v9/topic/com.ibm.db2.udb.admin.doc/doc/r0001954.htm]

or from the Configure Database window of the DB2 Control Center.
