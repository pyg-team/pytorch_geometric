APAR STATUS
 * CLOSED AS DOCUMENTATION ERROR.
    
   
   

ERROR DESCRIPTION
 *  One needs to remove a user called "tipadmin" on their Omnibus,
   prior to installing Impact 6.1, if you're using Object Server
   authentication for the "user registry" step of the install.
   
   
   
   Failure to do so resutls in following, within the
   wsadmin.traceout (from logs.zip), after the install fails:
   -------------------------------------------
   WASX7093I: Issuing message: "WASX7246E: Cannot establish "SOAP"
   connection to host "localhost" because of an authentication
   failure. Ensure that user and password are correct on the
   command line or in a properties file. Exception message (if
   any): "ADMN0022E: Access is denied for the getProcessType
   operation on Server MBean because of insufficient or empty
   credentials.""
   WASX7120E: Diagnostic information from exception with text
   "javax.management.JMRuntimeException: ADMN0022E: Access is
   denied for the getProcessType operation on Server MBean because
   of insufficient or empty credentials." follows:
   
    javax.management.JMRuntimeException: ADMN0022E: Access is
   denied for the getProcessType operation on Server MBean because
   of insufficient or empty credentials.
   -------------------------------------------
   
   
    
   
   

LOCAL FIX
 *  One needs to remove the "tipadmin" user on the Object Server
   first, if they're wanting to use Object Server authentication
   for their "user registry", during the time of installation.
   
   
    
   
   

PROBLEM SUMMARY
 *  One needs to remove a user called "tipadmin" on their Omnibus,
   prior to installing Impact 6.1, if you're using Object Server
   authentication for the "user registry" step of the install.
