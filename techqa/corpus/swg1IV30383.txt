ERROR DESCRIPTION
 *  When 5.1.1 is configured to authenticate to ObjectServer and
   ObjectServer uses SSL, then Impact is not able to make any
   connections to that ObjectServer, even to read events.
   
   This occurs when a user other than admin is used for the task
   and actioned in the script:
   
   $NCHOME/etc/tivoli-vmm4ncos/bin/install-vmm4ncos.sh.
   
   
   This copies the jconn3.jar to the lib/ext directory since the
   classes within it are used object server authentication.
   Unfortunately those classes cause a problem with the datasource
   when SSL is in place.
   
   
    
   
   

LOCAL FIX
 *  None yet.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * SSL authentication to Omnibus in 511 does not work. Also, if *
   * it is configured, Impact cannot make any connection to       *
   * ObjectServer via SSL.                                        *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   ObjectServer plugin used by Impact to authenticate with VMM did
   not handle SSL properly.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Modified ObjectServer plugin to implement SSL connection to
   ObjectServer in similar way to how mainline&#160;Impact code does so.
