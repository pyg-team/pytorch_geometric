ERROR DESCRIPTION
 *  This issue is reproduced when a solution has a custom view with
   a Property List container.  String properties when added inside
   the Property List container and configured to TextArea will
   reproduce this issue when running Case Client via Internet
   Explorer v11.  This issue is not reproducible with other
   browsers, such as Firefox or Chrome.  This issue is also not
   reproducible where the Property List container is not used.  It
   is
   also not reproducible with other containers.
   
   Steps to reproduce:
   1. Create a solution with a "single value" string property.
   2. Create a case type and add the string property to the case
   type.
   3. Add a customized view in this case type,
   4. set the Default view to the customized view
   5. Edit the view
     a. add a property list to the view
     b. add the string property inside the property list container
     c. set the Editor Settings for the string property to "Text
   area"
   6. Save and close the solution; deploy the solution
   7. Via IE 11, Logon to Case Client, and add a new case
   8. In the TextArea, enter text and then press the carriage
   return
   
   Expected results:
   Two lines are created in the text area.
   
   Actual result:
   There is only 1 line in the text area
   
   This issue is not reproducible with ICM 5.2.1 FP002.  It is
   reproducible with ICM 5.2.1 FP003 and later releases.
   
   
    
   
   

LOCAL FIX
 *  Possible workarounds:
   1. Using a different browser such as Chrome or Firefox
   2. Avoid using the property list container
   3. Issue is not reproducible using other container types
