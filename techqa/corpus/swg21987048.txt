CAUSE
From IDA 9.1.3, the IDA install only includes packaging for Windows environment. Previously, packages for all other environments, like Linux, would also be included in same package.

ENVIRONMENT
Windows.

DIAGNOSING THE PROBLEM
The files that are missing are not windows but Linux, SPARC, AIX, Mac, HP, etc. They are not windows files.

RESOLVING THE PROBLEM
In order to package IDA using PU, the command line PUCL.exe must be used. The GUI PU will expect to package IDA for all platforms it runs on, but in Version 9.1.3, IDA now packages these separately in different install images.





When using PUCL.exe, you provide the platform for windows pointing to -platform os=win32. 

(Note, even if the install is 64 bit, the win32 option will package the image correctly) 

To package IDA 9.1.3, the line command PUCL.exe must be run:

PUCL.exe copy com.ibm.rational.data.architect.v75_9.1.3.20150901_1548 
-platform os=win32 -repositories C:\installida9.1.3\disk1 
-target C:\ida_repository -acceptLicense 

where c:\install9.1.3 is directory where you uncompressed the IDA files downloaded.
and c:\ida_repository is your new repository. 

 

To list what is in the repository Run
pucl.exe listAvailablePackages -repositories C:\ida_repository 


Or open IBM Installation Manager, and add the repository to the Files - Preferences Panel. 

PLEASE NOTE: 

If 64 bit IDA is to be packaged ensure that you install PU using a 64 bit Installation Manager which is tolerant to 32 bit packages. (1.8.5 is tolerant) Do not install the 32 bit PU using the Installation manager that ships with it unless you are packaging 32 bit IDA. There is no 64 bit PU for Windows. 

Also make sure the SDPShared directory chosen does not point to a C:\Program Files (x86)\... directory if using 64 bit IDA. 

Other packages can then be added to the repository, by using the Packaging Utility GUI.
