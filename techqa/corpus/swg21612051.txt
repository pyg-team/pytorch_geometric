PROBLEM(ABSTRACT)
 You receive message DFHAP0001 reporting abend code 0C1/AKEA (abend0C4 abendAKEA) many times for different application modules running in CICS Transaction server for z/OS (CICS TS). The common factor with these abending modules is that they were all compiled with CA Optimizer. 

SYMPTOM
DFHAP0001 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/DFHAP/DFHAP0001.html] An abend (code 0C1/AKEA) has occurred at offset X'nnnn' in module mmmmm


CAUSE
CA Optimizer did not initialize properly. Message CAPC040I was issued.


DIAGNOSING THE PROBLEM
Reviewing a dump at the time of the error you will find the PSW points to 2 bytes of zeros, followed by an eye catcher of "CAOCEMCK". Here is this storage in browse mode:

0000C3C1 D6C3C5D4 C3D258F0 F01007FF *..CAOCEMCK.00...*

RESOLVING THE PROBLEM
Verify that CA Optimizer initialized correctly.
