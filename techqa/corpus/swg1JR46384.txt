PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  Users deploying SCA modules that have       *
   *                  JAX-RPC WS bindings may see deployment      *
   *                  failures.                                   *
   ****************************************************************
   * PROBLEM DESCRIPTION: Users deploying an SCA module may see   *
   *                      a org.xml.sax.SAXParseException if      *
   *                      the module contains a JAX-RPC WS        *
   *                      export binding.                         *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   User deploys an SCA module with JAX-RPC WS binding results in
   deployment failure with SAXParseException.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The JAX-RPC WS binding deployment code is fixed by using
   another related WebSphere WCCM API method that serializes out
   the service-qname correctly.
   
   Install/Dependency information is available in the readme.txt
   on FixCentral. When the user goes to the download files page
   in FixCentral, the readme will be available with the fix
   package as a separate download.
