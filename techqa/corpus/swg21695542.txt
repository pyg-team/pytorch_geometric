Title: IBM No data when monitoring 32bit application using 64bit agent - United States

Text:
no data; gksi; koq; itcam; itm TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 OQ agent does not collect data when running 64bit agent to monitor 32bit MSSQL server 

CAUSE
Agent is a 64bit process (WIX64) 

"OQ","KOQ(64-bit) CMA/Monitoring Agent for Microsoft SQL 
Server","WIX64","063100000","32681","KOQ64CMA.ver","0" 


Registry shows: 

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\eventlog\Application\MSSQL$TRACKIT] 
"CategoryCount"=dword:00000008 
"CategoryMessageFile"="C:\\Program Files (x86)\\Microsoft SQL 
Server\\MSSQL.1\\MSSQL\\Binn\\Resources\\1033\\sqlevn70.rll" 
"EventMessageFile"="C:\\Program Files (x86)\\Microsoft SQL Server\\MSSQL.1\\MSSQL\\Binn\\Resources\\1033\\sqlevn70.rll" 
"TypesSupported"=dword:000000ff 

This indicates MS SQL Server is instead a 32bit process. 


RESOLVING THE PROBLEM
Uninstall 64bit agent and install 32bit version. 

 ITM Agent and monitored application must be at same bit level.
