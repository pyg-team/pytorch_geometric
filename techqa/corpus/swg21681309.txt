Title: IBM Netool WebGUI AEL gives INVALID_PARAM_ERROR - United States

Text:
WebGUI; AEL; INVALID_PARAM_ERROR; HEMJS0423E TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Opening an AEL in OMNIbus WebGUI results in an empty event list and a yellow triangle with INVALID_PARAM_ERROR 

SYMPTOM
The WebGUI log may show below errors at the time issue occurred. The location of log can be found in below technote: 

http://www-01.ibm.com/support/docview.wss?uid=swg21502736 [http://www-01.ibm.com/support/docview.wss?uid=swg21502736]

[2014-08-07T14:28:59]:SEVERE:HEMJS0423E:[WebContainer : 16]:No DataSources currently available. Webtop service unavailable.
[2014-08-07T14:28:59]:FINER:HEMTrace:[WebContainer : 16]:doRedirectURL endofurl /Unavailable/?code=datasource real path /opt/IBM/tivoli/tipv2/profiles/TIPProfile/installedApps/TIPCell/isc.ear/OMNIbusWebGUI.war/Unavailable/?code=datasource is file false

The Object Server may show below errors at the same time when you try to open AEL's:

2014-08-07T14:29:10: Error: E-OBX-102-023: Failed to authenticate user root. (-3600:Denied)
2014-08-07T14:29:30: Error: E-OBX-102-057: User root@nc9053115058.tivlab.austin.ibm failed to login: Denied



RESOLVING THE PROBLEM
This problem can happen if you are using "&" in the password for the Object server. If this is the case then remove it from the password. After this update password in the ncwDataSourceDefinitions.xml file. Restart TIP/WebGUI. It should fix this issue. Same applies to other XML reserve characters (i.e. <, >, &, ' and "). 

Note: If password must contain "&" or any other special characters then you can encrypt the password and then use encrypted password in ncwDataSourceDefinitions.xml file. In this case you need to set encrypted="true" in ncwDataSourceDefinitions.xml file. 

Sample command to encrypt password is:
$ nco_pa_crypt 'password&1'
