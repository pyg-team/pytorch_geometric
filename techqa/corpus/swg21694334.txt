Title: IBM Config Editor crashes producing core dumps. - United States

Text:
SDI Security Directory Integrator crash coredump Java TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When using the SDI 7.2 Config Editor, the Config Editor crashes.

A javacore file will be created that shows information like:

CAUSE
The Config Editor crashing, is a result of an issue within Eclipse.

RESOLVING THE PROBLEM
To resolve the issue, the ibmditk file will need to be modified. In the <SDI Install Directory>/ibmditk (.bat for Windows and .sh for Linux/AIX), the following line must be edited. Add the parameter in BOLD to the line. 

---
$TDI_HOME_DIR/ce/eclipsece/miadmin" "$@" $nosplash -vm "$TDI_JAVAW_PROGRAM" -vmargs -Dorg.eclipse.swt.internal.gtk.cairoGraphics=false -Dcom.ibm.di.loader.IDILoader.path="$TDI_HOME_DIR"
---

By adding-Dorg.eclipse.swt.internal.gtk.cairoGraphics=false the Config Editor behaves properly without crashing.
