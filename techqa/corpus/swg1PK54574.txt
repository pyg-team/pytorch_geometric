Title: IBM PK54574: CENTRALIZED INSTALLATION MANAGER (CIM) IS UNABLE TO START THE UPDATEINSTALLER INSTALL PROCESS WITH NON-ROOT AUTHORITY - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After  (CIM)  Centralized installation manager pushes the
   (UPDI) UpdateInstaller image to a remote node,  CIM will start
   the UPDI install process.
   
   However if a non-root install is attempted, the install
   process will fail with the  following logged to the
   UPDI log.txt file:
   
   Install, com.ibm.ws.install.ni.ismp.actions.
   ISMPWarningDialogAction, wrn, INSTCONFFAILED :
   NONROOT_INSTALL_DISALLOWED : Set the non-root installation
   allowed setting to "true" in the response file before
   installing.
   
   
    
   
   

LOCAL FIX
 *  Code fix
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All users of WebSphere Extended Deployment   *
   *                 using the centralized installation manager   *
   ****************************************************************
   * PROBLEM DESCRIPTION: Centralized Installation Manager is     *
   *                      unable to start the UpdateInstaller     *
   *                      install process with non-root authority *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   After (CIM) centralized installation manager pushes the
   (UPDI) UpdateInstaller image to a remote node,  CIM will start
   the UPDI install process.
   
   However if a non-root install is attempted, the install
   process will fail with the  following logged to the
   UPDI log.txt file:
   
   Install, com.ibm.ws.install.ni.ismp.actions.
   ISMPWarningDialogAction, wrn, INSTCONFFAILED :
   NONROOT_INSTALL_DISALLOWED : Set the non-root installation
   allowed setting to "true" in the response file before
   installing.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The descriptor used to install UPDI has been updated to
   include the allowNonRootSilentInstall="true" option.
   
   
    
   
   

TEMPORARY FIX
