Text:
TIPL2; IAGLOBAL_INVALID_TIP_LOCATION; install error TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Tivoli Integrated Portal Fix Pack installation fails with error "IAGLOBAL_INVALID_TIP_LOCATION"

IAGLOBAL_INVALID_TIP_LOCATION: the installer could not locate the existing installation of the Tivoli Integrated Portal or the found installation is not a correct one. 
? The problem can be caused by a difference between bitness of the installer and the existing Tivoli Integrated Portal installation. 

Check the Tivoli Integrated Portal installation path that is defined in your response file. Use an uppercase letter to denote a disk drive.Windows 
Windows Use \\ as the path separators and do not leave separator at the end of your path. 


CAUSE
Incorrect path defined in sample_response.txt file

The value for IAGLOBAL_TIP_HOME path contains a slash ( / ) at the end, which causes the inventory step to fail,because the path registered in DE database as well, and being compared against.

<parameter name="IAGLOBAL_TIP_HOME" 
value="/usr/IBM/WebSphere/AppServer/"/> 


DIAGNOSING THE PROBLEM
Check the sample_response.txt file for what value/path is defined for "IAGLOBAL_TIP_HOME". 

 

Also, the TIPInstall_Inventory.log may contain an entry like this: 

[processReq] Invoking DEProcessReq CHECK... 
[echo] TIP2200 found: false 

Which is a bit misleading in this case. 


RESOLVING THE PROBLEM
Make sure the value for IAGLOBAL_TIP_HOME in the sample_response.txt file is correct and has no / at the end. 

Example: 

Good entry: IAGLOBAL_TIP_HOME=/usr/IBM/WebSphere/AppServer 

Bad entry: IAGLOBAL_TIP_HOME=/usr/IBM/WebSphere/AppServer/
