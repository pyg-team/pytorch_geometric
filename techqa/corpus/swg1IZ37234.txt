Title: IBM IZ37234: SECONDARY AUTHENTICTION FAILS ON NOKIA DEVICES. - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  On Nokia devices running the 6.1.2 release of the Mobility
   Client logins may fail when the Connection Manager is configured
   for 2 factor authentication such as LDAP-BIND or RADIUS.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All Nokia device users.  Nokia devices are   *
   *                 based on the Symbian Operating System.       *
   ****************************************************************
   * PROBLEM DESCRIPTION: During a login, when secondary          *
   *                      authentication and force password       *
   *                      change were requested the client would  *
   *                      hang.                                   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   During a login, when secondary authentication and force password
   change were requested the client would hang.
   
   
    
   
   

PROBLEM CONCLUSION
 *  There was an error creating the second worker thread that was
   causing the hang.  That error was corrected.  The fix for this
   problem is available from the Lotus Mobile Connect Level 2
   support center via a TESTFIX until such time as a new release
   of the product is made available.  This fix will be included
   in any release after 6.1.2.
   
   
    
   
   

TEMPORARY FIX
