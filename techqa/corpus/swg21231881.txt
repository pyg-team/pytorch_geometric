Title: IBM PLGC0022W: No definition is found for the specified server. Deployment of resource manager fails on WebSphere Application Server V6 Network Deployment (ND) when one or multiple web servers are defined in the deployment manager. - United States

Text:
Content Resource Manager WebSphere Application Server ND network deployment HTTP Server TECHNOTE (FAQ)

QUESTION
 Deployment of the resource manager fails when one or multiple web servers are defined in the deployment manager.

CAUSE
The plugin generation for the web servers defined in the WebSphere® Application Server fails due to an invalid configuration directory path used in calling the plugin generator.

ANSWER
There are two workarounds possible after applying the patch for DB2® Content Manager installation on WebSphere Application Server Network Deployment (see the link in the Related information section for more information about the patch): 

A) Remove the web server definitions from the dmgr and add them back later after running the DB2 Content Manager V8.3 installation program.

After the resource manager application is deployed, you need to remap the web modules of each application to the appropriate web servers and regenerate the plugin information for HTTP Server using the GenPluginCfg.sh utility in the "bin" directory of WebSphere Application Server.

B) Edit in the DB2 Content Manager installation patched files "icmrm.jacl" under <CM_Install_Files>/CM83/db2cmv8/config/icmrm.jacl

Search for the plugin generation line generation:

$AdminControl invoke $pluginCfg generate "{$configRoot} $cellName $nodeName $serverName true"

Comment out the line by adding a number sign (#) at the beginning.
Run the DB2 Content Manager installation again.

After the resource manager application is deployed, regenerate the plugin information for HTTP Server using the administration console or using the GenPluginCfg.sh/bat utility in the "bin" directory of WebSphere Application Server.
