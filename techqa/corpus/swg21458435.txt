RESOLVING THE PROBLEM
 

 1. CC_EXT_JNI_DIRS is no longer being used. CLASSPATH is being used. CLASSPATH will be set automatically during the installation of Salesforce Pack 1.5. 
 2. There is no longer a client side installation for Salesforce Pack 1.5 with Information Server 8.5. This means the installation of Salesforce Pack is only done on the Information Server 8.5 Server machine. 
 3. Once Salesforce Pack 1.5 is installed, you will have to manually change the CLASSPATH environment variable on the IBM InfoSphere DataStage server as follows:  * On Unix or Linux
       Edit the value for the CLASSPATH environment variable in the dsenv file from /apps/etc/com.ibm.ws.security.crypto.jar to /eclipse/plugins/com.ibm.isf.client_8.5.0.0/com.ibm.ws.security.crypto.jar. 
     * On Microsoft Windows
       Edit the value for the CLASSPATH environment variable in the System Properties from \apps\etc\com.ibm.ws.security.crypto.jar to \eclipse\plugins\com.ibm.isf.client_8.5.0.0\com.ibm.ws.security.crypto.jar. 
    
    
 4. You also need to remove the following two environment variables:  * CC_SALESFORCE_MSG_LEVEL 
     * CC_JNI_EXT_DIRS
