Title: IBM How to configure IBM Rational Performance Tester to use PVU licensing - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How do I configure IBM Rational Performance Tester (RPT) to support PVU licensing? 

ANSWER

PVU licensing is an alternative to using the IBM Rational Performance Tester (RPT) floating license with Virtual User licenses. 

Specific installation options and additional configuration steps required for RPT to use PVU licensing. All screen shots are from RPT 8.7 on Windows but the steps will also apply to previous versions of RPT. 

Installation options 

1. Install RPT with the PVU option 

Note: if RPT is already install use Installation Manager to modify RPT 

[/support/docview.wss?uid=swg21902683&aid=1] [/support/docview.wss?uid=swg21902683&aid=1] 

2. Install the agent with the IBM Rational Performance Test Server (PVU mode) - for use with IBM Rational Test Workbench option 

[/support/docview.wss?uid=swg21902683&aid=2] [/support/docview.wss?uid=swg21902683&aid=2] 

Note: if agent is already installed or if the mode used at installation time is not known then uninstall the agent and reinstall it. This is the only guaranteed way to know if the agent has been installed with the PVU option. 

Apply the license via activation kit 

If using authorized user license keys then follow these steps to apply the activation kits. 

Applying the license for RPT to use PVU is an extra step. In addition to applying the license to the RPT Workbench itself, there's the additional step of applying the license of the PVU feature. 

Each will require it's own activation kit. 

To apply the activation kits: 

1. Download the following activation kits from the Rational License Key Center 

 *  IBM Rational Test Workbench Performance Test v8.7 Activation Kit 
 *  IBM Rational Test Workbench Performance PVU v8.7 Activation Kit


You should now have these two files:

1. Make sure the Rational License Key Server (RLKS) has the IBM Rational Test Workbench Floating License key and not the IBM Rational Performance Tester Floating License key.

2. In Installation Manager choose Manage Licenses 

3. Highlight IBM Rational Performance Tester 8.7 and and IBM Rational Test Workbench Processor Value Unit (PVU) Support click on Configure Floating License Support 

[/support/docview.wss?uid=swg21902683&aid=5] [/support/docview.wss?uid=swg21902683&aid=5] 

 

4. Follow the dialogs to point to the RLKS that has the IBM Rational Test Workbench Floating License key and click Test Connection 

[/support/docview.wss?uid=swg21902683&aid=6] [/support/docview.wss?uid=swg21902683&aid=6] 

 

5. Both the RPT workbench and PVU support option should now have a License Type of Floating applied 

 

[/support/docview.wss?uid=swg21902683&aid=7] [/support/docview.wss?uid=swg21902683&aid=7] 

 

6. Install and use the LKAD on the RPT workbench to get the licenses from the RLKS 

[/support/docview.wss?uid=swg21902683&aid=8] [/support/docview.wss?uid=swg21902683&aid=8] 

 

Validate 

1. Create a schedule with 6 users 

2. Record a simple test that just accesses a web page then exits, example www.ibm.com [www.ibm.com] 

3. Add the test to the schedule 

4. For the user group click Run this group on the following locations (0 defined) 

Note: once RPT is configured to use PVU licensing, the option Run this group on the local computer is not supported. 

5. Choose Add New... 

[/support/docview.wss?uid=swg21902683&aid=9] [/support/docview.wss?uid=swg21902683&aid=9] 

6. Configure the location of the agent 

7. The operating system type must have PVU in it, for example: "Windows (PVU)" 

[/support/docview.wss?uid=swg21902683&amp;aid=10] 

8. Click agent status icon to make sure the agent is connected 

[/support/docview.wss?uid=swg21902683&amp;aid=11] 

9. Run the schedule 

[/support/docview.wss?uid=swg21902683&amp;aid=12] 

 

10. If floating licensing was used then the lmgrd.log file will show the ibmrpt_pvu feature was checked out and checked in.
