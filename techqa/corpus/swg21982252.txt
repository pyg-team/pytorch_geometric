Title: IBM Error creating new Object Store in IBM Filenet Content Manager - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 User recieving an error when creating objectstore:
The database access failed with the following error: ErrorCode -4,470, Message jcc][t4][10120][10898][3.63.75] Invalid operation: result set is closed. ERRORCODE=-4470, SQLSTATE=null'ObjectStore: "OS_name ". [jcc][t4][10120][10898][3.63.75] Invalid operation: result set is closed. ERRORCODE=-4470, SQLSTATE=null. Error Code: 0Error Source: FileNet.Api HR = 0x800710d9 HR Description =Unable to read from or write to the database. MinorCode = 0x00000a94 

RESOLVING THE PROBLEM
Datasources were created manually outside of Configuration Manager (CMUI). Datasources custom properties setting were incorrectly configured. 

To resolve this, ensure that the custom properties for the object store data sources has the following: 

For non XA data source: 
Name=webSphereDefaultIsolationLevel Value=2 Type=Integer 
Name=resultSetHoldability Value=1 Type=Integer 

For XA data source: 
Name=webSphereDefaultIsolationLevel Value=2 Type=Integer
