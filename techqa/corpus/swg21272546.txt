Title: IBM Java error java.lang.StringIndexOutOfBoundsException when using nco_confpack - United States

Text:
nco_os_migrate; omnibus; nco_confpack; java.lang.StringIndexOutOfBoundsException TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 nco_confpack is unable to run against an Object Server migrated with nco_os_migrate. 

SYMPTOM
Java errors: java.lang.RuntimeException: java.lang.StringIndexOutOfBoundsException atcom.micromuse.common.confpack.objectserver.MethodAccessor.invokeMethod(MethodAccessor.java:156)...


CAUSE
Single quotes in migrated tools can cause the error.

RESOLVING THE PROBLEM
 

There may be tools with external effects in the object server which contains extra, unpaired single quotes at the end of the contents of the Executable tab. Remove any lone single quotes found, and then try nco_confpack again.

To find these tools, one can either inspect each tool individually via the Administrator Config utility, or run the following sql against the Object Server to find tools that contain lone single quotes:

select ActionID,Name from tools.actions where ExternalEffect1 like '.*\'$';
