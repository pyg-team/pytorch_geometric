Title: IBM When entering stereotypes in IBM Counter Fraud Management, Case Manager fails to insert into the database with error: CIHCF0006E - United States

Text:
Stereotype field; Stereotypes field; Spaces TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 IBM Counter Fraud Management (ICFM) 1.5.0.4 Case Manager fails to execute an insert query when creating cases whose stereotypes contain spaces. An error message is displayed in the WebSphere SystemOut.log file, CIHCF0006E: Failed to access database or failed to execute a query. 

SYMPTOM
SystemOut.log in WebSphere Application Server displays an SQL error: 

 * E com.ibm.cf.dao.common.GenericTypeDAO insert CIHCF0006E: Failed to access database or failed to execute a query. SQL: INSERT INTO CFFACT.CASE_PROPERTY_TYPE (name, description, stereotype) values (?,?,?). Error code = -803. SQLState = 23505. ErrorMessage = DB2 SQL Error: SQLCODE=-803, SQLSTATE=23505, SQLERRMC=1;CFFACT.CASE_PROPERTY_TYPE, DRIVER=4.19.26. 
   
   com.ibm.websphere.ce.cm.DuplicateKeyException: DB2 SQL Error: SQLCODE=-803, SQLSTATE=23505, SQLERRMC=1;CFFACT.CASE_PROPERTY_TYPE, DRIVER=4.19.26 


CAUSE
Stereotypes in ICFM 1.5.0.4 may not contain spaces, even though the graphical user interface allows you to enter them.

RESOLVING THE PROBLEM
Do not enter spaces in the stereotype. To work around the issue, you may replace any spaces with an underscore--that is, the _ character.
