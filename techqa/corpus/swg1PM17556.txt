Title: IBM PM17556: UNCONFIG.SH UNEXPECTEDLY REINSTATES JVM OPTIONAL ARGUMENTS - United States

Text:
z/os SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When a generic JVM argument is present before an app server is
   configured with ITCAM for the first time and is removed
   afterwards, when the unconfig.sh script is run the JVM argument
   is unexpectely reinstated.
   
   
    
   
   

LOCAL FIX
 *  Manually remove the reinstated JVM arguments after Data
   collector unconfiguration.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:
   *
   * z/OS system with ITCAM 6.1 Data collector where JVM argument
   * modifications are performed after DC configuration.
   *
   ****************************************************************
   *
   * PROBLEM DESCRIPTION:
   *
   * In WebSphere z/OS systems configured with ITCAM Data
   * collector, when a generic JVM argument is present before the
   * application server is configured with ITCAM for the first time
   * and is removed afterwards, the removed JVM argument is
   * reinstated when the unconfig.sh script is run.
   *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this APAR is contained in the following maintenance
   package:

     | fix pack | 7.1.0-TIV-ITCAMAD-FP0001
