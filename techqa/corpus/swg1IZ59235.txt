APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In a policy that hibernates, you cannot have a function call
   prior to
   the Hibernate() call. If you do, the hibernation will not wake
   up when
   an Activate() is issued nor will it activate when it times out.
   This is true for Impact 5.1.0 only. This works in version 4.0.2.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Hibernation doesn't work in it was called from within a while
   loop or a user defined function. For example:
   while (i < 10 ) {
   hibernat(reason,key);
   }
   or
   function Test() {
   hibernate(reason,key);
   }
   
   
    
   
   

PROBLEM CONCLUSION
 *  fixed in 5.1.0.1-TIV-NCI-IF0001
