ERROR DESCRIPTION
 *  While using a custom webservice in combination with a business
   object mapping. When a certain element in the SOAP response from
   the webservice is null, the user application including the BO
   mapping fails with BOXMLSerializer throwing the following
   unexpected exception:
   
   java.lang.ClassCastException:
   com.ibm.ws.bo.bomodel.impl.BusinessObjectTypeImpl incompatible
   with org.eclipse.emf.ecore.EDataType
   
   Stack Dump = An error occurred while parsing native data: The
   error message is: java.lang.ClassCastException:
   com.ibm.ws.bo.bomodel.impl.BusinessObjectTypeImpl incompatible
   with org.eclipse.emf.ecore.EDataType.
   Caused By: java.lang.ClassCastException:
   com.ibm.ws.bo.bomodel.impl.BusinessObjectTypeImpl incompatible
   with org.eclipse.emf.ecore.EDataType
   at
   com.ibm.wsspi.sca.webservice.data.WebServiceNativeDataHolder.get
   Body(WebServiceNativeDataHolder.java:217)
   at
   com.ibm.ws.sca.internal.message.impl.ManagedMessageImpl.getBody(
   ManagedMessageImpl.java:1205)
   at
   com.ibm.ws.sca.internal.message.impl.ManagedMessageImpl.getBody(
   ManagedMessageImpl.java:1066)

 *  NO FIX AVAILABLE
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  WebSphere Process Server and                *
   *                  WebSphere Enterprise Service Bus v6.1.2.x   *
   *                  and v6.2.0.x users.                         *
   ****************************************************************
   * PROBLEM DESCRIPTION: A SOAP envelope containing a top        *
   *                      level XML element having                *
   *                      xsi:nil="true" was deserialized by      *
   *                      the business object                     *
   *                      framework, causing a                    *
   *                      ClassCastException to be thrown         *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Deserialization of top level XML elements having
   xsi:nil="true" was not supported.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Applying this interim fix will resolve the reported problem.
