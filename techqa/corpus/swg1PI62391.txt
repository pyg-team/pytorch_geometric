APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  A map called mainMap_001 have two output cards using a RUN
   function to the map called subMap_002.
   
   The subMap_002 simply return the text value on input card #1 to
   output card #1.
   
   The RUN function calls are configured as follows:
   
   Output Card #1 :
   
   =VALID (  RUN( "subMap_002", " -WM -OE1 " + ECHOIN ( 1, Input )
   ), FAIL ("running subMap_002 failed!!") )
   
   Output Card #2 :
   
   =VALID (  RUN( "subMap_002", " -WM -OE1 " ), FAIL ("running
   subMap_002 failed!!") )
   
   
   Issue: When running maps in ITX 9.0.0, output card #2 returns
   NONE of the application crashes
   
   
    
   
   

LOCAL FIX
 *  CQ48115
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   IBM Transformation Extender customers who use RUN maps and ECHO
   adapter
   
   Problem Description:
   The RUN map does not produce correct result or the process
   abends when the RUN map input card uses ECHO adapter and the top
   level map overrides at least one output card using ECHO adapter
   (e.g. -OE1).
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  Correct the code to proper initialize echo adapter buffers when
   running RUN map.
   
   Applies to:
   9.0.0.0
