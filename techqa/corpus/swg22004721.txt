OVERRIDE GETANDSET TECHNOTE (FAQ)

QUESTION
 Can IBM Transformation Extender use the GETANDSET function with the SPE Adapters -OVERRIDE file option? 

ANSWER
The IBM Transformation Extender's (ITX) GETANDSET function requires file input. In an ITX map rule, when you utilize the SPE adapter that invokes the -TRANSFORM option to run an ITX map similar to: 


GET("SPE", "-TV+ -TRANSFORM " + "-MAP My_map_using_GetAndSetFile" + " -OVERRIDE IN1=file://C:/temp/myinput.txt") 

The -OVERRIDE IN1=file://C:/temp/myinput.txt syntax converts the input to a STREAM object at the time that ITX invokes the map. The GETANDSET function only works with file input so you cannot take advantage of the functionality that the ITX GETANDSET function offers when using the -OVERRIDE IN1=file://C:/temp/myinput.txt syntax.
