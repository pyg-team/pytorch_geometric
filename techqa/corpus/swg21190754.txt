MustGather: WebSphere Application Server Web Server plug-in problems 




For releases of WebSphere Application Server for z/OS V7.0, V8.0, V8.5, V9.0 use the following procedures.

1. Turn on the server tracing using the TSO Modify command (Note: The trace string after tracejava should be on 1 single line without any blanks):

F <server>,tracedetail=(3,4) 
F <server>,tracejava='com.ibm.ws.webcontainer.*=all:com.ibm.wsspi.
webcontainer.*=all:HTTPChannel=all:GenericBNF=all' 

Replace <server> with the jobname of the Appserver's Control Region

2A. If you are using the HTTP Server for z/OS V530, enable -vv trace as follows:

a. Use the modify command 
F IMWEBSRV,APPL=-vv

b. Or at startup by editing the JCL for the started task

//IMWEBSRV PROC LEPARM='ENVAR("_CEE_ENVFILE=/etc/httpd.envvars")', 
// ICSPARM='-vv -r /etc/httpd.conf'

3. Enable plug-in trace: edit the plugin-cfg.xml file and change Loglevel to Trace. 
For example: 
<Log LogLevel="Trace" Name="\WebSphere\AppServer\logs\http_plugin.log"/> 
 * 
 * 
 *   * IBM HTTP Server for z/OS V530 joblog; for the Apache HTTP Server the Error and Access logs. 
    * WebSphere Application Server Control and Servant Regions joblog. 
    * The plug-in trace. 
    * The httpd.conf and plugin-cfg.xml files.
   
   


5. Disable tracing 

a. On the Application Server: 

f <server>,tracejava='TRACEINIT' 

b. On IBM HTTP Server for z/OS: 

f IMWEBSRV, APPL=-NODEBUG
