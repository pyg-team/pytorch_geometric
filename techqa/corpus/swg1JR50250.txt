Title: IBM JR50250: RULE BASED MAPPING WITH MORE THAN 1000 TABLES HIT AN ERROR: ORA-01795: MAX NUMBER OF EXPRESSIONS IN A LIST IS 1000 - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When we map (rule based mapping) more than 1000 tables in a
   single subscription, then we hit the oracle error as follows:
   [Oracle JDBC Driver][Oracle]ORA-01795: maximum number of
   expressions in a list is 1000
   
   
    
   
   

LOCAL FIX
 *  Have only 1000 mapings (rule based) in a single subscription.
   
   
    
   
   

PROBLEM SUMMARY
 *  IIDR 10.2.1 for Oracle Redo fails with the following error:
   [Oracle JDBC Driver][Oracle]ORA-01795: maximum number of
   expressions in a list is 1000 - while using a rule based
   subscriptions which results on more than 1000 tables in the
   subscription.
   
   This issue affects users running IIDR 10.2.1 Interim Fix 2 for
   Oracle Redo (and below).
   
   
    
   
   

PROBLEM CONCLUSION
 *  This issue is fixed by applying the following interim fixes
   depending on the product version:
   - IIDR 10.2.1 Interim Fix 3 for Oracle Redo; or
   - IIDR 10.2 Interim Fix 16 for Oracle Redo; or
   
   
    
   
   

TEMPORARY FIX
 *  Only map 1000 tables in a rule-based subscription.
