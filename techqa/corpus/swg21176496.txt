QUESTION
 I get a "java.lang.NoClassDefFoundError: org/eclipse/xsd/XSDParticleContent" error when using XML services, or any of the import / export samples that use TExportPackageICM.java. How do I correct this? 

CAUSE
If you are using the XML services feature or a feature that uses XML services internally, such as the TExportPackageICM, TExportICM, and TExportManagerICM API sample tools and you get the following error, you are missing some prerequisite jar files from your CLASSPATH. 

Error 

java.lang.NoClassDefFoundError: org/eclipse/xsd/XSDParticleContent
at com.ibm.mm.sdk.xml.PXMLDataInstanceServiceICM.extract(PXMLDataInstanceServiceICM.java(Compiled Code))
at com.ibm.mm.sdk.xml.DKXMLDataInstanceService.extract(DKXMLDataInstanceService.java(Compiled Code))


ANSWER
Pass the "xmlsdk" option to the cmbenv81.bat or cmbenv81.sh environment setup scripts. 

 * 
 * WIN: cmd> cmbenv81 xmlsdk
 * AIX: cmd> cmbenv81.sh xmlsdk


Alternatively, include the following libraries in your CLASSPATH. The libraries listed are installed in the DB2Â® Content Manager installation directory that is assigned by the environment variable <IBMCMROOT>.  * <IBMCMROOT>\lib\xsd.jar 
 * <IBMCMROOT>\lib\xsd.resources.jar; 
 * <IBMCMROOT>\lib\common.jar; 
 * <IBMCMROOT>\lib\common.resources.jar; 
 * <IBMCMROOT>\lib\ecore.jar; 
 * <IBMCMROOT>\lib\ecore.xmi.jar; 
 * <IBMCMROOT>\lib\ecore.resources.jar


As another alternative, include the following libraries in your CLASSPATH. 

 * <IBMCMROOT>\lib\plugins\org.eclipse.emf.common_2.0.0\runtime\common.jar 
 * <IBMCMROOT>\lib\plugins\org.eclipse.emf.common_2.0.0\runtime\common.resources.jar 
 * <IBMCMROOT>\lib\plugins\org.eclipse.emf.ecore_2.0.0\runtime\ecore.jar 
 * <IBMCMROOT>\lib\plugins\org.eclipse.xsd_2.0.0\runtime\xsd.jar 
 * <IBMCMROOT>\lib\plugins\org.eclipse.xsd_2.0.0\runtime\xsd.resources.jar
