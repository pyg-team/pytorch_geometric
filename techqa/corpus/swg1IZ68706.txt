APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Omnibus V7  BI Directional  and Uni Directional Gateways
   terminate as a service.
   
   Issue : when a Large Saf file is present, yet function if ran on
   command line.
   
   REPLICATION: Y -  for both types of binrary:
   
   
   
   Recreate:
   1.  Set a bi-directional gateway to use SAF file, and not
   resynchronization.
   2.  Connect the gateway to a primary and backup ObjectServer.
   3.  Shut down the backup ObjectServer.
   4.  Continue sending events to the primary, and the SAF file
   will grow in size.
   5.  Allow the SAF file to grow to a very large size, 50-60 MB
   should suffice.
   6.  Stop the gateway service, and start the backup ObjectServer.
   7.  Start the gateway service - it attempts to start, begins
   reading the SAF file, and aborts after 30 seconds, if the SAF
   file has not been completely read.
   
   Similar recreate steps apply to the uni-directional gateway.
   
   
    
   
   

LOCAL FIX
 *  None. Remove SAF to start gateway is required.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   Users that run OMNIbus gateways as Windows services.
   ****************************************************************
   PROBLEM DESCRIPTION:
   When a gateway service on Windows is started, and a large SAF
   file needs to be processed, the service would timeout and the
   gateway would not be started.
   ****************************************************************
   RECOMMENDATION:
   Apply the maintenance package listed in the Problem Conclusion.
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The SAF file processing will occur after the service is marked
   as started, so the timeout will no longer occur.
   
   The fix for this APAR is contained in the following maintenance
   packages:
   
        | fix pack | 7.1.0-TIV-NCOMNIbus-FP0012
        | fix pack | 7.2.0-TIV-NCOMNIbus-FP0009
        | fix pack | 7.2.1-TIV-NCOMNIbus-FP0008
        | fix pack | 7.3.0-TIV-NCOMNIbus-FP0002
