ERROR DESCRIPTION
 *  When Impact is installed "distributed" with TBSM, that is the U
   is shared with TBSM, but Impact server itself is installed on a
   seperate server the 2 name servers installed, one on TBSM, the
   other on Impact need to be configured into a name server
   cluster, with the cluster config. mirrored/defined on the TIP
   server using the nci_configuration_utility.
   
   This is currently not documented
   
   
    
   
   

LOCAL FIX
 *  Use nci_configuration_utility to setup the cluster on each
   server in the cluster.
   
   Ensure all 3+ server have the same configuration
   
   
    
   
   

PROBLEM SUMMARY
 *  When Impact is installed "distributed" with TBSM.  For example
   The customer has 3 servers for TBSM and Impact 6.1  :
    1.  TIP Server : Impact UI and TBSM UI
    2.  TBSM Data Server
    3.  Impact Data Server
   
   Both the TBSM Data Server and Impact Server are running
   nameservers independently and need to be configured together as
   a cluster, with the TIP server a member of the cluster.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Updated these topics in the Administration Guide:
   
   "Installation and migration" > "Installing in a shared
   environment"
   - "Installing the GUI Server into an existing TIP installation"
   ("What to do next" section)
   - "Installing both components into an existing TIP installation"
   ("What to do next" section)
   
   "Sharing nameservers with TBSM"
   "Configuring shared nameservers with TBSM"
