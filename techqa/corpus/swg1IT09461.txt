APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Do to the daylight savings time change yesterday at 2:00 am.
   SI is not able to extract any of the batches that were placed
   into CEU between 1:00 - 2:00 am.  We are able to extract the
   batches manually using the cmuextract command.  If we extract
   and re-add the batch to CEU then SI is able to extract
   successfully.  The issue is not specific to the data/batch or a
   mailbox.
   
   
    
   
   

LOCAL FIX
 *  STRRTC - 462268
   RJ/RJ
   Circumvention:
   Update to latest Connect:Enterprise UNIX Build
   
   
    
   
   

PROBLEM SUMMARY
 *  There was no logic to process batches that were created after
   day saving time ended at 2am
   
   
    
   
   

PROBLEM CONCLUSION
 *  If user sets environmental variable PROCESS_DST_CHANGE=1
   batches that were created an hour before the day saving time
   ended, are available for processing
   
   
    
   
   

TEMPORARY FIX
 *  There was no logic to process batches that were created after
   day saving time ended at 2am
