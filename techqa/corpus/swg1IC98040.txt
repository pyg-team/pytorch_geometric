APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When a Library referencing another Broker Library which
   references other Java projects is added to the BAR file, the
   StackOverflow error occurs as a result of the circular java to
   library compilation cycle.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All users of WebSphere Message Broker Toolkit V8.0 and IBM
   Integration Toolkit V9.0 who use WMB/IIB Java Libraries
   referencing other Java libraries in the workspace.
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM SUMMARY:
   When a WMB/IIB Application or Library uses another WMB/IIB
   Java library that references a java library in the workspace,
   a StackOverFlow error may occur when adding such an
   application or library to the BAR file. The problem is due to
   the circular dependency in the build cycle.
   
   
   
   There are a number of resource name changes between WebSphere
   Message Broker and IBM Integration Bus Version 9.0. For
   details visit
   http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e
   tools.mft.doc/bb23814_.htm [http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.etools.mft.doc/bb23814_.htm]
   
   
    
   
   

PROBLEM CONCLUSION
 *  The product now detects and bypasses the circular build cycles
   to avoid the exception while building the BAR file.
