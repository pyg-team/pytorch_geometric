Title: IBM PM54949: XMLLIB->XPATH() FUNCTION RETURNING INCORRECT RESULTS FROM AN ANYELEM FIELD IN AN XML FRAGMENT WHEN DATA IS UTF-8 - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using the XMLLIB->XPATH() function on an anyElem field in a
   UTF-8 encoded XML data fragment can lead to lost or incorrectly
   encoded output.
   
   
    
   
   

LOCAL FIX
 *  WTXCQ - WTX00039658
   PB / PB
   Circumvention: Use the WORD() function to extract the required
   fragment, and then use the SUBSTITUTE() function to re-map the
   XML escaped characters.
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   WebSphere Transformation Extender customers using XPATH function
   and UTF-8 encoding
   
   Problem Description:
   XMLLIB->XPATH() function can only handle native language data.
   
   Platforms Affected:
   ALL
   
   
    
   
   

PROBLEM CONCLUSION
 *  Allow XMLLIB->XPATH() to handle any language. Previously due to
   command line conversion in could only handle native language.
   Now command line is passed as UTF-8.
   
   Delivered in:
   8.2.0.6
   
   
    
   
   

TEMPORARY FIX
