Title: IBM Using CCDT in an Activation Specification for a Queue Manager Group of separate queue managers in WAS V7, V8.0 and V8.5 - United States

Text:
 WHITE PAPER

ABSTRACT
 This technical document describes the steps for configuring an Activation Specification in WebSphere Application Server V7, V8 and V8.5 that uses a Client Channel Definition Table (CCDT) file for a Queue Manager Group in WebSphere MQ V7 that use separate queue managers.
This document explains the usage of the sample MDB (SampleMDBEJBEA) 

CONTENT
+++ Related techdocs 

a) If you want to use a Connection Factory, consult the following related techdoc:
Using CCDT in a Connection Factory for a Queue Manager Group of separate queue managers in WAS V7 and V8.x [http://www.ibm.com/support/docview.wss?rs=171&uid=swg27035452]

b) The sample MDB (SampleMDBEJBEAR) is available from the following techdoc.
It also has the steps on how to create the JMS administrative objects and how to deploy the MDB.
Using WebSphere MQ V7 as JMS Provider for WebSphere Application Server V7, V8.0 and V8.5 [http://www.ibm.com/support/docview.wss?rs=171&uid=swg27016505]

c) For the case of a Queue Manager Group that is composed of a multi-instance queue manager, see the techdoc:
Using a CCDT file to connect to WebSphere MQ multi-instance queue managers from WebSphere Application Server V7 [http://www.ibm.com/support/docview.wss?rs=171&uid=swg27020701]
