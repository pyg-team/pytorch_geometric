LOCAL FIX
 *  You can safely ignore the java.io.NotSerializableException for
   ProcessContainerStartUpBean when shutting down a server.
   
   
    
   
   

PROBLEM SUMMARY
 *  The ProcessContainerStartUpBean class was declared to be
   serializable, but it has a non-serializable FlavorManagerBPC
   field.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix is included in a future release of IBM BPM that ensures
   the ProcessContainerStartUpBean class is serializable and
   serializing instances of this class during EJB passivation no
   longer result in a java.io.NotSerializableException.
