APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In the WebSphere Application Server 6.1.0.X, changing the Heap
   size more than 2048 MB fails with the following message.
   
    "Maximum Heap Size should be an integer in the range 0 through
   2048."
   
   To Recreate :
   
   - Click Application servers > server_name > Process Definition >
   Java Virtual Machine.
   - Enter the heap size value greater than 2048.
   
   WebSphere Application Server 6.1.0.5 or earlier release.
   
   
    
   
   

LOCAL FIX
 *  set JVM heap size via parameters -Xms, -Xmx in field "Generic
   JVM arguments" .
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: WebSphere Application Server version 6.1     *
   *                 users of the Administrative console          *
   *                 application trying to increase the heap      *
   *                 size more than 2048.                         *
   ****************************************************************
   * PROBLEM DESCRIPTION: The JVM Initial or Maximum heap size    *
   *                      couldn't be set more than 2 GB.         *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   After upgrade from Service Pack 6.1.0.2 to 6.1.0.3, it is not
   possible to set the JVM heap size bigger than 2GB via the
   Administrative Console in the fields :
   "Maximum heap size"/"Initial Heap Size"
   
   Navigate through the path:
   Application servers > server_name >
   Process Definition > Environment Entries >
   Java Virtual Machine.
   
   When trying to put a value more than 2048, the following error
   message is displayed:
   "Maximum/Initial Heap Size should be an integer in the range
   0 through 2048."
   
   The validation rule of disallowing a heap size greater than
   2048 is applicable to Z/OS only. It was incorrectly applied to
   all platforms.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code has been modified to make the JVM Heap Size
   restricted only for Z/OS machines.
   
   The fix for this APAR is currently targeted for inclusion in
   fixpack 6.1.0.9. Please refer to the recommended updates page
   for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
