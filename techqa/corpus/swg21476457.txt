Title: IBM Admin vs User  permissions issues in SamplePower - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Product installed without errors. Licensed without errors. 

When logged in with a 'User' account program will start and show the 'Tips' screen. I close that dialog then click on 'New' and I get the 
following:

Run-time error '339':
Component 'VSOCX6.OCX' or one of its dependencies not correctly 
registered: a file is missing or invalid.

If I log out and log in with my administrator account, the program works fine. What permissions need to be modified? 

RESOLVING THE PROBLEM
While logged in as the local administrator run one analysis within SamplePower. This will insure that the proper files are registered and registry keys are created. Once this is done, you should be able to log in as a member of the local 'Users' group and run SamplePower with no further errors
