Title: IBM Information Server 'dscmdexport' command crashes with "java/lang/OutOfMemoryError" - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempting to export a large project result in "java/lang/OutOfMemoryError" when using dscmdexport of IBM InfoSphere DataStage (Client tier). 

SYMPTOM

CAUSE
'dscmdexport' processes run as a Java program.
The export of a large project needs an increase of the maximum heap size.


RESOLVING THE PROBLEM
The client-side Java heap size can be increased by editing the file 

"<root installation>\ASBnode\conf\proxy.xml". 

Increase the "MaximumHeapSize" parameter (test with 512 and increase upto 1500) 

... 

 <!-- These settings overrides the ones used by DataStage client applications --> 

 <DSClient.JvmSettings> 

 <add key="MaximumHeapSize" value="512" /> 

 </DSClient.JvmSettings>
