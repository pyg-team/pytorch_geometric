Title: IBM Security Bulletin: Vulnerability in SSLv3 affects Forms Experience Builder (CVE-2014-3566) - United States

Text:
SSLv3 vulnerability; Forms Experience Builder and SSLv3 vulnerability; WebSphere Application Server and SSLv3 vulnerability SECURITY BULLETIN

SUMMARY
 SSLv3 contains a vulnerability referred to as the Padding Oracle On Downgraded Legacy Encryption (POODLE) attack. SSLv3 is enabled in WebSphere Application Server used by Forms Experience Builder. 

VULNERABILITY DETAILS
CVE-ID: CVE-2014-3566 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2014-3566]
DESCRIPTION: Product could allow a remote attacker to obtain sensitive information, caused by a design error when using the SSLv3 protocol. A remote user with the ability to conduct a man-in-the-middle attack could exploit this vulnerability via a POODLE (Padding Oracle On Downgraded Legacy Encryption) attack to decrypt SSL sessions and access the plaintext of encrypted connections.

CVSS Base Score: 4.3
CVSS Temporal Score: See https://exchange.xforce.ibmcloud.com/vulnerabilities/97013 [https://exchange.xforce.ibmcloud.com/vulnerabilities/97013] for the current score
CVSS Environmental Score*: Undefined
CVSS Vector: (AV:N/AC:M/Au:N/C:P/I:N/A:N)

AFFECTED PRODUCTS AND VERSIONS
IBM Forms Experience Builder 8.5
IBM Forms Experience Builder 8.5.1

REMEDIATION/FIXES
None

WORKAROUNDS AND MITIGATIONS
When Forms Experience Builder is running on WebSphere Application Server (including Network Deployment and Libery profiles), SSL v3 should be disabled using the directions stated in the following bulletin: 

 *  IBM WebSphere Application Server Security Bulletin related to CVE-2014-3566 [http://www-01.ibm.com/support/docview.wss?uid=swg21687173] 

Environments that leverage the IBM HTTP Server should also refer to the instructions provided in the following bulletin: 

 *  IBM HTTP Server Security Bulletin related to CVE-2014-3566 [http://www-01.ibm.com/support/docview.wss?uid=swg21687172] 


IBM recommends that you review your entire environment to identify areas that enable the SSLv3 protocol and take appropriate mitigation and remediation actions. The most immediate mitigation action that can be taken is disabling SSLv3. You should verify disabling SSLv3 does not cause any compatibility issues. GET NOTIFIED ABOUT FUTURE SECURITY BULLETINS
