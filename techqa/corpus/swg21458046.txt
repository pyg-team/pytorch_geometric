QUESTION
 Netcool/OMNIbus v7.4 : What's the best practice for an OMNIbus installation? 

ANSWER

Netcool/OMNIbus v7.4 uses the Tivoli Deployment Engine installer.
DE installs an installed product database in the users home directory [~user/.acsi_user], or /usr/ibm/common/acsi for a root user installation.

It is recommended to perform a non-root installation per product to ensure that issues with the Deployment Engine are isolated and easier to manage. Therefore for a Netcool/OMNIbus v7.4 installation create a user such as netcool, to administer the product.
e.g.
vi /etc/passwd
ncoadmin::100:100::/opt/netcool:/usr/bin/bash
netcool::101:100::/opt/netcool/v74:/usr/bin/bash
webgui::102:100::/opt/netcool/webgui:/usr/bin/bash
:wq
pwconv

Set netcool and webgui passwords as required.

vi /etc/group
ncoadmin::100:root,ncoadmin,netcool,webgui

mkdir /opt/netcool
mkdir /opt/netcool/v74
mkdir /opt/netcool/webgui
chown ncoadmin /opt/netcool
chgrp -R ncoadmin /opt/netcool
chown netcool /opt/netcool/v74
chown webgui /opt/netcool/webgui

The for each installation login as the application specific user;
e.g.
mkdir /tmp/install
chown netcool /tmp/install
su - netcool
cd /tmp/netcool
gzcat CZ9C8ML.tar.gz | tar xvf -
./install.sh -i console

In this case the Deployment Engine files will be installed in /opt/netcool/v74/.asci_netcool.

With the command to list the installed products being;
~netcool/.asci_netcool/bin/listIU.sh

Netcool/OMNIbus v7.4 uses another directory for ASCI:
.acsi_nrv74 -> /opt/nrv741/omnibus/.acsi_<host>
.acsi_<host>
