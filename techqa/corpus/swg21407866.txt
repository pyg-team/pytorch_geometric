Title: IBM AIX® ODM shows previous version after uninstalling OnDemand server or ODWEK - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Uninstalling IBM® DB2® Content Manager OnDemand server or OnDemand Web Enablement Kit (ODWEK) shows a previous version in the AIX Object Data Manager (ODM).

I am expecting no entry in the ODM. 

SYMPTOM
Running lslpp-l ars.srvr for the OnDemand server or lslpp -l ars.www for ODWEK after uninstalling the software shows a previous version in the ODM. 

For example, installing OnDemand server V8.4.1.0, installing the V8.4.1.2 fixpack, uninstalling OnDemand server, and running lslpp -l ars.srvr shows:

# lslpp -l ars.srvr
Fileset Level State Description
----------------------------------------------------------------------------
Path: /usr/lib/objrepos
ars.srvr 8.4.1.0 COMMITTED IBM Content Manager OnDemand
for AIX


CAUSE
OnDemand uses InstallShield to perform the product installation. InstallShield manages product versions using its own Vital Product Data (VPD) registry. 

In OnDemand V8.4.0.X and V8.4.1.X, the InstallShield version used will revert ODM entries to the previous software version instead of removing all matching ODM product entries when OnDemand is uninstalled.


RESOLVING THE PROBLEM
The remaining ODM entries will not prevent OnDemand V8.4.0.X or V8.4.1.X software from being installed. InstallShield performs product prerequisite checking against its own VPD registry. 

To remove the previous version of OnDemand from the ODM, you can either: 

 * Install and uninstall the OnDemand software reported in the ODM. 
 * Remove the ODM entries manually. A backup of the ODM should be taken before proceeding. Commands to remove the OnDemand server or ODWEK ODM entries can be found in the Related information link.
