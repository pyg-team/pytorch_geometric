Title: IBM Timeout error after 30 seconds on reports with Relational Queries - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Error message when running a template on a large data set that takes longer than 30 seconds to run, error generates exactly at 30 seconds:

Clarity.10017
Template build process failed to complete.

Clarity.10023
Error populating grid for maps.

Clarity.12005
RelationalQuery [RelDataQuery1] failed to execute.

System.Data.SqlClient.SqlException
Timeout expired. The timeout period elapsed prior to completion of the
operation or the server is not responding. 

SYMPTOM
Template fails to load after 30 seconds for templates with relational queries as the command time out kicks in after 30 seconds.


CAUSE
There is a command timeout for relational queries and if a large data set is being retrieved from SQL that requires more than 30 seconds you will receive an error in Clarity,

ENVIRONMENT
N/A

DIAGNOSING THE PROBLEM
N/A

RESOLVING THE PROBLEM
Add the tag TimeOut="x"¬ù tag to any relational queries failing where x is a value greater than 30 seconds.
