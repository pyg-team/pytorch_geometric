ERROR DESCRIPTION
 *  On Windows system interactive jobs does not work if the stream
   logon is specified with the DOMAIN.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  The problem happens when the user launching the job is a Domain
   user, and the
   streamlogon looks like <domain name>/<user name> (it is enough
   that the streamlogon
   is specified as <domain name>/<user name>).
   In this case when JOBMON,exe looks for active desktops, the
   owner of each active desktop
   is matched against the streamlogon user name: a check on the
   domain name is missing here,
   and the check between user name and domain/user names always
   fail.
   Now we added a furthur check on the domain of the user owning
   the active desktop, and the
   domain of the streamlogon user.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This apar will be fixed   8.6.0-TIV-TWS-FP0003,
   8.5.1-TIV-TWS-FP0005
