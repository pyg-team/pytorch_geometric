Title: IBM SE36370: MQM400 Inconsistencies noticed for the SSLPEER attribute of     CHGMQMCHL - United States

Text:
 SUBSCRIBE

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  SSL Peer name (SSLPEER) parameter of the CHGMQMCHL command
   sometimes shows additional unwanted characters at the end of
   the specified string. However the DSPMQMCHL will show the
   correct values specified for SSL Peer name (SSLPEER).
   
   
    
   
   

LOCAL FIX
 *  Use MQSC ALTER command to change the channel and add the
   SSLPEER data
   
   
    
   
   

PROBLEM SUMMARY
 *  SSL Peer name (SSLPEER) parameter of the CHGMQMCHL command
   sometimes shows additional unwanted characters at the end of
   the specified string. However the DSPMQMCHL will show the
   correct values specified for SSL Peer name (SSLPEER).
   
   Users Affected: All WMQ users on System i platform using SSL
   and specifying the value for SSLPEER.
   
   Platforms Affected: iSeries
   
   The SSL peer name was not null terminated and buffer was not
   initialized properly.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Code changes were carried out to initialize the buffer and to
   null  terminate the string.
