ERROR DESCRIPTION
 *  Description: when an existing site (custom) has a SCM check and
   letter removed
   from the site. TEMA SCM will still shown the checklist with 0
   check.
   
   steps:
   1. create a custom site
   2. add a SCM Check
   3. run ETL
   4. remove the only SCM check
   5. run ETL
   
   expect:
   the Checklist will not be appear on the TEMA SCA
   
   actual:
   the checklist will be appear on the TEMA SCA with 0 check
   
   
    
   
   

LOCAL FIX
 *  The workaround is to remove the offending custom
   site and readded back to TEM console.
   
   
    
   
   

PROBLEM SUMMARY
 *  This issue will be addressed in the next release of IBM Endpoint
   Manager Security Configuration Analytics.
