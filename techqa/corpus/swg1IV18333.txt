ERROR DESCRIPTION
 *  Problem title
   Vulnerability and Security scan on application interface -
   'session fixation'.
   
   Problem description
   
   We are currently running ITNCM version 6.3 and we ran VA
   scanner for the application (IBM Rational AppScan) and found
   few vulnerabilities with the following recomendation,
   
   1. Do not accept externally created session identifiers
   2. Do not allow caching of SSL pages
   3. Enforce the use of HTTPS when sending sensitive information
   4. Always use SSL and POST (body) parameters when sending
   sensitive information.
   5. Add the 'Secure' attribute to all sensitive cookies
   6. Add the 'HttpOnly' attribute to all session cookies
   7. Avoid storing sensitive session information in permanent
   cookies
   8. Remove sensitive information from HTML comments
   
   
    
   
   

LOCAL FIX
 *  No workaround.
