APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If a Java project is referenced by a WebSphere Message Broker
   library, and the WMB project is referencing the WMB library, the
   Java project will be compiled as an independent resource
   during BAR compilation. As a result, there will be 2 copies of
   JAR file created for the same Java project in the BAR file.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All users of WebSphere Message Broker Toolkit who build Broker
   Archive files for a Java project referenced by a WebSphere
   Message Broker library.
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM SUMMARY:
   If a Java project is referenced by a library, the BAR compiler
   compiles the Java project as an independent resource. This
   results in multiple copies of the same Java project in the BAR
   file.
   
   
    
   
   

PROBLEM CONCLUSION
 *  WebSphere Message Broker Toolkit now compiles the library
   referencing the Java project instead of compiling the Java
   project as an independent resource.
