Title: IBM FNRCE0002E: E_BAD_CLASSID: Class "SystemConfiguration" not found. - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The error FNRCE0002E: E_BAD_CLASSID: Class "SystemConfiguration" not found is logged in the Content Engine error log whenever IBM Content Collector (ICC) Configuration Manager is launched. 

SYMPTOM
The following error message is logged in the P8 server error log:

2016-01-04T14:02:23.647 75BEE0F7 ENG FNRCE0002E - ERROR method name: determineBadClassRetry principal name: p8AdminUser Global Transaction: false User Transaction: false Exception Info: Class "SystemConfiguration" not found.
com.filenet.api.exception.EngineRuntimeException: FNRCE0002E: E_BAD_CLASSID: Class "SystemConfiguration" not found.
at com.filenet.engine.metadatacache.MetadataCacheBase.determineBadClassRetry(MetadataCacheBase.java:243)


CAUSE
When ICC Configuration Manager is launched, some queries will be sent to IBM FileNet P8 Content Engine to determine if an object store configured is a File Plan Object Store (FPOS). This happens even if IBM Enterprise Records (IER) is not configured in the environment.

ENVIRONMENT
IBM Content Collector with FileNet P8 repository 

DIAGNOSING THE PROBLEM
The error message is harmless and can be ignored. 

RESOLVING THE PROBLEM
This issue is still being investigated.
