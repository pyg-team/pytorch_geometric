Title: IBM Install requires WebSphere Application Server 6.0 disks - United States

Text:
PM15334; IM; RAD; WAS; WTE TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
Installation of IBM Rational Application Developer for WebSphere Software version 7.5 requires WebSphere Application Server Test Environment 6.0 installation disks even when it is not selected for install.

SYMPTOM
The following warning appears when installing Rational Application Developer: 


"Installation Manager cannot find the files for one or more disks. If you continue with the installation, you will be prompted to provide the location of these files. If these files are not available, the installation will fail. See Details for more information."

Details states:

"C:\RAD75\RAD_SETUP
IBM WebSphere Application Server Version 6.0 Test Environment 6.0.2.29
Disks needed: 1. Disks not found:1."


[/support/docview.wss?uid=swg21437924&amp;aid=1]


 * The "Warning" appears on the Summary page 
 * There are no errors in the log files 
 * Installation prompts you for WebSphere Application Server 6.0 if you continue with the installation


CAUSE
This issue has been identified as a product defect under APAR PM15334.



RESOLVING THE PROBLEM
 

WORKAROUNDS:


 * Option #1: 1. Launch Installation Manager 
   
   2. Go to Repositories 
   
   3. Remove the repository for the RAD_SETUP directory. 
   
   4. Add the diskTag.inf files for RAD/disk1 and WAS70/disk1 to the repository list. 
   
   5. Proceed with the install.
   
   
   
   
 * Option #2: 
   Download the WebSphere Application Server 6.0 zip files and add them to the Installation Manager repository.
