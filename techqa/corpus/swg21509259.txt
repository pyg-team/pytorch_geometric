Title: IBM embedded WebSphere Application Server fix pack install procedure for IBM Tivoli Monitoring V6.2.2 - United States

Text:
embedded WAS; eWAS; embedded WebSphere Application Server; TEPS; fix pack; portal server; ITM; upgrade eWAS; security TECHNOTE (FAQ)

QUESTION
 How do we install a fix pack for the embedded WebSphere Application Server provided by IBM Tivoli Monitoring V6.2.2? 

CAUSE
IBM Tivoli Monitoring (ITM) provides embedded WebSphere Application Server (eWAS) for use with Tivoli Enterprise Portal Sever (TEPS). For maintenance purposes it may be necessary to install a fix pack for eWAS. 
You can use this procedure to install fix packs for a given eWAS version, but not to upgrade to a new eWAS version. For example, you can use it to install eWAS 6.1.0.39 on top of eWAS 6.1.0.35, but you cannot use it to install eWAS version 7.0 on top of eWAS version 6.1. 

The following table shows the different levels of eWAS that are provided by ITM 6.2.2: 

ITM version eWAS version 6.22.04.00 and later 6.1.0.35 6.22.00.00 through 6.22.03.00 6.1.0.23 
This technote applies specifically to eWAS, not the full version of WebSphere. Further, this technote applies only to V6.2.2 of ITM.

Step 6. Copy Policy Files  * 
 * 
 *  Unix: Run these commands:  * 
    * cd CANDLEHOME/
    * platform
    * /iw/java/jre/lib/security
      find CANDLEHOME/JRE -name "*policy.jar" -exec cp {} . \;
   
   
   Where platform is the ITM platform value that you determined in Step 2a. Windows: Run these commands:  * 
    * cd CANDLEHOME\
    * CNPSJ\java\jre\lib\security
    * 
      copy CANDLEHOME\InstallITM\*policy.jar
   
   

Step 7, Modify the ITM version information for the eWAS component  * 
 * 
 *  Unix: Edit CANDLEHOME\registry\iwplatform.ver to change the VRMF value so it reflects the new eWAS level. For example, change this:  * 
    * VRMF = 06103500
   
   
   To this:  * 
    * VRMF = 06104701
   
   
   Verify the change by running the command CANDLEHOME/bin/cinfo -t iw. Its output should reflect the new eWAS level. Windows: Edit CANDLEHOME\InstallITM\ver\KIWWICNS.ver to change the VRMF and DVRMF values so they reflect the new eWAS level. For example, change this:  * 
    * VRMF=06103500
    * 
    * DVRMF=06.01.00.35.00
   
   
   To this:  * 
    * VRMF=06104701
    * 
    * DVRMF=06.01.00.47.01
   
   
   Verify the change by running the command CANDLEHOME/InstallITM/kincinfo -t iw. Its output should reflect the new eWAS level.
