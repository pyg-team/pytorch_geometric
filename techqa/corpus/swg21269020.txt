Title: IBM During migration WebSphere Application Server hangs after it has finished installing the WebSphere Commerce EAR - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 If the migrated EAR is already in installedApps directory, and the following error exists in the SystemOut.log file, then WebSphere Application Server could be hanging:

[8/26/07 15:14:29:047 CDT] 00000030 SystemOut O Commerce updating file permission on application WC_swanson60d
[8/26/07 15:14:29:058 CDT] 00000030 SystemOut O Script file is: /tmp/WC_swanson60d2007.08.26.15.14.29.000031.sh
[8/26/07 15:15:59:510 CDT] 0000002b ThreadMonitor W WSVR0605W: Thread "SoapConnectorThreadPool : 0" (00000030) has been active for 679168 milliseconds and may be hung. There is/are 1 thread(s) in total in the server that may be hung.
[8/27/07 16:10:11:622 CDT] 0000002b ThreadMonitor W WSVR0605W: Thread "SoapConnectorThreadPool : 1" (0000003f) has been active for 767961 milliseconds and may be hung. 

There can be 2 threads in the server that may be hung.



CAUSE
Check the original EAR to see if it contains files with blank space in file names. 

RESOLVING THE PROBLEM
If the EAR file contains files with blank spaces in the file names, apply the WebSphere Application Server iFix 6.0.2.5-WS-WAS-IFWC-file-permission3 as documented in the WebSphere Commerce 6.0 fix pack installation guides. 

Additional details can also be found in the technote: http://www-1.ibm.com/support/docview.wss?uid=swg24015639 [http://www-1.ibm.com/support/docview.wss?uid=swg24015639].
