Title: IBM IV47490: "INVALID BINDING" IS DISPLAYED ON APPLICATION IMPORT WINDOW WHEN IMPORT TARGET OBJECT IS EMPTY. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  "Invalid Binding" is displayed on Application Import window
   and import fails when import tartget object is empty (has
   no records).
   
   This problem can be recreated with the following steps.
   
   1.  Login to ISCCD, and select "System Configuration"-
       "Platform Configuration"-"Database Configuration"
       application.
   2.  Create a new object with the following values.
   
   Object name: TESTDOMAIN
   Attributes: see below.
   
   Attribute                    Type    Leng  Req Per Search
   ---------------------------  ------- ----- --- --- --------
   DOMAINID                     ALN     20    0   1   WILDCARD
   DOMAINVALUE                  ALN     50    0   1   WILDCARD
   DESCRIPTION_LONGDESCRIPTION  LONGALN 32000 0   0   WILDCARD
   
   * The following columns are created by default.
   Attribute                    Type    Leng  Req Per Search
   ---------------------------  ------- ----- --- --- --------
   TESTDOMAINID                 BIGINT  19    1   1   EXACT
   DESCRIPTION                  ALN     50    0   1   WILDCARD
   HASLD                        YORN    1     1   1   NONE
   
   Leng  = Length
   Req   = Required?
   Per   = Persistent?
   Search= Search Type
   
   * Make this object available with Admin mode.
   
   3.  Select "System Configuration"-"Platform Configuration"
       -"Application Designer" application.
   4.  Create a new application with the following values.
   
   Application  : TESTDOMAIN
   Description  : Domain Management
   Main Object  : TESTDOMAIN
   Key Attribute: TESTDOMAINID
   Module Name  : SETUP (Administration)
   
   Select "Single Page App".
   
   5.  Select "Integration"-"Object Structures" application.
   6.  Create a new Object Structure with the following values.
   
   Object Structure: TESTDOMAIN
   Consumed By     : INTEGRATION
   Source Objects  : TESTDOMAIN
   
   7.  Select "Inbound Setting Restrictions" from "Select
       Action" menu.
   8.  Check "Restricted?" and "Override?" of TESTDOMAINID,
       and click OK.
   9.  Select "Add/Modify Application Export Support" from
       "More Actions" in the left pane.
   10. Click "New Row" and add TESTDOMAIN to "Application
       Export Support" (with default value).
   11. Select "Add/Modify Application Import Support" from
       "More Actions" in the left pane.
   12. Click "New Row" and add TESTDOMAIN to "Application
       Import Support" (with default value).
   13. Open "Administration"-"Domain Management" application
       (created at step #4).
   14. Register several records.
   15. Select "Application Export" from "Common Actions" menu
       in the left pane.
   16. Specify the Object Structure TESTDOMAIN, XML format,
       and export target directory, then export the records.
   17. Delete all of the records (click on Trash icon).
   18. Select "Application Import" from "Common Actions" menu
       in the left pane.
   
   "Invalid Binding" is displayed on Application Import window.
