Title: IBM Attempts to access individual activities within IBM Connections results in errors - United States

Text:
optBitFilterSize; J2CA0056I; CLFRA0648E; Connections; Activities; The Connection Manager received a fatal connection error from the Resource Adapter for resource jdbc/activities; com.ibm.db2.jcc.am.SqlException TECHNOTE (TROUBLESHOOTING)

PROBLEM
While users can login to IBM Connections and access the Activities
application, they are unable to access the activities themselves. This
includes previously existing activities as well as newly created
activities.

SYMPTOM
Users get errors in the browser similar to the following errors that occur in the systemout.log:

ConnectionEve A J2CA0056I: The
Connection Manager received a fatal connection error from the Resource
Adapter for resource jdbc/activities. The exception is:
com.ibm.db2.jcc.am.SqlException: The SQL statement or command failed
because of a database system error. (Reason "optBitFilterSize less than
min".). SQLCODE=-901, SQLSTATE=58004, DRIVER=4.13.127
[10/29/13 13:40:41:842 EDT] 0000008b OpenActivitie E
sun.reflect.NativeMethodAccessorImpl invoke0 CLFRA0091E: Internal error
com.ibatis.dao.client.DaoException:
CLFRA0648E: Error getting nodes in activity:
com.ibatis.common.jdbc.exception.NestedSQLException:
--- The error occurred while applying a parameter map.
--- Check the NodeSummary.getDescendentNodes-InlineParameterMap.
--- Check the statement (query failed).
--- Cause: com.ibm.websphere.ce.cm.StaleConnectionException: The SQL
statement or command failed because of a database system error. (Reason
"optBitFilterSize less than min".).


CAUSE
Database inconsistency/corruption.

RESOLVING THE PROBLEM
The customer ran maintenance on the OPNACT database. Database maintenance is documented in the following wiki articles:
