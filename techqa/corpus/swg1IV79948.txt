ERROR DESCRIPTION
 *  If you follow these steps to set the session timeout to
   "No timeout" at this link:
   .
   http://www-01.ibm.com/support/knowledgecenter/SSWPVP_2.5.0.3/com [http://www-01.ibm.com/support/knowledgecenter/SSWPVP_2.5.0.3/com]

   stall_setkeepalive.html?lang=en
   .
   You will run into an issue where when you try to login to the
   SKLM GUI you will received a message that you session has
   become invalid and it will kick you out of the GUI.
   
   
    
   
   

LOCAL FIX
 *  Until fix is available set the session timeout to some setting
   other than No Timeout.
