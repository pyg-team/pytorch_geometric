ERROR DESCRIPTION
 *  To reproduce this issue, the following configuration was set up:
   
   1 - Configure an OMNIbusEventReader with two policies
   2 - The two policies are editted versions of the supplied XML
   default policy. Each policy accesses an Apache server XML web
   page, using different login accounts
   3 - Based on this setup, the two different web pages do not get
   returned. Only one web page is returned, and the other user
   account is not used.
   
   
    
   
   

LOCAL FIX
 *  No workaround possible
   
   
    
   
   

PROBLEM SUMMARY
 *  When setting up two XML DSAs they will only use one username and
   password even if they are configured with two different ones.
   
   
    
   
   

PROBLEM CONCLUSION
 *  XML DSA now uses username and password for each DSA
