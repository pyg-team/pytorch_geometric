Title: IBM Install Core Dump During the Identity Insight 8.1 FP1 Upgrade - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 Why do I get "java/lang/OutOfMemoryError" and install core dump during the Identity Insight 8.1 FP1 upgrade? 

CAUSE
Check the size of the .com.zerog.registry.xml file in _uninst directory. If it is very large(3-4MB), then rename the _uninst dir and run the installer again.

ANSWER
The user will see a java.lang.OutOfMemoryError exception - and three core files will be produced: 


heapdump.*.phd
javacore.*.txt
Snap.*.trc
(the * are date/time markers, and will be different with each occurrence)

The core files were put in /tmp/install.dir.*
