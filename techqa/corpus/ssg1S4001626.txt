APAR / Issue DESCRIPTION APAR IT16630 AFTER UPGRADING COPY SERVICES MANAGER TO 6.1.2 or 6.1.3, USER IS UNABLE TO LOGIN TO SERVER 

Customers that have migrated to IBM Copy Services Manager from a Tivoli Productivity Center for Replication server originating at release 5.1.x or earlier, may experience an issue logging into the server after upgrading to the 6.1.2 or 6.1.3 version. 

A new feature introduced in the 6.1.2 version made a change to the database requiring the database to be updated to the newest level. The 6.1.3.1 version will now update the database on startup to prevent this issue and allow
the user to login after upgrade. Defect 61810 VIEW ADVANCED SETTINGS NOT DISPLAYING FOR DS8000 STORAGE SYSTEM AT 8.1.1 LEVEL 

There is an issue preventing the extraction of the DS8000 advanced settings if the Copy Services Manager server has not exported any data previously. Copy Services Manager interim fix pack 6.1.3.1 contains a fix for this issue. 

As a work around, attempt to export data from the Copy Services Manager server, such as the copy sets for a session, and then restart the server. INSTALLATION INSTRUCTIONS
