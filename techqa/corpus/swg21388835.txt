Title: IBM When you upgrade to V8.4.1, you must run the arsdb command for OnDemand Report Distribution - United States

Text:
 FLASH (ALERT)

ABSTRACT
 When you upgrade the IBMÂ® Content Manager OnDemand server to V8.4.1 and access Report Distribution objects in the OnDemand Administrator Client, the following error is shown: "The server failed while accessing Report Distribution data. Contact your system administrator."

The OnDemand system log displays the following similar error:

DB Error: [IBM][CLI Driver][DB2/AIX64] SQL0206N "R.AID" is not valid in the context where it is used. SQLSTATE=42703 -- SQLSTATE=42S22, SQLCODE=-206, File=arsrddb.c, Line=3478 


CONTENT
After you upgrade to OnDemand server V8.4.1, run arsdb [-I <OD_INSTANCE>] -uv and arsdb [-I <OD_INSTANCE>] -mv. If Report Distribution is installed, run the following commands:


 1. Ensure that you have taken a backup of either the OnDemand database or the ARSRDREPDT table before proceeding. 
 2. Update the ARSRDREPDT table:
    arsdb [-I <OD_INSTANCE>] -uRv 
 3. Maintain the database:
    arsdb [-I <OD_INSTANCE>] -mRv
    
    Note: For Oracle databases, run arsdb [-I <OD_INSTANCE>] -sRv instead of -mRv.

Ensure that you have taken a backup of either the OnDemand database or the ARSRDREPDT table before you runarsdb [-I <OD_INSTANCE>] -uRv.
