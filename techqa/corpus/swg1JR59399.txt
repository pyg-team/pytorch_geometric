ERROR DESCRIPTION
 *  When you use the manageprofiles command or the Profile
   Management Tool (PMT) GUI and omit both the federateLater and
   the noagent parameters (the latter parameter is not exposed in
   the PMT), the IBM BPM node-side extension checker runs and the
   node agent starts before the managed node profile has been
   created. As a result, the managed node profile is not yet
   recognized, which causes the following error to appear in the
   <managed_profile>/logs/addNode.log file:

LOCAL FIX
 *  Select the "Federate this node later" option and then later
   federate the node after it is created or use BPMConfig
   
   
    
   
   

PROBLEM SUMMARY
 *  No additional information is available.
   
   
    
   
   

PROBLEM CONCLUSION

TEMPORARY FIX
 *  Either select the "Federate this node later" check box in the
   PMT or use the BPMConfig -create -profile command instead of the
   PMT.
