QUESTION
 How can I apply a Fix Pack to Core/Classic Optim and Optim Solution Components (Optim Designer, Optim Manager, Optim Runtime Services, WAS-CE and Optim Service Interface)? 

ANSWER
Updates to Core/Classic Optim, and to Optim Solution Components, must be applied separately.

Core/Classic Optim

Follow the steps below to download fix packs for the Core/Classic Optim Product,

Go to IBM Fix Central by clicking on below link or in the related URL section
[http://www-933.ibm.com/support/fixcentral/]
http://www-933.ibm.com/support/fixcentral/ [http://www-933.ibm.com/support/fixcentral/]

Select Product
Product Group - Information Management
Select from Information Management - Optim
Installed Version - Choose your Optim Version
Platform - Select Platform
Continue
Identify Fixes - Continue
Download Fix Pack 

Follow the instructions in the readme.pdf file to apply the fix pack on the Core/Classic Optim product.

Optim Solution Components:

Updates can be performed using one of two modes:


 * GUI Mode 
 * Console Mode 

(Note: You can use GUI mode to perform an update on a Unix/Linux platform if you set $DISPLAY. If you do not want to set $DISPLAY, please follow the instructions for Console mode.) 

Each of these modes in turn consists of two methods: 
 * Online method requires internet connection 
 * Offline method does not require internet connection 

GUI Mode 
 * Open Installation Manager 
 * File->Preferences-> Repositories-> Add Repository-> 
 * Use either Online or Offline Method below: 
 *   * Online Method 
    * From Repositories for Online method (listed below), copy and paste the repositories for your Optim version 
    * Select the check box "Search service repositories during installation and updates" 
    * Offline Method 
    * Browse to the folder where the extracted .zip file is located from Repositories for Offline method (listed below) and choose repository.config file 
    * Do not Select Check Box "Search service repositories during installation and updates" 
    * Click on OK 
   
   


Console Mode 
 * Goto <INSTALLATION_MANAGER_HOME>/eclipse/tools/ 
 * Execute ./imcl -c for UNIX/Linux imcl -c for Windows 
 * Go to Preferences by typing P 
 * Select Repositories by typing 1 
 * Then Add Repositories by typing D 
 * Use either Online or Offline Method below: 
 *   * Online Method 
    * From Repositories for Online method, copy and paste the repositories for your Optim version and press enter 
    * Type S to select/mark "Search service repositories during installation and updates" 
    * Offline Method 
    * Copy and paste the location of repository.config from the extracted .zip file from Repositories for Offline method. Make sure repository.config is in the path copied 
    * Type S to not select/mark "Search service repositories during installation and updates" 
    * Select Apply Changes and Return to Preferences Menu by typing A 
    * Select Return to Main Menu by typing R 
    * Follow the above steps for all of the components 
    * Select Update by typing 2

   To deploy the optim.unified.web.app.war WAR file with the Administrative Console of WebSphere Application Server Community Edition: 
   1. If you are upgrading a optim unified web app which consists of Optim Manager and Optim Connection Manager WAR file, uninstall the previous version of the WAR file before you deploy the new WAR file. To uninstall the WAR file, click Web App WARs under Console Navigation, and then click Uninstall on the line that contains an URL of /optim.
   
   2. Under Console Navigation, click Deployer.
   
   3. Click Browse after Archive and use the Browse window to select the service interface WAR file. The WAR file is shared_installation_directory/web/app/optim.unified.web.app.war, where shared_installation_directory is the installation directory that you specified for the IBM Optim Shared package group. For example, the default location for the WAR file on Microsoft Windows is C:\IBM\InfoSphere\Optim\shared\web\app\optim.unified.web.app.war.
   
   4. Click Install. The WAR file can take several minutes to deploy.
