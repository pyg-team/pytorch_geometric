Title: IBM PAM users cannot change their password in Dashboard Application Services Hub - United States

Text:
jazzsm1121relnotes password PAM WebGUI TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When WebGUI is installed and the user repository is an ObjectServer, then the PAM users in that repository are not able to change their existing password using the console. 

RESOLVING THE PROBLEM
To enable PAM users, in the ObjectServer repository, to change their existing password using the Dashboard Application Services Hub console, upgrade to Dashboard Application Services Hub 3.1.2.1 Cumulative Patch 4 (or higher) and WebGUI 8.1 Fixpack 5 (or higher), and then execute the following commands: 

 1. From JazzSM [http://www-01.ibm.com/support/knowledgecenter/SSEKCU_1.1.2.1/com.ibm.psc.doc/ref/psc_r_pathnames.html]\profile\bin run the wsadmin command to add the existingpassword property to the PersonAccount entity:
    $AdminTask addIdMgrPropertyToEntityTypes { -name existingpassword -dataType string -entityTypeNames PersonAccount } 
 2. In the JazzSM [http://www-01.ibm.com/support/knowledgecenter/SSEKCU_1.1.2.1/com.ibm.psc.doc/ref/psc_r_pathnames.html]\profile\config\cells\cell_name\wim\config directory, locate and edit wimconfig.xml so that the custom property (existingpassword) does not undergo role checking:
    <config:attributeGroups>
    <config:groupName>unchecked</config:groupName>
    <config:attributeNames>existingpassword</config:attributeNames>
    <config:attributeNames>identifier</config:attributeNames>
    <config:attributeNames>createTimestamp</config:attributeNames>
    <config:attributeNames>modifyTimestamp</config:attributeNames>
    <config:attributeNames>entitlementInfo</config:attributeNames>
    </config:attributeGroups>
