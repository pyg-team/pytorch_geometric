Title: IBM JAR files to package for Rule Execution Server in Java SE - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 What JAR files are required in my application when I use Rule Execution Server (RES) in a Java SE environment? 

CAUSE
The executionserver/lib folder contains a number of JAR files. Only some of them are necessary if you use Rule Execution Server in a Java SE environment. 

ANSWER
When you use Rule Execution Server in a Java SE environment, you must package a number of JAR files in your application. The files are provided in the executionserver/lib folder. 

In section Setting the class path for Java SE [http://pic.dhe.ibm.com/infocenter/dmanager/v8r5/topic/com.ibm.wodm.family.config.javase/config_ds_res/tsk_res_config_jse_classpath.html], the documentation explains how to find those files with Ant by referencing the variable executionserver.j2se.classpath. 

If you need to extract those files or if you do not use Ant to build your application, you can use the tool below to help you. 

This Ant script provides 3 actions: 

 * display: to display the list of JAR files 
 * copy: to copy the required JAR files to a dedicated folder  * parameter: target.copy.dir
   
   
 * package: to zip the JAR files into a single portable archive named jse-res-jars.zip for redistribution  * parameter: target.zip.dir
   
   

To use the classpath-res-jse.xml file, copy it to the <odm_installdir>/executionserver/lib folder. 

classpath-res-jse.xml [/support/docview.wss?uid=swg21647368&aid=2]classpath-res-jse.xml [/support/docview.wss?uid=swg21647368&aid=1] 



NOTE
The Ant executable is available in the ODM installation in this directory: <odm_installdir>/shared/tools/ant/bin/ant 

 



EXAMPLE
On a DOS command line from the executionserver/lib directory, you can run the following commands: 

 

..\..\shared\tools\ant\bin\ant -f classpath-res-jse.xml display 

..\..\shared\tools\ant\bin\ant -f classpath-res-jse.xml copy -Dtarget.copy.dir=c:\myapplication\lib 

..\..\shared\tools\ant\bin\ant -f classpath-res-jse.xml package -Dtarget.zip.dir=c:\myapplication\zip
