LOCAL FIX

PROBLEM SUMMARY
 *  The issue was a consequence of a fix made for 195176, where we
   set the end address of a code cache segment to avoid a hang.
   However, sometimes this new special segment allocation can fail
   and in that situation we weren't resetting the end segment
   address.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This defect will be fixed in:
   7.0.0 SR4
   .
   Fix checked in to reset the end segment address when code cache
   allocation fails with specified start address.
