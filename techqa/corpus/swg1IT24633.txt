Title: IBM IT24633: AFTER SETTING MQSISETDBPARMS, BIP8207I MESSAGE DISPLAYED AFTER RESTARTING THE IIB SERVICES IN HIGH AVAILABILITY ENVIRONMENT. - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In a High Availability environment, after making a change to a
   DSN using mqsidetdbparms,
   the following message is displayed each time a  broker instance
   is restarted.
   
     "BIP8207I: The Message Broker has not been restarted since the
   last change made to this
      datasource by the mqsisetdbparms command"
   
   This APAR applies to IIB v9.0.0.7 and IIB v10.0.0.x.
   
   
    
   
   

LOCAL FIX
 *  Customer can Ignore this message after restarting the IIB
   services.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All users of IBM Integration Bus version 9.0.0.7 and above and
   version 10 using the Multi-Instance Broker.
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   In a High Availability environment, after making a change to a
   DSN using mqsidetdbparms, the following message is displayed
   each time a broker instance is restarted:
   "BIP8207I: The Message Broker has not been restarted since the
   last change made to this datasource by the mqsisetdbparms
   command"
   
   This APAR applies to IIB v9.0.0.7 and IIB v10.0.0.x.
   
   There are a number of resource name changes between WebSphere
   Message Broker and IBM Integration Bus Version 9.0.x
   For details visit
   http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e
   tools.mft.doc/bb23814_.htm [http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.etools.mft.doc/bb23814_.htm]
   
   
    
   
   

PROBLEM CONCLUSION
 *  The product now verifies that the IBM Integration Bus services
   restart status from shared location, and local registry, so that
   BIP8207I is validated and ignored correctly after restarting the
   IBM Integration Bus services.
