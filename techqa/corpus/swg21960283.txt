Title: IBM The S&P agent won't connect to the TEPS - United States

Text:
summarization; pruning; portal; server TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Getting getHistoryConfig request to the TEP Server failed in the S&P agent logs. 

SYMPTOM
Error showing in the S&P agent log:

RESOLVING THE PROBLEM
Check the S&P agent config file sy.ini (if unix or KSYENV, if Windows):

KSY_CNP_SERVER_HOST=<TEPS ip address or hostname> 
KSY_CNP_SERVER_PORT=1920

Make sure there are no extra characters and the port is set to 1920.
