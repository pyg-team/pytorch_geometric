Title: IBM JR28218: JDBC ADAPTER 6.0.2 UNABLE TO INSERT CLOB VALUES OF 4K OR LARGER - United States

Text:

APAR STATUS
 * CLOSED AS DOCUMENTATION ERROR.
    
   
   

ERROR DESCRIPTION
 *  IBM WebSphere Adapter for JDBC version 6.0.2 are unable to
   insert CLOB values of 4k and larger into Oracle 9i or 10g
   databases.
   
   Symptoms: Exception under Oracle 9i:
   ResourceAdapt E com.ibm.j2ca.jdbc.JDBCDBOperationHandler
   executePreparedCUDStatement CWYBC0301E: An operation on the
   database failed with a SQL exception for the following reason:
   No more data to read from socket.
   
   Exception when applying Oracle 10g:
   ResourceAdapt E com.ibm.j2ca.jdbc.JDBCDBOperationHandler
   executePreparedCUDStatement CWYBC0301E: An operation on the
   database failed with a SQL exception for the following reason:
   ORA-01460: unimplemented or unreasonable conversion requested
   
   
    
   
   

LOCAL FIX
 *  The usage of older Oracle drivers (9.2.0.8 or older) cause this
   error.  To resolve the problem you need to apply the latest
   (10.1.0.2 or higher) Oracle thin driver.
   
   The following technote was created for the issue:
   http://www-1.ibm.com/support/docview.wss?rs=0&q1=clob+4k&uid=swg
   21284638&loc=en_US&cs=utf-8&cc=us&lang=en [http://www-1.ibm.com/support/docview.wss?rs=0&q1=clob+4k&uid=swg21284638&loc=en_US&cs=utf-8&cc=us&lang=en]
   
   
    
   
   

PROBLEM SUMMARY
 *  A newer version of the Oracle thin driver (10.1.0.2 or
   higher) is needed.  A tech note has been created for the 6.0.2
   version of the adapter and the JDBC v6.1 user guide has been
   updated with the information.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Documentation has been created and updated to reflect the
   limitation.
