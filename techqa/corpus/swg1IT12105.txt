APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The user is unable to rename or delete the queue manager's
   certificate from the queue manager's key database.
   
   From the runmqakm program the following error text is seen:
   CTGSK2005W A database error occurred.
   
   From the runmqckm program the following error text is seen:
   An invalid password was provided, the key database has been
   corrupted or it is of the wrong type. Expected database type:
   CMS
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   Users with certificates stored in key repositories, when
   attempting to delete or rename a certificate.
   This issue affects users of the GSKit included with all levels
   of MQ 7.5.
   This issue affects users of the GSKit included with MQ 8.0.0.0
   and MQ 8.0.0.4.
   This issue does not affect users of the GSKit included with MQ
   8.0.0.1 through MQ 8.0.0.3.
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   The bundled GSKit software contained a defect which caused this
   problem.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The bundled GSKit defect has been fixed.
