ERROR DESCRIPTION
 *  SET CHLAUTH command used with ADDRESS (' ')
   leads to abend 5C5 with RC 00D400A6 in CSQMSCA
   (set CHLAUTH processor).
   
   Qmgr trace shows in CEXIT for rsxSetChlAuthMap
   return code 20009231.
   
   This return code means rrcE_INVALID_PARAMETER
   
   The parameter ADDRESS(' ') is causing this return code that
   results in this abend.
   
   
    
   
   

LOCAL FIX
 *  add a value for the ADDRESS parameter
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All users of WebSphere MQ for z/OS Version 8 *
   *                 Release 0 Modification 0.                    *
   ****************************************************************
   * PROBLEM DESCRIPTION: SET CHLAUTH('xx') TYPE(ADDRESSMAP) with *
   *                      keyword ADDRESS(' ') generates error    *
   *                      message CSQ9017E and abend 5C6-00D400A6 *
   *                      in CSQMSCA.                             *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When command SET CHLAUTH('xx') TYPE(ADDRESSMAP) ADDRESS(' ') is
   used to add or remove channel authentication records generates
   abend 5C6-00D400A6 in CSQMSCA to report that ADDRESS(' ')
   parameter is invalid.
   
   
    
   
   

PROBLEM CONCLUSION
 *  SET CHLAUTH('xx') TYPE(ADDRESSMAP) has been changed to issue
   error messages CSQM192I and CSQ9023E to report that parameter
   ADDRESS(' ') is invalid and that the command has terminated
   abnormally. No abends are generated.
