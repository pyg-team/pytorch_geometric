APAR STATUS
 * CLOSED AS NEW FUNCTION.
    
   
   

ERROR DESCRIPTION
 *  Any URI which contains the j_security_check keyword keeps
   triggering the login process even though a user is already
   authenticated.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  IBM WebSphere Application Server version    *
   *                  7.0 and 8.0 users                           *
   ****************************************************************
   * PROBLEM DESCRIPTION: URI which contains the string           *
   *                      j_security_check always redirects the   *
   *                      user to a login page.                   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   WebContainer always invokes the login process whenever the
   keyword j_security_check is presented anywhere in the request
   URI. This can prevent any resource&#131;??s name, which relates to
   or contains the j_security_check keyword, from being served.
   For example:  http://www.my.example/j_security_check_FAQ [http://www.my.example/j_security_check_FAQ]
   always redirects the user to login even though the
   j_security_check_FAQ is a non-secured resource in the server.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The WebContainer code was modified to allow the
   j_security_check keyword to be used as part of the resource&#131;??s
   name.  The login process is only invoked for URIs that end
   with only the string /j_security_check
   
   A new WebContainer customer property needs to be set to enable
   the behavior provided by this apar:
   
   com.ibm.ws.webcontainer.enableExactMatchJSecurityCheck =  true
   (default is false)
   
   Please refer to the following technote for instructions on
   enabling WebContainer custom properties:
   http://www.ibm.com/support/docview.wss?rss=180&uid=swg21284395 [http://www.ibm.com/support/docview.wss?rss=180&uid=swg21284395]
   The fix for this APAR is currently targeted for inclusion in
   fix packs 7.0.0.19 and 8.0.0.1
   Please refer to the recommended updates page for delivery
   information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
