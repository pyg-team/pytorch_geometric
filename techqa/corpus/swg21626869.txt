Title: IBM TIP 2.2 fix pack install fails on eWAS fix pack install step (WASFP) - United States

Text:
TIPL2; TIPL2INST; Tivoli Integrated Portal; installation; process_spawner; access denied; WASFP; 2.2.0.7; 2.2.0.9; 2.2.0.11 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Tivoli Integrated Portal 2.2 fix pack (e.g. 2.2.0.x) install fails on the "step_0000x_WASFP" step on a Windows platform. The corresponding <TIP HOME>/logs/update/<WAS FP>/update.log shows an error on "process_spawner.exe (Access is denied.) 

CAUSE
The cause has not yet been determined

RESOLVING THE PROBLEM
While the root cause has not yet been determined, the following procedure can be used to get the TIP fix pack installed:



 1. Copy all *.pak files from:
    
    <Fix pack unpack location>/cdimage.fixpack/COI/PackageSteps/WASFP/FILES/
    
    to:
    
    <TIP HOME>\WebSphereUpdateInstallerV7\maintenance\
    
 2. Stop TIP via the Windows service.
    
    
 3. Run the WAS Update Installer:
    
    <TIP HOME>\WebSphereUpdateInstallerV7\Update.exe
    
    click "Next" through the install panels to apply maintenance packages.
    
    
 4. If the packages install indicates "Success", rerun the TIP 2.2. fix pack installer: 
    
    Note: See the following DCF for addition required steps for TIP fix pack 2.2.0.13 or later:
    http://www-01.ibm.com/support/docview.wss?uid=swg21694191 [http://www-01.ibm.com/support/docview.wss?uid=swg21694191]
