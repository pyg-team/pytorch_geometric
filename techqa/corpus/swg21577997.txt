Title: IBM Illegal key size or default parameters error in IBM FileNet Process Engine - United States

Text:
UTCryptoKeyFile; PE; AE TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempts to launch workflow in Process Enginee may see the following error:

Problem initializing encryption/decryption with keyId b2f8d3e4, size 256 bits. java.home=D:\Program Files\IBM\WebSphere\AppServer\java\jre. Cause: java.security.InvalidKeyException: Illegal key size or default parameters. 

CAUSE
The crypto key file is either missing or corrupted after installing IBM FileNet Application Engine

ENVIRONMENT
Windows, Unix

RESOLVING THE PROBLEM
To resolve, follow steps below to generate new encryption keys: 

 1. Backup the UTCryptoKeyFile.properties file in the <AE_installed_location>\Authentication\ directory then rename or delete it.
    
    For example, in Windows the path should look like this:
    C:\Program Files\FileNet\Authentication\UTCryptoKeyFile.properties 
    
    In Unix: 
    /opt/FileNet/Authentication/UTCryptoKeyFile.properties
    
 2. Recreate the UTCryptoKeyFile.properties file by running the following command on a Windows system (change the path accordingly on a Unix system):
    
    C:\Program Files\FileNet\AE\Workplace\WEB-INF\lib>java -cp javaapi.jar com.filenet.wcm.api.util.MakeCryptoKeys -n 3 -s 128 > "C:\Program Files\FileNet\Authentication\UTCryptoKeyFile.properties" 
    
 3. Restart the Application Engine server.
