ERROR DESCRIPTION
 *  How do we suppress the IBM Attached Tools Directory from being
   created
   for Secure+? It is creating security issues by leaving behind
   word
   writable files
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   Sterling Connect:Direct for UNIX 4.2.0
   Problem Description:
   Invoking spadmin.sh, spcli.sh, or lcu.sh creates a
   /tmp/.com_ibm_tools_attach directory used by the Java Attach
   API.  The Java Attach API is not used in C:D, so the /tmp
   directory creation is unnecessary and seen by some as a
   possible security risk.
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  Resolution Summary:
   Fixed spadmin.sh, spcli.sh, and lcu.sh so that no
   /tmp/.com_ibm_tools_attach directory is created.
   Delivered In:
   Sterling Connect:Direct for UNIX 4.2.0.4.iFix065
