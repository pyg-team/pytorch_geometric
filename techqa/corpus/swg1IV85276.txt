ERROR DESCRIPTION
 *  AWSFAB025E You are updating or uninstalling, but the
   installation script cannot find an instance of the product in
   the IBM Workload Scheduler registry file. The registry file:
   C:\Windows\TWSRegistry.dat must contain an entry belonging to
   the following user: "BPPDDPP".
   Try to regenerate the installation registry running the command
   with the -recovInstReg option.
   .
   This error appears in the selfupdate.log file when the
   centralize update is called from TDWC 9.3.0. when the tws user
   is in lower case in the TWSRegistry.dat file and TDWC pass the
   user in uppercase to the twsinst.vbs script .
   
   
    
   
   

LOCAL FIX
 *  set in upper case the user name in TWSregistry.dat
   
   as TDWC is passing it in UPPERCASE.
   
   
    
   
   

PROBLEM SUMMARY
 *  This error AWSFAB025E appears in the selfupdate.log file when
   the centralize update is called from TDWC 9.3.0. when the tws
   user is in lower case in the TWSRegistry.dat file and TDWC pass
   the user in uppercase to the twsinst.vbs script
   
   
    
   
   

PROBLEM CONCLUSION
 *  This APAR will be fixed in 9.3.0-TIV-TWS-FP0003
