QUESTION
 How can you configure the T6 agent to support RPT scripts that use Entrust TruePass authentication? 

ANSWER

In Rational Performance Tester (RPT) 8.6 you can run tests against servers that require Entrust TruePass authentication. Roaming mode with TruePass applet version 7.0 and later are supported. Local mode, and versions of the TruePass applet prior to 7.0, are not supported.

Note: The minimum supported version of RPT is 8.6.0.2. This also requires hotfix R49670_TLS_1.2_8602_UpdateSite on the RPT workbench. Obtain the hotfix from Rational support.

Entrust TruePass scripts that run correctly in RPT workbench generate java.awt.HeadlessException errors like the following in the trace-robotic log when the scripts playback through the T6 agent:

[date-time] - MIN - <t6-server> - PlaybackThreadPoolWorker-<thread-id> - com.ibm.tivoli.transperf.playback.rpt.RPTPlaybackService.processStatusEvent() - Sending RT VP event: i=|0|, statusEvent=|SimTestStatusEvent Values: monitorName=|<profile-name>/<script-name>/|,URL=|null|,eventType=|null|,eventId=|2288.b96|,parentEventId=|2288.b78|,ownerId=|A1E5346629D399B0C29BB83730666635|,text=|Unable to load Entrust applet/class: java.awt.HeadlessException ...

Use the following procedure to allow these scripts to run under the T6 agent:

1) If the script uses custom jar files, see the following technote:

ITCAM4Tx - error with playback of custom coded RPT script on RRT T6 agent
http://www.ibm.com/support/docview.wss?uid=swg21459054 [http://www-01.ibm.com/support/docview.wss?uid=swg21459054]

Note: Do not set the Djava.awt.headless property in the managed_jvm.xml file on the T6 agent host.

2) Verify that xorg-x11-xauth is installed. 

yum install xorg-x11-xauth 

3) Add the following line in ITM_HOME/config/t6.ini. 

DISPLAY=:0.0 

4) Add the following line in /root/.bash_profile. 

export DISPLAY=:0.0 

5) Restart the T6 agent.
