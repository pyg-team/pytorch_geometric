Title: IBM PK93170; 6.1.0.5: ProcessCPU only monitor usage for 512 threads. - United States

Text:
PK93170; PK93170; PK93170 DOWNLOADABLE FILES

ABSTRACT
 PROCESS_ID_MAX_SIZE is too small is repeatedly logged in the log file native_stdout.log 

DOWNLOAD DESCRIPTION
PK93170 resolves the following problem:

ERROR DESCRIPTION: 
ERROR: ProcessCPU getCPU(timeval) PROCESS_ID_MAX_SIZE is too small is repeatedly loged in the native_stdout.log 

LOCAL FIX: 
N/A 

PROBLEM SUMMARY

USERS AFFECTED:
All users of WebSphere Extended Deployment on the operating system of Linux

PROBLEM DESCRIPTION:
PROCESS_ID_MAX_SIZE is too small is repeatedly logged in the log file native_stdout.log

RECOMMENDATION:
Apply PK93170

ProcessCPU only monitor usage for 512 threads. When this limit is breached the message "PROCESS_ID_MAX_SIZE IS TOO SMALL IS REPEATLY LOGGED" is logged in the log file native_stdout.log.

PROBLEM CONCLUSION: 
The code was updated to log a max of 10 messages to prevent log flood. The code was modified to monitor usage for 1,024 threads per process.
