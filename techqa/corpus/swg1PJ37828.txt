ERROR DESCRIPTION
 *  Description of Problem (full details) Arabic text disappears or
   becomes rubbish after using the print option to pdf on eForms
   4.0.2 patch 4
   
   How long has the problem been occurring (recent changes)? :
   always, never worked
   
   Does this occur on more than one station / server? : test and
   production system
   
   Is there a workaround? : NO
   
   What is the impact to the customer/system? : Cannot print eForms
   correctly
   
   
   Steps to Reproduce:
   
   1 Install the below configuration
   
   2 Build an eForm with the Designer and Arabic language entries,
   using
   one font type or different fonts
   
   3 Deploy the template on Workplace as both Document
   Policy/Workflow Policy
   
   4 In Workplace open the eForm and use the print option in
   the form; i.e., render the form to PDF.
   
   
   5 See the missing Arabic language and rubbish language as well.
   
   Actual Results: Arabic language gets deleted and or becomes
   rubbish
   
   Expected Results: To be able to see the eForm in PDF mode with
   the same basic look as within P8eF (html mode).
   
   Configuration/Environment :
   
   CE is 4.5.1
   
   PE is 4.5.1
   
   AE is 4.0.2 _patch 4
   
   5.0.3.2-eFD-FP002
   
   eForms is 4.0.2 patch 4
   
   database is MS-SQL 2008
   
   IBM WebSphere is 7.0.0.5
