APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  During the Typical Installation from launchpad, in the
   "Installation summary" panel, customer has the option to select
   "Use your support account into include updates with the
   installation".
   
   With valid IBM ID and password entered, the Typical
   Installation returns with warning of:
   "Sorry. Your credentials were not accepted. Please try agian.
   Launchpad has not searched for available fixes because it
   cannot currently connect to the update repository
   "http://public.dhe.ibm.com/software/websphere/bpm/respositories/
   server/wbpm/855/updates/"
   
   
    
   
   

LOCAL FIX
 *  De-select the option
   "Use your support account to include updates with the
   installation"
   and install any additional APAR fixes from Installation Manager
   afterwards.

PROBLEM CONCLUSION
 *  A fix is available for IBM BPM V8.5.5.0  that corrects the
   incorrect update repository URL.
   If you have internet access, you are prompted to update the
   launchpad when you start launchpad. Click Update.
   If you do not have internet access, you can modify the following
   properties file to direct Installation Manager to the updates.
   1.From
   http://public.dhe.ibm.com/software/websphere/bpm/repositories/se [http://public.dhe.ibm.com/software/websphere/bpm/repositories/se]
   rver/launchpad/855/, download the interim fix to you local
   system.
   2.Create the following file:
   Windows
    C:/users/user_name/bpm_updates.properties
   Linux or Unix
   /user_home_directory/bpm_updates.properties
   3.Add code in bpm_updates.properties to point the location of
   the interim fix in your local system, for example
   
   launchpad.1=interimfix_location
   
   where interimfix_location is the local folder of the interim fix
   you downloaded in step 1.
   4.Start launchpad. You are prompted to update the launchpad.
   5.Click Update.
