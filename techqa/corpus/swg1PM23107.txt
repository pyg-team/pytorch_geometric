Title: IBM PM23107: CQWeb (IHS) leaks server type and version in its response messages. - United States

Text:

APAR STATUS
 * CLOSED AS PERMANENT RESTRICTION.
    
   
   

ERROR DESCRIPTION
 *  The default settings in the IHS installed with CQWeb 7.1.x are
   such that the response messages from the CQWeb server include
   the HTTP server type and version in their headers.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The default settings in the IBM HTTP SERVER (IHS) installed
   with IBM Rational ClearQuest 7.1.x are such that the
   response messages from the ClearQuest Web server include the
   HTTP server type and version in their headers.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The IBM HTTP Server (IHS) configuration as installed with
   ClearQuest follows the standard IHS installation
   configuration. Technote # 1497838 has been created to point
   out this configuration issue and describes how it can be
   addressed.
