Title: IBM Workplace XT releases prior to 1.1.5.2-WPXT-IF001 will encounter Process Engine applet exceptions with 5.2.0.3-P8CPE-FP003 - United States

Text:
PE applets; Process Engine; WPXT; launching; WorkplaceXT TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After upgrading Content Platform Engine (CPE) to 5.2.0.3-P8CPE-FP003, customers using Workplace XT releases prior to 1.1.5.2-WPXT-IF001 will not be able to launch Process Engine applets such as Process Administrator and Process Configuration Console.


SYMPTOM
1. Unable to launch Process Engine applets from Workplace XT. 

2. Splash screen for the applet appears to hang.
3. Errors like java.lang.ClassNotFoundException: com.filenet.api.core.EngineObject are seen in the Java plug-in console on the client (browser) workstation.

CAUSE
The Jace.jar file is not loaded into the Java Cache, so the com/filenet/api/core/EngineObject class cannot be found.



RESOLVING THE PROBLEM
The recommended solution is to install or upgrade to 1.1.5.2-WPXT-IF001 (released Dec 23, 2013) or higher, which will completely resolve the issue. 


If unable to install 1.1.5.2-WPXT-IF001, the issue can be resolved by using a manual workaround involving the appletParameters.properties file.

Follow one of the two options below depending on whether any customizations have been made to the appletParameters.properties file (normally located in the <installpath>\IBM\FileNet\WebClient\WorkplaceXT\WEB-INF directory).

1. If no customizations have been made to the appletParameters.properties file in the installed directory of WorkplaceXT/WEB-INF

Replace the appletParameters.properties file in the installed directory of WorkplaceXT/WEB-INF with this appletParameters.properties file:
[/support/docview.wss?uid=swg21669964&aid=2]appletParameters.properties [/support/docview.wss?uid=swg21669964&aid=1]

Then follow the rebuild and redeployment instructions below.

or

2. If the appletParameters.properties file in the installed directory of WorkplaceXT/WEB-INF has been customized.

Open the appletParameters.properties file in the installed directory of WorkplaceXT/WEB-INF and make the following changes in bold.

# PE Applets Default jar/versions

Once appletParameters.properties file has been modified, rebuild the EAR and/or WAR file using the tools in the <installpath>\IBM\FileNet\WebClient\deploy directory.

After rebuilding the EAR file, redeploy the application to the application server. 

After successfully redeploying the EAR file, the Process Engine Java applets should now open correctly.
