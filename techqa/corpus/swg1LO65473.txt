Title: IBM LO65473: IMPORTED HOLIDAYS NOT DISPLAYING ON CORRECT DAYS ON SOME NOKIA S^3 DEVICES. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Customer reported that on some Nokia S^3 devices imported
   Holidays are showing up way off on the calendar, like months
   off. It appears after some investigation related to the devices
   not handling certain date formats correctly.  Working with Nokia
   on a fix.
   
   
    
   
   

LOCAL FIX
 *  NA
   
   
    
   
   

PROBLEM SUMMARY
 *  Customer reported that some imported Holidays were way off when
   synced to certain Nokia devices.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The problem was tracked to how some Nokia devices handle certain
   date time formats.  Updates to Lotus Traveler server and client
   for Nokia were needed to work around the issue.
   
   
    
   
   

TEMPORARY FIX
 *  This fix will be included in Lotus Traveler 8.5.3.2 and later
   releases.  For the latest recommended maintenance please see
   this technote:
   http://www.ibm.com/support/docview.wss?uid=swg24019529 [http://www.ibm.com/support/docview.wss?uid=swg24019529]
