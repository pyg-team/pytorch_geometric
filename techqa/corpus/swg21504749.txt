Title: IBM C&DS 4.2 installation issues with EAP 4.3.0, unable to start server. - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After installing Collaboration & Deployment Services 4.2, with Application Server JBoss Redhat 4.30, C&DS will not start. 

SYMPTOM
After installing C&DS 4.2 with Application Server Jboss Redhat 4.3.0: 

1. Windows Service is not created. 

2. Startserver.bat will not start the C&DS server. Errors: 

11:11:46,487 WARN [ServiceController] Problem starting service jboss.cache:service=TomcatClusteringCache 

 Caused by: java.lang.Exception: exception caused by UDP.start() 

 Caused by: java.lang.Exception: problem creating sockets (bind_addr=USUSHWSWD204/3.156.32.190, mcast_addr=230.1.2.7:45577) 

11:11:49,034 WARN [ServiceController] Problem starting service jboss:service=DefaultPartition org.jgroups.ChannelException: failed to start protocol stack org.jgroups.ChannelException: failed to start protocol stack 

 Caused by: java.lang.Exception: exception caused by UDP.start() 

 Caused by: java.lang.Exception: problem creating sockets (bind_addr=USUSHWSWD204/3.156.32.190, mcast_addr=228.1.2.3:45566) 

11:11:49,237 WARN [ServiceController] Problem starting service jboss.cache:service=EJB3SFSBClusteredCache 

org.jgroups.ChannelException: failed to start protocol stack at org.jgroups.JChannel.connect(JChannel.java:389) 

.. and so on.


ENVIRONMENT
Installation Environment: 

C&DS 4.2 

Jboss EAS (JBoss [EAP] 4.3.0.GA_CP09) 

Java 1.6 0_23 

Windows Server 2008 R2 64 bit 

SQL Server 2008 R2 

Standalone application server selected (not cluster)


RESOLVING THE PROBLEM
Reinstalled C&DS using Jboss Redhat 5.0 and customer is now able to start C&DS.
