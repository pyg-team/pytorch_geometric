APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Portal server1 admin not up & not showing the default view when
   it is up if portal profile created using portal profile
   template.
   
   First issue:
   The default view is set to My Tasks so there is a white frame
   on the right and on the left
   - There are no tasks currently selected as 'My tasks'
   - Add tasks
   Expected results:
   I would expect the view to be set to WebSphere Application
   Server like the one that comes with default profiles server1
   
   Second issue:
   Reproduction scenario:
   Install Portal 70
   Run enable-profiles
   Create a portal profile using
   Run the following command from the AppServer_root/bin
   directory:
   manageprofiles.sh -create -templatePath
   /usr/IBM/WebSphere/PortalServer/profileTemplates/default.portal
      -profileName snj_profile
      -profilePath /usr/IBM/WebSphere/snjprofile
      -cellName snjCell
      -nodeName snjNode
   
   Start server1 for the 2nd profile
   Access server1 console and get message
   SRVE0255E: A WebGroup/Virtual Host to handle /ibm/console has
   not been defined
   
   Server1 SystemOut.log shows
   
   [10/14/10 15:29:50:050 EDT] 00000010 webcontainer E
   com.ibm.ws.webcontainer.WebContainer handleRequest SRVE0255E: A
   WebGroup/Virtual Host to handle /favicon.ico has not been
   defined.
   [10/14/10 15:33:09:916 EDT] 00000010 util W
   com.ibm.ws.webcontainer.util.VirtualHostContextRootMapper map
   SRVE0316W: Request matches the context root [/ibm/console/*]
   under the virtual host alias of [*:10042].
   [10/14/10 15:33:09:929 EDT] 00000010 util W
   com.ibm.ws.webcontainer.util.VirtualHostContextRootMapper map
   SRVE0317W: You may need to add a new virtualhost alias of *:
    to the same virtual host that [*:10042] is under.
   [10/14/10 15:33:09:941 EDT] 00000010 webcontainer E
   com.ibm.ws.webcontainer.WebContainer handleRequest SRVE0255E: A
   WebGroup/Virtual Host to handle /ibm/console has not been
   defined.
   
   Expected results:
   Server1 of secondary profiles should be accessible after
   creating additional profiles
   
   Actual results:
   The virtual host for 2ndary profiles is not set correctly
   
   This APAR is a follow-up of PM28138 [http://www-01.ibm.com/support/docview.wss?uid=swg1PM28138].
   Opened for release 7001
