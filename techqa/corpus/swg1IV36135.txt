LOCAL FIX
 *  The exception occurred only when there is no match. Hence, the
   application can catch java.lang.StringIndexOutOfBoundsException
   and process it as a false condition.
   
   
    
   
   

PROBLEM SUMMARY
 *  Pattern match loop increment was not updated properly when input
   text has UTF-16 surrogate pair characters
   
   
    
   
   

PROBLEM CONCLUSION
 *  This defect will be fixed in:
   6.0.0 SR14
   6.0.1 SR6
   
   .
   Pattern match loop increment is modified to handle UTF-16
   surrogate pair.
