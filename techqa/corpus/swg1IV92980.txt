ERROR DESCRIPTION
 *  When installing the Secondary Impact server with multiple NICs
   in a cluster, the install fails with the error :
   
   18 Nov 2016 21:28:00,008 ERROR [ClusterMember] Error in calling
   syncUpWithNameserver()
   com.micromuse.response.common.RespException: Error converting
   role to Secondary. Exception:
   com.micromuse.response.common.RespException: Cannot get access
   to Primary cluster member
    at
   com.micromuse.response.broker.cluster.ClusterMember.syncUpWithNa
   meserver(ClusterMember.java:1696) ....
   
   To resolve this networking problem on a Secondary(and Primary)
   Impact servers
   with multiple NICs/IP adresses, please ensure the Impact IP
   address being used on its Primary AND Secondary servers, that is
   the IP address returned by pinging the host returned by the
   hostname command is the same IP address as that used by Impact
   in its :
   
   impact.server.ipaddress=<ipaddress>
   
   in its <impact home>/etc/<impact server name>_server.props
   file.
   
   Following these changes, please reattempt your install
   
   
    
   
   

LOCAL FIX
 *  Workaround outlined above
