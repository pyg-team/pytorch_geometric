 TECHNOTE (FAQ)

QUESTION
 Why I am getting the following exception when trying to access to Decision Center through Webseal:

7/24/13 16:19:00:008 CEST] 00000031 SecurityCheck E Invalid request 
[Referer - smps2scp/inteodm/teamserver] 
GET /teamserver/faces/home.jsp?r=jSLXwK7+jgOJH/yGgly2VG 
+n6d6ZJemRruCWRt7eUla5gsfOq6KCfrLBkA8Cl2fe 
Parameters: 
r: jSLXwK7 jgOJH/yGgly2VG n6d6ZJemRruCWRt7eUla5gsfOq6KCfrLBkA8Cl2fe 

ANSWER
This error message comes from Decision Center for checking the value of the Referer header against a pattern.

It is possible to disable this check for Decision Center Enterprise Console.


 * Edit the file jrules-teamserver-WAS8.ear/teamserver.war/WEB-INF/web.xml 
 * Change the value of the parameter ilog.rules.teamserver.check-referer from true to false under the securityCheckPointFilter filter section 
 * Save the changes 
 * Repack the teamserver.war file and re-deploy the modified jrules-teamserver-WAS8.ear file to the application server


For the Decision Center Business Console, comment out the securityCheckPointFilter  * Edit the file jrules-teamserver-WAS8.ear/decisioncenter.war/WEB-INF/web.xml 
 * Comment out the securityCheckPointerFilter section:

<filter-mapping>
<filter-name>securityCheckPointFilter</filter-name>
<url-pattern>/*</url-pattern>
</filter-mapping>  * Save the changes 
 * Repack the decisioncenter.war and re-deploy the modified jrules-teamserver-WAS8.ear file to the application server
