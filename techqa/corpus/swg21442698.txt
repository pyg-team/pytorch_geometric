Title: IBM Blank spaces in the value of the default_charset_value parameter might cause unexpected errors - United States

Text:
Configuration TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When a properties file has an invalid default encoding character set, import jobs fail with exception. 

SYMPTOM
Import jobs fail with UnsupportedEncodingException exception.


CAUSE
The presence of blank spaces in the default_charset_value parameter value causes the parameter value to become invalid.

DIAGNOSING THE PROBLEM
Check the GUI and Logs for an UnsupportedEncodingException exception.

RESOLVING THE PROBLEM
Check the syntax and value of the default_charset_value parameter in the $TOP/etc/default/common.properties file. 


The syntax of a valid parameter value is "default_charset_value=Value". For example, "default_charset_value=Cp1252". 

An example of an invalid syntax is "default_charset_value=<blankspace>Value" or "default_charset_value=value<blankspace>". For example, "default_charset_value=Cp1252 " or "default_charset_value= Cp1252". 

Simply remove any blank spaces that might be present in the value of the default_charset_value parameter.
