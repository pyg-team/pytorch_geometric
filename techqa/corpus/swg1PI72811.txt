PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server                                      *
   ****************************************************************
   * PROBLEM DESCRIPTION: Applications will not start if an       *
   *                      alternative bean was vetoed.            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   CDI throws an exception if an alternative defined in beans.xml
   does not exist. However the technical implimentation means that
   vetoed beans do not exist meaning this exception can be thrown
   when it is unexpected.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A new system property is added. If
   org.jboss.weld.bootstrap.allowExcludedAlternatives is set to
   true it becomes legal to veto beans defined as alternatives.
   See
   https://docs.jboss.org/weld/reference/latest/en-US/html/configur [https://docs.jboss.org/weld/reference/latest/en-US/html/configur]
   e.html#allow_vetoed_alternatives for more details
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 9.0.0.4.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
