   Steps to reproduce:
   Go to Integration > External Systems
   Query EXTSYS1 (make sure it is enabled)
   Click on External Systems tab
   Select MXCOAInterface
   Click on Data Import button
   Check Import Preview and select an xml file with the following
   xml data saved to the file:
   <?xml version="1.0" encoding="UTF-8"?>
   <SyncMXCOA xmlns="http://www.ibm.com/maximo"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   creationDateTime="2015-07-02T15:59:35-04:00" transLanguage="EN"
   baseLanguage="EN" messageID="1435867175424912228"
   maximoVersion="7 5
   20140827-1420 V7512--1" event="0">
     <MXCOASet>
       <CHARTOFACCOUNTS action="AddChange">
         <ACTIVEDATE>2011-02-10T16:00:00-04:00</ACTIVEDATE>
         <EXPIREDATE>2015-06-18T14:18:45-04:00</EXPIREDATE>
         <ACTIVE>0</ACTIVE>
         <GLACCOUNT>
           <VALUE>6000-229-000</VALUE>
         </GLACCOUNT>
         <ORGID>EAGLENA</ORGID>
       </CHARTOFACCOUNTS>
     </MXCOASet>
   </SyncMXCOA>
   If you remove <ACTIVEDATE> you get this error:
   BMXAA5598E - Processing of an inbound transaction failed. The
   processing exception is identified in document 1.
       BMXAA1169E - Expiration date must be after the active date.
   
   
    
   
   

LOCAL FIX
 *  change the GLACCOUNT format it should work
   from:
   <GLACCOUNT>
           <VALUE>6000-229-000</VALUE>
   </GLACCOUNT>
   to:
   <GLACCOUNT>6000-229-000</GLACCOUNT>
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * Maximo 7508 users who used MIF COA to import Chartofaccounts *
   * to set expiration date of a chartofaccount.                  *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * User could not use the same xml to import chartofaccounts to *
   * maximo to set expiration date of a chartofaccount in MAXIMO  *
   * 7508. The same xml used to work in 7507 and earlier          *
   * releases.                                                    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Change the GLACCOUNT in the import xml                       *
   *                                                              *
   * from:                                                        *
   *                                                              *
   * <GLACCOUNT>                                                  *
   *         <VALUE>6000-229-000</VALUE>                          *
   * </GLACCOUNT>                                                 *
   *                                                              *
   * to:                                                          *
   *                                                              *
   * <GLACCOUNT>6000-229-000</GLACCOUNT>                          *
   ****************************************************************
