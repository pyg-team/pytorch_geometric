CAUSE
This may be caused by Eclipse Bugzilla 264595 [https://bugs.eclipse.org/bugs/show_bug.cgi?id=264595] which is a duplicate of 236724 [https://bugs.eclipse.org/bugs/show_bug.cgi?id=236724]. See the Related information section.



RESOLVING THE PROBLEM
Solution A 

The Eclipse bugs are resolved in Eclipse 3.4.1 which is contained in Rational Application Developer version 7.5.1. 

Solution B 

Set MOZILLA_FIVE_HOME if browser is not installed dynamically (for example not installed as a Linux RPM package and just downloaded/extracted). 

You need to set the environment variable MOZILLA_FIVE_HOME to the folder containing your Mozilla Firefox installation. Refer to Eclipse SWT FAQ in the Related information section. 

Starting the product from command line
1. Open up command line terminal and go to the product install directory, opt/IBM/SDP75/). 
2. Enter MOZILLA_FIVE_HOME=/usr/lib/<location of Mozilla or Firefox>. 
3. Enter export MOZILLA_FIVE_HOME. 
4. Start the product using ./eclipse.

Starting the product from the menu
1. Open up command line terminal and enter gedit $HOME/.base_profile. 
2. Add the line MOZILLA_FIVE_HOME=/usr/lib/<location of Mozilla or Firefox>. 
3. Add the line export MOZILLA_FIVE_HOME 
4. Save and close the file. 
5. Log out and log back in. 
6. Start the product from the menu. 

See also the other linked technotes for resolution of other similar start-up issues.
