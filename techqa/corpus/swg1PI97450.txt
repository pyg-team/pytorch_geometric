PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All Content Manager OnDemand for z/OS 10.1   *
   *                 and above customers using arsdb -r.          *
   ****************************************************************
   * PROBLEM DESCRIPTION: arsdb -r fails with:                    *
   *                                                              *
   *                      ARS4125I Failed while processing sql    *
   *                      statement >CREATE UNIQUE INDEX          *
   *                      ARSSV100.arsapp_idx ON ARSSV100.arsapp  *
   *                       agid, aid, name                        *
   *                      INCLUDE  description, doc_type,         *
   *                      doc_comp_type, res_comp_type, idx_type, *
   *                      comp_obj_size, pid, upd_userid,         *
   *                      upd_date, db_exp_date,<                 *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   arsdb was trying to INCLUDE a list of columns in the index.  The
   variable used to contain the string is not big enough hold the
   entire string, causing the string to be truncated, and thereby
   creating an invalid SQL statement.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The variable is made large enough to hold the entire string.
