APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When trying to generate a report manually via the java
   interface (tools >> reports >> Define / Run) the following
   message is generated in the browser.
   "XQE-PLN-0148 User defined SQL is not permitted.
   When we access the IBM Control Center via WEB and select the
   "group work area" option, the following error is generated:
   "DPR-ERR-2058
   
   
    
   
   

LOCAL FIX
 *  STRRTC - 572446
   SF / SF
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Description of issue: Sometimes java console users get error
   XQE-PLN-0148 when attempting to run a report and web console
   users get error DPR-ERR-2058 when going to the workspaces.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Description of fix:  ICC roles are only being added to Cognos
   accounts when the Cognos Authentication Module search method
   when it needs to be called with the first authentication to
   ensure its in place at the time needed.  Added the loading of
   ICC users and roles to Cognos accounts at the time ICC connects
   to Cognos for the first time.
   
   
    
   
   

TEMPORARY FIX
 *  IT25792/R572446 (2018-07-25)
   Description of issue: Sometimes java console users get error
   XQE-PLN-0148 when attempting to run a report and web console
   users get error DPR-ERR-2058 when going to the workspaces.
   Description of fix:  ICC roles are only being added to Cognos
   accounts when the Cognos Authentication Module search method
   when it needs to be called with the first authentication to
   ensure its in place at the time needed.  Added the loading of
   ICC users and roles to Cognos accounts at the time ICC connects
   to Cognos for the first time.
