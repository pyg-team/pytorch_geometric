APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When running firmware versions containing APAR IC93084 [http://www-01.ibm.com/support/docview.wss?uid=swg1IC93084] and prior
   to this APAR, messages similar to the following might be seen in
   the logs when using SFTP:
   20140224T155640Z [0x80e005c7][ssh][info] : SSH client:  Killed
   by signal 15.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Affected are Datapower firmware releases 5.0.0.11-5.0.0.13,
   6.0.0.3-6.0.0.5 and 6.0.1.0-6.0.1.1 with configurations using
   SFTP client, i.e. SFTP Poller front-side handler or sftp:
   backend URLs.
   
   An info-level message "SSH client: Killed by signal 15." might
   be logged for transactions using the SFTP protocol. This message
   is benign and can be safely ignored.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in 5.0.0.14, 6.0.0.6 and 6.0.1.2.
   
   For a list of the latest fix packs available, please see:
   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
