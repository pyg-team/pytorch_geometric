Title: IBM Silent installation exits abnormally with InvocationTargetException and StackOverflowError - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 In some situations, silent installation of InfoSphere Warehouse 9.7 can exit abnormally with InvocationTargetException and StackOverflowError. 

SYMPTOM
The silent installation exits with the following exception: 

 
Configuring the installer for this system's environment...

Launching installer...

Preparing SILENT Mode Installation...

Invocation of this Java Application has caused an InvocationTargetException. This application will now exit. (LAX)

Stack Trace:
java.lang.StackOverflowError
at java.lang.String.equals(String.java:560)
at java.util.Hashtable$Entry.equalsKey(Unknown Source)
at java.util.Hashtable.get(Unknown Source)
at com.zerog.ia.installer.util.VariableManager.b(DashoA10*..)
...


CAUSE
The problem occurs when the installation response file that is specified in the command line does not exist on the system. 


RESOLVING THE PROBLEM
Make sure that the response file exists on the system and specify the fully-qualified file name in the installation command.
