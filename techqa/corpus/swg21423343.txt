QUESTION
 Collecting data for problems with ILOG Business Rule Management Systems Components . Collecting this MustGather information before calling IBM support will help you understand the problem and save time analyzing the data. 

ANSWER
ILOG Business Rule Management System comprises a set of modules that operate in different environments, and also work together to provide a comprehensive BRMS. 
Namely the components are:


 * Rule Studio (RS) 
 * Rule Execution Server (RES) 
 * Decision Validation Services (DVS) 
 * Rule Team Server (RTS) 
 * Rule Solution for Office (RSO)


Properly identifying the component with the problem will significantly improve the support team's response time. 

What to do first 
 1. Find out if you are using the latest update of the product. Use the help About menu in your JRules component to determine the version currently installed. Use the following document to find the version of JRules components [http://www-01.ibm.com/support/docview.wss?uid=swg21400738].
    For version 6.X and 7.0 The version number is defined as follow V.R.U where V is the version R the release and U the update.
    For version 7.1 the version number is defined as follow V.R.M.F where V is the version number R is the release number M is the Modification number and F is the Fix pack number. (Read more in this document [http://www.ibm.com/support/docview.wss?uid=swg27008656])
    
 2. Check the lifecycle information to see if you should upgrade to a newer release or version.  * For version before 7 check here [http://www-01.ibm.com/software/support/lifecycle/index_j.html] 
     * For version 7 and later check here [http://www-01.ibm.com/software/support/lifecycleapp/PLCSearch.wss?q=WebSphere+ILOG&scope=&sort=P&ibm-search.x=12&ibm-search.y=9] search for the WebSphere ILOG products.
       
       
    
    
 3. Check the release notes for the updates and newer version of the product or component:  * For version 7 and later check these release notes [http://www-01.ibm.com/support/docview.wss?uid=swg27023079] 
     * For version before 7 check these release notes [https://www14.software.ibm.com/webapp/iwm/web/reg/download.do?source=swg-ilogspt&lang=en_US&cp=UTF-8&S_PKG=JRules] ( requires login to the IBM ILOG Legacy Support )
       
       
    
    
 4. Install the latest update or upgrade as possible. Find the download you need here [http://www-01.ibm.com/support/search.wss?rs=2025&tc=SS6MTS&dc=D400&dtm]. Use the information in the following document to proceed Upgrading JRules to the latest V7.1 fix pack [http://www-01.ibm.com/support/docview.wss?uid=swg21474292]
    
 5. Search information related to your specific issue or question in:  * The documentation located in the infocenter for 7.0 [http://publib.boulder.ibm.com/infocenter/brjrules/v7r0m3/index.jsp]or infocenter for 7.1 [http://publib.boulder.ibm.com/infocenter/brjrules/v7r1/index.jsp] 
     * The support site [http://www.ibm.com/support/troubleshooting/us/en/?cm_re=masthead-_-supdl-_-troubleshoot] for existing technotes [http://www-01.ibm.com/support/search.wss?rs=2025&tc=SW930&dc=DB560+DB520&dtm], published APARs [http://www-01.ibm.com/support/search.wss?rs=2025&tc=SW930&dc=DB550+DB100&dtm]
       
    
    
 6. When nothing seems to be helpful to resolve your problem or answer your question it is time for you to collect the specific information that illustrate your issue and contact IBM Support . Follow the direction according to the component you are using so that the support team receives all the data it needs for a timely resolution.
