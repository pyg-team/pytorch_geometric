Title: IBM Security Bulletin: IBM® Db2® is vulnerable to buffer overflow (CVE-2018-1459). - United States

Text:
buffer overflow; CVE-2018-1459 SECURITY BULLETIN

SUMMARY
 Db2's rah.exe is vulnerable to a stack-based buffer overflow, caused by improper bounds checking, which could lead an attacker to execute arbitrary code. 

VULNERABILITY DETAILS
CVEID: CVE-2018-1459 [http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2018-1459]
DESCRIPTION: IBM DB2 for Linux, UNIX and Windows (includes DB2 Connect Server) is vulnerable to stack based buffer overflow, caused by improper bounds checking which could lead an attacker to execute arbitrary code.
CVSS Base Score: 8.4
CVSS Temporal Score: See https://exchange.xforce.ibmcloud.com/vulnerabilities/140210 [https://exchange.xforce.ibmcloud.com/vulnerabilities/140210] for the current score
CVSS Environmental Score*: Undefined
CVSS Vector: (CVSS:3.0/AV:L/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H) 

AFFECTED PRODUCTS AND VERSIONS
All fix pack levels of IBM Db2 V9.7, V10.1, V10.5, and V11.1 editions on Windows platform are affected. Unix-type platforms are not affected.

REMEDIATION/FIXES
The recommended solution is to apply the appropriate fix for this vulnerability. 
FIX:
The fix for DB2 V11.1 is in V11.1.3.3 iFix001, available for download from Fix Central [http://www-01.ibm.com/support/docview.wss?uid=swg24044872]. 

Customers running any vulnerable fixpack level of an affected Program, V9.7, V10.1, and V10.5, can download the special build containing the interim fix for this issue from Fix Central. These special builds are available based on the most recent fixpack level for each impacted release: DB2 V9.7 FP11, V10.1 FP6, and V10.5 FP9. They can be applied to any affected fixpack level of the appropriate release to remediate this vulnerability.
