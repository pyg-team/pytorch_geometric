Title: IBM Ant task to build MTDS WAR fails with "<XOM class> does not have a no-arg default constructor" - United States

Text:
decision service; HTDS; WODM; JRules; Rule designer; Rule Studio TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The ANT task provided in Rule Studio to build the web archive (WAR) of a Monitored Transparent Decision Service (MTDS) fails. In the stack trace, the root cause points at an Execution Object Model (XOM) class that "does not have a no-arg default constructor". 

SYMPTOM

CAUSE
The JAX-WS API used to generate the web service component requires that all XOM classes define a default, public, no-argument constructor.

ENVIRONMENT
Monitored Transparent Decision Services (MTDS)



DIAGNOSING THE PROBLEM
Refer to the stack trace generated by the Ant task. 

Note that you might need to follow the instructions of technote Ant task to generate MTDS fails without giving an error [http://www.ibm.com/support/docview.wss?uid=swg21589438] to see the error in the Ant task output.


RESOLVING THE PROBLEM
Add a no-arg default constructor to all the XOM classes listed in the stack trace.
