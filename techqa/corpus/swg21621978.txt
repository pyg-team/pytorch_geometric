Title: IBM TRC LDAP authentication not working - United States

Text:
ldap; userid; login; failure TECHNOTE (FAQ)

QUESTION
 If logging-in on the TRC using LDAP user I get "Invalid username/wrong password. Login Failed." 

CAUSE
Error shows in the log:
4-Jan-2013,13:48:31 ¬WebContainer : 5| ¬INFO | - Authenticating with LDAP server 4-Jan-2013,13:48:32 ¬WebContainer : 5| ¬ERROR| - LDAP: Authentication failure : francy 4-Jan-2013,13:48:32 ¬WebContainer : 5| ¬INFO | - Invalid username/wrong password. Login failed. 


ANSWER
First check if you can login with the default TRC Admin user. If so, you need to edit the ldap.properties file and search for ldap.userid and check if sAMAccountName is used. 

This should be used in order to use the userid only portion of the logon (without the UPN Suffix). 

If this is the case check also that 
ldap.userSearch=(sAMAccountName={0})
