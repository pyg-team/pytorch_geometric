Title: IBM PM26637: AUGMENTING A PROFILE WITH OSGI/JPA2 FEATURE PACK CAUSES VARIOUS PROBLEMS WITH JNDI - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After a profile is augmented with the OSGi/JPA2 feature pack
   JNDI may stop working when anything other than a standard JNDI
   implementation without security enabled is used.  Numerous
   different symptoms are possible.  The following exeptions are
   examples:
   
   Using LDAP:
   javax.naming.NamingException: Cannot get schema object as
   DirContext:
   CN=<cn>,CN=<cn>,DC=<dc>,DC=<dc>,DC=
   ibm,DC=com; Remaining name: ''
   at com.sun.jndi.ldap.LdapCtx.buildSchemaTree(LdapCtx.java:1614)
   at com.sun.jndi.ldap.LdapCtx.getSchemaTree(LdapCtx.java:1575)
   at com.sun.jndi.ldap.LdapCtx.c_getSchema(LdapCtx.java:1516)
   at
   com.sun.jndi.toolkit.ctx.ComponentDirContext.p_getSchema(Compone
   ntDirContext.java:415)
   ...
   
   LDAP with security enabled:
   com.ibm.websphere.wim.exception.WIMSystemException: CWWIM4520E
    The 'javax.naming.NotContextException: Expected an LdapContext
   object.' naming exception occurred during processing.
   at
   com.ibm.ws.wim.adapter.ldap.LdapConnection.search(LdapConnection
   .java:2866)
   at
   com.ibm.ws.wim.adapter.ldap.LdapConnection.checkSearchCache(Ldap
   Connection.java:2709)
   at
   com.ibm.ws.wim.adapter.ldap.LdapConnection.search(LdapConnection
   .java:2886)
   ...
   
   This problem can cause an administrator to be unable to login
   to a secured admin console.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server OSGi/JPA2 Feature Pack               *
   ****************************************************************
   * PROBLEM DESCRIPTION: Augmenting a profile with the           *
   *                      OSGi/JPA2 feature pack may cause JNDI   *
   *                      to stop working.                        *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   OSGi interacts closely with JNDI as part of normal operation.
   Certain aspects of the implementation of OSGi JNDI failed to
   correctly handle different configurations of JNDI itself,
   particularly LDAP.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This APAR corrects the handling of the various possible JNDI
   configurations.
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 1.0.0.5 of the OSGi/JPA2 feature pack.  Please refer
   to the Recommended Updates page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
