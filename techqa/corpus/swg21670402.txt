Title: IBM MDM Remote RMI/EJB invocation failing with ORB error - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The MDM Remote RMI/EJB invocation can fail with the following Object Request Broker (ORB) error:

Error: Service bean could not be looked up on server iiop://hostname:2809 and provider url com/dwl/base/requestHandler/beans/DWLServiceController
com.ibm.mdm.tools.metadata.common.exceptions.MetadataException: javax.naming.NamingException: Error getting WsnNameService properties [Root exception is org.omg.CORBA.TRANSIENT: initial and forwarded IOR inaccessible vmcid: IBM minor code: E07 completed: No]

orbtrc.<timetstamp>.txt and orbmsg.<timestamp>.txt

CAUSE
The security exception CAUGHT_EXCEPTION_WHILE_CONFIGURING_SSL_CLIENT_SOCKET: JSSL0130E: java.io.IOException: signals that an I/O exception of some sort has occurred. 

Reason: 
Read timed out Remote Host: x.xx.xx.xxx Remote Port: 9403 occurs when the server fails to resolve the client’s host correctly. This can be caused by a network DNS configuration issue.


RESOLVING THE PROBLEM
You may opt to manually add the client’s ip and hostname information to the hosts file of the remote server. This will be the host running Websphere and MDM.
