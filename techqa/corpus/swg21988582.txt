Title: IBM Column Analysis Task Failed on table(s) with large number of columns On Oracle Repository(Xmeta+IADB) - United States

Text:
Information Analyzer; Column Anlaysis; CA; Oracle; TNS-12519 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 On Oracle Repository(Xmeta+IADB), Column Analysis Task Failed on table with large number of columns. 

SYMPTOM
e.g 


Run Column Analysis Task on table P100K1KC which has 1000 columns and 100k rows.
The task contained 100 ds jobs in the background. 
We saw 46 finished, but 54 aborted with below error.

pxbridge(4),1: The connector could not establish a connection to data source IADBODBC using user iauser.
pxbridge(4),1: ODBC function "SQLConnect" reported: SQLSTATE = 08001: Native Error Code = -1: Msg = [IBM(DataDirect OEM)][ODBC Oracle Wire Protocol driver][Oracle]TNS-12519: TNS:no appropriate service handler found (CC_OdbcConnection::connect, file CC_OdbcConnection.cpp, line 582)



CAUSE
Oracle PROCESSES limitation has been exceeded (maximum connections permitted).

The default number of processes is 300 after Oracle(12c) is installed.

SQL> select value from v$parameter where name in ('processes','sessions','open_cursors');

VALUE
--------------------------------------------------------------------------------
300
480
1500


RESOLVING THE PROBLEM
Increase the number of processes to a higher number. e.g 1500 


sqlplus / as sysdba
SQL> alter system set processes=1500 scope=spfile;

SQL> shutdown immediate
SQL> startup
