ERROR DESCRIPTION
 *  When number of the connections increases, WebSphere MQ 8.0 queue
   manager performance declines quickly, and becomes slow or
   unresponsive then.
   The FDCs with Probe IDs XC307100 and/or XC308010 might also be
   generated with
   high CPU usage from channel processes.

   Component :- xlsRequestMutex
   Program Name :- amqrmppa
   Major Errorcode :- xecL_W_LONG_LOCK_WAIT

   xcsFFST

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   Users using MQ v8 queue manager with lot of MQ clients and/or
   with clients often connecting and disconnecting.
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   This problem occurred because of delay in scanning the channel
   status table.
   
   
    
   
   

PROBLEM CONCLUSION
 *  MQ has been modified to improve the performance when scanning
   the channel status table.
   
   ---------------------------------------------------------------
   The fix is targeted for delivery in the following PTFs:
