ERROR DESCRIPTION
 *  Problem description:
   
   SQW Problems when connecting against a remote DB2 database.
   Loading data from a DB2 table on another server (table source)
   into a  DB2 table in the local instance (table target) fails.
   The table target  is "not logged initially". The error
   indicates a logfile size problem.
   
   CQ defect number wsdbu00278944 has been opened to track this
   issue.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  'Not Logged Initially' option does not work for Table Target
   operator when inserting data to a remote database.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This issue is resolved with InfoSphere Warehouse v9.7.1 interim
   fix build SQW_9.7.1.v20100202.zip
