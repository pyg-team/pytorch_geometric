Title: IBM IV96830: LEASES WILL GET STUCK WHEN TRYING TO PROCESS THE STATUS WHEN A REVIEW ASSUMPTION IS INVOLVED - United States

Text:

APAR STATUS
 * CLOSED AS FIXED IF NEXT.
    
   
   

ERROR DESCRIPTION
 *  Doing a review assumption on an Amended Lease (where the lease
   was revised using Amendment option at least once), the lease is
   getting sutck in Processing Status.
   
   1. Create a lease with lease type = Expense Lease, with a
   commencement and base expiration date.
   2. Create a payment schedule with multiple schedules.
   3. Activate the lease
   4. Once the lease is active, Revise it using the option Data
   Amendment.
   5. change the accounting start date
   6. Create payment schedule in a revised lease
   7. Activate the lease and do a Review assumption where Review
   Type = Business Factors and Fiscal Period = 2015-01 and
   Recalculate schedules is selected.
   8. Once the Review assumption is submitted, the lease will get
   stuck in processing.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  This is caused by a lease is being reassessed/remeasured with
   periods between accounting start date and accounting end date
   that have been closed, earlier than today s date. The
   changes are made in the workflow, "triContract - Synchronous -
   Preliminary Calculations for Generating Amortization Schedule".
   NOTE: In this scenario, no calculation on the FLIs will be done
   since all the periods are closed. They shouldn't be changed.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The status of the lease will be in Active even when a lease is
   being reassessed/remeasured with periods between accounting
   start date and accounting end date that have been closed,
   earlier than today s date.
   This is targeted to the 1h2017 application/OM release.
