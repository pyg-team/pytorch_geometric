Title: IBM FileNet Content Manager CE_Operations Component Manager setStringProperty method produces error: "...EngineRuntimeException:E_BAD_VALUE" - United States

Text:
update field; failed to update TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When the Conductor Queue reaches the CE_Operations Component Manager step of the running workflow using the setStringProperty method, producing the exception, "com.filenet.api.exception.EngineRuntimeException:E_BAD_VALUE". 

SYMPTOM
 

Below is the error stack of the affected work item from the wobquery output of the Conductor queue.

"Work Performer Exception: com.filenet.api.exception.EngineRuntimeException;The value given for a property or list element lies outside the permitted range or value set, or exceeds the maximum length allowed." 


CAUSE
The string being passed into the setStringProperty function has the length bigger than the Maximum String Length set for that property (in characters).


RESOLVING THE PROBLEM
 

Use one of the methods below to fix the problem. 

 * Increase the Maximum String Length for the property using FileNetÂ® Enterprise Manager (FEM). Here are the steps : 
   
    1.  Launch FEM and go to Property Templates. 
    2.  Right click on the property and select Properties. 
    3.  Go to 'More' tab, and enter a value in the Maximum String Length input box.
   
   
   
 * Or limit the length of the string value passed into the setStringProperty() function to the length set for the property.
