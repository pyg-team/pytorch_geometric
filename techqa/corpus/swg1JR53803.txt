   and IBM Process Server for online deployment, you receive an SSL
    Handshake exception because the cacerts truststore is used to
   establish the SSL connection instead of the

LOCAL FIX
 *  Copy the Process Server certificate to the truststore that is
   being used (which you can determine by enabling deployment
   tracing).
   
   
    
   
   

PROBLEM SUMMARY
 *  When you configuring HTTPS to be used between Process Center and
    Process Server for online deployment, the wrong truststore
   might be used, which causes the  deployment to fail.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix is available that ensures the correct truststore is
   selected.
   
   On Fix Central (http://www.ibm.com/support/fixcentral), search
   for JR53803:
   
   1. Select IBM Business Process Manager with your edition from
   the product selector, the installed version to the fix pack
   level, and your platform, and then click Continue.
   2. Select APAR or SPR, enter JR53803, and click Continue.
   
   When you download fix packages, ensure that you also download
   the readme file for each fix. Review each readme file for
   additional installation instructions and information about the
   fix.
