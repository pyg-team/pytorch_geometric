Title: IBM GJSO030E AWSJCO005E CORBA MARSHAL Connection refused saving SAP job - United States

Text:
GJSO030E; AWSJCO005E; corba marshal; connection refused; jsc; fp5; unable TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 You are unable to retrieve or save SAP jobs when accessing them through the Job Scheduling Console (JSC) v8.3 Fix Pack (FP) 05. 

SYMPTOM
The symptom is a JSC pop-up with a WebSphere message:

Message GJSO030E 

 The window cannot close
Reason: AWSJCO005E WebSphere Application Server has given the following error: CORBA MARSHAL 0x4942f82a No; nested exception is org.omg.CORBA MARSHAL:
Trace from server ... Unable to read value from underlying bridge : java.net.ConnectException: Connection refused (errno 239) ... vmcid: IBM minor code: 89A


CAUSE
There are several compatibility issues between TWS v8.3 FP02 and JSC v8.3 FP03 or later.


ENVIRONMENT
JSC v8.3 FP03 or greater connecting to TWS v8.3 GA, FP01 or FP02



DIAGNOSING THE PROBLEM
Attempt to perform the same activity using a JSC v8.3 FP01 or FP02 client. 



RESOLVING THE PROBLEM
NOTE: Tivoli Workload Scheduler (TWS) v8.3 is no longer supported without a support extension contract as of September, 2012. Upgrade TWS v8.3 to a supported version. 

Support extension contract: Install Tivoli Workload Scheduler (TWS) v8.3 FP05 or later or revert to JSC v8.3 FP01 or FP02. It is recommended to install the latest Fix Pack on both the JSC and the TWS master.
