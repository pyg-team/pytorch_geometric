Title: IBM Running a Birt Report in TCR 3.1.3 fails with COG-BIRT-0011 - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 While running a BirtReport in HTML format in Tivoli Common Reporting 3.1.3, the Birt report is failing with the error message COG-BIRT-0011 

SYMPTOM
The full error message in TCR is similar to: 

COG-BIRT-0011 
An error has occurred while connecting to the Content Store 
Cause: java.lang.IllegalArgumentException: Adding text to an XML document must not be null.. Stack Trace: 
com.cognos.pogo.bibus.CommandExecutionException: Adding text to an XML 
document must not be null at com.cognos.pogo.bibus.BIBusCommand.handleDefaultException(BIBusCommand.j
ava:372) at com.cognos.pogo.bibus.BIBusCommand.execute(BIBusCommand.java:205) 
at com.ibm.cognos.birtservice.SaveBirt.reportOuptutToCM(SaveBirt.java:580)


CAUSE
When a HTML BirtReport does not contains a picture, this HTML BirtReport is failing. BirtReports in other formats like PDF and Excel are working fine.


RESOLVING THE PROBLEM
The issue is addressed in APAR IJ00515 and a new BirtAdapterA.jar file is available. This new BirtAdapterA.jar superseded the BirtAdapterA.jar attached in the technote "Tivoli Common Reporting (TCR) v3.1.2.1: Unable to run BIRT reports when predefined prompt values are set" 


http://www-01.ibm.com/support/docview.wss?uid=swg21996775 [http://www-01.ibm.com/support/docview.wss?uid=swg21996775] 
Please contact the IBM Support to get the new BirtAdapterA.jar file.

1. Stop Jazz for Service Management (JazzSM) profile server
ex: <JazzSM_Installed_location>/profile/bin/stopServer.sh server1

2. Switch to 
UNIX;
<JazzSM_Installed_location>/profile/installedApps/JazzSMNode01Cell/IBM Cognos.ear/p2pd.war/WEB-INF/lib/
Windows;
c:\Program Files\IBM\JazzSM\profile\installedApps\JazzSMNode01Cell\IBM Cognos.ear\p2pd.war\WEB-INF\lib\

3. Move existing BirtAdapterA.jar file into safe location

4. Download and copy attached BirtAdapterA.jar into above location

5. Start JazzSM profile server
ex: <JazzSM_Installed_location>/profile/bin/startServer.sh server1

6. run your HTML BirtReport again
