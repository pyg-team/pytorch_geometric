QUESTION
 You use CICS VSAM Recovery for z/OS (CICS VR) V4.2 or higher, and observe several dynamically allocated and uncataloged datasets with the ddname DWWHIST. You would like to know, why is CICS VR allocating the DWWHIST datasets? 

ANSWER
DWWHIST [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.cvr.doc/dwwig/allocdatasets.html] is an internal DD used by the CICS VR server address space for history datasets that are temporary. The data from these datasets is flushed to the DWWMSG [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.cvr.doc/dwwig/allocdatasets.html] dataset when the history scavenger runs.

You can run the history scavenger dynamically, if the hourly run is not sufficient by using the SETSMS command to specify SCAVENGE(HISTORY) in the CICSVR_GENERAL_CONTROL() [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.cvr.doc/dwwig/gencont.html] string. 

Module DWWITENV calls DWWUTENV to perform SVC99 DYNALLOC processing for the history datasets. The S99 parmlist only sets values for the ddname, dsname, blksize, blkln, disp, and primary and secondary space allocation values. No unit information is specified nor request to catalog the 
datasets.
