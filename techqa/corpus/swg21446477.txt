Title: IBM ITCAM4Tx - LoadRunner script playback failed with return code 111 - United States

Text:
robotic lr 71RRT-PLAY 71INS-RRT 71RRT-CONF TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After uploading a HP Mercury Load Runner test script to the AMC console agent, and creating a robotic profile to playback the associated transaction, the TEP "robotic script playback status" workspace shows : last run status = failed 

SYMPTOM
TEP "robotic script verification point failures" workspace shows return code 111 as violation. 

After having enabled the generic playback component extra logging, as doing below : 

 In t6-logging.properties file located under C:\IBM\ITM\TMAITM6, change the 
following entries to : 
BWM.trc.gp.mbean.level=DEBUG_MAX 
BWM.trc.gp.service.level=DEBUG_MAX 
BWM.trc.gp.install.level=DEBUG_MAX 
BWM.trc.gp.util.level=DEBUG_MAX 
- save the file, stop the RRT T6 agent, clear old logs from 
C:\program files\ibm\tivoli\common\BWM/logs directory 
- restart the RRT T6 agent 

- reproduced the problem 


C:\Program File\IBM\Tivoli\common\BWM\logs\trace-robotic.log file, after the 
playback has failed, is showing the following : 

1) the native loadrunner command in use : 

PlaybackThreadPoolWorker-0 - 
com.ibm.tivoli.transperf.playback.generic.util.Process.exec() - Native 
command stdout: command passed in is \bin/mdrv.exe -usr 
C:\IBM\ITM\tmaitm6\app\LOADRUNNER\scripts\eric1ibmdotcom\eric1ibmdotcom. 
usr -out C:\IBM\ITM\tmaitm6\app\LOADRUNNER\scripts\eric1ibmdotcom 
-drv_log_file C:\IBM\ITM\tmaitm6\app\LOADRUNNER\scripts\eric1ibmdotcom 
-no_drv_log > stdout 2> stderr 
timeout value passed in is 120000 

2) the origin of the error :

PlaybackThreadPoolWorker-0 - 
com.ibm.tivoli.transperf.playback.generic.util.Process.exec() - Native 
command stderr: Error creating command process 

3) the return code value:

PlaybackThreadPoolWorker-0 - 
com.ibm.tivoli.transperf.playback.generic.util.Process.exec() - Exit, 
return value = 111


CAUSE
T6 configuration setting : "LoadRunner Command home" is not set, but is left blank


RESOLVING THE PROBLEM
On the RRT T6 agent, use the "Tivoli Manage monitoring services" GUI : 

 <select the robotic agent > => right mouse click => reconfigure....=> 

on the "Mercury LoadRunner Configuration" panel, 

setup LoadRunner Command home field, like for example :

LoadRunner Command home = C:\Program Files\HP\LoadRunner

where "C:\program Files\HP\LoadRunner"
is the home directory where LoadRunner product was installed.
