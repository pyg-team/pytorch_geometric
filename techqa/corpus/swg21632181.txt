Title: IBM SecurityImpl (initialization failure) for TWS commands that access the DB - United States

Text:
AWSJCS011E; NoClassDefFoundError; SecurityImpl; UnsatisfiedLinkError; libtwssecurityjni.a; 0509-022; 0509-150; 0509-126; module TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Unable to successfully run commands such as planman, optman, or composer in Tivoli Workload Scheduler (TWS). 

SYMPTOM
The following message is displayed at the command line: 

AWSJCS011E An internal error has occurred. The error is the following: "
; nested exception is: com.ibm.ws.exception.WsEJBException: nested exception is: java.lang.NoClassDefFoundError:

com.ibm.tws.security.jni.SecurityImpl (initialization failure)".
AWSJCS011E An internal error has occurred. The error is the following: com.ibm.tws.security.jni.SecurityImpl (initialization failure)".


In WebSphere's SystemOut.log the following exceptions are found:

RuntimeException: java.lang.UnsatisfiedLinkErrr: <TWSHome>/bin/libtwssecurityjni.a ( 0509-022 Cannot load
module <TWSHome>/bin/lib/twssecurityjni.a.
0509-150 Dependent module libicuuc34.1.a could not be loaded.
0509-022 Cannot load module libicuuc34.1.a.
0509-026 System error: A file or directory in the path name does not exist.
0509-022 Cannot load module <TWSHome>/bin/libtwssecurityjni.a.
0509-150 Dependent module <TWSHome>/bin/libtwssecurityjni.a could not be loaded.)
at com.ibm.tws.security.jni.TWSSecurityAPIJNI.<clinit>(TWSSecurityAPIJNI
.java:27)
at java.lang.J9VMInternals.initializeImpl(Native Method)
..."


CAUSE
Missing library file libicuuc34.1.a from both <TWSHome>/bin and from /usr/Tivoli/TWS/ICU/3.4.1/lib


DIAGNOSING THE PROBLEM
ls -l <TWSHome>/bin/libicuuc34.1.a 

ls -l /usr/Tivoli/TWS/ICU/3.4.1/lib


RESOLVING THE PROBLEM
1. If libicuuc34.1.a is found in <TWSHome>/bin, then place a copy in /usr/Tivoli/TWS/ICU/3.4.1/lib. 

2. If libicuuc34.1.a is missing altogether then check for /usr/Tivoli/TWS/ICU/3.4.1/lib/libicuuc.a. If libicuuc.a is found, then create a soft link to libicuuc.a named libicuuc34.1.a: 

 a. As root: 

 # cd /usr/Tivoli/TWS/ICU/3.4.1/lib 

 # ln -sf libicuuc.a libicuuc34.1.a 

The required library should be found now.
