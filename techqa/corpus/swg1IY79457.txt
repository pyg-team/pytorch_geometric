ERROR DESCRIPTION
 *  A MQCSP (Connection security parameters structure) can be
   specified on a client MQCONNX call using the
   SecurityParmsOffset or SecurityParmsPtr fields in the MQCNO
   (Connection options structure) structure.
   If an MQCSP is specified on the MQCNO of a client MQCONNX call,
   then the client may fail to connect to the queue manager.
   This occurs when the client application and queue manager are
   on different machines, using different character sets (CCSID).
   The channel failure occurs on the queue manager side.
   If the queue manager is running on a UNIX platform an FFST
   showing a SIGSEGV within an amqrmppa or amqcrsta process is
   observed. If the queue manager is running on Windows,
   then an access violation is observed within an AMQRMPPA.EXE
   process.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   Customer passing an MQCSP structure to a client MQCONNX call.
   
   Platforms affected:
    All Distributed (iSeries, all Unix and Windows)
   ****************************************************************
   PROBLEM SUMMARY:
   The codepage conversion of the network flows between client and
   server, containing the MQCSP information, was performed
   incorrectly.
   This caused the server-side message channel agent to access
   invalid storage, and or raise a SIGSEGV  or access violation.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code was fixed to correct the codepage conversion.
