APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Environment: APM HTTP Server Agent 8.1.3
   
   
   OS: Linux/AIX
   
   Problem Description:
   KHU agent failed to discovery Apache HTTP server when the server
   is
   configurated without a default httpd.conf file
   
   Detailed Recreation Procedure:
   1) Install and start KHU agent.
   2) Configure Apache server and remove the default httpd.conf
   file
   3) Restart Apache.
   
   The KHU agent failed to discovery the Apache server due to
   "httpd -V"
   error.
   
   Related Files and Output: N/A
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  KHU agent failed to discover the running HTTP server when there
   is no default httpd.conf exist, and no HTTP Server agent would
   be shown in APM UI dashboard. In <APM agent install
   dir>/logs/khu-disc.log, there was an error recorded as "Failed
   to get server version!". And in <APM agent install
   dir>/tmp/khu/discovery/<md5>/compile_info, there was an error:
   "httpd: Could not open configuration file
   /etc/httpd/conf/httpd.conf: No such file or directory"
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this APAR is contained in the following maintenance
   packages:
     8.1.3.0-IBM-IPM-HTTP_SERVER-AGENT-XLINUX-IF0001
     8.1.3.0-IBM-IPM-HTTP_SERVER-AGENT-AIX-IF0001
