CAUSE
The target side database is busy and no response to CDC's bookmark query.

ANSWER
There is a system parameter you can use on your target datastore target_get_position_query_timeout_seconds, with default setting as 120 seconds. Try to set it to a higher value to workaround this exception. In the mean time, you also need to check from target database to see why the query takes a long time than expected.
