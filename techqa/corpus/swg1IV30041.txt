LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Configuring Oracle data source via LDAP was very difficult   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   Impact required a special sql.Datasource object to exist in the
   LDAP jndi registry in order to obtain an oracle connection
   through ldap. This approach does not make sense given the url
   available to make an oracle ldap connection, and in the case of
   Active  DIrectory this approach was not feasible to implement.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Customer now configures the data source in Impact with the url
   to connect to Oracle via LDAP.
   The fix for this apar is in 511FP2
