Title: IBM BIP2080E Broker failing to startup correctly - United States

Text:
BIP2080E Broker failing to startup correctly BIP2066E Broker fails start TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Your IBM Integration Bus (IIB) or WebSphere Message Broker (WMB) is failing to start up correctly. 

SYMPTOM
Following are the errors received:

BIP2238W: LIL '/usr/lpp/mqsi/lil/imbdfjps.lil' has been unloaded. 
BIP2066E: Broker NPP1BRK 
BIP2080E: Execution Group default failed to return an internal configuration Start message response within a 60 second time period.


CAUSE
The CLASSPATH and LIBPATH were incorrect. 

The imbdfjps.lil did not load, which caused the execution groups to not to start up correctly.

RESOLVING THE PROBLEM
 Alter the ENVFILE and set the CLASSPATH and LIBPATH to include: 

..../Java/bin/lib and ...Java/bin/bin 

After correcting the CLASSPATH and LIBPATH the Broker started without errors, and deploy works as expected.
