APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Tivoli portal client changes need to be made to recognize Java
   8 is installed can be set as the default java used.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Add support for the Tivoli Portal client to run with Java
   version 8.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Added necessary code to the Tivoli Portal clients startup files
   including java script to support Java version 8.
   
   Known Issues:
   Problem Summary 1:
   Using Java 8 64-bit with Internet Explorer version 11 fails with
   the message:
   
   This content cannot be displayed in a frame
   To help protect the security of information you enter into this
   website, the publisher of this content does not allow it to be
   displayed in a frame.
   What you can try:
   Open this content in a new windows
   
   Workaround:
   Internet Explorer verison 11 does not support running 64-bit
   plugins.   Use Java 8 32-bit with Internet Explorer version 11
   or use the WebStart client.
   
   
   Problem Summary 2:
   Java 8 fails with Firefox; the client does not fully initialize
   giving a blank screen.
   
   Workaround:
   Use Internet Explorer or the WebStart client.
   This defect will be fixed in Oracle Java 8 u60, targeted for
   August 2015.
   
   
   
   Note: Recent Java updates have caused connection problems when
   logging into the portal server via the Tivoli Enterprise Portal
   browser and WebStart clients. Refer to technote #1672459 for
   important information regarding the latest Java Support
   requirements:
   
   http://www.ibm.com/support/docview.wss?uid=swg21672459 [http://www.ibm.com/support/docview.wss?uid=swg21672459]
