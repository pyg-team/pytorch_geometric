ERROR DESCRIPTION
 *  When importing a change which contains a search-condition
   clause and the column name in the search condition is
   qualified by table OWNER and table name, the job fails
   with the following error because the CREATE TRIGGER stmt
   contains double quotes:
   
   Missing semicolon after triggered action
   ADB2PA:  The following statmeent could not be parsed:

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users of the DB2 Administration Tool for     *
   *                 z/OS who import a change which contains      *
   *                 a search-condition clause and the column     *
   *                 name in the search condition is qualified    *
   *                 by table OWNER and table name.               *
   ****************************************************************
   * PROBLEM DESCRIPTION: DB2 Admin imports a change which        *
   *                      contains a search-condition             *
   *                      clause and the column name in the       *
   *                      search condition is qualified by        *
   *                      table OWNER and table name:             *
   *                                                              *
   *                      For example:                            *
   *                      CREATE TRIGGER ...                      *
   *                      WHEN (>table_schema<.                   *
   *                      >table_name<.>column_name<              *
   *                      = >table_name<.>column_name<)           *
   *                      ...                                     *
   *                                                              *
   *                      and the import job fails with the       *
   *                      following error:                        *
   *                                                              *
   *                      Missing semicolon after triggered       *
   *                      action                                  *
   *                      ADB2PA: The following statement         *
   *                      could not be parsed :                   *
   *                                                              *
   *                      Token number =     nn, Token = ..       *
   *                      Unsuccessful parse of DDL statement.    *
   *                        ADB2CID - Now terminating             *
   *                      ADB0006E ADBCCM - Error(s) found.       *
   *                      Please review output.                   *
   *                                                              *
   *                      because the CREATE TRIGGER              *
   *                      statement contains double periods:      *
   *                                                              *
   *                      CREATE TRIGGER                          *
   *                      >schema<.>triggername< AFTER            *
   *                      update OF...                            *
   *                                                              *
   *                      (>schema<..>col_name<                   *
   *                      !=>schema<..>col_name<)                 *
   *                        ;END                                  *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Import job fails when parsing the CREATE TRIGGER statement
   because it contains double periods between the schema and
   column name.

 *  Problem has been resolved.
