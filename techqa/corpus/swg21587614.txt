Title: IBM Unable to populate the TCR IBM_TRAM.TIME_DIMENSION table - United States

Text:
TCR 2.1.1 1.3 ITM OS Agent Reports TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 call IBM_TRAM.CREATE_TIME_DIMENSION stored procedure fails with DB2 SQL errors SQLCODE "-204", SQLSTATE "42704" 

SYMPTOM
call IBM_TRAM.CREATE_TIME_DIMENSION('start_date', 'end_date', granularity_of_data); 

SQL0727N An error occurred during implicit system action type "1". 

Information returned for the error includes SQLCODE "-204", SQLSTATE 

"42704" and message tokens "IBM_TRAM.TIME_DIMENSION". SQLSTATE=56098 


CAUSE
This error can happen when IBM_TRAM.TIME_DIMENSION table does not exist. 

The execution of the command "call IBM_TRAM.CREATE_TIME_DIMENSION('start_date', 'end_date', granularity_of_data);" does not create the table IBM_TRAM.TIME_DIMENSION if it does not already exist. The table population fails when this condition occurs.


ENVIRONMENT
IBM Tivoli Common Reporting 2.1.1 on zLinux. This error can happen on other platforms . Tivoli Data Warehouse should be running on IBM DB2.



RESOLVING THE PROBLEM
1. Create the table TIME_DIMENSION using CREATE TABLE IBM_TRAM.TIME_DIMENSION command provided in "create_schema_IBM_TRAM.db2" file in ../Reports/db_scripts/db2 directory from the Agent Reports install package. 

2. Execute the command "call IBM_TRAM.CREATE_TIME_DIMENSION('start_date', 'end_date', granularity_of_data);" with actual parameter values.
