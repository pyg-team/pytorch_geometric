APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After applying the fix for APAR IV83868 [http://www-01.ibm.com/support/docview.wss?uid=swg1IV83868] Migration Manager will
   remove text search indexes from Oracle database in target
   environment.
   Steps to replicate:
   1) On Source System Goto System Configuration -> Migration ->
   Migration Collections.
   2) Press New Record button to create new Migration Collection.
   Collection Name: IV91204_COLL
   Description: COLLECTION TEST FOR APAR IV91204
   3) On Configurations table press New Row button.
   4) On Application field press action arrow to open Select
   Application dialog.
   5) Locate CONFIGUR application and click Go To Application arrow
   6) On Database Configuration Application select WORKLOG object
   and press Return With Value button.
   7) Save.
   8) Press Validate Migration Collection button on Toolbar to open
   Validate Dialog.
   9) Press Start button and close dialog
   10) Select Action Create package definition and press Yes on
   dialog.
   11) Goto System Configuration -> Migration -> Migration Manager
   Application.
   12) Select IV91204_COLL Package Definition.
   13) Press change Status button on toolbar to open Change Status
   Dialog.
   14) Select Approved package definition and Press OK button.
   15) Select Package Tab.
   16) On Packages table press Create button to open Upload
   Compiled Sources dialog.
   17) Change Processing Action to Replace and press Continue
   button
   18) When Processing Complete, press OK button to dismiss dialog.
   19) Select Action Manage Targets.
   20) Press New Row Button.
   Target Name: C
   Description: C
   Type: FILE
   File Path: c:\
   21) Press OK button to dismiss dialog.
   22) On Packages table press Distribute button to open Distribute
   Package dialog.
   23) Select Target C and press OK button.
   24) When Processing Complete, press OK button to dismiss dialog.
   25) On Target System Goto System Configuration -> Migration ->
   Migration Manager Application.
   26) Select Action Upload Package.
   27) Browse to find package
   C:\IV91204_COLL_server_name_20160822121943.zip
   28) Goto System Configuration -> Platform Configuration ->
   Database Configuration
   29) Select WORKLOG object and select indexes tab
   30) On Indexes table confirm there is a text search index on
   the description field.
   31) Goto System Configuration -> Migration -> Migration Manager
   Application.
   32) Select Action Deploy Package to open Deploy Package dialog.
   33) Select Package uploaded earlier, tick "Do you have a current
   backup?" checkbox and press Deploy button.
   34) Press OK to dismiss dialog.
   35) Goto System Configuration -> Platform Configuration ->
   Database Configuration
   36) Select WORKLOG object and select indexes tab
   When collection is deployed in target it removed the text search
   index from the target even though it was in the source system.
   
   
    
   
   

LOCAL FIX
 *  Recreate text search index after migration.
