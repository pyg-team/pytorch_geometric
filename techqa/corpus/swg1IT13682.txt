APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When an integration node is started on a Linux or UNIX platform
   where mqsiprofile is sourced in a command shell other than
   Bash, Healthcare Web Monitoring may be unavailable.
   
   On supported platforms, it may also not be possible to deploy
   the MedicalDeviceInput node.
   
   
   Additional Symptom(s) Search Keyword(s):
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   All users of IBM Integration Bus Healthcare Pack v4.0 that use
   the web-based monitoring component, or the MedicalDeviceInput
   node, and use a command shell on Linux or UNIX other than Bash.
   
   
   Platforms affected:
   AIX, Linux on Power platform, Linux on x86-64 platform, Linux on
   zSeries platform, Solaris x86-64 platform, Solaris SPARC
   platform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   When an integration node is started on a Linux or UNIX platform
   where mqsiprofile is sourced in a command shell other than Bash,
   the necessary environment variables may not be set up correctly.
   This can prevent the Healthcare Web Monitoring view from being
   available, and on supported platforms, it may not be possible to
   deploy the MedicalDeviceInput node.
   
   <i>There are a number of resource name changes between WebSphere
   Message Broker and IBM Integration Bus Version 9.0.  For details
   visit
   http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e [http://pic.dhe.ibm.com/infocenter/wmbhelp/v9r0m0/topic/com.ibm.e]
   tools.mft.doc/bb23814_.htm</i>
   
   
    
   
   

PROBLEM CONCLUSION
 *  The product mqsiprofile extension scripts now set up the
   environment variables correctly.
   
   
    
   
   

TEMPORARY FIX
