LOCAL FIX
 *  The range of values to be used for ConnectionMultiplicity is
   1-2147483647.
   
   
    
   
   

PROBLEM SUMMARY
 *  Server fails to start with a BAD_PARAM exception when a value of
   '0' is used for ConnectionMultiplicity.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This defect will be fixed in:
   5.0.0 SR10
   .
   The problem is resolved by validating the input for
   ConnectionMultiplicity property thus correcting the range of
   values permitted for ConnectionMultiplicity to be 1 -
   2147483647.
   .
   To obtain the fix:
   Install build 20090430 or later
