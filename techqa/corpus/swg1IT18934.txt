APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  For MDM 11.5, Workbench 11.5.0.2 does not generate SQL codes
   correctly for Alert entity with non-persistent fields.
   It throws following runtime error if you deploy generated one.
   Caused by: com.dwl.bobj.query.PersistenceException:
   com.ibm.pdq.runtime.exception.DataRuntimeException:
   [pdq][10103][3.200.75] An error prevented the update operation
   from completing successfully.;  Caused by:
   com.ibm.db2.jcc.am.SqlSyntaxErrorException: DB2 SQL Error:
   SQLCODE=-104, SQLSTATE=42601, SQLERRMC=,;_TP_CD,
   ALERT_TP_CD,;<space>, DRIVER=3.69.49
   Caused by: com.ibm.pdq.runtime.exception.DataRuntimeException:
   [pdq][10103][3.200.75] An error prevented the update operation
   from completing successfully.;  Caused by:
   com.ibm.db2.jcc.am.SqlSyntaxErrorException: DB2 SQL Error:
   SQLCODE=-104, SQLSTATE=42601, SQLERRMC=,;_TP_CD,
   ALERT_TP_CD,;<space>, DRIVER=3.69.49
   The generated invalid SQL looks like following.
   /**
      * <!-- begin-user-doc -->
      * <!-- end-user-doc -->
      *
      * @generated
      */
     public static final String getEObjAvivaAlertExtSql = "select
   , LAST_UPDATE_DT, LAST_UPDATE_USER, LAST_UPDATE_TX_ID from
   ALERT where ALERT_ID = ? ";
     /**
      * <!-- begin-user-doc -->
      * <!-- end-user-doc -->
      *
      * @generated
      */
     public static final String createEObjAvivaAlertExtSql =
   "insert into ALERT (DESCRIPTION, INSTANCE_PK, ENTITY_NAME,
   ALERT_ID, CREATED_BY_USER, END_DT, REMOVED_BY_USER, START_DT,
   ALERT_SEV_TP_CD, ALERT_TP_CD, , LAST_UPDATE_DT,
   LAST_UPDATE_USER, LAST_UPDATE_TX_ID) values( ?1.description,
   ?1.instancePK, ?1.entityName, ?1.alertIdPK, ?1.createdByUser,
   ?1.endDt, ?1.removedByUser, ?1.startDt, ?1.alertServTpCd,
   ?1.alertTpCd, ?1.lastUpdateDt, ?1.lastUpdateUser,
   ?1.lastUpdateTxId)";
   Some fields in SQL are missing.
   As you cT
   
   
    
   
   

LOCAL FIX
 *  Fix generated SQL.
