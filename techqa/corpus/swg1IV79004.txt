APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If you create an Operator View in the GUI and change the
   generated Policy to use DirectSQL rather than GetByFilter, this
   works fine at 6.1.1 GA, but fails in the later fixpacks.
   
   To reproduce:
   Create a simple Operator View that displays data from a table
   in a DB2 database:
   
   1. Create data source. E.g. LabTestDB (DB2)
   2. Create data type. E.g. MyTable (all data from one table)
   3. Create OpView:
      a) Operator View name E.g. TestOV
      b) Event panel - Top
      c) Action panel - Off
      d) Group Name - MyGroup
      e) By Filter
      f) Data Type - MyTable
   4) Save and run operator view - works fine.
   5) Edite the policy Opview to change the line. E.g.:
     result_InfoPanelBedrockData=GetByFilter("MyTable","",null);
    to
     result_InfoPanelBedrockData=DirectSQL( "LabTestDB", "select *
   from
     mytable", false );
   
   Operview fails with:
   Error 500: java.lang.NullPointerException
   
   
    
   
   

LOCAL FIX
 *  May be possible to edit generated HTML to work.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * If a user creates an Operator View in the GUI and then       *
   * changes the                                                  *
   * generated Policy to use DirectSQL rather than GetByFilter,   *
   * the                                                          *
   * operator view no longer displays the results.                *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   The method that generates the tabbed format of the query results
   in opview orgnode viewer class was updated in 611 FP2.
   this caused the issue.
   
   
    
   
   

PROBLEM CONCLUSION
 *  In the opvieworgnodesviewer class, a null pointer exception is
   being generated in the getcolumnnames method when
   columnNamesDeleted is being retrieved from the the
   SoapOrgNodeMemento.
   This is happening when DirectSQL is being used to query.
   A null pointer check was added to the code. This resolved the
   issue.
