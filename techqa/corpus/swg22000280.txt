Title: IBM Cannot load program saproute on AIX - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 saproute invocation fails:
exec(): 0509-036 Cannot load program saproute because of the following
errors: 

SYMPTOM
exec(): 0509-036 Cannot load program saproute because of the following 

errors:
0509-150 Dependent module libstdc++.a(libstdc++.so.6) could not be
loaded.
0509-022 Cannot load module libstdc++.a(libstdc++.so.6).
0509-026 System error: A file or directory in the path name does not
exist.


CAUSE
The runtime loader is trying to find the necessary runtime libraries
and cannot find the C++ standards runtime library


RESOLVING THE PROBLEM
On AIX server, install the RPM libraries for the appropriate AIX version. When libraries are installed for the specific AIX version running and the LIBPATH variable is setup to point to them, the saproute executable and the SAP extension runs successfully. 


Please refer to UNIX Administrator or AIX Support regarding RPM libraries for the appropriate AIX version

Required files in $EDI_ROOT/lib directory:
libstdc++.a
libgcc_s.a

libgcc-4.2.0-3.aix5.3.ppc.rpm
libstdcplusplus-4.2.0-3.aix5.3.ppc.rpm

rpm -qpl libgcc-4.2.0-3.aix5.3.ppc.rpm
/opt/freeware/lib/gcc/powerpc-ibm-aix5.3.0.0/4.2.0/libgcc_s.a

rpm -qpl libstdcplusplus-4.2.0-3.aix5.3.ppc.rpm
/opt/freeware/lib/gcc/powerpc-ibm-aix5.3.0.0/4.2.0/libstdc++.a
