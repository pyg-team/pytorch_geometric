Title: IBM How to workaround warning messages about a signature in RHEL kernel upgrade on NEC Orca system - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How to workaround warning messages about a signature in RHEL kernel upgrade on NEC Orca system 

CAUSE
When upgrading Red Hat kernel from version 5.5 to 5.7, 5.7 to 5.8 and 5.8 to 5.9, 
"kernelupgrade.sh" displayed the warning messages regarding to Header V3 DSA signature: NOKEY.


ANSWER
Required KEY is not registered before RHEL kernel upgrade and the error could be ignored. 


However, users can workaround this error message by installing the public release key as below.
After installing the public release key, there will no relevant NOKEY error message when upgrading RHEL kernel from version 5.5 to 5.7, 5.7 to 5.8 and 5.8 to 5.9.

*Install public release key ( as root ) 
[root@vmnps-dw21 nzscratch]# rpm --import /etc/pki/rpm-gpg/RPM-GPG-KEY-redhat-release
