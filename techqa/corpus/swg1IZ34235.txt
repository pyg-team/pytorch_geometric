Text:
  A FIX IS AVAILABLE
6.0.1.1-TIV-XEforMsg-IF0002 - OMXE for Messaging 6.0.1 FP01 IF02 [http://www-01.ibm.com/support/docview.wss?uid=swg24021966]

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After installing OMEGAMON XE for Messaging v6.0.1 Language
   Pack (C14CRML), it couldn't open TEP Browser with cache error.
   
   
   ENVIRONMENT:
   
   TEPS (All version) : All Platforms
   
   Recreation Steps:
   
   1. Install OMEGAMON XE for Messaging v6.0.1 Language Pack
   (C14CRMS) to TEPS.
   2. Open the TEP Browser.
   
   
   LOGS:
   
   Screenshot of the error message on ECuRep.
   
   
    
   
   

LOCAL FIX
 *  One of the following:
   
   1. Edit the version from 6.0.111 to 6.0.1.11 in applet.html.
   
   2. Edit the version from 6.0.111 to 6.0.1.11 in wbi.inf, then
   reconfigure the TEPS.
   
   
    
   
   

PROBLEM SUMMARY
 *  After installed OMEGAMON XE for Messaging V601 Language Pack
   (C14CRML) then Java error could pop up when accessing TEPS from
   TEP Browser client. For example, you will see
   .
   ====================
   Cache Error
   cache file can not be kept or updated.
   ====================
   .
   In this screen, Java exception is displayed if click the "see
   detail" button on the error message dialog. The root cause is
   that the version of the jar file (wbi_resources_l10n.jar) is
   6.0.111 which is not correct.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The correct version is 6.0.1.11 .
   
   
    
   
   

TEMPORARY FIX
 *  No temporary Fix. The formal fix is included in V601 FP01 IF02
   in which refreshed the Language Pack installation image.
