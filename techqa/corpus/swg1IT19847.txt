LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   This issue affects users of the:
   
     - WebSphere MQ V7.5 JCA Resource Adapter
     - IBM MQ V8.0.0.0 and V8.0.0.1 JCA Resource Adapter
   
   
   Platforms affected:
   MultiPlatform
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   After detecting a broken connection for a pooled JMS Connection
   object, the MQ JCA Resource Adapter invoked a callback on the
   object:
   
       javax.resource.spi.ConnectionEventListener
   
   which was previously registered by the application server to
   inform the container of the event.  This, in turn, resulted in
   the application server calling back into the MQ JCA Resource
   Adapter to perform clean-up and closure operations on the
   affected pooled ManagedConnection object.  The result of this
   clean-up was that a:
   
       java.util.ConcurrentModificationException
   
   was thrown by the MQ JCA Resource Adapter, and a FFDC was
   generated.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The MQ JCA Resource Adapter code has been updated to prevent a:
   
       java.util.ConcurrentModificationException
   
   being thrown in the scenario described above.
