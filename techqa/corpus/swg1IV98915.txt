APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If you have a policy that calls exit(); from within a
   syncronized block this will trigger an ExitPolicyException and
   the policy fails.
   
   Here is a simple example:
   
   synchronized( TestSyncBlock ) {
     log( "Now in the sync block - just exit(); ..." );
     exit();
     log( "Should never see this message, should always exit
   above." );
   }
   log( "As the syncronized block is a function," );
   log( "the exit should leave us here." );
   
   The log messages at the end will not be logged because the
   policy fails.
