Title: IBM PK83207; 7.0.0.3: update to the WebSphere MQ resource adapter - United States

Text:
PK83207; PK83207; PK83207 DOWNLOADABLE FILES

ABSTRACT
 Updates the WebSphere MQ Resource Adapter shipped with WebSphere Application Server to WebSphere MQ Fix Pack 7.0.0.2 

DOWNLOAD DESCRIPTION
PK83207 resolves the following problem:

ERROR DESCRIPTION: 
This APAR updates the WebSphere MQ resource adapter shipped with WebSphere Application Server to 7.0.0.2-k700-002-090421 

LOCAL FIX: 

PROBLEM SUMMARY

USERS AFFECTED:
Users of WebSphere Application Server Version 7.0 using WebSphere MQ as a JMS messaging provider.

PROBLEM DESCRIPTION:
Updates the WebSphere MQ Resource Adapter shipped with WebSphere Application Server to 7.0.0.2

RECOMMENDATION:
None

The following fixes are included in level 7.0.0.2-k700-002-090421 contained in this APAR that are not included in the level of the WebSphere MQ resource adapter shipped with WebSphere Application Server Fix Pack 7.0.0.3 (7.0.0.2-k700-002-090216):

WebSphere MQ reference IC59462: Client transport connections to WebSphere MQ Version 7.0 using conversation sharing, may create too many conversations on a single network connection, or too few.

WebSphere MQ reference 130635: Establishing client transport connection to WebSphere MQ might fail intermittently with 2059 MQRC_Q_MGR_NOT_AVAILABLE reason codes, even though the queue manager is available and other connections are created successful.

WebSphere MQ reference 130638: The following class cast exception is thrown, preventing an attempt to deactivate an endpoint for an activation specification from succeeding: java.lang.ClassCastException: com.ibm.mq.jms.MQXAConnectionFactory incompatible with com.ibm.mq.connector.ConnectionUsageTuple

PROBLEM CONCLUSION: 
The Interim Fix package containing this APAR can be applied to WebSphere Application Server Version 7.0 Fix Pack 7.0.0.1and Fix Pack 7.0.0.3. 

This APAR is not targeted for delivery in any future Fix Pack. WebSphere Application Server Fix Pack 7.0.0.5 will contain a more recent version of the WebSphere MQ resource adapter, under a different APAR. 

Please see the following document for important information for users of the WebSphere MQ messaging provider with WebSphere Application Server, including how to check the version of the resource adapter used by your application server http://www.ibm.com/support/docview.wss?rs=171&uid=swg21316899 [http://www.ibm.com/support/docview.wss?rs=171&uid=swg21316899]
