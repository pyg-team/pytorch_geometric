Title: IBM Setting the IDUC Flush Rate in Object Server clients - United States

Text:
Gate.IDUCflushrate; ObjectServer; IDUC; get iduc TECHNOTE (FAQ)

QUESTION
 The client has a property setting for IDUCFlushRate. What is IDUC and what should this be set to? 

CAUSE
The Object Server uses IDUC to notify a many clients of the event Insert, Delete, Update and Control notifications every IDUC cycle time. The Object Server IDUC cycle is controlled by the Object Server Granularity property, the default value is 60 seconds. The Object Server informs the client of all events which were inserted, deleted, or updated since the last cycle, and the client uses that information to further query the Object Server for event data. 

ANSWER
By default, the client will use the Object Server Granluarlity period of 60 seconds to receive event data. The client can override this by setting it's individual IDUCFlushRate property. Setting the client's IDUC flush rate will increase the load on the Object Server. For example, setting the client IDUC Flush Rate to 30 seconds will increase the client load on the Object Server 2x. The client IDUC rate should not be decreased to 10 seconds or lower as this will greatly impact the Object Server performance. 

When changing a client's flush rate, monitor the Object Server trigger statistics log and profiler report log for the impact on Object Server performance. For more information on these logs, see the Omnibus Administration guide section on Performance Tuning:
