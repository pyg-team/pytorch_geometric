Title: IBM Encountering error  BIP2241E after applying a WTX patch. - United States

Text:
WTX IIBM lil 'DTXPlugInNode' TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Encountering error BIP2241E after applying a WTX patch 

SYMPTOM
The following error is encountered.

BIP2241E: A Loadable Implementation Library (.lil,.jar, or .par) is not found for message flow node 
type 'DTXPlugInNode' 


CAUSE
WebSphere Transformation Extender (WTX) is using the 58.5 version of XML4c and IBM Integration Bus (IIB) is using 58.0.

ENVIRONMENT
Using WMB 8.0.0.1 and WTX 8.4.0.x to run a WTX map.

DIAGNOSING THE PROBLEM
Applying WTX CORE/PARSER patch that contains xml4c 585 XML Parser.

RESOLVING THE PROBLEM
Amend the PATH environment variable so that WTX in the front of the path. 





For example on Linux :

Change the dtxwmqi_84_64.sh script 

from 

LD_LIBRARY_PATH=${LD_LIBRARY_PATH}:${DTX_LIBS} 

To 

LD_LIBRARY_PATH=${DTX_LIBS}:${LD_LIBRARY_PATH}
