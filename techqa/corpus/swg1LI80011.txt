APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When issuing an HA switch command, the standby appliance does
   not covert to primary as expected. The primary reboots and
   continues as primary.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Time synchronization does not happen between the Active and
   Standby appliances.  Because of this, ha switch can not happen.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The problem is due to internal script that synchronizes time in
   standby appliance to that of active appliance is failing because
   of missing sntp command and incorrect parameters to sntp in the
   script file. Missing command has been added and script corrected
   to pass proper parameters to sntp.
   
   This fix is targeted for inclusion in Cast Iron fix packs
   7.5.1.1, 6.4.0.2 and 6.3.0.3.
   
   
    
   
   

TEMPORARY FIX
