 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In an environment where discovery is performed using only a
   ping or filefinder, and feedback is disabled, we will still
   discover the device. In the FullDiscovery stitcher we feed the
   old discovery with the new discovery. We need to skip that step
   if feedback is disabled.
   
   
    
   
   

LOCAL FIX
 *  Change the following code in FullDiscovery:
               int pingFinderRunning = 0;
               Record isPingFinderRunning = RetrieveSingleOQL(
                   "select m_Name from disco.managedProcesses where
                       m_Name = 'ncp_df_ping';"
               );
               // Added by Paul Stroud(2/15/16)
               // We need to determine if feedback is enabled
               Record feedbackCheckRec = RetrieveSingleOQL(
               "select m_FeedbackCtrl from disco.config;"
               );
               // Default is feedback is on.
               int doFeedback = 1;
               if(feedbackCheckRec != NULL)
               {
               doFeedback = ExecEvalOnRecord(feedbackCheckRec,
   eval(int,'&m_FeedbackCtrl') );
               }
               delete(feedbackCheckRec);
               if(isPingFinderRunning != NULL)
               {
                   pingFinderRunning = 1;
               }
               delete(isPingFinderRunning);
               if(pingFinderRunning == 1 AND doFeedback <> 0)

PROBLEM SUMMARY

   * In an environment where discovery is performed using only a  *

   * will still discover the device that no longer in seed file.  *
   ****************************************************************

   * | fix pack | 4.2.0-ITNMIP-FP0004                             *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  In the FullDiscovery stitcher we feed the old discovery with the
   new discovery. We need to skip that step if feedback is
   disabled.
   In FullDiscovery.stch, check to see if m_FeedbackCtrl from
   disco.config is enabled, and if not, don't seed the discovery
   with the previous discoveries data.
