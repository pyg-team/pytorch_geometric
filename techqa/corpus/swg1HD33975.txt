ERROR DESCRIPTION
 *  Workflow License are not immediately released
   after any      Web Editor Logout
   Scenario :
   1. Install SMARTEAM V5R14SP3 Web Editor
   environment.
   Installed License numbers are following.
   (Web Editor = 5, Workflow = 5)
   2. Perform IISRESET on the Web Editor Server.
   3. Login Web Editor client as user ID "A1"on
   terminal "T1" using
   Workflow.
   (Actual exhausted License status by i4blt : Web
   Editor = 1,
   Workflow = 1)
   4. Login Web Editor client as user ID "A2"on
   terminal "T2" using
   Workflow.
   (Actual exhausted License status by i4blt : Web
   Editor = 2,
   Workflow = 2)
   5. Login Web Editor client as user ID "A3"on
   terminal "T3" using
   Workflow.
   (Actual exhausted License status by i4blt : Web
   Editor = 3,
   Workflow = 3)
   6. Login Web Editor client as user ID "A4"on
   terminal "T4" using
   Workflow.
   (Actual exhausted License status by i4blt : Web
   Editor = 4,
   Workflow = 4)
   7. Logout Web Editor client "A2"on terminal "T2".
   (Actual exhausted License status by i4blt : Web
   Editor = 3,
   Workflow = 4) <= KO
   The Workflow License status must be the same as
   Web Editor (=3).
   8. Logout Web Editor client "A4"on terminal "T4".
   (Actual exhausted License status by i4blt : Web
   Editor = 2,
   Workflow = 4) <= KO
   The Workflow License status must be the same as
   Web Editor (=2).
   9. Login Editor client as user ID "B1"on
   terminal "U1" and open the
   SmartBox.
   (Actual exhausted License status by i4blt :Web
   Editor = 2,
   Workflow = 5) <= KO
   The Workflow License status must be the same as
   Web Editor
   plus 1 (=3).
   10. Login Editor client as user ID "B2"on
   terminal "U2" and try
   opening the SmartBox.
   Result :
   However, the exhausted Workflow License numbers
   have already
   reached the limit.
   "Lacking of Workflow License" message is
   displayed.
   It is very severe impact in the
   customer's "Approval Workflow"
   processes.
   thanks Baruch
   *note Tere is also BID-67741 to this issue
   .
