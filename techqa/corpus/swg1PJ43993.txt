PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * Customers using Workplace XT with CPE 5.2.1.3-P8CPE-FP004    *
   * and prior version, when associating an entry template with a *
   * folder, and the folder preference annotation class has a     *
   * storage policy pointing to multiple storage areas.           *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * If a folder preference annotation class has a storage policy *
   * pointing to multiple storage areas, when the user associate  *
   * an entry template with the folder in workplace XT, the       *
   * following error may show up in Workplace XT:                 *
   *  Failed to save folder preferences of                        *
   * {11111111-1111-1111-1111-111111111111} folder in             *
   * MyObjectStore object store.                                  *
   * com.filenet.api.exception.EngineRuntimeException:            *
   * FNRCE0057E:E_READ_ONLY: The method failed because an object  *
   * or property is read-only. Attempt to modify settable only on *
   * create property,StorageArea.                                 *
   *                                                              *
   * This is caused by the way the batch in Workplace XT          *
   * separates the annotation creation and content update into 2  *
   * steps, which may cause the storage area used by content      *
   * upload is different than the storage area used by creation.  *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Upgrade to CPE 5.2.1.4-P8CPE-FP004 or higher.                *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  This issue is fixed by detecting the storage area mismatch
   between the one picked up by content upload and the one picked
   up at annotation creation time, an internal storage area update
   is allowed to correct the storage area for the annotation to
   match the content upload storage area.
   
   Fixed in 5.2.1.4-P8CPE-FP004 and higher.
