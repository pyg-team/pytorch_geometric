APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When Datapower is used for authentication by MobileFirst
   Platform, it was observed that on android device the LTPAtoken
   provided by Datapower is not persisted when the application is
   force closed.
   In which case, even if the token has a not expired, on
   re-opening the application the user is forced to login to the
   application.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * User's who perform MFP authentication using gateway's like   *
   * IBM Datapower in Android Native and Hybrid Environments.     *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * When Datapower is used for authentication by MobileFirst     *
   * Platform, it was observed that on android device the         *
   * LTPA-token provided by DataPower is not persisted when the   *
   * application is force closed. In which case, even if the      *
   * token has a not expired, on re-opening the application the   *
   * user is forced to login to the application. This issue is    *
   * observed in android native and hybrid environments.          *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  Non-expired LTPA tokens provided by Datapower is being persisted
   when the application is force closed. Users are not challenged
   with Login as long as the client has valid LTPA token in android
   native and hybrid environments.
