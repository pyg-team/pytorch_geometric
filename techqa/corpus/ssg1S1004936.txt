Title: IBM How to collect a supportsave from a Brocade switch - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How to collect a supportsave from a Brocade switch 

CAUSE
Brocade switch supportsave is needed by IBM support to look into any issues related to SAN.

ANSWER
Collecting supportsaves from the switch CLI requires that you have an internal FTP or SCP server in your network. Steps to collect supportsave. 

1. Log onto the switch as admin using telnet or ssh.
2. At the command prompt type: supportsave
3. You will be prompted as below:
a) the IP address of your FTP server.
b) the user name/password for your ftp server.
c) to select a protocol. ie: FTP
d) the remote directory on ftp server.

4. The files generated by this command are compressed before being sent off the switch. You can then collect (zip, tar, etc.) these files together before sending to IBM Support.

Example:
IBM_2498_B40:FID128:admin> supportsave
This command collects RASLOG, TRACE, supportShow, core file, FFDC data
and then transfer them to a FTP/SCP/SFTP server or a USB device.
This operation can take several minutes.
NOTE: supportSave will transfer existing trace dump file first, then
automatically generate and transfer latest one. There will be two trace dump
files transferred after this command.

Now logon to the ftp server and zip/tar the file together before uploading to IBM.
