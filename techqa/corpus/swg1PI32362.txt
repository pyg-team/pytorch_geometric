PROBLEM CONCLUSION
 *  The problem has been resolved.
   
   When using option  'I - Change installation default parameters'
   on panel ADB2P and CM is disabled, DB2 Admin will issue new
   error message ADB909E / msgadb909e:
   
   ADB909E:
   The profile table has not been created, or the profile table is
   created but CM is disabled.
   
   Explanation:
   The 'Installation default parameters' option is not available
   because creation of the Change Management database and/or
   the CM option at install time were disabled.  DB2 Admin
   Tool will use DB2 Utility default value limits instead.
   
   System action:
   Processing stops.  The DB2 Admin utility panels will allow
   you to specify the PARALLEL parameter according to the
   DB2 utility limits.  See DB2 Utility Guide and Reference for
   more information about the PARALLEL keyword.
   
   User response:
   If DB2 Utility default value limits are sufficient, then no
   action is needed.
   
   If there is a need to enable the DB2 Admin Tool option
   'I - Change installation default parameters' on panel ADB2P
   'DB2 Admin Options', then the following steps must be
   performed by the installer:
   
   Instructions for DB2 Admin Tool installer:
   
   1. Using IBM Tools Customizer for z/OS, navigate to panel
   CCQPWRK 'Customizer Workplace: DB2 Admin Tool' and
   issue line command "E" for input field 'Product parameters'.
   
   2.  On panel CCQPPRD 'Product Parameters', scroll several
   pages to the 'Admin Tool Setup Task (Create and Upgrade)'
   section and enable the following options:
   
   Change Management database . . . . . . . YES (YES, NO)
   Enable CM on DB2 Admin Tool primary menu YES (YES, NO)
   
   3.  F3/END to navigate back to panel CCQPWRK 'Customizer
   Workplace: DB2 Admin Tool' and issue line command "G" to
   re-generate job template ADBSETUP 'Admin Tool Setup Task'.
   
   4.  Submit template ADBSETUP 'Admin Tool Setup Task' job.
   
   5.  Submit template ADBBIND 'Binds'.
