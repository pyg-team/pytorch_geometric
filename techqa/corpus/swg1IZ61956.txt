APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The DirectSQL function is overwriting the value of the
   Datasource property within the policy scope.  This is resulting
   in errors like:
   
   com.micromuse.common.datasource.SQLDataSourceException in
   policy: <error>
   
   
    
   
   

LOCAL FIX
 *  Where possible use alternative Impact functions to query your
   datasource.
   
   
    
   
   

PROBLEM SUMMARY
 *  DIRECTSQL OVERWRITES THE DATASOURCE VARIABLE WITHIN POLICY
   
   
    
   
   

PROBLEM CONCLUSION
 *  DirectSQL overwrites DataSource varialbe name with whatever is
   used in DirectSQL: DirectSQL("mySource",Query,null);. DataSource
   variable name will be assigned to mySource after DirectSQL is
   being executed. As a result, the next Action using DataSource
   variable name will connect to an incorrect database.
   The issue is fixed in Netcool/Impact 5.1 Interim Fix 0002
