COMMENTS
 *  Prior to applying this APAR, run the following SQL to
   determine if any of the ARSAPP columns are currently
   too big to fit in the updated tables:
   
   SELECT NAME, AGID FROM ARSUSER.ARSAPP WHERE
         LENGTH(FIXED_VIEW)   > 10698  OR
         LENGTH(INDEXER)      > 10698  OR
         LENGTH(PREPROCESSOR) >  10698;
   
   If any rows are returned, DO NOT APPLY this SYSMOD.  The
   columns will need to be reduced in size before this SYSMOD
   is applied.  Contact IBM Support for assistance.
   
   1. Rollup the changes from DB2 Content Manager OnDemand for
      Multiplatforms fixpacks 7.1.2.8 and 7.1.2.9
   
   2. A new ars.cfg setting, ARS_USER_EXITS_DIR, is provided.  If
      specified, this causes OnDemand to look in the specified
      directory for user exits.  Note That if this is specified,
      this is the only directory OnDemand will search.  It will not
      fall back to /usr/lpp/ars/bin/exits if the userexit cannot be
      loaded.
   
   3. Two new ars.cfg settings are provided.
      ARSMVS_TABLESPACE_COMPRESS=YES | NO
      ARSMVS_TABLESPACE_TRACKMOD=YES | NO
   
   Will cause OnDemand to put a COMPRESS YES | NO and/or a
   TRACKMOD YES | NO clause on the CREATE TABLESPACE
   for the application group data table tablespaces.
   
   Upgrading customers will need to perform the following after
   APPLYing this SYSMOD. Customers will need to make sure they have
   a valid database backup and recovery plan prior to performing
   this procedure:
   
    1. Customize and execute the ARSTSPCC member of
       SARSINST to create the additional tablespaces.
    2. Stop the OnDemand Server.
    3. Backup the OnDemand Database or make sure they
       have a current backup.
    4. Run "arsdb <-I instance_name> -vx ARSSYS"
    5. Run "arsdb <-I instance_name> -vx ARSAPP"
    6. Run "arsdb <-I instance_name> -vd ARSSYS"
    7. Run "arsdb <-I instance_name> -vd ARSAPP"
    8. Run "arsdb <-I instance_name> -vc ARSSYS"
    9. Run "arsdb <-I instance_name> -vc ARSAPP"
   10. Run "arsdb <-I instance_name> -vi ARSSYS"
   11. Run "arsdb <-I instance_name> -vi ARSAPP"
   12. Run "arsdb <-I instance_name> -vu"
   13. Start the OnDemand Server
