Title: IBM Saving a JDBC Agent fails with error: CNS-CNS-1933 - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempting to edit and save an existing JDBC Agent fails with error.


In the connection pool configuration for the affected datasource, increase the value for the maximum connections (exact steps vary depending on the application server being used). 

SYMPTOM
CNS-CNS-1933 Unable to validate JDBC agent . Ensure that the datasource or the JDBC URL is correct. 
Caused by: 
Connection not available, Timed out waiting for <n> 
Caused by: 
Connection not available, Timed out waiting for <n> 


CAUSE
This issue is the result of having no connections available in the connection pool for the selected datasource.

RESOLVING THE PROBLEM
In the connection pool configuration for the affected datasource, increase the value for the maximum connections (exact steps vary depending on the application server being used).
