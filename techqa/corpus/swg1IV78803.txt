PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: Users viewing TEP with an Oracle Java Runtime
   ****************************************************************
   * PROBLEM DESCRIPTION: The SOA topology would not render on an
   * Oracle JVM. A NoClassDefFoundError would be thrown, looking
   * for org.apache.xalan.client.XSLTProcessorApplet
   *
   * XSLTProcessorApplet is part of the standard IBM Java Runtime
   * but not part of the Oracle one. The code which depended on
   * this missing class was rewritten to remove the dependency.
   ****************************************************************
   * RECOMMENDATION: Apply IFix 7 when available.
   ****************************************************************
