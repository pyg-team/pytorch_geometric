ERROR DESCRIPTION
 *  The "Save sent mail" value of desktop policy is not reflected
   immediately.
   
   Steps to Reproduce:
   Prepare two users. ( Admin Notes, User Notes )
   1. User Notes start a Notes client.
   2. Go to File - Preferences - Mail - Sending and Receiving.
   3. Change the value of "Save copies of messages that I send:"
   item into "Never"
   4. Save a preferences.
   5. Admin Notes start Domino Administrator, and create a desktop
   setting doc.
   6. Open Preferences - Mail
   7. Change the value of "Save sent mail" item into "Always" and
   select "Set value and prevent changes".
   8. Set "MyDesktop" into Name, and save a Desktop setting doc.
   9. Create Desktop policy.
   Go to [Basic].
   - Policy Name: DTPol
   - Policy Type: Explict
   - Desktop: MyDesktop
   Go to [Policy Assignment]
   - Set "User Notes" into Users and Groups.
   10. Save a Desktop policy doc.
   11. User Notes restart a Notes client.
   12. Open a mail database.  Close a mail database after "Notes
   configuration settings have been refreshed" message was
   displayed in status bar.
   DCC finished and the Desktop policy was pushed.
   13. Go to File - Preferences - Mail - Sending and Receiving.
   The value of "Save copies of messages that i send:" item is
   "Never" and locked( gray out ).
   14. User Notes sent a mail. However, a copy message was not
   saved in his mail database.
   
   
    
   
   

LOCAL FIX
 *  Restart Notes client.The value of "Save copies of messages that
   I send:" item changed to "Always".
