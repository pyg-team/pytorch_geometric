Title: IBM 'Adapter not found' using the ITX Soap adapter on Sterling Integrator. - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 'Adapter not found' using the IBM Transformation Extender (ITX)/WebSphere Transformation Extender (WTX) Soap adapter on Sterling Integrator. 

SYMPTOM
'Adapter not found' using the ITX Soap adapter on Sterling Integrator.


CAUSE
Sterling Integrator is unable to locate the SOAP JAR files needed to run the ITX/WTX map.

ENVIRONMENT
Running a ITX/WTX map using the ITX/WTX Soap adapter on Sterling Integrator.

DIAGNOSING THE PROBLEM
The JNI log contains the following errors :

ERROR: Unable to retrieve classloaderinfo for jni env: 0x7bf9ee00, class: 

*ERR: in _mjpiWTXLogCallback (-2)

*ERR: Function failed (-3) in dtxjpiGetConfig()





RESOLVING THE PROBLEM
Add the following jar files to the external section of the ITX Dtx.ini file. 

dtxpi.jar, m4soap.jar, and xds4j.jar

For Example :

If ITX was installed in C:\IBM\WebSphere Transformation Extender 9.0.0 on windows.

You would add the following to the dtx.ini file.

[External Jar Files] 
jar1=C:\IBM\WebSphere Transformation Extender 9.0.0\m4soap.jar 
jar2=C:\IBM\WebSphere Transformation Extender 9.0.0\dtxpi.jar 
jar3=C:\IBM\WebSphere Transformation Extender 9.0.0\xds4j.jar
