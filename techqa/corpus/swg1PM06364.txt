PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  Users of WebSphere Application Server v7    *
   *                  Feature Pack for Service Component          *
   *                  Architecture (SCA) Refresh Pack v1.0.1      *
   ****************************************************************
   * PROBLEM DESCRIPTION: ClassCastException occurs when using    *
   *                      <ts:wireFormat.jmsText/> from SCA       *
   *                      Feature Pack.                           *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   There is no equals() method implementation for
   WireFormatJMSText. As a result, in
   JMSBindingReferenceBindingProvider.configureBindingChain(),
   the check to ensure that we don't add the
   WireFormatJMSTextReferenceInterceptor twice is failing. In the
   case where we are adding the intepretor twice, the
   classCastException occurs since it's already a JMSMessage
   after the first interceptor.
   
   
    
   
   

PROBLEM CONCLUSION
 *  dded equals() method implementation to WireFormatJMSBytes and
   WireFormatJMSText to address the issue.
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 1.0.1.3.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
