ERROR DESCRIPTION
 *  Batch : Running CATDUA batch with documents
   selected from ENOVIA VPM input elements are not
   seen
   .
   SCENARIO:
   1. Start ENIVOA VPM
   2. Launch CATIA V5R19SP02
   3. Click on Tools menu and select Utility.
      Batch monitor dialog box appears
   4. Double click on CATDUAV5 option.
      CATDUA dialog box appears
   5. Select Clean option in CATDUA dialog box
   6. Select document location option as ENOVIA VPM
   7. Click on browse option
   8. Click on connect to VPM option and make a connect
   9. Query for documents in the database
      (any document will do)
   .
   OBSERVATION1:
   Panel information appears saying N number of
   documents found
   .
   10. Click on OK from the search panel
   .
   PROBLEM:
   In the document selection area, in the multi list,
   selected documents are not seen. Selected lines
   are highlighted as blank lines.
   .
   EXPECTED RESULT:
   Input documents should be visualized.
   .
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Batch : Running CATDUA batch with documents
   selected from ENOVIA VPM input elements are not seen
   
   
    
   
   

PROBLEM CONCLUSION
 *  THIS PROBLEM WILL BE FIXED ON CATIA VERSION
   5 RELEASE 20 GA LEVEL.
   NOTE THAT THIS PROBLEM WILL ALSO BE FIXED
   ON V5R19SP04
   .
   Incident Diagnosis
   Records were selected but were invisible in list.
   .
   Correction:
   Records selected are now invisible in list.
   .
   Answer and By-Pass
   Inside 'Output Naming Definition' dialog, User
   must have attributes from two different VPM tables
   selected under 'Selected Attributes List'
   .
