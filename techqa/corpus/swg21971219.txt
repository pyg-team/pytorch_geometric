Title: IBM How to update driver from ojdbc6.jar to ojdbc7.jar in order to connect to Oracle 12.1.0.2 Database ? - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How to update driver from ojdbc6.jar to ojdbc7.jar in order to connect to Oracle 12.1.0.2 Database ? 

CAUSE
In order to connect to Oracle 12.1.0.2 database with SI525, database driver needs to be updated from ojdbc6.jar to ojdbc7.jar when JDK used by sterling integrator is JDK 1.7

ANSWER
 

1)Stop SI 

2)Run the command : ./install3rdParty.sh Oracle 12.1.0.2 -d "path of the Oracle driver"/ojdbc7.jar 
This command will install the new driver on your system and modify the configuration files 
3)Comment all the entries in dynamicclasspath.cfg and dynamicclasspath.cfg.in linked with the old driver: 

In the example below the &INSTALL_DIR for this installation is /products/gis/ 

dynamicclasspath.cfg 
#VENDOR_JAR=/products/gis/dbjar/jdbc/Oracle/ojdbc6.jar 
#DB_JAR=/products/gis/dbjar/jdbc/Oracle/ojdbc6.jar 
#DB_JAR=/products/gis/dbjar/jdbc/Oracle/ojdbc6.jar 

dynamicclasspath.cfg.in 
#DB_JAR=&INSTALL_DIR;/dbjar/jdbc/Oracle/ojdbc6.jar 

5)In sandbox.cfg 
Modify the path from ojdbc6.jar to the new path : 
JDBC_DRIVER=/products/gis/dbjar/jdbc/Oracle/ojdbc6.jar 
DB_DRIVERS=/products/gis/dbjar/jdbc/Oracle/ojdbc6.jar 

Will become 
JDBC_DRIVER=/products/gis/dbjar/Oracle/12.1.0.2/ojdbc7.jar 
DB_DRIVERS=/products/gis/dbjar/Oracle/12.1.0.2/ojdbc7.jar 

6)Modify the path from ojdbc6.jar to the new path ojdbc7.jar in the following files : 
AGENTDynamicclasspath.cfg
AGENTDynamicclasspath.cfg.in
ACTIVEMQDynamicclasspath.cfg
ACTIVEMQDynamicclasspath.cfg.in
APPDynamicclasspath.cfg
APPDynamicclasspath.cfg.in 

7)Restart SI 

8)Verify you are able to connect to the DB with the new driver
