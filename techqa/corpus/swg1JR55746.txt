APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When multiple logon and remember me is enabled, if a shopper is
   logged on with rememberMe=true and updateCookies=true, and if
   the user token is removed from the CTXMGMT table, the shopper
   cannot re-logon through REST services.
   
   A 401 error is returned and you might see a
   javax.ejb.ObjectNotFoundException error message.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  USERS AFFECTED:
   WebSphere Commerce Version 7 Feature Pack 8.
   
   PROBLEM ABSTRACT:
   401 error when shopper is logged on with rememberMe=true and
   updateCookies=true
   
   BUSINESS IMPACT:
   Shopper cannot logon through REST services.
   
   RECOMMENDATION:
   
   
    
   
   

PROBLEM CONCLUSION
 *  Handle the exception appropriately when WebSphere Commerce is
   trying to find the activity token by the id that is stored in
   the persistent cookie.
