APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Performance hit due to a combination of 3 issues:
   1. Multiple HTTP calls made for IID to build a picture of whats
   on the PC
   2. Retrieving archived information which adds a little
   functionality at the large performance cost
   3. Retrieving BPD/AIS information for snapshots which adds a
   little functionality at the large performance cost
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  User connecting from IID to Process Center  *
   ****************************************************************
   * PROBLEM DESCRIPTION: Refresh and Publish from IID to PC takes*
   *                      a long time when there are a large      *
   *                      number of PA/TKs, snapshots or bpd/AISs *
   ****************************************************************
   * RECOMMENDATION:  Ifix is Recommended                         *
   ****************************************************************
   Performance hit due to a combination of 3 issues:
   1. Multiple http calls made for IID to build a picture of whats
   on the PC
   2. Retrieving archived information which adds a little
   functionality at the large performance cost
   3. Retrieving bpd/ais information for snapshots which adds a
   little functionality at the large performance cost
   
   
    
   
   

PROBLEM CONCLUSION
 *  IMPORTANT: To be effective a mandatory server side fix is
   required, JR42484 [http://www-01.ibm.com/support/docview.wss?uid=swg1JR42484], see
   http://www-01.ibm.com/support/docview.wss?uid=swg1JR42484 for
   more information.
   
   The fix:
   Added the following functionality to improve performance for the
   3 issues:
   
   1. Multiple http calls consolidated into 1 call
   2. Disabled retrieving archived information
   3. Disabled retrieving bpd/ais information for snapshots
   
   The fix for version 7.5.1 is available in fix
   central and is also fixed in future releases.
