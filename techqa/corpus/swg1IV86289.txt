ERROR DESCRIPTION
 *  PROBLEM:
   After adding the value "SRFC" using internal value "SR" in
   domain TKCLASS (type SYNONYM), Ticket Template functionality
   stopped to work ie. when trying to use this option, it does not
   return ticket template and displays the error below:
   
   BMXAA6713E - The MBO fetch operation failed in the mboset with
   the SQL error code 1427. The record could not be retrieved from
   the database. See the log file for more details about the
   error. java.sql.SQLException: ORA-01427: single-row subquery
   returns more than one row
   
   PERFORMANCE ISSUE:  No
   STEPS TO REPRODUCE:
   1 Login in as maxadmin Go to System Configuration -> Platform
   Configuration -> Domains
   2 Open TKCLASS domain, and add a row with the following
   attributes (Internal Value: SR, Value: SRFC)
   3 Now go to SR application.   Service Desk -> Service Request
   4 Click icon to create a new SR
   5 Click on icon next to Common Request
   6  Notice the problem, there is no value to select
   7 If you are in template field and you hit <enter> you will get
   the error
   CURRENT ERRONEOUS RESULT:
   BMXAA6713E - The MBO fetch operation failed in the mboset with
   the SQL error code -811. The record could not be retrieved from
   the database.
   See the log file for more details about the error
   EXPECTED RESULT:
   No error should happen.
