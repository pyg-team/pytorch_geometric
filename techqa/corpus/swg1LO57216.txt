ERROR DESCRIPTION
 *  DESC: users not more able to delete Folders in own mail database
   error message :
        "You are not authorized to perform that operation."
   
   users have editor access with access level
   
   x delete documents
   x Replicate Documents
   x create shared folders/views
   
   
   if additional
   
   x create privat folders/views
   is enabled in ACL -> users also able to delete folders
   
   but this permission should not enabled
   
   Environment:
   Notes/Domino 802
   Windows 32 bit
   
   Steps to reproduce the issue:
   1. Install Notes/Domino 802
   2. Ensure you register an user with Editor access to his mail
   file and the ACL:
   x delete documents
   x Replicate Documents
   x create shared folders/views
   3. Create a shared Folder, from Create => Folder actions
   4. Delete the Folder from: Actions => Folder Options => Remove
   Folder
   You get the error message:
    "You are not authorized to perform that operation."
   
   If you add the ACL settings, Create privat folders/views, you
   are then able to delete the folder.
   
   
    
   
   

LOCAL FIX
 *  You can delete the folder by right-click on the Folder and then
   Remove Folder.
   
   Also, you can upgrade to Notes 851 or 852 as the issue is not
   occuring on these versions.
