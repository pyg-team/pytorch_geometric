Title: IBM IT06390: HTTPSCONNECTOR STARTUP FAILURE IF CIPHER NAMES STARTS WITH TLS - United States

Text:

WebSphere Message Broker V7.0 - Fix Pack 7.0.0.8 [http://www-01.ibm.com/support/docview.wss?uid=swg24041187]

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  As per IBM JSSE doc, cipher suite names may begin with "SSL" or
   "TLS". However, if you configure HTTPSConnector with a cipher
   name of TLS prefix, it will fail during startup with the
   following exception in the stdout.
   
   
   java.lang.IllegalArgumentException: CipherSuites may not be null
   at com.ibm.jsse2.s.(s.java:86)
   at com.ibm.jsse2.cd.setEnabledCipherSuites(cd.java:51)
   
   An example of configuring cipher names with TLS prefix is
   
   mqsichangeproperties  -e  -o HTTPSConnector -n
   ciphers -v
   \"TLS_RSA_WITH_AES_128_CBC_SHA,TLS_RSA_WITH_AES_256_CBC_SHA\"
   
   
    
   
   

LOCAL FIX
 *  Change the cipher suite name to start with the "SSL" prefix

   cipher name of TLS prefix, it will fail during start-up with the
   following exception in stdout

   at com.ibm.jsse2.cd.setEnabledCipherSuites(cd.java:51)

   \"TLS_RSA_WITH_AES_128_CBC_SHA,TLS_RSA_WITH_AES_256_CBC_SHA\"
   
   There are a number of resource name changes between WebSphere
   Message Broker and IBM Integration Bus Version 9.0. For details
   visit
