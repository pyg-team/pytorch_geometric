PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All Netcool/Impact users who are using       *
   *                 Hibernation Action FunctionN                 *
   ****************************************************************
   * PROBLEM DESCRIPTION: IN ORDER TO REMOVE THE HIBERNATION      *
   *                      SUCCESSFULLY, YOU MUST FIRST STRIP THE  *
   *                      POLICYSTATE:  FROM THE RETURN VALUE OF  *
   *                      GETHIBERNATIN                           *
   ****************************************************************
   * RECOMMENDATION:                                              *
   *                                                              *
   ****************************************************************
   Steps to replicate --------------------- NOE: replicating this
   issue assumes Impact 3.1 Fix Pack4 is not installed 1) Install
   and Configure Impact 3.1 Server. 2) Run Impact Server 3) Create
   a policy using Hibernate() action function  Please refer to
   Netcool/Impact 3.1 Policy Admin guide on how to use Hibernate()
   4) Create another Policy using RemoveHibernation() to remove
   the Hibernation Keys. The policy should use
   GetHiberatingPolicies() to select the Key that was created by
   the Hibernate() Action Function in step 3  Please refer to
   Netcool/Impact 3.1 Policy Admin guide on how to use
   RemoveHibernation() and GetHibernatingPolicies(). 5) Start
   HibernatingPolicyActivator service. 6) Run the policy you
   created in Step 3 to start the Hibernation 7) View the data
   items for Hibernation Data Type under the global Repository
   Data Types. You should a key create by Step 3 policy. 8) Run
   the policy you created in Step 4 to remove the Hibernation key
   8) Check The Hibrnation Data Items again. You will observe that
   the Key wasn't removed
