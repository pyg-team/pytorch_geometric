PROBLEM(ABSTRACT)
 You are using the CICS Explorer and are connecting to your CICSPlex SM (CPSM) Web User Interface (WUI) server CMCIPORT for the first time. The connection fails with messages DFHXS1111, DFHWB0361 and DFHAC2003. The WUI server has been configured with SEC=YES for security in the CICS system initialization table (SIT). 

SYMPTOM
You receive the following messages:

DFHXS1111 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/DFHXS/DFHXS1111.html] yyyy/mm/dd hh:mm:ss applid tranid Security violation by user userid { at netname | at console } portname for resource resource in class classname. SAF codes are (X'safresp',X'safreas'). ESM codes are (X'esmresp',X'esmreas').
DFHWB0361 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/DFHWB/DFHWB0361.html] yyyy/mm/dd hh:mm:ss applid An attempt to attach a CICS Web alias transaction for userid userid has failed because the user is not authorized to execute transaction tranid. Host IP address hostaddr. Client IP address: clientaddr. TCPIPSERVICE tcpipservice.
DFHAC2003 [http://pic.dhe.ibm.com/infocenter/cicsts/v5r1/topic/com.ibm.cics.ts.messages.doc/DFHAC/DFHAC2003.html] yyyy/mm/dd hh:mm:ss applid Security violation has been detected term id = termid, trans id = tranid, userid = userid.


CAUSE
The CICS Explorer userid does not have sufficient authority to attach either the CWXN or CWWU transactions.

ENVIRONMENT
CICS TS V4.1 or later and CICSPlexSM V4.1 or later WUI Server

RESOLVING THE PROBLEM
Ensure your CICS Explorer userid has permission to attach the CWXN and CWWU transaction identifiers.

If you continue to receive errors, see the MustGather for CICS Explorer errors and incorrect output [http://www.ibm.com/support/docview.wss?uid=swg21385861] to determine what documentation to gather before contacting CICS Support.
