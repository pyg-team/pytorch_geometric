APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When invokding a Sterling B2Bi business process that contains
   the SPE Deenvelope service, it can fail with an error when the
   following conditions are met:
   
   1) The input data file name contains a space
   2) The control numbers in the input data do not match
   
   Error message:
   
   "INVALID_CHARACTER_ERR: An invalid or illegal XML character is
   specified"
   
   
    
   
   

LOCAL FIX
 *  B2BSI - RTC536987
   TR / TR
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   Customers using ITXA with B2Bi
   
   Problem Description:
   An input filename with a space causes spedeenvelope service to
   fail with "INVALID_CHARACTER_ERR: An invalid or illegal XML
   character is specified"
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  Resolution Summary:
   SPE services in B2Bi updated to replace invalid XML characters
   in document names
   
   Delivered In:
   B2Bi 5262_4
