Title: IBM FileNotFoundException during installation - United States

Text:
install; zip; unzip TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 You receive a FileNotFoundException error when your WebSphere Operational Decision Manager product installation fails. 

SYMPTOM
The errors reported by the installer are:

CRIMC1029E: Adding plug-in org.eclipse.datatools.enablement.ibm.informix.dbdefinition_1.0.4.v201005
211230 to repository C:\Program Files\IBM\IBMIMShared failed.

CRIMC1085E: Resumable download failed for: C:\ODMInstallation\disk2\WDS\plugins\org.eclipse.datatools.enablement.ibm.informix.dbdefinition_1.0.4.v201005211230.jar.

'plug-in org.eclipse.datatools.enablement.ibm.informix.dbdefinition_1.0.4.v201005211230' does not exist (at C:\ODMInstallation\disk2\WDS\plugins\org.eclipse.datatools.enablement.ibm.informix.dbdefinition_1.0.4.v201005211230.jar).

java.io.FileNotFoundException: C:\ODMInstallation\disk2\WDS\plugins\org.eclipse.datatools.enablement.ibm.informix.dbdefinition_1.0.4.v201005211230.jar (The system cannot find the file specified.)


CAUSE
The issue might be with the version of WinZip used to extract the content of the WebSphere Operational Decision Manager or Operational Decision Manager TAR installer archives. 

You can find similar issues referring to old versions of Winzip reported online.

ENVIRONMENT
WinZip

DIAGNOSING THE PROBLEM
Looking at the content of the installer archives on the local disk, we see indeed that the file is there, but has a “.jar0000644” extension instead of ".jar":

org.eclipse.datatools.enablement.ibm.informix.dbdefinition_1.0.4.v201005211230.jar0000644

RESOLVING THE PROBLEM
Ensure that you have the latest version of Winzip or the latest version of 7-zip to extract the WebSphere Operational Decision Manager or Operational Decision Manager installer archives. 





Note: When installing with WinZip, consult these recommendations [http://www-01.ibm.com/support/docview.wss?uid=swg27023696#inst04].
