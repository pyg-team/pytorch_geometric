APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Used Command Assistance to generate a command to change to the
   module mapping of an application . When the command is run an
   error is seen:
   WASX7110E: Incorrect number of elements in option:....
     -
   These steps recreate the problem:
     1. Applications > Enterprise Applications >
        DefaultApplication.ear > Manage Modules
     2. Change the mapping to include a cluster and a server
     3. Click OK
     4. Click Command Assistance to see the command that was
        generated.  Mine is:
       AdminApp.edit('DefaultApplication.ear',
        '[ -MapModulesToServers
         [[ "Increment Enterprise Java Bean"
             Increment.jar,META-INF/ejb-jar.xml
             WebSphere:cell=mshenT41Cell61-11,node=Node61,
              server=server2+
              WebSphere:cell=Cell61,cluster=singleCluster
              13 moduletype.ejb "EJB Module" ]
          [ "Default Web Application"
             DefaultWebApplication.war,WEB-INF/web.xml
             WebSphere:cell=Cell61,node=Node61,
              server=server2+
              WebSphere:cell=Cell61,cluster=singleCluster
               13 moduletype.web "Web Module" ]]]' )
     5. When I ran this on wsadmin I got the same error
        "WASX7015E: Exception running command:..."
     6. These parts of the command should not be there:
            [ "Default Web Application"
               DefaultWebApplication.war,WEB-INF/web.xml
               WebSphere:cell=Cell61,node=Node61,
                server=server2+
                WebSphere:cell=Cell61,cluster=singleCluster]]]' )
        AdminConfig.save()
     8. Looking at the console I see the changes were made.
      -
   
   The server trace shows that the console application panel has
   passed the extra 3 columns (13 moduletype.ejb "EJB Module")
   data to the AdminAppCommand.  Those 3 columns are hidden
   columns in MapModulesToServers install task and should not
   be displayed (ModuleVersion, moduleType and moduletypeDisplay).
   
   
    
   
   

LOCAL FIX
 *  Modify the command by hand per step 7.
    -
   Search keywords: console admin wsadmin $Admin $Config Jython
   jacl
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of WebSphere Application Server   *
   *                  V6.1 who want to use the command assistance *
   *                  generated when editing an Enterprise        *
   *                  application.                                *
   ****************************************************************
   * PROBLEM DESCRIPTION: Extra columns are included in the       *
   *                      command assistance command that         *
   *                      prevents the command from running       *
   *                      successfully.                           *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Extra data that the AdminApp commands could not run with was
   being generated for command assistance.  This command could
   not run successfully.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The extra columns are no longer displayed as part of the
   command assistance AdminApp command so that the command can
   now run.
   
   The fix for this APAR is currently targeted for inclusion in
   fixpack 6.1.0.23.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
