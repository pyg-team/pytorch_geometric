Title: IBM JDBC Adapter V7.5 - RetrieveAll operation throws ResourceException: Resource Adapter instance is NULL - United States

Text:
jdbc adapter; Database Vendor TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 JDBC Adapter V7.5 throws ResourceException in the doDeepRetrieve() method when executing the RetrieveAll operation. The above exception is thrown when the "DatabaseVendor" property in the managed connection factory is not set. 

SYMPTOM
The exception below can be seen for the RetrieveAll operation: 

--------------------------------------------------------------------------------------------
com.ibm.j2ca.jdbc.JDBCRecord doDeepRetrieve() Exception occured
while doRecursiveRetrieve
javax.resource.ResourceException: Resource Adapter instance is
null
--------------------------------------------------------------------------------------------


RESOLVING THE PROBLEM
This issue has been resolved in JDBC Adapter Interim Fix v7.5.0.5_IF01. JDBC Adapter now works correctly even when the DatabaseVendor property is not specified in the MCF properties.
