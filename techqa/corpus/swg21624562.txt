Title: IBM When migrating from IBM Business Process Manager (BPM) V7.5.1 to V8.0, an UpgradeDatabase-related error shows in the log file - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After the migration process from V7.5.1 to 8.0 succeeds, the upgraded Process Server and Performance Data Warehouse databases show that the migration process succeeded. However, an error exists in the system log file that was generated after you executed the upgrade_7x.bat command script. 

SYMPTOM
You see the following messages in the system log file: 

com.lombardisoftware.tools.dbupgrade.UpgradeDatabase - DB2 SQL Error: SQLCODE=-204, SQLSTATE=42704, SQLERRMC=DB2INST2.AM_TASKS, DRIVER=4.11.69
com.ibm.db2.jcc.am.SqlSyntaxErrorException: DB2 SQL Error: SQLCODE=-204, SQLSTATE=42704, SQLERRMC=DB2INST2.AM_TASKS, DRIVER=4.11.69 at com.ibm.db2.jcc.am.gd.a(gd.java:676).
.
com.lombardisoftware.tools.dbupgrade.UpgradeDatabase - IBM Business Process Manager V8.0.0 Enterprise database upgrade program finished.


CAUSE
The error message occurred because the constraints are not being dropped correctly when the database upgrade process runs. SQL statement failed with DROP DB2INST2.AM_TASKS.


RESOLVING THE PROBLEM
This issue has been identified as a product defect under APAR JR45550 [http://www.ibm.com/support/docview.wss?uid=swg24034495]. Apply Version 8.0.1 Fix Pack 1 [http://www.ibm.com/support/docview.wss?uid=swg24034495], which includes APAR JR4550 and fixes this issue
