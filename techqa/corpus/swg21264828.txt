Title: IBM Application Status in the Administrative Console is unknown - United States

Text:
WASADM; cluster; application; status; console; adminconsole; appstatus; unknown TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Even though the Enterprise Applications are running properly, the Administrative Console reports the Applications' status as unknown. The Application Servers and NodeAgents status do report correctly, the issue is only with applications. 

SYMPTOM
After following the Administrative Console Mustgather [http://www-1.ibm.com/support/docview.wss?rs=180&uid=swg21199337], the Dmgr trace shows the following:

CAUSE
The problem is being caused by an erroneous file in the install_root/config/cells/[cellName]/clusters/ directory. This could be a .jar file, .xml file, .log file, etc...

There should be no files under the clusters directory, except a directory whose name is the cluster name.


RESOLVING THE PROBLEM
Remove the extra file from the configuration. Then synchronize the changes and restart all the servers (Dmgr, NodeAgent, and Application Servers).
