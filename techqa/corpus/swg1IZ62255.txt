ERROR DESCRIPTION
 *  Environment: Solaris 10
   
   Problem Description: The Status attribute of the Solaris Zone
   attribute group shows a wrong status (e.g. it shows the status
   of "shutting down" instead of the actual status of "running"
   reported by the OS command "zoneadm -list -cv").
   The problem is due to the fact that Solaris has broken the
   backward compatibility for the zone_status_t enum in the
   /usr/include/sys/zone.h header file in Solaris 10 Update 7:
   139555-08 (sparc) and 139556-08 (x86).
   
   Recreation procedure:
   1. Install the ITM UNIX OS Agent on Solaris 10 Update 7 or
   later.  This can be determined with the following command:
   
   # uname -a
   SunOS marte 5.10 Generic_139555-08 sun4v sparc
   
   Consider that the Kernel Update 139555-08 corresponds to Solaris
   10 Update 7.
   
   2. In the Tivoli Enterprise Portal, refresh the Solaris Zone
   workspace and check that the Status attribute does not
   correspond to one listed by "zoneadm -list -cv"
