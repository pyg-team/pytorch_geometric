CAUSE
Load Module for WLM Address Space was running in APF-Authorized library. Also user was pointing to the incorrect plan for their EDITSIN.

DIAGNOSING THE PROBLEM
Create the /tmp/edidproc directory under the HFS. Then rerun scenario and this will show the plan that is being pointed to. Duplicate the problem using Server commands from WDI Client. Check the messages in the tmp directory /tmp/edidproc.

RESOLVING THE PROBLEM
1. Point to the correct plan in the EDITSIN.




2. Remove the WDI Loadlib from APF-Authorized.
