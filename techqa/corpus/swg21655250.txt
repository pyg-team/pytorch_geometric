Title: IBM %GETCOL function exception - United States

Text:
%GETCOL function usage TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Exception occurred during replication. 
%GETCOL for table <SCHEMA>.<TABLENAME> returned no rows satisfying the selection
criteria. A SQL exception has occurred. The SQL error code is '-204'.
The SQL state is: 42704. 

SYMPTOM
Replication failed and the error message is found in cdc log: 

DB2 SQL Error:SQLCODE=-204, SQLSTATE=42704, SQLERRMC=<DBname>.<schemaname>.<tablename>,
DRIVER=4.15.82 A SQL exception has occurred. The SQL error code is
'-727'. The SQL state is: 56098. The error message is: DB2 SQL Error:
SQLCODE=-727, SQLSTATE=56098,
SQLERRMC=2;-204;42704;<DBname>.<schemaname>.<tablename>, DRIVER=4.15.82



CAUSE
The error indicates the table is not found due to user selected source database schema and table name instead of target database value


ENVIRONMENT
The error is observed in a replication setting from DB2 source to Sql Server target



RESOLVING THE PROBLEM
Expression with %GETCOL function is evaluated at the target side. User needs to check the %GETCOL function and select target side schema name and table name for the expression to work.
