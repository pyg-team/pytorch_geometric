ABSTRACT
 This document lists the fix packs and interim fixes required for the operation of IBM WebSphere Portal version 6.1.0.5 and 6.1.5.2 deployed on these levels of supporting software.

List of minimum levels of required fix packs and interim fixes for WebSphere Application Server and Process Server deployed with WebSphere Portal version 6.1.0.5 or 6.1.5.2.




The required fix pack level and additionally recommended maintenance for WebSphere Portal V6.1.0.5 or 6.1.5.2 to function properly when deployed with WebSphere Application Server or Process Server is listed in this document. The minimum fix pack levels required are: 


 * WebSphere Application Server: V6.1.0.31 or 7.0.0.11 (distributed platforms) 
 * [optional:] WebSphere Process Server (client code version tested): V6.2.0.2, 6.1.2.3 or 6.1.0.4 
 * [optional:] WebSphere Virtual Enterprise (formerly "Extended Deployment") V6.1.0.5 or 6.1.1.1


Download links for these fix packs (choose the appropriate platform from the page):  * WebSphere Application Server version 6.1.0.31 for distributed platforms: 

http://www.ibm.com/support/docview.wss?uid=swg27009661 [http://www.ibm.com/support/docview.wss?uid=swg27009661]  * WebSphere Application Server version 7.0.0.11 for distributed platforms: 

http://www.ibm.com/support/docview.wss?uid=swg24026852 [http://www.ibm.com/support/docview.wss?uid=swg24026852]  * WebSphere Process Server fix packs can be retrieved from http://www.ibm.com/software/integration/wps/support/download.html [http://www.ibm.com/software/integration/wps/support/download.html] 
 * WebSphere Virtual Enterprise (formerly "Extended Deployment") Server fix packs can be retrieved from http://www.ibm.com/software/webservers/appserv/extend/support/download.html [http://www.ibm.com/software/webservers/appserv/extend/support/download.html]



Installation and support for these fixes and fix packs is provided by the appropriate WebSphere product support teams, not WebSphere Portal support. When installing non-Portal interim fixes, you must use the appropriate Update Installer [http://www.ibm.com/support/docview.wss?uid=swg24012718] which is not provided with the downloads, and NOT the WebSphere Portal Update Installer. 

Service releases later than the levels listed above might also be used and in most cases are considered to be untested but supported as described in the WebSphere Portal Support Statement [http://publib.boulder.ibm.com/infocenter/wpdoc/v6r1m0/topic/com.ibm.wp.ent.doc_v6101/overview/inst_req_supt.html]. 

Customers choosing to install a later service level of WebSphere Application Server or Process Server must ensure on their own that the level they are using includes some or all of the co-requisite Application Server or Process Server interim fixes listed in the Information Center and Release Notes, and then install any additional individual interim fixes not included.
