APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Client gets SAXParseException while updating spec
   value with ?&amp;? inside <value> tag.
   
   Snippet of sample request xml:
   
   <ProductSpecValueBObj>
    <ProductSpecValueId/>
        ...
   
         <AttributeValueBObj>
      <Action>update</Action>
      <Path>/MyProductSpec</Path>
                             <Value>
      <![CDATA[
           <MyProductSpec
   xmlns="http://www.example.com/MyProductSpec/internal/00000001"
   xmlns:ps="http://www.example.com/MyProductSpec/internal/00000001
   ">
                   <MyProdName>My ProductName &amp; Testing
   </MyProdName>
   
         </MyProductSpec>
      ]]>
       </Value>
     </AttributeValueBObj>
   </ProductSpecValueBObj>
   
   Snippet of sample response xml:
   
   <DWLError>
                   <ComponentType>99</ComponentType>
                   <Detail>com.ibm.mdm.common.jpal.JPALException:
   org.xml.sax.SAXParseException: The entity name must immediately
   follow the '&amp;' in the entity reference.</Detail>
                   <ErrorMessage>Handle Schema Validation
   Failed.</ErrorMessage>
                   <ErrorType>READERR</ErrorType>
                   <LanguageCode>100</LanguageCode>
                   <ReasonCode>9606</ReasonCode>
                   <Severity>0</Severity>
   
   <Throwable>com.ibm.mdm.common.jpal.JPALException:
   org.xml.sax.SAXParseException: The entity name must immediately
   follow the '&amp;' in the entity reference.</Throwable>
               </DWLError>
   
   
    
   
   

LOCAL FIX
 *  This problem has been fixed in 9.0.2.25. Please download and
   install this PTF if you experience this problem.
