APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  ha_manage_mq_data_manually, ha_existing_data_path and
   ha_do_not_install_mq_advanced_components properties in MQ
   Multi-Instance pattern templates are not referenced in the
   links between active and standby nodes in High availability
   standby instance configuration.
   
   
    
   
   

LOCAL FIX
 *  To link these properties between active and standby nodes in a
   MQ multi-instance pattern template, redraw the "Data
   Dependencies" line or add the property references to the
   configuration.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   This issue affects users of the  IBM MQ Virtual System Pattern
   Type, who wish to make use of the MQ multi-instance pattern
   templates.
   
   
   Platforms affected:
   Linux on x86-64, AIX
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   A packaging issue within the IBM MQ Virtual System Pattern Type
   meant that the following link properties were not included in
   the links between active and standby nodes in the MQ
   multi-instance pattern templates:
   
   - ha_manage_mq_data_manually
   - ha_existing_data_path
   - ha_do_not_install_mq_advanced_components
   
   
    
   
   

PROBLEM CONCLUSION
 *  The IBM MQ Virtual System Pattern Type packaging has been
   updated to included these properties in the links between nodes
   in the multi-instance pattern templates.
