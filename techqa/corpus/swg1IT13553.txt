Title: IBM IT13553: ORA-01483 WHEN INSERTING MORE THAN 1996 BYTES INTO BLOB COLUMN WHEN TABLE ALSO HAS A TIMESTAMP COLUMN - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  An insert into an Oracle table fails with error ORA-01483 when
   the following conditions are true:
   
   1) More than 1996 bytes of data is being inserted into a BLOB
   column.
   2) A TIMESTAMP column follows the BLOB column.
   3) The Oracle database is using a Japanese character set, for
   example, NLS_CHARACTERSET = JA16SJIS.
   4) The DataDirect ODBC V7.1 Oracle driver is being used.
   
   The full error text is:
   BIP2322E: Database error: SQL State 'HY000'; Native Error Code
   '1483';
   Error Text '[unixODBC][IBM][ODBC Oracle Wire Protocol
   driver][Oracle]ORA-01483: invalid length for DATE or NUMBER bind
   variable'.
   
   
    
   
   

LOCAL FIX
 *  Use the DataDirect ODBC V7.0 Oracle driver from IBM Integration
   Bus v9.0.0.2 or earlier.
