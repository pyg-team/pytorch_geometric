ERROR DESCRIPTION
 *  The installation or upgrade of the Tivoli Storage Manager for
   Virtual Environments - Data Protection for VMware might stop at
   the vCenter credential validation step with the message :
   "unexpected error with vCenter validation"
   This can happen if the vCenter SSL certificate uses a key
   bigger than 2048 bits.
   This is also the minimum required key length starting with the
   vCenter Version 6.0.
   
   Tivoli Storage Manager Versions Affected:
   All supported versions of Tivoli Storage Manager for Virtual
   Environments - Data Protection for VMware on Windows and Linux
   
   Customer/L2 Diagnostics (If Applicable)
   Without canceling the installation process and while the error
   is displayed, get and save the regtool.log file located in the
   temporary installation files then cancel the installation.
   
   In the TDPVMwareInstallation.log, the following example
   messages are seen :
   
   1: eWAS_port_check: running
   ["C:\install\TSM4VE_71\ISP_FOR_VE_713_DP_VMW_WIN_ML_CN6M0ML\tsm4
   ve\X64\..\extra\jre\jre\bin\javaw.exe" -jar
   AvailablePortVerificator.jar -port 9080] Working
   Dir[C:\Users\u104017\AppData\Local\Temp\3\{E34A074D-DD5C-4FA1-AB
   54-9DEB04255412}]
   1: eWAS_port_check: Default port is not available.
   
   In the regtool.log, the following will be seen :
   
   <timestamp> com.ibm.tivoli.tsm.vmware.RegisterPlugin perform
   INFO: -validate_any option chosen
   ...
   <timestamp>
   com.ibm.tivoli.tsm.vmware.RegisterPluginbase.openConnection
   SEVERE: FFDC
   com.sun.xml.internal.ws.client.ClientTransportException:
       HTTP transport error: javax.net.ssl.SSLKeyException: RSA
   premaster secret error
       at com.sun.xml.internal.ws.transport.
       at ...
   ...
   INFO: Wrong user name or password for vCenter Server
   
   Initial Impact: Medium
   
   Additional Keywords: spectrum protect login connection DCF
   1972226 1972802
   
   
    
   
   

LOCAL FIX
 *  Ensure the vCenter certificate has a key no bigger than 2048
   bits.
   If the security requirements need this key to be bigger,
   contact IBM support.

PROBLEM CONCLUSION
 *  During the vCenter credential validation step, the Data
   Protection for VMware installer will detect when the vCenter
   Server SSL connection fails due to an RSA key encryption length
   issue. The installer will provide a technote to resolve this
   issue so installation can complete. See technote:
   http://www-01.ibm.com/support/docview.wss?uid=swg21976853 [http://www-01.ibm.com/support/docview.wss?uid=swg21976853]
