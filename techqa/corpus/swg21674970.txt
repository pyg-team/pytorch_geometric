Title: IBM Executing RFT script from command line results in "CRFCN0511E: Image [pause_playback.gif] not found" error - United States

Text:
RFT; java; jre; command line; CRFCN0511E; Image [pause_playback.gif] not found TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Attempts to execute IBM Rational Functional Tester (RFT) script from command line results in error: CRFCN0511E: Image [pause_playback.gif] not found. 

SYMPTOM
When running below command from Windows command line window to execute RFT script: 

java -classpath "C:\Program Files (x86)\IBM\SDP-RFT\FunctionalTester\bin\rational_ft.jar" com.rational.test.ft.rational_ft -datastore C:\IBM\rftworkspace\ietest -playback ietest -logfolder default -log ie-cmd-Log

Command returns below error message:

CAUSE
The java.exe used in command line refers to a 64 bits JRE, while only 32 bit JRE is supported by RFT.


RESOLVING THE PROBLEM
Switching to 32 bit JRE resolves the error, in one of the following ways: 

Option 1: Set system/user PATH to pointing to a 32 bit JRE 

Option 2: Use absolute path to call a 32 bit JREin command line
