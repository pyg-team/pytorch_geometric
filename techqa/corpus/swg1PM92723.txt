Title: IBM PM92723: INPLACE EDIT RESOURCE ENVIRONMENT VARIABLES MAY BE MISSING AFTER SERVER UPDATE - United States

Text:
  A FIX IS AVAILABLE
Fixes integrated in WebSphere Portal 8.0.0.1 Combined Cumulative Fixes [http://www-01.ibm.com/support/docview.wss?uid=swg24034497]



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When upgrading from 8000 directly to 8001 CF05 or 8001
   CF06, inplace editing may not work even after the
   'enable-new-8001-features' ConfigEngine task is executed.
   
   The problem occurs if the required inplace resource
   environment variables within the WCM ConfigService are missing
   on the updated environment.
   
   
    
   
   

LOCAL FIX
 *  To manually add the required environment variables:
   1. Execute the 'action-update-inplace-resource-environment'
   ConfigEngine task
   2. Restart the server
   
   
    
   
   

PROBLEM SUMMARY
 *  When upgrading from 8000 directly to 8001 CF05 or 8001 CF06,
   inplace editing may not work even after the
   'enable-new-8001-features' ConfigEngine task is executed.
   
   The problem occurs if the required inplace resource environment
   variables within the WCM WCMConfigService are missing on the
   updated environment.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The inplace editing resource environment variables will be added
   if they are missing on the server during any CF install.
