Title: IBM IT08738: UPGRADE JAVA RUNTIME ENVIRONMENT SUPPLIED WITH WEBSPHERE MQ 8.0 TO JAVA 7.1.3.0 - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The WebSphere MQ V8.0 installation contains a v7
   Java Runtime Environments (JREs), supplied within the JRE
   install component.
   New revisions of this JRE are released periodically to address
   program defects, and products which embed the JRE should
   be updated accordingly.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   USERS AFFECTED:
   The JRE which is embedded into WebSphere MQ is used
   to run the following Java components of the product:
   
     WebSphere MQ Managed File Transfer
     GSKit graphical interface
     WebSphere MQ Explorer
     WebSphere MQ Extended Reach (MQXR) service
   
   
   Platforms affected:
   AIX, Linux on Power, Linux on S390, Linux on x86, Linux on
   x86-64, Solaris SPARC, Solaris x86-64, Windows, Linux on zSeries
   
   ****************************************************************
   PROBLEM DESCRIPTION:
   This APAR updates the Java Runtime Environment (JRE) supplied
   with WebSphere MQ V8.0, to the level:
   
       Java 7.1 Service Refresh 3 (SR3)
   
   on all platforms except Solaris where the JRE is updated to
   level:
   
       Java 7.0 Service Refresh 9 (SR9)
   
   
    
   
   

PROBLEM CONCLUSION
 *  This APAR updates the JRE updates for WebSphere MQ V8.0 on the
   following platforms:
   
     AIX
     Linux (x86-32)
     Linux (x86-64)
     Linux (PPC64)
     Linux (PPC64LE)
     zLinux
     Solaris (x86-64) } Java 7.0 Service Refresh 9 (SR9)
     Solaris (SPARC) } Java 7.0 Service Refresh 3 (SR9)
     Windows 32-bit
     Windows 64-bit
   
   
   HP-UX has not been updated.
   
   ---------------------------------------------------------------
   The fix is targeted for delivery in the following PTFs:
