ABSTRACT
 This document contains system requirements for WebSphere MQ v5.3 for HP NonStop Server (Integrity). 

CONTENT
 

Operating Systems [/support/docview.wss?uid=swg27006797&amp;aid=1] HP NonStop Server H06.04 or later
- Plus a minimum of the following operating system patches and their associated requisite
patches (see the HP NonStop support site [https://onepoint.nonstop.compaq.com/buildpage.asp] for requisite details):  * SPR T8306H01^ABJ 
 * SPR T8994H01^AAM 
 * SPR T9050H02^AQG 
 * SPR T0288H02^AAG


Support for the HP Integrity NonStop Server BladeSystem is introduced with Fix Pack V5.3.1.4. The minimum supported operating system level is J06.03.00 with the following operating system patches:  * SPR T0694H01^AAC


Support for the HP Integrity NonStop BladeSystem NB54000c is introduced with Fix Pack V5.3.1.8. The minimum supported operating system level is J06.11.00 Hardware Requirements [/support/docview.wss?uid=swg27006797&amp;aid=1]  * Any HP NS-Series Integrity NonStop Server machine that is capable of running the HP NonStop Server H06.04, or later, operating system. 
 * From Fix Pack V5.3.1.4, any HP Integrity NonStop BladeSystem machine that is capable of running the J06.03.00 or later operating system release.


From Fix Pack V5.3.1.8, any HP Integrity NonStop BladeSystem NB54000c system that is capable of running the J06.11.00 or later operating system release Network Protocols [/support/docview.wss?uid=swg27006797&amp;aid=1] For the HP Integrity NonStop Server machine running H06.04 or later:  * NonStop TCP/IP or NonStop TCP/IPv6


From Fix Pack V5.3.1.4, for the HP Integrity NonStop BladeSystem running J06.03.00 or later:  * NonStop TCP/IP or NonStop TCP/IPv6 using the CLIM (with SPR T0694H01^AAC) or G4SA controller

Compilers [/support/docview.wss?uid=swg27006797&amp;aid=1]  * C and C+  * Open System Services (OSS) applications in C or C++: TNS/E Native C/C++ compiler (c89), or PIC linker (eld) and object file tool (enoft) 
    * Guardian applications in C: TNS C compiler (C), binder (BIND) and TNS/E Object Code Accelerator (OCA) or TNS/E Native C compiler (CCOMP), or PIC linker (ELD) and object file tool (ENOFT) 
    * Guardian applications in C++: TNS/E Native C++ compiler (CPPCOMP), or PIC linker (ELD) and object file tool (ENOFT) 
    * Windows cross-development environment applications in C or C++: HP Enterprise Toolkit (ETK) TNS/E Native C/C++ compiler (c89), or PIC linker (eld) and object file tool (enoft) 
   
   
 * COBOL  * OSS applications in COBOL85: TNS/E Native COBOL85 compiler (ecobol), or PIC linker (eld) and object file tool (enoft) 
    * Guardian applications in COBOL85: TNS COBOL89 compiler (COBOL), binder (BIND) and TNS/E Object Code Accelerator (OCA) or TNS/E Native COBOL85 compiler (ECOBOL), or PIC linker (ELD) and object file tool (ENOFT) 
    * Windows cross-development environment applications in COBOL85: HP Enterprise Toolkit (ETK) TNS/E Native COBOL85 compiler (ecobol), or PIC linker (eld) and object file tool (enoft) 
   
   
 * TAL  * Guardian applications in TAL: TNS TAL compiler (TAL), binder (BIND), and TNS/E Object Code Accelerator (OCA)
