CAUSE
This is caused by not enabling TLSv1.2 in IBM Docs while forcing the use of TLSv1.2 in HttpServer. 



RESOLVING THE PROBLEM
1. See the following procedure to configure your environment:
https://www.ibm.com/support/knowledgecenter/SSAW57_8.5.5/com.ibm.websphere.nd.doc/ae/tsec_transition_sp300.html [https://www.ibm.com/support/knowledgecenter/SSAW57_8.5.5/com.ibm.websphere.nd.doc/ae/tsec_transition_sp300.html]
Note: In step 3 of the preceding topic, select Strict but not transition. In step 17b, update ssl.client.props and set com.ibm.websphere.security.FIPSLevel equal to SP800-131. 

To make sure that the publishing issue is resolved, use the following link:
https://meisenzahl.org/2016/10/26/using-tls-1-2-only-configuration-with-ibm-connections/ [https://meisenzahl.org/2016/10/26/using-tls-1-2-only-configuration-with-ibm-connections/]
2. If you still do not see an Edit in IBM Docs button in Files, see hte following technote: 
http://www-01.ibm.com/support/docview.wss?uid=swg21997328 [http://www-01.ibm.com/support/docview.wss?uid=swg21997328]
