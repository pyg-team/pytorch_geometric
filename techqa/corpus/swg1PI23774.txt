Title: IBM PI23774: AppScan Standard multiple connections when publishing results to AppScan Enterprise - United States

Text:
 
APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  1) Expected behavior:
   AppScan Standard and ASE shouldn't crash when publishing results
   from AppScan Standard to ASE.
   2) Observed behavior:
   AppScan Standard and ASE crash when publishing results from
   AppScan Standard to ASE.
   3) Workarounds (if applicable):
   Use premium license for ASE
   4) Reproduction Steps:
       - Login to ASE as a standard/non-product administrator user.
      - Start a scan running in ASE
      - In ASD, open a scan (demo.testfire.net.scan), and then File
   ? Export ? Publish results to AppScan Enterprise
      - Let the on-load connectivity test finish
      - Click Test connectivity
      - Click Publish (should succeed)
      - Click Publish again after the last one finished, STD and
   ASE should crash during this attempt.
   5) Additional Technical detail:
   The user has basic ASE license, which has limited number of
   simultaneous connections.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   AppScan Standard and ASE crash when publishing results from
   AppScan Standard to ASE. This happens because ASE basic
   license is limited to 10 connections but AppScan uses many
   more connections.

 *  Fixed in 9.0.1
