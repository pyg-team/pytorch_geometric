ERROR DESCRIPTION
 *  Problem title
   NCM config modelling problem on Cisco 29xx
   .
   Problem description and business impact
   Using ITNcM v 6.4 on Linux platform we see modelling issue for
   Cisco 29xx with OS C2900-UNIVERSALK9-M-15.0(1)M7 using driver
   cisco_router_isrg2_15.x_v20130223.062211. We see issue with
   smartmodel in the interface section.
   
   We have the following configuration
   
   interface GigabitEthernet0/0
   description To_
    ip address
   ip verify unicast source reachable-via rx allow-self-ping
   ip flow ingress
   duplex auto
   speed auto
   no cdp enable
   standby 1 ip
   standby 1 timers 1 4
   standby 1 priority 105
   standby 1 preempt delay minimum 120
   standby 1 authentication md5 key-chain HSRP1
   standby 1 name
   standby 1 track 1 decrement 10
   !
   interface GigabitEthernet0/1
   description To_
   ip address
   ip verify unicast source reachable-via rx allow-default
   allow-self-ping
   ip flow ingress
   duplex auto
   speed auto
   no cdp enable
   standby 2 ip
   standby 2 timers 1 4
   standby 2 priority 105
   standby 2 preempt delay minimum 60
   standby 2 authentication md5 key-chain HSRP1
   standby 2 name Vendor
   standby 2 track 2 decrement 10
   !
   
   On interface Gig0/0, ITNcM models the configuration properly,
   however on  Gig0/1 interface, it doesn't model properly on the
   following section "ip verify unicast source reachable-via rx
   allow-default allow-self-ping"

 *  No workaround.
