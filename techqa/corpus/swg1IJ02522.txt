ERROR DESCRIPTION
 *  Product: IBM Tivoli Netcool/Impact
   Release: 7.1.0
   Severity: 3
   
   Problem Description: IBM Tivoli Netcool/Impact 7.1.0
   
   Need to update the Impact documentation to include what is
   replicated in an Impact cluster.
   
   A user assumed that the running state of a service in
   Impact is replicated in a cluster environment. i.e.
   
   1) Manually stop a service on the primary.
   2) Shutdown the primary.
   3) Secondary takes over as the acting primary.
   
   Note: The service is set to automatically start on startup
   
   4) After the secondary starts up the running state of the
   service is started (running).
   
   The user assumed if the service is stopped on the
   primary then when the secondary takes over as the acting
   primary the running state of the service on the secondary
   should also be stopped.
