LOCAL FIX
 *  Set the Java system property "swing.noxp=true" in a
   Java system properties file for the specific agent product
   to be configured.
   .
   Go to the $CANDLEHOME\TMAITM6 directory, and open the
   "k<pc>cma.ini" file and view the contents to check the
   value specified for "CanProd"
   .
   Then create a file with the following naming convention
   in the TMAITM6 directory:
       k<pc>config.sysprops.cfg
   Where you substitue the CanProd value found in the k<pc>cma.ini
   file for the <pc>:
   
   .
   Example for the VM agent:
   In kvmcma.ini:
    CanProd=kvm
   .
   Create a file in $CANDLEHOME\TMAITM6 directory:
    kkvmconfig.sysprops.cfg
   .
   In this file, create one line that contains:
     -Dswing.noxp=true
   
   
    
   
   

PROBLEM SUMMARY
 *  The Manage Tivoli Enterprise Services (MTEMS) tool's file
   browser button may not show the icon when the configuring an
   agent.
   
   Attempting to configure agents through MTEMS GUI does not bring
   up the Java configuration screen as Java is receiving a
   NullPointerException in Java SWING calls.
