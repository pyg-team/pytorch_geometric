Title: IBM CRFCN0601E, CRFCN0527E errors running RFT - United States

Text:
ITCAM; RFT; BWMGW0154E; "User interaction"; ForeGroundLockTimeout; registry; screen lock; 71RRT-RFTPLAN; 71RRT-RFTCONF; 71RRT-RFTREC; 71RRT-RFTPLAY; 71RRT-RFTTS; 71RRT-TS TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 RFT (Rational Functional Tester) script playback is not working correctly on the T6-Robotic Response Time agent. 

SYMPTOM
CRFCN0601E, CRFCN0527E 


CAUSE
The playback window must be active and unlocked for proper playback of RFT recordings.

DIAGNOSING THE PROBLEM
 

In the Robotic Response Time -> Playback view, the Verification Point Failures contain this error:

RESOLVING THE PROBLEM
Verify the screen on the playback agent is not locked. Watch the script playback several times on the t6 playback agent.


 * The RFT program install and running environment must have UAC (User Account Control) off on Windows 2008 and Windows 7. 
 * Look for other errors in the log that might indicate a more specific problem such as:  * - ERROR - piddles - PlaybackThreadPoolWorker-19 - com.ibm.tivoli.transperf.playback.rft.RFTPlaybackEngine.execute() - 
      Desktop is not active 
    * exception_message=CRFCN0557E: Activation failed when running under a Terminal Services environment. 
      This may be caused by using a minimized terminal window - try playing back without minimizing the terminal window (it does not need to be full-screen).
      
   
   
 * Verify the version of RFT is compatible with the ITCAM for Transactions agents Check Related URLs for compatibility information.
