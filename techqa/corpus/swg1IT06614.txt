Title: IBM IT06614: PUREAPPLICATION SYSTEM: AFTER REBOOTING A VIRTUAL MACHINE WITH MONITORING FOR WEBSPHERE IT GOES INTO SAFERUNNING STATE. - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After rebooting a WebSphere Application Server virtual machine
   that has the Monitoring for WebSphere agent (ITCAM) enabled it
   goes into SAFERUNNING state.
   
   Logs in the ITCAM for WebSphere agent contain the following
   error:
   
   Starting ITCAM Agent for WebSphere Applications ...
   KCIIN0521E ...Error encountered during product startup.
   ...
      Please check log file, if one exists, and env.config,
      yn.ini  or yn.environment if it exists in
      /opt/IBM/maestro/ITCAMWAS/config for possible errors.
   KCIIN0198E Unable to start agent. Please, check log file.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  There was a problem with the file system permissions for the
   user that starts the ITCAM for WebSphere agent.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A code fix is being provided that sets the
   correct file permissions for the user provided
   with the IBM System Monitoring version 1.0.2.3 service released
   with PureApplication System version 2.0.0.1 interim fix 2 and
   higher.
