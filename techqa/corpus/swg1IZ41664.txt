ERROR DESCRIPTION
 *  Webservice request can get rejected due to a typo in the xml
   namespace of the outbound SOAP header.
   
   Consider the following example:
   
   <soapenv:Header>
   <wsse:Security
   xmlns:wsse="http://docs.oasis-open.org/wss/2004/01/oasis-200401-
   wsswssecurity-secext-1.0.xsd">
   <wsse:UsernameToken>
   <wsse:Username>wcsadmin</wsse:Username>
   <wsse:Password
   Type="http://docs.oasis-open.org/wss/2004/01/oasis-200401-wssuse
   rname-token-profile-1.0#PasswordText">password123</wsse:Password
   >
   </wsse:UsernameToken>
   </wsse:Security>
   </soapenv:Header>
   
   The URLs:
   http://docs.oasis-open.org/wss/2004/01/oasis-200401-wsswssecurit [http://docs.oasis-open.org/wss/2004/01/oasis-200401-wsswssecurit]
   y-secext-1.0.xsd
   http://docs.oasis-open.org/wss/2004/01/oasis-200401-wssusername- [http://docs.oasis-open.org/wss/2004/01/oasis-200401-wssusername-]
   token-profile-1.0
   
   Should actually be:
   http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-wssecuri [http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-wssecuri]
   ty-secext-1.0.xsd
   http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-username [http://docs.oasis-open.org/wss/2004/01/oasis-200401-wss-username]
   -token-profile-1.0
   
   Notice the hyphen (-) between wss and wssecurity, and wss and
   username.
   
   The request can also get rejected due to IOWrapperException when
   parsing BOD messages.
   When WebSphere Commerce receives and handles a BOD response, it
   may report an error similar to the following:
   
   Exception thrown from ProcessOrderCmdImpl:
   CommerceSrvr  E com.ibm.commerce.
   command.ECCommandTarget executeCommand
   CMN0420E: The following command
   exception has occurred during processing:
   "java.lang.RuntimeException:
   org.eclipse.emf.ecore.resource.
   Resource$IOWrappedException: Premature
   end of file.".java.lang.RuntimeException:
   org.eclipse.emf.ecore.
   resource.Resource$IOWrappedException:
    Premature end of file.
   at com.ibm.commerce.foundation.services.invocation.
   AbstractSDOInvocationServiceObjectImpl.
   setXML(AbstractSDOInvocationServiceObjectImpl.java:165)
   at com.ibm.commerce.
   foundation.services.invocation.InvocationService.invoke
   (InvocationService.java:105)
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  USERS AFFECTED: All users that use webservice and have schema
   validation enabled on the remove service provider or return BOD
   messages that have new lines or spaces after the SOAP body tag
   instead of regular text.
   
   PROBLEM ABSTRACT: Webservice request is rejected due to
   incorrect xml namespace url in SOAP header or IOWrapperException
   when parsing BOD messages.
   
   BUSINESS IMPACT: MEDIUM: Impact on providing webservice.
   
   RECOMMENDATION:
   
   
    
   
   

PROBLEM CONCLUSION
 *  This APAR corrects the xml namespace url in the SOAP header of a
   webservice request.
