APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Using SORTUPBY function in a IBM transformation Extender
   (ITX) V9.0.0.2 map rule does not encounter an error as expected.
   The map completes successfully but there is no output created.
   
   SORTUPBY is a new feature in ITX 9.0.0.3.
   
   
    
   
   

LOCAL FIX
 *  ITXCQ - ITX00058959
   GK / GK
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   IBM Transformation Extender customers who compile maps in
   release X and try to run them in a release earlier than X.
   
   
   Problem Description:
   If a map encounters an unknown "opcode" in a rule, it was just
   returning NONE for that opcode.
   
   Platforms Affected:
   All
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code has been changed to set a "map must be recompiled"
   return code and abort the map.
   
   Applies to:
   9.0.0.3
   
   Fixed in the next service packs and releases.
