APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  PureApplication System at V1.1.0.2. V1.1.0.3 V1.1.0.4
   -
   There is a warning issued when the connection is lost which can
   be easily overlooked:
   "CWZIP6227W PureSystems Manager could not communicate with IBM
   Storwize V7000 system"
   
   Symptoms:
   
   - Extend and Capture and provisioning failing
   - Failed LUNs reporting in Hardware View and storage
   - Compute node in failed state.
   - Storge collection set does not return V7000 Storwize "SNAP"
   data.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  If connectivity is lost to the V7000 storage we need to attempt
   to recover and also do a better job of informing the customer of
   the problem.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The storage query code has been updated to attempt a recovery
   and also generate a "Call home" event that will be more visible
   in the Events page.
   
   
    
   
   

TEMPORARY FIX
