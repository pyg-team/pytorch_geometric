 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Events that went on hibernation after wake up, not all, but a
   small fraction of events were not process correctly.
   
   This issue appears that the hibernation does wake up but the
   policy is not being run. Could be a thread timing issue.
   
   
    
   
   

LOCAL FIX
 *  67651.999.760.nci201503260056v6110b681.jar
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Customers using hibernation may notice that                  *
   * Some events that are hibernated do not update the event in   *
   * the ObjectServer                                             *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   Customers using hibernation may notice that
   Some events that are hibernated do not update the event in the
   ObjectServer.
   All threads were using one instance of the Hibernate function so
   they were sharing data.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The parser engine was changed to load a new instance of
   functions as used so that the threads do not share the same
   function class instances.
