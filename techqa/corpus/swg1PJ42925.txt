 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In IBM Case Manager Configuration Tool, when running the
   Register the IBM Case Manager Services Plug-in, the test
   connection might fail with the following error when IBM Content
   Navigator (ICN) is not configured with the
   com.ibm.ws.security.server.lm.wsMapDefaultInboundLoginModule
   custom property in WebSphere.
   
   Register the IBM Case Manager Services Plug-in
   There was an error executing the Register the IBM Case Manager
   Services Plug-in task.
   The task failed because of the following error: Login error:
   com.ibm.security.krb5.KrbException, status code: 6
    message: Client not found in Kerberos database
   
   This will happen if ICN is incorrectly configured, and the
   following custom property is not set:
   com.ibm.ws.security.server.lm.wsMapDefaultInboundLoginModule
   
   In WebSphere, navigate to Global Security > JAAS > Application
   Logins > Navigator >
   com.ibm.ws.security.server.lm.wsMapDefaultInboundLoginModule.
   There should be a custom property named cookie, which should be
   set to true.
   
   Steps to reproduce, when ICN is configured for SPENGO Kerberos
   SSO
   1. Launch the Case Manager configuration tool and create a
   websphere development profile.
   2. Run all the tasks up to the "Register the IBM Case Manager
   Services Plugin" task.
   3. Select the test connection button on the "Register the IBM
   Case Manager Services Plugin" task.
   
   Expected Result:
   The test connection works successfully.
   
   Actual result:
   The test connection fails and when you run the task, the task
   also fails.
   
   
    
   
   

LOCAL FIX
 *  In WebSphere, navigate to Global Security > JAAS > Application
   Logins > Navigator >
   com.ibm.ws.security.server.lm.wsMapDefaultInboundLoginModule.
   There should be a custom property named cookie, which should be
   set to true.
