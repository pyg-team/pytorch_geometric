Title: IBM Unstructured Data Stage returns Apache Error - United States

Text:
unstructured; apache; xbean; xmlbeans TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When using Unstructured Data Stage it returns error messages similar to:
org/apache/xmlbeans/XmlOptions.setSaveAggresiveNamespaces()Lorg/apache/xmlbeans/XmlOptions
org.apache.poi.POIXMLDocument (initialization error) 

SYMPTOM
When attempting to load data using the Unstructured Data stage

CAUSE
Issue related to upgrade, engine tier machine may contain xmlbeans-2.3.0.jar and xbean.jar older than xmlbeans-2.3.0.jar in the same place.
The root cause of issue is that old xml beans jar(ASBNode/lib/java/xbeans.jar) file was not deleted during in-place upgrade installation. 


DIAGNOSING THE PROBLEM
Verify if xmlbeans and xbean both exists in the same folder.



RESOLVING THE PROBLEM
Confirm to have followed all the steps as described on the following Info Center link: http://pic.dhe.ibm.com/infocenter/iisinfsv/v9r1/index.jsp?topic=%2Fcom.ibm.swg.im.iis.ds.unstructureddatastage.usage.doc%2Ftopics%2Fuds_ex2_config_uds.html [http://pic.dhe.ibm.com/infocenter/iisinfsv/v9r1/index.jsp?topic=%2Fcom.ibm.swg.im.iis.ds.unstructureddatastage.usage.doc%2Ftopics%2Fuds_ex2_config_uds.html] 

If the issue persists, follow the steps below: 

1) Login to 'engine' tier machine.
2) Goto D:/IBM/InformationServer/ASBNode/lib/java folder
3) Check both xbean.jar and xmlbeans-2.3.0.jar exists in this folder. If both JAR file exists, go to 4)
4) Click 'Stop ASBAgent' icon on the desktop
5) Backup xbean.jar and remove this jar from ASBNode/lib/java folder.
6) Click 'Start ASBAgent' icon on the desktop.
