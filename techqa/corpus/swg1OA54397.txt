APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In HOD when trying to capture diagnostics for a transport
   trace, the trace does not show up when the settings are set
   to save on the server. This happens when using html page
   config.  The popup indicates the trace is saved, but gives no
   location and no trace can be found on the server.
   
   
    
   
   

LOCAL FIX
 *  Save to local or Java Console
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All  Host On-Demand version 12.0 users who are using HTML    *
   * Based model.                                                 *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * User was unable to save transport trace on server when using *
   * HTML based model.                                            *
   * While saving the trace file user gets "Saved on server"      *
   * message however the transport trace was not getting saved.   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * User can upgrade to Host On Demand Version 12.0.3.0 to       *
   * resolve this problem.                                        *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  User can upgrade to Host On Demand Version 12.0.3.0 to resolve
   this problem.
   With the latest version Transport trace gets saved on server
   only when user logs in by using his credentials."Save Location"
   option is retained in Config server based Model.
   
   
    
   
   

TEMPORARY FIX
 *  NA
