Title: IBM JAZZ 1.1.0 breaks Existing console integration Federation connections - United States

Text:
JazzSM 1.1.0; DASH 3.1.1.0; APMUI; Federation; Console; connections; Null; break TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 After JazzSM 1.1.0 was released, a bug was discovered around Federation which caused current connections to break:
"The page being launched does not exist: apmui_itportal-Federation.1418164510940" 

SYMPTOM
After the initial JazzSM 1.1.0 DASH 3.1.1.0 upgrade, the following error occurs when trying to access the connection in the Console Integration" menu: 


[/support/docview.wss?uid=swg21692723&amp;aid=1] 

 

When viewing the "connection profiles" in DASH, the ID's show up as null. 


[/support/docview.wss?uid=swg21692723&aid=2] [/support/docview.wss?uid=swg21692723&aid=2]


CAUSE
After JazzSM 1.1.0, DASH 3.1.1 was released, Development found issues around Federation connections. This was fixed in a refreshed version of the software and released November 18, 2014


ENVIRONMENT
JazzSM 1.1.0, Dashboard Application Services Hub version: 3.1.1.0-201410211752

RESOLVING THE PROBLEM
This is a known issue introduced in the initial release of JazzSM 1.1.0 DASH 3.1.1.0 and fixed with the latest refreshed version of JazzSM version 1.1.1 which includes DASH 3.1.1.0-201411062200 ( release date of Nov 6, 2014 )
