Title: IBM QRadar Incident Forensics fix for CVE-2015-0235 (GHOST) Vulnerability - United States

Text:
CVE-2015-0235; Forensics; QRadar Incident Forensics; GHOST; Vulnerability; update; script RELEASE NOTES

ABSTRACT
 This update contains a fix for CVE-2015-0235 (GHOST vulnerability) and installation instructions. To install this update, you must have QRadar Incident Forensics 7.2.4 Patch 1 installed. 

CONTENT
If your deployment is installed with QRadar Incident Forensics 7.2.4 Patch 1, you can install CVE-2015-0235 to update glibc libraries. For information on QRadar Incident Forensics Patch 1, see the following release notes: http://www.ibm.com/support/docview.wss?uid=swg27044477 [http://www.ibm.com/support/docview.wss?uid=swg27044477]. 

 


Before you begin 

Ensure that you take the following precautions: 

 * Back up your data before you begin any software upgrade. For more information, see the IBM Security QRadar Incident Forensics Administration Guide [http://public.dhe.ibm.com/software/security/products/qradar/documents/7.2.4/QIF/EN/b_forensics_ag.pdf]. 
 * To avoid access errors in your log file, close all open QRadar sessions. 
 * Verify that all changes are deployed on your appliances. The patch cannot install on appliances that have changes that are not deployed. 




About this task 

This update resolves the gblic vulnerability identified as CVE-2015-0235 for QRadar Incident Forensics.

NOTE: This patch is not intended to update core QRadar appliances. Two separate fix packs are available to update QRadar deployments (7.2.4 Patch 4 [https://ibm.biz/BdE3V4] fix pack & 7.1 MR2 Patch 10 [https://ibm.biz/BdE3Vr] fix pack. The CVE update below is only intended for QRadar Incident Forensics or QRadar Packet Capture appliances. 




Procedure 

 *  Download the CVE-2015-0235-fix.sh.gz file from the IBM Fix Central website: https://ibm.biz/BdETHA [https://ibm.biz/BdETHA] (IBM Shortened URL) and copy the file to your QRadar Incident Forensics appliance. Using SSH, log in to your QRadar Incident Forensics appliance as the root user. Copy the fix to the /tmp directory on the . 
   Note: If space in the /tmp directory is limited, copy the fix pack to another location that has sufficient space. Change to the directory where you copied the patch file. For example, cd /tmp. To extract the file, type the following command: gunzip CVE-2015-0235-fix.sh.gz. To set the permissions on the fix pack, type the following command: chmod +x CVE-2015-0235-fix.sh.gz To run the patch installer, type the following command:./tmp/CVE-2015-0235-fix.sh
   The following message is displayed:
   
   A reboot is required to ensure that all process with the libraries loaded have been restarted. This patch must be manually applied on all systems in your deployment, including HA Secondaries.
   
   Do you want to apply this patch? [y/n] y
   
   Press Y to install the update.
   
   The following updates are applied:
   1:glibc-common
   2:nss-softokn-freebl
   3:glibc
   4:glibc-headers
   5:glibc-devel
   6:nss-softokn
   7:nss-softokn-freebl
   8:glibc
   After the patch completes the installation, type the following command: reboot. *IMPORTANT* Repeat this procedure for any additional QRadar Incident Forensics appliances in your network. Administrators and users should clear their browser cache before logging in to the Console. 
 * 


Results After the appliance reboots, administrators can update any other QRadar Incident Forensics appliances in their network. We also recommend that administrators send an email to their team to inform them that they will need to clear their browser cache before logging in to the QRadar interface. 

 

 







Issues resolved in update CVE-2015-0235 for QRadar Incident Forensics

 * glibc-2.12-1.149.el6_6.5.x86_64.rpm
 * glibc-common-2.12-1.149.el6_6.5.x86_64.rpm
