Title: IBM Unable to use IBM Standardization Rules Designer - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 It's our first time attempting to login to the QualityStage Rules Designer. When we try to login we get HTTP request error 

SYMPTOM
It's our first time attempting to login to the QualityStage Rules Designer. When we try to login we get HTTP request error


DIAGNOSING THE PROBLEM

Whether the database is DB2 or Oracle, the above errors point to one of the following 

 1. the table does not exist 
 2. the table is under a different schema. SRD does not use the xmeta schema, so the errors should not refer to xmeta. 
 3. the connection details are incorrect
    
    

RESOLVING THE PROBLEM
 

 1. run the scripts in IS_install_path/SQLScripts/qssrd folder to create the database. Please see https://www.ibm.com/support/knowledgecenter/SSZJPZ_11.5.0/com.ibm.swg.im.iis.productization.iisinfsv.install.doc/topics/cfgodb_regreposodb.html [https://www.ibm.com/support/knowledgecenter/SSZJPZ_11.5.0/com.ibm.swg.im.iis.productization.iisinfsv.install.doc/topics/cfgodb_regreposodb.html] for more information 
 2. review the scripts and confirm that you can actually login to the database that was created 
 3. Login to the WebSphere Administration Console 
    Navigate to Resources -> JDBC -> Data Sources  1. Click on QSSRD DataSource 
     2. On the right hand side under "Related Items" click on "JAAS - J2C authentication data" 
     3. Click on QSSRD Credentials 
     4. Confirm the username ( usually srduser ) and update the password. 
     5. Click Apply 
     6. Restart WebSphere
