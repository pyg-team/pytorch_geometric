ERROR DESCRIPTION
 *  ENOVIA ADMIN: CATIA CRASH WHEN OPEN A
   PART FROM A SECOND CACHE SERVER
   .
   Scenario:
   1. Create three V5R13SP06 LCA seperate
   installations as follows:
   a) One vault cache server and its client
   b) Second vault cache server and its client
   c) Third master server and client accessing vault
   directly
   2. Open CATIA, ENOVIA on a client using the first
   cache server i.e. (a).
   3. Create a product in Enovia
   4. Send it to Catia
   5. Insert a part under the product in catia and
   model with certain geometry
   6. Save back to Enovia
   7. Open the same part in enovia on third client i.e.
   (c)
   8. Send it to catia : opens in catia, works OK, close
   it.
   9. Select the part click RMB and send the same
   part to local hard disk.
   10. Part is saved sucessfully on hard disk. File size
   shown OK.
   11. Open the same part in enovia on a client using
   the second cache server i.e.
   (b)
   10. Send it to catia
   .
   Result: CATIA closedown suddenly without any
   warning
   Expected result: Should be able to open in CATIA
   same as that of first cache server.
   .
   11. Though catia crashed, since enovia is still open,
   select the part click RMB and send the same part to
   local hard disk.
   .
   Result: File is saved. but filesize is shown 0 KB
   Expected result: Should be able to save same as
   first cache server.
   .
