Text:
Connection Pooling; Java; CLI; Standalone; App; Application TECHNOTE (FAQ)

QUESTION
 How do we achieve connection pooling in stand alone Java applicaitons? 

ANSWER
If you are not using an application server, you have to manage the connection pooling within the application code. JDBC/JCC driver can not achieve pooling. So, to make sure connections are managed and pooled properly; application has to be designed to support the extra feature. 
You can use JCC driver's APIs to define connection pools. Because, you are not using application server, you have to still manage connection resources and pools in your application.

Here is code snippet : 
..........
Class.forName("com.ibm.db2.jcc.DB2Driver").newInstance();
// Type 4, ConnectionPoolDataSource
DB2ConnectionPoolDataSource db2ds = new com.ibm.db2.jcc.DB2ConnectionPoolDataSource();

db2ds.setDatabaseName( sDBname );
db2ds.setServerName( sServer );
db2ds.setDriverType( 4 );
db2ds.setPortNumber( Integer.parseInt( sPortNumber ) );
db2ds.setDescription( sDescription );
db2ds.setUser( sUserid);
db2ds.setPassword( sPassword );

// Type 4, ConnectionPoolDataSource
pc = db2ds.getPooledConnection();
conn = pc.getConnection();
