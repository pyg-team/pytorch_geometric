Title: IBM SA94148: MQM400 CHANNEL SHUTDOWN WITHOUT AMQ6993 EXCEPTION - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Too many 'channel end' conditions log exception AMQ6993 as a
   way of forcing an FFST report. When a network crashes with
   lots of client connections, this can severely impact moving
   these channels to an inactive state.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Too many 'channel end' conditions log exception AMQ6993 as a
   way of forcing an FFST report. When a network crashes with
   lots of client connections, this can severely impact moving
   these channels to an inactive state.
   
   
    
   
   

PROBLEM CONCLUSION
 *  When a channel job ends, the following list of return codes will
   NOT cause an AMQ6993 'Program  ended abnormally' escape message
   to be logged :-
   *  rrcI_NORMAL_CHANNEL_END      (AMQ9001)
   *  rrcE_CONNECTION_CLOSED       (AMQ9209)
   *  rrcI_LISTENER_STOP_REQUEST   (AMQ9234)
   *  rrcE_STATUS_TABLE_FULL       (AMQ9513)
   *  rrcE_REMOTE_QM_TERMINATING   (AMQ9525)
   *  rrcI_QM_TERMINATING          (AMQ9542)
   *  rrcI_DISCINTERVAL_EXPIRED    (AMQ9545)
   *  rrcE_MAX_ACTIVE_CHANNELS     (AMQ9573)
   *  rrcI_NO_LISTENERS_FOUND      (AMQ9597)
   *  rrcI_LISTENERS_TERMINATING   (AMQ9598)
