WebSphere Application Server's SystemOut.log will contain an exception: "java.lang.OutOfMemoryError: Java heap space"


CAUSE
WebSphere Application Server instance ran out of Java heap space


ENVIRONMENT
Windows



DIAGNOSING THE PROBLEM
Review cpit_install_stdout.txt and SystemOut.log for the aforementioned exceptions and log entries



RESOLVING THE PROBLEM
1. Increase the maximum and minimum JVM heap size. 

 - Initial heap size: 512 MB 

 -Maximum heap size: 1024 MB 

2. Delete decompressed software folder to ensure there is enough space to re-run CPIT installation 

 - Go to C:\Program Files (x86)\IBM\cpit\decompressed and delete TDS, CE, WAS, WPXT folders 

 - Clear the recycling bin 

3. Replace the cpit.properties file 

 - Remove the cpit.properties file from the cpit folder where you are running the 5.2.0-CPIT-WIN.exe and replace it with the cpit.properties from C:\Program Files (x86)\IBM\cpit\install-scripts. This will allow the install to start at step 23. 

4. Run 5.2.0-CPIT-WIN
