Title: IBM Manually changing the TEMA hostname in ITCAM for WebSphere Data Collector. - United States

Text:
ITCAMforWebSphere TEMA kwjdc.properties hostname change TECHNOTE (TROUBLESHOOTING)

THIS DOCUMENT APPLIES ONLY TO THE FOLLOWING LANGUAGE VERSION(S):
 US English 

PROBLEM(ABSTRACT)
 If you need to manually change the hostname or IP address for the TEMA connection in the Data Collector. 

SYMPTOM
Hostname or IP address for the ITCAM for Websphere TEMA changes and now the Data Collector is not communicating with TEMA. No data is seen in the TEP for ITCAM for WebSphere. 


CAUSE
Hostname or IP address for the ITCAM for Websphere TEMA changes and now the Data Collector is not communicating with TEMA.

RESOLVING THE PROBLEM
Edit /usr/lpp/itcam/WebSphere/runtime/server/server/kwjdc.properties


Change: 

# Host name or IP address where the TEMA is located
com.ibm.tivoli.kwj.agenthostname= 

to your new name.

Save and restart the DC and TEMA.

 

PRODUCT ALIAS/SYNONYM
 ITCAM for Websphere
