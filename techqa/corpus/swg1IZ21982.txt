APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Current the level of the log message is not recorded in the log
   file. This will allow us to monitor the log file for important
   events while still outputting the more trivial ones in case they
   are needed for problem resolution.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  There is no facility for getting the log level printed out with
   a log entry.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Added the facility for causing the agent to output the log
   level. To do this you need to alter the LogPrefix inivar in
   init.cfg (while the agent is off) using the "!L" characters
   i.e.:
   
   LogPrefix=%Y-%m-%d!s%H:%M:%S!s(!L)!s
   
   The fix for this APAR is contained in the following maintenance
   packages:
      | fix pack | 4.0.0-TIV-SSM-FP0004
