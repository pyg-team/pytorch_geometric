Title: IBM Unable to access Metric Package after migrate Metric Store to Oracle 11g from 10g - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Unable to open metric package after repointing content store and metric store from an oracle 10g database to 11g. 

SYMPTOM
Receive error: 

'java.lang.NullPointerException' 
WHILE updating user DB 
WHILE An error occurred while retrieving user information. Check your IBM Cognos 8 configuration settings. 
WHILE checking DB session 
WHILE initializing CMM DB Session 
WHILE processing request for pid 'null' 
WHILE Executing page 'null', from pageMap 'Unknown, since pid 'null' 
doesn't exist in any of the page maps', via request:


RESOLVING THE PROBLEM
1. Open Cognos Administration--Configuration; 

 2. Delete the existing data source for the package; 

 3. Create a new data source with the same data source name; 

 Using the new data source you will be able to open the metric package.
