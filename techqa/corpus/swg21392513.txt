Title: IBM Enabling detailed logging on WebSphere Portal Version 6 for OmniFind Enterprise Edition - United States

Text:
OEE; WPS; debug TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 To troubleshoot issues related to OmniFind Enterprise Edition, how can I enable detailed logging on WebSphere Portal Version 6? 

SYMPTOM
Detailed logging can help troubleshoot the following issues: 

 * Users do not get the expected search results when the option to validate current credentials is enabled. 
 * Errors occur during search, especially when using the Search portlet in WebSphere Portal (ESSearchPortlet).


IBM Software Support might ask you to enable detailed logging or tracing for other issues so that the problem can be fully investigated. 

RESOLVING THE PROBLEM
Use the following procedures to enable or disable detailed logging on the WebSphere Portal, Version 6 server that you use with OmniFind Enterprise Edition. 

Enabling detailed logging on the WebSphere Portal server Version 6.0


 1. Log in to the WebSphere Application Server administrative console on the WebSphere Portal server. 
 2. Click Portlet Management in the navigation area to the left, and then click Portlets. 
 3. Change the Search by option to Title contains. 
 4. In the Search field, type enterprise search and then click the Search button. 
 5. Modify logging.level parameter to FINEST (or suggested level). 
 6. Save the change and restart the Portal server.



Disabling detailed logging on the WebSphere Portal server Version 6.0 
 1. Log in to the WebSphere Application Server administrative console on the WebSphere Portal server. 
 2. Click Portlet Management in the navigation area to the left, and then click Portlets. 
 3. Change the Search by option to Title contains. 
 4. In the Search field, type enterprise search and then click the Search button. 
 5. Modify logging.level parameter to SEVERE. 
 6. Save the change and restart the Portal server.


Enabling detailed logging on the WebSphere Portal server Version 6.1


 *  Log in to the WebSphere Application Server administrative console on the WebSphere Portal server. Open the Troubleshooting panel in the left pane, then click Logs and Trace. Click the WebSphere_Portal server, click Change Log Detail Levels, and then click the Runtime tab. Add the following description to a text area to define the logging level:
   
   com.ibm.es.*=finer 

 * Tip:
 * 


Disabling detailed logging on the WebSphere Portal server Version 6.1 
 *  Log in to the WebSphere Application Server administrative console on the WebSphere Portal server. Open the Troubleshooting panel in the left pane, then click Logs and Trace. In the right pane, click Change Log Detail Levels, then click the Runtime tab. Remove the defined logging level entry (com.ibm.es.*) from the text area.
   

For more information about logging and tracing in WebSphere Portal, see the Trace Logging topic in the WebSphere Portal Information Center.
