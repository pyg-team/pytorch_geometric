Title: IBM Subscription Framework generates exceptions that can be ignored - United States

Text:
 RELEASE NOTES

ABSTRACT
 The Subscription Framework uses a WebSphere work manager to run subscriber code. The WebSphere work manager occasionally logs error messages that a subscriber has not ended. You can ignore these error messages.


CONTENT
If a subscriber is running, you might see an exception similar to the following example in the WebSphere logs. You can ignore this exception message.

[1/30/15 9:42:29:662 PST] 0000007a ThreadMonitor W WSVR0605W: Thread "WorkManager.SubscribeWorkManager : 1" (0000007c) has been active for 705453 milliseconds and may be hung. There is/are 1 thread(s) in total in the server that may be hung.
at java.lang.Object.wait(Native Method)
...

You might also see an error message similar to the following example if a subscriber was running and WebSphere is shut down. You can ignore this error message.

[3/12/15 15:17:08:951 EDT] 00000165 ThreadMonitor W WSVR0606W: Thread "WorkManager.SubscribeWorkManager : 1" (00000165) was previously reported to be hung but has completed. It was active for approximately 700284 milliseconds. There is/are 0 thread(s) in total in the server that still may be hung.
