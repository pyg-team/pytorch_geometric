Title: IBM Sametime Install error: A package has an issue that cannot be resolved by the Installation Manager. - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM
When attempting to install Sametime Meeting Server, the installation fails with errors:
"CRIMA1217E: A problem occurred during the execution of the /opt/IBM/WebSphere/STMServerPN/build_meeting.xml file.
Explanation: A package has an issue that cannot be resolved by the Installation Manager.
User Action: Identify the package that has the issue by looking at the installation history in Installation Manager.
Error during "install" phase:
A problem occurred during the execution of the /opt/IBM/WebSphere/STMServerPN/build_meeting.xml file.
exec returned: 105 [/opt/IBM/WebSphere/STMServerPN/build_meeting.xml:1866] 

Note: the path in the error may be different if the server is on Windows. 

The meeting server fails to install.


SYMPTOM
There may be a partial WebSphere Application Server profile in the WebSphere profiles directory. 

(/opt/IBM/WebSphere/AppServer/profiles/<profilename>) 

In the partial profile directory, the logs indicate DB2 errors.


CAUSE
The db2 password has special characters.


ENVIRONMENT
Sametime 9.0.x 

Linux or Windows


RESOLVING THE PROBLEM
Perform the following steps to resolve the issue: 

 1. Change the db2 password to one with no special characters (only alphanumeric are allowed) [https://www.ibm.com/support/knowledgecenter/SSKTXQ_9.0.1/admin/install/inst_db2_server_winux.html]. 
 2. Remove any partial profile directory that may have been left behind from the failed installation attempt. (For example: /opt/IBM/WebSphere/AppServer/profiles/<profilename> 
 3. Update the db2 password in the Sametime System Console [https://www.ibm.com/support/knowledgecenter/SSKTXQ_9.0.1/admin/admin/admin_pw_db2.html] for each data source that is already defined. 
 4. Update your Connect to DB2 database prerequisite activities:  1. Prerequisite Connect to DB2 Database for Sametime Meeting Server [https://www.ibm.com/support/knowledgecenter/SSKTXQ_9.0.1/admin/install/inst_config_ssc_db2_ga_meet.html] 
     2. Prerequisite: Connect to DB2 Databases for Sametime Proxy Server [https://www.ibm.com/support/knowledgecenter/SSKTXQ_9.0.1/admin/install/inst_config_ssc_db2_ga.html] 
     3. Prerequisite: Connecting to a DB2 database for Sametime Advanced [https://www.ibm.com/support/knowledgecenter/SSKTXQ_9.0.1/admin/install/stv_inst_config_ssc_db2_ga.html]
    
    
 5. Save to the master configuration, synchronize nodes and restart all servers. 
 6. 

Once completed, re-run the Sametime Install that failed. RELATED INFORMATION
 How to change DB2 user password? [http://www-01.ibm.com/support/docview.wss?uid=swg21984232]
