PROBLEM SUMMARY
 *  If you have many entries in the LSW_PO_VERSIONS table of the IBM
    Process Server database, a value that is too small for the Java
    maximum heap size is set when the BPMUpdateSystemApp admin
   command runs to update the IBM BPM toolkits and process
   applications.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix will be incorporated into an IBM BPM V8.5.6.0 cumulative
   fix that increases the Java maximum heap size for running the
   BPMUpdateSystemApp admin command.
   
   NOTE: This fix might not work all your DEs. A fix that applies
   to all DEs will be available later.
