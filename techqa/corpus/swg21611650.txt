Title: IBM Could not find factory error after redeploying FileNet WorkplaceXT - United States

Text:
jsf-api-1.1_02.jar; jsf-impl-1.1_02.jar; faces; redeploying; could not find Factory; error code 500 TECHNOTE (TROUBLESHOOTING)

CAUSE
Missing files in the WEB-INF\lib install directory of WorkplaceXT which caused missing files in the newly created web_client.war or web_client.ear files.

DIAGNOSING THE PROBLEM
Compare the contents of the last known good web_client.war to the web_client.war that is throwing the error when deployed to <WorkplaceXT_Install_Dir>\WEB-INF.

RESOLVING THE PROBLEM
Replace the jsf*.jar files that are missing from the <WorkplaceXT_Install_Dir>WEB-INF\lib. They can be replaced by one of the following options, or WorkplaceXT can be reinstalled:


 *  Copy the missing files from a known good web_client.war. Copy the missing files from a different environment (such as TEST or DEV) as long as the WorkplaceXT versions are exactly the same. Restore the missing files from a file level backup. Uninstall and reinstall WorkplaceXT.
