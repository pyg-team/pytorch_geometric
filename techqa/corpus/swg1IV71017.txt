Title: IBM IV71017: MISSING INFO IN READMES FOR APS PREREQUISITE LIBRARY. - United States

Text:

APAR STATUS
 * CLOSED AS DOCUMENTATION ERROR.
    
   
   

ERROR DESCRIPTION
 *  During Authorization Policy Server installation through IBM
   Installation Manager an error occurs. Looking into Installation
   History, the gtk2-devel library is missing.
   Documentation states gtk is required on Linux but does not state
   gtk2-devel.
   
   
   AFFECTED PLATFORMS / VERSIONS:
   This issue affects Linux amd64 and similar platforms,  IBM
   Tivoli Monitoring Dashboards for Servers and Authorization
   Policy V6.3.0, V6.3.0 FP1 and V6.3.0 FP2.
   
   
   DIAGNOSTICS:
   IBM Installation Managershows a pop up window containing
   the following message:
   
   "CRIMA1217E: A problem occurred during the execution of the
   /opt/IBM/JAZZSM/AuthPolicyServer/PolicyServer/bin/com.ibm.tivoli
   .granular.deploy.xml file.
   
   Explanation: A package has an issue that cannot be resolved by
   Installation Manager.
   
   User Action: Identify the package that has the issue by looking
   at the installation history.
   In Installation Manager, click File > Installation History. In
   console mode, enter S: View InstallationHistory. Contact IBM
   support."
   
   Initial Impact:
   Policy Server installation can not be completed.
   
   
    
   
   

LOCAL FIX
 *  Install gtk2-devel library and repeat Authorization Policy
   Server Installation.
   
   
    
   
   

PROBLEM SUMMARY
 *  IBM Tivoli Monitoring Version 6.3 Installation guide states gtk
   is required on Linux it but does not state gtk2-devel.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A tech note has been created to address the incorrect
   information; it is reachable at the following link:
   http://www-01.ibm.com/support/docview.wss?uid=swg21699958 [http://www-01.ibm.com/support/docview.wss?uid=swg21699958]
