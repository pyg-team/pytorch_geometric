Title: IBM IY99104: SNMPTRAPACTION FUNCTION DOES NOT WORK WITH SNMPTRAPOID REPLACED BY NULL - United States

ERROR DESCRIPTION
 *  According to the Impact 4.0 DSA Guide it is possible to replace
   the SNMPTrapOID placeholder with a hard-coded NULL value when
   calling SNMPTrapAction.
   
   i.e. SnmpTrapAction(HostId, Port, VarIdList, ValueList,
   Community, Version, SysUpTime, Enterprise, GenericTrap,
   SpecificTrap, NULL);
   
   Currently Impact is not allowing this.  Error:
   
   com.micromuse.common.util.Null incompatible with
   java.lang.String
   
   
    
   
   

LOCAL FIX
 *  Use a valid value for SNMPTrapOID.

   *                      WITH SNMPTRAPOID REPLACED BY NULL FOR   *
   *                      SNMP VERSION 1                          *
   ****************************************************************
   * RECOMMENDATION:                                              *
   *                                                              *
   ****************************************************************
   Steps To Replicate: 1) Create SNMP based policy using
   SnmpTrapAction, or use the existing template policy. 2) Set
   Version variable =1 (SNMP verrsion) 3) Comment SnmpTrapOID or
   set it to NULL 4) Run the policy and observe the following
   error: com.micromuse.response.common.PolicyProcessingException:
   PolicyProcessingException in policy SNMPTrapActionTest.
   Throwable: java.lang.RuntimeException: Exception in policy:
   SNMPTrapActionTest at line 68. The 'SnmpTrapOid' value cannot
   be null
