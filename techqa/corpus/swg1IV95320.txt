ERROR DESCRIPTION
 *  Product: Tivoli Network Manager
   Version: 3.8
   Component:  ncp_poller
   
   Symptoms:
   After failover, the Poller should check the
   ncpoller.active_poll_events table for initial event state to
   prevent duplicate events being sent.
   
   
    
   
   

LOCAL FIX
 *  restart the backup poller after failback so that the event
   status is updated while the Primary is acting primary, then the
   when the Backup becomes acting primary the status will be
   correct.
