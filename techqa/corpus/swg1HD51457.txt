ERROR DESCRIPTION
 *  CASE SENSITIVE/INSENSITIVE BY DEFAULT IN
   SEARCH
   In Enovia Documentation following information is
   given for the search -
   When Performing an Advanced Search ,You can
   check how large the search will be before you
   actually run the search. Click the button located on
   the vertical toolbar to find out how many items
   currently saved to the database match your search
   criteria. You can also regulate the performance of
   Advanced Search by modifying the default settings.
   The default settings are available in the Settings
   Search window. You can access the Settings
   Search window by clicking the Properties button on
   the horizontal toolbar.
   The Case Sensitive check box is used to control
   your query results. This enables you to reduce the
   number of items displayed in search results. In
   order to change the default behavior of the Case
   Sensitive check box, the properties file must be
   modified before you log onto ENOVIA V5.
   When the property is set to FALSE, the Case
   Sensitive check box is checked by default and your
   query will be case sensitive.
   When the property is set to TRUE, the Case
   Sensitive check box is not checked by default and
   your query will not be case sensitive.
   Problem- It is not mentioned in Doc, by which
   properties file change,the default behaviour of the
   case sensitive check box is change.
