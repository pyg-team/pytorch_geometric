Title: IBM How to collect a "server tooling plugin" trace with Rational Application Developer 7.0 or Application Server Toolkit 6.1 - United States

Text:
trace; runtime; run time; plug-in; plugin; plug in TECHNOTE (FAQ)

QUESTION
 How do you collect a "server tooling plugin" trace with IBM Rational Application Developer for WebSphere Software (RAD) 7.0 or the IBM Application Server Toolkit for WebSphere Application Server (Toolkit) 6.1? 

CAUSE
In IBM Application Server Toolkit for WebSphere Application Server (Toolkit) 6.1 and IBM Rational Application Developer for WebSphere Software (RAD) 7.0, the component which provides the ability to create and work with a WebSphere Application Server is called the "Server Tools". 

This includes the ability to create, delete, start, and stop a server, as well as the ability to deploy, run, and debug applications on a local or remote Application Server runtime. When there is a problem with one of these features, it may be helpful to collect a Server Tools Plug-in Trace for review by IBM support.

The trace settings in this document are specifically for Application Server Toolkit V6.1 or Rational Application Developer 7.0 when using the Application Server 6.1 runtime, or Application Server 6.0 runtime, or Application Server 5.1 runtime. 

If you are using Rational Application Developer 6.0 with either the 6.0 or 5.0 Application Server runtime, review the following technote:


 * How to collect a "server tooling plugin" trace with Rational Application Developer 6.0 [http://www.ibm.com/support/docview.wss?uid=swg21293372]


If you are using Rational Application Developer 7.5 or Rational Application Developer Assembly and Deploy, review the following technote: 
 * How to trace Rational Application Developer for WebSphere Software [http://www.ibm.com/support/docview.wss?uid=swg21327307]

ANSWER
To capture a WebSphere Application Server 6.1 Server Tools Plug-in Trace do the following: 

 

 1. Exit Rational Application Developer 7.0 or Application Server Toolkit 6.1.
    
    
 2. Create a text file ".options" with the trace strings for the Application Server runtime you are using:
    
    If you are using the Application Server V6.1 runtime with Rational Application Developer V7.0, the .options file must contain:
    
    com.ibm.ws.ast.st.v61.core/debug=true
    com.ibm.ws.ast.st.v61.core/debug/error=true
    com.ibm.ws.ast.st.v61.core/debug/warning=true
    com.ibm.ws.ast.st.v61.core/debug/info=true
    com.ibm.ws.ast.st.v61.core/debug/details=false
    com.ibm.ws.ast.st.common.core/debug=true
    com.ibm.ws.ast.st.common.core/debug/info=true
    com.ibm.ws.ast.st.core/debug=true
    com.ibm.ws.ast.st.core/debug/info=true
    org.eclipse.wst.server.core/debug=true
    org.eclipse.wst.server.ui/debug=true
    com.ibm.ws.ast.st.v61.ui/debug=true
    com.ibm.ws.ast.st.v61.ui/debug/error=true
    com.ibm.ws.ast.st.v61.ui/debug/warning=true
    com.ibm.ws.ast.st.v61.ui/debug/info=true
    com.ibm.ws.ast.st.v61.ui/debug/details=false
    com.ibm.etools.wrd.websphere/debug=true
    
    If you are using the Application Server 6.0 runtime with Rational Application Developer 7.0, the .options file must contain:
    
    com.ibm.ws.ast.st.v6.core/debug=true
    com.ibm.ws.ast.st.v6.core/debug/error=true
    com.ibm.ws.ast.st.v6.core/debug/warning=true
    com.ibm.ws.ast.st.v6.core/debug/info=true
    com.ibm.ws.ast.st.v6.core/debug/details=false
    com.ibm.ws.ast.st.common.core/debug=true
    com.ibm.ws.ast.st.common.core/debug/info=true
    com.ibm.ws.ast.st.core/debug=true
    com.ibm.ws.ast.st.core/debug/info=true
    org.eclipse.wst.server.core/debug=true
    org.eclipse.wst.server.ui/debug=true
    com.ibm.ws.ast.st.v6.ui/debug=true
    com.ibm.ws.ast.st.v6.ui/debug/error=true
    com.ibm.ws.ast.st.v6.ui/debug/warning=true
    com.ibm.ws.ast.st.v6.ui/debug/info=true
    com.ibm.ws.ast.st.v6.ui/debug/details=false
    com.ibm.etools.wrd.websphere.v6/debug=true
    
    If you are using the Application Server V5.1 runtime with Rational Application Developer V7.0, the .options file must contain:
    
    com.ibm.etools.websphere.tools.v51/debug=false
    com.ibm.etools.websphere.tools.v51/debug/error=true
    com.ibm.etools.websphere.tools.v51/debug/warning=true
    com.ibm.etools.websphere.tools.v51/debug/info=false
    com.ibm.etools.websphere.tools.v51/debug/details=false
    com.ibm.ws.ast.st.common.core/debug=true
    com.ibm.ws.ast.st.common.core/debug/info=true
    com.ibm.ws.ast.st.core/debug=true
    com.ibm.ws.ast.st.core/debug/info=true
    org.eclipse.wst.server.core/debug=true
    org.eclipse.wst.server.ui/debug=true
    
    
 3. Put the .options file into the same directory where the eclipse.exe file is located.
    
    For Rational Application Developer 7.0, the default location is:
    C:\Program Files\IBM\SDP70
    
    For Application Server Toolkit 6.1, the default location is:
    C:\Program Files\IBM\WebSphere\AST\eclipse\
    
    
 4. Determine the command line arguments that should be used.
    
    To determine the command line arguments for Rational Application Developer 7.0, look inside the eclipse.ini file, and make note of all of the arguments that come after the line that says -vmargs. 
    
    For example, the default arguments for Rational Application Developer 7.0 are:
    
    -Xquickstart
    -Xms40m
    -Xmx768m
    -Xgcpolicy:gencon
    -Xscmx96m
    -Xshareclasses:singleJVM,keep
    -Xnolinenumbers
    -XX:MaxPermSize=512M
    
    To determine the command line arguments for Application Server Toolkit 6.1, look inside the ast.ini file, and make note of all of the arguments on the lines that begin with VMArgs=. 
    
    For example, the default arguments for Application Server Toolkit V6.1 are:
    
    -Dsecurity.overrideProperties=true 
    -Djava.security.properties="C:\Program Files\IBM\WebSphere\AST\runtimes\base_v61_stub\properties\java.security"
    -Dcom.ibm.CORBA.ConfigURL="file:C:\Program Files\IBM\WebSphere\AST\runtimes\base_v61_stub\properties\sas.client.props"
    -Dcom.ibm.SSL.ConfigURL="file:C:\ProgramFiles\IBM\WebSphere\AST\runtimes\base_v61_stub\properties\ssl.client.props"
    
    
 5. Start Rational Application Developer 7.0 or Application Server Toolkit 6.1 using a java.exe command with the -debug option and the other arguments from step 4.
    
    To start Rational Application Developer 7.0, go to the directory where the eclipse.exe file is located (C:\Program Files\IBM\SDP70), and use a Java command as follows:
    
    .\jdk\bin\java.exe -Xj9 -cp startup.jar -Xquickstart -Xms40m -Xmx768m -Xgcpolicy:gencon -Xscmx96m -Xshareclasses:singleJVM,keep -Xnolinenumbers -XX:MaxPermSize=512M org.eclipse.core.launcher.Main -product com.ibm.rational.rad.product.ide -debug .options > trace.out 2>&1
    
    Note: in the above command, the -product option needs to match the actual product, for example:
    
    - for Rational Application Developer 7.0 use:
    -product com.ibm.rational.rad.product.ide
    
    - for Rational Software Architect 7.0 use:
    -product com.ibm.rational.rsa.product.ide
    
    - for WebSphere Development Studio Client 7.0 use:
    -product com.ibm.etools.iseries.wdsc.welcome.ide
    
    - for WebSphere Integration Developer 6.1 use:
    -product com.ibm.wbit.feature.ide
    
    
    To start Application Server Toolkit 6.1, go to the directory where the eclipse.exe file is located ( C:\Program Files\IBM\WebSphere\AST\eclipse), and use a Java command as follows:
    
    .\jre\bin\java.exe -Xj9 -cp startup.jar -Dsecurity.overrideProperties=true -Djava.security.properties="C:\Program Files\IBM\WebSphere\AST\runtimes\base_v61_stub\properties\java.security" -Dcom.ibm.CORBA.ConfigURL="file:C:\Program Files\IBM\WebSphere\AST\runtimes\base_v61_stub\properties\sas.client.props" -Dcom.ibm.SSL.ConfigURL="file:C:\Program Files\IBM\WebSphere\AST\runtimes\base_v61_stub\properties\ssl.client.props" org.eclipse.core.launcher.Main -debug .options > trace.out 2>&1
    
    
 6. With the trace active, recreate the problem.
    
    The trace output will be written to a file called trace.out in the same directory.
