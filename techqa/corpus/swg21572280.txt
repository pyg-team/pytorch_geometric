Title: IBM UseSsl and the Alcatel-Lucent 5529 OAD probe - United States

Text:
nco_p_alcatel_5529_oad_v6; UseSsl; SSL; ssl; keystore; jms; alcatel-oad TECHNOTE (FAQ)

QUESTION
 Alcatel-Lucent 5529 OAD probe: How to configure UseSsl. 

CAUSE
Alcatel-Lucent 5529 OAD client certificate creation, export, and import into a probe specific Java KeyStore needs to be performed

ANSWER
 

To enable an SSL connection set the UseSsl property to 'true'. 

A Java keystore file is required for the HTTPS connection, access to which is specified using the CertificateStore and CertificateStorePassword properties. Set the TrustedStore and TrustedStorePassword properties to the same values. Then extract the root CA and provider certificate from the ems servers Java keystore, to create the clients Java keystore.

Example Alcatel EMS provided client certificate import;
Make sure the probes java is in the users path:
e.g. for 64-bit java probes in Netcool/OMNIbus 8.1
set path=( $NCHOME/platform/linux2x86/jre64_1.7.0/jre/bin $path)
rehash
java -version

Then create the store file:

keytool -genkey -alias sslclientcert -keyalg RSA -keystore ./alcateload.jks -keysize 2048
keytool -import -trustcacerts -alias sslclientcert -file ./alcatelclient.cert -keystore ./alcateload.jks 
keytool -list -v -keystore ./alcateload.jks

Import the required root CA and Issuing CA certificates in to the probes Java keystore [alcateload.jks] to ensure the certificate is trusted.

It is best to keep the Java KeyStore files in a unique directory in the probes directory;
e.g.
/opt/IBM/tivoli/netcool/omnibus/probes/java/keystore
If many probes are using SSL then create a probe specific directory;
e.g.
/opt/IBM/tivoli/netcool/omnibus/probes/java/keystore/nco_p_alcatel_5529_oad_v6

Example SSL property settings;
HttpHost : "198.162.20.21"
HttpPort : 8443 
UseSsl : "true" 
TrustStore : "/opt/IBM/tivoli/netcool/omnibus/probes/java/keystore/alcateload.jks" 
TrustStorePassword : "ECEDBJAGBJFHGD"
CertificateStore : "/opt/IBM/tivoli/netcool/omnibus/probes/java/keystore/alcateload.jks" 
CertificateStorePassword : "ECEDBJAGBJFHGD"

Please consult with the Alcatel EMS administrator or Alcatel support for JMS server certificate creation and ensure the client certificate is created and stored correctly.


 * Ensure a valid country code is used 
 * Ensure that the probes keystore password is set
