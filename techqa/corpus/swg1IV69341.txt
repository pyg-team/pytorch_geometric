APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  ERROR DESCRIPTION:
   Windows 2012 Server R2 does not appear on the SCA installation
   console for install. The installer for Windows 2012 Server R2
   has a failure. when attempting to install it manually.
   
   
    
   
   

LOCAL FIX
 *  The following is the procedure for getting around the Windows
   2012 R2 install bug for TEMA SCA.
   
   steps:
   1. extract TEMA SCA installation package
   
   2. browser into extracted package folder under
   ..\server\parts\CDROM_Installers\Disk1\InstData\Win32\VM
   3. right mouse click on tema.exe file to show context menu
   4. in the context menu, select "Troubleshoot compatibility"
   
   5. Select option "Try Recommanded settings"
   
   6. Click on "Test the program"
   
   
   7. Installer shall proceed without bugging out with error.
   
   
    
   
   

PROBLEM SUMMARY
 *  Problem Description:
   Unable to install SCA 1.5 on Windows 2012 R2
   
   Problem Summary:
   Windows 2012 R2 was not listed in the SCA dashboard and
   relevance; users were unable to install SCA 1.5.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This issue was resolved in SCM Reporting QA v.230.
