Title: IBM Unable to connect with JMS adapter (SCI39456) - United States

Text:
STERLINGPRI TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Unable to connect with JMS adapter (SCI39456) 

SYMPTOM
JMS Queue Adapter is unable to connect to a remote JMS Queue 
Error: Cannot instantiate class: weblogic.jndi.WLInitialContextFact
ory.Root exception is java.lang.ClassNotFoundException: weblogic.jndi.WLInitialContextFactory


CAUSE
The connection factory weblogic.jndi.WLInitialContextFactory is not available 
within JBOSS.

RESOLVING THE PROBLEM
Import the weblogic.jar file which contains the weblogic.jndi.WLInitialContextFactory.class into JBoss. 

To import the weblogic.jar file into Sterling Integrator running JBoss

1. Shut down SI with hardstop.sh
2. Copy the weblogic.jar file onto the Server running JBoss. The jar is located under the top level lib directory of the WebLogic install.
3. run bin/install3rdParty.sh to import the weblogic.jar
./install3rdParty.sh WebLogic 6_1 -j /path/to/weblogic.jar
4. Restart SI with run.sh
