Title: IBM PM40488: WTX CRASH OR OUT OF MEMORY ERROR OCCURS DURING XML XERCES VALIDATION OF INVALID DATA WHEN USING COMPONENT RULES - United States

 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  A WTX crash or out of memory error occurs during XML Xerces
   validation of invalid data when using component rules.
   
   On Windows the map ends with a 'Not enough memory' error.
   
   On Unix the map ends with a Bus Error(coredump).
   
   
    
   
   

LOCAL FIX
 *  WTXCQ - WTX00037708
   GK / PB
   Circumvention: None
   
   
    
   
   

PROBLEM SUMMARY
 *  Users Affected:
   WebSphere Transformation Extender customers using Xerces
   validation
   
   Problem Description:
   After XML validation fails, code executes a component rule for
   an element that is not built causing read from non-initialized
   memory address. Application may crash or fail with out of
   memory error.
   
   Platforms Affected:
   ALL
   
   
    
   
   

PROBLEM CONCLUSION
 *  If XML validation fails, validation component rules should not
   be executed.
   
   Delivered in:
   8.2.0.6, 8.3.0.5
