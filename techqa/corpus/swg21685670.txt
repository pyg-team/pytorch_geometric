Title: IBM Data Collector not connecting to MS - United States

Text:
CAMADALL DC not connectiong to MS codebase.http.header.enabled=true codebase TECHNOTE (TROUBLESHOOTING)

THIS DOCUMENT APPLIES ONLY TO THE FOLLOWING LANGUAGE VERSION(S):
 US English 

PROBLEM(ABSTRACT)
 ITCAM Data Collector not connecting to the Managing Server 

SYMPTOM
From the WebSphere SystemErr.log you will see these messages:

SystemErr R Check if IP address of Kernal in property file is correct OR if kernal is up & running 
SystemErr R Unable to load classes from Kernel com.cyanea.kernel.common.KernelException: java.lang.ClassNotFoundException: com.cyanea.kernel.util.JoinManager
[9/28/14 9:22:56:826 CDT] 000000ff SystemErr R Sun Sep 28 09:22:56 CDT 2014: kernel props used :


CAUSE
The WebSphere where the ITCAM Data Collector is running will be using Java version = 1.7.0, Java Compiler = j9jit26, Java VM name = IBM J9 VM

ENVIRONMENT
Any

DIAGNOSING THE PROBLEM
Review the WebSphere SystemOut.log for Java version = 1.7.0, Java Compiler = j9jit26, Java VM name = IBM J9 VM

RESOLVING THE PROBLEM
Change the value of the following property to "true" in
<MS_INSTALL>/etc/kl1.properties and <MS_INSTALL>/etc/kl2.properties:

codebase.http.header.enabled=true

Restart the Managing server.
Restart the WebSphere application server JVMs
