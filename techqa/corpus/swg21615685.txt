Title: IBM IIB or WMB Toolkit connection failure - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The IBM Integration Bus (IIB) or WebSphere Message Broker (WMB) Toolkit fails to connect to a remote broker, with error MQRC 2059 

SYMPTOM
You have correctly configured your WebSphere Message Broker Toolkit to connect to a remote broker, by using the configuration steps [http://www.ibm.com/support/knowledgecenter/SSKM8N_8.0.0/com.ibm.etools.mft.doc/be10200_.htm] from the product documentation. 

Your queue manager has been given the correct authorisation, but attempting to connect to the remote broker by using the WebSphere Message Broker Toolkit results in error MQRC 2059.
You can connect to your remote broker from the same machine without error by using the WebSphere Message Broker Explorer, or by completing the following steps:
1. Set the following environment variable: 

 * 
 * set MQSERVER=ChannelName/TCP/server-address(port) 
 * 
 * export MQSERVER=ChannelName/TCP/'server-address(port)' 

2. Open a WebSphere Message Broker command prompt, and enter the following test command:  * 


CAUSE
The network configuration of your IIB or WMB Toolkit installation is incorrect.

RESOLVING THE PROBLEM
In the IIB or WMB toolkit click Window -> Preferences -> General -> Network Connection. The Preferences window opens, and the Network Connections preferences are displayed.
