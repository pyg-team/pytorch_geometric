ERROR DESCRIPTION
 *  When using the administrative console to tune the session
   timeout , for example to set 120s:
   Enterprise Applications > teamserver-WAS85 > Session management
   > Session Timeout > Set timeout > 120
   Then save, stop  teamserver-WAS85, start  teamserver-WAS85, the
   application does not start, it is corrupted.
   
   
    
   
   

LOCAL FIX
 *  recreate sample server
   
   
    
   
   

PROBLEM SUMMARY
 *  Changing session timeout in WAS will stop it and start a
   duplicated applicationConfig created in the deployment
   settings, then the solution is to remove the code that is
   creating this unused/malformed section.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code is fixed
