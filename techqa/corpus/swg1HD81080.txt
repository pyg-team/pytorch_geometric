ERROR DESCRIPTION
 *  ABSTRACT:
   DMUSPA:EXPORTED XML FILE IN JAPANESE
   ENVIRONMENT NOT IN JAPANESE NOTATION
   SCENARIO:
   1. Start CATIAV5. Go to
   Tools\Customize\Click "Options" tab
   1-2. Select "Environment language (default)". In
   this case, on Japanese Windows
   2. Restart CATIAV5
   2-1. Open "12-2.CATProduct"
   2-2. Change to DMU Space Analysis W/B
   3. Click "Distance and Band Analysis" Icon
   4. Edit Distance and Band Analysis dialog appears
   4-1. Select "Minimum" and "Between two
   selections"
   4-2. Select each product as Selection
   4-3. Click "Apply" Button
   5. Results and Preview appear
   6. Click "Export As" Icon. Save with xml type. 6
   files are created
   7. Open created xml file
   RESULT:
   The exported xml files on Japanese are not in
   Japanese notation
   EXPECTED RESULT:
   The exported xml files on the Japanese
   environment should be in Japanese notation.
   .

PROBLEM CONCLUSION
 *  THIS PROBLEM IS PERMANENT RESTRICTION IN
   CATIA
   Incident Diagnosis:
   DMUSPA:EXPORTED XML FILE ON JAPANESE
   ENVIRONMENT ARE NOT IN JAPANESE NOTATION
   Restriction Explanation:
   No japanese style sheet available for xml export of
   Distance results.
   By-Pass:
   Use the Distance_JA.xsl file in the DMU Distance
   tab in Tools->Options->Digital Mockup->DMU
   Space Analysis to get the japanese output.
   Distance_JA.xsl file is available in R19SP05 &
   higher level.
