Title: IBM Logging onto FileNet Content Manager Application Engine (AE) causes the system to hang - United States

Text:
Application Engine; JVM TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Logging onto the Application Engine's Workplace causes the system to hang due to not being configured correctly in WAS application server Generic JVM arguments 

SYMPTOM
The following error occurs in the SystemOut.log from IBM ECM Websphere Enterprise 7.1 (WAS) 

[10/10/12 08:15:44:402 IST] 00000055 SystemOut O Failed to get primary port
[10/10/12 08:18:20:687 IST] 0000001f ThreadMonitor W WSVR0605W: Thread "WebContainer : 5" (00000044) has been active for 689949 milliseconds and may be hung. There is/are 2 thread(s) in total in the server that may be hung.


RESOLVING THE PROBLEM
 

 1. Login to WAS application server 
 2. Goto Application servers > server1 > Process definition > Java Virtual Machine 
 3. Under Generic JVM arguments field added the following for AE


-Djava.security.auth.login.config= C:\PROGRA~1 \FileNet\AE\CE_API\config\ jaas.conf.WebSphere 

Note: This is the default location for AE to be installed on Windows 64 bit server. Make sure you enter in the correct location for this file.
