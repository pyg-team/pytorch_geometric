Title: IBM Application Error in IBM Content Collector Configuration Manager when saving a task route - United States

Text:
icc; fips; md5 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 An application error is reported in the IBM Content Collector (ICC) configuration manager when you try to save a task route or other settings. 

SYMPTOM
The pop up window shows the following message: 

 * 
 * Error: 
 * 
   A severe error occurred.
   
   
 * Problem:
 * 
   Exception has been thrown by the target of an invocation.
   
   
 * Solution:
 * 
   Contact IBM Software Support with this error message (see Details).


Click on the Details button and the following stack trace is reported:  

 * Details
 * System.Reflection.TargetInvocationException: Exception has been thrown by the target of an invocation.
   at System.RuntimeMethodHandle.InvokeMethod(Object target, Object[] arguments, Signature sig, Boolean constructor)
   at System.Reflection.RuntimeConstructorInfo.Invoke(BindingFlags invokeAttr, Binder binder, Object[] parameters, CultureInfo culture)
   .....
   .....
   System.InvalidOperationException: This implementation is not part of the Windows Platform FIPS validated cryptographic algorithms.
   at System.Security.Cryptography.MD5CryptoServiceProvider..ctor()


CAUSE
Incompatible Windows security policy.

DIAGNOSING THE PROBLEM
Scroll down to the bottom of the detail section and verify that the following error is seen: 

 * 
 * 
 * 
 * 
 * 
 * System.InvalidOperationException: This implementation is not part of the Windows Platform FIPS validated cryptographic algorithms.
   at System.Security.Cryptography.MD5CryptoServiceProvider..ctor()

RESOLVING THE PROBLEM
To resolve the problem, go to the following: 

 * Navigate to Control Panel > Administrative Tools > Local Security Policy > Security Settings > Local Policy > Security Options. 
 * In security option, select and double click on System Cryptography: Use FIPS compliant algorithms for encryption, hashing, and signing 
 * Check if the option is enabled. If it is enabled, disable this option and try to save the changes in ICC configuration manager again.


If problem persists after disabling the option, contact IBM support for further assistance.
