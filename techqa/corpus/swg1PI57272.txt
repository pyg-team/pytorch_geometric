Title: IBM PI57272: A WARNING MESSAGE SHOULD BE PRINTED WHEN SUBSCRIBERONLY SET TO T RUE FOR A SYNDICATOR - United States

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When deployment.subscriberOnly set to true on a server or node
   of cluster, it can't work as a syndicator. If syndication pair
   created with the server or node being the syndicator,
   syndication update won't do anything. The status of the
   syndicator will stuck in queued after syndication pair is
   created for automatic syndication case.
   
   
   [2/14/16 20:28:37:380 CST] 00002beb SyndicatorTas 3   run() -
   Calling SyndicatorTaskManager.getInstance()
   .startUpdateTask(m_updateId)
   [2/14/16 20:28:37:380 CST] 00002beb SyndicatorTas 3   run() -
   Returned from call to SyndicatorTaskManager.getInstance()
   .startUpdateTask(m_updateId)
   
   
   Analysis: a warning message should remind that
   deployment.subscriberOnly should set to false for a syndicator
   Solution: a warning message will show up for the case.
