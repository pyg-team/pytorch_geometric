APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After upgrading to SC 5.2.11 or installing SC 5.2.11, login
   screen may render in an odd manner (no Spectrum Control logo and
    unusual position for login form)  and after clicking login
   the next page contains something like:
   {"result":{"type": "E","msgId":"BPCUI0501E","time":"Oct 1, 2016,
   12:00:00", "text":"An invalid request was rejected."}}
   
   
    
   
   

LOCAL FIX
 *  The issue can be resolved using the following steps:
   
   1. Create a backup of following files:
   C:\Program
   Files\IBM\TPC\ewas\profiles\WebServerProfile\installedApps\WebSe
   rverCell\WebServer.ear\TPC-GUI.war\WEB-INF\web.xml
   C:\Program
   Files\IBM\TPC\ewas\profiles\WebServerProfile\config\cells\WebSer
   verCell\applications\WebServer.ear\deployments\WebServer\TPC-GUI
   .war\WEB-INF\web.xml
   
   
   Modify the web.xml file in the GUI WAR.  This will need to be
   edited in two distinct versions of web.xml.
   C:\Program
   Files\IBM\TPC\ewas\profiles\WebServerProfile\installedApps\WebSe
   rverCell\WebServer.ear\TPC-GUI.war\WEB-INF\web.xml
   C:\Program
   Files\IBM\TPC\ewas\profiles\WebServerProfile\config\cells\WebSer
   verCell\applications\WebServer.ear\deployments\WebServer\TPC-GUI
   .war\WEB-INF\web.xml
   
   Locate the following line in the web.xml file, around line 100.
       <filter-mapping>
           <filter-name>CSRFFilter</filter-name>
           <url-pattern>/*</url-pattern>
       </filter-mapping>
   
   Change the code as marked below:
       <filter-mapping>
           <filter-name>CSRFFilter</filter-name>
           <url-pattern>/CSRF/*</url-pattern>
       </filter-mapping>
   
   Restart Spectrum Control.
   https://www.ibm.com/support/knowledgecenter/en/SS5R93_5.2.8/com.
   ibm.spectrum.sc.doc/fqz0_r_start_stop_tpc_services.html [https://www.ibm.com/support/knowledgecenter/en/SS5R93_5.2.8/com.ibm.spectrum.sc.doc/fqz0_r_start_stop_tpc_services.html]
