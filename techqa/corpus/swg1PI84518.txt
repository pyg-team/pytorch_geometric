PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of the IBM WebSphere Application  *
   *                  Server "single" application classloading    *
   *                  policy                                      *
   ****************************************************************
   * PROBLEM DESCRIPTION: Applications may incur classloading     *
   *                      problems or unexpected behaviors.       *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When an application server is configured with the "single"
   application classloader policy, WebSphere system applications
   and end-user applications share the same application
   classloader. The lack of isolation may cause unexpected
   behaviors or classloading problems whenever end-user and
   system applications provide the same open-source packages and
   resources.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Apply APAR PI84518 to enable WebSphere system applications to
   load from their own application classloaders whenever the
   "single" application classloader policy is configured on an
   application server.
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 9.0.0.8.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
