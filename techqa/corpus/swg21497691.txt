Title: IBM "Test connection failed ... this failure may be caused by the fact that under the default settings SQL Server does not allow remote connections" when testing database connection inside Controller Configuration - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Administrator logs onto the Controller application server, and launches "Controler Configuration". Inside the section "Database connections" the administrator creates a new connection to connect to a SQL database. User clicks the green 'tick' button to test the connection. User receives error message. 

SYMPTOM
Microsoft Data Link Error 

Test connection failed because of an error in initializing provider. An error has occurred while establishing a connection to the server. When connecting to SQL Server 2005, this failure may be caused by the fact that under the default settings SQL Server does not allow remote connections.
[OK]



CAUSE
SQL server has been configured without the option "Allow remote connections to this server" enabled. 

 * For more details, see separate Microsoft article #914277.

RESOLVING THE PROBLEM
Enable the SQL option "Allow remote connections to this server". 

 

Steps: 

The following instructions are based on SQL 2005: 

 1. Enable a short period of downtime (no users using any databases hosted on the SQL server) 
 2. Logon to the SQL server as an administrator 
 3. Launch "SQL Server Management Studio" 
 4. Inside "Object Explorer", right-click on the server, and choose 'Properties' 
 5. Click on section "Connections" 
 6. In the section "Remote server connections", tick the box next to "Allow remote connections to this server" 
 7. Click OK 
 8. Restart the SQL server 
 9. Test.
