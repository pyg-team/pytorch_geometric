ERROR DESCRIPTION
 *  SMARTEAM CATIA V5 INTEGRATION: Part Number
   is changed to UUID and New status if replace
   revision on CATIA
   .
   Scenario:
   
   1.Prepare the following structure:
   Product1
       + Product2
          + Part1
   2.(User A):Check out Product1 and load it to
   CATIA. Product2 and Part1 are in Copy File mode
   on lifecycle operation.
   3.(User B):Check out Product2 and load it to
   CATIA. Also Part1 is in Check Out mode on
   lifecycle operation.
   4.(User B):Modify Part1 something and then check
   in the whole of Product2.
   5.(User A):Define Part1 as work object and then
   select SmarTeam / Assembly Management/Replace
   with Selected Revision menu.
   6.(User A):Select object with the latest revision.
   Result: Part1 is replaced with the latest revision
   correctly.
   7.(User A):Define Product2 as work object and
   then select SmarTeam / Assembly Management /
   Replace with Selected Revision menu.
   8.(User A):Select object with the latest revision.
   Result: Part Nubmer of the CATPart instance is
   changed to UUID and then status to NEW on the
   specification tree.
   
   .
