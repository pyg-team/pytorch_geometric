ERROR DESCRIPTION
 *  product; Netcool/Impact
   Release 5.1.1
   Problem; customer has fp2 applied which includes fix for apar
   IV00394 [http://www-01.ibm.com/support/docview.wss?uid=swg1IV00394], long passwords are allowed for Imapct login, Howeveer,
   if customer tries to log into an OPview it fails because
   password limit
   
   
    
   
   

LOCAL FIX
 *  The fix is to remove the text  maxLength=16 from these files:
   
   opview.war/login_retry.jsp
   opview.war/login.jsp
   
   i.e. replace
   login.jsp:                  maxLength=16
   class="netcool_login_pword"><br>
   login_retry.jsp:                  maxLength=16
   class="netcool_login_pword"><br>
   
   with
   login.jsp:                  class="netcool_login_pword"><br>
   login_retry.jsp:
   class="netcool_login_pword"><br>
   
   
   This change is on the GUI server.
   Remember to make a backup of any file before changing.

 *  ****************************************************************

   * All Netcool/Impact users.                                    *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Opview does not accept passwords longer than 16 characters.  *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   There was a limit of 16 characters which has now been removed.
