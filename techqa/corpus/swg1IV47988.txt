APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Setting the WSSetDefaultPKGName value globally doesn't work when
   the WSInvoke function is inside a policy function
   
   
    
   
   

LOCAL FIX
 *  Be sure to set the WSSetDefaultPKGName value inside the same
   policy function where the the WSInvoke function is located.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Setting the WSSetDefaultPKGName value globally doesn't work  *
   * when                                                         *
   * the WSInvoke function is inside a policy function            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Call WSSetDefaultPKGName in each user defined function.      *
   ****************************************************************
   Variables that are being set in Web Service functions are not
   accessible outside of the user function in which they are
   called.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Web Service functions have been changed to access global
   variables and local variables.
