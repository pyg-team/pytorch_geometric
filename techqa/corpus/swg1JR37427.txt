PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  WebSphere Process Server Users Migrated     *
   *                  from WebSphere Interchange Server           *
   ****************************************************************
   * PROBLEM DESCRIPTION: IllegalArgumentException is thrown      *
   *                      when BO message with no verb is         *
   *                      sent to the migrated JMS/MQ Export.     *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The WebSphere Interchange Server's JMS/MQ connector has a
   DefaultVerb property.  When it's specified and BO message with
   no verb is send to the connector, it sets the DefaultVerb
   value to the deserialized BO.  But after migration, WebSphere
   Process Server doesn't respect the DefaultVerb property and
   causes the exception.
   
   
    
   
   

PROBLEM CONCLUSION
 *  To fix this error, the datahandler wrapper of MQ/JMS binding
   gets the verb from the DefaultVerb property and set it to the
   deserialized BG if it has no verb specified.
