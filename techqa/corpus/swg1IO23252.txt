APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Library server monitor returned error "GetDatabaseLock  Did not
   get database lock".  The error message logged in library server
   log is as below:
   
   "... GetDatabaseLock  Did not get database lock ..."
   
   
    
   
   

LOCAL FIX
 *  Restart library server monitor.
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * 999                                                          *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * When activating a quiesced Content Manager system, the       *
   * library server monitor does not release the database lock    *
   * flag as it should.  This prevents the library server monitor *
   * from correctly monitoring the status of associated resource  *
   * managers.                                                    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * No                                                           *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fixed in IBM Content Manager 8.5 fix pack 4
   
   
    
   
   

TEMPORARY FIX
 *  Restarting Library Server Monitor
