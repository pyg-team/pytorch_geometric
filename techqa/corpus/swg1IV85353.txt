ERROR DESCRIPTION
 *  Issue on Rave Line Chart (Dash 3.1.2.1). All ascending
   and
   descending option is not working.
   
   For example
   
   
   Rave Line Chart released in Dash 3.1.2.1 is showing dates from
   18th May
   to 31st May then 1st and 2nd June. Client is not accepting this
   format when descending
   it should be like from 2nd June, 1st June, 31st May, 30th, May,
   29th May
   and so on.
   
   Data Model is showing descending order:
   
   
    
   
   

LOCAL FIX
 *  Change the date data type as varchar/string, which is not
   acceptable by all customer
   
   
    
   
   

PROBLEM SUMMARY
 *  THis is the limitation with Rave feature. Also Rave is not in
   support.
   
   As alternative workaround is suggested in the published technote
