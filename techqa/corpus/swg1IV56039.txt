 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The current observed behavior (that is wrong) seems to be a side
   effect
   of an improvement contained in TWS 851 FP05 in managing
   interactive jobs
   for dynamic agents. For some reason it seems that when the user
   running
   the job is impersonated (this is done using Microsoft API) the
   profile
   that is associated to the user has a default timeout set to 60
   secs for
   vbs scripts (only for vbs script).
   This behavior is reported in some MSDN technotes and could
   require
   additional investigation with Microsoft.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  The problem applies to dynamic interactive jobs only.
   When the user running the job is impersonated the profile that
   is associated to
   the user has a default timeout set to 60 secs for vbs scripts
   (only for vbs script).
   We have reworked the way the logon is done to avoid this side
   effect.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The problem will be fixed in TWS 8.5.1 FP06, TWS 8.6 FP04,
   TWS 9.1 FP01, TWS 9.2 FP01
   
   
    
   
   

TEMPORARY FIX
