 This document provides a collection of updates and corrections to the 8.1 documentation for Fix Pack 3.

Documentation updates: 

IBM Datacap Taskmaster Capture support for IBM Security Access Manager 
http://www.ibm.com/support/docview.wss?uid=swg27045387 [http://www.ibm.com/support/docview.wss?uid=swg27045387]
This techdoc describes how Taskmaster Web can authenticate Taskmaster users who are already authenticated in the IBM Security Access Manager SSO environment. 


Enabling SSL for your LLLDAP configuration (LDAPS)
http://www.ibm.com/support/docview.wss?uid=swg27045687 [http://www.ibm.com/support/docview.wss?uid=swg27045687] 
You can enable LDAP connections over Secure Sockets Layer (SSL) for your LLLDAP configuration. 


Customizing the DotEdit Verify line items panel to handle large numbers of line items 
http://www.ibm.com/support/docview.wss?uid=swg27045485 [http://www.ibm.com/support/docview.wss?uid=swg27045485] 
Handling large numbers of line items in the Verify task consumes a lot of memory, which can cause the application to hang, crash, or have issues displaying controls in the panel. You can add a scroll bar to the line items panel, which enables the application to load manageable portions of the line items at a time, instead of attempting to load all of the line items at once. This customization makes it possible to load and process thousands of line items without crashing or degrading performance. 


Setting a variable in DotEdit to prevent Image View from returning focus to the first page (Main_Page)
http://www.ibm.com/support/docview.wss?uid=swg27045562 [http://www.ibm.com/support/docview.wss?uid=swg27045562] 
When you click a field on a trailing page and the field is associated with the main page, or the field does not have a zone location, focus jumps back to the first page. To keep focus on the current page, you can specify a setting to prevent focus from automatically returning to the first page. 


Setting up DotScan to undo rescanned pages
http://www.ibm.com/support/docview.wss?uid=swg27045038 [http://www.ibm.com/support/docview.wss?uid=swg27045038] 
For TWAIN and ISIS scanned pages, you can set up DotScan to undo rescanned pages to go back to a previous image. 


Running verification on batches and documents
https://www.ibm.com/support/knowledgecenter/SSZRWV_8.1.0/com.ibm.dc.develop.doc/dcadg606.htm [https://www.ibm.com/support/knowledgecenter/SSZRWV_8.1.0/com.ibm.dc.develop.doc/dcadg606.htm]
By default Taskmaster Web loads only the data file for the current image when a Verify task is run. You cannot run cross document or cross batch validations. To be able to run validation on an entire batch or a document, you can configure the caching option by setting the LoadDoc value in the <verify>.set.xml file to load all of the data files in the current batch or only for current document.
