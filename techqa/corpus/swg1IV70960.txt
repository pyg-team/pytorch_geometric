PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * Maximo users of linked documents who have attached a domain  *
   * to DOCLINKS.DOCUMENT                                         *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * BMXAA6713E - The MBO fetch operation failed in the mboset    *
   * with the SQL error code -206. The record                     *
   * could not be retrieved from the database. See the log file   *
   * for more details about the error.                            *
   *                                                              *
   * com.ibm.db2.jcc.am.SqlSyntaxErrorException: "DOCTYPE" is not *
   * valid in the context where it is used.. SQLCODE=-206,        *
   * SQLSTATE=42703                                               *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Apply latest fix pack.                                       *
   ****************************************************************
   Reproduced in 7.5.0.6 and 7.5.0.7
   
   1. Create a Custom ALN-Domain (Upper(20))
   2. Insert some example items into the domain
   3. In Database Configuration, modify the DOCLINKS object by
   attaching the new domain to the DOCUMENT attribute.
   4. Configure this change.
   5. Modify  LIBRARY.xml
    a) Search for id="addnewattachments_file_1_4"
      <multiparttextbox dataattribute="document"
   descdataattribute="description" id="addnewattachments_file_1_4"
   label="Name the document"/>
   
    b) Modify this line as follows:
      <multiparttextbox dataattribute="document"
   descdataattribute="description" id="addnewattachments_file_1_4"
   lookup="valuelist" inputmode="REQUIRED" label="Name the
   document"/>
   
   6. Import the changed LIBRARY.xml
   
   Go to PR Application,try to attach a document you will have the
   option to select the value from the field but no results
   displayed for the domain values.
   
   The following error message is in the logs: BMXAA6713E - The MBO
   fetch operation failed in the mboset with the SQL error code
   -206.
   The record could not be retrieved from the database. See the log
   file for more details about the error.
   
   com.ibm.db2.jcc.am.SqlSyntaxErrorException: "DOCTYPE" is not
   valid in the context where it is used..
   SQLCODE=-206,SQLSTATE=42703
   
   
    
   
   

PROBLEM CONCLUSION
 *  Error message "BMXAA6713E - The MBO fetch operation failed in
   the mboset with the SQL error code -206. The record could not be
   retrieved from the database. See the log file for more details
   about the error.
   com.ibm.db2.jcc.am.SqlSyntaxErrorException: "DOCTYPE" is not
   valid in the context where it is used..
   SQLCODE=-206,SQLSTATE=42703" is no longer encountered when a
   domain is attached to DOCLINKS.DOCUMENT.
   
   The fix for this APAR is contained in the following maintenance
   package:
     | release\fix pack\interim fix for Release 7.5.0.9 and Release
   7.6.0.1 of Base Services
