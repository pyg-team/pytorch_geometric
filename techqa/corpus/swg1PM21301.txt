APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Session timeout is not working for Trust Association
   Intercepter (TAI) authenticated requests. When the first
   request after session timeout comes in, a new session is
   created although timeout.resume.session=false.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Session timeout is not working for Trust Association
   Intercepter (TAI) authenticated requests. When the first request
   after session timeout comes in, a new session is created
   although timeout.resume.session=false.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Modified Portal behavior so that session does not get
   re-created implicitly after session timeout even for TAI
   authenticated requests.
   
   Failing Module(s):
      Authorization/Authentication (login/logout)
   
   Affected Users:
      All
