Title: IBM PK64600: PROCLIB.EKM2ENV shipped with the JVM is out-of-date - United States

Text:
AIX SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The Encryption Key Manager Component of the Java Platform (EKM)
   has been changed to start using the command
   com.ibm.keymanager.EKMServer
   
   Whereas previously it used com.ibm.keymanager.KMSAdminCmd.
   
   This apar addresses updating the shipped Sample JCL/Env files.
   The EKM Documentation has already been updated to include this
   detail.
   
   
   Error Message:
   java com.ibm.keymanager.KMSAdminCmd
   <configfile name>&#168; : read the config properties from file.
   -filename <filename> : read EKM input commands from a file.
   -i : execute ekm commands interactively.
   <ekmcli> -ekmuser <username> -ekmpassword <password>
   
   This may be an additional Error Message:
   
   IOS627E PRIMARY INTERFACE WITH ENCRYPTION KEY MANAGER CANNOT BE
   CONNECTED DUE TO SOCKET FAILURE ERNO=0468
   11 EKM TASK NOT RESPONDING. CONTACT MVS OR STORAGE SUPPORT -
   REPLY Y
   
   .
   Stack Trace: N/A
   .
   N/A
   
   
    
   
   

LOCAL FIX
 *  Change your invocation to use EKMServer
   
   For Java SDK 5, this is fixed in SR8 APAR PK69074 [http://www-01.ibm.com/support/docview.wss?uid=swg1PK69074]
   For Java SDK 6, this will be fixed in SR2
   
   
    
   
   

PROBLEM SUMMARY
 *  Updating  the class file from KMSAdmin to EKMServer in script
   PROCLIB.EKM2ENV
   
   
    
   
   

PROBLEM CONCLUSION
 *  This defect will be fixed in:
   1.4.2 SR11 (z/OS 31-bit)
   .
   NA
   .
   To obtain the fix:
   Install build 20080426 or later
   
   
    
   
   

TEMPORARY FIX
 *  NA
