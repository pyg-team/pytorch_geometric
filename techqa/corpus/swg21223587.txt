Title: IBM Workaround for installing DB2 Content Manager Version 8.3 when WebSphere Application Server Version 6 is installed in a directory with spaces. - United States

Text:
db2 content manager websphere application server network deployment directory with space rc=142 TECHNOTE (FAQ)

QUESTION
 The installation of DB2® Content Manager Version 8.3 fails on Windows® if the 
WebSphere® Application Server Version 6 installation directory contains spaces (for 
example : C:\Program Files\WebSphere).

CAUSE
If the WebSphere Application Server installation directory has spaces, the deployment of
the resource manager application will fail while trying to regenerate the plug-in information 
for IBM HTTP Server. 

ANSWER
The solution is to change the WebSphere variable, USER_INSTALL_ROOT, to specify the 
installation directory using the Windows 8.3 naming format so that the location does not 
contain spaces. The USER_INSTALL_ROOT variable can be updated in the WebSphere 
Application Server Administration Console. 

For example, if WebSphere Application Server is installed in C:\Program Files\WebSphere\AppServer, 
the USER_INSTALL_ROOT variable should be changed to C:\PROGRA~1\WebSphere\AppServer. 
To determine the equivalent file name for your directories that contain spaces, you can run 
the command dir /x from a DOS command prompt. 

Follow these steps to update the USER_INSTALL_ROOT variable: 

1. Logon to the WebSphere Application Server Administration Console 
(for example: http://hostname:9060/ibm/console) 

2. Go to Environment –> WebSphere Variables as shown in Figure 1.

Look for the entry USER_INSTALL_ROOT. 
It will probably be listed on Page 2 of the WebSphere variables as shown in Figure 2.

If you are not using WebSphere Application Server Network Deployment,
you can run the DB2 Content Manager installation again and you should find that this
error no longer occurs.

If you are using WebSphere Application Server Network Deployment (ND),
you can proceed with the installation of DB2 Content Manager as described in the Technote link provided below.
