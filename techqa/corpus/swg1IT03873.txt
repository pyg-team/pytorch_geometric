APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When certain MQ Objects are in transition state from up to down
   or down to up, if 'show mq-connections' is run it could cause a
   conflict resulting in an unexpected immediate restart.
   
   
    
   
   

LOCAL FIX
 *  Ensure no MQ status providers are run when an MQ Object is in
   transition state.
   
   
    
   
   

PROBLEM SUMMARY
 *  Affected are users of MQ Queue Manager Object and MQ/MQFTE Front
   Side Protocol Handler.
   
   DataPower might restart while querying MQ Connection Status.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Fix is available in 6.0.0.10, 6.0.1.6, and 7.0.0.3.
   
   For a list of the latest fix packs available, please see:
   http://www-01.ibm.com/support/docview.wss?uid=swg21237631 [http://www-01.ibm.com/support/docview.wss?uid=swg21237631]
