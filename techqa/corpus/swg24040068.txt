ABSTRACT
 This patch upgrades OpenSSL for Netcool/Reporter to fix security vulnerabilities. 

DOWNLOAD DESCRIPTION
Bug/Enhancement Identification


This patch upgrades OpenSSL for Netcool/Reporter to version 1.0.1m to fix below security vulnerabilities:


 1.  NULL pointer dereference when compling using no-ssl3 option (CVE-2014-3569) 
 2.  Bignum squaring (BN_sqr) may produce incorrect results on some platforms, including x86_64 (CVE-2014-3570) 
 3.  OpenSSL is vulnerable to a denial of service, caused by a NULL pointer dereference when handling malicious messages. By sending a specially-crafted DTLS message, a remote attacker could exploit this vulnerability to cause a segmentation fault. (CVE-2014-3571) 
 4.  An OpenSSL client will accept a handshake using an ephemeral ECDH ciphersuite using an ECDSA certificate if the server key exchange message is omitted (CVE-2014-3572) 
 5.  Certificate's fingerprint might be changed by modifying the contents of the signature algorithm or the encoding of the signature (CVE-2014-8275) 
 6.  An OpenSSL server will accept a DH certificate for client authentication without the certificate verify message (CVE-2015-0205) 
 7.  A memory leak can occur in the dtls1_buffer_record function under certain conditions (CVE-2015-0206) 
 8.  Use After Free following d2i_ECPrivatekey error (CVE-2015-0209) 
 9.  Segmentation fault in ASN1_TYPE_cmp (CVE-2015-0286) 
 10. ASN.1 structure reuse memory corruption (CVE-2015-0287) 
 11. X509_to_X509_REQ NULL pointer deref (CVE-2015-0288) 
 12. PKCS#7 NULL pointer dereference (CVE-2015-0289) 
 13. OpenSSL could allow a remote attacker to execute arbitrary code on the system, caused by an error when processing base64 encoded data. An attacker could exploit this vulnerability using specially-crafted base 64 data to corrupt memory and execute arbitrary code on the system and cause a denial of service. (CVE-2015-0292) 
 14. DoS via reachable assert in SSLv2 servers (CVE-2015-0293) 
 15. RC4 Bar Mitzvah Attack for SSL/TLS (CVE-2015-2808)

PREREQUISITES
Netcool/Reporter 2.2.0.0 release is installed. Netcool/Reporter Fix Pack 2.2.0.9 is installed.
Netcool/Reporter 2.2.0.9 Interim Fix 1, 2.2.0.9-TIV-NCReporter-IF0001:
ITNR-2.2-Linux_ApacheOpenssl-IV29878-988-0.tar.gz is installed.

INSTALLATION INSTRUCTIONS
(All references to INFOHOME indicate the root directory where Netcool/Reporter 
is installed.)

1. Shut down Netcool/Reporter.

2. Back up your Netcool/Reporter file system and repository database 

3. This patch is delivered as a compressed UNIX tar file, 2.2.0.9-TIV-NCReporter-Linux-IF0005.tar.gz. Extract it into a temporary directory (for example, /opt/tmp) as follows: 

% cd /opt/tmp 
% gzip -dc 2.2.0.9-TIV-NCReporter-Linux-IF0005.tar.gz | tar xvf -

A subdirectory, 2.2.0.9-TIV-NCReporter-Linux-IF0005 is created under the temporary directory(for example,/opt/tmp/2.2.0.9-TIV-NCReporter-Linux-IF0005).

All files and utilities required to install the patch are extracted into this subdirectory.

4. Change to the directory 2.2.0.9-TIV-NCReporter-Linux-IF0005, which contains the patch files to be installed.

5. Run the patch installer "setup.sh" script. It might stop the patch process if the $INFOHOME environment variable cannot be found. You might need to change the permission to execute the setup.sh script using chmod.

For example:

% chmod 755 setup.sh
The patch installer backs up existing files into the $INFOHOME/backup/vpatch_2.2.0.9-IF0005 directory.

[Optional] if you want to rollback this upgrade process, run the following command:

% setup.sh rollback

6. Add $APACHE_HOME/lib into your environment variable LD_LIBRARY_PATH. For example: LD_LIBRARY_PATH=/netcool/reporter/apache/lib:$LD_LIBRARY_PATH;export LD_LIBRARY_PATH

7. After the patch is processed, change to the directory APACHE_HOME/bin and run below commands:

OPENSSL_CONF=../conf/openssl.cnf;export OPENSSL_CONF
openssl req -config ../conf/openssl.cnf -new -out ../conf/server.csr
openssl rsa -in privkey.pem -out ../conf/server.key
openssl x509 -in ../conf/server.csr -out ../conf/server.crt -req -signkey ../conf/server.key -days 365 

8. Re-start Netcool/Reporter.
