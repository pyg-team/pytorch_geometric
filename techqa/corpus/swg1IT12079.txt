Title: IBM IT12079: PUREAPPLICATION SYSTEM: DEPLOYMENTS THAT REQUIRE USER TO UPLOAD AN ARTIFACT AT DEPLOY TIME HAVE THE NAME ENCODED TWICE - United States

Text:
  A FIX IS AVAILABLE
IBM PureApplication System Version 2.1.2.0 [http://www-01.ibm.com/support/docview.wss?uid=swg24041499]


SUBSCRIBE

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  User created a deployment  that required a deploy time artifact
   "my test.pem". After the upload the artifact name was displayed
   on the console as
   "deployment_artifacts/5/artifacts/my%20test.pem".
   
   During the deployment the URI became
   "deployment_artifacts/5/artifacts/my%2520test.pem"
   
   
    
   
   

LOCAL FIX
 *  Name your artifacts using alphanumeric characters that do not
   require URL encoding.
   
   
    
   
   

PROBLEM SUMMARY
 *  The artifact path was being encoded twice.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A code fix is being provided in PureApplication System version
   2.1.2.0.
   
   
    
   
   

TEMPORARY FIX
