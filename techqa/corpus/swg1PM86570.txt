Title: IBM PM86570: SWITCHING HISTORIC CHARTING DATA SEVERAL TIMES WILL CAUSE DMGR CRASH AND REPORT TOO MANY OPEN FILES EXCEPTION - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Switch Historic Charting data several times will cause the
   deployment manager to crash and report too many open files
   exception
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of IBM WebSphere Application      *
   *                  Server V8.5                                 *
   ****************************************************************
   * PROBLEM DESCRIPTION: Administrative console crashes when     *
   *                      too many historic reports are opened.   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   When too many reports are opened in Runtime Operations, the
   administrative console will crash. There are accompanying
   messages in SystemOut about too many files being opened.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The code was updated so that files are closed when they are no
   longer in use.
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 8.5.5.0.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
   
   
    
   
   

TEMPORARY FIX
