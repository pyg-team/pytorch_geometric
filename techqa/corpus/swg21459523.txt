PROBLEM(ABSTRACT)
 You purge the JVM with CEMT PERFORM JVM PURGE JVMPROFILE(DFHJVMPR). When you try to start 4 JVMs with CEMT PERFORM JVMPOOL START JVMCOUNT(4) CEXECKEY JVMPROFILE(DFHJVMPR), it fails with an AbendAEXZ. 

SYMPTOM
DFHDU0203I A transaction dump was taken for dump code: AEXZ [http://pic.dhe.ibm.com/infocenter/cicsts/v4r2/topic/com.ibm.cics.ts.messages.doc/cics_mc/AExx_abend_codes/AEXZ.html].
DFHAC2236 Transaction CEMT abend AEXZ in program DFHEMTD term termid. Updates to local recoverable resources will be backed out.


CAUSE
CICS group DFHJAVA is not installed.

DIAGNOSING THE PROBLEM
A review of the dump shows that CICS is unable to attach transaction CJPI. The trace shows an SJIS_REASON of x'D' that means SJIS_CJPI_ATTACH_FAILED. Transaction CJPI [http://pic.dhe.ibm.com/infocenter/cicsts/v4r2/topic/com.ibm.cics.ts.systemprogramming.doc/topics/dfha726.html] is defined in group DFHJAVA [http://pic.dhe.ibm.com/infocenter/cicsts/v4r2/topic/com.ibm.cics.ts.resourcedefinition.doc/topics/dfha46k.html] but the DFHJAVA group is not installed.

RESOLVING THE PROBLEM
Install CICS group DFHJAVA. [http://publib.boulder.ibm.com/infocenter/cicsts/v4r1/index.jsp?topic=/com.ibm.cics.ts.resourcedefinition.doc/topics/dfha46k.html]The DFHJAVA group should be part of your startup group list GRPLIST [http://pic.dhe.ibm.com/infocenter/cicsts/v4r2/topic/com.ibm.cics.ts.doc/dfha2/parameters/dfha2_grplist.html].
