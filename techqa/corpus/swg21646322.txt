Title: IBM A CWSIT0006E error occurs shortly after IBM Process Designer for IBM Business Process Manager (BPM) starts - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 You change the SIB_ENDPOINT_SECURE_ADDRESS port of the profile after the profile is created. When you attempt to start IBM Process Designer, you see a CWSIT0006E error. 

SYMPTOM
The folllowing text shows the error that IBM Process Designer throws shortly after starting:

Caused by: com.ibm.websphere.sib.exception.SIResourceException: 
CWSIT0006E: It was not possible to contact any of the specified 
bootstrap servers. Please see the linked exception for further details. 
Bootstrap connections were attempted to: [NLWSL1095.ddns.slb.atosorigin-
asp.com:7306:BootstrapSecureMessaging]


CAUSE
IBM Process Designer throws the error as it tries to connect to the Bootstrap Secure Messaging bus on the wrong port.

DIAGNOSING THE PROBLEM
The SIB_ENDPOINT_SECURE_ADDRESS port has been changed after the profile was created. There are stale references to the old port in the Javaâ„¢ Message Service (JMS) resources for the endpoint.

RESOLVING THE PROBLEM
To resolve this issue, complete the following steps:




 1. Log into the Integrated Solution Console.
    
    
 2. Expand Resources > JMS > Queue Connection Factories. Click each entry in the table and check the Provider Endpoints to see if the the old port information exists. If the old port information exists, update it with the correct port number.
    
    
 3. Expand Resources > JMS > Topic Connection Factories. Click each entry in the table and check the Provider Endpoints to see if the the old port information exists. If the old port information exists, update it with the correct port number.
