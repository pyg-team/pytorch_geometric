Title: IBM Ruleset parsing fails with IncompatibleClassChangeError - United States

Text:
 TECHNOTE (FAQ)

QUESTION
 How to prevent an error such as the one shown below upon ruleset parsing?

ilog.rules.res.session.IlrSessionCreationException: An error occurred while the rule session was created:
com.ibm.rules.res.xu.internal.XUException: The ruleset parsing process failed.
Caused by: java.lang.IncompatibleClassChangeError: org.objectweb.asm.ClassVisitor
at ilog.jit.asm.IlxASMClassTranslator.translate(IlxASMClassTranslator.java:120)
at ilog.jit.asm.IlxASMClassBuilder.buildClass(IlxASMClassBuilder.java:61)
at ilog.rules.engine.sequential.platform.IlrSEQTaskFactory.buildHierarchy(IlrSEQTaskFactory.java:507)
... 

ANSWER
Make sure that the Rule Execution Server XU resource adapter has the option "Isolate this resource provider" checked if you are using WebSphere Application Server 8.5 or 8.5.5, as per the instructions in documentation: Deploying the XU RAR [http://www-01.ibm.com/support/knowledgecenter/SSQP76_8.6.0/com.ibm.odm.distrib.config.was/config_ds_res_was8/tsk_was_deploy_xu.html?lang=en] 

("For WebSphere Application Server version 8.5 or 8.5.5, select the Isolate this resource provider option.")
