APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  WebSphere does not establish SSL connection for CICS publish
   call as it directs the call to TCP port. The call from CICS does
   not support CSIv2 tag that indicates SSL connection.  This
   resulted in SECJ0306E (Authentication Error) as the certificate
   sent from CICS was not read.
   
   
    
   
   

LOCAL FIX
 *  With APAR PK45120, WebSphere supports SSL Tag20 to establish
   connection with CICS and other servers that does not implement
   CSIv2 tags.
   
   Following configuration is required in order to utilize
   additional support PK45120 has introduced.
   
   
   
   CONFIGURATION FOR WebSphere 6.0.2.x
   
   Set following three properties.
   
   1.  com.ibm.CORBA.SSLClientAuthentication=true.
   
   To add this to the ORB properties from the GUI, go to Servers ->
   Application Servers ->
   click on <server name> -> Container Services -> ORB Service ->
   Custom Properties.
   Add this property from the panel and set it to true.
   
   
   2. com.ibm.CSI.activeProtocol=BOTH
   
   To configure this property from the GUI, go to Security ->
   Global security.
   set "Active protocol" to "CSI and SAS" from pulldown.
   
   
   3.
   com.ibm.CSI.claimTLClientAuthenticationRequired/Supported=true
   
   To configure this propertiy,  go to Security -> Global security
   ->
   CSIv2 inbound authentication
   
   Under "Client certificate authentication" select radiobutton to
   either "Supported" or "Required"
   
   
   
   CONFIGURATION FOR WebSphere 6.1.0.x
   
   Set following values in specified panels.
   
   
   1. Set CSIv2 inbound authentication to Supported or Required.
   
   Go to Global Security -> Authentication Protocol -> CSIv2
   inbound authentication
   Set "Client certificate authentication" to "Supported" or
   "Required"
   
   2. Set CSIv2 inbound transport to SSL-required or SSL-supported
   
   Go to Global Security -> Authentication Protocol -> CSIv2
   inbound transport
   Set "transport" to "SSL-required" or "SSL-supported"
   
   
   For WebSphere v6.1, please refer to following article in
   InfoCenter for details.  "Configuring CSIv2 for Secure Sockets
   Layer client authentication"
   
   http://publib.boulder.ibm.com/infocenter/wasinfo/v6r0/index.jsp?
   topic=/com.ibm.websphere.nd.doc/info/ae/ae/tsec_configsas.html [http://publib.boulder.ibm.com/infocenter/wasinfo/v6r0/index.jsp?topic=/com.ibm.websphere.nd.doc/info/ae/ae/tsec_configsas.html]
   
   Keywords: rebind_corba_object csiv2 security
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: IBM WebSphere Application Server version 6.0 *
   *                 and 6.1 users of CICS sending ORB requests   *
   *                 over SSL.                                    *
   ****************************************************************
   * PROBLEM DESCRIPTION: Authentication errors such as SECJ0306E *
   *                      are thrown when attempting to           *
   *                      authenticate.                           *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   Application Server does not support authentication using a
   certificate if ORB requests are without a CSIv2 tag. This
   results in authentication failure such as SELJ0306E when an
   ORB request tries to access resources with a valid certificate.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Implemented function that enables certificate authentication
   for ORB requests without CSIv2 tags in order to support
   interoperability with CICS.
   
   This APAR requires following configuration in addition to
   applying the APAR.
   
   Configuration for WebSphere Application Server version 6.0.2.x
   
   1.  com.ibm.CORBA.SSLClientAuthentication=true.
   
   To add this to the ORB properties from the GUI, go to Servers
   -> Application Servers ->
   click on <server name> -> Container Services -> ORB Service ->
   Custom Properties.
   Add this property from the panel and set it to true.
   
   
   2. com.ibm.CSI.activeProtocol=BOTH
   
   To configure this property from the GUI, go to Security ->
   Global security.
   set "Active protocol" to "CSI and SAS" from pulldown.
   
   
   3. com.ibm.CSI.claimTLClientAuthenticationRequired
   or com.ibm.CSI.claimTLClientAuthenticationSupported
   =true
   
   To configure this property, go to Security -> Global security
   -> CSIv2 inbound authentication
   
   Under "Client certificate authentication" select radiobutton to
   either "Supported" or "Required"
   
   
   Configuration for WebSphere Application Server version 6.1.0.x
   
   1. Set CSIv2 inbound authentication to Supported or Required.
   
   Go to Global Security -> Authentication Protocol -> CSIv2
   inbound authentication
   Set "Client certificate authentication" to "Supported" or
   "Required"
   
   2. Set CSIv2 inbound transport to SSL-required or SSL-supported
   
   Go to Global Security -> Authentication Protocol -> CSIv2
   inbound transport
   Set "transport" to "SSL-required" or "SSL-supported"
   
   "Configuring CSIv2 for Secure Sockets Layer client
   authentication"
