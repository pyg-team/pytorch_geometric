Title: IBM When opening a work item from an in-basket in Case Manager Client, it failed to load the IBM form with CDEWG5405E error - United States

Text:
blank; IBM form; ICM; 5.1; 5.1.1; Case Form; widget TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 The customer configured the Case Form Widget to use a form template. It was unable to load the IBM form in Case Manager Client with CDEWG5405E error when opening a work item from an in_basket. 

SYMPTOM

CAUSE
The problem was due to the log4j-1.2.7.jar file of IBM Cognos software on the server configured in the ICM Classpath. See the screen shot from running to http://<server name>:<port #>/CaseManager/CASEREST/v1/info below. 
[/support/docview.wss?uid=swg21673198&aid=1] [/support/docview.wss?uid=swg21673198&aid=1]


ENVIRONMENT
Windows 2008 R2, IBM Case Manager (ICM) 5.1.1, ICM development profile with the stand-alone WAS profile



RESOLVING THE PROBLEM
To resolve the CDEWG5405E error, you must remove the log4j-1.2.7.jar file from WAS Classpath. You can edit the setupCmdLine.bat being called by the WAS startServer.bat file to remove the reference log4j-1.2.7.jar file
