Title: IBM NotSerializableException when accessing WebSphere Commerce Administration Tools - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 When accessing WebSphere Commerce Administration tools like Accelerator, you encounter a NotSerializableException in your Systemout.log file. 

SYMPTOM
Error seen in Systemout.log file when making a request to the tool: 

0000002f CommerceSrvr E com.ibm.commerce.struts.StrutsActionHelper canCallerAccessView CMN0409E: The following error occurred during processing: 
"java.rmi.MarshalException: Exception occurred in server thread; nested exception is: java.io.NotSerializableException". java.rmi.MarshalException: Exception occurred in server thread; nested exception is: java.io.NotSerializableException 
at com.ibm.rmi.util.ProxyUtil.copyObjects(ProxyUtil.java:220) 
at com.ibm.CORBA.iiop.UtilDelegateImpl.copyObjects(UtilDelegateImpl.java:797) 
at javax.rmi.CORBA.Util.copyObjects(Util.java:309) 


CAUSE
Incorrect configuration in the WebSphere Application Server Administration Console.


RESOLVING THE PROBLEM
There is a setting in the WebSphere Application Server Administration console called 'Pass by reference'. This value should be enabled for Commerce environments. 


To enable:

Click Servers > Server Types > Application servers > server_name > Container services > ORB Service and select Pass by reference.

More information can be found about this configuration here:

http://www.ibm.com/support/knowledgecenter/en/SSAW57_7.0.0/com.ibm.websphere.nd.doc/info/ae/ae/uorb_rsetg.html [http://www.ibm.com/support/knowledgecenter/en/SSAW57_7.0.0/com.ibm.websphere.nd.doc/info/ae/ae/uorb_rsetg.html]
