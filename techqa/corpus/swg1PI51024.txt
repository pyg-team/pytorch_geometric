APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  In IBM Rational Quality Manager (RQM), escaped XML entities in
   the summary attribute of requirements are displayed in test case
   PDF/CSV without unescaping them
   
   
   Steps to reproduce:
   1. create a requirement with the name: "Hello, <name>"
   2. create a test case
   3. link the above test case to the above requirement
   4. export the test case to a summary PDF
   
   Expected Results:
   In the Requirement Link section of the PDF, there should be a
   requirement listed with the name "Hello, <name>"
   
   Actual Results:
   There is a Requirement Link, but its name is "Hello,
   &lt;name&gt;"
   
   Workaround:
   There is no workaround
   
   Review the following work item on Jazz.net for further details:
   https://jazz.net/jazz02/web/projects/Rational%20Quality%20Manage
   r#action=com.ibm.team.workitem.viewWorkItem&id=143327 [https://jazz.net/jazz02/web/projects/Rational%20Quality%20Manager#action=com.ibm.team.workitem.viewWorkItem&id=143327]
