APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  If you are using IBM Business Process Manager (BPM) V8.5.7, you
   might notice high CPU usage when you run a task search,
   accompanied by report about hung threads from the application
   server. A deeper analysis of the hung threads stacks shows that
   all the threads are hung in the
   com.ibm.bpm.dashboards.data.TaskListItem.fetchUserInfo method.
   
   PRODUCTS AFFECTED
   IBM Business Process Manager (BPM) Advanced
   IBM BPM Standard
   IBM BPM Express
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  This issue is caused by hung threads.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix will be included in a future release of IBM Business
   Automation Workflow that synchronizes access to a shared
   resource in
   com.ibm.bpm.dashboards.data.TaskListItem.fetchUserInfo, fixing
   the hung threads issue.
   
   
    
   
   

TEMPORARY FIX
 *  Not applicable.
