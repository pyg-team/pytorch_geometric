 *  When you start an IBM Business Process Manager (BPM) application

   start, which involves certain initialization steps. In rare
   cases, these operations might lock each other and cause the
   transactions to time out. In the SystemOut.log file of the

LOCAL FIX
 *  Restart the server.
   
   
    
   
   

PROBLEM SUMMARY
 *  Certain initialization steps that occur when the server starts
   require access to EJB resources. These resources become
   available when the EJB module they are contained in has started.
   When the initialization steps started, all resources had not
   started. As a result, two resources waited on each other,
   causing the timeout.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A fix that changes the initialization steps to run when all
   resources are guaranteed to be available is included in an IBM
   Business Process Manager (BPM) V8.5 cumulative maintenance
   vehicle. For more information, see "Product maintenance
   strategy for IBM Business Process Manager V8.5.6.0 and later" at
   http://www.ibm.com/support/docview.wss?uid=swg21964995. [http://www.ibm.com/support/docview.wss?uid=swg21964995.]
