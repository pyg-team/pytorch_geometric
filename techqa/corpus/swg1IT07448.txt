ERROR DESCRIPTION
 *  The below Call Home events could occur while upgrading IBM
   PureApplication System from version 2.0.0.1 to a 2.0.0.1
   interim fix version.
   
   CWZIP9001E An unexpected internal error
   [com.ibm.purescale.global.GlobalException: Null object returned
   obtaining records from table global_ipconfig] has occurred.
   
   CWZIP6105E System management is offline
   
   CWZIP9001E An unexpected internal error
   [com.ibm.pdq.runtime.exception.DataRuntimeException:
   [pdq][0][2.7.116] Caused by:
   com.ibm.db2.jcc.am.DisconnectNonTransientConnectionException:
   [jcc][t4][2043][11550][4.14.113] Exception
   java.net.NoRouteToHostException:
   
   CWPZC9001E: Could not obtain Connection from
   org.apache.commons.dbcp.PoolingDataSource;
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  There were cron jobs being scheduled during the system upgrade
   that were causing these events to be generated.
   
   
    
   
   

PROBLEM CONCLUSION
 *  A code fix is being delivered in PureApplication System version
   2.1.0.0 to prevent these events from occurring.
