APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Duplicate entries of the same custodian are returned on the
   People in Scope page and Hold Notice Recipients page as a
   result of importing a custodian using CSV file with a different
   email address than the one used when this custodian was added
   to the Hold Notice.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Duplicate entries of the same custodian are returned on the
   People in Scope page and Hold Notice Recipients page as a result
    of importing a custodian using CSV file with a different email
   address than the one used when this custodian was added to the
   Hold Notice.
   
   
    
   
   

PROBLEM CONCLUSION
 *  This issue is now fixed.
   A new configuration parameter is added to control whether to
   allow duplicate custodian entries to be imported or not. The
   component parameter is
   LEGACY_BEHAVIOR_CONFIGURATION.DISABLE_EMAIL_CHECK_FOR_SCOPE_TARG
   ET_IMPORT. If the paramter value is set to yes, it enables
   legacy behavior. If it is set to no, the duplicate entries are
   skipped.
   The fix is available from fix central in Atlas 6.0.3.2 iFix 2.
