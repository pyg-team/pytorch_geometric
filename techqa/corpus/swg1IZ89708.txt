ERROR DESCRIPTION

   FITS MR1015106556 Description
   =============================
   Ability to stitch ITCAM for AD data to ITCAM for SOA data in TT
   Topology View when using the Apache AXIS2 SOAP engine
   Customer would like to see an end to end view of a transaction
   that passes through the following components
   .
   [MQ on Z]  ---> [MQ on Distributed] ---> [MB on Distributed*]
   ---(SOAP calls)---> [Apache Axis2 SOAP engine in embedded HTTP
   server in WAS ]  ---(SOAP calls)---->  [MB on Distributed*]
   *=same server
   .
   At the moment it is not possible to stitch such a transaction
   together in the TT topology view
   
   
    
   
   

LOCAL FIX

   together in the TT topology view
