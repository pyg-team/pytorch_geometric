QUESTION
 What are some examples of the mkreplica -import command for IBM Rational ClearQuest MultiSite? 

ANSWER
Run the command "mkreplica -import" in a command window. 

Running this command without further arguments will present the full syntax for the command. 

Included in the output are the database parameters for the three vendor databases supported for production MultiSite usage, IBM DB2, Microsoft SQL Server, and Oracle.


if <vendor> == DB2,
<db-info> := Database name 

<db-params> := -server server-name 

-dbo.login dbo-name [dbo-pwd] 

[-con.nectopts connect-options] 

if <vendor> == ORACLE, 

<db-info> := Oracle SID 

<db-params> := -server server-name 

-dbo.login dbo-name [dbo-pwd] 

[-con.nectopts connect-options] 

if <vendor> == SQL_SERVER, 

<db-info> := Physical Database Name 

<db-params> := -server server-name 

-dbo.login dbo-name [dbo-pwd] 

-rw.login rw-login [rw-pwd] 

[-con.nectopts connect-options] 

[-ro.login ro-login [ro-pwd]] (schema repo only)




Example fo r DB2 


multiutil mkreplica -import -site SITE2 -repository DB2repoDB -vendor DB2 -server DB2HOST -dbo DB2repoUser DB2pass -con "" -database DB2userDB -vendor DB2 -server DB2HOST -dbo DB2dbUser DB2pass -con "" C:\temp\packet.xml


Example for Oracle 


multiutil mkreplica -import -site SITE2 -repository ORASID -vendor ORACLE -server ORAHOST -dbo ORArepoUser ORApass -con "LOB_TYPE=CLOB" -database ORASID -vendor ORACLE -server ORAHOST -dbo ORAdbUser ORApass -con "LOB_TYPE=CLOB" C:\temp\packet.xml


Example for SQL SERVER 


multiutil mkreplica -import -site SITE2 -repository SQLrepoDB -vendor SQL_SERVER -server SQLHOST -dbo SQLrepoUser SQLpass -rw SQLrepoUser SQLpass -ro SQLrepoUser SQLpass -con "" -database SQLuserDB -vendor SQL_SERVER -server SQLHOST -dbo SQLdbUser SQLpass -rw SQLdbUser SQLpass -con "" C:\temp\packet.xml

Failure to create back-up copies can limit your ability to recover from a an upgrade failure, design change issues or other unforeseen failures.
