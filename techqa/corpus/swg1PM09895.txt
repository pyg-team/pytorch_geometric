PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  Users of the default messaging provider     *
   *                  for IBM WebSphere Application Server        *
   *                  Version 6.1                                 *
   ****************************************************************
   * PROBLEM DESCRIPTION: If an SIBus messaging engine (ME) is    *
   *                      started on a server that is not the     *
   *                      preferred server and moves to the       *
   *                      preferred server when that becomes      *
   *                      available, dynamic config updates are   *
   *                      attempted on the original server as     *
   *                      well as the current server. The         *
   *                      updates on the original server cause    *
   *                      CWSID0003E messages in the              *
   *                      SystemOut.log file.                     *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   This APAR applies when an ME has started on a non-preferred
   server and then been stopped as the preferred server becomes
   available.
   When a dynamic configuration reload is attempted after a
   change, a CWSID0003E error is seen in the SystemOut.log. For
   example, after addition of a new queue, the SystemOut.log shows
   CWSID0003E: An internal error occurred; reason:
   com.ibm.ws.sib.admin.impl.JsLocalizer: Unable to modify
   Message Point: uuid=<uuid> targetUuid=<targetUuid>
   and the FFDC shows:
   Stack Dump =
   com.ibm.websphere.sib.exception.SINotPossibleInCurrentConfigurat
   ionException: CWSIP0211E: Messaging engine <ME-Name> on bus
   <Bus-Name> is not started.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this APAR changes the dynamic configuration code
   of the SIB Service such that SIB does not attempt to
   dynamically update the config for a messaging engine that
   started and then stopped. The fix must be applied in
   conjunction with PK84660 [http://www-01.ibm.com/support/docview.wss?uid=swg1PK84660].
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 6.1.0.33.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
