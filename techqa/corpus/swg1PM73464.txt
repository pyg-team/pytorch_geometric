APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  - change session id timeout to 2 minutes via WAS Admin console
   -> server -> portal -> session mgmt
   - restart server
   - login and open toolbar via 'edit'
   - wait for session timeout
   - click something in the toolbar - e.g. content tab
   - A new session will be created that will never timeout.
   Note:  If this condition is occurring the session count in the
   Portal server will gradually rise.  If anonymous sessions are
   enabled the count will rise more quickly.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  Can not get browsers to timeout if shelf is open
   
   - change session id timeout to 2 minutes via WAS Admin console
   -> server -> portal -> session mgmt
   - restart server
   - login and open toolbar via 'edit'
   - wait for session timeout
   - click something in the toolbar - e.g. content tab
   - now the user has a session that will not timeout anymore!
   
   
    
   
   

PROBLEM CONCLUSION
 *  Take care of session timeout.
