DIAGNOSING THE PROBLEM
Compare library files in both the <twshome>/tws/bin and /usr/Tivoli/TWS/ICU/3.4.1/lib directories. Use checksum to compare or take note of different file sizes, for example. 

NOTE: This can be caused by any corrupted library file. To determine what library files the netman binary needs, use the following command as the twsuser from the <twshome>/bin directory: 

$ldd netman 

If there is a problem (for example, with libicudata34.a), something similar to the following will be displayed: 

bash-3.00$ /usr/maestro/tws/bin $ ldd netman

RESOLVING THE PROBLEM
There are several recommendations to resolve this issue: 

1. If the file sizes are different in the <twshome>/tws/bin and /usr/Tivoli/TWS/ICU/3.4.1/lib directories, rename the suspected library file in /usr/Tivoli/TWS/ICU/3.4.1/lib to /usr/Tivoli/TWS/ICU/3.4.1/lib/<library_file_name>.old and copy the library file from <twshome>/bin into /usr/Tivoli/TWS/ICU/3.4.1/lib. 

2. If you are unsure of the integrity of the library file(s) in both directories mentioned above, rename the suspect file(s) to <library_file_name.old> and copy the file(s) from a working AIX FTA (same TWS version) to the system having the issue. 

3. Uninstall TWS base code and reinstall, then stop Netman and install the TWS Fix Pack.
