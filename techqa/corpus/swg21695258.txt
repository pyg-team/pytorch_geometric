DIAGNOSING THE PROBLEM
Make sure that stproxyconfig.xml is on it's location i.e The deployment manager's stproxyconfig.xml file is typically located in 
the following directory: 

WebSphere_AppServer_install_root/profiles/Deployment_Manager_Profile_Nam
e/config/cells/Cell_Name/nodes/Node_Name/servers/STProxyServer/ 

For example, on IBM AIXÂ® or Linux:

des\STProxyNode1\servers\STProxyServer 

If that is not the case then check formatting in stproxyconfig.xml

RESOLVING THE PROBLEM
Checked stproxyconfig.xml and found the following issue with "clusterlist" tag
<clusterlist>xyx.server.com<clusterlist/>

Changed it to <clusterlist>xyx.server.com</clusterlist>
Saved the file, Synched all nodes and restarted the server. Issue was resolved
