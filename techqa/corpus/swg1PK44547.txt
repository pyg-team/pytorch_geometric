ERROR DESCRIPTION
 *  A SOAP message is sent by WebServices client application that
   contains ws-addressing fields.  The wsa addressing headers are
   processed in a handler both on the SOAP request and SOAP
   response processing in WebSphere Appserver.  The wsa fields are
   unaffected by the wsa addressing processing on the request.
   But when the response processing is done, the it does not
   preserve the wsa addressing fields.
   .
   Here is an example of the request sent by the webservices client
   with the wsa fields in it
   <soapenv:Envelope
   xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"
   xmlns:soapenc="http://schemas.xmlsoap.org/soap/encoding/"
   xmlns:xsd="http://www.w3.org/2001/XMLSchema"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xmlns:wsa="http://schemas.xmlsoap.org/ws/2004/08/addressing"
   xmlns:hl7="urn:hl7-org:v3">
      <soapenv:Header>
         <wsa:MessageID>uuid:123</wsa:MessageID>
         <wsa:Action>Service1/QUPA_IN01</wsa:Action>
         <wsa:To>http://to</wsa:To>
         <wsa:From>
            <wsa:Address>http://from</wsa:Address>
         </wsa:From>
         <hl7:communicationFunctionRcv>
            <hl7:device>
               <hl7:id extension="1234" root="1.2.3.34"/>
            </hl7:device>
         </hl7:communicationFunctionRcv>
         <hl7:communicationFunctionSnd>
            <hl7:device>
               <hl7:id extension="6789" root="1.2.3.34"/>
            </hl7:device>
         </hl7:communicationFunctionSnd>
         <wsa:ReplyTo>
            <wsa:Address>http://from</wsa:Address>
         </wsa:ReplyTo>
      </soapenv:Header>
      <soapenv:Body>
         <abc/>
      </soapenv:Body>
   </soapenv:Envelope>
   .
   SAAJ code to get the wsa data fails to find the wsa fields when
   processing the SOAP response.  The following SAAJ call will not
   find the wsa fields even though they were sent on the SOAP
   message.  The message is the SOAPMessage.  i is an Iterator on
   the elements in the SOAP header.
   
       i = message.getSOAPHeader().getChildElements();
       obj = i.next();
       if (obj instanceof org.w3c.dom.Element)
       {
          elem = (SOAPElement)obj;
   
          if (elem.getLocalName().equals("MessageID")) wsaMessageId
   = elem.getFirstChild().getNodeValue();

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: All IBM WebSphere Application Server         *
   *                 version 6.1 users of Web Services Addressing *
   ****************************************************************
   * PROBLEM DESCRIPTION: Web Services Addressing service         *
   *                      removed addressing information from a   *
   *                      SOAP message                            *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   A SOAP message is sent by a WebServices client application
   that  contains ws-addressing fields.  The wsa addressing
   headers are processed in a handler both on the SOAP request
   and SOAP response processing in WebSphere Application Server.
   The wsa fields are unaffected by the wsa addressing processing
   on the request. But when the response processing is done, the
   addressing handler removes the wsa addressing fields.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Class WSAddressingClientHandler will be changed so that the
   Web Services addressing fields are only removed when
   necessary.
   The fix for this APAR is currently targeted for inclusion in
   fixpack 6.1.0.11. Please refer to the recommended updates page
   for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
