ERROR DESCRIPTION
 *  Approver Initials:    RL
   Severity:             3
   Reported Release:     623
   Compid:               5724C04AS Tivoli Application Support
   
   PROBLEM DESCRIPTION:
   Running itmasi.sh silent from SQL 6.2.3 FP1 and installing TEMS
   support on RHEL 6 64 bit. Did install setupLinux.bin as
   required. Still get wrong libkbb.sp loaded by the ksminst.
   TEPS support did install just fine (Running TEPS and TEMS on
   the same server)
   
   RECREATE INSTRUCTIONS:
   Running itmasi.sh silent from SQL 6.2.3 FP1 CD-Image onto lx8266
   machine.
   
   itmasi.sh $CANDLEHOME
   
   
    
   
   

LOCAL FIX
 *  If $CANDLEHOME/lx8266/ms/lib/libkbb.so does not exist,
   then copy libkbb.so from $CANDLEHOME/tmaitm6/lx8263
   before performing the rest of the work-arounds steps.
   
   cd /opt/app/Tivoli/itm
   cp tmaitm6/lx8263/lib/libkbb.so lx8266/ms/lib
   
   The rest of the work-around for this problem is:
   
   Step1: Enter into $CANDLEHOME
   Step2: Backup libkbb.so under $CANDLEHOME/tmaitm6/lx8266/lib/
   Step3: Copy libkbb.so from $CANDLEHOME/lx8266/ms/lib/ to
          $CANDLEHOME/tmaitm6/lx8266/lib/
   
   cd /opt/app/Tivoli/itm
   cp tmaitm6/lx8266/lib/libkbb.so tmaitm6/lx8266/lib/libkbb.so.bak
   cp lx8266/ms/lib/libkbb.so tmaitm6/lx8266/lib
   
   Step4: Re-run the ifix installer.
   
   Step5: Restore libkbb.so under $CANDLEHOME/tmaitm6/lx8266/lib
   cd /opt/app/Tivoli/itm
   cp tmaitm6/lx8266/lib/libkbb.so.bak tmaitm6/lx8266/lib/libkbb.so
