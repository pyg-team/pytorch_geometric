 *  IBM Rational AppScan 7.8.0.2 does not include the 'Host:' reques

LOCAL FIX

PROBLEM SUMMARY
 *  IBM Rational AppScan 7.8.0.2 does not include the "Host:"
   request-header line during an HTTP/1.1 CONNECT request to a
   proxy in order to access an HTTPS site.
   
   As a result some proxies may return a 400 error code to the
   request as per RFC 2616's specification.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The "Host:" header was added to the HTTP connect request.
