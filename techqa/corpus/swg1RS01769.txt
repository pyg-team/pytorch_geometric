ERROR DESCRIPTION
 *  When running IloTaskController.getTaskState() from Processing
   Service on another users' instance than the one which created
   the solve task, returned status is always NOT_STARTED, whereas
   getState() correctly returns READY,RUNNING,COMPLETED.
   
   
    
   
   

LOCAL FIX
 *  For ODM Entreprise 3.7 the solution is to use:
   ilog.odm.processingsvc.IloJobController.getState()
   instead of:
   ilog.odm.processingsvc.IloTaskController.getTaskState().
   
   We have updated the java classes in your example:
   
   TaskControllerTest\Nurses_java\src\customAction\
   RemoveNurseActionHandler.java
   and
   TaskControllerTest\TestTaskController\src\
   TestTaskController.java
   to use both the getState() and getTaskState() functions and show
   a comparison of the function behaviour.
   
   Please check how the task status line is displayed:
   "taskState= " + taskCtrl.getTaskState() + "
   state="+taskCtrl.getState()
   +" phase="+taskCtrl.getCompletionPhase()
