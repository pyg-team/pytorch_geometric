Title: IBM Tivoli Integrated Portal Poodle vulnerability CVE-2014-3566 - United States

Text:
TIPL2; TIPL2SEC; Poodle vulnerability CVE-2014-3566 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Tivoli Integrated Portal and the products deployed with it are effected by SSLv3 Poddle vulnerability CVE-2014-3566 

CAUSE
See below technote for information about this vulnerability. 

http://www-01.ibm.com/support/docview.wss?uid=swg21687173 [http://www-01.ibm.com/support/docview.wss?uid=swg21687173]


RESOLVING THE PROBLEM
This vulnerability is fixed in Websphere FP 37 but this version of Webshpere is not included in the current version of Tivoli Integrated Portal (version 2.2.0.15). TIP 2.2.0.17 will include eWAS 7.0.0.39 which will have the fix for this vulnerability. 


To address this issue you have below choices:
1. Install iFIX mentioned in above technote, or
2. Implement below work around, or
3. Wait till TIP 2.2.0.17 is available. 

Steps to implement work around are as follows:

1. Go to Security > SSL certificate and key management > SSL configurations 
2. The collection of all SSL configurations is listed there. For each SSL configuration in the list, the SSL protocol need to be modified to use TLS. 
3. Select NodeDefaultSSLSettings then click Quality of protection (QoP) settings under Additional Properties on the right. 
4. Set the "Protocol" drop down to "TLS" . 
5. Apply/Save. 
6. Restart TIP
