APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Users Affected:
   ITCAM for Application Diagnostics 7.1 users who monitor eXtreme
   Scale standalone environment with KeyStoreLoginAuthenticator
   security settings.
   
   ---------------------------------------------
   Problem Description:
   When WebSphere eXtreme Scale standalone environment is
   configured to use KeyStoreLoginAuthenticator security settings,
   then monitoring agent is unable to connect to catalog servers.
   
   The following exception can be seen in the logs:
   java.lang.SecurityException: Supplied credential is null
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ***************************************************************
   * USERS AFFECTED: ITCAM for Application Diagnostics 7.1 users
   * who try to configure monitoring of Websphere eXtreme Scale
   * standalone environment with KeyStoreLoginAuthenticator
   * security settings.
   ***************************************************************
   * PROBLEM DESCRIPTION:
   * Cannot configure monitoring for standalone WXS environment
   * with KeyStoreLoginAuthenticator security enabled.
   ***************************************************************
   * RECOMMENDATION: Use ITCAM for AD 7.1.0.3 interim fix 5
   * (7.1.0.3-TIV-ITCAMfAD-IF0005) or newer to resolve this issue.
   ***************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix for this APAR corrects the way of passing credentials
   while connecting to secured standalone catalog server.
   
   The fix for this APAR is contained in the following maintenance
   packages:
   
     | interim fix | 7.1.0.3-TIV-ITCAMAD-IF0005
