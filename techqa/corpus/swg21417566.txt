Title: IBM MustGather: Collecting IBM WebSphere DataPower SOA Appliance diagnostic information for hash or digital signature-related issues. - United States

Text:
MustGather; MustGatherDocument TECHNOTE (FAQ)

QUESTION
 What diagnostic information should I collect for a hash or signature-related problem on IBM WebSphere DataPower SOA Appliance? 

CAUSE
There could be various problems: 

 * Wrong certificate used on the validation side. 
 * Canonicalization issue.

ANSWER
The following information describes the documentation needed by IBM support to investigate this problem scenario. Gather as much information listed in this document as possible. 
1. Follow the instructions at this link first on collecting IBM WebSphere DataPower SOA Appliance Security and AAA Information. [http://www.ibm.com/support/docview.wss?uid=swg21404383] [http://www.ibm.com/support/docview.wss?uid=swg21404383]

2. Additionally, run a test in your environment and collect the following information. Make sure to collect all data from a single test run such that the timestamps can be correlated. 

 * Debug log files from DataPower appliance 
 * Probe capture from the DataPower appliance 
 * Export of the config from DataPower appliance 
 * Steps on how the input was created
