APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When setting up Policy Output Parameters for Array Of Impact
   Object fields one is unable to select an individual field for
   deletion by the first column select box.  However, if one
   selects the field as Key Field then the first column select box
   is selected and one can delete the field (so there is a
   workaround).  Reported on FP6 and FP7.
   
   Replication steps:
   Create a simple test Policy
   Click on "Configure Policy Settings"
   In the "Policy Output Parameters" section click on "New"
   Select "Array Of Impact Object" for the format
   Click on "Schema Definition" button
   Click on "Add New Field" button
   Provide a field name and click "OK"
   Click on "Add New Field" button
   Provide a different field name and click "OK"
   Now try to select an individual field to delete it by clicking
   on the select box at the start of the line.  Nothing happens.
   
   Select the Key Field tick box for a field and the first column
   selection box is also ticked and one can delete the field this
   way.
   
   
    
   
   

LOCAL FIX
 *  As a workaround: select the field as Key Field then the first
   column select box is selected and one can delete the field.
