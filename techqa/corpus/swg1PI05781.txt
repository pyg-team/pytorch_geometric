Title: IBM PI05781: Unable to install RSA V8.5.5 when V8.5.1 + V8.5.5 media used as local repositories - United States

Text:

APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Abstract:
   Attempts to install IBM Rational Software Architect V8.5.5 when
   V8.5.1 + V8.5.5 media used as local repositories result in
   throwing the error - CRIMC1020E.
   
   Problem:
   In case Rational Software Architect  v8.5.1 and v8.5.5 media are
   set as local repositories rather than remote repositories, then
   IBM Rational Software Architect V8.5.5 installation will fail
   with the following errors.
   CRIMC1020E: Plugin
   com.ibm.check.was.extension.install_1.1.0.v20110601_2118 not
   found in C:\RSA8.5.1\disk1, and C:\RSA8.5.5\disk1
                           Plugin
   com.ibm.rational.cdi.check64bit_1.0.0.v20110601_2118 not found
   in C:\RSA8.5.1\disk1, and C:\RSA8.5.5\disk1
   
   It has been identified as the RSA 8.5.5 media defect.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  there was an old version of
   com.ibm.check.was.extension.install_1.1.0 in cdi-common and used
    in 855, when RSA upgrading, it looks for the new one
   v20110601_2118 which included in 851 version, can't find it,
   throw out an error.
   
   
    
   
   

PROBLEM CONCLUSION
 *  com.ibm.rational.cdi.check64bit_1.0.0.v20140222_1420 was
   generated to be used in the RSA upgrading.
   
   
    
   
   

TEMPORARY FIX
