   SA 3.3:
   =======
   This readme file describes how to install the APAR OA42205 for
   IBM Tivoli System Automation for z/OS (SA z/OS) monitoring agent
   for the Tivoli Enterprise Portal Server TEP(S).
   
   You must have installed an ITM environment V6.1 or higher and
   the related SA Application support.
   The following packed files for the SA z/OS TEP exploitation are
   available after your SMP/E installation:
   
       ING.SINGPWS1(KAH42205)
       KAH42205 is the packed file for Windows or Linux.
   
   To install the fix, follow the instructions below.
   1. Download the member (KAH42205) from the data set on the host
      system to your workstation in binary mode using, for example,
      FTP.
   2. Rename KAH42205 to KAH42205.zip on the workstation.
   
   Windows:
   --------
   3. unpack with an appropriate tool (WINZIP or PKZIP)
   The unpacked zip file KAH42205 contains the parts for APAR
         OA42205.
            kah_resources.jar        (English version)
            kah_resources_ja.jar     (Japanese version)
            readme_OA42205.txt
      4. Stop the Tivoli Enterprise Portal Server (TEPS)
      5. save or rename the related old .jar files and copy the new
         .jar files into
         C:\IBM\ITM\CNB\classes
         C:\IBM\ITM\CNP\
      6. Restart the Tivoli Enterprise Portal Server (TEPS)
      7. Reconfigure the Tivoli Enterprise Portal Server (TEPS)
      8. Execute the C:\IBM\ITM\CNPS\BuildPresentation.bat file
      9. If you use the Tivoli Enterprise Portal in browser mode,
         clear the Java Plug-in cache to load the new jar files
         when restarting the browser client.
   
   Linux:
   ------
   3. unpack to the current directory with the command:
      unzip KAH42205.zip
      The unpacked file KAH42205 contains the parts for APAR
      OA42205.
         kah_resources.jar        (English version)
         kah_resources_ja.jar     (Japanese version)
            readme_OA42205.txt
   4. Stop the Tivoli Enterprise Portal Server (TEPS)
   5. save or rename the related old .jar files and copy the new
      .jar files in .../opt/IBM/ITM/ls3266/cw/classes
   6. Restart the Tivoli Enterprise Portal Server (TEPS)
   7. Reconfigure the Tivoli Enterprise Portal Server (TEPS)
   8. Execute the command: /opt/IBM/ITM/bin/itmcmd execute cq
      InstallPresentation.sh
   9. If you use the Tivoli Enterprise Portal in browser mode,
      clear the Java Plug-in cache to load the new jar files when
      restarting the browser client.
   
   NOTE:
   -----
   If you did not have previously installed the language pack
   then you do not have to copy the Japanese resource bundle
   kah_resource_ja.jar.
   Just copy the English version kah_resource.jar.
