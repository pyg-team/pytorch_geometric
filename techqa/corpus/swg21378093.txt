Title: IBM OracleEBS EJB 2.1 project publish failed to WebSphere Application Server 6.1 or 7.0 - United States

Text:
RN7.5.2; SRX TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 java.lang.NoClassDefFoundError during publishing OracleEBS EJB 2.1 project to WebSphere Application Server 6.1 or 7.0 

SYMPTOM

CAUSE
 

The ffdcSupport.jar and other dependency files have not been added to the classpath of EJB project.


RESOLVING THE PROBLEM
 

 

1. If the target runtime server is WebSphere Application 6.1, you need to add these dependency files which are under base_61_stub\adapterJars folder to the classpath of EJB project: 

 * aspectjrt.jar 
 * ffdc.jar 
 * ffdcSupport.jar 
 * icu4j_3_4_1.jar

2. If the target runtime server is WebSphere Application7.0, you need to add those dependency files which are under base_70_stub\adapterJars folder:  * aspectjrt.jar 
 * ffdc.jar 
 * ffdcSupport.jar 
 * com.ibm.ffdc.jar 
 * com.ibm.icu.jar
