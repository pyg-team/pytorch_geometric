CAUSE
SVN Repository corrupt

RESOLVING THE PROBLEM
To rebuild the SVN repository on your Secondary server : 

1. Move the impact/etc directory to a backup directory 
2. Move the impact/policy directory to a backup directory 
3. Change to the $NCHOME/impact directory 
- run: $NCHOME/impact/platform/<arch>/svn/bin/svn checkout 
file:///$NCHOME/impact/platform/ [file:///$NCHOME/impact/platform/]<arch>/svn/repository/trunk/etc 
- run: $NCHOME/impact/platform/<arch>/svn/bin/svn checkout 
file:///$NCHOME/impact/platform/ [file:///$NCHOME/impact/platform/]<arch>/svn/repository/trunk/policy 

This will recreate the policy and etc directories from SVN and will restore everything in the .svn directories. After do this, start the server and it will replicate from the Primary.
