APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The signature NumberFormat.format(java.math.BigDecimal) was
   part of the icu4j library version 3.4.1 and it was not
   deprecated. In the version 3.4.5 jar the signature has been
   removed  causing  problems for the BPC Explorer as part of
   WebSphere Process Server 61.
   
   Unfortunately, the BPC Explorer uses this signature. If a
   WebSphere Process Server server is upgraded the Enterprise
   Application "BPC Explorer" need not be upgraded as well. In
   which case, the already installed BPC Explorer would use a
   non-existing signature.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  All users of WebSphere Application Server   *
   *                  6.1.0.3 and later with applications that    *
   *                  use the ICU4J 3.4.5 library.                *
   ****************************************************************
   * PROBLEM DESCRIPTION: The exception                           *
   *                      java.lang.NoSuchMethodError:            *
   *                      com.ibm.icu.text.NumberFormat.format    *
   *                      (java.math.BigDecimal) can be thrown.   *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The ICU4J 3.4.5 library distributed with the WebSphere
   Application Server 6.1.0.3 and later contains the ICU4J 3.4.5
   library from the Eclipse SDK. The
   NumberFormat.format(java.math.BigDecimal) method does not
   exist in that version of ICU4J. An application compiled with
   the previous version of ICU4J distributed with the WebSphere
   Application Server may be using this method and when run on a
   later release of Application Server may encounter the exception
   described above.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The missing method has been added back into the ICU4J 3.4.5
   library included with the WebSphere Application Server.
   
   The fix for this APAR is currently targeted for inclusion in
   fix pack 6.1.0.19.  Please refer to the Recommended Updates
   page for delivery information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
