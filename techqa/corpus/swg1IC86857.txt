 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  Tivoli Integrated Portal is installed as a component with TPC.
   Logs from this embedded installation are not collected by the
   TPC Service tool (service.bat/service.sh).
   The service tool should collect the following output and logs:
   Operating system Report the current level
   AIX&#65449;   oslevel -r
   Windows&#65449;  "systeminfo" output
   Linux&#65449;   uname  arv
   Output of  java  version  command.
   Windows platform:
   Collect the output of the following commands:
   C:\Program Files\IBM\Common\acsi\bin\de_version.cmd
   C:\Program Files\IBM\Common\acsi\bin\listIU.cmd
   C:\Program Files\IBM\Common\acsi\bin\de_lsrootiu.cmd
   Collect the following files:
   The product installation log from the installation user's home
   directory - "IA-TIPInstall-00.log" and the similarly named
   uninstall log from the same directory
   Contents of the Deployment Engine (DE) logs directory
   C:\Program Files\IBM\Common\.acsi\logs
   Contents of the TIP logs and eWAS server logs directories
   <TIP HOME>\logs
   <TIP HOME>\profiles\TIPProfile\logs\
   Contents of the Common Object Installer (COI) _uninst directory
   <TIP HOME>\_uninst
   AIX/Linux Platform:
   Collect the output of the following commands:
   /usr/ibm/common/acsi/bin/de_version
   /usr/ibm/common/acsi/bin/listIU.sh
   /usr/ibm/common/acsi/bin/de_lsrootiu.sh
   Collect the following files:
   The product installation log from /root - "IA-TIPInstall-00.log"
   and the similarly named uninstall log from the same directory.
   Contents of the Deployment Engine (DE) logs directory
   /usr/ibm/common/acsi/logs
   Contents of the TIP logs and eWAS server logs directories
   <TIP HOME>/logs
   <TIP HOME>/profiles/TIPProfile/logs/
   Contents of the Common Object Installer (COI) _uninst directory
   <TIP HOME>/_uninst

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All TPC 5.1.0 users who have TIP installed in a custom       *
   * location and run service.bat tool                            *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * If TIP is installed in custom path, service.bat tool does    *
   * not collect the TIP installation logs.                       *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Upload the tip_path\logs manually.                           *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  We need to use absolute paths for TIP installation directory
   instead of relative paths to TPC install location.  The fix for
   this APAR is targeted for the following maintenance package:
   
   | fix pack | 5.1.1-TIV-TPC-FP0001 - target March 2013
   http://www-01.ibm.com/support/docview.wss?&uid=swg21320822 [http://www-01.ibm.com/support/docview.wss?&uid=swg21320822]
   
   The target dates for future fix packs do not represent a formal
   commitment by IBM.  The dates are subject to change without
   notice.
