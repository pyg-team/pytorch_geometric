CAUSE
The problem was found to be a typo in wimconfig.xml.

RESOLVING THE PROBLEM
 

The problem was found to be a typo which was introduced when manually editing the following section in wimconfig.xml: 


 *  <config:groupConfiguration> 
   <config:memberAttributes dummyMember="uid=dummy" 

name="uniqueMember" objectClass="groupOfUniqueNames" scope="direct"/>  *  <config:memberAttributes name="nsRole" objectClass="ldapsubentry" scope="direct" /> 
   <config: membershipAttributes name="nsRole" scope="direct" /> 
   </config:groupConfiguration> 


The correct configuration parameter is config:membershipattribute (singular); the incorrect example notes this as config:membershipAttributes. 

NOTE: Whenever you see the following message, it is most likely cause by an incorrect setting in the wimconfig.xml file:  * Exception while loading dynamically re-ordered service class com.ibm.wps.services.puma.PumaStoreService
