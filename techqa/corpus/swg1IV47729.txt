ERROR DESCRIPTION
 *  Same problem as APAR IZ44233 [http://www-01.ibm.com/support/docview.wss?uid=swg1IZ44233].
   
   Reproduced on 7.5.0.2, 7.5.0.4
   
   Steps to reproduce (from original APAR):
   
   MEA: enable object structure MXASSET and all necessary steps.
   Note ASSETSPEC is part of the object structure.
   Table Domain:
   1. Go to Domains and create a new TABLE domain called
   MYASSETDOMAIN(ex)
   2. In the object area, insert a new row and give "ASSET".  You
   can give PERSON or any other maximo tables. The object "ASSET"
   entered here is not related to the Asset app we will test.
   Save.
   Classification:
   1. Go to Classification app, create a new classification called
   MYDOMAINCLASS(ex).
   2. In use with table window, new row, enter ASSET
   3. In Attributes table window, new row, enter MYATTR. Click Yes
   to the question Attribute MYATTR does not exist....
   4. Enter the following values:
       Data Type: TABLE
       Domain: MYASSETDOMAIN
       Lookup Attribute: ASSETNUM
       Lookup Name: ASSET
   5. Save.
   
   Asset App:
   1. Go to Asset App and create a new asset called MYASSET, do not
   save yet.
   2. Click Specification tab, and in the Clasification field, type
   MYCLASS. In Asset app, you have to type so the table window will
   populate.
   3. In the Table Value field of the attribute MYATTR, select a
   valid assetnum such as 7500.
   4. Save the asset.
   
   XML output file:
   Change your output file to Sync/Sync. Also
   change  ASSET.<PURCHASEPRICE>0.0</PURCHASEPRICE> to
   <PURCHASEPRICE>18.0</PURCHASEPRICE>
   Change ASSETSPEC <TABLEVALUE>7500</TABLEVALUE> to
   <TABLEVALUE>11200</TABLEVALUE>
   
   External System: Import the xml back.
   
   Asset App: Bring up MYASSET and you should see purchase price 18
   and table value in Specification tab to be 11200.
   
   If you can succesfully complete the above steps without seeing
   the TABLE VALUE does not validate error, then it works.
   
   Additionally, in the TABLEVALUE tag in the xml, try to give an
   invalid ASSETNUM such as XYZZ.  The import should fail because
   the value XYZZ does not exist in the ASSET table derived from
   the table domain MYASSETDOMAIN.   You should see TABLEVALUE XYZZ
   is not valid.
   ex..Change ASSETSPEC <TABLEVALUE>XYZZ</TABLEVALUE>
