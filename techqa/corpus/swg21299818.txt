You would like to know what documentation you must collect (MustGather) so that the WebSphere Message Broker Support team can diagnose your problem. If you gather this documentation before contacting support it will expedite the troubleshooting process, and save you time.

 * WebSphere Message Broker version, release, and maintenance level 
 * Operating system version, release, and maintenance level 
 * Related products version, and release levels if applicable 
 * Components installed 
 * Number of machines (include operating systems) 
 * Number of brokers (include operating systems)

General 
Required (MustGather) data:  *  If the problem is associated with connectivity, you can collect the CMP and MQ Java client trace as per following steps

1. open Toolkit > Window > Preferences > broker Development > Broker Runtime 
2. Select both CMP API and MQ Java client trace and provide the location of these files 
3. Click Apply 
4. Recreate the problem and provide the 2 trace files.  *  [http://www.ibm.com/support/docview.wss?rs=849&uid=swg21295454] [http://www.ibm.com/support/docview.wss?rs=849&uid=swg21295454]If the problems are associated with toolkit functionality, send us the snapshot of the problematic scenario. 
 * .metadata directory within the workspace
