ERROR DESCRIPTION
 *  The help resources jar that is included in the Self Describing
   Agent (SDA) support files is not signed with a current
   certificate.
   Environment: KRZ - 6.3.1.2-TIV-ITM_KRZ-IF0003
   OS: Linux
   Log files:
   No logs provided. Customer received "Failed to validate
   certificate. The application will not bet executed" when
   logging into TEPS, along with
   ERROR:
   java.security.cert.CertificateException: Found unsigned entry in
   resource:http://lrv140wm.europe.intranet:15200/classes/krz_reso
   urces.jar
   DETAILED RECREATION PROCEDURE:
   1) Upgrade to 6.3.1.2-TIV-ITM_KRZ-IF0003 (using the SDA option)
   2) Launch TEPS
   
   
    
   
   

LOCAL FIX
 *  Use the manual upgrade instead of the SDA option.

 *  The help resources jar that is included in the Self Describing
   Agent support files is not signed with a current certificate.
   When launching the portal using the Java(R) Web Start
   client,&#160;the message "Failed&#160;to&#160;validate&#160;certificate.&#160; The
   application&#160;will&#160;not&#160; be&#160;executed"&#160;is received with details:
   "java.security.cert.CertificateException:&#160;Found&#160;unsigned&#160;entry
   in resource:
   http://<your.itm.server>:15200/classes/krz_resources.jar"
