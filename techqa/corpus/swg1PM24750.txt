PROBLEM SUMMARY
 *  About IBM Lotus Web Content Management cumulative fix strategy
   
   All the fixes or maintenance changes implemented on a specified
   ILWCM (IBM Lotus Web Content Management) version are bundled
   together into a single package named as "Cumulative fix". Such
   "Cumulative Fixes" (or CFs) are released at regular intervals so
   that customers can apply these on their environments. This will
   ensure that they have the latest corrective or adaptive changes
   incorporated into that version of ILWCM.
   
   The latest available cumulative fix always supercedes the
   previous cumulative fixes. That is, the current CF contains all
   the fixes from all previous CFs and a new set of fixes
   implemented after that. At any point in time, a customer only
   needs to apply the latest released CF to ensure that they have
   all the maintenance changes implemented for that ILWCM version.
   Cumulative fix Details
   
   This is cumulative Fix 38 for IBM Web Content Management version
   6.1.5.x on Websphere Portal version WP v6.1.5.x
   
   Affected Users: All ILWCM Users
   Platforms: All
   Fix pack: This fix will be integrated into fixpack WP v6.1.5.2
   Requisites
   
   Pre-requisite Portal Component: WCM 6.1.5.0 or WCM 6.1.5.1
   Pre-requisite interim fixes: None.
   Co-requisite fixes: PM10931 [http://www-01.ibm.com/support/docview.wss?uid=swg1PM10931](PZN - for 6.1.5.0 only).
