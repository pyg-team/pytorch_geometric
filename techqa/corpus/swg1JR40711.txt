APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After enabling the store and forward feature, the LTPA token may
   expire within the timeout leniency after recovering the
   environment.
   The ifix will solve this issue to allow that the LTPA token can
   be refreshed within the timeout leniency.
   
   
    
   
   

LOCAL FIX
 *  N/A
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:  Customers using LTPA token, Failed          *
   *                  Event Manager (FEM) and store & forward     *
   *                  feature in WebSphere Process Server (WPS).  *
   ****************************************************************
   * PROBLEM DESCRIPTION: LTPA token expired with timeout         *
   *                      leniency when enabling the store and    *
   *                      forward feature.                        *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   After enabling the store and forward feature, the LTPA token
   may
   expire within the timeout leniency after recovering the
   environment.
   
   
    
   
   

PROBLEM CONCLUSION
 *  The fix resolves the issue by allowing LTPA token be
   refreshed within the timeout leniency.
   
   The fix will enable the failed event manager to store the
   message with the extend security header and the user id as
   same. When resubmit the message. The header will be restored
   and the LTPA token can be refreshed by SCA related code then.
