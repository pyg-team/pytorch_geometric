APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After attempting to confirm multiple EVERY jobs to SUCC via DWC
   simultaneously only one of the set will show the state SUCC
   from the DWC view.  The following message is returned if the
   action is retried for the instances that do not show SUCC:
   "AWSUI0796E The specified action cannot be performed.
   Reason: AWSJSY404E The Symphony plan operation
   "confirmJobInstance" could not be completed because an error
   has occurred while accessing the Symphony plan. The error
   message is: "AWSBIN076E The operation cannot be performed. The
   job or the current instance of the job is in an incorrect
   state.".
   Looking at the involved jobs via conman show all with the state
   set to SUCC.  Also, confirming the state via conman does not
   reflect a SUCC state in DWC.
   
   
    
   
   

LOCAL FIX
 *  In the state where job states via conman do not agree with the
   job states seen via DWC the following command can be launched
   on the MDM to synchronize the job states:
   $ planman resync
