Title: IBM Login Error You cannot login at this time.  Contact the system administrator - United States

Text:
TPAESECURITY; TPAELDAP; loginerror.jsp; NoLogin TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Login Error trying to hit the Maximo URL 

SYMPTOM
Receive this URL rather than your normal URL:
http://hostname/maximo/webclient/login/loginerror.jsp?group=system&key=NoLogin [http://hostname/maximo/webclient/login/loginerror.jsp?group=system&key=NoLogin]


CAUSE
Security has changed in the web.xml files

DIAGNOSING THE PROBLEM
If you see this URL http://hostname/maximo/webclient/login/loginerror.jsp?group=system&key=NoLogin [http://hostname/maximo/webclient/login/loginerror.jsp?group=system&key=NoLogin] rather than your normal login URL see the resolution

RESOLVING THE PROBLEM
Security has some how been changed, check the web.xml files 
IBM\SMP\maximo\applications\maximo\maximouiweb\webmodule\WEB-INF
- IBM\SMP\maximo\applications\maximo\mboweb\webmodule\WEB-INF
- IBM\SMP\maximo\applications\maximo\meaweb\webmodule\WEB-INF
- IBM\SMP\maximo\applications\maximo\maxrestweb\webmodule\WEB-INF

Search these files for useAppServerSecurity and see if the value is set
to 0 or 1. It should be 0 if you are not using application server security (LDAP) and 1 if you are.
