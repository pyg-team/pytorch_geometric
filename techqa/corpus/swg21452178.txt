Title: IBM DFHIS1027 with IPCONN defined as USERAUTH VERIFY running ACF2 - United States

Text:
DFHIS1027 IPIC IPCONN CPMI ACF2 ACF01200 INVALID PASSWORD AUTHORITY CTG9631E ECI_ERR_SECURITY_ERROR -27 websphere message broker 5655M1500 R650 650 R500 500 HCI6500 5655-M15 5655M15 kixinfo retention32 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 You have a CICS Transaction Server for z/OS (CICS TS) IP interconnectivity (IPIC) connection defined with USERAUTH set to VERIFY in the IPCONN definition. You verify the correct userid and password are passed in the request but you receive messages DFHIS1027 and ACF01200. 

SYMPTOM
DFHIS1027 [http://publib.boulder.ibm.com/infocenter/cicsts/v3r2/index.jsp?topic=/com.ibm.cics.ts.messages.doc/cics_mc/dfhis/IS1027.html] Security violation has been detected using IPCONN ipconn and transaction id CPMI by userid userid
ACF01200 INVALID PASSWORD/AUTHORITY
CTG9631E ECI_ERR_SECURITY_ERROR -27


CAUSE
Missing fix for CA ACF2

RESOLVING THE PROBLEM
Apply fix TC8790M for CA ACF2 from [/support/docview.wss?uid=swg21452178&amp;aid=1]CA Technologies [http://www.ca.com/].
