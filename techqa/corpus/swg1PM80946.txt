APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  The following two INSERT INTO EDFMsgRelNo statements in
   update8214to8215.sql script incorrectly pass a '1' character
   value (due to enclosed quotes), for the third column
   DATELENGTHIND, even though this column is defined as SMALLINT.
   
   INSERT INTO EDFMsgRelNo VALUES ('09A','Release 2008 - A','1');
   INSERT INTO EDFMsgRelNo VALUES ('09B','Release 2008 - B','1');
   
   
    
   
   

LOCAL FIX
 *  WTXCQ - WTX00042117
   LV / LV
   Circumvention: Change '1' to 1 in the offending statements.
   
   
    
   
   

PROBLEM SUMMARY
 *  Users affected:
   Websphere TRansformation Extender pack for TMGR
   
   Problem Description:
   Two INSERT INTO EDFMsgRelNo statements in update8214to8215.sql
   script passes a character value (due to enclosed quotes), for
   column DATELENGTHIND, even though this column is defined as
   SMALLINT.
   
   Platforms affected:
   ALL
   
   
    
   
   

PROBLEM CONCLUSION
 *  Removed quotes on the 3rd element of the EDFMsgRelNo entries in
   all sql script files that had them incorrectly

   TMGR 8.2.1.5, TMGR 8.2.1.6 and future releases
