APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When you install the Windows Guardium STAP 10.2.30.12 it may
   crash the server
   .
   This was due to the following hard coded path in the Guardium
   source code for DumpFolder - which was an incorrect assumption.
   .
   C:\Program Files\IBM\Windows S-TAP\Bin\..\Logs
   .
   
   
    
   
   

LOCAL FIX
 *  This is fixed in Windows STAP version 10.2.30.15 and higher.
   An option to set WER_DUMP_FOLDER is also available in Windows
   STAP version 10.2.30.15 and higher.
   .
   All the latest GPU Fix Pack , Sniffer, Security and Agent
   patches are available from Fix Central -
   Note the below 3 url lines needs to be pasted IN FULL to see
   the fixes
   http://www-933.ibm.com/support/fixcentral/swg/selectFixes?parent
   =ibm~Information+Management&product=ibm/Information+Management/I [http://www-933.ibm.com/support/fixcentral/swg/selectFixes?parent=ibm~Information+Management&product=ibm/Information+Management/I]
   nfoSphere+Guardium&release=All&platform=All&function=all
   .
   Some ad-hoc patches may not be placed on Fix Central - In that
   event - or if the patch(es) are not yet shown on Fix Central
   and no later GPU or fixpack is listed then please contact IBM
   Security Guardium Support
