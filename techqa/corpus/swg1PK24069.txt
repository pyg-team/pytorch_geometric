Title: IBM PK24069: SESSION TIMED OUT PAGE DOES NOT HAVE THE OK BUTTON TO GO BACK TO ADMINISTRATIVE CONSOLE LOGIN PAGE. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  When session gets timed out or the same user gets login with
   different browser then we come to a session timed out page but
   there "OK" button is missing to redirect to adminconsole Login
   Page.
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED: WebSphere Application Server version 6.0.2   *
   *                 administrative console users who get logged  *
   *                 out.                                         *
   ****************************************************************
   * PROBLEM DESCRIPTION: The OK button is missing in the Session *
   *                      Invalid page when a session is timed    *
   *                      out.                                    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   ****************************************************************
   The OK button was missing because the condition given was
   causing a conflict which restricted the OK from coming up.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Code has been corrected to allow the OK button to come up.
   
   The fix for this APAR is currently targeted for
   inclusion in fixpack 6.0.2.11.
   Please refer to the recommended updates page for delivery
   information:
   http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980 [http://www.ibm.com/support/docview.wss?rs=180&uid=swg27004980]
   
   
    
   
   

TEMPORARY FIX
