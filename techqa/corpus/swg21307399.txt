Title: IBM How to rename or refactor WebContent folder to a different name - United States

WebContent; root; refactor; rename; 1307399 TECHNOTE (FAQ)

QUESTION
 How do you rename or refactor the IBM® Rational® Application Developer WebContent folder to root? 

CAUSE
The early versions of Rational Application Developer (RAD) v7 did not have a clean way to modify WebContent folder via web project's properties. 
Starting with RAD v7.0.0.4, renaming the WebContent folder can be achieved by refactoring the folder name. However, on Microsoft® Windows® Platforms, refactoring may fail to modify java-output-path attribute in the org.eclipse.wst.common.component file. 

The following bug reports exist that discuss the limitation of previous RAD versions and APAR PK36592:

https://bugs.eclipse.org/bugs/show_bug.cgi?id=102981 [https://bugs.eclipse.org/bugs/show_bug.cgi?id=102981]
https://bugs.eclipse.org/bugs/show_bug.cgi?id=194294 [https://bugs.eclipse.org/bugs/show_bug.cgi?id=194294]


ANSWER
Below are the steps for modifying the WebContent directory to root directory. 

 

For a new web project: 

 1. RAD 7 > File > New Project > Dynamic Web Project
    
    
 2. Next > Create Project Name > Select Target Runtime > Create new EAR project or Select an existing project 
    
    
 3. Next > Select Project Facets
    
    
 4. Next > Change Context Root, Content Directory and Java Source Directory > Finish


For an existing web project: 

Note: Users should backup their projects before performing the following steps. 

 1. RAD v7.0.0.6 > J2EE > Project > Uncheck “Build Automatically”
    This step ensures that RAD does not build your projects while you are modifying the projects’ .classpath file.
    
    
 2. Open Resource Perspective > Open .classpath file with XML Editor
    
    
 3. Modify classpathentry kind="output" path to "root/WEB-INF/classes"
    
    [/support/docview.wss?uid=swg21307399&aid=1] [/support/docview.wss?uid=swg21307399&aid=1]
    
    
 4. Save .classpath
    
    
 5. Open .settings\org.eclipse.wst.common.component with XML editor
    
    
 6. Modify wb-resource deploy-path's source-path and java-out-path's value
    
    [/support/docview.wss?uid=swg21307399&aid=2] [/support/docview.wss?uid=swg21307399&aid=2]
    
    
 7. Save org.eclipse.wst.common.component file
    
    
 8. Project Explorer > Refactor WebContent to root
    
    
 9. Project > clean > select "Start a build immediately" > OK
