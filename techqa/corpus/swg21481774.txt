Title: IBM S1000 [DataDirect][Oracle Wire Protocol driver] ServerName cannot be used with HostName - United States

Text:
 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 I have installed Clementine 12 on a Solaris 10 server with an Oracle 9.2.0.8 database on a different server (myserver1, myserver2, myserver3).
I am getting the following error messages when selecting my datasource and attempting to connect the Clementine Server to the Oracle databases.

Failed to connect to data source me@MyData Oracle Wire Protocol
S1000 [DataDirect][Oracle Wire Protocol driver] ServerName cannot be used with HostName, PortNumber, SID, or ServiceName.
Failed to connect to datasource: MyData Oracle Wire Protocol; UID=me;PWD=***** 

RESOLVING THE PROBLEM
Below is a sample of the Oracle Data Source being used on this Solaris machine. In this particular case the message is stating the ServerName can not be used with HostName.

First, remove the shipped information within the < > symbols including the symbols and value that are not being used for example ServerName, TNSNamesFile and others as well. 

If using a definied TNSNamesFile for the Oracle database, you want to specify the TNSNamesFile and the ServerName which is definied within the TNSNamesFile.

If a TNSNameFile is not being used by Oracle database, then you must specify a HostName, PortNumber, SID, ServiceName. The HostName may need to be domain qualified depending on your network setup
