APAR STATUS
 * CLOSED AS PROGRAM ERROR.
    
   
   

ERROR DESCRIPTION
 *  After applied Fixpack 2 on Imapct 6.1.0.1, rule created by Event
   Isolation and Correlation have problem. When put value '(.*)' in
   Regular Expression to extract field, the value is strored as
   '&;amp;#40;*&;amp;#41' in DB2 and shown as
   '&;amp;#40;*&;amp;#41' in
   Impact GUI.
   
   
    
   
   

LOCAL FIX
 *  Update unescapeHTMLCharacters funciont in policy
   NCI_EIC_utils.ipl,
   from
   encodedStr
   ={"&#39;","&amp","\\\\&quot;","&quot;","&lt;","&gt;"};
   replacementStr = {"'","&",'"','"',"<",">"};
   
   to
   
   encodedStr =
   {"&#39;","&amp","\\\\&quot;","&quot;","&lt;","&gt;","&#40;","&#4
   1;"};
   replacementStr = {"'","&",'"','"',"<",">","(",")"};
   
   
    
   
   

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All Impact Users                                             *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * When creating an event rule in Event Isolation and           *
   * Corelation, if the following regex metacharacters, (.*),     *
   * are added to the                                             *
   * Regular Expression in the variable assignments section, the  *
   * regex is                                                     *
   * shown as '&;amp;#40;*&;amp;#41' in the Impact GUI, this      *
   * causes correlation to fail.                                  *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * NONE                                                         *
   ****************************************************************
   the regex metacharacters, left and right parentheses are not
   being encoded correctly in EIC.
   
   
    
   
   

PROBLEM CONCLUSION
 *  Encoding was added to cater for left and right parenthesis to be
   used in users's regex.

   maintenance packages:

   |Fix Pack|7.1.0-TIV-NCI-FP0003|
