Title: IBM DFHDB8222 Connection from CICS TS 4.2 to IMS fails. DBCTL return code is 36 - United States

Text:
DFHDB8222 ; return code 36 ; RC36 DFHDB8111 DFHDB8102 ; MSGDFHDB8222 MSGDFHDB8111 MSGDFHDB8102 ; CICS TS 4.2 ; 5655S9700 R670 670 R700 700 HCI6700 5655-S97 5655S97 KIXINFO kixdcfcic670 retention42 TECHNOTE (TROUBLESHOOTING)

PROBLEM(ABSTRACT)
 Your CICS region fails to connect to IMS and you receive message DFHDB8222 Connection has failed. DBCTL return code is 36 (RC36). You also receive message DFHDB8111 with return code 36 and DFHDB8102. This occurs during CICS startup or when you enter transaction CDBC to connect to IMS from CICS Transaction Server for z/OS (CICS TS) V4.2. 

SYMPTOM
The following message sequence appears in the CICS log:
F CICS1,CDBC CON SUF(00)
+DFHDB8222 Connection has failed. DBCTL return code is 36.
+DFHDB8111 E Connection has failed. DBCTL return code 36.
+DFHDB8102 I Disconnection from DBCTL is now complete.

CAUSE
IMS is not recognizing the new parameters being passed by CICS in the participant adapter parameter list (PAPL).

RESOLVING THE PROBLEM
Apply the required PTF(s) applicable to your IMS release: 

 * IMS V12 New function APARs:  * PM31420 [http://www.ibm.com/support/docview.wss?uid=isg1PM31420] (PE PTF UK70991) 
    * PM45414 [http://www.ibm.com/support/docview.wss?uid=isg1PM45414] (PTF UK71251) 
    *  [http://www.ibm.com/support/docview.wss?uid=isg1PM47327] [http://www.ibm.com/support/docview.wss?uid=isg1PM47327]PM47327 [http://www.ibm.com/support/docview.wss?uid=isg1PM47327] (PTF UK73103) - fixing APAR for PM31420
   
   
   
 * IMS V11 toleration only - PM31729 [http://www.ibm.com/support/docview.wss?uid=swg1PM31729] (PTF UK67278)
   
   
 * IMS V10 toleration only - PM31730 [http://www.ibm.com/support/docview.wss?uid=swg1PM31730] (PTF UK67279)
