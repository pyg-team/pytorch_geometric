CAUSE
security.xml is corrupted. 


ENVIRONMENT
ND (Network Deployment) cell topology or Possible with Single Server profile



RESOLVING THE PROBLEM
Find the last known good back of security.xml and try to restore the file. If you don't have good backup then this might required to rebuild the environment from scratch. Other possibility would be to compare with other environment and try fixing the missing tag, but not guaranteed that this will fix the issue. So overall if you have good backup great, if not then rebuild the environment. 

For zOS specific client: 

Solution for z/OS WebSphere 8.0 Deployment Manager start-up failure. 

BBOO0222I: ADMN0015I: The administration service is initialized.

BBOO0035W TERMINATING THE CURRENT PROCESS, REASONC9C21C4F. and IEF472I W8DN1AG W8DN1AG - COMPLETION CODE - SYSTEMDC3 USER0000 REASON000C0008 

Probable cause: The operator issued CANCEL commands against the DMGR and DEMN address spaces while P DEMN was in process for a normal shutdown. 

Doing so apparently left an empty (0 byte) security.xml file, but a good one in the backup directory.
