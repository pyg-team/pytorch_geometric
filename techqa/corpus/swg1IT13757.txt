Title: IBM IT13757: THE SETTING FOR BACKUPIDPREFIX IN THE FCM PROFILE FILE IS IGNORED. - United States

Text:
 SUBSCRIBE
You can track all active APARs for this component.



APAR STATUS
 * CLOSED AS DOCUMENTATION ERROR.
    
   
   

ERROR DESCRIPTION
 *  The setting for option BACKUPIDPREFIX in the CLIENT section of
   the profile file (the default name is 'profile') is ignored by
   IBM Tivoli Storage FlashCopy Manager for Unix and Linux.
   Instead default prefixes are used.
   ORACL__ for plain Oracle environments
   <SID>___ for Oracle in SAP environments
   DB2___ for DB2 environments
   GEN___ for custom applications.
   
   This parameter should specify a string that is added before the
   backup ID that is generated by FlashCopy Manager and should be
   used to separate the backups within the same repository.
   
   
   Data Protection Versions Affected:
   All supported versions of Tivoli Storage FlashCopy Manager for
   UNIX and Linux.
   
   
   Initial Impact: LOW
   
   
   Additional Keywords:
   TSM FCM profile client BACKUPIDPREFIX option
   
   
    
   
   

LOCAL FIX

PROBLEM SUMMARY
 *  ****************************************************************
   * USERS AFFECTED:                                              *
   * All users                                                    *
   ****************************************************************
   * PROBLEM DESCRIPTION:                                         *
   * Parameter information in the docs, but this is ignored by    *
   * the product and cannot be set by the user: BACKUPIDPREFIX    *
   ****************************************************************
   * RECOMMENDATION:                                              *
   * Remove the information form the Client sections              *
   ****************************************************************
   
   
    
   
   

PROBLEM CONCLUSION
 *  The parameter is listed so Users would assume that they can set
   this parameter. that is not the case. Remove it from the docs
   and update the tecnote for Doc updates 4.1
