ERROR DESCRIPTION
 *  ABSTRACT: DMU SPA: WRONG RESULTS FOR XML -
   IMPORT IN TO ENOVIA DMU ON WINDOWS.
   SCENARIO:
   1. Start CATIA V5 with cache activated.
   2. Set the option Tools > Options > Digital MockUp
   > DMU Space Analysis > DMU Clash > During
   initital computation to "None"
   3. Open the attached product TEST1.CATProduct
   4. Double click on the Interference Results.1.
   5. Click on the Export As icon and select a storage
   folder and a name and save the XML file.
   6. Run the macro  and select the xml in the
   CATTemp folder in the folder with the name
   selected in the step 5
   (tempFile.xml). Click OK.
   7. Click OK in the warning message.
   8. Double click on the Interference Results.1 newly
   created. Result: value= -8
   9. Close the product.
   10. Set the option Tools > Options > Digital
   MockUp > DMU Space Analysis> DMU Clash >
   During initital computation to "Compute penetration
   depth or minimum distance"
   11. Activate the option Tools > Options >
   Infrastructure > Product Infrastructure> Product
   Visualization > "Do not activate default shapes on
   open".
   12. Open the attached product TEST1.CATProduct
   13. Double click on the Interference Results.1.
   14. Click on the Export As icon and select a storage
   folder and a name and save the XML file.
   15. Run the macro  and select the xml in the
   CATTemp folder in the folder with the name
   selected in the step 5 (tempFile.xml). Click OK.
   16. Click OK in the warning message.
   17. Double click on the Interference Results.1
   newly created. Result: value= -8
   RESULT:
   The value obtained after importing the interfernece
   results using the macro is wrong .
   .

PROBLEM CONCLUSION
 *  THIS PROBLEM WILL BE FIXED ON ENOVIA
   VERSION 5 RELEASE 20GA LEVEL.
   AND ON VERSION 5 RELEASE 19SP02
   Incident Diagnosis
   After the import of an existing XML clash result, the
   value of the penetration depth is incorrect in
   German language settings with comma as the
   decimal character.
   Correction Explanation
   the decimal point in the code is replaced by the
   local character
